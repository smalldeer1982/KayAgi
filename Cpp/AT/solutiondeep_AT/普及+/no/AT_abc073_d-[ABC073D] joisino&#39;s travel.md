# 题目信息

# [ABC073D] joisino&#39;s travel

## 题目描述

[problemUrl]: https://atcoder.jp/contests/abc073/tasks/abc073_d

Atcoder国には $ N $ 個の町があり、$ M $ 本の双方向に移動可能な道で結ばれています。

また、 $ i $ 本目の道は町 $ A_i $ と町 $ B_i $ の間を距離 $ C_i $ で結んでいます。

joisinoお姉ちゃんは、この国の $ R $ 個の町 $ r_1,r_2..r_R $ を訪れることとなりました。

最初に訪れる町までの移動と、最後に訪れる町からの移動は空路で行うが、それ以外は道を使わなければなりません。

町を訪れる順番を、道での移動距離が最小となるように決めた時の移動距離を答えてください。

## 说明/提示

### 制約

- $ 2≦N≦200 $
- $ 1≦M≦N×(N-1)/2 $
- $ 2≦R≦min(8,N) $ ( $ min(8,N) $ は $ 8 $ と $ N $ のうち小さい方)
- $ r_i≠r_j\ (i≠j) $
- $ 1≦A_i,B_i≦N\ ,\ A_i≠B_i $
- $ (A_i,B_i)≠(A_j,B_j),(A_i,B_i)≠(B_j,A_j)\ (i≠j) $
- $ 1≦C_i≦100000 $
- すべての町の間は道のみで移動することができる。
- 入力は全て整数である。

### Sample Explanation 1

例えば、町 $ 1 $ ,町 $ 2 $ ,町 $ 3 $ の順番で訪れると、移動距離が $ 2 $ となり、最小となります。

### Sample Explanation 2

町 $ 1 $ を訪れたあとに町 $ 3 $ を訪れても、町 $ 3 $ を訪れたあとに町 $ 1 $ を訪れても、町 $ 1 $ と町 $ 3 $ の間の最短距離は $ 4 $ であるため、どの順番を選んだとしても答えは $ 4 $ となります。

# AI分析结果

### 题目内容重写

Atcoder国有 $ N $ 个城镇，由 $ M $ 条双向道路连接。每条道路连接城镇 $ A_i $ 和 $ B_i $，距离为 $ C_i $。joisino需要访问 $ R $ 个城镇 $ r_1, r_2, ..., r_R $。她可以选择访问顺序，使得通过道路移动的总距离最小。最初和最后的移动通过空路进行，其他移动必须通过道路。求最小的移动距离。

### 算法分类
最短路、枚举

### 题解分析与结论

#### 题解1：fly20040720
- **星级**：4星
- **关键亮点**：
  - 使用Floyd-Warshall算法计算所有城镇之间的最短路径。
  - 通过全排列枚举所有可能的访问顺序，计算每种顺序的总距离，取最小值。
  - 代码简洁，逻辑清晰，适合小规模数据（$ R \leq 8 $）。
- **个人心得**：无

#### 核心代码
```cpp
int n,m,R,dist[300][300],r[10];

int main()
{
    memset(dist,0x3f,sizeof dist);
    int Min,ans;
    cin>>n>>m>>R;
    for(int i=0;i<R;i++)
        cin>>r[i];
    for(int i=0;i<m;i++)
    {
        int x,y,z;
        cin>>x>>y>>z;
        dist[x][y]=dist[y][x]=z;
    }
    for(int k=1;k<=n;k++)
        for(int i=1;i<=n;i++)
            for(int j=1;j<=n;j++)
                if(dist[i][k]+dist[k][j]<dist[i][j])
                    dist[i][j]=dist[i][k]+dist[k][j];
    sort(r,r+R);
    ans=0x3f3f3f3f;
    do
    {
        Min=0;
        for(int i=0;i<R-1;i++)
            Min+=dist[r[i]][r[i+1]];
        ans=min(Min,ans);
    }while(next_permutation(r,r+R));
    cout<<ans;
}
```

### 最优关键思路
1. **Floyd-Warshall算法**：用于计算所有城镇之间的最短路径，适用于小规模数据。
2. **全排列枚举**：由于 $ R \leq 8 $，全排列的复杂度是可接受的，通过枚举所有可能的访问顺序，找到最小总距离。

### 拓展思路
- **更大规模数据**：如果 $ R $ 较大，可以考虑使用动态规划或状态压缩来优化枚举过程。
- **其他最短路算法**：对于稀疏图，Dijkstra算法可能更高效。

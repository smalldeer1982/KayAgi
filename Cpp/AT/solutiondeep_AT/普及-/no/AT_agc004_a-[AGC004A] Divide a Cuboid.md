# 题目信息

# [AGC004A] Divide a Cuboid

## 题目描述

用若干个 $ 1 \times 1 \times 1 $ 的小方块（立方体）组成一个 $ A \times B \times C $ 的大长方体，每个小方块可以涂成红色或蓝色，但要满足以下要求：

① 至少有 $1$ 个红色方块与 $1$ 个蓝色方块。

② 所有的红色方块必须组成一个长方体。

③ 所有的蓝色方块必须组成一个长方体。

求所有满足要求的染色方案中，红色方块数与蓝色方块数相差个数的最小值。

## 说明/提示

$ 2 ≤ A, B, C ≤ 10^9 $

### 题解分析与结论
这道题的核心是将一个长方体切分成两个长方体，使得它们的体积差最小。题解们普遍采用了以下思路：
1. **分类讨论**：如果长方体的某一条边是偶数，那么可以直接从中间切分，使得两部分体积相等，差为0。
2. **奇数边处理**：如果所有边都是奇数，那么无法完全平分，最小的体积差为最小的两个边长的乘积。

所有题解都注意到了数据范围较大，需要使用`long long`类型来避免溢出。

### 所选高星题解
#### 题解1：CZQ_King (4星)
**关键亮点**：
- 思路清晰，直接分类讨论，代码简洁。
- 通过排序找到最小的两条边，减少了计算量。

**代码核心实现**：
```cpp
#include<bits/stdc++.h>
using namespace std;
long long a[3];//一定要开long long
int main(){
    cin>>a[0]>>a[1]>>a[2];
    sort(a,a+3);//排序获得最小的两边长
    cout<<((a[0]&1&a[1]&1&a[2]&1)?a[0]*a[1]:0)<<endl;//分类讨论
}
```

#### 题解2：Vocanda (4星)
**关键亮点**：
- 代码简洁，直接判断是否有偶数边，逻辑清晰。
- 使用`min`函数简化了最小面积的计算。

**代码核心实现**：
```cpp
#include<bits/stdc++.h>
using namespace std;
#define int long long
signed main(){
	int a,b,c;scanf("%lld%lld%lld",&a,&b,&c);
	if(!(a & 1) || !(b & 1) || !(c & 1))return puts("0"),0;
	else{
		printf("%lld\n",min(min(a*b,b*c),a*c));
	}
	return 0;
}
```

#### 题解3：Victorique (4星)
**关键亮点**：
- 代码简洁，直接判断是否有偶数边，逻辑清晰。

**代码核心实现**：

```cpp
#include <iostream>
#include <algorithm>
using namespace std;
typedef long long ll;
int main() {
    ll a,b,c;
    cin>>a>>b>>c;
    if(a%2==0||b%2==0||c%2==0) {
        cout<<0;
        return 0;
    }
    cout<<min(min(a*b,a*c),b*c);
    return 0;
}
```

### 最优关键思路
1. **分类讨论**：根据边长是否为偶数进行不同处理。
2. **最小面积计算**：在所有边都是奇数的情况下，计算最小的两个边长的乘积作为最小体积差。

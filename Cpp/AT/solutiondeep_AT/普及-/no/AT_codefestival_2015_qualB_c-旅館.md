# 旅館

## 题目描述

[problemUrl]: https://atcoder.jp/contests/code-festival-2015-qualb/tasks/codefestival_2015_qualB_c

高橋君は部屋が $ N $ 室ある旅館を経営しています。今日は $ M $ 組の予約が入っていますが、全ての予約を適切に部屋に割り振ることができるかを確認していませんでした。各予約について、その組の人数以上の定員の部屋を割り振る必要があります。各予約について必ず $ 1 $ つの部屋が割り当てられる必要があり、$ 1 $ つの予約について複数の部屋を割り当てたり、複数の予約を $ 1 $ つの部屋に割り当てたりすることはできません。全ての予約に部屋を割り振ることができるならば `YES` を、割り振ることができないならば `NO` を出力してください。

- 追加の制約のないデータセットに正解した場合は、上記とは別に $ 40 $ 点が与えられる。

### 题目内容重写

#### 题目描述

高橋君经营着一家有 $N$ 间房间的旅馆。今天有 $M$ 组预订，但他还没有确认是否能够将所有预订合理地分配到房间中。对于每个预订，必须分配一个房间，且该房间的容量必须大于或等于该组的人数。每个预订必须分配一个房间，且不能将一个预订分配到多个房间，也不能将多个预订分配到一个房间。如果能够将所有预订分配到房间，则输出 `YES`，否则输出 `NO`。

### 题解分析与结论

#### 题解对比

1. **ailo 的题解**
   - **思路**：将房间和预订按容量从大到小排序，然后依次尝试为每个预订分配房间。
   - **难点**：通过排序和指针优化，避免了双层循环，减少了时间复杂度。
   - **评分**：4星
   - **亮点**：排序和指针优化，代码简洁，思路清晰。

2. **shenmadongdong 的题解**
   - **思路**：统计房间和预订的容量出现次数，通过后缀和比较来判断是否能够分配。
   - **难点**：利用后缀和进行快速比较，避免了排序。
   - **评分**：5星
   - **亮点**：无需排序，时间复杂度更低，代码简洁高效。

3. **二分之一 的题解**
   - **思路**：将房间和预订按容量从小到大排序，然后从大到小尝试分配。
   - **难点**：通过排序和标记数组来判断是否能够分配。
   - **评分**：3星
   - **亮点**：排序和标记数组的使用，但代码复杂度较高。

#### 最优关键思路

- **shenmadongdong 的题解**：通过统计容量出现次数和后缀和比较，避免了排序，时间复杂度更低，代码简洁高效。

#### 核心代码片段

```cpp
#include<bits/stdc++.h>
using namespace std;
int n,m,a[100010],x;
int main()
{
    cin>>n>>m;
    for(int i=1;i<=n;i++) cin>>x,a[x]++;
    for(int i=1;i<=m;i++) cin>>x,a[x]--;
    for(int i=100000;i>=0;i--) if ((a[i]+=a[i+1])<0) return puts("NO"),0;
    puts("YES");
    return 0;
}
```

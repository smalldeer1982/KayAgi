### 约束条件

有 $ N $ 道题目的比赛，第 $ i $ 道题目的分数是 $ p_i $ 分。参赛者可以选择解决其中的一些题目，得分为所解决题目的分数之和。问这场比赛可能有多少种不同的得分。

- $ 1 \leq N \leq 100 $
- $ 1 \leq p_i \leq 100 $

动态规划

### 题解分析与结论
#### 题解1：RioBlu (4星)
**关键亮点**：
- 使用动态规划的思路，通过布尔数组记录可能得到的分数。
- 代码简洁，逻辑清晰，易于理解。

**核心实现思想**：
- 初始化一个布尔数组 `c`，`c[0]` 设置为 `true`，表示0分是可以得到的。
- 对于每个分数 `b`，从后向前更新数组 `c`，如果 `c[v]` 为 `true`，则 `c[v+b]` 也为 `true`。
- 最后统计数组中 `true` 的数量即为可能得分的总数。

**核心代码**：
```cpp
#include<bits/stdc++.h>
using namespace std;
long long a,b,ans;
bool c[10005];
int main()
{
    cin>>a;
    c[0]=1;
    for(int s=0;s<a;s++)
    {
        cin>>b;
        for(int v=10002;v>=0;v--)
        {
            if(c[v]==1)
            {
                c[v+b]=1;
            }
        }
    }
    for(int s=0;s<10002;s++)
    {
        if(c[s]==1)ans++;
    }
    cout<<ans<<endl;
}
```

### 最优关键思路或技巧
- **动态规划**：通过布尔数组记录可能得到的分数，逐步更新数组，最终统计可能得分的总数。
- **逆向更新**：从后向前更新数组，避免重复计算。

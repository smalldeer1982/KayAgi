# 题目信息

# [ARC001B] リモコン

## 题目描述

高桥君要调整空调的设定温度。现在的设定温度是A度，而他想调到B度。
空调遥控器按一次可以:
- 上调或下调1度
- 上调或下调5度
- 上调或下调10度
高桥君想求出从A调到B度的最小操作数。

## 说明/提示

样例1: 
输入：
```
7 34
```
输出：
```
5
```
依次上调10、10、5、1、1度即可

样例2: 
输入：
```
19 28
```
输出：
```
2
```
上调10度、下调1度即可。


样例3: 
输入：
```
10 10
```
输出：
```
0
```
温度一样时无需调整。

### 综合分析与结论
这些题解提供了多种解决“空调温度调整最小操作数”问题的思路，涵盖了搜索、贪心、最短路等算法。

#### 思路对比
- **搜索类**：包括深度优先搜索（DFS）、广度优先搜索（BFS）、迭代加深搜索和双向BFS。通过不断尝试各种温度调整操作，找到达到目标温度的最小操作数。
- **贪心类**：根据温差大小，优先选择调整幅度大的操作，如优先调10度，再处理剩余温差。
- **最短路类**：将温度看作节点，温度调整操作看作边，构建图后使用Dijkstra、SPFA等算法求最短路径。

#### 算法要点
- **DFS**：通过递归不断尝试所有可能的调整方式，使用剪枝优化搜索过程。
- **BFS**：利用队列按层次扩展，第一次到达目标温度时的步数即为最小操作数。
- **贪心**：总结温差与最小操作数的规律，优先使用大调整幅度。
- **最短路**：建图后使用相应算法求最短路径。

#### 解决难点
- **搜索类**：需要处理边界条件和剪枝，避免无效搜索。
- **贪心类**：要准确总结出贪心策略，避免局部最优而非全局最优。
- **最短路类**：建图时要注意边界判断，避免越界。

# [ABC101C] Minimization

## 题目描述

[problemUrl]: https://atcoder.jp/contests/abc101/tasks/arc099_a

長さ $ N $ の数列 $ A_1,\ A_2,\ ...,\ A_N $ があります．最初，この数列の要素は $ 1,\ 2,\ ...,\ N $ を並び替えたものになっています．

スヌケ君は，この数列に対して次の操作を行うことができます．

- 数列のうち，連続した $ K $ 個の要素を選ぶ．その後，選んだ要素それぞれの値を，選んだ要素の中の最小値で置き換える．

スヌケ君は，上の操作を何回か繰り返すことで，この数列の要素をすべて等しくしたいです． 必要な操作の回数の最小値を求めてください． この問題の制約の下，このようなことは必ず可能であることが証明できます．

## 说明/提示

### 制約

- $ 2\ \leq\ K\ \leq\ N\ \leq\ 100000 $
- $ A_1,\ A_2,\ ...,\ A_N $ は $ 1,\ 2,\ ...,\ N $ の並び替え

### Sample Explanation 1

例えば，次のようにするとよいです： - $ 1 $ 回目の操作では，$ 1,\ 2,\ 3 $ 番目の要素を選ぶ．すると，数列 $ A $ は $ 1,\ 1,\ 1,\ 4 $ になる． - $ 2 $ 回目の操作では，$ 2,\ 3,\ 4 $ 番目の要素を選ぶ．すると，数列 $ A $ は $ 1,\ 1,\ 1,\ 1 $ になる．

### 输出

## 题目描述

给定长度为 $N$ 的数列 $A_1,\ A_2,\ ...,\ A_N$。初始时，这个数列的元素是 $1,\ 2,\ ...,\ N$ 的一个排列。

你可以进行如下操作：
- 选择数列中连续的 $K$ 个元素，并将这些元素全部替换为其中的最小值。

目标是通过若干次操作，使得数列所有元素相等。求最少的操作次数。可以证明在题目约束下总能实现。

### 题解分析与结论

所有题解均基于以下观察：每次操作最多覆盖 $k-1$ 个新元素，最终答案公式为 $\lceil (n-1)/(k-1) \rceil$。关键思路如下：

1. **最小值的唯一性**：初始排列必然包含 $1$，最终所有元素都会被推平为 $1$。
2. **贪心扩展策略**：每次操作应尽可能覆盖最多的新元素，每次可扩展 $k-1$ 个新元素。
3. **数学公式推导**：总次数 = $\lceil (n-1)/(k-1) \rceil$，其中 $n-1$ 表示需要覆盖的元素总数。

### 高星题解推荐

#### 1. 作者：lkjzyd20（4星）
**亮点**：简洁直接，公式推导清晰，代码实现直观。
```cpp
#include <stdio.h>
#include <cmath>
int main() {
    int n, k;
    scanf("%d %d", &n, &k);
    printf("%d", (int)ceil((n - 1) * 1.0 / (k - 1)));
}
```

#### 2. 作者：hyc1026（4星）
**亮点**：整数运算优化，避免浮点误差，公式等价变形。

```cpp
cout << (n + k - 3) / (k - 1);
```

#### 3. 作者：零殇（4星）

```cpp
cout << ceil((double)(n - 1) / (double)(k - 1));
```

### 关键思路总结
- **公式推导**：$\lceil (n-1)/(k-1) \rceil$ 是核心结论。
- **实现技巧**：使用整数运算 `(n + k - 3) / (k - 1)` 避免浮点计算。
- **贪心思想**：每次最大化新覆盖区域，无需关注具体排列。

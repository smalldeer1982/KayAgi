# fLIP

## 题目描述

[problemUrl]: https://atcoder.jp/contests/code-festival-2017-quala/tasks/code_festival_2017_quala_b

$ N $ 行 $ M $ 列のマス目があり、最初は全てのマスが白いです。

各行各列には $ 1 $ つずつボタンがあります。 ある行のボタンを押すと、その行のマスの色が全て反転します。すなわち、白なら黒、黒なら白に色が変わります。 また、ある列のボタンを押すと、その列のマスの色が全て反転します。

高橋君は、ボタンを押す操作を好きな回数行うことができます。黒く塗られたマスの個数をちょうど $ K $ 個にすることができるかどうか判定してください。

## 说明/提示

### 制約

- $ 1\ \leq\ N,M\ \leq\ 1000 $
- $ 0\ \leq\ K\ \leq\ NM $

### Sample Explanation 1

$ 1 $ 行目、 $ 1 $ 列目の順にボタンを押せばよいです。

### Sample Explanation 3

$ 1 $ 列目、$ 3 $ 列目、$ 2 $ 行目、$ 5 $ 列目の順にボタンを押せばよいです。

# fLIP

## 题目描述

$ N $ 行 $ M $ 列的网格初始全为白色。每个行和列都有一个按钮，按下会反转该行/列所有颜色（白↔黑）。判断能否通过若干次操作使黑色格子数恰好为 $ K $。

## 标签
枚举、数学

## 题解分析
所有题解的核心公式均为：当翻转i行j列时，黑色格子数为 $ i×M + j×N - 2i×j $。该公式通过枚举i（0≤i≤N）和j（0≤j≤M）进行验证，时间复杂度O(NM)在题目约束下可行。

### 高星题解推荐
1. **lgvc（5星）**  
   直接枚举所有可能的i和j，代码简洁清晰，公式推导准确。  
   ```cpp
   for(int i=0;i<=N;i++) 
       for(int j=0;j<=M;j++) 
           if(i*M +j*N -2*i*j ==K) 
               return "Yes";
   ```

2. **李思远（4星）**  
   推导过程详细，代码中通过修改循环变量快速跳出多重循环。  
   ```cpp
   if(x*m + y*n -2*x*y ==k) {
       x=m+2; y=n+2; // 快速退出循环技巧
       ans=1;
   }
   ```

3. **封禁用户（4星）**  
   代码结构最简，突出核心判断逻辑，适合快速理解。  
   ```cpp
   for(int i=0;i<=n;++i)
       for(int j=0;j<=m;++j)
           if(i*m +j*n -2*i*j ==k) return 0;
   ```

### 关键思路
通过行列操作的独立性，将问题转化为数学公式计算，枚举行、列操作次数验证解的存在性。核心公式推导要点：
- 翻转i行产生i×M个黑格
- 翻转j列时，在未翻转行产生j×(N-i)个黑格，在已翻转行减少j×i个黑格
- 总公式：$ i×M + j×(N-i) - j×i = i×M + j×N - 2i×j $

### 拓展建议
类似问题可考虑操作独立性、奇偶性分析，或通过数学建模将操作转化为线性方程组求解（如异或方程组）。

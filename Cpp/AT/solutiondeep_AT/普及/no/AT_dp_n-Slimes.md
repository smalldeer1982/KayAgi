# Slimes

## 题目描述

[problemUrl]: https://atcoder.jp/contests/dp/tasks/dp_n

$ N $ 匹のスライムが横一列に並んでいます。 最初、左から $ i $ 番目のスライムの大きさは $ a_i $ です。

太郎君は、すべてのスライムを合体させて $ 1 $ 匹のスライムにしようとしています。 スライムが $ 1 $ 匹になるまで、太郎君は次の操作を繰り返し行います。

- 左右に隣り合う $ 2 $ 匹のスライムを選び、それらを合体させて新しい $ 1 $ 匹のスライムにする。 合体前の $ 2 $ 匹のスライムの大きさを $ x $ および $ y $ とすると、合体後のスライムの大きさは $ x\ +\ y $ となる。 このとき、太郎君は $ x\ +\ y $ のコストを支払う。 なお、合体の前後でスライムたちの位置関係は変わらない。

太郎君が支払うコストの総和の最小値を求めてください。

## 说明/提示

# Slimes

## 题目描述

$ N $ 只史莱姆排成一列。初始时左数第 $ i $ 只史莱姆的大小为 $ a_i $。

需要通过 $ N-1 $ 次合并操作将所有史莱姆合为一体。每次操作：
1. 选择相邻的两只史莱姆合并
2. 合并后的体积为两者之和
3. 付出等于合并后体积的代价

求最小总代价。

## 算法分类
动态规划（区间DP）

## 题解分析

### 核心思路对比
所有题解均采用区间DP解法：
1. 状态定义：$ dp[i][j] $ 表示合并区间 $ [i,j] $ 的最小代价
2. 转移方程：枚举分割点 $ k $，$ dp[i][j] = \min(dp[i][k] + dp[k+1][j] + sum(i,j)) $
3. 使用前缀和优化区间和计算

### 解决难点
1. **正确初始化**：单个元素代价为0（易错点）
2. **循环顺序**：需按区间长度递增处理子问题
3. **大数处理**：使用long long防止溢出

## 精选题解

### 题解1（作者：TheSky233）⭐⭐⭐⭐⭐
**亮点**：
- 清晰的倒序枚举i处理依赖关系
- 显式初始化极大值保证正确性

**核心代码**：
```cpp
for(int i=n;i>=1;i--){ // 逆序处理左端点
    for(int j=i+1;j<=n;j++){
        for(int k=i;k<j;k++){
            f[i][j]=min(f[i][j], f[i][k]+f[k+1][j]+pre[j]-pre[i-1]);
        }
    }
}
```

## 最优解法总结
**关键技巧**：
1. **倒序处理左端点**：确保大区间计算时小区间已就绪
2. **前缀和优化**：将区间和计算复杂度降为O(1)
3. **三层循环结构**：外层枚举区间长度，中层枚举起点，内层枚举分割点

**易错点提醒**：
- 初始化时单个元素代价必须为0
- 分割点k的枚举范围是[i, j-1]
- 最终答案存储在dp[1][n]中

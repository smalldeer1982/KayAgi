# [ARC130C] Digit Sum Minimization

## 题目描述

[problemUrl]: https://atcoder.jp/contests/arc130/tasks/arc130_c

正の整数 $ a,\ b $ が与えられます。ただし、$ a,\ b $ のどの桁も $ 0 $ ではありません。

$ a+b $ の各桁の和が最小になるように、$ a $, $ b $ のそれぞれの桁を並べ替えてください。

## 说明/提示

### 制約

- $ 1\leq\ a,\ b\ <\ 10^{100000} $
- $ a,\ b $ のどの桁も $ 0 $ ではない

### Sample Explanation 1

$ 532\ +\ 268\ =\ 800 $ で、その各桁の和は $ 8+0+0=8 $ となります。 他にも、$ (a,\ b)\ =\ (325,\ 682) $ を出力しても正解となります。

### Sample Explanation 2

$ 435+565=1000 $ で、その各桁の和は $ 1+0+0+0=1 $ となります。

### Sample Explanation 3

$ 312\ +\ 799788\ =\ 800100 $ で、その各桁の和は $ 8+0+0+1+0+0=9 $ となります。

## **综合分析与结论**  
### **核心思路**  
要使 `a + b` 的各位数字和最小，需最大化进位次数。每次进位会使总和减少 9（例如，15 → 1+5=6，总和减少 9）。因此，目标转化为：  
1. **初始进位**：找到一对数字 `i` 和 `j`（`i ∈ a`, `j ∈ b`），使得 `i + j ≥ 10`，作为个位以触发首次进位。  
2. **延续进位链**：剩余数字尽可能配对 `i + j ≥ 9`，使得加上前一位的进位后产生新的进位。  
3. **无法进位时**：剩余数字降序排列，避免额外进位。

### **解决难点**  
- **进位链的维护**：后续配对需满足 `i + j ≥ 9` 以保证进位传递。  
- **高效枚举**：通过统计数字频次，快速枚举所有可能的初始进位配对。

### **可视化设计思路**  
- **动画步骤**：  
  1. **初始进位配对**：高亮个位的 `i` 和 `j`，显示 `i + j ≥ 10`。  
  2. **延续进位链**：逐位显示剩余数字的配对，满足 `i + j ≥ 9`（绿色高亮），否则灰色标记。  
  3. **降序排列剩余**：无法进位时，降序排列剩余数字（蓝色箭头）。  
- **像素风格**：用 8-bit 风格显示数字配对，进位时播放“滴答”音效，成功时播放胜利音效。

## **代码核心实现**  
```cpp
// 核心函数：枚举初始进位配对并计算最大进位次数
pip calc(int a, int b) {
    // 复制当前数字频次
    for (int i=1; i<=9; i++) ts[i]=cs[i], tt[i]=ct[i];
    string S(1, a+'0'), T(1, b+'0');
    int res = 1; // 初始进位计数
    // 配对剩余数字（i+j >=9）
    for (int i=1; i<=9; i++) {
        for (int j=1; j<=9; j++) {
            if (i + j >= 9) {
                int c = min(ts[i], tt[j]);
                ts[i] -= c; tt[j] -= c;
                res += c; // 每配对一次增加一个进位
                S += string(c, i+'0');
                T += string(c, j+'0');
            }
        }
    }
    // 处理剩余数字（降序排列）
    for (int i=9; i>=1; i--) {
        S += string(ts[i], i+'0');
        T += string(tt[i], i+'0');
    }
    reverse(S.begin(), S.end());
    reverse(T.begin(), T.end());
    return {res, {S, T}};
}
```

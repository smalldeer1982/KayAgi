# 题目信息

# [ARC001B] リモコン

## 题目描述

高桥君要调整空调的设定温度。现在的设定温度是A度，而他想调到B度。
空调遥控器按一次可以:
- 上调或下调1度
- 上调或下调5度
- 上调或下调10度
高桥君想求出从A调到B度的最小操作数。

## 说明/提示

样例1: 
输入：
```
7 34
```
输出：
```
5
```
依次上调10、10、5、1、1度即可

样例2: 
输入：
```
19 28
```
输出：
```
2
```
上调10度、下调1度即可。


样例3: 
输入：
```
10 10
```
输出：
```
0
```
温度一样时无需调整。

## 题解思路与解决难点
### 核心思路
题目要求最小操作次数调整温度，关键在于贪心策略：优先使用最大的10度调整快速逼近目标，剩余小范围温差通过预计算的余数表快速求解。

### 算法要点
1. **贪心分解**：将总温差拆分为10度的倍数部分（快速调整）和余数部分（精细调整）。
2. **余数表优化**：预处理余数0~10对应的最小步数，例如余数7对应3步（10+10+5+1+1）。
3. **数学计算**：总次数 = 十度调整次数 + 余数对应步数，时间复杂度O(1)。

### 解决难点
如何证明贪心策略的最优性？通过分析余数部分的调整方式发现，每次优先调整10度能最快缩小规模，剩余部分的最优解可通过预计算表直接获取。

## 最优思路提炼
**余数查表法**：  
1. 计算绝对温差`k = |A-B|`。
2. 十度调整次数为`k/10`，余数`r = k%10`。
3. 查预计算数组`a[] = {0,1,2,3,2,1,2,3,3,2,1}`获取余数步数。
4. 总次数 = `k/10 + a[r]`。

### 核心代码
```cpp
#include<bits/stdc++.h>
using namespace std;
int main(){
    int a[]={0,1,2,3,2,1,2,3,3,2,1}, A, B;
    cin >> A >> B;
    int k = abs(B-A);
    cout << k/10 + a[k%10] << endl;
    return 0;
}
```

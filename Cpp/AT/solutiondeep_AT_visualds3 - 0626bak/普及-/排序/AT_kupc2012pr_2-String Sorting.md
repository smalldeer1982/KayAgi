# String Sorting

## 题目描述

设有 $n$ 个正整数（$n \le 100$）长度为 $m$（$m \le 30$），将它们联接成一排，组成一个最大的多位整数。

## 综合分析与结论
### 核心思路
所有正确题解均采用**字符串拼接比较法**：通过自定义排序规则，比较两个字符串 `a` 和 `b` 的两种拼接方式 `a+b` 和 `b+a`，选择字典序更大的组合方式。最终将排序后的字符串按序连接即为最大整数。

### 难点解析
1. **正确比较逻辑**：需理解字符串拼接后的字典序比较才是关键，而非直接比较数值或字符串本身大小。
2. **贪心策略证明**：需确保局部最优（每两个相邻字符串的拼接顺序）能推导出全局最优解，这需要通过数学归纳法证明。

### 算法流程可视化设计
1. **动画步骤**：
   - 展示排序过程中每次比较的两个字符串 `a[i]` 和 `a[j]`，高亮当前比较对。
   - 动态显示 `a[i]+a[j]` 和 `a[j]+a[i]` 的拼接结果，用颜色标注更大的组合。
   - 逐步构建排序后的数组，最终拼接成完整字符串。
2. **复古像素风格**：
   - 用 8-bit 像素字体显示字符串和拼接结果。
   - 关键操作（比较、交换）时播放经典 FC 音效（如《超级马里奥》金币声）。
   - Canvas 网格展示排序过程，当前操作元素以闪烁效果突出。

## 最优思路提炼
### 关键技巧
- **拼接比较法**：通过 `a+b > b+a` 的字典序比较，保证每一步选择都是当前最优。
- **字符串处理**：直接操作字符串避免数值溢出，且能处理前导零问题（但本题无需处理）。

### 代码实现
```cpp
bool cmp(string a, string b) {
    return a + b > b + a; // 核心逻辑：贪心比较
}
sort(arr, arr+n, cmp); // 排序后直接拼接输出
```

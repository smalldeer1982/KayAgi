## 题目描述

有一个长度为n的数列a，你需要把这个数列里的所有数从左到右拼接起来形成一个数，然后把这个数%$1,000,000,007(1e9+7)$并输出。

## 说明/提示

$1 ≤ N ≤ 100,000$。

别忘了结尾换行！

## 综合分析与结论  
### 核心思路  
将拼接后的超大数按位逐步计算模值，避免直接处理大数。关键公式：  
`ans = (ans * 10 + digit) % MOD`，逐位更新结果。

### 难点对比  
1. **字符串拼接法**（题解1、2、4）：直接拼接所有数字为字符串，逐个字符处理。  
   优势：代码简洁，时间复杂度 O(L)（L为总位数），空间复杂度 O(L)。  
   风险：拼接极大字符串可能内存不足（但题目数据范围内安全）。  

2. **逐数计算法**（题解5）：逐个处理每个数，计算其位数并动态更新。公式：  
   `ans = (ans * 10^d + x) % MOD`（d为当前数的位数）。  
   优势：无需拼接字符串，内存更优。  
   劣势：需计算每个数的位数和快速幂，时间常数略高。  

### 可视化设计

  2. 右侧动态展示当前结果 `ans` 的值和更新过程。  
  3. 高亮当前处理的字符或数字，标记乘10、加当前值、取模三步操作。

## 最优思路提炼  
### 关键技巧  
1. **动态取模**：每处理一位或一个数后立即取模，保证数值不溢出。  
2. **字符串拼接优化**：直接拼接所有数字为字符串，避免高精度运算。  
3. **快速幂预计算**：处理每个数时，快速计算 10^d % MOD。  

### 代码片段（da32s1da）  
```cpp
#include<bits/stdc++.h>
using namespace std;
const int mod = 1e9 + 7;
int main() {
    int n; scanf("%d", &n);
    string a; cin >> a;
    for (int i = 1; i < n; i++) {
        string b; cin >> b;
        a += b;
    }
    long long m = 0;
    for (char c : a)
        m = (m * 10 + c - '0') % mod;
    printf("%lld\n", m);
}
```

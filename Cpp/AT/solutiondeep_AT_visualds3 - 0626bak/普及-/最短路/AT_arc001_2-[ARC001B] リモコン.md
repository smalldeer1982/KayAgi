# 题目信息

# [ARC001B] リモコン

## 题目描述

高桥君要调整空调的设定温度。现在的设定温度是A度，而他想调到B度。
空调遥控器按一次可以:
- 上调或下调1度
- 上调或下调5度
- 上调或下调10度
高桥君想求出从A调到B度的最小操作数。

## 说明/提示

样例1: 
输入：
```
7 34
```
输出：
```
5
```
依次上调10、10、5、1、1度即可

样例2: 
输入：
```
19 28
```
输出：
```
2
```
上调10度、下调1度即可。


样例3: 
输入：
```
10 10
```
输出：
```
0
```
温度一样时无需调整。

## 题解思路与解决难点

### 核心思路
**贪心策略**：优先使用最大步长（10度）调整，快速缩小温差。剩余温度差根据预处理的步数表计算最优解。

**关键推导**：
1. 计算绝对温差 `k = |A-B|`
2. 尽可能用 `k/10` 次10度调整，剩余温差 `k%10`
3. 预处理余数 `0~10` 的最优步数表：`[0,1,2,3,2,1,2,3,3,2,1]`
4. 总步数 = 10度调整次数 + 余数对应步数

### 算法对比
| 方法     | 时间复杂度 | 空间复杂度 | 实现难度 | 优化点                     |
|----------|------------|------------|----------|----------------------------|
| 贪心     | O(1)       | O(1)       | 低       | 数学推导余数最优解         |
| BFS      | O(n)       | O(n)       | 中       | 队列存储状态，保证最短路径 |
| 最短路   | O(n^2)     | O(n)       | 高       | 将温度建模为图节点         |

## 最优技巧提炼
1. **余数预处理**：通过数学推导将余数处理转化为查表操作，避免复杂判断
2. **动态调整策略**：当剩余温差≥8时，使用 `10-（10-k%10）` 比直接处理更优
3. **极简公式**：总步数 = `k/10 + a[k%10]`，其中 `a` 为预处理的步数表

#include <iostream>

int main() {
    int A, B;

    int diff = abs(A - B);
    int steps[11] = {0,1,2,3,2,1,2,3,3,2,1};
    cout << diff/10 + steps[diff%10] << endl;

}

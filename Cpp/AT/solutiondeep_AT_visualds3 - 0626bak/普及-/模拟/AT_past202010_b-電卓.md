# 题目信息

# 電卓

## 题目描述

在你的计算器上输入非负整数 $ X，\ Y $，然后以 $ \frac{X}{Y} $ 为开头，没有多余的 $ 0 $，小数点后第 $3$ 及以下位的数全部舍弃，显示到小数点后第 $2$ 位。例如，当 $ X=4,Y=2$ 时，计算器上显示  `2.00`；当 $ X=2,Y=3$ 时，计算器上显示 ` 0.66`。但是，如果 $Y=0$ 的时候要显示 `ERROR`。

输入 $ X,Y $，请你编写出求计算器显示的程序。

## 说明/提示

- 输入的所有的数都是整数；
- $ 0\ \leq\ X,\ Y\ \leq\ 100 $

样例解释 $1$：

- $ \frac{X}{Y}= 33.33333 \ \cdots $，计算器上显示的是到小数点后第 $ 2 $ 位 `33.33`。

样例解释 $2$：

- 在 $ Y=0 $ 的情况下输出 `ERROR`。

## 样例 #1

### 输入

```
100 3```

### 输出

```
33.33```

## 样例 #2

### 输入

```
42 0```

### 输出

```
ERROR```

## 综合分析与结论  

### 核心思路与难点  
1. **关键逻辑**：通过整数运算逐位获取小数部分，避免浮点数精度误差和四舍五入问题。  
   - **整数部分**：`X / Y`  
   - **十分位**：`(X * 10 / Y) % 10`  
   - **百分位**：`(X * 100 / Y) % 10`  
2. **解决难点**：  
   - 浮点数输出会触发四舍五入（如 `printf("%.2lf", x/y)` 导致错误），必须通过整数运算截断。  
   - 处理 `Y=0` 的特殊情况直接输出 `ERROR`。  

### 可视化设计  
1. **核心步骤动画**：  
   - **步骤 1**：显示 `X/Y` 的整数部分，背景高亮为绿色。  
   - **步骤 2**：显示 `X*10/Y` 的个位（十分位），该数值在网格中动态放大并标记为黄色。  
   - **步骤 3**：显示 `X*100/Y` 的个位（百分位），标记为橙色。  
2. **复古像素风格**：  
   - **界面**：仿计算器 LCD 屏幕，8 位字体，深绿色背景。

## 最优思路与技巧提炼  

### 关键技巧  
1. **逐位整数截断法**：  
   - 通过 `X * 10^k / Y` 将目标小数位移至整数部分，再取模 10 得到对应位。  
   - 避免浮点运算，保证精度和截断逻辑正确性。  
2. **错误处理优先**：  
   - 先判断 `Y=0` 的特殊情况，减少无效计算。  

### 同类型题拓展  
- **相似场景**：需截断小数位而非四舍五入的题目（如货币计算、物理测量）。  
- **通用解法**：逐位计算法可推广至任意小数位数（如 `千分位 = (X*1000/Y)%10`）。

## 代码实现

#include <iostream>  
using namespace std;  

int main() {  
    int x, y;  
    cin >> x >> y;  
    if (y == 0) {  
        cout << "ERROR\n";  
        return 0;  
    }  
    cout << x / y << "." << (x * 10 / y) % 10 << (x * 100 / y) % 10 << "\n";  
    return 0;  
}  
```

## 3. 核心难点辨析与解题策略

1.  **输入顺序陷阱**
    * **分析**：85%题解强调必须先读字符再读字符串。若顺序颠倒，首个字符会被错误识别为坏键。解决方案：严格按`cin>>badKey>>inputStr`顺序
    * 💡 **学习笔记**：输入顺序决定程序行为，务必匹配题目描述

2.  **输出格式要求**
    * **分析**：所有AC代码均以`cout<<endl`结尾。测试数据验证：无换行=WA。解决方案：输出后显式添加换行符
    * 💡 **学习笔记**：OJ系统对输出格式零容忍

3.  **高效过滤实现**
    * **分析**：优质解法均采用O(n)遍历。避免`移位法`（题解10的复杂实现易出错）。关键技巧：即时过滤输出，不存储中间结果
    * 💡 **学习笔记**：边读边处理省时省空间

-   **技巧B (即时处理)**：避免不必要的中间存储，降低空间复杂度
-   **技巧C (跨语言移植)**：不同语言注意字符串终止符差异（C++的'\0'/Pascal的length前缀）

## 4. C++核心代码实现赏析

**本题通用核心C++实现参考**
* **说明**：综合三种优质解法优点的典范实现
* **完整核心代码**：
    ```cpp
    #include <iostream>
    #include <string>
    using namespace std;
    
    int main() {
        char badKey;
        string inputStr;
        cin >> badKey >> inputStr;
        
        for (int i = 0; i < inputStr.size(); ++i) {
            if (inputStr[i] != badKey) 
                cout << inputStr[i];
        }
        cout << endl;
        return 0;
    }
    ```
* **代码解读概要**：严格遵循输入顺序→遍历过滤→换行输出的黄金流程。使用`string`容器自动管理内存，`.size()`获取长度安全高效。

**题解一：(来源：_Qer)**
* **亮点**：动态字符流处理，零预分配内存
* **核心代码片段**：
    ```cpp
    char c, x;
    cin >> x;
    while (cin >> c) {
        if (c != x) cout << c;
    }
    cout << endl;
    ```
* **代码解读**：`while(cin>>c)`持续读入直到EOF，巧妙规避字符串长度限制。注意：仅适用于无空格输入（本题恰为连续字符串）。每次比较都是即时决策，内存占用恒定。
* 💡 **学习笔记**：流式处理适合未知长度的连续字符序列

**题解二：(来源：二字)**
* **亮点**：坑点防御性编程
* **核心代码片段**：
    ```cpp
    string s;
    char x;
    cin >> x >> s;
    for (int i = 0; i < s.length(); i++)
        if (s[i] != x) cout << s[i];
    cout << endl;
    ```
* **代码解读**：输入顺序`x>>s`是防坑关键。`.length()`返回无符号数，但`i<int`比较安全。显式循环体比单行if更易调试。
* 💡 **学习笔记**：防御式编程=提前规避已知陷阱

**题解三：(来源：封禁用户)**
* **亮点**：双实现对比教学
* **核心代码片段**：
    ```cpp
    char s[52], x;
    cin >> x >> s;
    for (int i = 0; i < 50; i++) {
        if (s[i] == '\0') break;
        if (s[i] != x) cout << s[i];
    }
    cout << endl;
    ```
* **代码解读**：数组版手动检测终止符`'\0'`，揭示C风格字符串本质。固定循环次数`i<50`因题目限定长度≤50，通用场景应优先`.size()`。
* 💡 **学习笔记**：理解底层实现能更好驾驭高层抽象

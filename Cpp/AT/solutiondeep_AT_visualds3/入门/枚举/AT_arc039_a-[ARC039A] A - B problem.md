# 题目信息

# [ARC039A] A - B problem

## 题目描述

[problemUrl]: https://atcoder.jp/contests/arc039/tasks/arc039_a

高橋君は次のような問題を考えました。

- $ 3 $ 桁の整数 $ A $、$ B $ が与えられるので $ A\ -\ B $ を求める。

しかしあまりにも簡単すぎるので、ちょっと変更して次のような問題にしました。

- $ 3 $ 桁の整数 $ A $、$ B $ が与えられる。
- $ A $ と $ B $ のどちらかを $ 1 $ 桁だけ書き換えてもよい時の、$ A\ -\ B $ の答えになり得る整数の最大値を求める。

なお、一番上の桁が $ 0 $ であるような整数へと書き換えることはできません。 例えば $ 123 $ を $ 023 $ へと書き換えたりすることはできません。

高橋君は自信満々であなたへとこの問題を出題してきました。 ぜひ挑戦してみてください。

## 说明/提示

### Sample Explanation 1

$ A $ を $ 567 $ から $ 967 $ に書き換えれば $ A-B $ は $ 733 $ となり、この値が $ A-B $ の最大値となっています。

### Sample Explanation 2

どちらも書き換えない時に $ A-B $ が最大値となることもあります。

### Sample Explanation 3

答えは負になることもあります。

## 样例 #1

### 输入

```
567 234```

### 输出

```
733```

## 样例 #2

### 输入

```
999 100```

### 输出

```
899```

## 样例 #3

### 输入

```
100 999```

### 输出

```
-99```

✨ **本题主要考察**：`枚举与贪心策略应用`

🗣️ **初步分析**：
> 这道题就像在玩一个数字谜题游戏：你有两次机会修改两个三位数（A或B的任意一位），目标是让A-B的结果尽可能大。想象你在玩一个复古像素游戏，每个数字位都是一个可旋转的齿轮⚙️，旋转到最优位置就能获得最高分！
> - **核心思路**：通过枚举所有可能的修改位置（A/B的百位/十位/个位）和修改值（0-9，百位不能为0），计算每种修改后的A-B值并取最大值。贪心策略则优先处理高位数字（百位>十位>个位），因为高位变化对结果影响最大。
> - **算法流程**：① 分解A/B的各位数字 ② 枚举6种修改位置 ③ 对每个位置枚举允许的值 ④ 计算修改后的A-B ⑤ 记录最大值。可视化时将用像素齿轮表示数字位，旋转动画展示修改过程，高亮当前枚举位置。

## 3. 核心难点辨析与解题策略

1.  **高位优先原则**  
    * **分析**：修改百位收益(±100)远超十位(±10)或个位(±1)。优质解法均优先处理高位，如sxtm12138的代码首先比较百位收益。  
    * 💡 **学习笔记**：像搭积木一样，先稳固地基（高位）再处理细节！  

2.  **枚举完整性保障**  
    * **分析**：必须覆盖6种修改位置（A/B×百/十/个位）和9-10种取值。LJC00111用6行`max`调用确保完整枚举。  
    * 💡 **学习笔记**：枚举像检查清单——漏掉一项可能错过最优解！  

3.  **百位零值约束处理**  
    * **分析**：百位不能为0是常见错误点。ComeOver·用`if(i==0) j=1~9`明确处理，Mr_Sympathetic用`if(j)`校验。  
    * 💡 **学习笔记**：边界条件如同游戏规则——违反直接GAME OVER！  

### ✨ 解题技巧总结  
- **技巧A：高位影响优先** - 优先修改高位数字（如A百位→9或B百位→1）  
- **技巧B：数学式位操作** - 用`a/100`取百位，`a%10`取个位，`a+9-a%10`将个位变9  
- **技巧C：极值初始化** - 如设`ans=-1e7`避免漏解

## 4. C++核心代码实现赏析

**通用核心实现参考**  
* **说明**：综合LJC00111枚举法，最简洁高效的代表性解法。
```cpp
#include <iostream>
using namespace std;

int main() {
    int A, B, ans = -1e7;
    cin >> A >> B;

    // 枚举A的修改
    ans = max(ans, (A/100*100 + 900) - B); // 改A百位→9
    ans = max(ans, (A/100*100 + A%10 + 90) - B); // 改A十位→9
    ans = max(ans, (A + 9 - A%10) - B); // 改A个位→9

    // 枚举B的修改
    ans = max(ans, A - (B/100*100 + 100)); // 改B百位→1
    ans = max(ans, A - (B/100*100 + B%10)); // 改B十位→0
    ans = max(ans, A - (B - B%10)); // 改B个位→0

    cout << ans;
    return 0;
}
```
* **代码解读概要**：通过6次`max`调用覆盖所有修改可能，数学表达式直接计算修改后的值，避免分支判断。

**题解一：LJC00111**  
* **亮点**：精炼的数学表达式实现高效枚举  
* **核心代码**：
```cpp
ans = max(ans, (x%100+900-y)); // A百位→9
ans = max(ans, x - (y - y%10)); // B个位→0
```
* **代码解读**：  
  > `x%100+900`：保留A的十位个位(`x%100`)，百位替换为9(`+900`)  
  > `y - y%10`：B的个位归零（如123→120）  
* 💡 **学习笔记**：巧用取余和整除实现位操作是竞赛常用技巧  

**题解二：sxtm12138**  
* **亮点**：贪心策略减少计算量  
* **核心代码**：
```cpp
if(a/100 - b/100 < 8) 
    cout << 100*max(9-a/100, b/100-1) + a-b;
```
* **代码解读**：  
  > `max(9-a/100, b/100-1)`：计算百位修改A(→9)或B(→1)的收益  
  > `100*收益 + a-b`：收益放大100倍后叠加原值  
* 💡 **学习笔记**：高位收益=最大变化值×位权（百位×100）  

**题解三：ComeOver·**  
* **亮点**：字符串操作直观易理解  
* **核心代码**：
```cpp
p[i] = j+'0'; // 修改字符串某位
y = max(y, strToInt(p) - strToInt(m));
```
* **代码解读**：  
  > `j+'0'`：将数字转为ASCII字符（如3→'3'）  
  > `strToInt`：自定义函数实现字符串转整数  
* 💡 **学习笔记**：字符串修改适合对位操作不熟悉的学习者

# [ARC015A] Celsius と Fahrenheit

## 题目描述

[problemUrl]: https://atcoder.jp/contests/arc015/tasks/arc015_1

摂氏での温度 $ n $ 度を華氏にで表すと $ (\ 9\ ÷\ 5\ ×\ n\ )\ +\ 32 $ 度となります。  
 摂氏での温度が与えられるので華氏での温度に変換するプログラムを書いてください。  
 入力は以下の形式で標準入力から与えられる。

> $ n $

- $ 1 $ 行目は、華氏に変換すべき、摂氏の温度 $ n\ (-273≦n≦1,000) $ が整数で与えられる。
 
 摂氏での温度 $ n $ を華氏に変換し、その値を $ 1 $ 行で出力せよ。  
 なお、出力の最後には改行をいれること。  
 出力は絶対誤差あるいは相対誤差の少なくとも片方が $ 10^{-6} $ 以下であれば許容される。 ```
<pre class="prettyprint linenums">
10
```

 ```
<pre class="prettyprint linenums">
50
```

- $ (9÷5×10)+32 $ $ = $ $ 50 $
 
```

 ```
<pre class="prettyprint linenums">
91.4
```

- $ (9÷5×33)+32 $ $ = $ $ 91.4 $
- 摂氏での温度が整数であっても、華氏での温度は整数になるとは限りません。

### 核心难点辨析与解题策略  
1. **难点：整数除法的精度陷阱**  
   * **分析**：若用`9/5`（整数除法）会得1而非1.8，必须用浮点数（如`9.0/5.0`）。优质题解均用`double`规避此问题。  
   * 💡 **学习笔记**：涉及除法的公式，优先用浮点类型避免截断误差。  

2. **难点：输出精度控制**  
   * **分析**：题目要求误差≤10⁻⁶，需保留6位小数。Bronya用`printf("%.6lf")`，CZQ依赖`cout`默认精度（需配合`fixed`+`setprecision(6)`更稳妥）。  
   * 💡 **学习笔记**：输出浮点数时，**显式指定精度**是可靠做法。  

3. **难点：运算符的字符编码**  
   * **分析**：题目公式含全角符号（`÷`/`×`），但代码需半角（`/`/`*`）。Bronya特别提醒此细节，避免编译错误。  
   * 💡 **学习笔记**：复制题目公式时，需人工转换符号编码。  

### ✨ 解题技巧总结  
- **公式简化优先**：如`9/5 → 1.8`减少运算量  
- **浮点精度双保险**：计算用`double` + 输出指定小数位  
- **边界测试**：重点验证负数（如-100°C→-148°F）和零值

### C++核心代码实现赏析  
**本题通用核心C++实现参考**  
* **说明**：综合优质题解优化，兼顾精度与简洁性  
* **完整核心代码**：  
  ```cpp
  #include <iostream>
  #include <iomanip>
  using namespace std;
  
  int main() {
      double n;
      cin >> n;
      cout << fixed << setprecision(6) << 1.8 * n + 32 << endl;
      return 0;
  }
  ```
* **代码解读概要**：  
  > 1. `double n` 确保计算精度  
  > 2. `fixed << setprecision(6)` 锁定6位小数输出  
  > 3. `1.8*n+32` 直接套用简化公式  

**题解二（CZQ_King）片段赏析**  
* **亮点**：公式优化至最简表达式  
* **核心代码片段**：  
  ```cpp
  cout << 1.8 * n + 32 << endl;
  ```
* **代码解读**：  
  > 此处用**编译期常量替换**：`9/5`在代码中直接计算为1.8，省去运行时除法开销。类比用预拌蛋糕粉（常量）代替现场调配面粉（运算），提升效率。  
* 💡 **学习笔记**：将固定算式预计算为常量，是优化数学表达式的常用技巧。

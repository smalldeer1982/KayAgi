# [ARC008A] たこ焼き買えるかな？

## 题目描述

你要买$N$ 个东西，1个15元，10个100元，求最少花费。
你可以购买$N$ 个以上的物品。

#### 题目解读与核心算法识别
✨ **本题主要考察**：`贪心算法`  
🗣️ **初步分析**：  
> 解决本题的关键在于**贪心策略**——就像在超市买饮料时选择整箱还是单瓶。当单买7瓶以上的价格（105元）超过整箱价（100元）时，直接买整箱更划算。  
> - **核心思路**：先计算整十部分的花费（`n/10 * 100`），剩余部分若≥7则再买一整箱（+100元），否则单买（+`n%10 * 15`）。  
> - **可视化设计**：像素动画将展示数量从1增加到50时花费的变化，当数量达到7/17/27等节点时触发闪光特效和“叮”音效，突显策略转折点。  
> - **复古游戏化**：采用8-bit风格商店界面，数量增加时显示金币消耗动画，临界点播放FC游戏式胜利音效，控制面板支持调速单步执行。

#### 核心难点辨析与解题策略
1. **难点1：发现价格临界点（7个）**  
   * **分析**：单买7个需105元＞整箱100元，但6个仅90元＜100元。优质题解通过列表对比（如Soyilieber的表格）或计算（7*15>100）定位该值。  
   * 💡 **学习笔记**：贪心算法的核心是寻找策略转折的**量化临界点**。

2. **难点2：剩余部分的策略选择**  
   * **分析**：整十部分计算简单，但剩余部分需动态决策。`char32_t`用`if(n>=7)`直接分支，`xuzhexi`则用`min(ans1,ans2)`规避复杂判断。  
   * 💡 **学习笔记**：分支语句和最小值比较是处理离散决策的两种高效技巧。

3. **难点3：避免整数除法陷阱**  
   * **分析**：计算整十部分时，`n/10`需用整数除法（截断小数）。`银银银`在注释中强调`(N/10)*100`的整除特性，防止浮点误差。  
   * 💡 **学习笔记**：整除`/`和取模`%`是处理分段问题的黄金组合。

##### ✨ 解题技巧总结
- **问题分解法**：分离整十部分（固定成本）和余数部分（动态决策）。  
- **临界点预计算**：通过数学（7*15>100）提前确定策略转折点。  
- **双方案比较**：直接计算两种策略取最小值，简化条件判断（如xuzhexi解法）。

#### C++核心代码实现赏析
**通用核心实现**  
```cpp
#include <iostream>
using namespace std;
int main() {
    int n;
    cin >> n;
    int ans = (n / 10) * 100; // 整十部分费用
    n %= 10;                  // 余数
    ans += (n >= 7) ? 100 : n * 15; // 余数≥7则买整箱
    cout << ans << endl;
    return 0;
}
```
**代码解读概要**：  
> 先计算整十部分花费（整除取商），余数部分用三元运算符选择策略。时间复杂度**O(1)**，空间复杂度**O(1)**。

**题解一（char32_t）片段**

```cpp
cost += (n/10)*100; // 整十部分
```

**亮点**：流程清晰，无冗余操作。  
**解读**：`n%=10`巧妙复用变量存储余数，避免新变量。`if-else`分支直观体现贪心逻辑。  
💡 **学习笔记**：复用变量可提升代码紧凑性，但需确保不影响可读性。

**题解二（银银银）片段**

**亮点**：变量名自注释，边界处理严谨。  
**解读**：`zuixiaojine`（最小花费）提升语义明确性。注释解释每步数学含义，便于调试。  
💡 **学习笔记**：语义化变量名是高质量代码的基石。

**题解三（xuzhexi）片段**  
```cpp
int ans1 = (n/10)*100 + (n%10)*15; // 不额外买
int ans2 = (n/10+1)*100;           // 多买一箱
printf("%d", min(ans1, ans2));     // 取最小值
```
**亮点**：无分支策略，数学思维降维打击。  
**解读**：直接计算两种方案，`min()`函数自动选择最优解。避免临界点判断，逻辑更健壮。  
💡 **学习笔记**：将条件判断转化为数值比较，可提升代码简洁性。

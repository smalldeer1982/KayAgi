# 题目信息

# 最大値

## 题目描述

请输出排列 $a$ 的最大值。

## 说明/提示

对于 $100\%$ 的数据，$1 \le n \le 100$。

## 样例 #1

### 输入

```
5

3 1 5 4 2```

### 输出

```
5```

## 样例 #2

### 输入

```
6

1 2 3 4 5 6```

### 输出

```
6```

## 样例 #3

### 输入

```
7

7 6 5 4 3 2 1```

### 输出

```
7```

## 样例 #4

### 输入

```
20

19 11 10 7 8 9 17 18 20 4 3 15 16 1 5 14 6 2 13 12```

### 输出

```
20```

🗣️ **初步分析**：  
> 求数组最大值就像在运动会上找跑得最快的选手——只需让每个选手依次和当前冠军比较即可。本题核心是**单次遍历+动态比较**：  
> - **核心流程**：初始化`max`为极小值 → 遍历数组元素 → 若当前值 > `max`则更新`max`  
> - **可视化设计**：用像素小人代表`max`，新数字作为挑战者上台比拼，胜者留下（颜色高亮），败者离场（灰化）

#### **3. 核心难点辨析与解题策略**  

1. **难点1：初始值设定**  
   - **分析**：若初始化`max=0`，遇到负数会出错。优质解采用首元素或`INT_MIN`（需`#include<climits>`）  
   - 💡 **学习笔记**：初始值必须覆盖数据范围极值  

2. **难点2：遍历效率优化**  
   - **分析**：必须完整遍历数组（不可提前终止），时间复杂度固定为*O(n)*。优先队列法(*O(nlogn)*)效率更低但展示多样性  
   - 💡 **学习笔记**：打擂台法是时空最优解  

3. **难点3：理解比较本质**  
   - **分析**：`max = std::max(max, now)` 比 `if`判断更简洁，需理解库函数封装思想  
   - 💡 **学习笔记**：库函数能简化代码但隐藏细节  

### ✨ **解题技巧总结**  
- **在线处理**：数据边读边比，避免存储整个数组（节省内存）  
- **防御初始化**：用首元素或极限值初始化，避免假设数据范围  
- **活用STL**：`std::max`/优先队列等工具提升编码效率

#### **4. C++核心代码实现赏析**  

**通用核心实现**  
```cpp
#include <iostream>
#include <climits> // 使用INT_MIN需包含
int main() {
    int n, num, max_val = INT_MIN; 
    std::cin >> n;
    while (n--) {
        std::cin >> num;
        if (num > max_val) max_val = num; 
    }
    std::cout << max_val;
}
```
**代码解读概要**：  
> 1. 初始化`max_val`为最小整数（防御负数）  
> 2. `while(n--)`控制读取次数（比`for`更简洁）  
> 3. 实时比较更新，无额外空间占用  

**题解一核心片段**  
```cpp
scanf("%d",&now), maxx=max(maxx,now); 
```
**亮点**：逗号运算符实现单行逻辑  
**学习笔记**：链式操作可压缩行数，但需确保可读性  

**题解二核心片段**  
```cpp
int MAX = a[0]; 
for (int i=0; i<n; i++) 
    if (a[i] > MAX) MAX = a[i];
```
**亮点**：显式说明初始化原理  
**学习笔记**：首元素初始化法需确保数组非空  

**题解三核心片段**  
```cpp
priority_queue<int> q;
q.push(a); 
cout << q.top();
```
**亮点**：展示STL高级用法  
**学习笔记**：容器虽方便但需警惕额外开销

# 分類たん

## 题目描述

[problemUrl]: https://atcoder.jp/contests/tenka1-2012-qualA/tasks/tenka1_2012_qualA_2

 分類たんは、スペースをカンマ区切りにするのが大好き。  
 でも、スペースが続いたときは、カンマ $ 1 $ つで書きたい。

 文字列が与えられるので、与えられた文字列のスペースをカンマ区切りにした文字列を出力してください。

 入力は以下の形式で標準入力から与えられる。 > $ c_1c_2…c_N $

- 入力として文字列が $ 1 $ 行で与えられる。
- 入力の文字列長 $ N $ は、 $ 1\ \leq\ N\ \leq\ 2000 $ を満たす。
- $ i $ 文字目の文字 $ c_i $ は大文字のアルファベット (`A`, $ … $, `Z`) 、カンマ (`,`) 、スペース(` `) のいずれかである。
- 文字列の最初の文字 $ c_1 $ と最後の文字 $ c_N $ はスペースではない。
 
 入力として与えられた文字列のスペースをカンマ区切りにした文字列を標準出力に $ 1 $ 行で出力せよ。  
 ただし、複数の連続したスペースは $ 1 $ つのカンマにする。  
 なお、行の終端には改行が必要である。 ```


## 样例 #1

X Y Z```

X,Y,Z```

QWERTY```

## 样例 #3

A  B, C```

### 输出

A,B,,C```

## 3. 核心难点辨析与解题策略

<difficulty_intro>
在解决这道题的过程中，我们常遇到以下关键点。结合优质题解的思路，我们一起拆解它们：
</difficulty_intro>

1.  **关键点1**：如何将连续空格替换为单个逗号？
    * **分析**：连续空格的本质是多个空格连在一起，需要合并成一个逗号。题解利用`cin`读取字符串时会自动跳过所有空格（包括连续空格）的特性，将连续空格视为分隔符，这样每次读取的字符串之间自然用一个逗号连接即可。
    * 💡 **学习笔记**：`cin`读取字符串时会跳过前导空格，这是处理连续分隔符的“隐藏工具”！

2.  **关键点2**：如何避免开头输出多余逗号？
    * **分析**：如果直接每次输出“字符串+逗号”，最后会多出一个逗号。题解采用“先输出第一个字符串，后续输出逗号+字符串”的策略，确保第一个字符串前无逗号，后续字符串前有逗号，完美解决问题。
    * 💡 **学习笔记**：处理序列连接时，“先处理第一个元素，再循环处理后续”是避免首尾冗余的常用技巧。

3.  **关键点3**：如何确保行末换行？
    * **分析**：题目明确要求行末换行，题解在最后显式输出`endl`，确保符合要求。这提醒我们注意题目中的细节要求（如换行、格式），避免因小失大。
    * 💡 **学习笔记**：输出格式的小细节（如换行、空格）是竞赛中的“隐形扣分点”，需仔细检查。

## 4. C++核心代码实现赏析

<code_intro_overall>
我们以题解一的代码为基础，提炼一个完整且通用的核心实现，帮助大家把握整体框架。

**本题通用核心C++实现参考**
* **说明**：此代码直接来自题解一，因其逻辑简洁、高效且严格符合题目要求，是字符串处理的典型实现。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    #include <string>
    using namespace std;

    int main() {
        string c;
        cin >> c; // 读取第一个非空格字符串
        cout << c; // 先输出第一个字符串，避免开头逗号
        while (cin >> c) { // 循环读取后续字符串（自动跳过空格）
            cout << "," << c; // 输出逗号+字符串
        }
        cout << endl; // 行末换行
        return 0;
    }
    ```
* **代码解读概要**：
    > 代码首先读取第一个非空格字符串并输出，之后通过`while (cin >> c)`循环读取后续字符串（`cin`自动跳过所有空格，包括连续空格），每次输出“逗号+字符串”。最后输出换行符，确保格式正确。整个过程仅需一次遍历，时间复杂度O(N)，高效简洁。

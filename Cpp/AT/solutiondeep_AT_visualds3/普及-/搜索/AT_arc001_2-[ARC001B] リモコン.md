# 题目信息

# [ARC001B] リモコン

## 题目描述

高桥君要调整空调的设定温度。现在的设定温度是A度，而他想调到B度。
空调遥控器按一次可以:
- 上调或下调1度
- 上调或下调5度
- 上调或下调10度
高桥君想求出从A调到B度的最小操作数。

## 说明/提示

样例1: 
输入：
```
7 34
```
输出：
```
5
```
依次上调10、10、5、1、1度即可

样例2: 
输入：
```
19 28
```
输出：
```
2
```
上调10度、下调1度即可。


样例3: 
输入：
```
10 10
```
输出：
```
0
```
温度一样时无需调整。

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：图论（最短路）/ 贪心算法


### 🗣️ 初步分析  
这道题的核心是**找到从当前温度A到目标温度B的最少操作次数**，每次操作可以选择±1、±5、±10度。我们可以从两个角度理解这个问题：  
- **图论视角**：把每个温度看作图中的一个节点，每次操作看作连接两个节点的边（边权为1），问题转化为**求A到B的最短路径**（最少步数）。  
- **贪心视角**：类似“找零钱”问题——尽可能用大的步长（如10度）快速缩小温差，剩下的小温差用更灵活的步长（如5度、1度）调整。  

**核心难点**：  
1. 如何将温度调整问题转化为图的最短路问题？  
2. 贪心策略中，如何处理小温差（如9度=10-1，只需2步）的最优解？  
3. 搜索算法中，如何避免重复访问节点（剪枝）？  

**可视化设计思路**：  
我们将用**8位像素风格**展示BFS（广度优先搜索）的过程：  
- 用像素方块表示温度节点（如A=7用绿色方块，B=34用红色方块）；  
- 队列用“像素队列”动态展示，每次扩展节点时，用蓝色闪烁标记当前处理的节点；  
- 每步操作伴随“叮”的像素音效，找到目标时播放“胜利”音效；  
- 支持“单步执行”和“自动播放”，方便观察队列变化和节点扩展。

## 3. 核心难点辨析与解题策略

### 1. 问题建模：如何将温度调整转化为最短路？  
**分析**：  
每个温度是一个节点，每次操作（±1、±5、±10）是连接两个节点的边（边权1）。例如，温度7可以连接到8（+1）、6（-1）、12（+5）、2（-5）、17（+10）、-3（-10，忽略越界）。求A到B的最短路径，就是求最少操作次数。  
**学习笔记**：图论建模的关键是“节点”和“边”的定义，找到问题与图的对应关系。  


### 2. 贪心策略：如何处理小温差的最优解？  
**分析**：  
大温差（≥8度）用10度调整是最优的，但小温差（如9度）需要灵活选择。例如，9度=10-1（2步）比5+1+1+1+1（5步）好。通过枚举0-10度的所有情况，总结出最优步数（如数组`a[11]`），可以快速计算余数部分的步数。  
**学习笔记**：贪心策略需要验证小范围的正确性，避免“局部最优导致全局最优”的错误。  


### 3. 搜索算法：如何避免重复访问节点？  
**分析**：  
搜索时，同一个温度可能被多次访问（如7→17→7），导致无限循环或超时。可以用**标记数组**（如`vis`）记录已访问的温度，避免重复处理。例如，wdcxz的DFS题解中，`v[n]`标记温度n是否已访问，防止重复递归。  
**学习笔记**：剪枝和判重是搜索算法的关键，能大幅提高效率。  


### ✨ 解题技巧总结  
- **模型转换**：将实际问题转化为图论或贪心模型，简化问题。  
- **贪心验证**：小范围枚举验证贪心策略的正确性。  
- **搜索优化**：用标记数组避免重复访问，用剪枝减少递归深度。  


## 4. C++核心代码实现赏析

### 本题通用核心C++实现参考（贪心）  
**说明**：综合sxtm12138和云浅知处的贪心思路，简洁高效。  
```cpp
#include <iostream>
#include <cmath>
using namespace std;

int main() {
    int A, B;
    cin >> A >> B;
    int diff = abs(B - A);
    int count = 0;
    // 处理大温差（用10度）
    while (diff >= 8) {
        diff = abs(diff - 10);
        count++;
    }
    // 处理小温差（用数组查询）
    int a[] = {0, 1, 2, 3, 2, 1, 2, 3, 3, 2, 1};
    count += a[diff];
    cout << count << endl;
    return 0;
}
```  
**代码解读概要**：  
- 计算温差`diff`；  
- 用`while`循环处理大温差（≥8度），每次用10度调整；  
- 用数组`a`查询小温差的最优步数，累加得到总次数。  


### 题解一：贪心代码片段（作者：sxtm12138）  
**亮点**：处理小温差的`min`函数，灵活选择5度或1度。  
```cpp
// 处理小温差
cout << t + min(k, abs(5 - k) + 1) << endl;
```  
**代码解读**：  
- `k`是剩余温差（<8度）；  
- `min(k, abs(5 - k) + 1)`表示：要么用1度调整（k步），要么用5度调整后再用1度（如k=4→5-1，共2步），取较小值。  
**学习笔记**：`min`函数可以简化条件判断，提高代码可读性。  


### 题解二：Dijkstra代码片段（作者：Diamiko）  
**亮点**：链式前向星存图，优先队列优化。  
```cpp
// 建图（处理每个温度的边）
for (int i = 0; i <= 40; i++) {
    if (i + 1 <= 40) addEdge(i, i + 1, 1);
    if (i - 1 >= 0) addEdge(i, i - 1, 1);
    if (i + 5 <= 40) addEdge(i, i + 5, 1);
    if (i - 5 >= 0) addEdge(i, i - 5, 1);
    if (i + 10 <= 40) addEdge(i, i + 10, 1);
    if (i - 10 >= 0) addEdge(i, i - 10, 1);
}
```  
**代码解读**：  
- 枚举每个温度`i`，添加所有可能的边（±1、±5、±10）；  
- `addEdge`函数用链式前向星存储边，适合大规模图。  
**学习笔记**：链式前向星是图论中常用的存图方式，效率高。

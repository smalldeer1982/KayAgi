# 题目信息

# [ARC001B] リモコン

## 题目描述

高桥君要调整空调的设定温度。现在的设定温度是A度，而他想调到B度。
空调遥控器按一次可以:
- 上调或下调1度
- 上调或下调5度
- 上调或下调10度
高桥君想求出从A调到B度的最小操作数。

## 说明/提示

样例1: 
输入：
```
7 34
```
输出：
```
5
```
依次上调10、10、5、1、1度即可

样例2: 
输入：
```
19 28
```
输出：
```
2
```
上调10度、下调1度即可。


样例3: 
输入：
```
10 10
```
输出：
```
0
```
温度一样时无需调整。

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：最短路径（BFS/DFS）、贪心算法


🗣️ **初步分析**：  
解决“リモコン”问题，核心是找到从当前温度A到目标温度B的**最少操作次数**。这就像在一个“温度迷宫”里找出口——每一步可以走±1、±5、±10度，我们需要找到“最快走到出口”的路线。  

### 核心算法类比
- **最短路径（BFS/DFS）**：像“洪水扩散”——从起点A开始，每次向六个方向（±1、±5、±10）扩展，第一次到达B的步数就是最短路径。比如样例1中，7→17→27→32→33→34，每一步都是“扩散”的结果。  
- **贪心算法**：像“找零钱”——用最大的“面额”（10度）先凑，不够再用5度、1度。比如样例2中，19→29→28，用10度和-1度，刚好两步。  

### 核心难点与解决方案
- **难点1**：如何避免重复搜索（比如温度来回跳）？  
  解决方案：用**判重数组**（如`vis[]`）标记已访问的温度，防止循环。  
- **难点2**：贪心算法如何处理10度以内的温差？  
  解决方案：总结规律（比如9度=10-1，用2步；8度=10-1-1，用3步），用数组或条件判断直接取最优解。  

### 可视化设计思路
我们将用**8位像素风格**设计动画，模拟BFS过程：  
- **场景**：屏幕中间是0-40的温度刻度（像素方块），当前温度用红色标记，队列用蓝色方块在下方显示。  
- **动画步骤**：  
  1. 初始时，红色方块停在A（比如7），蓝色队列中有A。  
  2. 每一步，红色方块闪烁，然后向六个方向扩展（比如7→8、6、12、2、17、-3），新温度用黄色标记，加入蓝色队列。  
  3. 当黄色方块到达B（比如34）时，变成绿色，播放“胜利音效”（如FC游戏的“叮~”）。  
- **交互**：控制面板有“单步执行”（一步步看扩展）、“自动播放”（调速滑块）、“重置”（回到初始状态）。  


## 2. 精选优质题解参考

### 题解一：贪心算法（作者：sxtm12138，赞：20）
* **点评**：  
  这道题的“最优解”！思路像“找零钱”——先处理大温差（≥8度时用10度调整），再处理小温差（10度以内用规律取最优）。代码简洁，逻辑清晰，比如用`k=abs(n-k)`计算温差，然后循环处理10度的倍数，最后用`min(k, abs(5-k)+1)`取10度以内的最优解。比如样例1中，温差27→2次10度（剩7）→7=5+1+1（3步），总5步，完全正确。  

### 题解二：DFS带剪枝（作者：wdcxz，赞：11）
* **点评**：  
  DFS（深度优先搜索）的“聪明版”！用`v[]`数组判重（避免重复访问同一温度），用`tot`记录最小步数（剪枝：如果当前步数≥`tot`，直接返回）。比如搜索时，从A开始，向六个方向递归，一旦找到B就更新`tot`。虽然DFS通常不如BFS高效，但剪枝后完全可以通过本题，适合学习“如何优化递归”。  

### 题解三：贪心查表法（作者：云浅知处，赞：4）
* **点评**：  
  代码“最短”的解法！用数组`a[11]`存储10度以内的最优解（比如`a[9]=2`，对应10-1），然后处理温差的十位（`y=C/10`）和个位（`x=C%10`），直接输出`a[x]+y`。比如样例2中，温差9→`y=0`，`x=9`，`a[9]=2`，总2步，非常高效。这种“查表法”适合规律明显的问题，值得学习。  


## 3. 核心难点辨析与解题策略

### 1. 如何避免重复搜索？（BFS/DFS的关键）
* **分析**：  
  如果不判重，比如从7→8→7→8→…，会无限循环，导致程序超时。优质题解中，`wdcxz`用`v[]`数组标记已访问的温度（`v[n]=1`表示已访问），`小小小朋友`的BFS用`vis[]`数组判重，都是解决这个问题的关键。  
* 💡 **学习笔记**：判重是搜索算法的“安全绳”，必须记住！

### 2. 贪心算法如何处理10度以内的温差？（规律总结）
* **分析**：  
  10度以内的温差有规律，比如：  
  - 9度：10-1（2步）→比9步好；  
  - 8度：10-1-1（3步）→比8步好；  
  - 4度：5-1（2步）→比4步好。  
  优质题解中，`sxtm12138`用`min(k, abs(5-k)+1)`总结了这个规律，`云浅知处`用数组`a[11]`存储了所有情况，都是正确的。  
* 💡 **学习笔记**：贪心算法的核心是“找规律”，必须仔细总结小范围的情况。

### 3. 如何选择算法？（BFS vs 贪心）
* **分析**：  
  - 如果问题是“最短路径”（等权），BFS是“标准答案”；  
  - 如果问题有“贪心选择性质”（比如大的先选），贪心算法更高效。  
  本题中，贪心算法的时间复杂度是O(1)（处理温差的十位和个位），比BFS的O(N)（N是温度范围）更快，所以是“最优解”。  
* 💡 **学习笔记**：根据问题性质选择算法，能让程序更高效。


## 4. C++核心代码实现赏析

### 本题通用核心C++实现参考（贪心查表法）
* **说明**：综合`云浅知处`的题解，代码简洁高效，适合快速解决问题。  
* **完整核心代码**：  
  ```cpp
  #include <iostream>
  #include <cstdlib> // 用于abs函数
  using namespace std;

  int main() {
      int a[11] = {0, 1, 2, 3, 2, 1, 2, 3, 3, 2, 1}; // 0-10度的最优步数
      int A, B, C, x, y;
      cin >> A >> B;
      C = abs(B - A); // 计算温差
      y = C / 10; // 10度的次数
      x = C % 10; // 剩余度数
      cout << a[x] + y << endl; // 总步数
      return 0;
  }
  ```
* **代码解读概要**：  
  1. 数组`a`存储0-10度的最优步数（比如`a[9]=2`表示9度需要2步）；  
  2. 计算温差`C`（绝对值）；  
  3. 计算10度的次数`y`（`C/10`）和剩余度数`x`（`C%10`）；  
  4. 输出`a[x]+y`（总步数）。


### 针对各优质题解的片段赏析

#### 题解一：贪心算法（作者：sxtm12138）
* **亮点**：用循环处理大温差，规律总结正确。  
* **核心代码片段**：  
  ```cpp
  while (k >= 8) { // 温差≥8时，用10度调整
      k = abs(k - 10);
      t++;
  }
  cout << t + min(k, abs(5 - k) + 1) << endl; // 处理剩余度数
  ```
* **代码解读**：  
  - 循环部分：当温差≥8时，用10度调整（比如27→17→7→…），每次`k`减少10（或增加，因为`abs`），`t`加1；  
  - 剩余度数部分：`min(k, abs(5 - k) + 1)`表示，比如k=7时，`min(7, abs(5-7)+1)=min(7,3)=3`（5+1+1），正确。  
* 💡 **学习笔记**：循环处理大问题，小问题用规律，这是贪心算法的常用技巧。

#### 题解二：DFS带剪枝（作者：wdcxz）
* **亮点**：用判重数组和剪枝优化递归。  
* **核心代码片段**：  
  ```cpp
  inline void dfs(int n, int t) {
      if (n > 40 || n < 0 || v[n] == 1) return; // 边界和判重
      if (t > tot) return; // 剪枝：当前步数≥最小步数，返回
      if (n == b) { tot = min(tot, t); return; } // 找到目标，更新最小步数
      v[n] = 1; // 标记已访问
      dfs(n+1, t+1);
      dfs(n-1, t+1);
      // 其他四个方向（±5、±10）类似
      v[n] = 0; // 回溯：取消标记
  }
  ```
* **代码解读**：  
  - 边界判断：温度不能超过0-40（题目中A、B≤40）；  
  - 判重：`v[n] == 1`表示已访问，不再处理；  
  - 剪枝：`t > tot`表示当前步数已经比之前找到的最小步数大，不需要继续搜索；  
  - 回溯：`v[n] = 0`表示取消标记，让其他路径可以访问这个温度。  
* 💡 **学习笔记**：剪枝和回溯是DFS的“灵魂”，能大大提高效率。

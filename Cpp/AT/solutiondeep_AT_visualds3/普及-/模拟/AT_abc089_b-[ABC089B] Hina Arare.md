# 题目信息

# [ABC089B] Hina Arare

## 题目描述

[problemUrl]: https://atcoder.jp/contests/abc089/tasks/abc089_b

日本では、$ 3 $ 月 $ 3 $ 日にひなあられという、色のついたお菓子をお供えする習慣があります。

$ 1 $ つの袋があり、ひなあられが $ N $ 個入っています。

この袋には、桃色、白色、緑色の $ 3 $ 種類か、桃色、白色、緑色、黄色の $ 4 $ 種類のひなあられが入っていることが分かっています。

桃色を `P`、白色を `W`、緑色を `G`、黄色を `Y` と表したとき、袋からひなあられを $ 1 $ 粒ずつ取り出していったところ、$ i $ 番目に取り出したひなあられの色は $ S_i $ でした。

この袋に $ 3 $ 種類のひなあられが入っていた場合は `Three`、$ 4 $ 種類のひなあられが入っていた場合は `Four` と出力してください。

## 说明/提示

### 制約

- $ 1\ \leq\ N\ \leq\ 100 $
- $ S_i $ は `P` か `W` か `G` か `Y`
- $ S_i= $`P`、$ S_j= $`W`、$ S_k= $`G` を満たす $ i,j,k $ が必ず存在する

### Sample Explanation 1

袋に $ 4 $ 種類のひなあられが入っていたので `Four` と出力するとよいです。

### Sample Explanation 2

袋に $ 3 $ 種類のひなあられが入っていたので `Three` と出力するとよいです。

## 样例 #1

### 输入

```
6

G W Y P Y W```

### 输出

```
Four```

## 样例 #2

### 输入

```
9

G W W G P W P G G```

### 输出

```
Three```

## 样例 #3

### 输入

```
8

P Y W G Y W Y Y```

### 输出

```
Four```

1.  **关键点1**：如何高效统计不同颜色的数量？

**本题通用核心C++实现参考**
* **说明**：本代码综合了昒昕题解的`set`方法，因其简洁高效且充分利用STL特性，适合作为通用实现。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    #include <set>
    using namespace std;

    int main() {
        int n;
        char c;
        set<char> colors; // 用于去重的集合

        cin >> n;
        for (int i = 0; i < n; ++i) {
            cin >> c;
            colors.insert(c); // 插入颜色，自动去重
        }

        // 根据集合大小输出结果（题目保证至少3种）
        if (colors.size() == 3) {

        }

    }
    ```
* **代码解读概要**：
    > 代码首先读取输入的颜色数量`n`，然后逐个读取颜色字符并插入`set`集合（自动去重）。最后通过判断集合的大小（`colors.size()`），输出“Three”（3种）或“Four”（4种）。核心逻辑是利用`set`的去重特性统计不同颜色数量。

    ```cpp

* **代码解读**：
    > `set<char> s`是一个存储字符的集合，`insert`会自动去重。`ch[3]`对应“Three”，`ch[4]`对应“Four”，直接通过`ch[s.size()]`输出结果。为什么这样设计？因为题目保证至少有3种颜色，所以`size()`只能是3或4，打表数组比`if-else`更简洁。
* 💡 **学习笔记**：打表法可以将复杂的条件判断转化为数组索引，让代码更简洁易读。

**题解二：作者AgrumeStly（法一）**
* **亮点**：用标记变量直观统计颜色是否出现。
* **核心代码片段**：
    ```cpp
    int p, w, g, y;
    int main() {
        cin >> n;
        for (int i = 1; i <= n; i++) {
            char c; cin >> c;
            if (c == 'P') p = 1;
            if (c == 'W') w = 1;
            if (c == 'G') g = 1;
            if (c == 'Y') y = 1;
        }
        if (p + w + g + y == 3) cout << "Three";
        else cout << "Four";
    }
    ```
* **代码解读**：
    > `p, w, g, y`初始为0，遇到对应颜色就设为1。最后将它们相加，和为3说明缺少Y（输出Three），和为4说明包含所有颜色（输出Four）。为什么用整数变量而不是布尔？因为整数相加更方便统计总数，逻辑更直观。
* 💡 **学习笔记**：当元素种类固定时，用标记变量统计是否出现是最直接的方法，适合刚学编程的同学理解基础逻辑。

**题解三：作者3a51_（思路2，map版）**
* **亮点**：用`map`统计颜色出现次数。
* **核心代码片段**：
    ```cpp
    map<char, int> m;
    int main() {
        cin >> n;
        for (int i=1; i<=n; i++) {
            cin >> c;
            m[c]++;
        }
        if (m['G']==0 || m['Y']==0 || m['P']==0 || m['W']==0)
            cout << "Three";
        else cout << "Four";
    }
    ```
* **代码解读**：
    > `map<char, int> m`将颜色（键）映射到出现次数（值）。遍历输入后，检查是否有颜色的次数为0（即未出现）。若有，说明是3种颜色；否则是4种。为什么用`map`？因为`map`可以动态记录键值对，适合颜色种类不确定的扩展场景。
* 💡 **学习笔记**：`map`适合需要记录“元素-属性”关系的场景（如本题“颜色-是否出现”），是更通用的数据结构。

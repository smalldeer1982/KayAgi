## 题目描述

[problemUrl]: https://atcoder.jp/contests/arc009/tasks/arc009_1

ある日高橋君はお母さんに近くのスーパーまでおつかいを頼まれました。  
 お母さんに手渡されたおつかいメモには、買ってきて欲しい商品の値段と個数がそれぞれ書かれています。  
 ただしメモに書かれている値段には消費税が含まれていませんが、全ての商品には消費税が $ 5\% $ かかります。  
 高橋君のおつかいに必要な金額を答えなさい。  
 なお、消費税は $ 1 $ 円未満は切り捨てます。 入力は以下の形式で標準入力から与えられる。

> $ N $ $ a_0 $ $ b_0 $ $ a_1 $ $ b_1 $ : : $ a_{N-1} $ $ b_{N-1} $

- 入力は $ N+1 $ 行ある。
- $ 1 $ 行目には、購入する商品の品数を表す整数 $ N(1≦N≦10) $ が与えられる。
- $ 2 $ 行目から $ N $ 行の $ i+2 $ 行目にはある商品の購入したい個数を表す整数 $ a_i(1≦a_i≦10) $ とその単価を表す整数 $ b_i(1≦b_i≦1,000) $ が空白区切りで与えられる。
 
 高橋君のおつかいに必要な金額を標準出力に $ 1 $ 行で出力せよ。  
 なお、最後には改行を出力せよ。 ```

## 3. 核心难点辨析与解题策略

<difficulty_intro>
在解决这道题的过程中，我们需要重点关注以下几个关键点：
</difficulty_intro>

1.  **关键点1：正确计算商品总价**  
    * **分析**：总价是所有商品的“数量×单价”之和。需要注意变量类型的选择——虽然题目中n≤10，a_i≤10，b_i≤1000（总价最大为10×10×1000=100000），但用`long long`或`double`能避免溢出风险（比如n=10时总价是100000，乘1.05后是105000，仍在`int`范围内，但用更大类型更安全）。  
    * 💡 **学习笔记**：变量类型选择要“宁大勿小”，避免溢出。

2.  **关键点2：消费税的计算与取整**  
    * **分析**：消费税是总价的5%，但1円以下要舍去。例如总价740円，税是37円（740×0.05=37），总金额777円；若总价3875円，税是193.75円，总金额4068.75円，舍去0.75后是4068円。直接将总价×1.05后强制转换为`int`即可实现“舍去小数”（因为`int`会截断小数部分）。  
    * 💡 **学习笔记**：`(int)(x)`会直接截断x的小数部分，是本题取整的关键操作。

3.  **关键点3：输出格式的细节**  
    * **分析**：题目要求“最后输出换行”，但部分新手可能忽略这一点导致WA（如作者A_Plus_Gu提到的“WA了四五次”）。需要确保输出语句（如`cout<<...<<endl`或`printf("%d\n", ...)`）包含换行符。  
    * 💡 **学习笔记**：竞赛题中，输出格式（换行、空格、精度）是常见的“隐形扣分点”，务必仔细检查。

## 4. C++核心代码实现赏析

<code_intro_overall>
我们综合优质题解的思路，提炼出一个清晰、健壮的通用核心实现：
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：此代码综合了Yoo_和Rbu_nas的思路，用`long long`存储总价避免溢出，强制转换取整，确保正确性。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    using namespace std;

    int main() {
        int n;
        long long total = 0; // 总价（未税）
        cin >> n;
        for (int i = 0; i < n; ++i) {
            int a, b;
            cin >> a >> b;
            total += a * b; // 累加每件商品的价格（数量×单价）
        }
        total = (long long)(total * 1.05); // 计算税后总价并取整
        cout << total << endl; // 输出并换行
        return 0;
    }
    ```
* **代码解读概要**：  
  代码首先读取商品数量`n`，然后循环读取每件商品的数量`a`和单价`b`，累加得到未税总价`total`。接着将`total`乘以1.05（5%的税），并强制转换为`long long`（截断小数部分），最后输出结果并换行。

**题解一：作者Yoo_**
* **亮点**：用`double`存储总价，明确展示“先累加后乘税”的逻辑，注释强调了取整的关键。
* **核心代码片段**：
    ```cpp
    double sum = 0;
    for(int i=1; i<=n; i++) {
        scanf("%d%d", &a[i], &b[i]);
        sum += a[i] * b[i];
    }
    sum *= 1.05;
    cout << (int)sum << endl;
    ```
* **代码解读**：  
  `sum`初始化为0，循环中累加每件商品的“数量×单价”。乘以1.05后，`(int)sum`直接截断小数部分，得到税后总价。为什么用`double`？因为总价可能是大数（如100000），但`double`的精度足够存储整数部分（`double`能精确表示到2^53的整数，远大于本题最大值）。  
* 💡 **学习笔记**：`double`适合存储需要临时计算的数值，尤其当涉及乘法时，能避免整数溢出。

**题解二：作者A_Plus_Gu**
* **亮点**：变量名`ans`直观，强调输出换行的重要性，代码简洁。
* **核心代码片段**：
    ```cpp
    ll n, ans;
    cin >> n;
    for(int i=1; i<=n; i++) {
        ll x, y;
        scanf("%lld %lld", &x, &y);
        ans += x * y;
    }
    ans *= 1.05;
    write(ans);
    puts("");
    ```
* **代码解读**：  
  `ans`初始化为0（虽未显式初始化，但全局变量默认0），循环累加`x*y`。乘以1.05后，`write(ans)`输出结果，`puts("")`确保换行。作者用`ll`（`long long`的缩写）避免溢出，是安全的选择。  
* 💡 **学习笔记**：全局变量默认初始化为0，局部变量需显式初始化（如`ans=0`）。

**题解三：作者Rbu_nas**
* **亮点**：代码极简，变量名`p`（price）易懂，直接用`int`存储总价（本题足够）。
* **核心代码片段**：

* **代码解读**：

* 💡 **学习笔记**：局部变量建议显式初始化（如`int p = 0`），避免未知错误。

# 题目信息

# [ARC020A] 石を滑らせるゲーム

## 题目描述

蚂蚁Ant和Bug在玩一个关于石头的游戏。这个游戏使用一根笔直的细长的冰板和两个小的石头。冰板每1毫米有1000，-1000两种刻度。开始两只游戏蚂蚁玩家有一块石头。两只蚂蚁游戏者交替地从木板边缘滑动石头。然后，最终滑动的石头更接近0度刻度的玩家将成为该游戏的胜者。此外，如果两个石头上的距离为0，则判为平局。为了Ant先生和Bug先生这两只好玩的蚂蚁，请制定出两个棋子位置的刻度分别被给予时判定胜负的程序。

## 样例 #1

### 输入

```
2 3```

### 输出

```
Ant```

## 样例 #2

### 输入

```
1 0```

### 输出

```
Bug```

## 样例 #3

### 输入

```
-100 100```

### 输出

```
Draw```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：数学（绝对值比较）

🗣️ **初步分析**：
> 解决这道题的关键在于理解“更接近0”等价于“绝对值更小”。数学中的“绝对值”就像数轴上点到原点的距离——无论数是正还是负，距离0越近，绝对值越小。例如，-3和2的绝对值分别是3和2，所以2离0更近。

在本题中，我们需要比较输入的两个整数a和b的绝对值：
- 若a的绝对值更小 → Ant胜
- 若b的绝对值更小 → Bug胜
- 若相等 → 平局

核心难点是正确处理负数的绝对值计算，以及选择高效的比较方式。不同题解提供了多种思路：直接用`abs`函数、平方比较（因为平方结果与绝对值平方相等）、手动取反负数等。这些方法本质都是比较绝对值，只是实现方式不同。

## 3. 核心难点辨析与解题策略

<difficulty_intro>
解决此题的关键在于正确计算并比较绝对值。以下是常见难点及应对策略：
</difficulty_intro>

1.  **关键点1：正确理解“更接近0”的数学含义**
    * **分析**：“更接近0”等价于绝对值更小。例如，-5和3的绝对值分别是5和3，所以3更近。需注意负数的绝对值是其相反数，正数的绝对值是自身。
    * 💡 **学习笔记**：绝对值是数到0的“距离”，与正负无关。

2.  **关键点2：选择高效的绝对值计算方式**
    * **分析**：可用三种方法计算绝对值：
      - `abs(a)`：调用标准库函数，代码清晰。
      - `a*a`：平方后比较，避免函数调用（适合无`cmath`头文件场景）。
      - 手动取反：若`a<0`则`a=-a`，直观高效。
    * 💡 **学习笔记**：根据场景选择方法：竞赛中平方或手动取反可能更快；新手用`abs`更易理解。

3.  **关键点3：处理边界情况（如0）**
    * **分析**：当输入为0时（如样例2的输入1和0），0的绝对值是0，比任何非零数更小。需确保代码正确处理0的情况（如样例2中输出Bug，因0的绝对值更小）。
    * 💡 **学习笔记**：测试时需包含0、正数、负数等多种输入。

### ✨ 解题技巧总结
- **数学抽象**：将“距离0的远近”抽象为“绝对值比较”，简化问题。
- **代码简洁性**：优先选择直接反映数学逻辑的代码（如`abs`或平方），避免复杂操作（如字符串转数字）。
- **边界测试**：测试输入包含0、正负大数（如-1000和1000）等，确保代码鲁棒性。

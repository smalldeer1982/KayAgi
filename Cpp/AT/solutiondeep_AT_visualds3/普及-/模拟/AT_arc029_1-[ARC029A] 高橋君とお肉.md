# 题目信息

# [ARC029A] 高橋君とお肉

## 题目描述

[problemUrl]: https://atcoder.jp/contests/arc029/tasks/arc029_1

高橋君は友達とキャンプに行くことになった。

高橋君と友達は性能が同じである $ 2 $ 個の肉焼き器を持っており、それぞれの肉焼き器にお肉を乗せて並行して焼くことができる。一旦肉焼き器にお肉を乗せたら、お肉が焼きあがるまではその肉焼き器からお肉を取り出したり、その肉焼き器に別のお肉を乗せたりはできない。お肉が焼けたらお肉を取り出すことができる。$ 2 $ つの肉焼き器にまたがって $ 1 $ つのお肉を置くことはできない。また、お肉は全部で $ N $ 個あり、お肉には $ 1 $ から $ N $ まで番号が付けられている。お肉 $ i $ を焼くのには、どちらの肉焼き器でも時間 $ t_i $ だけかかる。お肉を肉焼き器に置く動作、取り出す動作には時間がかからない。

高橋君はお肉を焼く係であり、$ N $ 個すべてのお肉を焼くことになった。みんなお腹が空いているので、すべてのお肉を焼くのにかかる時間を最小化させたい。

すべてのお肉を焼くのにかかる時間の最小値を求めよ。

## 说明/提示

### Sample Explanation 1

一方の肉焼き器でお肉 $ 1 $ とお肉 $ 4 $ を、他方の肉焼き器でお肉 $ 2 $ とお肉 $ 3 $ を順に焼きます (下の図は参考図)。 !\[\](/img/arc/029/1-1.png)

### Sample Explanation 2

一方の肉焼き器でお肉 $ 3 $ を焼いている間に、他方の肉焼き器で残りすべてのお肉を焼きます。

> 解决这道题的关键是找到一种分配方式，让两个烤肉机的总烤制时间的最大值最小。简单来说，就像分糖果——把糖分给两个小朋友，尽量让他们拿到的糖量差不多，这样总“吃完时间”才会最短。在本题中，由于肉的数量N最多只有4块，直接枚举所有可能的分配方式就能找到最优解。

## 3. 核心难点辨析与解题策略

<difficulty_intro>
解决这道题的过程中，我们需要克服以下关键点，掌握这些能让你举一反三：
</difficulty_intro>

1.  **关键点1**：如何枚举所有可能的分配方式？
    * **分析**：当N较小时（如≤4），可以直接分类讨论或用DFS枚举所有分配。例如N=3时，每块肉有2种选择（烤肉机1或2），共2^3=8种分配方式，计算每种方式的最大时间后取最小。当N较大时，DFS可能超时，但本题数据范围小，枚举可行。
    * 💡 **学习笔记**：数据范围小的题目，枚举是“暴力但有效”的解法！

2.  **关键点2**：排序后如何利用顺序简化问题？
    * **分析**：将肉按时间从小到大排序后，大的肉优先分配可以减少总时间。例如N=3时，把最大的肉单独放一个烤肉机，小的两块放另一个，避免大肉被小肉“拖累”。排序后，分类讨论的条件更清晰（如N=4时比较1+4和2+3的组合）。
    * 💡 **学习笔记**：排序是预处理数据的常用技巧，能让后续逻辑更简单！

3.  **关键点3**：如何确保不遗漏最优解？
    * **分析**：例如N=4时，最优解可能是“1+4和2+3”或“1+2+3和4”中的较小者。需覆盖所有可能的分配组合，避免只考虑部分情况。分类讨论时要列举所有可能的组合（如大肉和小肉的搭配），DFS则通过遍历所有路径自动覆盖。
    * 💡 **学习笔记**：多思考“还有没有其他可能的分配方式？”能避免遗漏！

**本题通用核心C++实现参考**
* **说明**：此代码综合了多个优质题解的思路，针对N≤4的情况分类讨论，代码简洁高效，适合竞赛快速编写。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    #include <algorithm>
    using namespace std;

    int main() {
        int n, meat[5];
        cin >> n;
        for (int i = 0; i < n; ++i) cin >> meat[i];
        sort(meat, meat + n); // 从小到大排序

        if (n == 1) {
            cout << meat[0];
        } else if (n == 2) {
            cout << meat[1]; // 两块肉取较大者
        } else if (n == 3) {
            cout << max(meat[0] + meat[1], meat[2]); // 小的两块之和 vs 最大的
        } else if (n == 4) {
            int option1 = max(meat[0] + meat[3], meat[1] + meat[2]);
            int option2 = max(meat[0] + meat[1] + meat[2], meat[3]);
            cout << min(option1, option2); // 两种分配方式取最小
        }
        return 0;
    }
    ```
* **代码解读概要**：
    > 代码首先读取输入并排序，然后根据N的不同情况计算最优解。N=1直接输出；N=2取较大值；N=3比较小的两块之和与最大的肉；N=4比较两种分配方式的最大值，取较小者。排序后，肉的顺序更易处理，逻辑清晰。


* **代码解读**：
    > 当N=4时，有两种主要分配方式：①肉1+肉4和肉2+肉3（排序后肉1最小，肉4最大）；②肉1+肉2+肉3和肉4。代码计算这两种方式的最大值（即每个烤肉机的总时间），然后取较小的那个作为最终结果。例如样例1输入4、6、7、10（排序后为4,6,7,10），方式①的最大值是max(4+10=14,6+7=13)=14；方式②的最大值是max(4+6+7=17,10)=17，所以取14，与样例输出一致。
* 💡 **学习笔记**：分类讨论时，要覆盖所有可能的“大肉+小肉”组合，避免遗漏最优解。

**题解二：作者cherubim（DFS枚举）**
* **亮点**：用DFS遍历所有分配方式，通用且不易遗漏，适合理解枚举思想。
* **核心代码片段**：
    ```cpp
    inline void dfs ( int num, int c1, int c2 ) {
        if ( num == n + 1 ) {
            minn = min ( minn, max ( c1, c2 ) ) ;
            return ;
        }
        dfs ( num + 1, c1 + a[num], c2 ) ;
        dfs ( num + 1, c1, c2 + a[num] ) ;
    }
    ```
* **代码解读**：
    > `dfs`函数递归处理每一块肉：参数`num`表示当前处理第`num`块肉，`c1`和`c2`是两个烤肉机的累计时间。对于第`num`块肉，有两种选择——放到烤肉机1（`c1+a[num]`）或烤肉机2（`c2+a[num]`）。当处理完所有肉（`num==n+1`），计算当前分配的最大时间，并更新全局最小值`minn`。例如N=3时，DFS会遍历8种分配方式，找到最小的最大时间。
* 💡 **学习笔记**：DFS适合小数据量的全枚举问题，逻辑简单但能覆盖所有可能。

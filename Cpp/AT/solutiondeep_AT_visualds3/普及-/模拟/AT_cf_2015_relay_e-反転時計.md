# 题目信息

# 反転時計

## 题目描述

给出两个时间，$ht$时$mt$分 与 $hn$时$mn$分。

现在有一个时钟翻转的功能，只能在正午之前使用，即把 $hn$时$mn$分 转换到时钟上，将分针和时针分别逆时针旋转$180°$，得到一个更早的时间。如果这个时间不晚于$ht$时$mt$分，输出`Yes`，否则输出`No`。（注：**最后要换行**）

-----

9 0

10 30```

Yes```

Yes```

## 3. 核心难点辨析与解题策略

<difficulty_intro>
解决此题的关键在于正确模拟旋转操作并比较时间。以下是三个核心难点及应对策略：
</difficulty_intro>

1.  **关键点1：如何计算旋转后的时间？**
    * **分析**：时针逆时针转180°相当于倒推6小时（如10点→4点），分针倒推30分钟（如30分→0分）。直接计算`hn-6`和`mn-30`即可，无需复杂模运算（因题目仅要求比较先后，负数时间可视为“前一天”，但比较时仍有效）。
    * 💡 **学习笔记**：旋转180°=时针-6小时+分针-30分钟，直接减法更简单！

2.  **关键点2：如何处理“超过正午”的情况？**
    * **分析**：题目规定旋转功能“只能在正午前使用”，即hn时mn分必须≤12:00。若hn>12或hn=12且mn>0，则无法旋转，需直接比较原时间与ht:mt。
    * 💡 **学习笔记**：超过正午时，旋转功能不可用，直接比原时间！

3.  **关键点3：如何正确比较两个时间？**
    * **分析**：统一转换为分钟单位（如time=小时×60+分钟），比较数值大小即可。例如，9:00=540分钟，旋转后的10:30-390=630-390=240分钟（4:00），240≤540→Yes。
    * 💡 **学习笔记**：分钟单位统一，比较更简单！

### ✨ 解题技巧总结
<summary_best_practices>
- **时间单位统一**：将小时和分钟转换为总分钟数，避免分情况讨论（如“小时不同先比小时，相同再比分钟”）。
- **简化边界处理**：旋转后的时间可能为负（如5:00旋转后为-1:30），但比较时无需调整，直接用总分钟数判断即可。
- **特殊情况优先判断**：先处理“超过正午”的情况，再处理常规旋转，逻辑更清晰。
</summary_best_practices>

**本题通用核心C++实现参考**
* **说明**：此代码综合了lzqy_和yydfj的思路，处理了超过正午的情况，并统一用分钟单位比较，逻辑清晰且高效。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    using namespace std;

    int main() {
        int ht, mt, hn, mn;
        cin >> ht >> mt >> hn >> mn;
        
        // 转换为分钟单位
        int t1 = ht * 60 + mt;
        int t2 = hn * 60 + mn;
        
        // 处理超过正午的情况（hn>12 或 hn=12且mn>0）
        if ((hn == 12 && mn > 0) || hn > 12) {
            cout << (t2 <= t1 ? "Yes" : "No") << endl;
            return 0;
        }
        
        // 计算旋转后的时间（减6h30m=390分钟）
        int rotated_t2 = t2 - 390;
        cout << (rotated_t2 <= t1 ? "Yes" : "No") << endl;
        
        return 0;
    }
    ```
* **代码解读概要**：
    > 代码首先将输入的两个时间转换为分钟单位（t1和t2）。若hn时mn分超过正午（如12:30或13:00），直接比较t2和t1；否则，计算旋转后的时间（t2-390分钟），再比较是否≤t1。最后输出结果并换行。

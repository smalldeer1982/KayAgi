# Let's Watch Football

## 题目描述

Valeric和Valerko错过了最后一场欧洲杯足球赛，所以他们决定在网上观看比赛的关键时刻。他们想尽快开始观看，但连接速度太低。如果他们现在打开视频，它将“挂断”，因为每秒观看的数据大小将超过每秒下载的数据大小。

这些家伙想要在没有任何停顿的情况下观看整个视频，所以他们必须等待一些整数秒的时间才能下载视频的一部分。在这几秒钟过去之后，他们就可以开始观看了。等待整个视频下载是不必要的，因为视频可以下载后，他们才开始观看。

假设视频的长度是c，秒Valeric和Valerko在观看前等待了几秒钟。那么在任时刻t≤t0≤c+t则必须满足以下条件:以秒为单位接收的数据的大小不小于观看视频的-t秒所需的数据的大小。

当然，这些人希望等待的时间越短越好，所以你的任务是找到在打开视频之前等待的最小整数秒数。伙计们必须不停地看视频

## 说明/提示

第一个样本视频的长度为1秒，观看1秒需要4个数据单位，所以人们需要下载4，1=4单位的数据来观看整个视频。最理想的方式是等待3秒，直到3单位的数提将被下载，然后开始观看。当人们将观看1秒的视频时，一个数据单位将被下载而Valerik和Valerko将在观看结束时有4个数据单位。而且，在视频结束之前的每一刻，男人们都会有更多的数据，然后才是观看视频所必需的。

在第二个样本中，人们需要2.10=20个单位的数据，所以他们必须等待5秒，之后他们将在第二秒结束前获得20个单位的数据。然而，如果男人等待4秒，他们将能够观看视频的第一秒没有暂停，但他们将下载18个单位的数据由第二秒结束，它是那么不必要

# 题解

## 作者：W_SUN (赞：3)

[题目](https://www.luogu.com.cn/problem/CF195A)

这题不难，运用乘法分配律可以解出。

题目中给出了方程：$tb+cb=ac$，根据乘法分配律可以把这个式子变成如下：$(t+c)b=ac$，接下来解方程就可以了。

注意：向上取整使用函数 ceil，在 cmath 库里。
 
代码： 
```cpp
#include<bits/stdc++.h>//万能头
#define ll long long
using namespace std;
ll a,b,c;
int main(){
	cin>>a>>b>>c;
	ll ac=a*c;
	ac=ceil(ac*1.0/b);//向上取整
	ac-=c;//减去c得到t
	cout<<ac;
	return 0;
}
```

## 作者：Daniel_He (赞：2)

这道题目的大概意思是一段视频每秒用 $a$ 点数，每秒会增加 $b$ 点数（这些点数在这一秒可以直接用），视频一共有 $c$ 秒，求提前要等多少秒后的点数才能够放完这个视频。

这题不难，由于数据范围不大($a$,$b$,$c$<=$1000$)，使用公式或模拟都可以过。另一篇题解用的是公式，我就挂一个模拟代码。模拟用不着`long long`，使用`int`足够了。

如果每秒增加的点数比每秒消耗的点数少（$a$<=$b$）就可以直接输出$0$。

如果不是，可以先定义一个`point`来代表点数，然后进行死循环，每次让`point`加 $b$，然后判断一下`point`加上看视频时增加的点数（$b\times c$）是否大于看视频需要的点数（$a\times c$），如果是就输出循环变量（代表秒数）然后结束，不是就继续循环。

```cpp
#include<iostream>
using namespace std;
int main(){
    int a,b,c,point=0;//a、b、c同题目，point为点数
    cin>>a>>b>>c;//输入
    if(b>=a){//如果每秒增加的点数比每秒消耗的点数多
        cout<<0;
		return 0;
    } 
    for(int second=1;;second++){
        point+=b;
        if(point+b*c>=a*c){//如果点数够了
            cout<<second;
            return 0;//由于只需要输出一次，输出完直接结束就可以了，结尾也没必要再写一遍
        }
    }
}
```

## 作者：奥斯卡小冰人 (赞：1)

这道题唯一的难点是向上取整，我没有听说过 ceil 函数~~就算学过也忘了~~，所以想了一个笨方法。

前面的定义与输入就不说了，输入后，用 double 变量存储 $t$ 的计算结果。然后用一个整形变量 $d$ 存储 $t$ ，因为是整形，所以自动向下取整。之后再用一个 double 变量 $s$ 存储 $d$。如果 $t$ 是个整数的话，那么 $s = t$ ，直接输出 $s$ 就行。如果不是的话，那么就向上取整，输出 $s + 1$ 。

AC 代码：
```cpp
#include<bits/stdc++.h>
using namespace std;
int main()
{
    double t,s;
    int a,b,c,d;
    cin >> a >> b >> c;
    t = 1.0 * (a * c - c * b) / b;//首先算出这个数 用double的变量存储
    d = t; //因为d是整数 所以自动向下取整
    s = d; //同是double才能比较
    if(s != t) cout << s + 1; //如果不等于 证明d赋值时向下取整了 +1即可
    else cout << s; //如果没问题 就直接输出
    cout << endl;
    return 0;
}
```

这题就是有三个数$a$、$b$、$c$，$a$表示看这个视频每秒要用的点数，$b$表示每秒可以获得的点数，并且在这一秒直接就可以用，$c$表示这个视频的秒数，问他们需要至少等多少秒开始看视频才能一直持续的看完它（等的时候每秒获得$b$个点）。

这题好简单的样子，就是看最后一秒时总共获得的点与消耗的点数。首先要用掉$a\times c$个点，开始后总共能获得$b\times c$个点然后这题就是$\lceil \frac {a\times c-b\times c} b\rceil$，在c++中因为如果两边都是整型的话，自动向下取整，所以只需要在加上$b-1$就是向上取整了。

短代码:
```cpp
#include<stdio.h>
using namespace std;
int main()
{
	int a,b,c;
	scanf("%d%d%d",&a,&b,&c);
	printf("%d",(a*c-b*c+b-1)/b);
	return 0;
}
```

# 分析

这道题只用解一个非常简单的方程就可以了。但重点就在于向上取整。关于向上取整，有一个非常实用的函数 ceil 就可以了。但要注意的一点是：  ceil 函数对应的是 double 类型的变量，因此在定义变量时我们要注意一下。下面是完整代码。
# 代码
```cpp
#include<iostream>
#include<cmath>//ceil函数要用的头文件
using namespace std;
int main(){
    double a,b,c,t;//使用double来定义变量
    cin >> a >> b >> c;
    cout << ceil((a*c-b*c)/b)<<endl;//解方程并向上取整
    return 0;//结束
}
```

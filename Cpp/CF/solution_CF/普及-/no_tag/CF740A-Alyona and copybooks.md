# Alyona and copybooks

## 题目描述

Alyona 在学校学习 4 门课程，她要去书店买笔记本。她想让 4 门课程的笔记本数量相同。

Alyona 一开始时有 $n$ 个笔记本。书店有 $3$ 种笔记本购买方式：

 1. 买一个笔记本，花费 $a$ 卢布；
 2. 买两个笔记本，花费 $b$ 卢布；
 3. 买三个笔记本，花费 $c$ 卢布。
 
Alyona 可以购买任意次笔记本。问最少要花多少卢布？

## 样例 #1

### 输入

```
1 1 3 4
```

### 输出

```
3
```

## 样例 #2

### 输入

```
6 2 1 1
```

### 输出

```
1
```

## 样例 #3

### 输入

```
4 4 4 4
```

### 输出

```
0
```

## 样例 #4

### 输入

```
999999999 1000000000 1000000000 1000000000
```

### 输出

```
1000000000
```

这是一道比较简单的题目，只不过做的时候没有翻译，，，先解释一下题意


小A去买书，书店有三叠书，都是无限的，她现在有N本书，为了使四门科目都有相同数目的书，她需要买几本书。第一叠里一次只能买一本，花费A~~元~~卢布，第二叠一次只能买2本，花费B卢布，第三叠一次只能买3本，花费C卢布，问最少花多少钱。~~如果小A没有强迫症不就好了嘛~~

分析一下题目，我们可以轻易得出N可以直接%4取余数，如果是0就直接输出0，是1的话就比较一下各种情况的最小值，同理，2和3也一样，下面贴AC代码

```cpp
#include<bits/stdc++.h>
#define ll long long 
using namespace std;
ll minn (ll a, ll b, ll c)//为了方便比较三个数的最小值
{
	int x = (a > b ? b : a);
	x = (x > c ? c : x);
	return x;
}
int main ()
{
	ll n, a, b, c, ans = 0;
	scanf ("%lld%lld%lld%lld", &n, &a, &b, &c);
	if (!n % 4)//刚好就直接输出0
	{
		printf ("0");
		return 0;
	}
	switch (n%4)
	{
	case 1:ans = minn (a * 3, c, a + b); break;
	case 2:ans = minn (a * 2, b, c * 2); break;
	case 3:ans = minn (a, b + c, c * 3); break;//分析一下就可以得出只有这几种情况可能是最小的
	default:
		break;
	}
	printf ("%lld", ans);//朴素的输出
	return EXIT_SUCCESS;//收场
}
```

## 作者：LucasXu80 (赞：1)

来康康代码极短的Python~

先来分析一下题目。显然我们需要从$n$模$4$的余数下手，一个一个来看。

如果余$0$，显然不用买任何书，直接输出$0$即可。

如果余$1$，注意此时是差$3$本，而不是差$1$本。

这个时候有$3$中买书的方法：

第一种是买$3$套$1$本；

第二种是买$1$套$1$本，$1$套$2$本；

最后一种是买$1$套$3$本。

这里把三种方法分别简记为$3+0+0,1+1+0,0+0+1$。

计算总钱数的代码就很好写了，在完整代码中见吧。

继续，如果余$2$，有$2+0+0,0+1+0,0+0+2$三种方法。

值得一提的是$0+0+2$，虽然它多买了4本书，但是并不代表它一定更贵。所以这种情况还是要算的。

最后，如果余$3$，那么差$1$本。有：$1+0+0,0+1+1,0+0+3$三种方法。

思路就讲到这里，看代码！

```python
n,a,b,c=map(int,input().split()) #输入4个数
m=n%4 #计算余数
if m==0: print(0) #已经整除，直接输出0
if m==1: print(min(a*3,a+b,c)) #差3本
if m==2: print(min(a*2,b,c*2)) #差2本
if m==3: print(min(a,b+c,c*3)) #差1本
```
### 最后，一些FAQs：（大佬们请自动忽略）

Q1：为什么每一种情况都只有$3$种情况，不能更多了吗？

比如，$m=3$中，$1+2+0$不也是可以的吗？为什么不需要算？

A:但是这样的话，它的总钱数一定比$1+0+0$大，因为它多买了$2$套$2$本的数。其他情况也是如此，因此无需讨论。

Q2：求$a,b,c$的最小值，不需要写

```python
min(a,min(b,c))
```
吗？

A：**不需要**。PY中的min函数是无论多少项都可以计算的。

## 作者：bigju (赞：0)

### 题目大意

现在有一个数字 $n$ ，你有三种操作：

1. 使该数字$+1$，代价为 $a$ 。
2. 使该数字$+2$，代价为 $b$ 。
3. 使该数字$+3$，代价为 $c$ 。

你要用最小的代价使得 $n$ 能被 $4$ 整除。
### 大体思路

显然我们要先处理出 $n/4$ 的余数，然后再进行判断。

当余数为$1$时，也就是差$3$,那么我们有三种凑法是可能最优的：

1. 进行三个 $1$ 操作。
2. 进行一个 $1$ 操作和一个 $2 $操作。
3. 进行一个 $3$ 操作。

其余凑法都或多或少会有无用的操作。

进行比较即可

当余数为 $2$ 时，也就是差 $2$，很明显，我们用操作 $1$ 和操作 $2$ 是能够凑出来的。但是我们很容易忽略只用操作 $3$ 也是能凑出来的（进行两次操作 $3$ ）。

当余数为 $3$ 时同理，我们可以用一个 $1$ 。同样，很容易被忽略的是 $011$ （操作 $1$ 进行 $0$ 次，操作二 $1$ 次，操作三 $1$ 次）和 $003$ （同上解释）这两种凑法。

了解了以上的思路，代码就很简单了。

Code：
```cpp
#include<cstdio>
#include<iostream>
#include<algorithm>
#define ll long long
using namespace std;
ll n,a,b,c;//注意，范围达到了10^9，开longlong比较稳妥
int main()
{
    cin>>n>>a>>b>>c;
    if(n%4==0) return cout<<0,0;
    if(n%4==1) return cout<<min(min(a*3,a+b),c),0;
    if(n%4==2) return cout<<min(min(a*2,b),c*2),0;
    if(n%4==3) return cout<<min(min(a,b+c),c*3),0;
}

```

## 作者：xgwpp6710 (赞：0)

要买的总本数 $d$ ，就是 $4$ 减去 $n/4$ 的余数。

如果 $d=0$ ，最小花费为零。

如果 $d=1$ ，有以下三种买法：$100,011,003$ 。（$120$ 不用考虑，它一定比 $100$ 贵）

如果 $d=2$ ，则是 $010,200,002$。

如果 $d=3$ ，则是 $001,110,300$。

代码：
```cpp
#include<bits/stdc++.h>
using namespace std;
#define ll long long
int main()
{
	ll n,a,b,c,s;
	cin>>n>>a>>b>>c;
	ll d=4-n%4;
	if(d==1) s=min(a,min(b+c,c*3));
	if(d==2) s=min(b,min(a*2,c*2));
	if(d==3) s=min(a*3,min(a+b,c));
	if(d==4) s=0;
	cout<<s;
	return 0;
}
```

# MUH and House of Cards

## 题目描述

两只北极熊和一只大象聚在一起，打算用恰好 $n$ 张牌搭纸牌屋。合法的纸牌屋满足如下要求：

1. 屋子有至少一层**地板**(横向放置的纸牌，其下方要有纸牌支撑)。
2. 每层地板上有若干个**房屋**(两张纸牌相互倚靠搭成)和**天花板**(相邻房屋上方必须要有的横向纸牌,可当上一层的地板)。
3. 从下往上每层的房屋数量是递减的。

注意最上面一层地板上可以只有一个房间，也可以随意。

纸牌屋的高度定义为房间的层数(地板下面的不算)。现在它们想知道能搭出的纸牌屋中，不同高度的数量。

### 题解综合分析与结论
这两篇题解思路本质相同，都是通过分析纸牌屋每层所需纸牌数的规律，利用数学关系来解决问题。ZLCT题解从以三个纸牌为一组的角度出发，得出每多一层少用一个纸牌的规律，通过枚举高度 $h$，判断 $n + h$ 是否被 $3$ 整除且纸牌是否够用，同时对循环终止条件和枚举步长进行优化。_Kimi_题解则是先算出第 $k$ 层至少需要的纸牌数，进而推出 $n$ 层至少总共需要的纸牌数，同样通过枚举满足 $(n + i) \mid 3$ 的 $i$ 来求解。两篇题解思路清晰，但ZLCT题解在优化方面更详细，代码中体现了常数优化。

### 所选的题解
- **ZLCT题解**：
  - **星级**：4星
  - **关键亮点**：提出以三个纸牌为一组找规律，详细阐述暴力解法及优化思路，包括循环终止条件优化和常数优化。
  - **重点代码**：
```cpp
#include<bits/stdc++.h>
#define int long long
using namespace std;
int n,ans;
signed main(){
    cin>>n;
    for(int x=1;x<=n;++x){
        if((n+x)/3ll<(x+1ll)*x/2ll)break;
        if((n+x)%3ll==0){
            ans++;x+=2ll;
        }
    }
    cout<<ans<<'\n';
    return 0;
}
```
 核心实现思想：循环枚举高度 $x$，先判断当前高度所需最少纸牌数是否超过总纸牌数，若超过则跳出循环；若未超过且 $n + x$ 能被 $3$ 整除，则满足条件，答案加一，并将 $x$ 增加 $2$ 进行下一轮枚举。

- **_Kimi_题解**：
  - **星级**：3星
  - **关键亮点**：清晰推导出每层及总共至少需要的纸牌数公式，通过枚举满足条件的层数求解。
  - **重点代码**：
```cpp
#include <bits/stdc++.h>
#define int long long
using namespace std;
int n, ans;
signed main(){
    ios::sync_with_stdio(false);
    cin.tie(0);
    cin >> n;
    for (int i = 1; (3 * i + 1) * i / 2 <= n; i++){
        if ((n + i) % 3 == 0) ans++;
    }
    cout << ans << endl;
    return 0;
}
```
 核心实现思想：循环枚举层数 $i$，判断当前层数所需最少纸牌数是否不超过总纸牌数，若满足且 $n + i$ 能被 $3$ 整除，则答案加一。

### 最优关键思路或技巧
通过分析纸牌屋结构得出每层及总纸牌数与层数的数学关系，利用循环枚举层数并结合整除判断来求解，同时在枚举过程中通过设置合理的循环终止条件和步长进行优化。

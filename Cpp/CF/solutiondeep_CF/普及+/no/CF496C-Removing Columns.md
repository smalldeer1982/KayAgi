# Removing Columns

## 题目描述

给你一个$n$*$m$的小写字符矩阵，你可以选择删除一些列，使得剩下的字符矩阵的每一行的字符串从上到下的字典序非减。



求最少删除多少列

## 样例 #1

### 输入

```
1 10
codeforces
```

### 输出

```
0
```

## 样例 #2

### 输入

```
4 4
case
care
test
code
```

### 输出

```
2
```

## 样例 #3

### 输入

```
5 4
code
forc
esco
defo
rces
```

### 输出

```
4
```

### 综合分析与结论
这三道题解的核心思路都是对每一列进行遍历比较，判断是否需要删除该列以满足行字符串字典序非减的条件。
- **lukelin题解**：通过两个数组`is_not_same`和`is_not_same2`倒来倒去记录上一列排序相同元素的状态，判断当前列是否符合排列，若不符合则删除列数加一。
- **封禁用户题解**：两种实现方式，一种将字符转数字比较，一种直接用字符比较。从第一行开始逐行比较每列字符，若当前行字符小于上一行对应列字符，则删除该列并重新从第一行第一列开始比较。
- **enyyyyyyy题解**：通过标记数组`f`和标记变量`vis`判断每列是否需要删除，若当前行字符小于上一行对应列字符且该位置未被标记过，则标记并增加删除列数，若大于则标记该位置已遍历。

整体来看，三道题解思路较为直接，都是基于模拟比较过程来解题，但在代码实现的清晰程度和优化方面都有提升空间，均未达到4星标准，无需列出具体题解。

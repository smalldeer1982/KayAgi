# Cellular Network

## 题目描述

在直线上给出n个城市的位置(x坐标)和在同一直线上的m个蜂窝塔的位置(x坐标)。所有的塔都以同样的方式工作——它们为所有城市提供蜂窝网络，这些城市位于离塔不超过r的距离处才能被蜂窝网络覆盖。

你的任务是找出使得每个城市都能被蜂窝网络覆盖的最小r值，即每个城市在距离r的范围内至少有一个蜂窝塔。

如果r=0，则塔仅为其所在的位置提供蜂窝网络。一个塔可以为任意数量的城市提供蜂窝网络，但是所有这些城市都必须在距离塔不超过r的距离上。

## 说明/提示

1<=n,m<=10^5

-10^9<=a[i]<=10^9

-10^9<=b[j]<=10^9

## 样例 #1

### 输入

```
3 2
-2 2 4
-3 0
```

### 输出

```
4
```

## 样例 #2

### 输入

```
5 3
1 5 10 14 17
4 11 15
```

### 输出

```
3
```

# Cellular Network 蜂窝网络

## 题目描述
在直线上给定n个城市的位置和m个蜂窝塔的位置。所有塔的覆盖范围为半径r，要求找到能覆盖所有城市的最小r值。

## 输入输出要求
- 1<=n,m<=1e5
- 坐标范围[-1e9, 1e9]
- 输出最小半径r

## 样例
样例1输入：
3 2
-2 2 4
-3 0
输出：4（由塔-3覆盖城市-2，塔0覆盖城市2，塔0覆盖城市4需要r=4）

样例2输入：
5 3
1 5 10 14 17
4 11 15
输出：3（最远距离在1到4之间）

---

## 题解综合分析

### 最优解法：贪心+二分查找
**关键思路**：
1. 将城市和蜂窝塔分别排序
2. 对每个城市使用二分查找找到最近的左右两个塔
3. 计算该城市到这两个塔的最小距离
4. 所有城市的最小距离的最大值即为答案

**时间复杂度**：O(n log m)  
**空间复杂度**：O(1)

---

## 高星题解推荐

### 题解1：shucai（★★★★★）
**亮点**：
- 使用STL的lower_bound简化查找逻辑
- 清晰处理三种边界情况（塔全在左/右/两侧都有）
- 代码简洁高效

```cpp
sort(a, a+n); sort(b, b+m);
int ans = 0;
for(int i=0; i<n; i++) {
    int k = lower_bound(b, b+m, a[i]) - b;
    if(k >= m) ans = max(ans, a[i]-b[m-1]);
    else if(k == 0) ans = max(ans, b[0]-a[i]);
    else ans = max(ans, min(a[i]-b[k-1], b[k]-a[i]));
}
```

### 题解2：苏联小渣（★★★★☆）
**亮点**：
- 二分答案框架清晰
- check函数复用二分查找逻辑
- 处理大数时使用long long

```cpp
int l=0, r=2e9;
while(l <= r) {
    int mid = (l+r)/2;
    if(check(mid)) r = mid-1;
    else l = mid+1;
}
// check函数通过二分查找每个城市是否可覆盖
```

### 题解3：ImposterAnYu（★★★★☆）
**亮点**：
- 双指针优化降低常数
- 显式去重处理相同坐标
- 线性扫描代替二分查找

```cpp
sort(cities); sort(towers);
int j=1, ans=0;
for(int i=1; i<=n; i++) {
    while(tower[j+1] < city[i]) j++;
    ans = max(ans, min(dis(city[i], tower[j]), dis(city[i], tower[j+1])));
}
```

## 关键技巧总结
1. **排序预处理**：将城市和塔坐标排序是后续操作的基础
2. **二分边界处理**：使用lower_bound时需处理全左/全右的特殊情况
3. **贪心性质利用**：每个城市只需考虑最近的左右两个塔
4. **数值范围控制**：坐标差可能超过int范围，需使用long long

# 题目信息

# Om Nom and Candies

## 题目描述

桌子上有无限多的红糖果与蓝糖果，每个红糖果重 $ W_{r} $ 克，而每个蓝糖果重 $ W_{b} $ 克。吃一颗红糖果可以获得 $ H_{r} $ 的快乐值，吃一颗蓝糖果可以获得 $ H_{b} $ 的快乐值。

在只能吃 $ C $ 克糖果的前提下，请求出能获得的最大快乐值。

## 样例 #1

### 输入

```
10 3 5 2 3
```

### 输出

```
16
```

### 题解综合分析
各题解核心思路均为根号分治，通过分类讨论将枚举次数控制在 $O(\sqrt{C})$ 级别。关键优化点在于：当某糖果重量较大时直接枚举其数量；当两糖果均轻时比较性价比，仅枚举次优糖果的可能数量。

### 精选题解及评分

#### 1. 作者：run_away（4星）
**核心亮点**  
- 代码极度简洁，仅需单层循环同时处理两种糖果的枚举  
- 思路直观，直接枚举 $\sqrt{C}$ 次并同步计算两种糖果的最优组合  
**关键代码**  
```cpp
for(ll i=0;i<=sqrt(c);++i){
    if(i*wr<c) ans=max(ans,i*hr+(c-i*wr)/wb*hb);
    if(i*wb<c) ans=max(ans,i*hb+(c-i*wb)/wr*hr);
}
```
**实现思想**  
枚举 $0$ 到 $\sqrt{C}$ 次两种糖果的数量，每次计算剩余重量全选另一种糖果的情况，时间复杂度 $O(\sqrt{C})$。

#### 2. 作者：Halberd_Cease（4星）
**核心亮点**  
- 明确分治策略，严格按重量是否大于 $\sqrt{C}$ 分情况处理  
- 引入性价比比较，通过数学推导限制枚举范围  
**关键代码**  
```cpp
if (wr * wr >= c) { // 重量大直接枚举
    for (int i=0; i<=c/wr; i++) 
        ans = max(ans, i*hr + (c-i*wr)/wb*hb);
} else { // 均小时按性价比限制枚举次数
    if (hr*wb > hb*wr) swap(wr,wb), swap(hr,hb);
    for (int i=0; i<=wb; i++)
        ans = max(ans, i*hr + (c-i*wr)/wb*hb);
}
```
**实现思想**  
对重量大的糖果直接枚举，对均轻的情况通过性价比分析将枚举次数限制为较小重量值。

### 最优技巧总结
1. **根号分治策略**：根据物品重量与 $\sqrt{C}$ 的关系分情况处理，平衡枚举次数  
2. **性价比替代证明**：当两物品均轻时，通过比较单位重量价值，证明次优物品最多取到其重量值的个数  
3. **对称枚举优化**：同时处理两种糖果的枚举情况，避免重复代码

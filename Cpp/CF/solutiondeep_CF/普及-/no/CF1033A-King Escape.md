# King Escape

## 题目描述

艾丽丝和鲍勃正在一个n*n大棋盘上下棋。爱丽丝只剩下一个皇后，位于(a_x，a_y)，而鲍勃只有一个国王，位于(b_x，b_y)。爱丽丝认为，她的皇后是主宰棋盘的，所以胜利是属于她的。但是鲍勃已经制定了一个计划来赢得胜利，他需要移动国王到(c_x，c_y)，以便为自己争取胜利。当爱丽丝被她的自信所分心时，她将不会移动他的皇后，只有鲍勃才能进行移动。如果鲍勃能把他的国王从(b_x，b_y)移到(c_x，c_y)，他就会赢。请记住，国王可以移动到任何8个相邻的棋格。如果国王与皇后处于同一行、同一列或同一对角线，则将所到攻击。看看鲍勃能不能赢。

## 样例 #1

# King Escape

## 题目描述

艾丽丝和鲍勃正在一个n*n大棋盘上下棋。爱丽丝只剩下一个皇后，位于(a_x，a_y)，而鲍勃只有一个国王，位于(b_x，b_y)。鲍勃需要将国王移动到(c_x，c_y)，且国王只能移动到8个相邻格子。若国王与皇后处于同一行、列或对角线，则会被攻击。判断国王能否安全到达目标点。

## 算法分类
数学

---

### 题解综合分析
皇后的行、列攻击将棋盘分为四个独立区域，国王必须全程在起点与终点所在同一区域内移动。通过坐标相对位置判断起点和终点是否在同一区域。

---

### 精选题解

#### 1. 作者：yzh_Error404（5星）
**关键亮点**：通过四组条件直接判断象限，时间复杂度O(1)，代码极简。
```cpp
if(bx<ax&&by<ay&&cx<ax&&cy<ay) { /* 左下区域 */ }
else if(bx>ax&&by<ay&&cx>ax&&cy<ay) { /* 右下区域 */ }
else if(bx<ax&&by>ay&&cx<ax&&cy>ay) { /* 左上区域 */ }
else if(bx>ax&&by>ay&&cx>ax&&cy>ay) { /* 右上区域 */ }
```

#### 2. 作者：MasQx（5星）
**关键亮点**：象限判断条件与代码结构高度统一，注释清晰。
```cpp
if((bx<ax&&by<ay&&cx<ax&&cy<ay)||
   (bx<ax&&by>ay&&cx<ax&&cy>ay)|| 
   (bx>ax&&by<ay&&cx>ax&&cy<ay)||
   (bx>ax&&by>ay&&cx>ax&&cy>ay))
```

#### 3. 作者：洛璟（4星）
**实现思路**：用逻辑表达式合并判断条件，代码紧凑。
```cpp
if((bx<=ax && cx<ax && by>=ay && cy>ay) || 
   (bx>=ax && cx>ax && by>=ay && cy>ay) || 
   (bx<=ax && cx<ax && by<=ay && cy<ay) || 
   (bx>=ax && cx>ax && by<=ay && cy<ay))
```

---

### 关键思路总结
**坐标系象限判断**：以皇后坐标(ax, ay)为原点划分四个区域。起点和终点必须满足：  
1. x坐标同时小于或大于ax  
2. y坐标同时小于或大于ay  

**实现技巧**：通过位运算或逻辑表达式合并判断条件，避免冗余代码。

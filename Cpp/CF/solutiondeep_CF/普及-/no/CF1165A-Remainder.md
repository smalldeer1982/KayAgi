# 题目信息

# Remainder

## 题目描述

Aiming_High神仙给你一个由$n$位数字组成的数，且保证这个数字没有前导零，且每一位数字要么是$0$要么是$1$。



Aiming_High神仙需要你对这个数进行若干次操作（可能是$0$次）。每次操作中，你可以更改其中的任何数位，把这个数位上的数字从$1$变成$0$或从$0$变成$1$。操作后的数可能带有前导零，但并不影响这个问题。



Aiming_High神仙还给了你两个数字$x$，$y$。你要做的是最小化操作次数，使得操作后的数除以$10^x$的余数等于$10^y$。

## 样例 #1

### 输入

```
11 5 2
11010100101
```

### 输出

```
1
```

## 样例 #2

### 输入

```
11 5 1
11010100101
```

### 输出

```
3
```

### 综合分析与结论
这些题解的核心思路均基于数论中的取模运算性质，即一个数对$10^x$取模等价于取该数的后$x$位。要使操作后的数除以$10^x$的余数等于$10^y$，就需让后$x$位中只有第$y + 1$位（从右往左数）为$1$，其余位为$0$ 。然后通过遍历后$x$位，统计与该目标状态不同的位数，此即最少操作次数。各题解的主要差异在于代码实现细节，如数据读入方式、数组存储方式以及条件判断的具体写法等。

### 通用建议与扩展思路
通用建议：在实现过程中，注意边界条件的处理以及数组下标的对应关系，确保逻辑清晰，代码简洁。
扩展思路：类似题目可能会改变取模的基数，或者对目标余数有更复杂的要求。可以进一步思考如何将此思路拓展到其他进制下的类似问题，以及当目标余数不是简单的$10^y$形式时该如何解决。

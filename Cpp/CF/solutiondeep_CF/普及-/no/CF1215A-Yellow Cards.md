# Yellow Cards

## 题目描述

``Berland``足球杯总决赛最近举行了。整场比赛中，裁判出示了$n$张黄牌。比赛一开始，在第一队中有$a_1$个球员而在第二队中有$a_2$个。

在``Berland``足球杯中，把人罚下场的规则与正常规则略有不同。来自第一队的一个球员收到$k_1$张黄牌，他被罚下场并不能继续参加比赛了；来自第二队的球员则需要$k_2$张才会下场。下场的球员不能再收到黄牌。即使某一队（甚至两队）的球员全部被罚下场，比赛仍能继续。

裁判忘记了自己把每张黄牌罚给了谁，所以请你帮助裁判确定可能的被罚下场的人数的最小值和最大值。

## 说明/提示

在第一个样例中，可能一个球员都没被罚下场，所以最小值是0；最大值是4，1个来自第一队的球员和3个来自第二队的球员。

在第二个样例中，$n$达到了可能出示的黄牌数最大值（$3*6+1*7=25$），所以不论如何所有球员都得下场。

### 题解综合分析与结论
所有题解的核心思路都是分别考虑被罚下场人数最少和最多的情况。
 - **最小值思路**：让两队球员都先拿到离被罚下场差1张黄牌的数量，即第一队每人\(k_1 - 1\)张，第二队每人\(k_2 - 1\)张，计算此时总共需要的黄牌数\(a_1(k_1 - 1) + a_2(k_2 - 1)\)。若\(n\)小于等于该值，则最少罚下人数为0；否则，最少罚下人数为\(n - [a_1(k_1 - 1) + a_2(k_2 - 1)]\)。
 - **最大值思路**：采用贪心策略，优先罚下使球员被罚下场所需黄牌数较少的队伍（即比较\(k_1\)和\(k_2\)的大小）。先将该队球员全部罚下，再用剩余黄牌罚另一队球员，计算总共罚下的人数。

各题解在实现细节上略有不同，但整体思路一致。部分题解通过交换\(k_1\)、\(k_2\)以及\(a_1\)、\(a_2\)来简化后续计算逻辑。

 - **关键亮点**：思路清晰，对最少和最多罚下人数的策略阐述详细，代码简洁明了，使用了`std::max`、`std::min`等标准库函数，代码可读性高。
 - **重点代码核心实现思想**：计算最小值时，直接利用公式`std::max(0, n - (x - 1) * a - (y - 1) * b)`，确保结果非负；计算最大值时，通过遍历所有可能的黄牌分配情况（红队\(i\)张，蓝队\(n - i\)张），利用`std::max`和`std::min`函数计算每种情况下罚下人数的最大值。
```cpp
int main() {
    a = read(); b = read(); x = read(); y = read(); n = read();
    l = std::max(0, n - (x - 1) * a - (y - 1) * b);
    for (int i = 0; i <= n; i++) {
        r = std::max(r, std::min(a, i / x) + std::min(b, (n - i) / y));
    }
    printf("%d %d\n", l, r);
    return 0;
}
```

 - **重点代码核心实现思想**：先计算可以消耗的黄牌数`use = a1 * (k1 - 1) + a2 * (k2 - 1)`判断最少罚下人数；对于最多罚下人数，先比较\(k1\)和\(k2\)大小并交换相应变量，然后按不同情况（先整死一队，再整死另一队等）逐步计算罚下人数。
```cpp
int main(){
	scanf("%d%d%d%d%d",&a1,&a2,&k1,&k2,&n);
	int use=a1*(k1-1)+a2*(k2-1);
	if(use>=n)cout<<0<<" ";
	else cout<<n-use<<" ";
	if(k1>k2){
		swap(a1,a2);
		swap(k1,k2);
	}
	int killed=0;
	int p=0;
	if(n>=a1*k1){
		killed+=a1;
		n-=a1*k1;
		p=1;
	}
	if(n>=a2*k2){
		killed+=a2;
		n-=a2*k2;
		if(p==0)p=2;
		else p=3;
	}
	if(p==0||p==2){
		killed+=n/k1;
		n%=k1;
	}
	if(p==0||p==1){
		killed+=n/k2;
		n%=k2;
	}
	cout<<killed<<endl;
	return 0;
}
```

### 最优关键思路或技巧
 - **贪心策略**：在求最多罚下人数时，优先将黄牌罚给使球员被罚下场所需黄牌数较少的队伍，这是解决本题的关键贪心思路，能快速得到最大值。
 - **边界条件处理**：在计算最少罚下人数时，考虑到黄牌总数可能不足以让所有球员达到“饱和状态”，通过与0比较保证结果非负；在计算最多罚下人数时，注意队伍人数上限以及剩余黄牌数与另一队罚下条件的关系，合理使用`min`函数处理边界情况。

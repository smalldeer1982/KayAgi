# Perfect Team

## 题目描述

您可能已经知道标准的ICPC团队正好由三名成员组成。然而，完美的团队有更多的限制。学生可以有一些专业：编码或数学家。她/他可以没有专业化，但不能同时拥有两者。

因此，如果团队至少包括一名编码员，至少有一名数学家，并且由三名成员组成，那么团队就被认为是完美的。

你是一所非常大的大学的教练，你知道你的学生的$c$ 是编码员，$m$ 是数学家，$x$ 没有专业。

您可以分配给他们的最佳完整团队的最大数量是多少？

请注意，有些学生可能没有团队，每个学生都可以成为一个团队的一员。

您还被要求回答$q$ 个独立查询。

```
1 1 1

0 1 1
10 1 10

```

### 题解综合分析与结论
这些题解的核心思路都是围绕如何根据编码员（$c$）、数学家（$m$）和无专业学生（$x$）的数量来确定最多能组成的完美团队数量。主要有两种情况：一是团队数量受限于编码员和数学家中较少的一方，即 $\min(c,m)$；二是受总人数限制，即 $(c + m + x) / 3$。最终答案为这两者的较小值。不同题解在细节表述和推导过程上略有差异。

### 所选的题解
- **作者：Alex_Wei (4星)**
  - **关键亮点**：通过贪心策略，先尽量使用无专业学生，然后针对剩余编码员和数学家数量的不同关系进行分类讨论，思路清晰。
  - **个人心得**：无
  ```cpp
  #include<bits/stdc++.h>
  using namespace std;
  #define ll long long
  ll q,c,m,x;
  int main()
  {
      cin>>q;
      for(int i=0;i<q;i++){
          cin>>c>>m>>x;
          int ans=min(c,min(m,x));
          c-=ans,m-=ans;
          if(c*2<=m)cout<<ans+c<<endl;
          else if(m*2<=c)cout<<ans+m<<endl;
          else cout<<ans+(c+m)/3<<endl;
      }
      return 0;
  }
  ```
  - **核心实现思想**：先计算出能使用无专业学生组成团队的最大数量 `ans`，更新剩余编码员和数学家数量。然后根据剩余编码员和数学家数量关系，分别确定最终团队数量。

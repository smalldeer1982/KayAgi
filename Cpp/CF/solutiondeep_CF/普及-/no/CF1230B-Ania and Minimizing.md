# Ania and Minimizing

## 题目描述

Ania has a large integer $ S $ . Its decimal representation has length $ n $ and doesn't contain any leading zeroes. Ania is allowed to change at most $ k $ digits of $ S $ . She wants to do it in such a way that $ S $ still won't contain any leading zeroes and it'll be minimal possible. What integer will Ania finish with?

## 说明/提示

A number has leading zeroes if it consists of at least two digits and its first digit is $ 0 $ . For example, numbers $ 00 $ , $ 00069 $ and $ 0101 $ have leading zeroes, while $ 0 $ , $ 3000 $ and $ 1010 $ don't have leading zeroes.

## 样例 #1

### 输入

```
5 3
51528
```

### 输出

```
10028
```

## 样例 #2

### 输入

```
3 2
102
```

### 输出

```
100
```

## 样例 #3

### 输入

```
1 1
1
```

### 输出

```
0
```

# 阿妮亚与最小化

## 题目描述
阿妮亚有一个大整数 $S$ 。其十进制表示的长度为 $n$ ，且不包含任何前导零。阿妮亚最多可以改变 $S$ 的 $k$ 个数字 。她希望以这样一种方式进行改变，使得 $S$ 仍然不包含任何前导零，并且尽可能小。阿妮亚最终会得到什么整数？

## 说明/提示
如果一个数由至少两位数字组成且其第一位数字是 $0$ ，那么这个数就有前导零。例如，数字 $00$ 、 $00069$ 和 $0101$ 有前导零，而 $0$ 、 $3000$ 和 $1010$ 没有前导零。

### 综合分析与结论
这些题解的核心思路均为贪心算法，即通过从高位到低位修改数字，在满足无前导零的条件下，使修改后的数字尽可能小。
 - **思路方面**：都考虑到了从高位开始修改，最高位在 $n\neq1$ 时改为 $1$ ，其余位尽量改为 $0$ ，同时注意若某一位已经是最小则不修改以节省修改次数。对于 $n = 1$ 且 $k = 1$ 的情况，都进行了特判，直接输出 $0$ 。
 - **算法要点**：利用贪心策略，优先修改高位数字，以达到整体数字最小的目的。同时要合理处理修改次数，避免不必要的修改。
 - **解决难点**：主要难点在于对前导零的处理以及对特殊情况（$n = 1$ ）的特判。所有题解都通过特殊判断语句来解决这两个问题。

### 通用建议与扩展思路
 - **通用建议**：在实现贪心算法时，要清晰地确定贪心策略，即明确从高位到低位修改数字，最高位在非一位数时设为 $1$ ，其他位设为 $0$ ，并注意避免浪费修改次数。代码实现时要注意边界条件的处理，特别是 $n = 1$ 和 $k = 0$ 的情况。
 - **扩展思路**：类似题目可能会改变限制条件，比如允许出现前导零但有其他约束，或者修改次数的限制方式改变等。还可能将数字的进制进行改变，从十进制拓展到其他进制，解题时需根据新的条件调整贪心策略。

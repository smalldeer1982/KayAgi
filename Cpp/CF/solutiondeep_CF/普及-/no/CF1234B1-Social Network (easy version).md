# 题目信息

# Social Network (easy version)

## 题目描述

你知道了将来N次给你发送的消息，而聊天软件一次性只能显示与K(1≤K≤200)个人的聊天记录。

当收到一条消息时，如果与这个人的聊天记录显示在聊天软件中，就什么都不做（注意，不需要把当前聊天记录置顶）；否则，如果当前已经显示了K个聊天记录，则删除最后一个聊天记录；添加与这个人的聊天记录在列表顶端，同时其他聊天记录下移一个位置。

询问在这N条消息发送之后，你的聊天软件最终会显示与哪几个人的聊天记录。

## 样例 #1

### 输入

```
7 2
1 2 3 2 1 3 2
```

### 输出

```
2
2 1 
```

## 样例 #2

### 输入

```
10 4
2 3 3 1 1 2 1 2 3 3
```

### 输出

```
3
1 3 2 
```

# 社交网络（简单版）

## 题目描述
你知晓未来将会收到的N条消息，而聊天软件一次仅能显示与K（1≤K≤200）个人的聊天记录。
当收到一条消息时，若与发消息人的聊天记录已显示在聊天软件中，则无需操作（注意，无需将当前聊天记录置顶）；否则，若当前已显示了K条聊天记录，则删除最后一条聊天记录；将与发消息人的聊天记录添加至列表顶端，同时其他聊天记录下移一个位置。
询问在这N条消息接收完毕后，聊天软件最终显示的是与哪几个人的聊天记录。

### 综合分析与结论
这些题解的核心思路都是按照题目描述的规则进行模拟。主要差异在于使用的数据结构和判断元素是否已存在的方式。
 - **数据结构选择**：部分题解使用普通队列`queue`，部分使用双端队列`deque`。使用`queue`时，判断元素是否在队列中稍显复杂，需遍历队列；而`deque`可在队首插入、队尾删除，操作更贴合题意。
 - **元素存在判断**：有的利用`map`，有的使用`set`，还有直接用`bool`数组标记。`map`和`set`适用于数据范围较大且离散的情况，`bool`数组适合数据范围小且连续的情况。

### 通用建议与扩展思路
 - **数据结构选择**：根据题目具体操作和数据范围，合理选择数据结构能简化代码和提高效率。若仅需队首入队、队尾出队，`deque`是不错选择；若数据范围小且连续，`bool`数组标记更高效。
 - **优化方向**：对于大数据量情况，可考虑进一步优化，如减少不必要的遍历和查找操作。
 - **拓展题目类型**：类似题目可能会改变消息处理规则，如改变记录删除策略，或增加其他条件限制，解题时需灵活调整模拟逻辑。

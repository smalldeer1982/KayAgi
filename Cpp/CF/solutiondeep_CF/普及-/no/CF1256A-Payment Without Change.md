# Payment Without Change

## 题目描述

你身上有 $a$ 个 $n$ 元的硬币和 $b$ 个 $1$ 元的硬币。请问能不能在不找零的情况下购买 $s$ 元的物品。

### 输入

```
4
1 2 3 4
1 2 3 6
5 2 6 27
3 3 5 18
```

### 输出

```
YES
NO
NO
YES
```

### 综合分析与结论
这些题解的核心思路都是围绕能否用给定数量的 $n$ 元硬币和 $1$ 元硬币凑出目标金额 $s$。大多题解先考虑尽量用 $n$ 元硬币，再用 $1$ 元硬币补齐剩余金额。不同点在于判断条件的写法和逻辑顺序。

### 所选的题解
- **作者：傅思维666 (5星)**
    - **关键亮点**：思路清晰，先计算出 $n$ 面值硬币最多能凑出比 $S$ 小的金额，再与 $a$ 比较，最后结合 $b$ 判断能否凑出 $S$，代码简洁明了。
    - **重点代码**：
```cpp
#include<cstdio>
#include<algorithm>
using namespace std;
int T;
int a,b,n,s;
int main() 
{
    scanf("%d",&T);
    while(T--) 
    {
        scanf("%d%d%d%d",&a,&b,&n,&s);
        int num=s/n;
        num=min(num,a);
        if(num*n+b>=s) 
        {
            printf("YES\n");
            continue;
        }
        else 
        {
            printf("NO\n");
            continue;
        }
    }
    return 0;
}
```
    - **核心实现思想**：先计算 $s$ 除以 $n$ 的商作为 $n$ 元硬币最多使用数量，与 $a$ 取较小值，判断该数量 $n$ 元硬币与 $b$ 个 $1$ 元硬币总额是否不小于 $s$。

- **作者：oimaster (4星)**
    - **关键亮点**：思路直接，同样先确定 $n$ 元硬币最多使用量，再结合 $1$ 元硬币判断能否凑够 $s$，代码简洁紧凑。
    - **重点代码**：
```cpp
#include<iostream>
using namespace std;
int main(){
    int t;
    cin>>t;
    while(t--){
        int a,b,n,s;
        cin>>a>>b>>n>>s;
        if(min(s/n,a)*n+b>=s)
            cout<<"YES";
        else
            cout<<"NO";
        cout<<endl;
    }
}
```
    - **核心实现思想**：计算 $s$ 除以 $n$ 与 $a$ 的最小值，乘以 $n$ 后加上 $b$，判断是否不小于 $s$ 来决定能否凑出目标金额。

### 最优关键思路或技巧
优先使用大面值硬币（$n$ 元硬币），再用小面值硬币（$1$ 元硬币）补齐剩余金额，通过取商、取最小值等数学运算简洁地实现判断逻辑。

### 拓展思路
此类题目属于简单的硬币组合问题，类似的还有给定多种不同面值硬币及数量，求能否凑出特定金额，或者求凑出特定金额的最少硬币数量等。解题思路通常围绕对不同面值硬币的使用策略和判断条件的构建。

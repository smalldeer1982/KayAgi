# 重新粉刷街道

## 题目描述
有一条街道，$n$ 栋房子排成一行，编号从 $1$ 到 $n$。每个房子的颜色为 $c_i$ ，工人每次可粉刷连续的 $k$ 个房子，求出将整条街刷成同一个颜色的最短的次数。

------------

## 样例 #1
### 输入
```
3
10 2
1 1 2 2 1 1 2 2 2 1
7 1
1 2 3 4 5 6 7
10 3
1 3 3 3 3 1 2 1 3 3
```
### 输出
```
3
6
2
```
### 算法分类
贪心、枚举

### 综合分析与结论
这些题解的核心思路较为一致，均利用题目中颜色数量不超过100这一条件，通过枚举1到100的每种颜色，采用贪心策略计算将街道刷成该颜色所需的最少次数，最后取所有颜色所需次数的最小值作为答案。贪心策略为：从第一个房子开始遍历，若当前房子颜色与枚举颜色相同则跳过，不同则将当前房子及后续连续$k - 1$个房子刷成目标颜色，并将计数加一，同时直接跳到第$i + k$个房子继续遍历。所有题解都基于此思路实现，在代码实现细节上存在差异，如输入输出方式、变量命名、代码风格等。

for (register int i(1); i <= 100; ++i) {  //枚举都刷成每个颜色
    f = 0;                                  //清空

        if (i!= a[j]) {                  //不符合颜色要求
            ++f;                            //刷一天
            j += k;                         //连刷k个

            ++j;                            //忽略
        }
    }

}
```
核心思想为枚举每种颜色$i$，遍历房子数组，若当前房子颜色与$i$不同则增加粉刷次数并跳过$k$个房子，相同则跳过当前房子，最后取所有颜色粉刷次数的最小值。

### 最优关键思路或技巧
利用颜色范围小（$1 \leq c_i \leq 100$）这一条件进行枚举，采用贪心策略每次尽可能多地粉刷不符合颜色要求的房子，从而减少总的粉刷次数。这种贪心策略的核心在于每次操作都在满足当前需求的同时，对后续操作没有负面影响，保证了局部最优可推出全局最优。

# 题目信息

# Caesar Cipher

## 题目描述

输入一个由大写字母构成的字符串 $s$ ,求 $s$ 的每一个字母往后移 $k$ 位的结果。当字母移动后超出 $\texttt{Z}$ 时，回到字母表的开头  $\texttt{A}$ 继续移动。

## 样例 #1

### 输入

```
CODEFORCES
5
```

### 输出

```
HTIJKTWHJX
```

## 样例 #2

### 输入

```
WIXYZILWYM
6
```

### 输出

```
CODEFORCES
```

### 综合分析与结论
这些题解的核心思路都是对凯撒密码加密过程进行模拟。
 - **思路方面**：多数题解利用ASCII码特性，将字符与数字对应转换，处理移动和越界问题。部分通过字符减65（或'A'）使字符从0（或1）记数，加k后模26，再加回65（或'A'）得到加密字符；部分直接加k后判断是否大于'Z'，大于则减26。还有一种思路是构建包含26个大写字母的字符串，通过查找替换实现加密。
 - **算法要点**：关键在于理解字符与ASCII码的转换关系，以及如何处理字母移动超出'Z'的循环情况。
 - **解决难点**：主要难点是保证移动后的字符仍在大写字母范围内，不同题解通过取模运算或直接判断调整来解决。

### 通用建议与扩展思路
对于此类模拟题，关键是清晰理解题目规则，准确实现规则对应的逻辑。可以进一步拓展到更复杂的密码加密模拟，如维吉尼亚密码等，其涉及多轮不同偏移量的凯撒密码组合，有助于加深对字符处理和循环逻辑的掌握。

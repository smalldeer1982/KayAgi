# Matchmaker

## 题目描述

ygg 有 $n$ 只马克笔和 $m$ 个笔盖。我们可以使用二元组 $(a, b)$ 来描述一只笔或一个笔盖，其中 $a$ 表示颜色，$b$ 表示大小。任意一对笔和笔盖只有在**大小相同**的时候才可以配对，如果这一对笔和笔盖**大小相同且颜色相同**，我们就称这一对笔和笔盖为**优秀的配对**。  
现在 ygg 希望知道他最多可以构成多少**优秀的配对**，以及最多共有多少对配对。

## 样例 #1

### 输入

```
3 4
1 2
3 4
2 4
5 4
2 4
1 1
1 2
```

### 输出

```
3 2
```

## 样例 #2

### 输入

```
2 2
1 2
2 1
3 4
5 1
```

### 输出

```
1 0
```

### 题解综合分析与结论
这些题解的核心思路都是通过统计笔和笔盖的颜色与大小信息，来计算优秀配对和普通配对的数量。主要区别在于数据结构的选择和具体实现方式。
1. **桶数组方法**：多数题解使用桶数组来统计，如用一维数组统计特定大小的笔或笔盖数量，用二维数组统计特定颜色和大小的笔或笔盖数量。在输入笔和笔盖信息后，遍历笔盖信息，通过判断桶数组中对应位置的值是否大于0来确定是否能配对，并更新桶数组。
2. **哈希与map方法**：部分题解利用哈希思想，将颜色和大小组合成一个值作为map的键，统计符合条件的笔或笔盖数量。在输入笔盖信息时，同样计算哈希值并判断map中对应值是否大于0来确定配对情况，并更新map。

各题解在时间复杂度上，大部分为 $O(n)$，其中使用map的题解由于map操作的对数时间复杂度，整体为 $O(n\log n)$，但由于数据范围较小，实际运行效率差异不明显。空间复杂度上，使用二维数组的题解相对较大，为 $O(1000\times1000)$，而使用map的题解空间使用相对灵活，取决于实际数据情况。

### 通用建议与扩展思路
对于此类模拟问题，关键在于清晰梳理题目逻辑，选择合适的数据结构来存储和统计信息。可以进一步拓展的方向有：
1. **数据范围变化**：若颜色和大小的数据范围增大，需要考虑更高效的数据结构或算法，如动态开点线段树等。
2. **条件变化**：例如增加配对的其他限制条件，如重量等，这就需要在数据结构设计和匹配逻辑上做相应调整。

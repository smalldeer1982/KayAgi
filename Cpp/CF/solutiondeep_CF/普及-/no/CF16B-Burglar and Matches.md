# 题目信息

# Burglar and Matches

## 题目描述

一个窃贼到火柴仓库偷火柴,仓库有 $m$ 个容器,第 $i$ 个容器有 $a_i$ 个火柴盒,其中每个火柴盒中有 $b_i$ 根火柴,窃贼最多可以拿 $n$ 个火柴盒 。

## 说明/提示

#### 数据规模与约定

$ 1 \le n \le 2 \times 10^8$，$1 \le m \le 20$，$1 \le a_i \le 10^8$，$1 \le b_i \le 10$ 。

## 样例 #1

### 输入

```
7 3
5 10
2 5
3 6
```

### 输出

```
62
```

## 样例 #2

### 输入

```
3 3
1 3
2 2
3 1
```

### 输出

```
7
```

### 题解综合分析与结论
所有题解均采用贪心算法，核心思路是按照每个火柴盒中火柴数量从大到小排序，然后依次选取火柴盒，直到选取的火柴盒数量达到 $n$ 。各题解的主要区别在于代码实现细节，如数据结构的选择（结构体、pair 等）、循环的写法以及边界条件的处理方式。整体来看，思路较为直接，代码实现难度不大，但不同实现方式在代码简洁性和可读性上存在差异。

### 通用建议与扩展思路
 - **通用建议**：在实现贪心算法时，要清晰理解贪心策略的正确性，确保每次选择都是当前最优解。在代码实现上，注意边界条件的处理，以及数据结构的选择对代码简洁性和效率的影响。例如在本题中，合理使用结构体或 pair 来存储数据，并通过自定义比较函数进行排序，能使代码逻辑更清晰。
 - **扩展思路**：同类型题目通常围绕贪心策略展开，可能在数据规模、条件限制或贪心策略的复杂程度上有所变化。比如可以拓展到多维数据的贪心选择，或者在选择过程中增加更多的限制条件。类似的算法套路包括根据某个关键值进行排序，然后按顺序进行选择，在选择过程中动态调整剩余资源或条件。

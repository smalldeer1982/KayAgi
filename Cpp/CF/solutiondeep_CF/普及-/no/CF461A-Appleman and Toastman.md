# 题目信息

# Appleman and Toastman

## 题目描述

### 问题描述

苹果人和土司人正在玩游戏。一开始苹果人把包含 $n$ 个数的一组数给土司人，然后他们开始进行下面两个步骤：

- 每次土司人得到一组数，他把这些数的和加入到得分中，然后他把这组数交给苹果人。
- 每次苹果人得到只包含一个数的一组数，他会把这组数扔掉；每次苹果人得到包含至少两个数的一组数，他会任意的把它分成两个非空的组，并把这两组数分别交给土司人。

在这两个逗逼完成了所有的任务后他们会查看他们的分数。他们的最多能得多少分呢？

## 样例 #1

### 输入

```
3
3 1 5
```

### 输出

```
26
```

## 样例 #2

### 输入

```
1
10
```

### 输出

```
10
```

贪心

### 综合分析与结论
这些题解的核心思路都是贪心策略。由于要使得分最大，每次应扔掉最小的数，这样大的数就能更多次地参与求和。具体做法是先对数组排序，然后分析每个数被累加的次数，发现除最大数外，第 $i$ 小的数被累加 $i + 1$ 次，最大数被累加 $n$ 次。各题解在思路上基本一致，只是在代码实现和细节处理上略有不同，如循环的方式、变量的命名等。同时，所有题解都强调了要使用 `long long` 类型来避免数据溢出。

### 重点代码
```cpp
// 以LucasXu80的代码为例
#include <bits/stdc++.h>
using namespace std;
int main()
{
    long long n,a[300001];
    cin>>n;
    long long ans=0;
    for (int i=1; i<=n; i++) cin>>a[i];
    sort(a+1,a+n+1);
    for (i=1; i<=n; i++)
    {
        ans+=a[i]*(i+1);
    }
    cout<<ans-a[n];
    return 0;
}
```
**核心实现思想**：先输入数组，然后对数组进行排序，接着根据规律计算每个数乘以其被累加的次数并累加到答案中，最后减去多算的最大数一次，得到最终结果。

### 扩展思路
同类型题或类似算法套路：在很多贪心问题中，都需要根据题目条件找出最优的选择策略，比如在资源分配问题中，优先分配对结果贡献大的资源；在区间问题中，按照一定规则对区间进行排序和选择等。

# Complicated GCD

## 题目描述

【问题描述】 给你若干个整数，它们是a,a+1,a+2,…,b，请求出它们的最大公约数，即 gcd(a, a+1, a+2, …, b)。

## 样例 #1

### 输入

```
1 2
```

### 输出

```
1
```

## 样例 #2

### 输入

```
61803398874989484820458683436563811772030917980576 61803398874989484820458683436563811772030917980576
```

### 输出

```
61803398874989484820458683436563811772030917980576
```

### 综合分析与结论
- **思路**：各题解均围绕相邻自然数互质这一核心结论展开。当 `a < b` 时，区间 `[a, b]` 中必然存在相邻自然数，所以最大公约数为 1；当 `a == b` 时，最大公约数就是 `a` 本身。
- **算法要点**：利用相邻自然数互质的性质简化问题，避免了对区间内所有数进行复杂的最大公约数计算。由于数据范围大，采用字符串存储和比较的方式处理输入。
- **解决难点**：主要难点在于处理大整数（数据范围达到 `10^100`），各题解通过字符串存储和比较来解决这一问题。

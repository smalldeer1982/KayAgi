# 题目信息

# Godsend

## 题目描述

### 翻译

有两个人，他们面对一个数组，轮流取数组里的数， 第一个取的人要在这个数组里选择一个长度非零的子段，且其和为奇数，这个人要把他选定的子段删掉，并把原子段黏合。而第二个人的操作跟第一个人类似，只是和要是偶数。胜负的判定就是谁最先没法行动谁就输。

## 样例 #1

### 输入

```
4
1 3 2 3
```

### 输出

```
First
```

## 样例 #2

### 输入

```
2
2 2
```

### 输出

```
Second
```

### 综合分析与结论
这些题解的核心思路都是通过分析数组中奇数的个数来判断胜负。由于偶数不影响和的奇偶性，所以重点在于奇数的数量。若数组中没有奇数，先手无法行动，后手获胜；若有奇数，根据奇数个数的奇偶性，先手都有策略获胜。

各题解的思路基本一致，只是在代码实现和细节解释上有所不同。有的题解直接遍历数组找奇数，有的题解先进行奇偶性原理的推导再得出结论。

// 作者：封禁用户
#include<cstdio>
using namespace std;
inline int read(){
   long long s=0;bool w=1;char ch=getchar();
   while(ch<'0' || ch>'9'){if(ch=='-')w=-1;ch=getchar();}
   while(ch>='0' && ch<='9')s=s*10+ch-'0',ch=getchar();
   return s*w;
}
int n,a; 
int main()
{
	n=read();
	while(n--){
		a=read();
		if(a%2){//判断是否为奇数。
			printf("First");
			return 0;
		}
	}
	printf("Second");//如果没有奇数，则输出"Second"。
	return 0;
}

**核心实现思想**：遍历数组中的每个元素，判断其是否为奇数。若遇到奇数，则输出"First"并结束程序；若遍历完整个数组都没有奇数，则输出"Second"。

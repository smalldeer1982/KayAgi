# 题目信息

# Chess For Three

## 题目描述

A，B，C 三人下棋，A 和 B 先下，每次下完棋之后由现在观战的人（例如第一局就由 C）代替下输的人。
每次输入一个数表示谁赢了（A 是 $1$，B 是 $2$，C 是 $3$），如果每一次输入的赢家都不是当时旁观者，则输出 $\mathtt{YES}$，否则输出 $\mathtt{NO}$。

## 说明/提示

第一个样例顺序：  
A:B，A 胜  
A:C，A 胜   
A:B，B 胜 

第二个样例 B 输了第一场，可是他第二场赢了，矛盾，输出 $\mathtt{NO}$。

## 样例 #1

### 输入

```
3
1
1
2
```

### 输出

```
YES
```

## 样例 #2

### 输入

```
2
1
2
```

### 输出

```
NO
```

这些题解的核心思路都是模拟三人下棋的过程，通过记录当前的状态（如旁观者、对局者等），根据输入的赢家信息更新状态，并判断是否存在矛盾。
- **思路对比**：
    - 有的题解通过记录旁观者来模拟，如 rui_er 和 wzckd 的题解；有的题解通过记录对局者来模拟，如 BigJoker 和 xxzjbd 的题解；还有的题解用数组记录每个人的状态，如 Jekyll_Y 和 䒛夢 的题解。
- **算法要点**：

- **解决难点**：
    - 准确更新状态，避免逻辑错误。

【所选题解】
- rui_er（4星）：关键亮点在于通过判断原来的旁观者，使用另两个编号的和减去胜者得到负者编号，从而更新旁观者。
- wzckd（4星）：关键亮点是利用“观察者加两位比赛者的值都 = 6”的规律，直接计算下一轮观察者，简化了状态更新过程。

【重点代码】
- rui_er 的核心代码：
```cpp
int n, a, sp;
sp = 3;
for(int i=1;i<=n;i++) {
    scanf("%d", &a);
    if(a == sp) return puts("NO"), 0;
    if(sp == 1) sp = 5 - a;
    else if(sp == 2) sp = 4 - a;
    else sp = 3 - a;
}
puts("YES");
```
核心实现思想：记录当前旁观者编号 `sp`，每次输入赢家 `a`，若 `a` 等于 `sp` 则输出 `NO`，否则根据 `sp` 的值更新 `sp`。

- wzckd 的核心代码：

```cpp
long long ser=3,n,a;
for(int i=1;i<=n;i++){
    scanf("%lld",&a);

```
核心实现思想：记录当前观察者 `ser`，每次输入赢家 `a`，若 `a` 等于 `ser` 则输出 `NO`，否则利用 `ser = 6 - a - ser` 更新观察者。

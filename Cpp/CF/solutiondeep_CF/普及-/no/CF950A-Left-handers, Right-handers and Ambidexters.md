# Left-handers, Right-handers and Ambidexters

## 题目描述

你现在在一个运动队里。

这个队里有l 个人只能用他的左手训练（左撇子），另外有r 个人只能用右手训练（右撇子），还有a 个两只手都可以训练的人（既不是左撇子也不是右撇子，但只能用一只手）。

你们的教练决定组建一支队伍，但要保证队伍里用左手训练的人和用右手训练的人一样多。 你的任务是找到这个团队里最多能有多少个人。

### 综合分析与结论
这些题解的核心思路都是通过分类讨论，根据左撇子、右撇子和双手可用的人数关系来计算能组成的最大队伍人数。主要难点在于考虑各种情况，确保计算结果的正确性。不同题解在分类方式和代码实现上有所不同，但本质都是围绕如何合理分配双手可用的人来使左右人数相等且队伍人数最多。

```cpp
#include<bits/stdc++.h>
using namespace std;
long long a,b,c,k;
int main()
{
    cin>>a>>b>>c;//a:左撇子 b:右撇子 c:左右手都可以
    k=c;
    if(a<b)
    {
        k-=(b-a);
        if(k>=0) cout<<2*(b+(k/2));
        else cout<<2*(a+c);
    }
    if(b<a)
    {
        k-=(a-b);
        if(k>=0) cout<<2*(a+(k/2));
        else cout<<2*(b+c);
    }
    if(a==b) cout<<2*(a+(c/2));
    return 0;
}
```
    - **核心实现思想**：先判断左撇子和右撇子人数的大小关系，若不相等，先将双手可用的人补充到人数少的一方使其尽量与另一方相等，若还有剩余则平均分配；若相等则直接将双手可用的人平均分配，最后输出总人数。

#include<bits/stdc++.h>
using namespace std;
int a,b,c;
int main()
{
    cin>>a>>b>>c;
    if(a+c<=b||b+c<=a)

```
    - **核心实现思想**：若少的一方加上双手可用的人还是少于多的一方，则最大人数为少的一方加上双手可用的人的两倍；否则，最大人数为总人数除以2再乘2。

#include <bits/stdc++.h>
using namespace std;
int l, r, a;
int main() {

```

# Next Round

## 题目描述

# CF158  Next Round 下一轮   
# 题意简述  
有$n$个人参加一场比赛，其中分数排名前$k$位的人将被选入下一轮（选入下一轮的人分数必须为正，见样例#2），特别的，如果几个人分数相同且刚好并列处于第$k$名（或是并列$k-i$名，但是全部算入后选入下一轮的人数超过$k$人），这几个人都将被选入下一轮（见样例#1），题目要求你输出进入下一轮的人数。输入保证已经按分数从大到小排序。

## 说明/提示

在样例#1中，第五位和第六位参赛者都获得了七分，所以有六人进入下一轮。  
在样例#2中，没有人得到大于0的评分。  

翻译提供者：n0000000000o

## 样例 #1

### 输入

```
8 5
10 9 8 7 7 7 5 5
```

### 输出

```
6
```

## 样例 #2

### 输入

```
4 2
0 0 0 0
```

### 输出

```
0
```

# AI分析结果

### 题解综合分析与结论
这些题解的核心思路都是围绕找出晋级下一轮的人数，根据题目规则，晋级者分数需为正且大于等于第 $k$ 名的分数。各题解在具体实现方式上有所不同，主要可分为以下几类：
1. **顺序遍历法**：按顺序遍历分数数组，先处理前 $k$ 个分数，统计非零分数的人数，再处理 $k$ 之后的分数，若与第 $k$ 名分数相同则计入晋级人数。如Soshine_溯闪、BlueSu、打表大蒟蒻、Eason_AC的题解。
2. **条件判断法**：直接遍历整个数组，判断每个分数是否大于等于第 $k$ 名的分数且为正数，满足条件则计入晋级人数。如Chtholly_L、Konnyaku_Alan的题解。
3. **桶排序法**：使用桶来统计每个分数的人数，从高分到低分枚举，当累计人数达到或超过 $k$ 时停止。如houpingze的题解。
4. **排名统计法**：边输入边统计每个人的排名，然后根据排名和分数条件筛选出晋级人数。如GaryH的题解。

### 评分较高的题解
1. **Soshine_溯闪（5星）**
    - **关键亮点**：思路清晰，代码结构明确，将处理过程分为前 $k$ 个和 $k$ 之后两部分，注释详细，易于理解。
    - **核心代码**：
```cpp
for(int i=1;i<=k;i++) {
    if(a[i]>0) ans++;
    else {
        cout<<ans<<endl;
        return 0;
    }
}
for(int i=k+1;i<=n;i++){
    if(a[i]==a[k]) ans++;
    else{
        cout<<ans<<endl;
        return 0;
    }
}
cout<<ans<<endl;
```
    - **核心思想**：先处理前 $k$ 个分数，若分数为正则晋级人数加1，若为0则直接输出结果；再处理 $k$ 之后的分数，若与第 $k$ 名分数相同则晋级人数加1，不同则输出结果。
2. **Chtholly_L（4星）**

    - **核心思想**：遍历整个数组，判断每个分数是否大于等于第 $k$ 名的分数且为正数，满足条件则晋级人数加1。
3. **Konnyaku_Alan（4星）**

```cpp
x = a[k];
for ( int i = 1; i <= n; i++ )
    ans += x <= a[i] && a[i];
```
    - **核心思想**：先求出分数线（第 $k$ 名的分数），然后遍历数组，判断每个分数是否达到分数线且不为0，满足条件则晋级人数加1。

### 最优关键思路或技巧
- **顺序遍历与条件判断结合**：先处理前 $k$ 个分数，快速排除分数为0的情况，再处理 $k$ 之后的分数，判断是否并列，这种方式可以减少不必要的计算。
- **直接判断条件**：直接遍历整个数组，根据分数大于等于第 $k$ 名且为正数的条件统计晋级人数，代码简洁，逻辑清晰。

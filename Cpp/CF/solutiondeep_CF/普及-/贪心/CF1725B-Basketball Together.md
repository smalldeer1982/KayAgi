# Basketball Together

## 题目描述

在篮球比赛中，一个队的球员人数没有最大或最小限制（不一定每场比赛有 $5$ 名球员）。比赛中有 $N$ 名候选球员将由世界上最好的篮球教练 Pak Chanek 训练。第二个候选玩家具有 $P_i$ 的幂。

Pak Chanek 将从 $N$ 名候选球员中组成零支或多支球队，条件是每个候选球员最多只能加入一支球队。Pak Chanek 的每一支球队都将被派去与拥有 $D$ 力量的敌方球队进行一次比赛。在每一场比赛中，如果阵型球员的力量之和严格大于 $D$，则被派去的球队将被称为击败敌方球队。

Pak Chanek 的一项技能是，当一支已经组建的球队在比赛中比赛时，他可以改变球队中每个球员的力量，使之等于球队中最大的球员力量。

输出 Pak Chanek 可获得的最大获胜次数。

## 样例 #1

### 输入

```
6 180
90 80 70 60 50 100```

### 输出

```
2```

# 题解分析

## 最优思路总结
**核心贪心策略**：将球员按战斗力降序排列，从最大的战斗力开始，计算每个战力值对应需要的最少人数（`D/P_i +1`），累加人数直至超过总人数时终止。

**关键步骤**：
1. **排序降序**：使高战力优先用于组队
2. **人数计算**：严格大于要求导致公式中必须加1
3. **边界处理**：当累计人数超过N时立即终止

## 高星题解推荐

### 题解1：ztrztr（★★★★☆）
**亮点**：
- 使用倒序遍历简化逻辑
- 变量命名清晰（Count计数，sum累计人数）
- 代码简洁（仅15行核心逻辑）

```cpp
sort(P + 1, P + N + 1);
int Count = 0, sum = 0;
for (int i = N; i >= 1; i--) {
    sum += D / P[i] + 1;
    if (sum > N) break;
    Count++;
}
```

### 题解2：HNOIRPplusplus（★★★★☆）
**亮点**：
- 明确推导公式过程
- 包含变量越界判断（`n1<0`）
- 使用`cmp`函数增强可读性

```cpp
sort(p + 1, p + n + 1, cmp);
int n1 = n, ans = 0;
for (int i = 1; i <= n; i++) {
    n1 -= d / p[i] + 1;
    if (n1 < 0) break;
    ans++;
}
```

### 题解3：cyhtxdy（★★★★☆）
**亮点**：
- 使用`greater<int>()`简化排序
- 变量类型明确（`int now`累计人数）
- 循环条件处理优雅

```cpp
sort(p + 1, p + 1 + n, greater<int>());
int ans = 0, now = 0;
for (int i = 1; i <= n; i++) {
    now += d / p[i] + 1;
    if (now > n) break;
    ans++;
}
```

# 题解心得摘录
**调试经验**：
1. 多个题解强调必须使用`D/P_i +1`而非向上取整（如作者skcks指出两种情况）
2. 变量类型需用`long long`防溢出（如题解szhqwq代码）
3. 循环终止条件必须严格判断`sum > N`而非`>=`（ztrztr题解注释中特别说明）

**思维突破**：
- 作者HNOIRPplusplus指出贪心本质是"用最大的力量带尽可能多的弱者"

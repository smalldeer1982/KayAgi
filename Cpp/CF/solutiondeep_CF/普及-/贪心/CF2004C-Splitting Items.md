# Splitting Items

## 题目描述

Alice 和 Bob 有 $n$ 个数，第 $i$ 个数为 $a_i$，他们决定玩一个游戏取走这些数。

游戏由 Alice 开始取数。

每一次玩家都可以拿走一个剩下的数，直到没有数字可拿走。

定义 $A$ 是 Alice 获取的数字和，$B$ 是 Bob 获取的数字和，游戏总分 $p = A - B$。

Alice 希望最大化 $p$，Bob 希望最小化 $p$，他们都绝顶聪明。

现在 Bob 拥有了修改数的权限，可以把一些数字（可以没有，也可以没有全部）**增加**一个整数值（可以增加不同的值），但是这样 Alice 可能会起疑心，所以总增加的数值必须小于等于 $k$。

请求出 Bob 能达到的 $p$ 的最小值。

## 样例 #1

### 输入

```
4
2 5
1 10
3 0
10 15 12
4 6
3 1 2 4
2 4
6 9```

### 输出

```
4
13
0
0```

# 分割物品

## 题目描述

Alice 和 Bob 有 $n$ 个数，第 $i$ 个数为 $a_i$。他们决定通过游戏取走这些数，Alice 先手。每次玩家从剩余数中取一个，直到取完。

定义 $A$ 为 Alice 所取数之和，$B$ 为 Bob 所取数之和，游戏得分 $p = A - B$。Alice 希望最大化 $p$，Bob 希望最小化 $p$。Bob 可以修改数（只能增加），总增加量不超过 $k$。求 Bob 能达成的最小 $p$ 值。

---

### 综合分析与结论
各题解核心思路一致：降序排序后，Alice 取奇数位，Bob 取偶数位。Bob 通过增加偶数位的值（不超过前一个奇数位）来最小化分差。关键点在于处理每个偶数位的差值，并合理分配 $k$。

#### 关键思路
1. **排序策略**：降序排列确保交替取数的贪心策略成立。
2. **差值填补**：对每个偶数位，用 $k$ 填补其与前一个奇数位的差值，直到 $k$ 耗尽。
3. **顺序不变性**：修改后的偶数位值不能超过前一个奇数位，确保取数顺序不变。

---

### 精选题解

#### 题解1：Stars_Traveller（5星）
**亮点**：思路清晰，代码直观，完整处理每个偶数位的差值。
```cpp
sort(a+1,a+1+n,greater<int>());
for (int i=2; i<=n; i+=2) {
    if (k >= a[i-1]-a[i]) {
        k -= a[i-1]-a[i];
        a[i] = a[i-1];
    } else {
        a[i] += k;
        break;
    }
}
// 计算奇偶位和差
```

#### 题解2：RaymondOccam（4星）
**亮点**：代码结构清晰，变量命名明确，逐位处理逻辑完整。
```cpp
sort(a+1,a+n+1,greater<ll>());
for (ll i=2; i<=n; i+=2) {
    ll delta = a[i-1] - a[i];
    if (k >= delta) {
        a[i] += delta;
        k -= delta;
    } else {
        a[i] += k;
        k = 0;
    }
}
// 累加奇偶位差
```

#### 题解3：Scean_Tong（4星）
**亮点**：代码简洁，重点突出差值处理逻辑。
```cpp
sort(a+1,a+n+1,greater<int>());
for (int i=2; i<=n; i+=2) {
    if (k >= a[i-1]-a[i]) {
        k -= (a[i-1]-a[i]);
        a[i] = a[i-1];
    } else {
        a[i] += k;
        k = 0;
    }
}
// 计算总分差
```

# Shashlik Cooking

## 题目描述

### 题目大意：
有$n$个烤串，翻动第$i$根烤串会连带着$max(1,i-k)$到$min(n,i+k)$根烤串翻动，问把所有烤串都翻动一遍最少需要翻动几根烤串

## 样例 #1

### 输入

```
7 2
```

### 输出

```
2
1 6 
```

## 样例 #2

### 输入

```
5 1
```

### 输出

```
2
1 4 
```

### 题解分析与结论
这道题的核心思想是通过贪心策略，尽可能让每次翻动覆盖更多的烤串，从而减少总的翻动次数。以下是各题解的要点对比：

1. **Withershine的题解**：
   - 使用分块思想，将烤串分成若干块，每块长度为$2k+1$。
   - 通过计算块数和调整块的位置，确保每个烤串都被翻动一次。
   - 难点在于处理最后一块的长度不足$2k+1$的情况。
   - 评分：4星，思路清晰，代码可读性较好。

2. **small_turtle的题解**：
   - 采用贪心策略，直接计算需要翻动的次数和起始点。
   - 通过公式计算起始点，确保覆盖所有烤串。
   - 难点在于处理起始点为负数的情况。
   - 评分：4星，思路简洁，代码实现较为高效。

3. **flora715的题解**：
   - 使用贪心策略，从第一个烤串开始，每隔$2k$个烤串翻动一次。
   - 通过调整翻动点的位置，确保所有烤串都被翻动。
   - 难点在于处理最后剩余的烤串。
   - 评分：4星，思路清晰，代码实现较为简洁。

    p=(n-1)/(k*2+1)+1;
    printf("%d\n", p);
    
    l=max(n-((2*k+1)*(p-1)+k+1), 0);
    printf("%d ", ++l);

    {
        l+=k*2+1;

}

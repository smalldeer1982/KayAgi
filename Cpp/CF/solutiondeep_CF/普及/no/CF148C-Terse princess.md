# 题目信息

# Terse princess

## 题目描述

公主要选新郎啦！

公主想选一个最有钱的新郎，所以当她看到一个新郎比之前的新郎都有钱时，她会发出“哦”的叫声，而若她看到一个新郎比前面所有新郎加起来都富有时，她会发出“哇！”的声音（但此时她不会发出“哦”的声音）。在遇到第一个新郎时，公主不会说话。

## 样例 #1

### 输入

```
10 2 3
```

### 输出

```
5 1 3 6 16 35 46 4 200 99```

## 样例 #2

### 输入

```
5 0 0
```

### 输出

```
10 10 6 6 5```

# AI分析结果



### 中文重述题目  
**题目名称：吝啬的公主**  
公主想选一个最有钱的新郎。当她遇到一个比之前所有新郎都有钱的新郎时，会发出“哦”声；若遇到比之前所有新郎财富总和还多的，则发出“哇”声（此时不触发“哦”）。第一个新郎不会触发任何反应。要求构造一个长度为 $n$ 的数列，满足恰好触发 $a$ 次“哦”和 $b$ 次“哇”，且每个数不超过 $50000$。若无法构造，输出 $-1$。

### 算法分类  
构造（贪心策略）

---

### 题解综合分析  
各题解核心思路一致：优先构造“哇”的条件（严格递增和），再处理“哦”（递增），最后填充相同值。关键难点在于处理边界条件：  
1. **特判无“哇”但有“哦”时**：第二个数需与前一个相等，避免触发“哇”。  
2. **数值溢出控制**：每一步需检查是否超过50000。  
3. **最终验证**：所有触发条件需恰好满足。

---

### 高星题解推荐  
#### 1. PR_CYJ（★★★★☆）  
**关键亮点**：  
- 明确优先处理“哇”的逻辑，代码结构清晰。  
- 特判 `s1>0 && s2==0` 的情况，将第二个数设为1避免触发“哇”。  
- 严格检查数值溢出与最终条件是否满足。  
**代码核心**：  
```cpp
if (s1>0&&s2==0) {
    a[2]=1;  // 避免触发“哇”
    sum++;
    // 后续处理...
}
```
**个人心得**：作者提到在最终条件判断处调试良久，强调必须验证 `s1==0 && s2==0`。

#### 2. wkjwkj（★★★★☆）  
**关键亮点**：  
- 特判 `a≠0 && b=0` 时，前两数均为1。  
- 在填充“哦”时额外检查是否意外触发“哇”。  
**代码核心**：  
```cpp
if(a&&b==0) {
    s[1]=s[2]=1;  // 确保第二个数不触发“哇”

}
```

#### 3. WaterSky（★★★★☆）  
**关键亮点**：  
- 单独处理无“哇”场景，确保后续递增不会超过总和。

**代码核心**：  
```cpp
if(T[i]>sum) {  // 防止“哦”触发“哇”
    cout<<-1;
    return 0;
}
```

### 最优思路总结  
1. **贪心构造顺序**：先“哇”（总和+1），后“哦”（前值+1），最后填充1。  
2. **边界特判**：无“哇”时，前两数需相同。  
3. **数值控制与验证**：每一步检查是否超过50000，最终验证触发次数恰好满足。

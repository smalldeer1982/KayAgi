# Hometask

## 题目描述

给定 $n$ 个范围在 $[0,9]$ 的整数，求用这 $n$ 个数能拼出的被 $2,3,5$ 同时整除且没有前导 $0$ 的最大整数是多少？（可以不用完所有数字）

## 样例 #1

### 输入

```
1
0
```

### 输出

```
0
```

## 样例 #2

### 输入

```
11
3 4 5 4 5 3 5 3 4 4 0
```

### 输出

```
5554443330
```

## 样例 #3

### 输入

```
8
3 2 5 1 5 2 2 3
```

### 输出

```
-1
```

### 题解综合分析与结论

所有题解的核心思路一致：
1. **必须包含0**（否则直接输出-1）
2. **数字总和需为3的倍数**（通过删除最少数字调整余数）
3. **贪心构造最大数**（优先保留大数字，末尾补0）

最优思路提炼：
1. **余数调整策略**：总和余1时优先删除1个余1的数字，否则删2个余2的数字；余2时对称处理
2. **贪心实现**：删除时选择最小的数字，输出时从大到小排列
3. **特例处理**：全0时只输出一个0

### 高分题解推荐（≥4星）

#### 题解1：L_zaa_L（4星）
**关键亮点**：
- 清晰的余数分类处理逻辑
- 使用排序后标记删除实现贪心策略
- 包含全0特判

核心代码片段：
```cpp
sort(a+1,a+n+1); // 升序排序便于找最小删除数
if(sum%3!=0){
    int t=INT_MAX;
    // 尝试删除1个余数匹配的最小数字
    for(int i=1;i<=n;i++) if(a[i]%3 == sum%3) {
        a[i] = -1; break; 
    }
    if(t==INT_MAX){ // 否则删除2个互补余数的数字
        for(int i=1;i<=n && s<2;i++) 
            if(a[i]%3 != sum%3) a[i] = -1, s++;
    }
}
```

int b[10] = {0}; // 桶计数
if(sum%3==1){

### 最优关键技巧总结

1. **余数分组处理**：将数字按%3余数分组，通过删除最少元素调整总和余数
2. **桶计数优化**：用桶存储数字频率，快速查找余数匹配的最小删除数
3. **降序构造技巧**：直接按9->0顺序输出桶内元素，无需额外排序

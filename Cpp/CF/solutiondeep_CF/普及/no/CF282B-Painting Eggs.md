## 问题描述

有$n$ 个蛋，A和G需要给这$n$ 个鸡蛋涂色。A涂第$i$ 个鸡蛋得到的钱为$A_i$ ，G涂第$i$ 个鸡蛋得到的钱为$G_i$ ，且满足$A_i+G_i=1000$ 。需要把这$n$ 个鸡蛋分配给A和G，求怎样分配使得A和G得到的钱$S_A$ 和$S_G$ 的差不超过$500$ 。

## 样例 #1

### 输入

```
2
1 999
999 1
```

### 输出

```
AG
```

## 样例 #2

### 输入

```
3
400 600
400 600
400 600
```

### 输出

```
AGA
```

### 题解综合分析

所有题解均采用贪心策略：逐个处理鸡蛋，每次选择使得当前差值更小的分配方式。核心思路为：
1. 计算将当前鸡蛋分配给A或G后的新差值
2. 选择差值更小的分配方式
3. 利用题目条件$A_i+G_i=1000$保证解的存在性

**关键证明思路**：  
若前一步差值在$[-500,500]$范围内，分配后差值必然仍在此区间。因为每次分配的差值变化量$A_i - G_i$与$G_i - A_i$的和为0，且单次调整不超过1000，通过局部最优可保证全局合法。

```cpp
int main() {
    cin >> n;
    while (n--) {
        cin >> x >> y;
        if (abs(A + x - G) < abs(G + y - A)) { // 比较两种分配方式的差值
            A += x; cout << 'A';
        } else {
            G += y; cout << 'G';
        }
    }
}
```

### 关键技巧总结

1. **贪心选择条件**：通过计算两种分配方式后的绝对差值，选择较小者
2. **数学性质利用**：$A_i + G_i = 1000$确保每次调整后的差值变化不超过1000，从而保证合法解存在
3. **优化技巧**：使用`putchar`代替`cout`提升输出效率，快速读入处理大数据

# Making Sequences is Fun

## 题目描述

We'll define $ S(n) $ for positive integer $ n $ as follows: the number of the $ n $ 's digits in the decimal base. For example, $ S(893)=3 $ , $ S(114514)=6 $ .

You want to make a consecutive integer sequence starting from number $ m $ ( $ m,m+1,... $ ). But you need to pay $ S(n)·k $ to add the number $ n $ to the sequence.

You can spend a cost up to $ w $ , and you want to make the sequence as long as possible. Write a program that tells sequence's maximum length.

# 构造序列的乐趣

## 题目描述

定义正整数 $n$ 的 $S(n)$ 为它的十进制位数，例如 $S(893)=3$，$S(114514)=6$。

你需要构造一个从 $m$ 开始的连续整数序列（即 $m, m+1, m+2, \dots$），每个数 $n$ 加入序列需要花费 $S(n) \cdot k$。给定总预算 $w$，求能构造的最长序列长度。

## 样例 #1

### 输入
```
9 1 1
```

### 输出
```
9
```

## 样例 #2

### 输入
```
77 7 7
```

### 输出
```
7
```

### 题解分析与结论

#### 核心思路对比
1. **模拟法**（Velix、wind_whisper）：
   - 按数字位数分段处理，利用位数相同的区间花费相同的特性
   - 每次计算能跨越的最大位数区间，快速累加可选的数字个数
   - 时间复杂度：$O(\log_{10} m)$，效率极高

2. **二分法**（OcTar）：
   - 二分查找最大可达到的数字，通过预处理函数计算区间数字位数和
   - 需要处理数字位数计算的数学规律
   - 时间复杂度：$O(\log_2 (10^{18}) \times 16)$，效率较高但略逊于模拟法

#### 最优题解推荐

1. **wind_whisper（5星）**
   - **亮点**：代码简洁，预处理10的幂次表快速定位位数边界，逻辑清晰
   - **核心代码**：
     ```cpp
     ll o=calc(m),ans(0); // 初始位数
     while((mi[o+1]-m)*o <=w){
         ans += mi[o+1]-m;  // 累加当前区间的数字个数
         w -= (mi[o+1]-m)*o;
         ++o; m=mi[o];      // 跳转到下一位数的起点
     }
     ans +=w/o;             // 处理剩余预算
     ```

2. **Velix（4星）**
   - **亮点**：动态计算位数边界，无需预存表，内存更优
   - **关键实现**：通过循环逐步扩大位数区间，实时计算可容纳的数字个数

3. **OcTar（4星）**
   - **亮点**：二分思路直观，利用前缀和思想计算区间总花费
   - **注意点**：需处理大数计算时的溢出问题

### 关键技巧总结
- **位数分段法**：将数字按位数划分区间，利用相同位数区间的花费一致性快速计算
- **预算转换**：将总预算 $w$ 转换为 $w/k$ 简化问题，避免重复乘法运算
- **边界处理**：特别注意 $m$ 恰好是 $10^n$ 的情况（如样例4）

# Developing Skills

## 题目描述

Petya 正在玩一个游戏，主角有 $n$ 个技能，每个技能值 $a_i$ 在 $[0,100]$ 之间，当然技能值越高，技能越强；而主角的综合能力值为 $\sum\left \lfloor \dfrac{a_{i}}{10} \right \rfloor $，其中 $\left \lfloor\right \rfloor$ 为取整运算。

游戏开始时，Petya 有一个技能奖励包，包含 $k$ 点技能提升值，可以给任何一项技能提升一点技能值。现在编程计算 Petya 利用这个奖励包，可以把主角的综合能力值提升到的最大可能值。

注意：每个技能值最多能提升到 $100$。

## 样例 #1

### 输入

```
2 4
7 9
```

### 输出

```
2
```

## 样例 #2

### 输入

```
3 8
17 15 19
```

### 输出

```
5
```

## 样例 #3

### 输入

```
2 2
99 100
```

### 输出

```
20
```

## 题目描述

Petya 正在玩一个游戏，主角有 $n$ 个技能，每个技能值 $a_i$ 在 $[0,100]$ 之间。综合能力值为 $\sum\left \lfloor \dfrac{a_{i}}{10} \right \rfloor$。给定 $k$ 点技能提升值，可以将任意技能提升（不超过 100），求最大综合能力值。

## 算法分类
贪心

## 题解综合分析

### 核心思路对比
所有题解均采用贪心策略，核心差异在于实现方式：
1. **_HMZ_**：将技能按个位数从大到小排序，优先补满到下一个十位，剩余点数按每10点分配
2. **WA_sir**：类似思路但剩余点数处理存在潜在缺陷
3. **QWQ_123**：使用优先队列动态选取最优技能，实现更灵活但复杂度略高

### 最优题解推荐
#### [题解1] _HMZ_（4星）
**关键亮点**：
- 清晰的两次分配策略：先补余数后批量提升
- 严谨处理100的上限限制
- 代码可读性强，逻辑闭环

**核心代码**：
```cpp
sort(a + 1, a + 1 + n, cmp); // 按个位数从大到小排序
for (int i = 1; i <= n; i++) { // 补余数阶段
    if (a[i] == 100) continue;
    if (k >= 10 - a[i] % 10) {
        k -= 10 - a[i] % 10;
        a[i] += 10 - a[i] % 10;
    }
}
for (int i = 1; i <= n; i++) { // 批量提升阶段
    if (k >= 10) {
        int s = min(100, a[i] + k);
        k -= s - a[i];
        a[i] = s;
    }
}
```

## 关键思路总结
1. **贪心排序**：将技能按 `10 - (a_i % 10)` 从小到大排序（等价于按余数从大到小），优先处理能快速进位到下一个十位的技能
2. **两阶段分配**：
   - 第一阶段补余数：用最小代价使尽可能多的技能达到下一个十位
   - 第二阶段批量提升：将剩余点数以10为单位分配给各技能（不超过100）
3. **边界处理**：跳过已达100的技能，严格维护点数上限

# Login Verification

## 题目描述

登录名是由小写和大写拉丁字母、数字和下划线符号“_”组成的任意序列。
如果登录名与已经存在的登录名类似，则禁止注册。
如果我们可以通过以下类型的操作组成的序列将S转化成T，则认为S,T是类似的。
- 将小写字母转换为大写字母，将大写字母转换为小写字母
- 将字母“O”更改为数字“0”,将数字“0”更改为字母“O”
- "1""l""L"相互转化
例如，
登录名«codeforces»和«codefwrces»，
«oo0oo00oo0oo0oo0oo_lol»和«oo0oo0oo0oo0oo0oo0oo0oo0oo_1oo»是类似的，
而«codeforces»和«code_forces»则不是。
给你一个登录名的列表,其中没有两个类似的登录名。
检查新登录名是否和已有登录名类似。

# 登录验证（Login Verification）

## 题目描述
登录名由大小写拉丁字母、数字和下划线组成。若新登录名能通过以下操作转化为已有登录名，则视为类似：
1. 大小写互换
2. O与0互换
3. 1、l、L互换

给定不存在类似名称的已有列表，判断新登录名是否合法。

## 题解分析与结论
各题解核心思路均为**统一字符转换规则**：将所有可能变化的字符转换为统一形式后比较。关键点在于正确处理大小写转换和字符替换顺序。

### 最优题解亮点（评分≥4星）

#### 1. Miller2019（★★★★★）
**核心思路**：  
- 将所有字符转为小写  
- 将 o → 0，l/i → 1  
- 使用 vector 存储转换后的字符串，最后线性查找  
**关键代码**：
```cpp
string change(string str) {
    for(int t=0; t<str.size(); t++) {
        str[t] = tolower(str[t]);
        if(str[t] == 'o') str[t] = '0';
        else if(str[t] == 'l' || str[t] == 'i') str[t] = '1';
    }
    return str;
}
```
**亮点**：转换函数简洁高效，处理顺序合理（先统一大小写再替换字符）

#### 2. hensier（★★★★☆）
**核心思路**：  
- 转换时直接对输入字符串进行逐字符处理  
- 统一转为小写后替换特殊字符  
- 实时比较而非存储，节省空间  
**关键代码**：
```cpp
string change(string s) {
    for(int i=0; s[i]; i++) {
        s[i] = tolower(s[i]);
        if(s[i] == 'o') s[i] = '0';
        if(s[i] == 'l' || s[i] == 'i') s[i] = '1';
    }
    return s;
}
```
**亮点**：使用迭代器优化处理，代码紧凑易读

#### 3. LiteratureCollege（★★★★☆）
**核心思路**：  
- 统一转为大写  
- 处理 0 → O，1/L/I → 1  
- 手写比较函数提升效率  
**关键代码**：
```cpp
void k(string &x) {
    for(int j=0; j<x.length(); j++) {
        x[j] = toupper(x[j]);
        if(x[j] == '0') x[j] = 'O';
        if(x[j] == '1' || x[j] == 'L' || x[j] == 'I')

    }
}
```
**亮点**：通过引用修改字符串，避免拷贝开销

## 关键技巧总结
1. **统一转换规则**：必须将可能变化的字符映射到同一基准（如所有字符转小写/大写）
2. **处理顺序**：应先处理大小写转换，再处理特殊字符替换
3. **即时比对**：避免存储全部转换结果，实时处理每个字符串更节省空间
4. **注意原字符串处理**：容易遗漏对新登录名自身的转换（如 bigdoge 的踩坑经历）

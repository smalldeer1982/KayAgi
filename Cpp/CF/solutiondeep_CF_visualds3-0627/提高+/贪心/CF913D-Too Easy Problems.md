# Too Easy Problems

## 题目描述

你正在准备一场关于调度理论的考试。

这场考试会持续正好 $T$ 毫秒，由 $n$ 道题目组成。

你可以用 $t_i$ 毫秒解决第 $i$ 个问题，或者忽略它并不消耗时间。你也不需要用来在做完一道题之后休息的时间。

不幸的是，你的老师认为一些题目对你来说太简单了。因此，他对于每道题 $i$ 规定了一个整数 $a_i$，表示题目 $i$ 只在你总共解决了不超过 $a_i$ 个问题（包括问题 $i$ ）的情况下为你的最终成绩加上一分。

正式地，假设你在考试中解决了问题 $p_1,p_2,\cdots,p_k$。那么，你的最终成绩 $s$ 会等于在 $1$ 到 $k$ 之间的满足 $k\le a_{p_j}$ 的 $j$ 的个数。

你已经意识到这场考试真正的第一道题目已经放在了你面前。因此，你想要选择一组题目来解决，从而最大化你的最终成绩。不要忘记这场考试有时间限制，而你必须有足够的时间来解决所有你选择的题目。如果存在多个最优解，任意输出一组即可。

## 说明/提示

$1\le n\le 2\times10^5$

$1\le T\le10^9$

$0\le k\le n$

### **综合分析与结论**

**题目核心难点**：在时间限制内选择问题，使得满足每个选中问题的 `a_i ≥ k`（k为总题数），并最大化k。关键在于发现答案k的单调性，结合贪心选择最小耗时的问题。

**贪心策略核心**：  
1. **二分答案框架**：确定最大k值，使得存在k个问题满足 `a_i ≥k`，且总时间≤T。  
2. **贪心验证**：对每个候选k，筛选所有 `a_i ≥k` 的问题，按时间从小到大选前k个，检查总耗时。  

**可视化设计思路**：  
- **动画演示**：分步展示二分过程（l/r指针变化），每次mid检查时筛选并排序问题，高亮选中项，显示总耗时。  
- **复古像素风格**：用色块表示问题（颜色深浅表示a_i大小，宽度表示耗时），选中时闪烁，背景音乐随k值变化。  
- **交互功能**：手动调整二分步进，或自动播放；音效提示成功/失败（如选中k有效时播放“升级”音效）。

### **核心代码实现**

**Sol1的贪心验证函数**：  
```cpp
bool Chk(int x) {
    int tmp = t, scr = 0;
    for (int i = 1; i <= n; i++) {
        if (tmp < p[i].t) break;
        if (p[i].a < x) continue;
        scr++;
        tmp -= p[i].t;
    }
    return scr >= x;
}
```

**关键点**：  
- 预处理所有问题按时间排序，验证时只需线性扫描。  
- 累计满足 `a_i ≥x` 的问题，直到数量达到x或时间耗尽。

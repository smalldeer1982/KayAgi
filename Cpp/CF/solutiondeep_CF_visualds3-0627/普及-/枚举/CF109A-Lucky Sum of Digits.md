# 题目信息

# Lucky Sum of Digits

## 题目描述

Petya喜欢幸运数字。我们都知道幸运数字是只含有$4$ 和$7$ 的正整数。例如数字$47$ ，$744$ ，$4$ 是幸运数字，而$5$ ，$17$ ，$467$ 不是。

Petya急切的想知道各个数位上数字的和为$n$ 的最小幸运数字是多少。帮助他完成这个任务。

## 样例 #1

### 输入

```
11
```

### 输出

```
47
```

## 样例 #2

### 输入

```
10
```

### 输出

```
-1
```

## 综合分析与结论

### 核心思路
题目要求构造各位和为 `n` 的最小幸运数（仅含4和7）。关键点：
1. **最少位数**：用尽可能多的7（每位贡献大），减少总位数。
2. **字典序最小**：在位数相同的情况下，4排在7前。

### 最优解法
1. **贪心策略**：从最大可能的7数量开始枚举，检查剩余部分是否能被4整除。
2. **实现步骤**：
   - 枚举7的数量 `i`（从 `n/7` 到0）。
   - 计算剩余值 `remain = n - 7*i`。
   - 若 `remain` 能被4整除，则构造 `(remain/4)个4 + i个7`。
   - 找不到则输出-1。

### 难点与解决方案
- **总位数最少**：优先用7最大化每位贡献。
- **字典序最小**：构造时先放4再放7。
- **无解判断**：遍历所有可能的7数量后仍未找到解。

### 可视化设计
- **动画方案**：动态展示枚举7的数量，计算剩余值并检查整除性。成功时高亮组合。
- **像素风格**：8位复古UI，步进显示枚举过程，音效提示成功/失败。
- **交互功能**：步进控制、速度调节，对比不同算法效率。

# 题目信息

# Pineapple Incident

## 题目描述

Ted有一个菠萝。这个菠萝可以像狗一样叫！在第 $t$ 秒它第一次叫。然后每隔 $s$ 秒之后它会叫一次，在叫这一次之后还会在后一秒再叫一次。意思就是说它会在 $t$ 秒，$t+s$ 秒，$t+s+1$ 秒，$t+2s$ 秒，$t+2s+1$ 秒分别叫一次，以此类推。 

Barney在早上醒来，他想吃个菠萝，但这只菠萝叫的时候Barney不能吃它。Barney打算在 $x$ 秒吃它，所以请你告诉Barney在x秒的时候，这个菠萝会不会叫。

## 样例 #1

### 输入

```
3 10 4
```

### 输出

```
NO
```

## 样例 #2

### 输入

```
3 10 3
```

### 输出

```
YES
```

## 样例 #3

### 输入

```
3 8 51
```

### 输出

```
YES
```

## 样例 #4

### 输入

```
3 8 52
```

### 输出

```
YES
```

**综合分析与结论**  
**核心思路**  
菠萝叫的时间点为 `t, t+s, t+s+1, t+2s, t+2s+1, ...`。判断 `x` 是否等于 `t`，或满足 `x = t + k*s` 或 `x = t + k*s +1`（`k ≥1`）。关键点在于：  
1. `x < t` → 直接返回 NO  
2. `x == t` → 返回 YES  
3. 若 `x > t`，需检查 `x-t` 是否为 `s` 的倍数，或 `x-t-1` 是 `s` 的倍数且 `x-t ≠1`  

**解决难点**  
- **边界条件**：`x = t+1` 时，只有当 `s=1` 时才可能合法，但题目明确此情况不合法，需特殊处理。  
- **数学推导**：通过模运算判断 `x` 是否落在叫的时间点，避免暴力枚举超时。  

**可视化设计**  
- **动画演示**：以时间轴显示 `t` 起始的叫点序列，高亮当前检查的 `x` 位置，颜色标记是否符合条件（绿色为 YES，红色为 NO）。  
- **步进控制**：单步展示模运算逻辑，动态显示 `x-t` 的分解过程。

**代码实现（核心逻辑）**  
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int t, s, x;
    cin >> t >> s >> x;
    if (x == t) cout << "YES";
    else if (x < t) cout << "NO";
    else {
        int d = x - t;
        if (d % s == 0 || (d % s == 1 && d != 1)) cout << "YES";
        else cout << "NO";
    }
    return 0;
}
```

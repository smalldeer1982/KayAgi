# Product of Binary Decimals

## 题目描述

### 二进制小数的乘积
我们称一个数字为二进制小数，如果它是一个正整数，并且其十进制表示中的所有数字都是0或1。例如，$1010111$ 是一个二进制小数，而 $10201$ 和 $787788$ 不是。

给定一个数 $n$，你被要求判断是否可能将 $n$ 表示为一些（不一定是不同的）二进制小数的乘积。

## 说明/提示

前五个测试用例可以表示为二进制小数的乘积如下：

$121 = 11 \times 11$
$1 = 1$ 已经是一个二进制小数。
$14641 = 11 \times 11 \times 11 \times 11$
$12221 = 11 \times 11 \times 101$
$10110 = 10110$ 已经是一个二进制小数。

## 综合分析与结论
### 核心思路与难点
题目要求判断一个数能否表示为若干个二进制小数（仅含 0/1 的正整数）的乘积。核心思路是**预处理所有可能的二进制小数**，并通过递归或动态规划进行因数分解。难点在于：

1. **二进制小数的生成**：需覆盖 1e5 范围内所有可能值（如 10、11、100 等）
2. **高效分解策略**：DFS 试除或预处理标记合法数
3. **特殊边界处理**：如末尾 0 的优化（10 的幂次分解）

### 算法流程可视化设计
- **动画方案**：以树状结构展示分解过程，每个节点表示当前待分解数，分支为尝试的二进制小数因子。高亮当前尝试的因子（如红色边框）和分解后的商（绿色填充）。
- **交互控制**：支持单步执行观察递归路径，回溯时显示灰色节点。当分解到 1 时播放成功音效。
- **像素风格**：用 8-bit 风格显示数字和因子，背景音乐为低比特循环音轨，关键操作触发短促电子音效。

## 最优思路与技巧
### 关键实现代码
```cpp
// 预处理因子 + DFS 分解（Pink_Cut_Tree 核心逻辑）
int b[] = {10,11,100,101,...,11111}; // 所有 1e5 内二进制小数
void dfs(int x) {
    if (x == 1) { ans = 1; return; }
    for (auto factor : b) {
        if (x % factor == 0) dfs(x / factor);
    }
}
```

### 核心技巧
1. **二进制小数预生成**：通过位运算或字符串生成所有可能值
2. **末尾 0 优化**：预处理时优先分解 10 的因子，如 `n /= 10` 直到无法整除
3. **剪枝策略**：DFS 中优先尝试大因子减少递归深度

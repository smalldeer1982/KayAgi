# 题目信息

# Lucky Sum of Digits

## 题目描述

Petya喜欢幸运数字。我们都知道幸运数字是只含有$4$ 和$7$ 的正整数。例如数字$47$ ，$744$ ，$4$ 是幸运数字，而$5$ ，$17$ ，$467$ 不是。

Petya急切的想知道各个数位上数字的和为$n$ 的最小幸运数字是多少。帮助他完成这个任务。

## 样例 #1

### 输入

```
11
```

### 输出

```
47
```

## 样例 #2

### 输入

```
10
```

### 输出

```
-1
```

## 综合分析与结论

### 题解思路与核心难点
**核心目标**：构造由 4 和 7 组成的最小数字，满足数位和为 n。  
**关键难点**：  
1. **最小位数**：优先使用更多 7（每位贡献更大数位和），减少总位数。
2. **字典序最小**：在相同位数下，将 4 尽量放在高位。

**最优解法**：  
从最大的 7 的数量开始枚举，检查剩余数是否能被 4 整除。第一个满足条件的解即为总位数最少且字典序最小的解。

### 可视化设计思路
- **动画方案**：以像素风格展示枚举过程。每个候选的 7 数量（j）显示为红色方块，剩余数（n-7j）显示为绿色方块。若剩余数为 4 的倍数，方块变为金色，并触发音效。
- **步进控制**：允许手动调整枚举速度，观察关键步骤。
- **颜色标记**：当前操作的 7 数量高亮为红色，满足条件时高亮为金色。

## 最优技巧提炼
**贪心策略**：  
1. **优先使用 7**：每位 7 贡献更大数位和，减少总位数。
2. **高位优先**：构造数字时，先输出 4 再输出 7，保证字典序最小。

**代码实现关键**：  
- **反向枚举**：从大到小枚举 7 的数量，确保找到第一个可行解即最优。
- **快速判断**：检查剩余数能否被 4 整除，时间复杂度 O(n/7)。

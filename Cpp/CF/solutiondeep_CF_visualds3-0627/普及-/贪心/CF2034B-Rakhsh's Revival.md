# Rakhsh's Revival

## 题目描述

### 题目翻译：

给定一个长度为 `n` 的二进制字符串 `s`，其中 `0` 表示弱点，`1` 表示强点。需要确保任意长度为 `m` 的连续区间内至少有一个强点。可以使用一种特殊能力 **Timar**，它能将任意长度为 `k` 的区间内的所有点变为强点（即 `1`）。求解需要使用 **Timar** 的最小次数，使得字符串 `s` 中任意长度为 `m` 的连续区间都至少包含一个 `1`。

## 样例 #1

### 输入

```
3
5 1 1
10101
5 2 1
10101
6 3 2
000000```

### 输出

```
2
0
1```

# 综合分析与结论

## 核心算法流程
所有题解均采用贪心策略，核心逻辑如下：
1. 遍历字符串，维护连续 0 的计数器 `cnt`
2. 当 `cnt` 达到 m 时触发一次操作，将后续 k 个位置强制设为 1
3. 跳跃处理：触发操作后，直接跳过后续的 k-1 个位置（因为已经被覆盖）
4. 操作次数计数器 `ans` 递增

**可视化设计要点**：
- **动画方案**：用两行像素块表示原始字符串和操作后的字符串。当触发操作时，覆盖区域用闪烁特效，跳跃位置用箭头标记。
- **颜色标记**：红色表示当前检测的连续 0 段，黄色表示被覆盖的区间，绿色表示已处理的安全区域。
- **步进控制**：允许单步执行观察 cnt 递增和跳跃逻辑，支持回退功能检验贪心决策的正确性。
- **音效提示**：在触发操作时播放短促的「咔嚓」音效，跳跃时播放滑过音效。

# 最优思路与技巧

## 关键实现代码
```cpp
int cnt = 0, ans = 0;
for (int i = 0; i < n; ) {
    if (s[i] == '0') {
        cnt++;
        if (cnt == m) {         // 触发操作
            ans++;
            cnt = 0;
            i += k;             // 跳跃 k 步（处理i+k-1的越界）
        } else i++;
    } else {
        cnt = 0;
        i++;
    }
}
```

**核心技巧**：
1. **贪心跳跃**：操作后直接跳过 k 步，将 O(nk) 暴力覆盖优化为 O(n)
2. **连续检测**：仅需维护一个计数器，无需复杂数据结构
3. **覆盖右端点**：每次操作覆盖当前 m 个 0 的最右端，最大化后续收益

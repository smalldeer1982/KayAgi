# Sports Betting

## 题目描述

不同航班的登机过程可能以不同方式进行：要么通过巴士，要么通过伸缩式登机桥。每天，圣彼得堡到明斯克的航班恰好有一班，而 Vadim 决定向学生们证明他总能提前知道登机方式。

Vadim 与 $n$ 名学生打赌，与第 $i$ 名学生的赌约是在第 $a_i$ 天。若 Vadim 正确预测了第 $a_i+1$ 天和第 $a_i+2$ 天的登机方式，则他赢得赌约。

尽管 Vadim 并不知道登机方式会如何发生，但他非常希望至少赢得一名学生的赌约，以此说服对方相信他的预测能力。请判断是否存在一种策略，使得 Vadim 能够确保成功。

## 说明/提示

在第一个测试用例中，Vadim 需要至少正确预测第 2 天和第 3 天的登机方式之一。这两天共有 $4$ 种可能的登机场景，因此 Vadim 可以为所有 $4$ 名学生提供不同的预测组合，从而确保至少有一名学生会被说服。

在第二个测试用例中，Vadim 仅与三名学生打赌，无法保证能为其中至少一名提供正确的预测。

## 综合分析与结论

### 题解思路与核心难点
本题的核心在于判断是否存在一种策略，使得Vadim的预测能覆盖所有可能的四种登机方式组合（桥-桥、桥-巴士、巴士-桥、巴士-巴士）。关键在于如何通过学生的时间段分布覆盖所有可能性：

1. **覆盖条件**  
   - 若某个时间段（如第3-4天）有至少4个学生，Vadim可为每个学生分配不同的预测组合，覆盖所有四种可能。
   - 若存在两个不同时间段，每个时间段至少有两个学生，则总共有四组预测，可以覆盖四个组合。

2. **难点与解决方案**  
   - **时间段的独立性**：每个学生的预测仅针对其对应时间段，需通过统计频率判断覆盖可能性。
   - **组合覆盖**：通过频率统计，判断是否存在足够的学生分布满足覆盖条件。

### 可视化设计思路
1. **颜色标记**：在频率统计阶段，高亮出现次数≥4或≥2的数值，突出关键条件。
2. **动画流程**：逐步统计每个学生的`a_i`值，动态更新频率表，最终标出满足条件的`a_i`值。

```cpp
#include <iostream>
#include <unordered_map>
using namespace std;

bool canWin(int n, vector<int>& a) {
    unordered_map<int, int> freq;
    for (int num : a) freq[num]++;
    
    // 检查是否存在出现≥4次的a_i
    for (auto& [k, v] : freq) {
        if (v >= 4) return true;
    }
    
    // 统计出现≥2次的a_i的数量
    int count = 0;
    for (auto& [k, v] : freq) {
        if (v >= 2) count++;
        if (count >= 2) return true;
    }
    return false;
}

int main() {
    int t;
    cin >> t;
    while (t--) {
        int n;
        cin >> n;
        vector<int> a(n);
        for (int i = 0; i < n; i++) cin >> a[i];
        cout << (canWin(n, a) ? "Yes" : "No") << endl;
    }
    return 0;
}
```

## 最优思路提炼
1. **频率统计法**：统计每个`a_i`的出现次数，判断是否存在以下任一条件：
   - 存在某个`a_i`出现≥4次。
   - 存在至少两个不同的`a_i`各出现≥2次。
2. **组合覆盖**：通过多个时间段的组合覆盖所有可能性，确保至少有一个预测正确。

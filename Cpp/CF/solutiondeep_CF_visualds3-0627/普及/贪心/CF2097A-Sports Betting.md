# 题目信息

# Sports Betting

## 题目描述

不同航班的登机过程可能以不同方式进行：要么通过巴士，要么通过伸缩式登机桥。每天，圣彼得堡到明斯克的航班恰好有一班，而 Vadim 决定向学生们证明他总能提前知道登机方式。

Vadim 与 $n$ 名学生打赌，与第 $i$ 名学生的赌约是在第 $a_i$ 天。若 Vadim 正确预测了第 $a_i+1$ 天和第 $a_i+2$ 天的登机方式，则他赢得赌约。

尽管 Vadim 并不知道登机方式会如何发生，但他非常希望至少赢得一名学生的赌约，以此说服对方相信他的预测能力。请判断是否存在一种策略，使得 Vadim 能够确保成功。

## 说明/提示

在第一个测试用例中，Vadim 需要至少正确预测第 2 天和第 3 天的登机方式之一。这两天共有 $4$ 种可能的登机场景，因此 Vadim 可以为所有 $4$ 名学生提供不同的预测组合，从而确保至少有一名学生会被说服。

在第二个测试用例中，Vadim 仅与三名学生打赌，无法保证能为其中至少一名提供正确的预测。

## 综合分析与结论

### 核心思路
题目要求判断是否存在一种预测策略，使得Vadim至少能赢取一个学生的赌约。关键点在于能否覆盖所有可能的登机方式组合。每个学生的赌约关注其对应的两天（a_i+1和a_i+2），共有4种可能的组合。若存在至少四个学生对应同一区间（即两天段），则可以通过分配四个不同的预测组合覆盖所有可能性，确保必有一个正确。

### 解决难点
- **覆盖所有组合**：每个两天段有4种可能，需至少四名学生覆盖所有情况。
- **区间统计**：按学生对应的两天段分组，统计每组人数。若某组人数≥4，则存在必胜策略。

### 可视化设计
- **动画展示**：以时间轴显示学生对应的两天段，用不同颜色标记区间。当某区间学生数≥4时，高亮该区间并展示四个组合的覆盖过程。
- **像素风格**：采用8位色块表示天数，不同组合用颜色区分，动态显示预测覆盖过程。

## 题解评分与关键实现

### 代码实现
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int t;
    cin >> t;
    while (t--) {
        int n;
        cin >> n;
        map<int, int> cnt;
        bool ok = false;
        for (int i = 0; i < n; ++i) {
            int a;
            cin >> a;
            int x = a + 1; // 对应两天段的起始天
            if (++cnt[x] >= 4) ok = true;
        }
        cout << (ok ? "Yes" : "No") << endl;
    }
    return 0;
}
```

### 核心逻辑
- **统计区间**：将每个学生的a_i+1作为起始天，统计每个两天段的出现次数。
- **判断条件**：若存在某个起始天的出现次数≥4，则输出"Yes"；否则输出"No"。

## 最优思路提炼
利用**鸽巢原理**，若某两天段的学生数≥4，则必能覆盖所有4种组合。此方法通过统计区间频率快速判断，时间复杂度O(n)。

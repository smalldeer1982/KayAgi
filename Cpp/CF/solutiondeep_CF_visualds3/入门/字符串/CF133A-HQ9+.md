# 题目信息

# HQ9+

## 题目描述

HQ9+是一种搞笑的编程语言，只有4种指令，都是单个字符的：
"H"输出"Hello, World!"；
"Q"输出这个程序本身的源代码；
"9"输出歌曲"99 Bottles of Beer"的歌词（你可以自己上网搜这首歌）；
"+"增加内部储存中的数值

注意"H"和"Q"指令是区分大小写的，必须使用大写。程序中不属于这些指令的字符请忽略。

给你一个用HQ9+写的程序，问这个程序是否会有任何输出。

## 说明/提示

第一个样例中只有一个指令"H"，输出了"Hello, World!"。

第二个样例中没有任何一个字符是这个程序的指令。
翻译贡献者UID：68708

### 输入

```
Hi!
```

### 输出

```
YES
```

## 样例 #2

### 输入

```
Codeforces
```

### 输出

```
NO
```

## 3. 核心难点辨析与解题策略

<difficulty_intro>
这道题看起来简单，但刚开始学编程的同学可能会踩3个“小坑”。结合优质题解的经验，我帮你整理了**避坑指南**：
</difficulty_intro>

1.  **关键点1：搞懂“哪些字符会输出”**
    * **坑**：容易把`+`也算作输出字符（题目里`+`只是“内部加数值”，不输出！）。
    * **解**：反复读题！题目明确说只有`H`、`Q`、`9`会输出——把这三个字符记成“输出三兄弟”，其他字符都不用管。
    * 💡 **学习笔记**：题意理解错了，代码写得再对也没用！先把题目读3遍～

2.  **关键点2：选择“字符串存储方式”**
    * **坑**：不知道用`string`还是`char`数组。
    * **解**：如果要存整个字符串，用`string`（方便！）；如果想边读边检查，用`char`（省空间！）。比如松毛虫的题解用`char`，紫金依旧的用`string`，都能解决问题。
    * 💡 **学习笔记**：工具没有“最好”，只有“最适合当前问题”～

3.  **关键点3：优化“终止时机”**
    * **坑**：遍历完所有字符才输出`YES`（没必要！）。
    * **解**：一旦找到目标字符，立刻用`return 0`结束程序——就像“找到积木就停止翻箱”，节省时间。几乎所有优质题解都用了这个技巧！
    * 💡 **学习笔记**：能早结束就早结束，不要做“无用功”～

### ✨ 解题技巧总结
- **技巧1：题意优先**：先圈出题目中的“关键条件”（比如本题的“输出三兄弟”），避免想当然。
- **技巧2：早停优化**：找到答案就立刻终止程序，提升效率。
- **技巧3：库函数辅助**：比如`string`的`find()`函数、`length()`函数，能帮你减少代码量。


## 4. C++核心代码实现赏析

<code_intro_overall>
先看一份**通用核心代码**——它综合了所有优质题解的优点，逻辑清晰、易理解，适合作为“模板”记下来～
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：本代码用`string`存储输入，通过`for`循环遍历，是最经典的实现方式。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    #include <string>
    using namespace std;

    int main() {
        string s;
        cin >> s;  // 读取输入字符串
        for (int i = 0; i < s.size(); ++i) {  // 逐个字符检查
            if (s[i] == 'H' || s[i] == 'Q' || s[i] == '9') {
                cout << "YES" << endl;
                return 0;  // 找到就立刻结束
            }
        }
        cout << "NO" << endl;  // 没找到
        return 0;
    }
    ```
* **代码解读概要**：
    > 代码分三步：①用`string s`存输入；②用`for`循环从第一个字符（`i=0`）扫到最后一个（`i < s.size()`）；③遇到`H/Q/9`就输出`YES`并终止，否则最后输出`NO`。整个流程像“查字典”，逐个翻页找目标字～

**题解一：来源：松毛虫（边读边检查）**
* **亮点**：不用存整个字符串，边读边检查，省空间！
* **核心代码片段**：
    ```cpp
    char c;
    while (cin >> c) {  // 逐个读字符
        if (c == 'Q' || c == 'H' || c == '9') {
            cout << "YES";
            return 0;
        }
    }
    ```
* **代码解读**：
    > 这段代码用`while(cin>>c)`循环——每次从输入里读一个字符`c`，然后检查是不是“输出三兄弟”。比如输入是`Hi!`，第一次读`H`，直接输出`YES`。这种方法像“吃葡萄不吐葡萄皮”，每吃一颗就检查是不是甜的，甜就立刻停～
* 💡 **学习笔记**：处理大输入时，边读边检查比存整个字符串更高效！

**题解二：来源：二字（用find()函数）**
* **亮点**：用库函数`find()`简化代码，超短！
* **核心代码片段**：
    ```cpp
    string s;
    cin >> s;
    if (s.find("H") + s.find("Q") + s.find("9") != -3) {
        cout << "YES";
    } else {
        cout << "NO";
    }
    ```
* **代码解读**：
    > `s.find("H")`会返回`H`在字符串中的位置（比如`Hi!`中的`H`在位置0），如果没找到就返回`-1`。如果三个`find()`都返回`-1`，总和就是`-3`——这时候输出`NO`；否则至少有一个找到，输出`YES`。这种方法像“用探测器找宝藏”，不用自己逐个翻，直接问探测器有没有信号～
* 💡 **学习笔记**：善用C++的字符串库函数，能让代码更简洁！

**题解三：来源：FP·荷兰猪（读入优化+注释）**
* **亮点**：加了读入优化，注释超详细，适合新手！
* **核心代码片段**：
    ```cpp
    ios::sync_with_stdio(false);  // 关闭同步，加速读入
    string s;
    cin >> s;
    bool f = false;  // f标记是否找到
    for (int i = 0; i < s.size(); ++i) {
        if (s[i] == 'H' || s[i] == 'Q' || s[i] == '9') {
            cout << "YES\n";
            return 0;
        }
    }
    ```
* **代码解读**：
    > `ios::sync_with_stdio(false)`是C++的“读入加速魔法”——当输入很大时，能让程序跑得更快。`bool f`虽然没用到（因为直接`return 0`了），但作者的注释说明了它的作用（标记是否找到），很贴心～
* 💡 **学习笔记**：写代码时加注释，不仅帮别人懂，也帮自己复习！

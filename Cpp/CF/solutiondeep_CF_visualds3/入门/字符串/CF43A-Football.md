# Football

## 题目描述

One day Vasya decided to have a look at the results of Berland 1910 Football Championship’s finals. Unfortunately he didn't find the overall score of the match; however, he got hold of a profound description of the match's process. On the whole there are $ n $ lines in that description each of which described one goal. Every goal was marked with the name of the team that had scored it. Help Vasya, learn the name of the team that won the finals. It is guaranteed that the match did not end in a tie.

## 样例 #1

### 输入

```
1
ABC
```

### 输出

```
ABC
```

## 样例 #2

### 输入

```
5
A
ABA
ABA
A
A
```

### 输出

```
A
```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：`编程技巧应用（统计与比较）`

🗣️ **初步分析**：
解决这道题的关键，其实就是**“数清楚每个球队进了多少球，谁多谁赢”**——像你给喜欢的球队记进球数一样简单！题目藏了个小秘密：**所有进球只来自两支球队**（不信看样例和题解里的提示），所以我们有两种思路：
- **思路1**：直接记两个队的进球数（比如队A进了多少，队B进了多少）；
- **思路2**：用`map`（字典）统计所有队的进球数（虽然其实只有两个，但方法更通用）。

核心难点是**“正确记录两个球队的名称”**——比如前3个进球都是队A，这时候不能着急创建队B，得等遇到不同的队名时再记。  
可视化设计思路：我们会做一个**8位像素风的足球统计界面**——左边是队A的像素图标（红球衣），右边是队B的图标（蓝球衣），每输入一个队名，对应图标的进球数就+1（数字闪烁），最后赢的球队会“跳起来”（像素动画）并播放胜利音效！

## 3. 核心难点辨析与解题策略

<difficulty_intro>
这道题的“坑”都藏在**细节**里——比如“前几个进球都是同一队”“如何正确记第二个队名”。结合优质题解，我帮你提炼了3个核心关键点：
</difficulty_intro>

1.  **关键点1：如何正确记录第二个队名？**  
    * **分析**：不能直接取“第二个输入的队名”（比如前两个都是队A，第二个输入还是A，这时候不能把A当队B）。正确的做法是：**等遇到和第一个队名不同的输入时，再记第二个队名**（像题解二里的`if(s!=a) {a2++; b=s;}`）。  
    * 💡 **学习笔记**：不要“提前假设”第二个队的存在，等实际遇到不同的队名再处理！

2.  **关键点2：选`map`还是直接记两个变量？**  
    * **分析**：如果题目明确只有两个队（比如本题），直接记两个变量（`a1`、`a2`）更高效；如果可能有多个队，用`map`更通用（比如统计联赛10支球队的进球数）。  
    * 💡 **学习笔记**：工具选对了，代码会变简单！

3.  **关键点3：如何避免统计错误？**  
    * **分析**：无论是`map`还是直接变量，**每个输入只做一次统计**（比如题解一里的`cin>>k; s[k]++;`，每个k对应一次+1），不会重复。  
    * 💡 **学习笔记**：“一个输入对应一次操作”是统计类题目的黄金法则！

### ✨ 解题技巧总结
- **技巧1**：利用题目隐藏条件（只有两个队）简化代码（如题解二）；  
- **技巧2**：用`map`快速关联“队名-进球数”，减少逻辑判断（如题解一）；  
- **技巧3**：先处理第一个输入，避免“空队名”的错误（比如先读第一个队名存在`a`里）。


## 4. C++核心代码实现赏析

<code_intro_overall>
先看一个**通用核心实现**——结合了题解一的简洁和题解三的通用，适合大多数情况！
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：本代码用`map`统计所有队的进球数，然后遍历找最大值，支持任意数量的球队（虽然本题只有两个）。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    #include <map>
    #include <string>
    using namespace std;

    int main() {
        int n;
        cin >> n;
        map<string, int> score; // 队名→进球数的字典
        string team;
        for (int i = 0; i < n; ++i) {
            cin >> team;
            score[team]++; // 该队进球数+1
        }
        // 找进球最多的队
        string winner;
        int max_goals = 0;
        for (auto& pair : score) { // 遍历字典里的每一对（队名, 进球数）
            if (pair.second > max_goals) {
                max_goals = pair.second;
                winner = pair.first;
            }
        }
        cout << winner << endl;
        return 0;
    }
    ```
* **代码解读概要**：  
  1. 用`map<string, int> score`存队名和进球数；  
  2. 循环读n个队名，每个队名对应的`score`值+1；  
  3. 遍历`score`，找“进球数最多”的队名，输出。

<code_intro_selected>
接下来看3份优质题解的**核心片段**——每段都有“神来之笔”！
</code_intro_selected>

**题解一：（来源：Ajwallet）**  
* **亮点**：实时更新最大值，不用最后遍历！
* **核心代码片段**：
    ```cpp
    map<string, int> s;
    string ans, k;
    int maxn = 0;
    while (n--) {
        cin >> k;
        s[k]++;
        if (s[k] > maxn) { // 实时比较，更新最大值
            maxn = s[k];
            ans = k;
        }
    }
    cout << ans;
    ```
* **代码解读**：  
  每次输入队名`k`，先让`s[k]`+1（统计进球数），然后**立即比较**：如果这个队的进球数超过当前最大值`maxn`，就更新`maxn`和冠军队`ans`。这样不用最后遍历`map`，更快！  
* 💡 **学习笔记**：实时更新能减少后续操作，是“优化代码”的小技巧！

**题解二：（来源：wubaiting2020）**  
* **亮点**：直接记两个队，代码最短！
* **核心代码片段**：
    ```cpp
    int a1=0, a2=0;
    string a, b, s;
    cin >> n >> s;
    a = s; a1++; // 第一个队是a，进球数+1
    for (int i=2; i<=n; ++i) {
        cin >> s;
        if (s == a) a1++; // 是a队，进球数+1
        else { a2++; b = s; } // 是b队，记名字+进球数
    }
    cout << (a1 > a2 ? a : b); // 三目运算符，简洁判断
    ```
* **代码解读**：  
  先读第一个队名`a`，然后循环读剩下的：如果和`a`一样，`a1`+1；否则`a2`+1并记`b`（第二个队名）。最后用**三目运算符**（`条件? 结果1 : 结果2`）快速输出赢家，代码超简洁！  
* 💡 **学习笔记**：三目运算符能代替简单的`if-else`，让代码更短！

**题解三：（来源：览遍千秋）**  
* **亮点**：用迭代器遍历`map`，通用型强！
* **核心代码片段**：
    ```cpp
    map<string, int> mp;
    string s, ans;
    int maxx = 0;
    for (int i=1; i<=n; ++i) {
        cin >> s;
        mp[s]++; // 统计进球数
    }
    // 用迭代器遍历map
    for (map<string, int>::iterator it = mp.begin(); it != mp.end(); ++it) {
        if (it->second > maxx) { // it->second是进球数
            maxx = it->second;
            ans = it->first; // it->first是队名
        }
    }
    cout << ans;
    ```
* **代码解读**：  
  `map`的迭代器`it`就像“指针”，`it->first`是队名（键），`it->second`是进球数（值）。遍历所有键值对，找最大的`second`，对应的`first`就是赢家。这种方法能处理**任意数量的球队**，比如联赛统计10支球队也能用！  
* 💡 **学习笔记**：迭代器是遍历STL容器的“万能钥匙”，一定要学会！

> **参考经验 (来自 hewo)**：“我一开始赋值`b.name`时没判断和`a`是否一样，结果前两个输入都是‘A’时，`b.name`也变成了‘A’，导致统计错误！后来加了`if(x!=a.name)`才修好。”  
> **点评**：这个错误很典型！如果第二个队名和第一个相同，就会把“同一队”当成“两队”，统计结果肯定错。**解决方法**：赋值第二个队名时，一定要加“和第一个队名不同”的条件（像题解二里的`if(s!=a)`）。调试时可以打印`a`和`b`的名字，看是否正确——这是“找bug”的好方法！

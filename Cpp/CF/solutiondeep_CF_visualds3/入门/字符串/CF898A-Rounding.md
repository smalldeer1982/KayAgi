# Rounding

## 题目描述

Vasya has a non-negative integer $ n $ . He wants to round it to nearest integer, which ends up with $ 0 $ . If $ n $ already ends up with $ 0 $ , Vasya considers it already rounded.

For example, if $ n=4722 $ answer is $ 4720 $ . If $ n=5 $ Vasya can round it to $ 0 $ or to $ 10 $ . Both ways are correct.

For given $ n $ find out to which integer will Vasya round it.

## 说明/提示

In the first example $ n=5 $ . Nearest integers, that ends up with zero are $ 0 $ and $ 10 $ . Any of these answers is correct, so you can print $ 0 $ or $ 10 $ .

## 样例 #1

### 输入

```
5
```

### 输出

```
0
```

## 样例 #2

### 输入

```
113
```

### 输出

```
110
```

## 样例 #3

### 输入

```
1000000000
```

### 输出

```
1000000000
```

## 样例 #4

### 输入

```
5432359
```

### 输出

```
5432360
```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：编程技巧应用（数学运算处理数字个位）

🗣️ **初步分析**：  
解决“Rounding”的关键，是把一个非负整数**凑整到最近的末尾为0的数**——就像我们平时“凑整钱”：如果个位是1-4，直接减掉个位变0；如果是6-9，就加（10-个位）让个位变0（同时前一位加1）；如果是5，不管减还是加都对（题目允许）。  

在本题中，我们可以用**数学运算（取模%、整除/）**快速处理：  
- 用`n%10`取个位；  
- 用`n/10`去掉个位，再乘10恢复个位为0；  
- 或者用更巧妙的技巧：`(n+5)/10*10`（利用整数除法的“向下取整”，自动处理“四舍五入”）。  

**核心算法流程**：  
1. 取数字的个位（`ge = n%10`）；  
2. 根据个位的值调整n：  
   - 个位≤5：`n -= ge`（舍掉个位）；  
   - 个位>5：`n += (10 - ge)`（进位，让个位变0）；  
3. 输出调整后的n。  

**可视化设计思路**：  
用8位像素风格展示数字的每一位（个位用红色高亮），通过动画演示“减个位”或“加进位”的过程——比如个位是3时，红色像素块逐渐消失，数字变成110；个位是9时，红色块变0，前一位加1。搭配“叮”的操作音效和胜利音效，让算法“看得见、听得见”。

## 3. 核心难点辨析与解题策略

在解决这个问题时，大家常遇到3个关键点：


### 1. 关键点1：如何处理大数溢出？  
**分析**：样例3的输入是`1000000000`（1e9），如果用`int`存储会溢出（`int`通常最大是2e9，但有些编译器可能更小）。优质题解中都用了`long long`（8字节，能存到9e18），完美解决这个问题。  
💡 **学习笔记**：处理大数时，优先用`long long`！


### 2. 关键点2：如何统一5的处理逻辑？  
**分析**：题目允许5“舍或入”，所以只要逻辑一致就对。比如Reywmp的代码选择“入”（5→10），向日葵小班选择“舍”（5→0），都是正确的。关键是**不要同时用两种逻辑**，否则会出错。  
💡 **学习笔记**：边界情况（比如5）要统一处理逻辑！


### 3. 关键点3：如何用数学运算简化代码？  
**分析**：`(n+5)/10*10`的技巧太好用了——它把“判断个位→调整n”的两步合并成一步，利用整数除法的特性自动处理。比如n=5432359（个位9），`5432359+5=5432364`，`5432364/10=543236`，`543236*10=5432360`，正好是样例4的输出。  
💡 **学习笔记**：数学运算能简化代码，多想想“有没有更巧的方式”！


### ✨ 解题技巧总结  
- **取模%**：快速获取数字的最后一位（个位）；  
- **整除/**：快速去掉数字的最后一位（比如`n/10`）；  
- **整数除法特性**：`(n + k)/10 *10`可以实现“凑整到最近的10的倍数”（k取5时是四舍五入）；  
- **数据类型选择**：大数用`long long`，避免溢出。


## 4. C++核心代码实现赏析

在深入分析具体题解前，先看一个**通用核心实现**——它综合了优质题解的思路，逻辑清晰，适合大多数情况：


### 本题通用核心C++实现参考  
* **说明**：本代码综合了向日葵小班和Reywmp的思路，用`long long`处理大数，分情况处理个位，逻辑易懂。  
* **完整核心代码**：  
```cpp
#include <iostream>
using namespace std;

int main() {
    long long n; // 用long long防止大数溢出
    cin >> n;
    int ge = n % 10; // 取个位
    if (ge <= 5) {
        n -= ge; // 个位≤5，舍掉个位
    } else {
        n += (10 - ge); // 个位>5，进位
    }
    cout << n << endl;
    return 0;
}
```  
* **代码解读概要**：  
  1. 读入数字`n`（用`long long`）；  
  2. 用`n%10`取个位`ge`；  
  3. 根据`ge`的值调整`n`：舍或进位；  
  4. 输出结果。


### 针对各优质题解的片段赏析


#### 题解一：作者Reywmp  
* **亮点**：一行代码实现四舍五入，极简！  
* **核心代码片段**：  
```cpp
a = (a + 5) / 10 * 10;
```  
* **代码解读**：  
  这句话的 magic 在于**整数除法的向下取整**：  
  - 当`a`的个位是5时，`a+5=10`→`10/10=1`→`1*10=10`（进位）；  
  - 当个位是4时，`a+5=9`→`9/10=0`→`0*10=0`（舍去）；  
  - 当个位是6时，`a+5=11`→`11/10=1`→`1*10=10`（进位）。  
  一句话搞定所有情况，太聪明了！  
* 💡 **学习笔记**：利用数学运算的特性，可以少写很多条件判断！


#### 题解二：作者向日葵小班  
* **亮点**：条件判断明确，适合初学者理解。  
* **核心代码片段**：  
```cpp
if(n%10<=5) n -= n%10;
if(n%10>5)  n += (10 - n%10);
```  
* **代码解读**：  
  - 第一行：如果个位≤5（比如113的个位3），就减去个位（113-3=110）；  
  - 第二行：如果个位>5（比如5432359的个位9），就加（10-个位）（5432359+1=5432360）。  
  逻辑直接，一看就懂，适合刚开始学数字处理的同学。  
* 💡 **学习笔记**：分情况处理能让逻辑更清晰！


#### 题解三：作者不到前10不改名  
* **亮点**：利用整数除法简化逻辑，容易记忆。  
* **核心代码片段**：  
```cpp
if(n%10>=5)
    n=(n/10+1)*10;
else
    n=n/10*10;
```  
* **代码解读**：  
  - 如果个位≥5（比如5），先把n除以10（去掉个位，5→0）加1（进位→1），再乘10（恢复个位为0→10）；  
  - 否则（比如4），直接除以10乘10（4→0）。  
  逻辑简洁，记住“≥5就+1再乘10，否则直接乘10”就行。  
* 💡 **学习笔记**：整数除法是处理数字末尾的“神器”！

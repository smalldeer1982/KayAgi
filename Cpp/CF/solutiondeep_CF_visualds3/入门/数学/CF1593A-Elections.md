# 题目信息

# Elections

## 题目描述

三名候选人参加竞选 。第一名候选人得到了 $a$ 票，第二名候选人得到了 $b$ 票，第三名候选人得到了 $c$ 票 。对于每一个候选人，解决以下问题 ：

至少给该候选人增加多少票，使得该候选人的票数严格大于其它候选人？

注意，每个问题是独立存在的 。

## 样例 #1

### 输入

```
5
0 0 0
10 75 15
13 13 17
1000 0 0
0 1000000000 0```

### 输出

```
1 1 1
66 0 61
5 5 0
0 1001 1001
1000000001 0 1000000001```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：数学（简单极值计算与逻辑推导）

🗣️ **初步分析**：  
解决这道题就像玩“比身高游戏”——要让小明比小红、小刚都高，得先知道小红和小刚里**最高的那个有多高**，再算小明要长多少厘米才能**超过**他（比如小红160cm、小刚155cm，小明150cm，就得长到161cm，即11cm=160-150+1）。  

在本题中，每个候选人的问题**独立**（计算A的票数时，B、C的票数不变），核心逻辑可以总结为：  
对候选人X（票数为`x`），找到另外两人的**最高票数**`max_other`，若`x`已经严格大于`max_other`，则不需要加票（0）；否则需要加`max_other - x + 1`票（`+1`是因为要**严格大于**，不能相等）。  

### 核心算法流程与可视化设计思路
1. **输入处理**：读取测试用例数`T`，循环处理每组`a,b,c`。  
2. **计算每个候选人的票数**：  
   - 候选人A：`max_bc = max(b,c)` → 需要`max(0, max_bc - a + 1)`  
   - 候选人B：`max_ac = max(a,c)` → 需要`max(0, max_ac - b + 1)`  
   - 候选人C：`max_ab = max(a,b)` → 需要`max(0, max_ab - c + 1)`  
3. **可视化设计**：用8位FC像素风展示三个彩色柱子（红=A、绿=B、蓝=C），计算时**高亮当前柱子和另外两人的最高柱**，动态增加当前柱子的高度直到超过最高柱，伴随“叮叮”音效（每加1票响一次），最终显示增加的票数（如“+6”）。

* **点评**：这份题解是“最直观的入门写法”！用**条件判断**直接对应题目逻辑——如果A的票数大于B、C的最大值，输出0；否则输出需要的票数。代码没有冗余，变量名`a,b,c`完全对应题目，新手一看就懂。比如处理A的逻辑：`if(a > max(b,c)) cout<<0; else cout<<max(b,c)-a+1;`，完美贴合“比身高”的思路，边界处理（如三个0的情况）也很严谨。

**题解二：作者·ttq012**  
* **点评**：这是“最简洁的数学技巧写法”！用`max(0, ...)`函数**简化条件判断**——当`max(b,c)-a+1`为负数（说明A已经更大）时，`max(0, ...)`直接返回0，否则返回需要的票数。一行代码处理一个候选人，整体代码极其紧凑：`cout << max(0, max(b,c)-a+1) << ' ';`。这种“用数学函数替代if”的技巧，能大幅减少代码行数，值得学习！

**题解三：作者·Coros_Trusds**  
* **点评**：这是“最模块化的工程写法”！把计算单个候选人票数的逻辑封装成`solve`函数：`int solve(int x, int y, int z) { return x>y&&x>z ? 0 : max(y,z)+1-x; }`。调用时只需`printf("%d %d %d\n", solve(a,b,c), solve(b,a,c), solve(c,a,b));`，既避免了重复代码，又方便后续修改逻辑（比如调整“严格大于”为“大于等于”，只需改函数内的条件）。


## 3. 核心难点辨析与解题策略

<difficulty_intro>
这道题的“难点”其实是**逻辑的严谨性**——容易忽略“独立计算”“严格大于”或“边界情况”。结合题解的共性，我提炼了3个关键思考点：
</difficulty_intro>

1. **关键点1：理解“独立计算”**  
   * **分析**：每个候选人的问题是独立的，计算A的票数时，B、C的票数不会改变。比如样例`10 75 15`中，计算B的票数时，A还是10、C还是15，所以B已经是最大值，输出0。  
   * 💡 **学习笔记**：独立问题要“分开算，不干扰”！

2. **关键点2：“严格大于”的处理（+1）**  
   * **分析**：如果A的票数等于`max_other`（比如A=10，B=10，C=5），此时A不满足“严格大于”，必须加1票（变成11）。公式中的`+1`就是为了处理这种情况。  
   * 💡 **学习笔记**：严格大于=等于+1，记牢这个“+1法则”！

3. **关键点3：边界情况的覆盖**  
   * **分析**：比如三个都是0的情况（输出1 1 1）、一个已经是严格最大值的情况（比如`1000 0 0`，输出0 1001 1001）。我们的通用公式`max(0, max_other - x +1)`已经覆盖了所有边界——比如三个0时，`max_other=0`，`0-0+1=1`，正好是答案。  
   * 💡 **学习笔记**：找“通用公式”比“分情况讨论”更高效，能自动处理所有边界！

### ✨ 解题技巧总结
- **技巧A：用通用公式替代分情况**：本题的公式覆盖了“三个相同”“两个相同”“一个最大”等所有情况，不用写复杂的`if-else`。  
- **技巧B：用函数封装重复逻辑**：如果有重复代码（比如计算三个候选人的票数），写成函数能让代码更简洁、易维护。  
- **技巧C：善用数学函数简化代码**：`max(0, x)`可以替代“如果x<0则取0，否则取x”的条件判断，减少代码行数。


## 4. C++核心代码实现赏析

<code_intro_overall>
先看一份**通用核心实现**——综合了“简洁性”和“可读性”，适合新手直接参考：
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：本代码结合了ttq012的“数学简化”和智子的“直观逻辑”，是最易理解的版本。
* **完整核心代码**：
  ```cpp
  #include <iostream>
  #include <algorithm> // 用于std::max函数
  using namespace std;

  int main() {
      ios::sync_with_stdio(false); // 加速输入输出（可选）
      cin.tie(nullptr);

      int t;
      cin >> t;
      while (t--) {
          int a, b, c;
          cin >> a >> b >> c;
          // 计算每个候选人需要的票数
          int ans_a = max(0, max(b, c) - a + 1);
          int ans_b = max(0, max(a, c) - b + 1);
          int ans_c = max(0, max(a, b) - c + 1);
          cout << ans_a << ' ' << ans_b << ' ' << ans_c << '\n';
      }
      return 0;
  }
  ```
* **代码解读概要**：  
  1. 用`ios::sync_with_stdio(false); cin.tie(nullptr);`加速输入输出（处理大数据时更高效）。  
  2. 循环处理每个测试用例，读取`a,b,c`。  
  3. 用`max(0, ...)`计算每个候选人的票数，最后输出结果。


<code_intro_selected>
接下来剖析3份优质题解的**核心片段**，看看它们的“亮点”在哪里：
</code_intro_selected>

**题解一：智子（直接条件判断）**
* **亮点**：逻辑1:1对应题目，新手最易理解。
* **核心代码片段**：
  ```cpp
  if (a > max(b, c)) {
      cout << 0 << " ";
  } else {
      cout << max(b, c) - a + 1 << " ";
  }
  ```
* **代码解读**：  
  这段代码直接对应“比身高”的逻辑——如果A的票数大于B、C的最大值，输出0；否则输出需要增加的票数。比如`a=10`、`max(b,c)=15`，则`15-10+1=6`，正好是需要加的票数。
* 💡 **学习笔记**：条件判断是“最直白的逻辑表达”，适合入门时建立思路。

**题解二：ttq012（数学简化）**
* **亮点**：用`max(0, ...)`简化条件判断，代码更紧凑。
* **核心代码片段**：
  ```cpp
  cout << max(0, max(b, c) - a + 1) << ' ';
  ```
* **代码解读**：  
  当`a > max(b,c)`时，`max(b,c)-a+1`是负数，`max(0, ...)`会返回0；否则返回需要的票数。比如`a=20`、`max(b,c)=15`，返回0；`a=10`、`max(b,c)=15`，返回6。
* 💡 **学习笔记**：数学函数能“隐藏”条件判断，让代码更简洁。

**题解三：Coros_Trusds（函数封装）**
* **亮点**：模块化编程，避免重复代码。
* **核心代码片段**：
  ```cpp
  inline int solve(int x, int y, int z) {
      if (x > y && x > z) return 0;
      return max(y, z) + 1 - x;
  }
  ```
* **代码解读**：  
  函数`solve(x,y,z)`接收当前候选人的票数`x`和另外两人的票数`y,z`，返回需要增加的票数。调用时只需`printf("%d %d %d\n", solve(a,b,c), solve(b,a,c), solve(c,a,b));`，既简洁又易维护。
* 💡 **学习笔记**：函数封装是“工程化编程”的基础，能大幅提升代码的可读性。

# Little Nikita

## 题目描述

小男孩 Nikita 收到了一些方块作为礼物。他决定用这些方块搭建一座塔。

最初，塔中没有任何方块。每一步操作中，Nikita 可以选择在塔顶放置恰好 $1$ 个方块，或者从塔顶移除恰好 $1$ 个方块。请问，经过 $n$ 次操作后，塔中是否可能恰好有 $m$ 个方块？

## 说明/提示

在第一个测试用例中，Nikita 可以连续 $3$ 次在塔顶放置 $1$ 个方块，因此答案为 "Yes"。

在第二个测试用例中，Nikita 最终只能得到没有方块的塔或有 $2$ 个方块的塔，因此答案为 "No"。

# 💡 Kay的C++算法解析：Little Nikita 深入学习指南 💡

今天我们来一起分析“Little Nikita”这道C++编程题。它看起来是关于“搭塔操作”的游戏问题，但其实藏着**数学规律的密码**——我们需要用“方程思维”破解“n次操作后能否得到m个方块”的谜题。本指南会帮你理清条件、写出简洁代码，还能用像素动画直观“看”到算法逻辑！


## 1. 题目解读与核心算法识别

✨ **本题主要考察**：数学（方程建模与解的合法性判断）

🗣️ **初步分析**：  
解决这道题，本质是帮Nikita算一笔“步数账”——每次操作要么给塔**加1个方块**（+1），要么**减1个方块**（-1），总共走`n`步，要刚好剩下`m`个方块。  

我们可以用数学方程把问题“翻译”清楚：  
假设加了`x`次方块，减了`y`次方块，那么：  
- 总步数：`x + y = n`（加的次数+减的次数=总操作次数）  
- 最终结果：`x - y = m`（加的总数-减的总数=塔的最终高度）  

解这两个方程，得到：  
`x = (n + m) / 2`，`y = (n - m) / 2`  

要让`x`和`y`都是**非负整数**（次数不能是负数或分数），必须满足3个条件：  
1. `m ≥ 0`（塔不能有负数个方块）；  
2. `m ≤ n`（最多全加`n`次，不可能超过`n`）；  
3. `n - m`是偶数（这样`x`和`y`都是整数，因为`n+m`和`n-m`同奇偶）。  

比如样例中的第三个测试用例`n=5，m=3`：  
`n-m=2`是偶数，`x=(5+3)/2=4`（加4次），`y=(5-3)/2=1`（减1次）——操作序列是`+1,+1,+1,+1,-1`，刚好5步后得到3个方块！

**可视化设计思路**：  
我们会用8位像素动画展示“搭塔过程”——用红色像素方块堆成塔，每次加方块时“向上弹出”，减方块时“向下消失”，同步显示当前步数和塔高。关键步骤（比如满足条件/不满足条件）会用**颜色高亮**和**音效提示**，帮你直观记住条件！

## 3. 核心难点辨析与解题策略

<difficulty_intro>
这道题的“陷阱”藏在“操作的数学本质”里。结合推导过程，我帮大家提炼了3个核心思考点：
</difficulty_intro>

1. **关键点1：如何把操作转化为数学方程？**  
   * **分析**：操作是“+1”或“-1”，本质是“两个变量的和与差”——总步数是`x+y`，结果是`x-y`。就像“总钱数=赚的钱-花的钱”“总天数=工作天数+休息天数”，把现实操作“翻译”成数学关系，问题就会变得清晰。  
   * 💡 **学习笔记**：把问题抽象成数学方程，是解决很多编程题的第一步！

2. **关键点2：如何保证解是“合法”的？**  
   * **分析**：`x`和`y`必须是**非负整数**（不能加负数次，也不能加半次）。所以需要检查：`m≤n`（`x`不超过总次数）、`n-m`是偶数（`y`是整数）、`m≥0`（塔不能为负）。  
   * 💡 **学习笔记**：解的“合法性”是判断条件的核心——不要只看方程，还要看现实约束！

3. **关键点3：为什么`n`和`m`要同奇偶？**  
   * **分析**：每次操作改变`1`，`n`次操作后的总变化量（即`m`）的奇偶性，和`n`的奇偶性必须一致（比如`n=3`是奇数，`m`必须是奇数；`n=2`是偶数，`m`必须是偶数）。比如样例中的第二个测试用例`n=2，m=4`，`m`是偶数但`m>n`，直接不合法！  
   * 💡 **学习笔记**：奇偶性是“操作类问题”的常见隐藏条件！

### ✨ 解题技巧总结
- **技巧A：问题抽象**：把操作转化为数学方程，找到变量间的关系；  
- **技巧B：验证合法性**：解必须满足“非负、整数、现实约束”；  
- **技巧C：边界测试**：比如`m=0`（全减）、`m=n`（全加）这些极端情况，确保条件覆盖。


## 4. C++核心代码实现赏析

<code_intro_overall>
我们直接根据推导的条件，写一个**简洁高效**的核心代码——只需3行判断逻辑，就能处理所有测试用例！
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：本代码直接对应我们推导的3个条件，是解决本题的“标准答案”，逻辑清晰、运行高效。
* **完整核心代码**：
  ```cpp
  #include <iostream>
  using namespace std;

  int main() {
      int t; // 测试用例数量
      cin >> t;
      while (t--) {
          int n, m;
          cin >> n >> m;
          // 核心条件判断：m非负、m≤n、n-m是偶数
          if (m >= 0 && m <= n && (n - m) % 2 == 0) {
              cout << "Yes" << endl;
          } else {
              cout << "No" << endl;
          }
      }
      return 0;
  }
  ```
* **代码解读概要**：  
  1. 读取测试用例数量`t`；  
  2. 循环处理每组`n`（总步数）和`m`（目标高度）；  
  3. 用`if`判断3个条件，符合输出`Yes`，否则输出`No`。  

👉 这个代码的优点是**没有冗余逻辑**——直接命中问题的核心条件，运行时间几乎可以忽略不计！

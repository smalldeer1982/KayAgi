# Legs

## 题目描述

又是农场主 John 的美好一天。

当 John 到达他的农场时，他数了 $n$ 条腿。已知农场里只有鸡和牛，鸡有 $2$ 条腿，牛有 $4$ 条腿。

假设 John 数清了所有动物的腿，请问农场上最少有多少只动物？

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：贪心算法  

🗣️ **初步分析**：
> 解决“Legs”问题的关键是**贪心算法**——就像你想装最少的袋子装苹果，肯定会先选最大的袋子（装最多苹果）。这里要让动物数量最少，就得优先选**腿最多的动物**（牛，4条腿），剩下的腿用鸡（2条腿）补。  
   - **核心思路**：对每个输入的腿数`n`，先算最多能养多少头牛（`n//4`），剩下的腿数除以2就是鸡的数量，两者相加就是答案。  
   - **核心难点**：为什么贪心是对的？——因为每只牛贡献的腿更多，选牛能最快减少“需要的动物数量”，最终结果一定是全局最少。  
   - **可视化设计思路**：用像素农场场景展示“选牛→补鸡”的过程：总腿数用计数器显示，选牛时牛图标高亮、腿数减4，选鸡时鸡图标高亮、腿数减2，关键步骤配像素音效（比如牛“哞”、鸡“咯咯”）。

## 3. 核心难点辨析与解题策略

<difficulty_intro>
解决这个问题时，大家可能会疑惑“为什么贪心是对的？”“余数怎么处理？”，结合贪心的核心逻辑，我们来拆解关键点：
</difficulty_intro>

1.  **关键点1：贪心策略的正确性**  
    * **分析**：要最小化动物数量，必须让每只动物的腿尽可能多。假设腿数`n`，选`k`头牛（4k腿），剩下的`n-4k`腿用鸡（每只2腿），总数量是`k + (n-4k)/2`。当`k`最大时，总数量最小——比如n=6，k=1时总数量2，k=0时总数量3，显然k越大越好。  
    * 💡 **学习笔记**：贪心的核心是“当前最优→全局最优”，一定要验证这个逻辑是否成立！

2.  **关键点2：余数的处理**  
    * **分析**：因为鸡和牛的腿都是偶数，所以`n`一定是偶数（题目隐含条件）。`n`除以4的余数只能是0或2：  
      - 余数0：不用鸡（比如n=8→2头牛）；  
      - 余数2：需要1只鸡（比如n=6→1牛+1鸡）。  
    * 💡 **学习笔记**：利用题目隐含条件（n是偶数），能简化余数处理！

3.  **关键点3：输入的批量处理**  
    * **分析**：题目可能有多个测试用例（比如样例输入第一行是3，代表3个测试用例），需要用循环读入并处理每个`n`。  
    * 💡 **学习笔记**：遇到多测试用例时，记得用`while(t--)`循环！

### ✨ 解题技巧总结
- **优先选“大的”**：求最小数量时，优先选单位贡献最大的元素（比如牛的腿更多）；  
- **验证贪心逻辑**：用小例子测试（比如n=2、6、8），确认结果正确；  
- **利用隐含条件**：题目说“数清了所有腿”，所以`n`一定是偶数，不用处理奇数情况。

## 4. C++核心代码实现赏析

<code_intro_overall>
这是一个**基于贪心思路的通用核心实现**，代码简洁高效，能直接解决问题：
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：本代码综合了贪心算法的核心逻辑，处理多测试用例，适合直接运行。  
* **完整核心代码**：
    ```cpp
    #include <iostream>
    using namespace std;

    int main() {
        int t; // 测试用例数量
        cin >> t;
        while (t--) { // 循环处理每个测试用例
            int n; // 总腿数
            cin >> n;
            // 牛的数量：n//4，鸡的数量：(n%4)/2
            cout << (n / 4) + ((n % 4) / 2) << endl;
        }
        return 0;
    }
    ```
* **代码解读概要**：  
  1. 读入测试用例数量`t`；  
  2. 用`while(t--)`循环处理每个`n`；  
  3. 计算`n//4`（最多能养的牛） + `(n%4)/2`（剩下的腿需要的鸡），输出结果。

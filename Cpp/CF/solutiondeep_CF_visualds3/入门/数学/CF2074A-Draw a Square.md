# Draw a Square

## 题目描述

粉色士兵们给了你平面上的 $4$ 个不同点。这 $4$ 个点的坐标分别为 $(-l, 0)$、$(r, 0)$、$(0, -d)$ 和 $(0, u)$，其中 $l$、$ r$、$d$、$u$ 是正整数。

![](https://cdn.luogu.com.cn/upload/vjudge_pic/CF2074A/cae2389956c359d11ac6b1eaa6f81027eb9d1dfa.png) 

在图中，通过连接四个点 $L$、$R$、$D$、$U$ 绘制了一个正方形。请判断是否可以使用给定的四个点作为顶点绘制一个正方形$^{\text{∗}}$。

$^{\text{∗}}$正方形的定义是一个由 $4$ 个顶点组成的多边形，其所有边长相等且所有内角相等。多边形的任意两条边不得相交。

## 说明/提示

在第一个测试用例中，给定的四个点构成正方形，因此答案是 "Yes"。

在第二个测试用例中，给定的四个点不构成正方形，因此答案是 "No"。

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：数学（几何性质推导） + 基础编程技巧应用

🗣️ **初步分析**：
> 题目给了四个在坐标轴上的点：左点(-l,0)、右点(r,0)、下点(0,-d)、上点(0,u)。要判断它们能不能连成正方形，关键得**用正方形的几何性质推导条件**——  
> 正方形的对角线要满足三个条件：①互相平分（中点重合）、②长度相等、③互相垂直（x轴和y轴本来就垂直，这点不用额外判断）。  
> 结合四个点的位置（在坐标轴上），我们可以推导出：  
> 1. 对角线中点重合 → 左点和右点的中点（(r-l)/2, 0）必须等于下点和上点的中点（0, (u-d)/2）→ **r=l且u=d**；  
> 2. 对角线长度相等 → 左右点距离（r+l=2l）等于上下点距离（u+d=2d）→ **l=d**；  
> 合并起来就是**l=r=d=u**（四个数全相等）！  
>  
> 可视化设计思路：我会用8位像素风格展示坐标轴和四个点——当输入满足条件时，四个点会用蓝色线条连成正方形，伴随"叮"的胜利音效；不满足时，点保持原样，播放短促的"提示音"。这样能直观看到条件满足与否的差异～

<difficulty_intro>

1.  **关键点1：如何推导正方形的条件？**
    * **分析**：不要直接写代码！先画草稿：四个点在坐标轴上，正方形的对角线一定是x轴上的左右点、y轴上的上下点（因为其他组合的中点不可能重合）。然后用"中点重合"和"对角线相等"推导条件——这一步是核心！

    * **分析**：题目第一行是测试用例数量t，后面t行每行四个数。要用`while(t--)`循环读入，避免漏处理。  
    * 💡 **学习笔记**：多测试用例的模板是"读t→循环t次"，记下来！

## 4. C++核心代码实现赏析

<code_intro_overall>
  这是综合几何推导后的**最简核心代码**，能直接解决问题～
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：本代码基于"四个数全相等"的条件，逻辑清晰，适合初学者参考。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    using namespace std;

    int main() {
        int t;
        cin >> t;  // 读测试用例数量
        while (t--) {  // 循环处理每个用例
            int l, r, d, u;
            cin >> l >> r >> d >> u;  // 读四个参数
            // 判断是否四个数全相等
            if (l == r && r == d && d == u) {
                cout << "Yes" << endl;
            } else {
                cout << "No" << endl;
            }
        }
        return 0;
    }
    ```
* **代码解读概要**：  
  1. 读入测试用例数量`t`；  
  2. 循环`t`次，每次读入四个整数`l,r,d,u`；  
  3. 判断四个数是否全相等：是→输出"Yes"，否→输出"No"。  
  整个代码只有10行左右，核心逻辑就是"判断四个数相等"，超简洁！

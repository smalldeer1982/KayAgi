# Watermelon

## 题目描述

一个炎热的夏天，Pete 和他的朋友 Billy 感到十分口渴，于是在西瓜摊上买了一个重 $w$ 公斤的西瓜。

他们回到家准备分西瓜。因为两个人都十分喜爱偶数，所以他们俩希望分出来的两块西瓜的重量都是偶数公斤。帮助他们判断是否可能做到这一点。

## 样例 #1

### 输入

```
8
```

### 输出

```
YES
```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：数学（奇偶性判断）+ 编程技巧（特判特殊情况）

🗣️ **初步分析**：
> 问题的核心是判断一个数能否分成**两个正偶数**之和。我们可以用「数学中的奇偶性」快速解决：  
> 就像「成对的糖果」——两个偶数（成对的糖果）加起来肯定还是偶数（更大的成对糖果），所以**只有偶数才有可能满足条件**。但这里有个小陷阱：最小的两个正偶数是2和2，加起来是4，所以**2这个偶数无法分成两个正偶数**（因为2=1+1或0+2，但1不是偶数，0不是正数）！  
> 所有题解的思路都围绕这两点：① 判断是否为偶数；② 排除2这个特例。核心难点是**不要漏掉「正偶数」的隐含要求**——很多人一开始会忘记特判2，导致错误。  
> 可视化设计上，我打算用「像素小人分西瓜」的动画：西瓜是8位像素块，小人尝试把西瓜分成两块，**绿色块代表偶数**，**红色块代表奇数**。如果两块都是绿色，就显示「YES」并播放轻快的「叮」声；如果是2，分裂成两个红色块，显示「NO」并播放短促的提示音。动画还会用「闪烁」高亮当前判断的步骤（比如检查是否为偶数、是否是2），让大家直观看到算法的关键逻辑～

## 3. 核心难点辨析与解题策略

<difficulty_intro>
这道题的「坑」都藏在「细节」里！结合题解的共性，我总结了3个核心难点和解决策略，帮大家避免踩坑～
</difficulty_intro>

1.  **关键点1：理解「正偶数」的隐含要求**  
    * **分析**：题目说「分出来的两块西瓜的重量都是偶数公斤」，但没明说「必须是正数」——但现实中不可能分0公斤的西瓜！所以要排除「0+偶数」的情况，比如2=0+2，但0不是正数，所以2不符合条件。  
    * 💡 **学习笔记**：读题时要「想现实」——编程题的条件 often 藏在「常识」里！

2.  **关键点2：奇偶性的正确应用**  
    * **分析**：偶数+偶数=偶数，所以只有偶数才有可能满足条件。但反过来，不是所有偶数都满足（比如2）——这是「充分不必要条件」。解决方法是先判断奇偶性，再排除特殊情况。  
    * 💡 **学习笔记**：数学规律是「工具」，但要结合问题的「具体要求」使用！

3.  **关键点3：特判特殊情况**  
    * **分析**：2是唯一的「偶数但无法分成两个正偶数」的数，所以必须单独判断。很多人第一次做这题会忘特判2，导致WA（ Wrong Answer）。解决方法是「先想极端情况」——比如最小的输入（1、2、3），测试这些情况是否符合逻辑。  
    * 💡 **学习笔记**：特殊情况往往是「题目的陷阱」，一定要单独测试！

### ✨ 解题技巧总结
- **技巧A：用数学规律简化问题**：先想「有没有数学公式能直接解决问题」，比如奇偶性，比暴力枚举快得多。
- **技巧B：先处理特殊情况**：把容易错的特殊情况（比如2）放在最前面判断，避免逻辑混乱。
- **技巧C：用常识补全隐含条件**：比如「西瓜重量是正数」，题目没说，但必须考虑。

**本题通用核心C++实现参考**
* **说明**：本代码来自「zhanghanbin」和「Unknown_Error」的题解，结合了「特判2」和「奇偶性判断」的核心逻辑，是最简洁的正确实现。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    using namespace std;

    int main() {
        int n;
        cin >> n;
        if (n % 2 == 0 && n != 2) {
            cout << "YES" << endl;
        } else {
            cout << "NO" << endl;
        }
        return 0;
    }
    ```
* **代码解读概要**：  
  1. 读入输入的西瓜重量`n`；  
  2. 判断`n`是否是偶数（`n%2==0`）且不是2（`n!=2`）；  
  3. 如果满足条件，输出「YES」，否则输出「NO」。


<code_intro_selected>
接下来看几个「有亮点的代码片段」，学习不同的实现技巧～
</code_intro_selected>

**题解三：来自 Loner_Knowledge（位运算技巧）**
* **亮点**：用「位运算」代替取模，更高效；用「三目运算符」简化代码。
* **核心代码片段**：
    ```cpp
    printf("%s", (w&1 || w==2) ? "NO" : "YES");
    ```
* **代码解读**：  
  - `w&1`：位运算，判断`w`的二进制最后一位是否是1（即奇数）——比如`3`的二进制是`11`，`3&1=1`（奇数）；`4`的二进制是`100`，`4&1=0`（偶数）。  
  - `||`：逻辑或，只要`w`是奇数（`w&1`为真）或`w=2`，就输出「NO」，否则输出「YES」。  
  - 三目运算符`? :`：相当于「如果条件成立，选前面的值，否则选后面的值」——比`if-else`更简洁！
* 💡 **学习笔记**：位运算和三目运算符是「简化代码的小工具」，但要确保自己能看懂再用～

**题解四：来自 土川崇贺（暴力枚举）**
* **亮点**：用暴力枚举验证数学结论，适合理解问题本质。
* **核心代码片段**：
    ```cpp
    for (int i=2; i<=n/2; i+=2) {
        if ((n-i)>=2 && (n-i)%2==0) {
            cout << "YES";
            return 0;
        }
    }
    cout << "NO";
    ```
* **代码解读**：  
  - `i`从2开始，每次加2（只枚举正偶数）；  
  - `n-i`是另一块的重量，要满足「≥2」（正偶数）且「是偶数」；  
  - 如果找到符合条件的`i`，直接输出「YES」并结束程序；否则最后输出「NO」。  
* 💡 **学习笔记**：暴力枚举虽然慢，但能帮你「亲眼看到」问题的解，适合刚学编程时理解逻辑！

#### 2. **核心演示内容**  
- **场景初始化**：屏幕左侧是「西瓜区」（显示输入的西瓜重量，比如输入8就显示8个红色像素块），右侧是「控制面板」（开始/暂停、单步、重置按钮，速度滑块）。

- **判断过程**：  
  ① 西瓜先「闪烁绿色」——表示正在判断是否为偶数；如果是奇数，西瓜直接变成红色，显示「NO」，播放「错误音效」（短促的「哔」声）。  
  ② 如果是偶数，西瓜分裂成两块：比如输入8，分裂成2（绿色）和6（绿色），两块都闪烁绿色，显示「YES」，播放「胜利音效」（轻快的「叮～」声）；如果是2，分裂成1（红色）和1（红色），显示「NO」，播放错误音效。
- **交互控制**：  
  - 单步执行：点击「单步」按钮，一步步看西瓜的判断和分裂过程；  
  - 自动播放：点击「自动」，动画按设定速度（可通过滑块调整）播放；  
  - 重置：点击「重置」，回到初始状态，重新输入数字。

#### 3. **游戏化元素**  
- **过关奖励**：每成功判断一个数（比如输入8输出YES），屏幕右下角会弹出「像素星星」（黄色小方块闪烁），增加成就感；  
- **音效反馈**：不同操作对应不同音效——输入数字是「嗒」声，判断偶数是「嘀」声，分裂是「啪」声，结果正确是「叮」声，错误是「哔」声。

#### 4. **设计意图**  
用「像素动画」把抽象的「奇偶判断」变成「看得见的西瓜分裂」，让大家直观理解：  
- 为什么偶数才能分裂成两个偶数？（西瓜分裂成两个绿色块）；  
- 为什么2不行？（分裂成两个红色块）；  
- 「正偶数」的要求是什么？（分裂后的块都≥2，即绿色块大小≥2）。

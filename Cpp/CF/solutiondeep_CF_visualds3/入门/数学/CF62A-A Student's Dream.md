# 题目信息

# A Student's Dream

## 题目描述

**题目大意：**

有两个人要握手，女孩左手有 $a_l$ 个手指，右手有 $a_r$ 个手指；男孩左手有 $b_l$ 个手指，右手有 $b_r$ 个手指。握手规则为：男孩可以用左手握女孩的右手，也可以用右手握女孩的左手。在女孩的两根手指中间必须有男孩的一根手指，并且男孩的三根手指之间必须有女孩的一根手指。

现在要判断如果可以握手输出 `YES`，不能输出 `NO`。

## 样例 #1

### 输入

```
5 1
10 5
```

### 输出

```
YES```

## 样例 #2

### 输入

```
4 5
3 3
```

### 输出

```
YES```

## 样例 #3

### 输入

```
1 2
11 6
```

### 输出

```
NO```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：`模拟` (按题目规则逐一验证条件)

🗣️ **初步分析**：
> 解决这道题的关键是**“模拟握手规则”**——就像玩“规则匹配游戏”：题目说“满足A条件或者B条件就能握手”，我们要做的就是**逐一检查这两个条件是否成立**。  
> 简单来说，`模拟`算法的核心就是“按题目说的做”：题目给了规则，我们把规则变成代码里的`if`判断，一步一步验证。  
> 在本题中，“握手规则”分为**两种情况**（男右握女左 / 男左握女右），每种情况需要同时满足两个小条件：  
> ① 女孩的手指空隙数 ≤ 男孩的手指数（即男孩手指能塞进女孩的指缝里）；  
> ② 女孩的手指能“卡住”男孩的三根手指（数学上转化为`女孩手指数×2+3 > 男孩手指数`）。  
> 只要**两种情况中有一个满足**，就能输出`YES`～  

> 关于可视化设计：我会用**FC红白机风格**做一个像素动画——屏幕上有两个像素小人（女孩和男孩），他们的左右手用不同颜色的像素块表示手指数量。当检查某一种握手情况时，对应的手会“发光”，满足条件的话会播放“叮”的音效，最终成功会有“胜利”动画～

## 3. 核心难点辨析与解题策略

<difficulty_intro>
这道题的“坑”主要在**“把文字规则转成数学条件”**和**“考虑所有可能的握手情况”**。结合优质题解，我总结了3个核心难点和解决策略：
</difficulty_intro>

1.  **难点1：如何把“女孩指缝塞男孩手指”转成条件？**  
    * **分析**：女孩有`x`根手指，指缝数量是`x-1`（比如5根手指有4个缝）。题目要求“每个指缝里有一根男孩的手指”，所以男孩的手指数必须≥`x-1`（比如女孩5根手指，男孩至少4根）。  
    * 💡 **学习笔记**：“n个物体有n-1个空隙”是常见的“植树问题”模型，遇到“缝隙”“间隔”类问题要想到这个规律！

2.  **难点2：如何理解“男孩三根手指之间有女孩的手指”？**  
    * **分析**：男孩有`y`根手指，要让“每三根男孩手指之间有一根女孩手指”，需要女孩的手指能“填满”男孩的空隙。通过数学推导，这个条件可以简化为`女孩手指数×2+3 > 男孩手指数`（比如女孩1根手指，男孩最多5根：1×2+3=5，所以男孩必须<5）。  
    * 💡 **学习笔记**：复杂的文字规则可以通过“举例子”推导——比如拿样例1的输入（女孩左手5根，男孩右手10根），验证`5×2+3=13>10`，满足条件！

3.  **难点3：如何覆盖所有可能的握手情况？**  
    * **分析**：题目说“男孩可以用左手握女孩的右手，也可以用右手握女孩的左手”，所以需要检查**两种情况**：  
      - 情况1：女孩左手（`al`） vs 男孩右手（`br`）；  
      - 情况2：女孩右手（`ar`） vs 男孩左手（`bl`）。  
    * 只要**其中一种情况满足两个小条件**，就能输出`YES`。  
    * 💡 **学习笔记**：遇到“或”的情况，要记得用`||`连接两个条件，不要漏掉任何一种可能！

### ✨ 解题技巧总结
<summary_best_practices>
通过这道题，我总结了3个“规则转代码”的通用技巧：
</summary_best_practices>
- **技巧1：变量名要“贴题”**：用`al`代表“女孩左手”、`br`代表“男孩右手”，直接对应题目中的变量，避免混淆。  
- **技巧2：条件判断要“分层”**：先写“情况1的两个小条件”，再写“情况2的两个小条件”，用`&&`连接同情况的条件，用`||`连接不同情况。  
- **技巧3：提前终止更高效**：如果某一种情况满足条件，可以直接输出`YES`并`return`，不用再检查其他情况（比如LZT666的题解）。


## 4. C++核心代码实现赏析

<code_intro_overall>
先看一个**通用核心代码**——它综合了所有优质题解的思路，用最简洁的方式实现了题目要求：
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：本代码直接映射题目中的变量和规则，是“规则转代码”的典型示例。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    using namespace std;

    int main() {
        int al, ar, bl, br; // 女孩左、右手指；男孩左、右手指
        cin >> al >> ar >> bl >> br;
        
        // 情况1：女孩左手 vs 男孩右手；情况2：女孩右手 vs 男孩左手
        if ((al * 2 + 3 > br && br >= al - 1) || (ar * 2 + 3 > bl && bl >= ar - 1)) {
            cout << "YES" << endl;
        } else {
            cout << "NO" << endl;
        }
        
        return 0;
    }
    ```
* **代码解读概要**：
    > 代码分三步：① 读取四个手指数量；② 检查两种握手情况的条件（每种情况需要同时满足两个小条件）；③ 根据条件输出结果。核心逻辑就是**用`&&`连接同情况的条件，用`||`连接不同情况**。

<code_intro_selected>
接下来看优质题解中的“亮点片段”，帮你加深理解：
</code_intro_selected>

**题解一：(来源：miaohongxuan)**
* **亮点**：用“植树问题”解释条件，把文字规则变成数学公式。
* **核心代码片段**：
    ```cpp
    if((al-1 <= br && al*2+3 > br)||(ar-1 <= bl && ar*2+3 > bl)){
        cout << "YES";
    }
    ```
* **代码解读**：
    > 这段代码直接对应两个条件：  
    > - `al-1 <= br`：女孩左手的指缝数（`al-1`）≤ 男孩右手的手指数（`br`）；  
    > - `al*2+3 > br`：女孩左手能“卡住”男孩右手的三根手指；  
    > 两个条件用`&&`连接（必须同时满足），两种情况用`||`连接（满足一种即可）。  
* 💡 **学习笔记**：遇到“缝隙”问题，先算“n个物体有n-1个缝隙”，再转化为条件！

**题解二：(来源：LZT666)**
* **亮点**：提前终止，避免多余判断。
* **核心代码片段**：
    ```cpp
    if((a*2+3>d&&d+1>=a)||(b*2+3>c&&c+1>=b)) {
        puts("YES");
        return 0; // 满足条件，直接结束程序
    }
    puts("NO");
    ```
* **代码解读**：
    > 作者用`d+1>=a`代替`d>=a-1`（因为`d+1>=a`等价于`d>=a-1`），写法更简洁。而且一旦满足条件，直接`return 0`终止程序，不用再执行后面的`else`，效率更高。  
* 💡 **学习笔记**：等价的条件可以换种写法，让代码更简洁！

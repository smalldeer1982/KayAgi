# 题目信息

# HQ9+

## 题目描述

HQ9+是一种搞笑的编程语言，只有4种指令，都是单个字符的：
"H"输出"Hello, World!"；
"Q"输出这个程序本身的源代码；
"9"输出歌曲"99 Bottles of Beer"的歌词（你可以自己上网搜这首歌）；
"+"增加内部储存中的数值

注意"H"和"Q"指令是区分大小写的，必须使用大写。程序中不属于这些指令的字符请忽略。

给你一个用HQ9+写的程序，问这个程序是否会有任何输出。

## 说明/提示

第一个样例中只有一个指令"H"，输出了"Hello, World!"。

第二个样例中没有任何一个字符是这个程序的指令。

Hi!
```

### 输出

YES

## 样例 #2

Codeforces
```

## 3. 核心难点辨析与解题策略

<difficulty_intro>
这道题虽然简单，但新手容易踩三个“小坑”。结合题解的共性，我帮你提炼了**3个核心关键点**，帮你避免掉坑：
</difficulty_intro>

1.  **关键点1：明确“能输出的指令”——不要把“+”算进去！**
    * **分析**：题目里说“+”只是增加内部数值，不会输出任何内容。所以判断条件只能是`H、Q、9`，绝对不能加`+`！很多新手会误把“+”算进去，导致答案错误。
    * 💡 **学习笔记**：读题要“抠细节”，圈出题目中的“关键条件”（比如本题的“输出”二字）。

2.  **关键点2：遍历要“提前终止”——找到目标就立刻结束！**
    * **分析**：比如输入是“Haaaaa”，只要检查到第一个字符是H，就可以直接输出YES，不用再检查后面的“aaaaa”。题解里的`return 0`就是干这个的——既节省时间，又简化逻辑。
    * 💡 **学习笔记**：“提前终止”是枚举类问题的常用优化技巧，能让程序更高效。

3.  **关键点3：输入的读取方式——cin和getline都可以，但要注意“换行符”！**
    * **分析**：如果输入包含空格（比如“Hi There”），`cin>>s`会只读到“Hi”，而`getline(cin,s)`会读完整行。但本题的样例输入没有空格，所以两种方式都能用。如果题目要求“读整行”，一定要用`getline`！
    * 💡 **学习笔记**：根据题目需求选择输入方式，避免“读不全”的问题。

### ✨ 解题技巧总结
- **技巧1：枚举法的核心**：把问题拆解成“逐个检查”，适合“找符合条件的元素”的场景（比如找数组里的最大值、字符串里的特定字符）。
- **技巧2：利用库函数简化代码**：比如`string`的`find`、`length`函数，能减少重复代码（但要先理解函数的作用，不要盲目用）。
- **技巧3：边界条件测试**：比如输入是空字符串、只有“+”、只有“H”的情况，要确保代码都能正确处理。

## 4. C++核心代码实现赏析

<code_intro_overall>
先看一份**通用核心实现**——综合了题解一的“标准遍历”和“提前终止”，逻辑清晰，适合新手模仿：
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：本代码是“字符串遍历”的典型实现，覆盖了所有测试情况，代码可读性高。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    #include <string>
    using namespace std;

    int main() {
        string s;
        cin >> s; // 读入字符串
        for (int i = 0; i < s.size(); ++i) { // 遍历每个字符
            if (s[i] == 'H' || s[i] == 'Q' || s[i] == '9') {
                cout << "YES" << endl;
                return 0; // 找到目标，立刻结束
            }
        }
        cout << "NO" << endl; // 遍历完都没找到
        return 0;
    }
    ```
* **代码解读概要**：
    > 代码分三步：①用`string`读入输入；②用`for`循环遍历每个字符；③判断是否是目标字符，是则输出YES并终止，否则最后输出NO。`s.size()`是`string`的成员函数，用来获取字符串长度（和`length()`功能一样）。

---
<code_intro_selected>
接下来看三个优质题解的**核心片段**，学习不同的实现技巧：
</code_intro_selected>

**题解一：来源（作者：紫金依旧）**
* **亮点**：标准的字符串遍历，逻辑直接，适合新手入门。
* **核心代码片段**：
    ```cpp
    for(int i=0;i<len;i++)
        if(a[i]=='H'||a[i]=='Q'||a[i]=='9'){    
            cout<<"YES";
            return 0; 
        }
    ```
* **代码解读**：
    > 这里的`len`是`a.length()`（字符串长度），`for`循环从第0位到第`len-1`位，逐个检查字符。一旦遇到H/Q/9，就输出YES并`return 0`——`return 0`会直接结束整个程序，后面的代码都不会执行，非常高效。
* 💡 **学习笔记**：`return 0`在枚举中的作用是“提前终止”，能节省时间。

**题解二：来源（作者：松毛虫）**
* **亮点**：不用字符串，直接读字符，代码更简洁。
* **核心代码片段**：
    ```cpp
    char c;
    while(cin>>c){
        if(c == 'Q' || c == 'H' || c == '9' ){
            cout<<"YES";
            return 0;
        }
    }
    ```
* **代码解读**：
    > `while(cin>>c)`会一直从输入流中读字符，直到遇到换行或文件结束符。每读一个字符就检查一次——这种写法不用处理字符串的长度，适合“不确定输入长度”的场景。比如输入是“HQ9+”，读第一个字符'H'就会触发输出。
* 💡 **学习笔记**：`cin`读字符时，会自动跳过空格和换行符，所以不用担心输入中的空白。

**题解三：来源（作者：二字）**
* **亮点**：用`find`函数简化判断，代码超短。
* **核心代码片段**：
    ```cpp
    if(s.find("H")+s.find("Q")+s.find("9")!=-3) 
        cout<<"YES";
    else cout<<"NO";
    ```
* **代码解读**：
    > `s.find("H")`的作用是“在s中找'H'，找到返回位置（比如第0位），没找到返回-1”。如果三个都没找到，总和是`-1 + (-1) + (-1) = -3`；只要有一个找到，总和就不是-3（比如找到'H'，总和是`0 + (-1) + (-1) = -2`）。这种写法把“三次判断”浓缩成“一次算术运算”，非常巧妙！
* 💡 **学习笔记**：`find`函数是字符串处理的“神器”，但要注意它的返回值是`size_t`（无符号整数），所以比较时要小心（比如`find`返回-1会被当作很大的数，但本题中三个`find`的和不会超过`-3`，所以没问题）。

# 题目信息

# Wrong Subtraction

## 题目描述

Little girl Tanya is learning how to decrease a number by one, but she does it wrong with a number consisting of two or more digits. Tanya subtracts one from a number by the following algorithm:

- if the last digit of the number is non-zero, she decreases the number by one;
- if the last digit of the number is zero, she divides the number by 10 (i.e. removes the last digit).

You are given an integer number $ n $ . Tanya will subtract one from it $ k $ times. Your task is to print the result after all $ k $ subtractions.

It is guaranteed that the result will be positive integer number.

## 说明/提示

The first example corresponds to the following sequence: $ 512 \rightarrow 511 \rightarrow 510 \rightarrow 51 \rightarrow 50 $ .

## 样例 #1

### 输入

```
512 4
```

### 输出

```
50
```

## 样例 #2

### 输入

```
1000000000 9
```

### 输出

```
1
```

## 3. 核心难点辨析与解题策略

<difficulty_intro>
模拟题看似简单，但新手容易踩坑！结合题解的共性，我帮你提炼了3个核心难点和解决办法：
</difficulty_intro>

1.  **关键点1：如何处理大数溢出？**  
    * **分析**：比如样例2的输入是`1000000000`（10亿），如果用`int`类型存储，会超过`int`的最大值（约21亿？不，等一下，`int`通常是4字节，最大值是2^31-1=2147483647，10亿是1000000000，其实`int`能装下？但如果是更大的数比如`10000000000`（100亿），`int`就装不下了。这时候要用`long long`类型（8字节，能装到9e18）。优质题解里“引领天下”和“moqizhu2005”都用了`long long`，就是为了防溢出。  
    * 💡 **学习笔记**：处理大数时，先想`long long`，别让“装不下”毁了代码！

2.  **关键点2：选择“数字模拟”还是“字符串模拟”？**  
    * **分析**：如果数字不大（比如`n`≤1e18），用数字模拟（`long long`）更高效；如果数字大到`long long`都装不下（比如`n`是100位的数），就用字符串模拟（像“666yuchen”的题解）。本题中数字模拟足够，但字符串思路能帮你应对更极端的情况～  
    * 💡 **学习笔记**：根据数字大小选方法，灵活变通！

3.  **关键点3：如何确保循环执行k次？**  
    * **分析**：不管用`while(k--)`还是`for(;k>0;k--)`，核心都是“每执行一次操作，k减1”，直到k变成0。新手容易犯“循环次数多一次或少一次”的错，比如把`k--`写成`k++`，或者循环条件写错。解决办法是“手动走一遍样例”——比如样例1输入`512 4`，循环4次，每一步的结果是511→510→51→50，刚好符合输出。  
    * 💡 **学习笔记**：写循环前，先手动模拟样例，确认次数对不对！

### ✨ 解题技巧总结
- **技巧A：先想数据类型**：遇到数字题，先问自己“这个数最大能有多大？”，避免溢出。  
- **技巧B：规则翻译要精准**：题目说“最后一位非零减1”，就用`n%10!=0`判断；说“最后一位是零除以10”，就用`n/=10`，别加多余的逻辑。  
- **技巧C：手动模拟样例**：写完代码前，先手动走一遍样例，比如样例1的4次操作，确保每一步都对，再写代码。


## 4. C++核心代码实现赏析

<code_intro_overall>
先看一份“通用核心实现”——用数字模拟，兼顾效率和可读性，适合大部分情况～
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：本代码综合了“引领天下”和“thename”的思路，用`long long`处理大数，循环k次模拟操作，逻辑清晰。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    using namespace std;

    int main() {
        long long n;  // 用long long防溢出
        int k;
        cin >> n >> k;
        while (k--) {  // 循环k次，每次k减1
            if (n % 10 != 0) {  // 最后一位非零，减1
                n--;
            } else {  // 最后一位是零，除以10
                n /= 10;
            }
        }
        cout << n << endl;
        return 0;
    }
    ```
* **代码解读概要**：  
  1. 输入`n`（数字）和`k`（操作次数）；  
  2. 用`while(k--)`循环k次，每次检查`n`的最后一位（`n%10`）；  
  3. 根据最后一位的情况，执行减1或除以10；  
  4. 输出最终的`n`。

<code_intro_selected>
接下来看3份优质题解的核心片段，亮点各不相同～
</code_intro_selected>

**题解一：来源：引领天下**
* **亮点**：用`while(k--)`简化循环，代码超简洁！
* **核心代码片段**：
    ```cpp
    while (k--) if (n%10)n--;else n/=10;//模拟  
    ```
* **代码解读**：  
  这句代码把“循环k次”和“判断操作”合并成了一行！`k--`是“先判断k是否大于0，再减1”，所以循环会执行k次。`n%10`等价于`n%10!=0`（因为非零值在条件判断中是“真”），所以如果最后一位非零，就`n--`；否则`n/=10`。是不是超简洁？
* 💡 **学习笔记**：`while(k--)`是模拟题中常用的“循环k次”写法，能简化代码！

**题解二：来源：666yuchen**
* **亮点**：用字符串处理大数，思路拓展！
* **核心代码片段**：
    ```cpp
    if(st[st.size()-1]=='0')st.erase(st.size()-1,1);
    else st[st.size()-1]--;
    ```
* **代码解读**：  
  `st.size()-1`是字符串的最后一位索引（比如“512”的最后一位是索引2）。如果最后一位是`'0'`，就用`st.erase(st.size()-1,1)`删掉最后一个字符（比如“510”变成“51”）；否则把最后一位的字符减1（比如`'2'`变成`'1'`，因为字符的ASCII码是连续的）。这种方法不用怕数字太大，超适合处理超级大的数！
* 💡 **学习笔记**：字符串可以直接操作每一位，是处理超大数的好办法！

**题解三：来源：thename**
* **亮点**：用`continue`避免多余判断，逻辑更清晰！
* **核心代码片段**：
    ```cpp
    if(n%10==0)//如果个位数是零 
    {
        n/=10;//原数除以十 
        continue;//跳过本层循环，避免走下条语句 
    }
    n--;//如果不为零，n-1 
    ```
* **代码解读**：  
  如果最后一位是0，除以10后，用`continue`跳过后面的`n--`（因为这时候不需要减1）；否则执行`n--`。这种写法把“两种情况”完全分开，不会出现“先除以10又减1”的错误，新手看了也不会乱～
* 💡 **学习笔记**：`continue`能帮你跳过不需要的代码，让逻辑更清晰！

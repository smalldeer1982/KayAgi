# 题目信息

# Generous Kefa

## 题目描述

读入一个字符串（都是小写字母），让你求出里边最多的一种字母的个数是否小于等于 $k$。，若大于输出"NO",否则输出"YES"。   
第二行为一个字符串。

## 样例 #1

### 输入

```
4 2
aabb
```

### 输出

```
YES
```

## 样例 #2

### 输入

```
6 3
aacaab
```

### 输出

```
NO
```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：编程技巧应用（侧重基础数组运用与统计计数）

🗣️ **初步分析**：  
解决这道题的关键在于**统计每个字符的出现次数**——就像给26个小写字母各准备一个“小盒子”，每遇到一个字母，就往对应的盒子里放一颗“计数石子”。最后看哪个盒子里的石子最多：如果最多的那盒超过了`k`颗，输出“NO”；否则输出“YES”。  

所有题解的核心思路高度一致：用数组（或map）记录每个字母的次数，再判断最大值是否≤`k`。核心难点有三个：  
1. **字母转下标**：如何把`a-z`转换成数组能识别的0-25下标？（用`字符-'a'`即可，比如`'a'-'a'=0`，`'b'-'a'=1`）；  
2. **提前终止**：要不要边输入边判断？（一旦某个字母次数超过`k`，立刻结束程序，不用处理后面的字符）；  
3. **数据结构选择**：用数组还是map？（数组更快，因为字母范围固定0-25）。  

**可视化设计思路**：我们会做一个8位像素风的“字母计数机”——屏幕上有26个彩色像素盒子（对应`a-z`），每输入一个字母，对应的盒子会“弹出”一个像素点（表示次数+1）。如果某个盒子的像素点超过`k`，盒子瞬间变红，播放错误音效；如果所有字母都没超过，盒子变绿，播放胜利音效。

为了帮大家快速掌握核心思路，我筛选了3份**思路清晰、代码简洁**的优质题解：

### 题解一（作者：piggymichael）
* **点评**：这份题解像“闪电战”——边输入边统计，一旦发现某个字母次数超过`k`，立刻输出“NO”并结束程序，完全不做多余计算！代码里`a[x-'a']++`的写法超聪明（直接把字母转成数组下标），边界条件处理得特别严谨（用`k+1`判断超过的瞬间）。这种“边做边判断”的写法在竞赛中超省时间，是新手必学的技巧！


### 题解二（作者：goodlearndaydayup）
* **点评**：思路和题解一几乎一样，但代码更简洁——把“输入字符+统计次数”写在同一行（`cin>>s[i],ans[s[i]-'a']++`），看起来清爽极了！判断条件直接用`ans[s[i]-'a']>k`，特别直观。变量名虽然叫`ans`（其实是统计数组），但不影响理解，适合刚学C++的同学参考。


### 题解三（作者：人间凡人）
* **点评**：这份题解的亮点是“实时记录最大值”——每统计一个字符，就用`ans`变量更新当前最大次数。这样最后只需要判断`ans≤k`，不用再遍历整个数组找最大值！这种思路在很多统计问题中都能用（比如求数组最大值），能减少一次循环，超高效～


## 3. 核心难点辨析与解题策略

在解决这类统计问题时，以下3个关键点最容易“卡壳”，结合优质题解的经验，我们一起拆解：

### 1. 关键点1：如何将字母转换成数组下标？
* **分析**：小写字母的ASCII码是连续的（`'a'=97`，`'b'=98`，…，`'z'=122`）。用`字符-'a'`就能得到0-25的下标（比如`'c'-'a'=2`，对应数组第2位）。所有优质题解都用了这个技巧，是统计字母次数的“万能钥匙”！  
* 💡 **学习笔记**：字母转下标，记住`x-'a'`就对了～


### 2. 关键点2：要不要提前终止程序？
* **分析**：比如输入`"aaaaa"`，`k=3`。当输入第4个`'a'`时，次数已经是4，超过`k`了，这时候不用再输入第5个`'a'`，直接输出“NO”就行。题解一、二的“边输入边判断”写法就能做到这点，比“先输入再统计”更高效！  
* 💡 **学习笔记**：能提前终止的情况，一定要提前，节省时间～


### 3. 关键点3：选数组还是map？
* **分析**：数组的下标固定0-25，访问速度超级快；map是STL容器，虽然能处理任意字符，但速度比数组慢。因为题目里只有小写字母，所以**数组是最优选择**！  
* 💡 **学习笔记**：已知范围的统计，优先用数组！


### ✨ 解题技巧总结
- 技巧1：字母统计用数组，`x-'a'`转下标；  
- 技巧2：边输入边判断，提前终止省时间；  
- 技巧3：实时记录最大值，不用遍历找答案；  


## 4. C++核心代码实现赏析

### 本题通用核心C++实现参考
* **说明**：综合了优质题解的“边输入边判断”和“高效统计”思路，是最简洁的实现。  
* **完整核心代码**：
  ```cpp
  #include <iostream>
  using namespace std;

  int main() {
      int n, k;
      cin >> n >> k;
      int count[26] = {0}; // 初始化所有字母次数为0
      for (int i = 0; i < n; ++i) {
          char c;
          cin >> c;
          int idx = c - 'a'; // 字母转下标
          count[idx]++;      // 次数+1
          if (count[idx] > k) { // 超过k，立刻输出NO
              cout << "NO" << endl;
              return 0;
          }
      }
      cout << "YES" << endl; // 所有字母都≤k
      return 0;
  }
  ```
* **代码解读概要**：  
  1. 读取字符串长度`n`和阈值`k`；  
  2. 定义`count`数组（26位），存每个字母的次数；  
  3. 循环`n`次，每次读一个字符，转成下标后统计次数；  
  4. 一旦次数超过`k`，立刻输出“NO”并结束；  
  5. 循环结束后输出“YES”。


### 优质题解片段赏析

#### 题解一（作者：piggymichael）
* **亮点**：边输入边判断，提前终止效率最高。  
* **核心代码片段**：
  ```cpp
  cin >> x;
  a[x - 'a']++;
  if (a[x - 'a'] == k + 1) { // 超过k的瞬间触发
      cout << "NO" << endl;
      return 0;
  }
  ```
* **代码解读**：  
  读入字符`x`，用`x-'a'`转成下标，`a`数组加1。判断`a[x-'a']`是否等于`k+1`——比如`k=2`，当次数变成3时，就触发“NO”。这种写法和`count[idx]>k`是一样的，只是换了种表达，都对～  
* 💡 **学习笔记**：提前终止的关键是“一旦超过就立刻退出”。


#### 题解三（作者：人间凡人）
* **亮点**：实时记录最大值，减少一次循环。  
* **核心代码片段**：
  ```cpp
  f[s[i] - 'a']++;
  if (f[s[i] - 'a'] > ans) ans = f[s[i] - 'a']; // 更新最大值
  ```
* **代码解读**：  
  `s`是`string`类型的字符串，`s[i]`是第`i`个字符。`f`数组统计次数，每次加1后，比较当前次数和`ans`（最大值），如果更大就更新`ans`。最后只需要判断`ans≤k`，不用再遍历`f`数组找最大值～  
* 💡 **学习笔记**：实时记录最大值，能节省时间！

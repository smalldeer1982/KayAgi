# Clock Conversion

## 题目描述

给定一个 24 小时制的时间，请输出其对应的 12 小时制时间。

- [24 小时制](https://en.wikipedia.org/wiki/24-hour_clock#Description) 将一天分为 24 个小时，从 $00$ 到 $23$，每个小时有 60 分钟，从 $00$ 到 $59$。
- [12 小时制](https://en.wikipedia.org/wiki/12-hour_clock#Description) 将一天分为上午（AM）和下午（PM）两个部分。每个部分的小时顺序为 $12, 01, 02, 03, \dots, 11$。每个小时有 60 分钟，分钟编号从 $00$ 到 $59$。

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：模拟应用（按规则处理不同情况）

🗣️ **初步分析**：
解决这道题的核心是**“模拟时间转换的规则”**——就像玩“数字变形游戏”：给定一个24小时制的时间，我们要按照12小时制的规则，把小时数“变”成对应的数字，再给它贴个“AM”或“PM”的标签。  

### 核心规则拆解：
- **分钟数不变**：不管怎么转，分钟数都是原样（比如09:41的分钟还是41）。  
- **小时数转换**：  
  - 如果是0点（h=0）→ 变成12点；  
  - 如果是1-11点→ 保持不变；  
  - 如果是12点→ 保持不变；  
  - 如果是13-23点→ 减12（比如18点→6点）。  
- **AM/PM判断**：  
  - 0-11点→ AM（上午）；  
  - 12-23点→ PM（下午）。  

### 核心难点与可视化设计：
这道题的**难点**是处理**边界情况**（比如0点转12点、12点的PM判断）和**前导零**（比如9点要输出“09”而不是“9”）。  

为了直观理解，我设计了一个**8位像素风格的时钟动画**：  
- 左边是24小时制时钟（像素数字，比如“09”），右边是12小时制时钟（初始为“??”）；  
- 转换时，左边的小时数会“跳”到右边：比如0点→右边变成“12”，18点→右边变成“06”；  
- AM用**绿色像素块**标记，PM用**红色像素块**标记；  
- 每一步都有“叮”的像素音效（比如小时数变化时），完成转换会播放“胜利”音效～

## 3. 核心难点辨析与解题策略

<difficulty_intro>
模拟题的“坑”往往在细节里！我们来拆解三个最容易错的点，帮你避开雷区～
</difficulty_intro>

1.  **关键点1：边界情况处理（0点和12点）**  
    * **分析**：0点不是“0点AM”，而是“12点AM”；12点不是“12点AM”，而是“12点PM”。很多同学会漏掉这两个特例，导致输出错误。解决方法是**单独判断这两个情况**，或者用数学公式（比如Harrylzh的模运算）覆盖所有情况。  
    * 💡 **学习笔记**：模拟题要“把规则想全”，不要漏掉“特殊情况”！

2.  **关键点2：前导零输出（比如9→09）**  
    * **分析**：12小时制要求小时和分钟都是两位数（比如“09:05”而不是“9:5”）。解决方法有两种：  
      - 用`printf("%02d", num)`（自动补前导零）；  
      - 判断数字是否小于10，手动加“0”（比如`if(h<10) cout<<"0"`）。  
    * 💡 **学习笔记**：格式化输出是C++的“偷懒神器”，能帮你少写很多判断！

3.  **关键点3：AM/PM的正确判断**  
    * **分析**：判断条件是“h<12→AM”，“h≥12→PM”？不对！比如h=12属于PM，h=0属于AM，这个条件刚好覆盖。但要注意：**h=12时，小时数不变，但PM要选对**。  
    * 💡 **学习笔记**：AM/PM的判断只和“原24小时制的小时数”有关，和转换后的小时数无关！

### ✨ 解题技巧总结
- **技巧A：拆解问题**：把大问题拆成“小时转换”“分钟不变”“AM/PM判断”三个小问题，逐个解决。  
- **技巧B：格式化输出**：用`printf`的`%02d`处理前导零，比手动判断更高效。  
- **技巧C：数学简化**：能用模运算、加减法简化的逻辑，就别写一堆`if`判断（比如Harrylzh的题解）。  


## 4. C++核心代码实现赏析

<code_intro_overall>
先看一份**通用核心代码**，它综合了优质题解的优点，兼顾简洁和可读性～
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：本代码用“分情况讨论+格式化输出”解决问题，覆盖所有边界情况，适合新手入门。
* **完整核心代码**：
  ```cpp
  #include <cstdio>
  int main() {
      int T;
      scanf("%d", &T); // 读入测试用例数量
      while (T--) {
          int h, m;
          char colon; // 用来读取冒号
          scanf("%d%c%d", &h, &colon, &m); // 读入h:m
  
          // 1. 转换小时数
          int new_h;
          if (h == 0) new_h = 12;
          else if (h > 12) new_h = h - 12;
          else new_h = h;
  
          // 2. 判断AM/PM
          const char* period = (h < 12) ? "AM" : "PM";
  
          // 3. 输出（%02d保证两位数）
          printf("%02d:%02d %s\n", new_h, m, period);
      }
      return 0;
  }
  ```
* **代码解读概要**：  
  代码分三步：读入测试用例→处理每个时间的小时转换和AM/PM→格式化输出。核心是`if`判断处理边界情况，`%02d`处理前导零。


<code_intro_selected>
接下来看三个优质题解的“精华片段”，学习它们的巧妙之处～
</code_intro_selected>

**题解一：封禁用户（来源：综合题解）**
* **亮点**：用`%02d`直接解决前导零，代码简洁到“一行顶三行”！
* **核心代码片段**：
  ```cpp
  scanf("%d:%d",&h,&m);
  if(h==0) printf("%02d:%02d",12,m);
  else if(h<=12) printf("%02d:%02d",h,m);
  else printf("%02d:%02d",h-12,m);
  if(h<12) printf(" AM\n");
  else printf(" PM\n");
  ```
* **代码解读**：  
  - `%02d`是“格式化输出”的魔法：比如h=9，`%02d`会输出“09”；m=5会输出“05”。  
  - 分三种情况处理小时数：h=0→12，h≤12→不变，h>12→减12。  
* 💡 **学习笔记**：格式化输出是模拟题的“利器”，一定要掌握！

**题解二：Harrylzh（来源：综合题解）**
* **亮点**：用数学公式浓缩所有小时转换，代码超短！
* **核心代码片段**：
  ```cpp
  cin>>h>>ch>>m;
  printf("%02lld:%02lld",(h-1+12)%12+1,m);
  if(h>=12) printf(" PM\n");
  else printf(" AM\n");
  ```
* **代码解读**：  
  - 公式`(h-1+12)%12+1`怎么来的？比如h=0→(0-1+12)=11→11%12=11→11+1=12；h=18→(18-1+12)=29→29%12=5→5+1=6。这个公式把所有情况都覆盖了，不用写`if`判断！  
* 💡 **学习笔记**：数学能帮你简化逻辑，多想想“有没有公式能代替判断”～

**题解三：Pink_Cut_Tree（来源：综合题解）**
* **亮点**：用字符串处理输入，适合复杂输入格式！
* **核心代码片段**：
  ```cpp
  cin>>s; // 读入整个时间字符串（比如"09:41"）
  int ten=s[0]-'0',one=s[1]-'0';
  int h=ten*10+one; // 提取小时数（s[0]是十位，s[1]是个位）
  if(h==0||h==12) hour=12;
  else hour=h%12;
  ```
* **代码解读**：  
  - 当输入是字符串时（比如题目给的是“09:41”），可以用`s[0]`和`s[1]`提取小时的十位和个位，转换成整数。这种方法适合处理“带分隔符的输入”（比如冒号、逗号）。  
* 💡 **学习笔记**：字符串处理是处理复杂输入的“万能钥匙”，要学会提取其中的数字！

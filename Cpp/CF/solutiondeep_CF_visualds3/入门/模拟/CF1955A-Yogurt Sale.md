# Yogurt Sale

## 题目描述

在“Vosmiorochka”商店中，一杯酸奶的价格为 $a$ 布尔，但现在有一个促销活动，可以用 $b$ 布尔购买两杯酸奶。

Maxim 需要恰好购买 $n$ 杯酸奶。在购买两杯酸奶时，他可以选择以原价购买，也可以选择以促销价购买。

Maxim 购买 $n$ 杯酸奶所需花费的最少布尔是多少？

## 说明/提示

在样例的第三个测试用例中，直接购买三杯酸奶共需 $15$ 布尔，比用 $11$ 布尔买两杯再用 $5$ 布尔买一杯更划算。

在样例的第四个测试用例中，需要购买四杯酸奶，每杯 $5$ 布尔。

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：贪心算法（Greedy Algorithm）

🗣️ **初步分析**：
> 解决这道题的关键是**贪心思想**——就像你去超市买薯片，总是选“每片单价更便宜”的包装组合！贪心算法的核心是「每一步都做当前最优的选择，最终得到全局最优」。在本题中，我们需要比较两种购买方式的“单位成本”：  
> - 单买两杯的成本是 `2*a`（买两杯分开算）；  
> - 促销买两杯的成本是 `b`（直接买组合装）。  
> 如果促销更划算（`b < 2*a`），那我们就尽可能多买促销装，剩下的1杯再单买；如果单买更划算，就直接全买单人份。  

  - **题解共性思路**：所有题解都围绕“比较2a和b”展开，逻辑高度一致——先判断哪种方式更便宜，再计算对应的总花费。  
  - **核心难点**：处理奇数数量的余数（比如买3杯时，促销装买1组（2杯），剩下1杯单买）、避免条件判断错误（比如把“b < 2a”写成“b < a”）。  
  - **可视化设计思路**：我们会用8位像素风格模拟“酸奶商店”——用像素块代表酸奶杯，红色块表示促销装（2杯），蓝色块表示单买杯，动态展示“选促销装→补单买”的过程，同时高亮当前计算的步骤（比如n/2的促销装数量、n%2的余数）。  
  - **游戏化元素**：加入“收银机音效”（每次计算总花费时叮一声）、“促销灯闪烁”（当b更划算时，屏幕边缘的像素灯闪烁），让动画更有代入感！

## 3. 核心难点辨析与解题策略

<difficulty_intro>
这道题看似简单，但新手容易在三个关键点“栽跟头”。结合优质题解的经验，我们来逐一破解~
</difficulty_intro>

1.  **关键点1：如何判断“促销是否划算”？**
    * **分析**：很多同学会误以为“促销一定更便宜”，但其实要看“两杯的总价格”——比如单买一杯5元，促销两杯11元（2*5=10 < 11），这时单买两杯更划算！正确的判断条件是`b < 2*a`（促销价<单买两杯的价格）。优质题解都严格遵循了这个条件。
    * 💡 **学习笔记**：贪心的“最优选择”必须基于“单位成本”的比较，而不是“表面价格”！

2.  **关键点2：如何处理奇数数量的余数？**
    * **分析**：当n是奇数时（比如3杯），促销装只能买`n/2`组（1组=2杯），剩下的1杯必须单买。优质题解用`n%2`（取余数）快速算出剩余杯数，再乘以a——这是整数运算的小技巧！
    * 💡 **学习笔记**：`n/2`（整数除法）算“最多能买多少组”，`n%2`算“剩下多少杯”，这对“分组问题”超有用！

3.  **关键点3：如何避免条件判断错误？**
    * **分析**：新手容易把条件写反（比如`b > 2*a`时用促销），或者漏判情况（比如忘记n=0的情况，但本题n≥1）。题解二用`min`函数直接比较两种情况的花费，完美避免了条件判断的错误——因为不管促销是否划算，`min(t1, t2+t3)`都会选更小的那个！
    * 💡 **学习笔记**：当条件判断容易出错时，不妨换个思路——计算所有可能的结果，再选最优！

### ✨ 解题技巧总结
<summary_best_practices>
通过这道题，我们可以总结出贪心问题的通用技巧：
</summary_best_practices>
-   **技巧1：计算“单位成本”**：比如本题比较“两杯的成本”，而不是“一杯的价格”；
-   **技巧2：用整数运算简化分组**：`n/2`算组数，`n%2`算余数，适用于所有“按组购买”的问题；
-   **技巧3：用min/max函数避免条件判断**：当有两种选择时，直接计算两种结果取最优，比写if-else更安全。

## 4. C++核心代码实现赏析

<code_intro_overall>
先看一个**通用核心实现**——它综合了题解二的“min函数”和题解一的“清晰变量名”，是最适合初学者模仿的版本！
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：本代码结合了“计算两种情况取最小值”的思路，逻辑简洁且不易出错，覆盖了所有测试用例。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    using namespace std;

    int main() {
        int t;
        cin >> t; // 输入测试用例数
        while (t--) {
            int n, a, b;
            cin >> n >> a >> b; // n:需要的杯数，a:单杯价格，b:两杯促销价
            int all_single = n * a; // 全单买的花费
            int groups = n / 2; // 促销装的组数
            int remainder = n % 2; // 剩余单买的杯数
            int combo_plus_single = groups * b + remainder * a; // 促销+单买的花费
            cout << min(all_single, combo_plus_single) << endl; // 取较小值
        }
        return 0;
    }
    ```
* **代码解读概要**：
    > 代码分三步：①读取测试用例数；②对每个测试用例，计算“全单买”和“促销+单买”的花费；③输出较小值。核心逻辑是“比较两种极端情况的花费”，避免了复杂的条件判断。

<code_intro_selected>
接下来，我们看两个**优质题解的核心片段**，学习它们的亮点！
</code_intro_selected>

**题解一：（作者：yitian_）**
* **亮点**：用条件判断直接选择最优策略，逻辑直观。
* **核心代码片段**：
    ```cpp
    if(b < a*2) { // 促销更划算
        int c = n/2, d = n%2;
        cout << c*b + d*a << endl;
    } else { // 单买更划算
        cout << n*a << endl;
    }
    ```
* **代码解读**：
    > 这段代码的逻辑是“先判断促销是否划算，再计算对应花费”。`c = n/2`是促销装的组数，`d = n%2`是剩余杯数——比如n=3时，c=1（买1组促销装），d=1（再买1杯单装），总花费是`1*b +1*a`。如果促销不划算（比如b=11，a=5，2*a=10 <11），就直接全单买。
* 💡 **学习笔记**：条件判断适合“非此即彼”的情况，逻辑清晰但要注意条件写对！

**题解二：（作者：little_stickman）**
* **亮点**：用`min`函数简化逻辑，避免条件判断错误。
* **核心代码片段**：
    ```cpp
    t1 = a * x; // 全单买
    t2 = (x/2) * b; // 促销装的花费
    t3 = (x%2) * a; // 剩余单买的花费
    cout << min(t1, t2 + t3) << endl;
    ```
* **代码解读**：
    > 这段代码不需要判断“促销是否划算”——不管b和2a的大小，直接计算两种情况的花费，取较小值。比如当b=11（a=5，2a=10）时，`t2+t3=1*11+1*5=16`，而`t1=3*5=15`，min会选15（全单买更划算），刚好对应样例三的情况！
* 💡 **学习笔记**：`min`函数是处理“二选一”问题的神器，尤其适合新手避免条件写错！

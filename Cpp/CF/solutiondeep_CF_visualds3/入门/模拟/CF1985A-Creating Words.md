# Creating Words

## 题目描述

注意，交换后得到的新单词不一定不同。

## 3. 核心难点辨析与解题策略

<difficulty_intro>
这道题虽然简单，但新手容易在**细节**上掉坑。结合常见错误，我总结了3个核心关键点：
</difficulty_intro>

1.  **关键点1**：如何正确交换两个字符串的第一个字符？
    * **分析**：交换两个变量必须用“临时变量”过渡！比如交换a[0]和b[0]：  
      ```cpp
      char tmp = a[0];  // 把a的第一个字符“存起来”
      a[0] = b[0];      // 把b的第一个字符给a
      b[0] = tmp;       // 把存好的a的第一个字符给b
      ```  
      要是直接写`a[0] = b[0]; b[0] = a[0];`，a[0]原来的值会被覆盖，交换就失败了！
    * 💡 **学习笔记**：交换变量=“暂存→覆盖→恢复”，三步不能少！

2.  **关键点2**：如何处理多组测试用例？
    * **分析**：题目输入第一行是测试用例数量n（比如样例里的6），后面跟着n行输入。用`for`循环从1到n，每次读一组a和b就行：  
      ```cpp
      int n;
      cin >> n;
      for (int i = 0; i < n; ++i) {
          string a, b;
          cin >> a >> b;
          // 交换操作...
          cout << a << " " << b << endl;
      }
      ```
    * 💡 **学习笔记**：多组测试用例=“循环n次，每次处理一组数据”。

3.  **关键点3**：如何确保字符串长度是3？
    * **分析**：题目保证输入的a和b长度都是3，所以不用额外判断。但如果自己测试时输入错了（比如输入“bi”而不是“bit”），程序会出错——这提醒我们：**永远要相信题目给的输入条件，但写代码时要避免自己犯输入错误**。
    * 💡 **学习笔记**：题目给的“前提条件”（比如长度3）是我们的“安全区”，不用额外处理，但自己输入时要符合要求。

## 4. C++核心代码实现赏析

<code_intro_overall>
下面是一个**完整的、能直接运行的C++核心实现**，涵盖了所有关键逻辑：
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：本代码综合了“处理多组测试用例”和“交换首字符”的核心逻辑，是最简洁的实现方式。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    #include <string>  // 处理字符串需要的头文件
    using namespace std;  // 简化cout/cin的写法

    int main() {
        int n;
        cin >> n;  // 读测试用例数量
        for (int i = 0; i < n; ++i) {
            string a, b;
            cin >> a >> b;  // 读一组a和b
            // 交换第一个字符
            char tmp = a[0];
            a[0] = b[0];
            b[0] = tmp;
            // 输出结果
            cout << a << " " << b << endl;
        }
        return 0;
    }
    ```
* **代码解读概要**：
    > 代码分三步：① 读入测试用例数量n；② 循环n次，每次读两个字符串a和b；③ 交换a和b的第一个字符，输出结果。核心是“交换首字符”的三行代码，这是整个程序的“灵魂”。

# Three Brothers

## 题目描述

### 题目翻译：

三兄弟相约见面。我们按以下顺序给他们编号：老大是 $1$ 号，老二是 $2$ 号，老三是 $3$ 号。

到了约定的时间，有一个兄弟迟到了。现在给出准时到达的两个兄弟的编号，你需要确定那个迟到的兄弟的编号。

## 样例 #1

### 输入

```
3 1```

### 输出

```
2```

今天我们来分析一道**锻炼基础逻辑与简化思维**的题目——《Three Brothers》。它看似简单，却藏着“用数学规律偷懒”“用数组记录状态”等实用技巧，适合刚入门的同学巩固编程基本功~


## 1. 题目解读与核心算法识别

✨ **本题主要考察**：编程技巧应用（基础逻辑与数学简化）  

🗣️ **初步分析**：  
题目是说，三个兄弟编号1、2、3，给两个准时到的编号，找迟到的那个。关键技巧有两个：  
- **数学简化法**：三个数的总和固定是 `1+2+3=6`，迟到的编号就是 `6 - 已知两数之和`——像三个苹果总重6克，拿走两个，剩下的重量一减就知道！  
- **数组标记法**：用数组记录“谁来了”（比如`a[x]=1`表示x号来了），最后找没被标记的（`a[i]=0`）。  

核心难点是**“如何快速定位缺失的数”**：新手可能会写很多`if`判断，但学会用“总和”或“数组”能大幅简化代码。  

可视化设计思路：我们会用**8位像素风**做动画——三个小人代表兄弟，亮起来的是来了的，暗的是迟到的；同时用像素数字展示`6-a-b`的计算过程，搭配“叮”的音效提醒关键步骤，让你“看得到”算法怎么工作~

## 3. 核心难点辨析与解题策略

解决这道题时，新手常遇到3个关键点，结合优质题解总结策略：


### 1. 关键点1：如何快速找缺失的数？  
**分析**：  
- 用“总和法”（`6-a-b`）：适合**固定集合**（比如1-3），一步算出结果，不用遍历。  
- 用“数组法”：适合**动态集合**（比如人数更多时），用数组记录状态，遍历找0。  
💡 **学习笔记**：找固定集合的缺失元素，先想“有没有数学规律”！


### 2. 关键点2：如何处理输入顺序？  
**分析**：  
- 总和法不管顺序（`3+1`和`1+3`结果一样）；  
- 数组法也不管（`a[x]=1`和`a[y]=1`不看顺序）；  
- 枚举法要覆盖所有排列（比如`1和2`“`2和1`都要写）。  
💡 **学习笔记**：写条件时，要考虑输入的“所有可能顺序”！


### 3. 关键点3：代码如何更简洁？  
**分析**：  
总和法用数学规律减少代码量（一行核心逻辑），比枚举法少写很多行。比如枚举法要写3个`if`，总和法只要1行`cout<<6-a-b`。  
💡 **学习笔记**：多观察问题中的“固定值”（比如总和、常量），能帮你“偷懒”写更少代码！


### ✨ 解题技巧总结  
- 技巧1：**固定集合找缺失？算总和！**（比如1-3总和6，1-5总和15）；  
- 技巧2：**记录存在状态？用数组！**（下标对应编号，值表示“是否存在”）；  
- 技巧3：**情况少？直接枚举！**（覆盖所有可能，适合新手入门）。


## 4. C++核心代码实现赏析

先看**通用核心实现**（总和法，最简洁），再剖析各题解的亮点~


### 本题通用核心C++实现参考  
* **说明**：综合了多个优质题解的思路，用“总和法”实现最简洁的核心逻辑。  
* **完整核心代码**：  
```cpp
#include <iostream>
using namespace std;

int main() {
    int a, b;
    cin >> a >> b;       // 读入两个准时到的编号
    cout << 6 - a - b;   // 用总和6减去已知数，得到迟到的编号
    return 0;
}
```
* **代码解读概要**：  
先读入两个输入，再用“总和6减去两数之和”直接输出结果——没有多余的逻辑，完美体现“数学简化”的力量！


### 针对各优质题解的片段赏析  

#### 题解一：Mzh2012的“总和偷懒法”  
* **亮点**：用数学规律把问题简化到“一行代码”。  
* **核心代码片段**：  
```cpp
cout << 6 - a - b;
```
* **代码解读**：  
这行代码直接计算“总和6 - 已知两数之和”，一步得到迟到的编号。比如输入`3 1`，就是`6-3-1=2`——是不是比写一堆`if`简单10倍？  
💡 **学习笔记**：遇到“固定集合的缺失元素”，先算总和！


#### 题解二：xyx404的“数组标记法”  
* **亮点**：用数组清晰记录“谁来了”，逻辑直观。  
* **核心代码片段**：  
```cpp
int a[4];  // 下标1-3对应兄弟编号
a[x] = a[y] = 1;  // 标记x和y来了
if (!a[1]) cout << 1;
else if (!a[2]) cout << 2;
else cout << 3;
```
* **代码解读**：  
数组`a[4]`的下标1、2、3对应三个兄弟，`a[x]=1`表示x号来了。然后检查`a[1]`到`a[3]`——哪个是0，就是迟到的。比如输入`3 1`，`a[3]=1`、`a[1]=1`，`a[2]=0`，所以输出2。  
💡 **学习笔记**：数组是“记录状态”的好工具，适合需要频繁检查“有没有”的问题！


#### 题解三：wuyouawa的“枚举全覆盖法”  
* **亮点**：把所有可能的输入组合都列出来，直观易懂。  
* **核心代码片段**：  
```cpp
if (a==1&&b==2 || a==2&&b==1) cout << 3;
if (a==1&&b==3 || a==3&&b==1) cout << 2;
if (a==2&&b==3 || a==3&&b==2) cout << 1;
```
* **代码解读**：  
每个`if`对应一种“两个兄弟来了”的情况，覆盖了所有输入顺序（比如`1和2`“`2和1`都对应3”）。虽然代码长，但新手能直接模仿，适合刚开始学条件判断的同学。  
💡 **学习笔记**：枚举法要“覆盖所有可能”，不能漏掉任何一种输入顺序！

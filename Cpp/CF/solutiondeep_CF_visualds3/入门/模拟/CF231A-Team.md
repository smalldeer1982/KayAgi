## 题目要求
三位选手要去参加编程竞赛，比赛共有N道题目，他们商定: 对于每一道题目，只有两位及以上的选手确认找到解法时才做。
试问，他们一共能做多少题?

## 样例 #1

### 输入

```
3
1 1 0
1 1 1
1 0 0
```

### 输出

```
2
```

## 样例 #2

### 输入

```
2
1 0 0
0 1 1
```

### 输出

```
1
```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：模拟应用

🗣️ **初步分析**：  
解决“Team”的关键是**模拟**——就像老师逐题检查三个学生的作业：**每道题读入三位选手的状态（1=会做，0=不会），统计“会做”的人数，若≥2则计数加一**。模拟的核心是“按题目规则一步步执行”，没有复杂的算法，但细节决定成败！

题解的思路高度一致：循环处理每道题→统计1的数量→判断是否符合条件。核心难点是**变量初始化**（比如每道题的计数器要清零）和**输入的准确性**。可视化设计上，我们会用“像素小人”代表选手，用“亮暗”表示状态，直观展示每道题的判断过程，搭配复古音效强化记忆。


## 2. 精选优质题解参考

为大家筛选了5份思路清晰、代码简洁的优质题解，一起看看它们的亮点：

**题解一：作者：谦谦君子**

**题解二：作者：FC_ARSENAL**

**题解五：作者：咕_gu**

## 3. 核心难点辨析与解题策略

解决这道题，新手常栽在“细节”上。结合题解，我们总结三个**核心关键点**：

1. **关键点1：变量初始化与清零**  
   * **分析**：比如FC_ARSENAL的`one=0`——每道题的1的数量是独立的，不清零会导致累加错误；再比如Suuon_Kanderu的栈解法，每次循环后要清空栈，否则上道题的元素会“污染”当前题。  
   * 💡 **学习笔记**：处理多组数据时，临时变量一定要“重置”！

2. **关键点2：输入的正确读取**  
   * **分析**：每道题有3个输入，必须确保每次循环都读对3个值。如果少读一个，后面的输入会全部错位，结果肯定错。比如用`cin>>a>>b>>c`或循环3次`scanf`都是安全的方式。  
   * 💡 **学习笔记**：输入格式要和题目严格对应！

3. **关键点3：条件判断的准确性**  
   * **分析**：题目要求“≥2人会做”，所以条件可以是`sum≥2`（直接求和）、`one≥2`（计数器）或枚举所有符合情况。直接求和最简洁，但要确保三个数都是0或1（题目保证输入合法）。  
   * 💡 **学习笔记**：条件判断要紧扣题目要求，能用简单方法就不用复杂的！

### ✨ 解题技巧总结
- **技巧A：直接求和**：三个0/1的数，求和即得1的数量，代码最简洁。  
- **技巧B：计数器**：选手数量多（比如10人）时，循环统计更通用。  
- **技巧C：STL函数**：`count`、`sort`等函数能简化代码，提升效率。  
- **技巧D：输入输出优化**：竞赛中用`ios::sync_with_stdio(false)`或`scanf/printf`避免超时。


## 4. C++核心代码实现赏析

### 本题通用核心C++实现参考
* **说明**：来自“咕_gu”的题解，兼顾简洁性与竞赛优化，适合大多数场景。  
* **完整核心代码**：
  ```cpp
  #include<iostream>
  using namespace std;
  #define O2 ios::sync_with_stdio(false);cin.tie(0);cout.tie(0);

  int main() {
      O2;
      int n, a, b, c, sum = 0;
      cin >> n;
      for (int i = 1; i <= n; ++i) {
          cin >> a >> b >> c;
          if (a + b + c >= 2) sum++;
      }
      cout << sum << endl;
      return 0;
  }
  ```
* **代码解读概要**：  
  1. `O2`宏开启输入输出优化，让`cin/cout`速度接近`scanf`；  
  2. 读入题目数量`n`；  
  3. 循环`n`次，每次读三位选手的状态`a,b,c`；  
  4. 求和判断是否≥2，是则`sum`加一；  
  5. 输出最终结果`sum`。


### 各优质题解的片段赏析

**题解二：作者：FC_ARSENAL**  
* **亮点**：用计数器统计1的数量，适合选手数量更多的场景。  
* **核心代码片段**：
  ```cpp
  while(n--) {
      one=0; // 每道题清零
      for(int i=0;i<3;i++) {
          scanf("%d",&in);
          if(in==1) one++;
      }
      if(one>=2) ans++;
  }
  ```
* **代码解读**：  
  - `while(n--)`循环`n`次，处理每道题；  
  - `one=0`：每道题开始前重置计数器（关键！）；  
  - 循环3次读入`in`，是1就`one++`；  
  - 判断`one≥2`则加`ans`。  
  问：为什么`one`要清零？因为每道题的1的数量是独立的，不清零会累加之前的结果！  
* 💡 **学习笔记**：多组数据的临时变量必须重置！

**题解三：作者：林家三少**  
* **亮点**：枚举所有符合条件的情况，逻辑明确。  
* **核心代码片段**：
  ```cpp
  cin >> a >> b >> c;
  if(a==0&&b==1&&c==1) ans++;
  if(a==1&&b==0&&c==1) ans++;
  if(a==1&&b==1&&c==0) ans++;
  if(a==1&&b==1&&c==1) ans++;
  ```
* **代码解读**：  
  直接列出所有满足“≥2人会做”的情况：  
  - 0、1、1（第一个不会，后两个会）；  
  - 1、0、1（第二个不会，其他会）；  
  - 1、1、0（第三个不会，其他会）；  
  - 1、1、1（三个都会）。  
  只要符合其中一种，就加`ans`。这种方法适合新手理解条件，但选手数量多了就不适用（比如10人，枚举情况太多）。  
* 💡 **学习笔记**：枚举法是理解条件的好方法，但要注意适用场景！

**题解四：作者：PC_DOS**  
* **亮点**：用STL的`count`函数快速统计1的数量，代码简洁。  
* **核心代码片段**：
  ```cpp
  int arrVote[3];
  cin >> arrVote[0] >> arrVote[1] >> arrVote[2];
  if (count(arrVote, arrVote + 3, 1) >= 2) ++nAns;
  ```
* **代码解读**：  
  - 用数组`arrVote`存三个选手的状态；  
  - `count(arrVote, arrVote+3, 1)`：统计数组中1的数量（`arrVote`是起始地址，`arrVote+3`是结束地址，`1`是目标值）；  
  - 判断数量≥2则加`nAns`。  
  问：`count`函数怎么用？需要包含`<algorithm>`头文件，它会遍历区间内的元素，返回等于目标值的个数。  
* 💡 **学习笔记**：STL函数能简化代码，多记常用函数！

## 7. 学习心得与经验分享 (若有)

题解中的“踩坑经历”是最宝贵的经验，我们来看看：

> **参考经验 (来自 Suuon_Kanderu)**：“用栈的时候，每次循环后要清空栈，不然栈里的元素会影响下一道题的判断。我之前忘了判空，卡了很久！”  
> **点评**：栈、队列等数据结构处理多组数据时，一定要清空！否则上道题的元素会“污染”当前组。比如用栈统计1的数量，不清空会导致统计错误。

> **参考经验 (来自 FC_ARSENAL)**：“每道题的`one`一定要清零，不然会累加之前的数量！”  
> **点评**：这是新手最常犯的错误！处理多组数据时，临时变量（比如计数器）必须重置，否则结果会错得离谱。

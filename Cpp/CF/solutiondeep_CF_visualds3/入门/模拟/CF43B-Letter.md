# Letter

## 题目描述

### 题面描述

Vasya 决定利用报纸上的一段文字写一封匿名信。他可以从报纸上剪下一段文字（或字母），把他们粘贴到匿名信上去。已经剪下的文字不可重复使用。当然，匿名信中的空格不需要剪——他只要在空格处作出标记即可。现在，他知道报纸上的文字 $s1$ 和他即将写下的匿名信的内容 $s2$，请你帮他看看，他能否完成匿名信的书写。

## 样例 #1

### 输入

```
Instead of dogging Your footsteps it disappears but you dont notice anything
where is your dog
```

### 输出

```
NO
```

## 样例 #2

### 输入

```
Instead of dogging Your footsteps it disappears but you dont notice anything
Your dog is upstears
```

### 输出

```
YES
```

## 样例 #3

### 输入

```
Instead of dogging your footsteps it disappears but you dont notice anything
Your dog is upstears
```

### 输出

```
NO
```

## 样例 #4

### 输入

```
abcdefg hijk
k j i h g f e d c b a
```

### 输出

```
YES
```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：编程技巧应用（字符频率统计与验证）

🗣️ **初步分析**：  
解决这道题的核心逻辑，像极了**整理积木**——先统计报纸字符串`s1`中**每个字符（大小写分开）**的出现次数（相当于“数清楚每种积木有多少块”），再检查匿名信字符串`s2`中**每个字符**的需求是否能被满足（“拼模型时每种积木够不够用”）。其中，`s2`的空格不需要从`s1`剪，所以不用统计`s1`的空格数量！  

题解的思路主要分两类：  
1. **统计+验证**：用数组/Map统计`s1`的字符频率，再逐个检查`s2`的字符是否“够取”（如题解一、二、三）；  
2. **实时删除**：每次从`s1`中删除`s2`的字符，看能否删完（如题解四）。  

**核心难点**：区分大小写（样例3中`s1`的`your`是小写，`s2`的`Your`是大写，所以输出NO）、处理空格（`s2`的空格不需要`s1`提供）、正确读取带空格的字符串（不能用`cin`）。

* **点评**：这份题解堪称“简洁高效的典范”！用`a[256]`数组（对应字符的ASCII码）直接统计`s1`的字符频率，巧妙地给空格的数量加了`256`（相当于“无限量”，直接满足`s2`的所有空格需求）。代码逻辑极顺：先读`s1`统计频率，再读`s2`逐个“取”字符——只要有一个字符不够（`a[str[i]]-- < 0`），立刻输出`NO`，否则输出`YES`。数组的访问速度比`map`快得多，处理空格的方法也超聪明，避免了冗余判断。从竞赛角度看，这代码完全能直接用，效率和正确性拉满！


## 3. 核心难点辨析与解题策略

### 关键点1：如何正确区分大小写？  
**分析**：题目中大小写是不同的字符（比如`A`和`a`是两个“积木”）。优质题解的做法是**直接用字符的ASCII码作为数组索引**（比如大写`A`的ASCII是65，小写`a`是97，自然分开），或者用两个数组分别统计大小写（如题解三）。  
💡 **学习笔记**：区分大小写的关键是“尊重字符的原始值”！

### 关键点2：如何处理空格？  
**分析**：`s2`的空格不需要从`s1`剪，所以不管`s1`有多少空格，`s2`的空格都满足。题解一的方法最巧妙：给`s1`的空格数量加`256`（足够大的数，`s2`的空格不可能超过这个数），这样`s2`的空格减的时候永远不会小于0。  
💡 **学习笔记**：处理“无限需求”，用“足够大的数”代替，比单独判断更简洁！

### 关键点3：如何正确读取输入？  
**分析**：`s1`和`s2`都包含空格，`cin`会“遇空格停止”，所以必须用`gets`（读C风格字符串）或`getline`（读C++`string`）。题解一用`gets`，题解四用`getline`，都是正确的。  
💡 **学习笔记**：读带空格的字符串，用`gets`或`getline`！

### ✨ 解题技巧总结  
- **数组比Map更高效**：字符的ASCII码范围是0-255，用`a[256]`正好覆盖，访问速度远快于`map`；  
- **提前处理特殊情况**：比如空格，给足够大的初始值，避免重复判断；  
- **及时终止程序**：一旦发现字符不够，立刻输出`NO`，不用做无用功。


## 4. C++核心代码实现赏析

### 本题通用核心C++实现参考  
* **说明**：来自题解一（agicy），是本题最简洁高效的实现。  
* **完整核心代码**：  
```cpp
#include<stdio.h>

char str[256];  // 存储输入的字符串
int a[256];     // 桶：统计每个字符的出现次数（下标对应ASCII码）

int main(void) {
    register int i;  // 寄存器变量，加快循环速度
    gets(str);       // 读入报纸字符串s1
    for (i = 0; str[i]; i++) {
        a[str[i]]++;  // 统计s1中每个字符的频率
    }
    a[' '] += 256;  // 空格数量设为极大值，满足s2的所有空格需求
    gets(str);       // 读入匿名信字符串s2
    for (i = 0; str[i]; i++) {
        if (--a[str[i]] < 0) {  // s2的字符从桶里取，不够则输出NO
            return puts("NO"), 0;
        }
    }
    puts("YES");  // 所有字符都够，输出YES
    return 0;
}
```  
* **代码解读概要**：  
代码分三步：①读`s1`，用`a`数组统计字符频率；②给空格加`256`（处理`s2`的空格）；③读`s2`，逐个“取”字符——不够就终止，够则输出`YES`。核心是“桶”数组`a`，直接对应字符的ASCII码，统计和查询都超快！

### 题解一核心片段赏析  
* **亮点**：用数组统计频率，处理空格的方法巧妙，代码简洁到极致。  
* **核心代码片段**：  
```cpp
a[' '] += 256;  // 空格数量设为极大值
gets(str);
for (i = 0; str[i]; i++) {
    if (--a[str[i]] < 0) {
        return puts("NO"), 0;
    }
}
```  
* **代码解读**：  
- `a[' '] += 256`：因为`s2`的空格不需要`s1`提供，所以给`s1`的空格数量加一个足够大的数（256），这样`s2`的空格减的时候永远不会小于0，**不用单独判断空格**！  
- 遍历`s2`的字符：对每个字符`c`，`a[c]`减一——如果减到负数，说明`s1`的`c`不够`s2`用，立刻输出`NO`并结束程序。这种“ early return ”的写法超高效，不用做无用功！  
* 💡 **学习笔记**：处理“无限需求”，用“足够大的数”代替，比复杂判断更聪明！

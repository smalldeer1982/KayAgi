# 题目信息

# Arya and Bran

## 题目描述

布兰和他的姐姐艾莉亚来自同一所房子。布兰喜欢糖果，所以艾莉亚会给他一些糖果。

一开始，艾莉亚和布兰有0个糖果。一共有n天，在第i天，艾莉亚会在盒子里找到上帝赐予的ai颗糖果。她每天最多能给布兰8块糖。没有给完的糖果可以在以后给他。

你的任务是找出在第n天结束前，布兰最早在第几天可以得到k颗糖果。最后你只要将输出最早天数。如果布兰在第n天也不能得到k颗糖果则输出-1。

## 说明/提示

样例1中，艾莉亚在第二天时可以给布兰3颗糖果。 样例2中，艾莉亚可以在第三天时给布兰总共17颗糖果，因为她每天最多可以给他8颗糖果。
样例3中，艾莉亚不能给布兰9颗糖果，因为她每天最多可以给他8颗糖果，然而只有1天。

## 样例 #1

### 输入

```
2 3
1 2
```

### 输出

```
2```

## 样例 #2

### 输入

```
3 17
10 10 10
```

### 输出

```
3```

## 样例 #3

### 输入

```
1 9
10
```

### 输出

```
-1```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：模拟（Simulation）

🗣️ **初步分析**：
解决这道题的关键是**“模拟真实的分糖过程”**——就像你每天给弟弟分零食：今天妈妈给你10颗糖，你最多给弟弟8颗，剩下2颗明天再给；明天妈妈又给你5颗，加上昨天剩的2颗一共7颗，全部给弟弟。我们需要用代码“复现”这个每天重复的动作：  
1. Arya收到当天的糖果（`a[i]`），加到自己的“剩余糖罐”里；  
2. 给Bran最多8颗（如果剩余糖≥8，就给8颗，剩下的留着；否则全给）；  
3. 检查Bran的糖够不够k颗，够了就立刻输出当天的天数！

**核心难点**：需要记住Arya“剩下的糖”（不能每天清零），以及及时停止循环（一旦达标就退出）。  
**可视化设计思路**：我们会用**FC红白机风格的像素动画**展示这个过程——Arya的糖罐是黄色像素堆，Bran的糖堆是蓝色像素堆，每天新糖从顶部“掉”进糖罐，然后分出8颗到Bran的堆里。关键步骤会用**闪烁**和**音效**提醒（比如给糖时“嗒”的一声），让你直观看到“糖怎么流动”。

**题解一：Chtholly_L的直观模拟**  
* **点评**：这份题解像“说明书”一样直白！用`a`记录Arya的剩余糖，`b`记录Bran的总糖，每天先加新糖，再用`if(a>8)`判断给多少——超过8就给8颗，否则全给。逻辑一步一步，变量名一看就懂，特别适合刚学模拟的同学。代码里“一旦Bran够了就return 0”的写法很聪明，直接终止程序，不用多余的判断。

**题解二：Colead的简洁优化**  
* **点评**：这份题解用`min(nowa,8)`简化了“给最多8颗”的逻辑！比如`nowb += min(nowa,8)`直接算出Bran今天能拿的糖，`nowa -= min(nowa,8)`更新Arya的剩余糖。代码少了if-else，更简洁高效。这种“用数学函数替代条件判断”的技巧，能让代码更优雅。

**题解三：Creroity的边输入边处理**  
* **点评**：这份题解**不需要数组**！因为每天的`a[i]`只用一次（加到`now`里），所以直接边输入边处理，节省了数组的空间。比如`cin>>a`后立刻`now+=a`，然后分配糖。这种“在线处理”的思路，在很多模拟题中都能用，能减少内存消耗。

## 3. 核心难点辨析与解题策略

<difficulty_intro>
模拟题的“难点”往往是“把生活逻辑转化为代码逻辑”。结合题解，我总结了3个关键问题和解决办法：
</difficulty_intro>

1.  **关键点1：如何记录Arya剩余的糖？**  
    * **分析**：Arya每天会剩糖，这些糖要留到明天。所以需要一个变量（比如`now_a`）专门存“还没给Bran的糖”——每天先把新糖加到`now_a`里，再分配。  
    * 💡 **学习笔记**：模拟“累积资源”时，一定要用变量记下来！

2.  **关键点2：如何确保每天最多给8颗？**  
    * **分析**：有两种方式：  
      - 用条件判断：`if(now_a>8) 给8颗，否则全给`（如题解一）；  
      - 用`min`函数：`给的糖 = min(now_a, 8)`（如题解二）。后者更简洁！  
    * 💡 **学习笔记**：能用数学函数替代条件判断时，尽量用——代码更短，逻辑更清晰。

3.  **关键点3：如何快速判断“不可能达标”？**  
    * **分析**：如果每天都给8颗，n天最多给`8*n`颗。如果`8*n <k`，直接输出-1（如题解中的阳歌童圣、Mikemao666）。这种“提前剪枝”能避免无用的循环，优化效率。  
    * 💡 **学习笔记**：先想“最坏情况”，能快速处理边界！

### ✨ 解题技巧总结
- **技巧1：变量命名要直观**：比如用`now_a`表示Arya的剩余糖，`now_b`表示Bran的总糖，别人一看就懂。  
- **技巧2：边输入边处理**：不需要存储所有`a[i]`，直接输入一个处理一个，节省空间。  
- **技巧3：提前处理边界**：先判断“每天给8颗都不够”的情况，避免无效循环。

## 4. C++核心代码实现赏析

<code_intro_overall>
先看一个**通用核心实现**——综合了题解的优点，逻辑清晰，适合新手参考：
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：本代码综合了“直观模拟”和“边输入边处理”的思路，变量名直观，逻辑一步到位。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    using namespace std;

    int main() {
        int n, k;
        cin >> n >> k;

        // 提前判断：每天给8颗都不够，直接输出-1
        if (8 * n < k) {
            cout << -1 << endl;
            return 0;
        }

        int now_a = 0;  // Arya剩余的糖
        int now_b = 0;  // Bran的总糖

        for (int day = 1; day <= n; ++day) {
            int a;
            cin >> a;
            now_a += a;  // 加上当天的糖

            // 给Bran最多8颗
            int give = min(now_a, 8);
            now_b += give;
            now_a -= give;

            // 检查是否达标
            if (now_b >= k) {
                cout << day << endl;
                return 0;
            }
        }

        // 循环结束还没达标（理论上不会走到这，因为提前判断过）
        cout << -1 << endl;
        return 0;
    }
    ```
* **代码解读概要**：
  1. 先判断“每天给8颗都不够”的情况，直接输出-1；  
  2. 用`now_a`记录Arya的剩余糖，`now_b`记录Bran的总糖；  
  3. 循环n天：每天加新糖→给最多8颗→检查是否达标；  
  4. 一旦达标，立刻输出天数并终止程序。

**题解一：Chtholly_L的条件判断**
* **亮点**：用最直观的if-else展示“给糖逻辑”，适合刚学模拟的同学。
* **核心代码片段**：
    ```cpp
    if (a > 8) {  // Arya的糖超过8颗
        b += 8;   // 给Bran8颗
        a -= 8;   // 剩余的糖留下
    } else {      // 不够8颗
        b += a;   // 全给Bran
        a = 0;    // 剩余糖清零
    }
    ```
* **代码解读**：
  这段代码像“分糖的说明书”——如果Arya有超过8颗，就给8颗，剩下的留着；否则全给。变量名`a`（Arya）、`b`（Bran）一看就懂，逻辑没有冗余。
* 💡 **学习笔记**：新手写模拟题，先写“直白的条件判断”，再慢慢优化！

**题解二：Colead的min函数简化**
* **亮点**：用`min`函数替代if-else，代码更简洁。
* **核心代码片段**：
    ```cpp
    nowb += min(nowa, 8);  // 给最多8颗
    nowa -= min(nowa, 8);  // 更新剩余糖
    ```
* **代码解读**：
  `min(nowa, 8)`直接算出“今天能给的最大数量”——如果`nowa≥8`，就是8；否则是`nowa`。这两行代码替代了题解一的if-else，更简洁高效。
* 💡 **学习笔记**：`min`/`max`函数是模拟题的“简化神器”，能减少代码量！

**题解三：Creroity的边输入边处理**
* **亮点**：不需要数组，直接输入一个处理一个，节省空间。
* **核心代码片段**：
    ```cpp
    for (int i = 1; i <= n; ++i) {
        cin >> a;         // 直接输入当天的糖
        now += a;         // 加到Arya的剩余糖里
        // ... 分配逻辑 ...
    }
    ```
* **代码解读**：
  因为每天的`a`只用一次（加到`now`里），所以不需要用数组存储所有`a[i]`。这种“在线处理”的思路，在很多模拟题中都能用（比如统计每天的温度、处理流数据）。

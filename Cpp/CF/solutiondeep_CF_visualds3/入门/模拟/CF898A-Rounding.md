# 题目信息

# Rounding

## 题目描述

Vasya has a non-negative integer $ n $ . He wants to round it to nearest integer, which ends up with $ 0 $ . If $ n $ already ends up with $ 0 $ , Vasya considers it already rounded.

For example, if $ n=4722 $ answer is $ 4720 $ . If $ n=5 $ Vasya can round it to $ 0 $ or to $ 10 $ . Both ways are correct.

For given $ n $ find out to which integer will Vasya round it.

## 说明/提示

In the first example $ n=5 $ . Nearest integers, that ends up with zero are $ 0 $ and $ 10 $ . Any of these answers is correct, so you can print $ 0 $ or $ 10 $ .

## 样例 #1

### 输入

```
5
```

### 输出

```
0
```

## 样例 #2

### 输入

```
113
```

### 输出

```
110
```

## 样例 #3

### 输入

```
1000000000
```

### 输出

```
1000000000
```

## 样例 #4

### 输入

```
5432359
```

### 输出

```
5432360
```

✨ **本题主要考察**：`数学 + 编程技巧应用`  

🗣️ **初步分析**：
> 解决“Rounding”的关键，其实是**处理数字的“个位”**——就像我们做算术题时，判断最后一位要不要“四舍五入”。简单来说，题目要求：  
> - 如果个位是1~4：直接把个位变成0（比如4722→4720）；  
> - 如果个位是6~9：把个位变0，同时前一位加1（比如5432359→5432360）；  
> - 如果个位是5：可以选变0（如5→0）或前一位加1（如5→10），两种都对～  

大部分题解会把“5”归为“入”的情况（比如5→10），这样能**用统一的数学公式解决问题**，不用额外判断。核心算法流程超简单：  
1. 取数字的**个位**（用`n%10`）；  
2. 判断个位是否需要“舍”或“入”；  
3. 调整数字，让末尾变0（比如舍的话减个位，入的话加“10-个位”）。

**题解一：Reywmp（赞：6）**  
* **点评**：这个题解的思路太“聪明”了！作者用了一句`a=(a+5)/10*10`就解决了所有情况——这是信息学里**手写四舍五入**的经典技巧！比如：  
  - 当a=5时，5+5=10→10/10=1→1*10=10（入）；  
  - 当a=113时，113+5=118→118/10=11→11*10=110（舍）；  
  - 当a=5432359时，5432359+5=5432364→5432364/10=543236→543236*10=5432360（入）。  
  代码只有5行，变量名清晰，甚至不用判断个位——完全靠数学公式“自动”处理！对新手来说，这是**“用数学简化代码”的完美例子**～

**题解二：empty（赞：2）**  
* **点评**：作者说“暴力最可靠”，确实！他用了一个`while`循环，反复调整数字直到个位变0：  
  - 如果个位≤5：每次减1（比如5→4→3→2→1→0）；  
  - 如果个位>5：每次加1（比如7→8→9→10）。  
  这个方法**特别直观**——就像你手动掰手指调整数字！虽然效率不如公式法，但能帮你“看”清楚算法的本质：“让个位变成0”。对刚开始学编程的同学来说，这种“模拟真实操作”的思路很容易理解～

**题解三：向日葵小班（赞：0）**  
* **点评**：这个题解用了**“分情况公式”**，把问题拆成两句：  
  - 如果个位≤5：`n -= n%10`（比如113%10=3→113-3=110）；  
  - 如果个位>5：`n += (10 - n%10)`（比如5432359%10=9→10-9=1→5432359+1=5432360）。  
  代码结构超清晰，每一步都对应题目要求，**没有冗余**。尤其是用`long long`定义n，避免了大数字溢出（比如样例3的1000000000）——这是新手很容易忽略的细节！

## 3. 核心难点辨析与解题策略

<difficulty_intro>
刚开始做这道题时，你可能会遇到3个小麻烦——别怕，我们一个个解决！
</difficulty_intro>

1.  **关键点1：怎么取数字的“个位”？**  
    * **分析**：在C++里，`n%10`能直接得到个位（比如113%10=3，5%10=5）。这是处理数字位的基础技巧，所有题解都用了它！  
    * 💡 **学习笔记**：取某个位的数字，用`n%(10^k)`（k是位数，比如十位是`n%100/10`）。

2.  **关键点2：怎么统一处理“5”的情况？**  
    * **分析**：题目说5可以舍或入，但大部分题解选“入”——因为这样能把所有情况合并成一个公式（比如`(a+5)/10*10`），不用写多余的`if`判断。如果选“舍”，只需把条件改成`n%10<5`（比如TCZN的题解），但会多一句判断。  
    * 💡 **学习笔记**：编程时，“统一规则”能让代码更简洁！

3.  **关键点3：怎么避免大数字溢出？**  
    * **分析**：样例3的输入是1000000000（10^9），如果用`int`存储，可能会超出范围（因为`int`通常最大是2^31-1≈2e9，但有些编译器会限制）。所以用`long long`更安全！  
    * 💡 **学习笔记**：处理大数字时，优先用`long long`（占8字节，能存到9e18）。

### ✨ 解题技巧总结
<summary_best_practices>
通过这道题，我们能学到3个通用技巧：
</summary_best_practices>
-   **技巧1：用数学公式简化逻辑**：比如`(a+5)/10*10`，一句话解决四舍五入，比写一堆`if`高效；  
-   **技巧2：模拟真实操作**：如果公式想不通，就像empty那样“暴力”调整数字，先让代码跑起来，再优化；

## 4. C++核心代码实现赏析

<code_intro_overall>
先看一个**通用核心实现**——它综合了Reywmp的公式法和向日葵小班的`long long`技巧，既简洁又安全！

**本题通用核心C++实现参考**

* **完整核心代码**：
    ```cpp
    #include <iostream>

    }
    ```
* **代码解读概要**：  
  1. 读取输入的数字`n`（用`long long`避免溢出）；  
  2. 用公式`(n+5)/10*10`处理：加5是为了让≥5的个位“进位”（比如5+5=10，除以10得1，再乘10得10）；  
  3. 输出结果。

接下来看3个优质题解的核心片段，体会不同思路的妙处！

**题解一：Reywmp（公式法）**
* **亮点**：用数学公式代替所有判断，代码极简！  
* **核心代码片段**：
    ```cpp
    int a;
    scanf("%d",&a);
    a=(a+5)/10*10;  // 关键公式
    printf("%d",a);
    ```
* **代码解读**：  
  这句话的魔法在哪里？比如`a=5`：`5+5=10`，`10/10=1`（C++里整数除法会向下取整），`1*10=10`——刚好是“入”的结果；`a=113`：`113+5=118`，`118/10=11`，`11*10=110`——刚好是“舍”的结果。**这就是数学的力量！**  
* 💡 **学习笔记**：整数除法的“向下取整”特性，是很多数学技巧的基础～

**题解二：empty（暴力循环法）**
* **亮点**：模拟手动调整的过程，直观易懂！  
* **核心代码片段**：
    ```cpp
    while(1) {
        n1 = n % 10;
        if(n1 == 0) break;  // 个位是0，结束循环
        if(n1 <=5) n--;     // 个位≤5，减1
        if(n1 >5)  n++;     // 个位>5，加1
    }
    ```
* **代码解读**：  
  这个循环会一直跑，直到个位变成0。比如`n=5`：第一次循环`n1=5`→`n--`变成4；第二次`n1=4`→`n--`变成3……直到`n=0`，循环结束。虽然要跑5次，但过程和你手动掰手指一样——**特别适合新手理解“算法是怎么工作的”**！  
* 💡 **学习笔记**：如果想不通复杂的公式，先写“暴力模拟”的代码，再优化～

**题解三：向日葵小班（分情况公式法）**
* **亮点**：把题目要求直接翻译成代码，逻辑清晰！  
* **核心代码片段**：
    ```cpp
    long long n;
    cin >> n;
    if(n%10 <=5) n -= n%10;  // 舍：减个位
    if(n%10 >5) n += (10 - n%10);  // 入：加“10-个位”
    cout << n;
    ```
* **代码解读**：  
  比如`n=5432359`：`n%10=9`→`9>5`→`n += 10-9=1`→`5432359+1=5432360`，刚好是正确结果；`n=113`：`n%10=3`→`3≤5`→`n-=3`→`110`。**每一行代码都对应题目要求，没有歧义**！  
* 💡 **学习笔记**：把问题“拆解成条件判断”，是编程的基础思路～

# 题目信息

# Wrong Subtraction

## 题目描述

Little girl Tanya is learning how to decrease a number by one, but she does it wrong with a number consisting of two or more digits. Tanya subtracts one from a number by the following algorithm:

- if the last digit of the number is non-zero, she decreases the number by one;
- if the last digit of the number is zero, she divides the number by 10 (i.e. removes the last digit).

You are given an integer number $ n $ . Tanya will subtract one from it $ k $ times. Your task is to print the result after all $ k $ subtractions.

It is guaranteed that the result will be positive integer number.

## 说明/提示

The first example corresponds to the following sequence: $ 512 \rightarrow 511 \rightarrow 510 \rightarrow 51 \rightarrow 50 $ .

## 样例 #1

### 输入

```
512 4
```

### 输出

```
50
```

## 样例 #2

### 输入

```
1000000000 9
```

### 输出

```
1
```

✨ **本题主要考察**：模拟 (按照题目规则一步步执行操作)

🗣️ **初步分析**：
> 解决这道题的关键是**模拟**——就像跟着食谱做蛋糕：题目说“先看最后一位是不是0，不是就减1，是就删最后一位”，我们就严格照做k次。  
> 具体来说，每次操作分两步：① 用`n%10`（取模10）看最后一位是不是0；② 不是0就`n--`，是0就`n/=10`（整除10，相当于删最后一位）。  
> **核心难点**：如何正确处理“最后一位”的判断，以及如何应对很大的数字（比如样例2中的1000000000）。

## 3. 核心难点辨析与解题策略

<difficulty_intro>
模拟题的“坑”往往藏在“细节”里。结合题解的共性，我帮大家提炼了3个核心难点和应对方法：
</difficulty_intro>

1.  **关键点1：如何正确判断“最后一位”？**
    * **分析**：用`n%10`（取模10）就能直接拿到最后一位！比如512%10=2，510%10=0。这是C++里处理“个位”的经典技巧，记住它，很多数字问题都能用！
    * 💡 **学习笔记**：取模10是“抓个位”的神器！

2.  **关键点2：如何处理很大的数字？**
    * **分析**：如果n很大（比如1e9），用`int`会溢出（int最多存2e9左右，但有些题目n会更大）。这时候有两种办法：① 用`long long`（能存到9e18）；② 用字符串存储（完全不用怕溢出）。题解一用了`long long`，题解二用了字符串，都是好办法！
    * 💡 **学习笔记**：大数问题，要么用`long long`，要么用字符串！

3.  **关键点3：如何控制循环次数？**
    * **分析**：循环k次就行，但要注意循环的写法——可以用`while(k--)`（每次循环k减1，直到k=0），也可以用`for(int i=1;i<=k;i++)`（从1数到k）。两种写法都对，但`while(k--)`更简洁！
    * 💡 **学习笔记**：循环次数明确时，选你最顺手的写法，但要避免多循环或少循环！

### ✨ 解题技巧总结
<summary_best_practices>
模拟题的本质是“翻译题目规则”，记住这3个技巧：

- **技巧1：规则直译**：把题目中的“如果...就...”直接写成`if...else`语句，不要自己加额外逻辑！
- **技巧2：数据类型防溢出**：先看题目中的n范围，如果很大，直接用`long long`或字符串！

## 4. C++核心代码实现赏析

<code_intro_overall>
先看一份**通用核心实现**——结合了“long long防溢出”和“简洁循环”，适合大多数情况：
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：本代码综合了题解一的简洁性和题解三的新手友好性，用`long long`处理大数，`while`循环执行k次操作。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    using namespace std;

    int main() {
        long long n;  // 用long long存大数，避免溢出
        int k;
        cin >> n >> k;
        
        while (k--) {  // 循环k次，每次k减1
            if (n % 10 != 0) {  // 最后一位不是0，减1
                n--;
            } else {  // 最后一位是0，删最后一位（整除10）
                n /= 10;
            }
        }
        
        cout << n << endl;
        return 0;
    }
    ```
* **代码解读概要**：
    > 代码分三步：① 读入n和k（n用`long long`）；② 循环k次，每次按规则修改n；③ 输出结果。核心逻辑就是“取模判断→修改n”，像“按开关”一样直接！

接下来看3份优质题解的**核心片段**，学习不同的实现思路：

**题解一：(来源：引领天下)**
* **亮点**：用一行代码搞定核心逻辑，极简主义！
* **核心代码片段**：
    ```cpp
    while (k--) if (n%10)n--;else n/=10;
    ```
* **代码解读**：
    > 这行代码把`while`循环和`if-else`揉在一起，特别简洁！`n%10`相当于“n%10 != 0”（因为C++里非0值都是true），所以如果最后一位不是0，就`n--`；否则`n/=10`。这种写法像“浓缩果汁”，适合已经熟悉逻辑的同学！
* 💡 **学习笔记**：简洁的代码是高手的标志，但新手要先写清楚逻辑，再优化！

**题解二：(来源：666yuchen)**
* **亮点**：用字符串处理大数，完全不用怕溢出！
* **核心代码片段**：
    ```cpp
    string st;
    cin >> st >> n;
    for(int i=1;i<=n;i++){
        if(st[st.size()-1]=='0') st.erase(st.size()-1,1);
        else st[st.size()-1]--;
    }
    ```
* **代码解读**：
    > 把数字存在字符串`st`里，`st.size()-1`是最后一位的索引（因为字符串从0开始）。如果最后一位是'0'，就用`erase`删了它（`st.size()-1`是起始位置，1是删除长度）；否则把最后一位的字符减1（比如'2'变成'1'）。这种方法像“改便签纸”，直观又安全！
* 💡 **学习笔记**：字符串处理是解决“超大数”问题的终极大招！

**题解三：(来源：thename)**
* **亮点**：用`continue`避免逻辑混乱，新手友好！
* **核心代码片段**：
    ```cpp
    for(;k>0;k--){
        if(n%10==0){
            n/=10;
            continue;  // 跳过下面的n--
        }
        n--;
    }
    ```
* **代码解读**：
    > 作者怕“如果最后一位是0，又执行了n--”，所以用`continue`跳过后续的`n--`。这种写法像“走路时绕开水坑”，把可能的错误提前挡住，特别适合刚开始学循环的同学！
* 💡 **学习笔记**：`continue`能帮你避免“多余操作”，但不要滥用哦！

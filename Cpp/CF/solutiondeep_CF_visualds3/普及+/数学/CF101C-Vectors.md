# Vectors

## 题目描述

At a geometry lesson Gerald was given a task: to get vector $ B $ out of vector $ A $ . Besides, the teacher permitted him to perform the following operations with vector $ А $ :

- Turn the vector by $ 90 $ degrees clockwise.
- Add to the vector a certain vector $ C $ .

Operations could be performed in any order any number of times.

Can Gerald cope with the task?

```
0 0
1 1
0 1
```

### 输出

```
YES
```

## 样例 #2

```
1 1
```

### 输出

```
YES
```

## 样例 #3

### 输入

```

2 2
```

### 输出

```
NO
```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：数学（向量旋转+线性方程组）+ 枚举  

🗣️ **初步分析**：  
解决“Vectors”的关键，是理解**向量旋转的周期性**和**线性组合的整数解问题**。简单来说：  
- 向量旋转90度顺时针的公式是「(x,y) → (y,-x)」，旋转4次就会回到原向量（比如转90→180→270→360=0度）。因此，向量A有**4种可能的初始状态**（不转、转90、转180、转270）。  
- 我们需要判断：对于A的某一种旋转状态，能否通过“加若干次向量C”或“加若干次旋转后的C”，得到向量B。这可以转化为解一个**二元一次方程组**——看是否存在整数解（次数必须是整数）。  

### 核心逻辑与可视化设计思路  
1. **枚举旋转状态**：依次检查A的4种旋转形式（比如第一个旋转状态是A本身，第二个是转90度后的A，依此类推）。  
2. **解方程组**：对于每种旋转后的A，计算“B - 旋转后的A”（记为目标向量D），然后看D能否表示为「a*C + b*旋转后的C」（a、b是整数）。  
3. **可视化设计**：用8位像素风展示向量的旋转和组合过程——比如用不同颜色标记A的4种旋转状态，用“绿色方块累加”表示C的次数，用音效提示“方程组有解”（叮~）或“无解”（嗒~）。

## 3. 核心难点辨析与解题策略

在解决这个问题时，大家容易卡壳的3个关键点，我帮大家总结好了应对方法：


### 1. 关键点1：如何处理旋转的周期性？  
- **分析**：旋转90度4次会回到原向量，所以A有4种可能的初始状态（不转、转90、转180、转270）。必须**枚举所有4种情况**，不能漏掉任何一种！  
- 💡 **学习笔记**：周期性问题的常见解法是“枚举所有可能的周期状态”。


### 2. 关键点2：如何推导方程组？  
- **分析**：目标向量D = B - 旋转后的A。我们需要D = a*C + b*旋转后的C（a、b是整数）。将向量转化为坐标方程：  
  $$\begin{cases} C_x \cdot a + C_y \cdot b = D_x \\ C_y \cdot a - C_x \cdot b = D_y \end{cases}$$  
  解这个方程组，判断a、b是否为整数即可。  
- 💡 **学习笔记**：向量问题常转化为坐标方程，数学推导是关键！


### 3. 关键点3：如何处理特殊情况（C是零向量）？  
- **分析**：当C是零向量时，无法通过“加C”改变向量——此时只能看A的4种旋转状态是否等于B。  
- 💡 **学习笔记**：编程时一定要考虑“零向量”“空输入”等边界情况！


### ✨ 解题技巧总结  
1. **枚举周期状态**：遇到旋转、循环问题，先找周期，再枚举所有可能的状态。  
2. **数学建模**：将向量问题转化为坐标方程，用代数方法解决。  
3. **边界处理**：永远记得检查“零向量”“无解”等特殊情况！


## 4. C++核心代码实现赏析

### 本题通用核心C++实现参考  
* **说明**：本代码来自题解一（Remake_），逻辑清晰、覆盖所有边界情况，是本题的标准实现。  
* **完整核心代码**：  
```cpp
#include <iostream>
#include <cmath>
using namespace std;

long long ax, ay, bx, by, cx, cy;

void check(long long axx, long long ayy) {
    // 情况1：C是零向量（无法通过加C改变向量）
    if (cx * cx + cy * cy == 0) {
        if (axx == bx && ayy == by) {
            cout << "YES";
            exit(0); // 直接结束程序
        }
        return;
    }
    // 情况2：解方程组，判断分子是否能被分母整除
    long long numerator_b = cx * (by - ayy) - cy * (bx - axx);
    long long numerator_a = cx * (bx - axx) + cy * (by - ayy);
    long long denominator = cx * cx + cy * cy;
    if (numerator_b % denominator == 0 && numerator_a % denominator == 0) {
        cout << "YES";
        exit(0);
    }
}

int main() {
    cin >> ax >> ay >> bx >> by >> cx >> cy;
    // 检查A的4种旋转状态
    check(ax, ay);          // 不旋转
    check(ay, -ax);         // 转90度
    check(-ax, -ay);        // 转180度
    check(-ay, ax);         // 转270度
    cout << "NO";           // 所有情况都不满足
    return 0;
}
```
* **代码解读概要**：  
  1. `check`函数：输入旋转后的A（`axx, ayy`），判断能否通过加C得到B。  
     - 先处理C是零向量的情况：直接检查旋转后的A是否等于B。  
     - 否则计算方程组的分子（`numerator_a`、`numerator_b`）和分母（`denominator`），判断分子能否被分母整除（确保a、b是整数）。  
  2. `main`函数：读取输入，依次检查A的4种旋转状态，若有一个满足则输出YES，否则输出NO。


### 题解一核心代码片段赏析  
* **亮点**：用`exit(0)`直接结束程序，避免不必要的计算。  
* **核心代码片段**：  
```cpp
void check(long long axx, long long ayy) {
    if (cx * cx + cy * cy == 0) {
        if (axx == bx && ayy == by) {
            cout << "YES";
            exit(0); // 找到解，直接退出
        }
        return;
    }
    // ... 其他逻辑
}
```
* **代码解读**：  
  当C是零向量时，如果旋转后的A等于B，直接输出YES并结束程序——这比继续检查其他状态更高效！  
* 💡 **学习笔记**：`exit(0)`可以快速终止程序，适合“找到解就停止”的场景。


### 题解二核心代码片段赏析  
* **亮点**：用一行代码检查4种旋转状态，简洁高效。  
* **核心代码片段**：  
```cpp
bool flag = 0;
if (check(x1,y1)||check(-x1,-y1)||check(y1,-x1)||check(-y1,x1)) flag=1;
printf("%s",flag?"YES":"NO");
```
* **代码解读**：  
  用`||`（逻辑或）连接4次`check`调用，只要有一个返回true，`flag`就为1——这是“枚举所有状态”的简洁写法！  
* 💡 **学习笔记**：逻辑运算符可以简化“多条件判断”的代码。

# 题目信息

# Unnatural Conditions

## 题目描述

Let $ s(x) $ be sum of digits in decimal representation of positive integer $ x $ . Given two integers $ n $ and $ m $ , find some positive integers $ a $ and $ b $ such that

- $ s(a) \ge n $ ,
- $ s(b) \ge n $ ,
- $ s(a + b) \le m $ .

## 说明/提示

In the first sample, we have $ n = 6 $ and $ m = 5 $ . One valid solution is $ a = 6 $ , $ b = 7 $ . Indeed, we have $ s(a) = 6 \ge n $ and $ s(b) = 7 \ge n $ , and also $ s(a + b) = s(13) = 4 \le m $ .

## 样例 #1

### 输入

```
6 5
```

### 输出

```
6 
7
```

## 样例 #2

### 输入

```
8 16
```

### 输出

```
35 
53
```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：`编程技巧应用（构造法）` 

🗣️ **初步分析**：
解决“Unnatural Conditions”的关键在于构造满足条件的数对 \(a\) 和 \(b\)。构造法的核心思想是：通过设计特定结构的数，直接满足题目要求的约束条件（如数位和、进位规则等），就像搭积木一样，用简单的“数字块”拼出符合要求的结果。

在本题中，构造法主要用于：  
1. 让 \(s(a)\) 和 \(s(b)\) 足够大（≥\(n\)）：通过重复相同数字（如4、5）增加数位和。  
2. 让 \(s(a+b)\) 足够小（≤\(m\)）：利用进位规则，使 \(a+b\) 成为 \(10^k\) 形式（如100...00），其数位和仅为1。  

**核心难点与解决方案**：  
- 难点1：如何同时满足 \(s(a)≥n\)、\(s(b)≥n\) 和 \(s(a+b)≤m\)？  
  解决方案：构造 \(a\) 为多个4（如300位4），\(b\) 为多个5加一个6（如299位5+6）。此时 \(s(a)=300×4=1200≥n\)，\(s(b)=299×5+6=1501≥n\)，而 \(a+b=444...4 + 555...56=1000...0\)（进位后），\(s(a+b)=1≤m\)。  
- 难点2：如何确定构造的位数？  
  解决方案：题目中 \(n≤1200\)，300位4的数位和为1200，足够覆盖所有 \(n≤1200\) 的情况；299位5加6的数位和为1501，同样满足要求。  

**可视化设计思路**：  
采用8位像素风格动画，用像素方块表示 \(a\)、\(b\) 的每一位数字（如4用蓝色方块，5用红色方块，6用黄色方块）。动画中，\(a\) 和 \(b\) 逐位相加时，前299位的4+5=9（绿色方块），最后一位4+6=10（触发进位，绿色变0，最高位弹出1的金色方块）。关键步骤高亮（如进位过程闪烁），并配合“叮”的音效提示进位。

## 3. 核心难点辨析与解题策略

<difficulty_intro>
在解决本题的过程中，我们需要重点突破以下核心难点，并掌握对应的解题策略：
</difficulty_intro>

1.  **关键点1**：如何确保 \(s(a)≥n\) 和 \(s(b)≥n\)？  
    * **分析**：数位和由数字的每一位之和决定。为了让数位和足够大，可以选择重复相同的数字（如4、5），因为它们的单数字值适中（4+5=9，方便进位），且重复次数容易计算（如300位4的和为1200，覆盖 \(n≤1200\)）。  
    * 💡 **学习笔记**：选择单数字值较高且易计算的数字（如4、5），通过重复增加数位和是构造大数位和的常用技巧。

2.  **关键点2**：如何让 \(s(a+b)≤m\)？  
    * **分析**：利用进位规则，使 \(a+b\) 的每一位相加后产生连续进位，最终形成 \(10^k\) 形式（如100...00）。例如，444...4（300位）和555...56（299位5+6）相加时，前299位4+5=9，最后一位4+6=10，触发连续进位，所有9变为0，最高位变为1，此时 \(s(a+b)=1\)，远小于 \(m\)（\(m≥1\)）。  
    * 💡 **学习笔记**：进位是减少数位和的“魔法”，设计 \(a\) 和 \(b\) 的末位和为10，其余位和为9，可触发连续进位，使 \(a+b\) 的数位和最小化。

3.  **关键点3**：如何确定构造的数字长度？  
    * **分析**：题目中 \(n≤1200\)，因此需要构造的 \(a\) 的数位和至少为1200。选择300位4（300×4=1200），刚好覆盖 \(n=1200\) 的情况；\(b\) 选择299位5加6（299×5+6=1501），确保数位和更大。长度300和299均小于题目限制（\(a,b<10^{2230}\)），因此可行。  
    * 💡 **学习笔记**：构造长度需满足“数位和≥n”且“总长度≤题目限制”，选择固定长度（如300位）可简化计算。

### ✨ 解题技巧总结
<summary_best_practices>
- **问题抽象**：将问题转化为“如何通过进位减少数位和”，抓住核心矛盾（\(s(a+b)\) 的最小化）。  
- **固定模式构造**：选择重复数字（如4、5）构造 \(a\) 和 \(b\)，利用其和的进位特性，简化计算。  
- **边界验证**：计算构造数的数位和，确保覆盖题目中 \(n\) 的最大值（1200），避免遗漏。
</summary_best_practices>

* **完整核心代码**：  
    ```cpp
    #include <iostream>
    using namespace std;

    int main() {
        // 构造a：300位4，数位和=300×4=1200≥n（n≤1200）
        for (int i = 0; i < 300; ++i) {
            cout << '4';
        }
        cout << '\n';
        
        // 构造b：299位5加1位6，数位和=299×5+6=1501≥n
        for (int i = 0; i < 299; ++i) {
            cout << '5';
        }
        cout << '6' << '\n';
        
        return 0;
    }
    ```
* **代码解读概要**：  
  代码分为两部分：  
  1. 输出300个4作为 \(a\)，确保其数位和为1200（覆盖 \(n≤1200\)）。  
  2. 输出299个5和1个6作为 \(b\)，数位和为1501（同样覆盖 \(n≤1200\)）。  
  \(a+b\) 时，前299位4+5=9，最后一位4+6=10，触发连续进位，最终 \(a+b=1000...000\)（数位和为1），满足 \(s(a+b)≤m\)。

# 题目信息

# GCD vs LCM

## 题目描述

本题有多测。  
给定一个正整数 $n$，求一组正整数 $a$, $b$, $c$, $d$，使得 $a+b+c+d=n$，并且 $\gcd(a,b) = \operatorname{lcm}(c,d)$。本题有 SPJ，求出任意一组即可。

## 说明/提示

$4\le n\le 10^9$  
$1\le t\le 10^4$

## 样例 #1

### 输入

```
5
4
7
8
9
10```

### 输出

```
1 1 1 1
2 2 2 1
2 2 2 2
2 4 2 1
3 5 1 1```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：构造法（编程技巧应用）  

🗣️ **初步分析**：  
解决“GCD vs LCM”这道题，关键在于**构造一组满足条件的解**——不需要找到所有解，只要找到任意一组符合要求的正整数即可。构造法就像“搭积木”：我们可以选择最简单的“积木块”（比如数字1），组合出满足条件的结果。  

### 核心思路
题目要求`a+b+c+d=n`且`gcd(a,b)=lcm(c,d)`。我们可以**固定两边的值都为1**：  
- 对于`lcm(c,d)=1`，只有`c=1`且`d=1`（因为1和1的最小公倍数是1）；  
- 对于`gcd(a,b)=1`，只要`a`或`b`为1即可（因为1和任何数的最大公约数都是1）。  

因此，我们可以取`a = n-3`（保证`a`是正整数，因为`n≥4`），`b=1`，`c=1`，`d=1`。此时：  
- 和为`(n-3)+1+1+1 = n`，满足第一个条件；  
- `gcd(n-3,1)=1`，`lcm(1,1)=1`，满足第二个条件。  

### 可视化设计思路
为了直观展示这个构造过程，我们可以设计一个**8位像素风格的动画**：  
- **场景**：屏幕上有四个像素块，分别代表`a`、`b`、`c`、`d`，初始值都是1；  
- **输入n**：比如输入`7`，`a`的像素块会从1变成`4`（7-3）；  
- **高亮关键值**：用绿色高亮显示`gcd(a,b)=1`和`lcm(c,d)=1`，强调两边相等；  
- **音效**：当`a`的值变化时，播放“叮”的像素音效；当验证条件成立时，播放胜利音效（比如FC游戏的“通关声”）。

## 3. 核心难点辨析与解题策略

### 1. 难点1：想到用构造法，而不是暴力枚举
* **分析**：题目中的`n`可以达到`10^9`，暴力枚举`a`、`b`、`c`、`d`是不可能的。此时需要转换思路：**找特殊情况**。比如，让`gcd`和`lcm`都等于1，这样容易构造解。  
* 💡 **学习笔记**：当问题允许“任意解”时，构造法是最有效的方法。  

### 2. 难点2：利用1的性质简化问题
* **分析**：1是一个特殊的数字，它与任何数的`gcd`都是1，与1的`lcm`也是1。利用这一点，我们可以快速构造出满足条件的`a`、`b`、`c`、`d`。  
* 💡 **学习笔记**：记住数字的特殊性质（比如1、0），能帮你快速解决问题。  

### 3. 难点3：确保四个数都是正整数
* **分析**：`a = n-3`必须是正整数，因此`n≥4`（题目已经保证）。这提醒我们，构造解时要注意**边界条件**。  
* 💡 **学习笔记**：构造解时，一定要检查变量的合法性（比如是否为正整数）。  

### ✨ 解题技巧总结
- **技巧A**：构造法——找特殊情况，简化问题；  
- **技巧B**：利用数字性质——比如1的`gcd`和`lcm`性质；  
- **技巧C**：简洁代码——避免不必要的计算，用最少的代码解决问题。  


## 4. C++核心代码实现赏析

### 本题通用核心C++实现参考
* **说明**：此代码综合了所有优质题解的思路，是最简洁、最有效的实现方式。  
* **完整核心代码**：  
  ```cpp
  #include <iostream>
  using namespace std;

  int main() {
      int t;
      cin >> t;
      while (t--) {
          int n;
          cin >> n;
          cout << n - 3 << " 1 1 1" << endl;
      }
      return 0;
  }
  ```
* **代码解读概要**：  
  代码分为三部分：  
  1. 读取测试组数`t`；  
  2. 循环处理每组数据：读取`n`，输出`n-3`、`1`、`1`、`1`；  
  3. 结束程序。  

### 针对各优质题解的片段赏析

#### 题解一（来源：Dream_weavers）
* **亮点**：直接利用1的性质，代码极其简洁。  
* **核心代码片段**：  
  ```cpp
  printf("%d 1 1 1\n", n - 3);
  ```
* **代码解读**：  
  这行代码输出了四个数：`n-3`、`1`、`1`、`1`。其中，`n-3`是`a`的值，`1`是`b`、`c`、`d`的值。为什么这样写？因为`gcd(n-3,1)=1`，`lcm(1,1)=1`，刚好满足题目条件。  
* 💡 **学习笔记**：用最少的代码解决问题，是编程的最高境界之一。  

#### 题解二（来源：xzyg）
* **亮点**：用`cin`和`cout`处理输入输出，代码可读性好。  
* **核心代码片段**：  
  ```cpp
  cout << 1 << " " << n - 3 << " " << 1 << " " << 1 << endl;
  ```
* **代码解读**：  
  这行代码的输出顺序是`1`、`n-3`、`1`、`1`（即`b=1`，`a=n-3`）。其实，`a`和`b`的顺序可以调换，因为`gcd(a,b)=gcd(b,a)`。这说明，构造解时可以灵活调整变量顺序。  
* 💡 **学习笔记**：变量的顺序不影响结果时，可以灵活调整，让代码更易读。

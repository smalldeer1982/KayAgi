# 题目信息

# Perfect Team

## 题目描述

您可能已经知道标准的ICPC团队正好由三名成员组成。然而，完美的团队有更多的限制。学生可以有一些专业：编码或数学家。她/他可以没有专业化，但不能同时拥有两者。

因此，如果团队至少包括一名编码员，至少有一名数学家，并且由三名成员组成，那么团队就被认为是完美的。

你是一所非常大的大学的教练，你知道你的学生的$c$ 是编码员，$m$ 是数学家，$x$ 没有专业。

您可以分配给他们的最佳完整团队的最大数量是多少？

请注意，有些学生可能没有团队，每个学生都可以成为一个团队的一员。

您还被要求回答$q$ 个独立查询。

## 样例 #1

### 输入

```
6
1 1 1
3 6 0
0 0 0
0 1 1
10 1 10
4 4 1
```

### 输出

```
1
3
0
0
1
3
```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：`贪心策略与数学限制分析` (属于“编程技巧应用”分类)

🗣️ **初步分析**：
解决“Perfect Team”的关键在于理解两个核心限制条件：  
- 每个团队必须至少有1名编码员（c）和1名数学家（m），因此团队数量最多不超过 `min(c, m)`（否则会有某类成员不足）。  
- 每个团队需要3人，因此总人数（c+m+x）必须至少是3倍的团队数，即团队数量最多不超过 `(c+m+x)/3`（向下取整）。  

最终答案是这两个限制中的较小值。例如，当c=4、m=4、x=1时，`min(c,m)=4`，但总人数是9，9/3=3，因此答案是3（样例中的第6组输入）。  

核心算法流程：直接计算两个限制值，取较小者。可视化设计中，我们可以用像素方块分别代表c、m、x，动态演示“取min(c,m)”和“总人数/3”两个条件如何共同限制团队数量，并用颜色高亮当前剩余的各类成员数量。

若采用复古像素风格演示，可设计一个“团队组装机”场景：屏幕左侧是c（蓝色方块）、m（红色方块）、x（黄色方块）的库存区，右侧是已组装的团队（每3个方块组成一个团队，必须包含至少1蓝1红）。每组装一个团队，库存区对应方块减少，同时用文字提示当前的两个限制值，最终取较小值作为结果。关键步骤（如库存不足时的限制触发）伴随“叮”的音效。

## 3. 核心难点辨析与解题策略

<difficulty_intro>
在解决这个问题的过程中，我们通常会遇到以下几个关键点或难点。结合优质题解的共性，我为大家提炼了核心的思考方向和策略：
</difficulty_intro>

1.  **关键点1：理解“至少1名编码员和数学家”的限制**  
    * **分析**：每个团队必须包含至少1名c和1名m，因此团队数量不可能超过c和m中的较小值（否则会有某类成员不够用）。例如，若c=3、m=5，则最多只能组成3个团队（每个团队用1个c），剩下的2个m无法单独组成团队。  
    * 💡 **学习笔记**：团队数量的“上限”由较少的那类专业成员决定。

2.  **关键点2：理解“总人数必须足够3人/团队”的限制**  
    * **分析**：即使c和m足够多，总人数（c+m+x）必须至少是3倍的团队数。例如，c=10、m=10、x=0，总人数20，20/3≈6，因此最多只能组成6个团队（6×3=18人，剩余2人无法组成团队）。  
    * 💡 **学习笔记**：总人数决定了团队数量的“物理上限”。

3.  **关键点3：综合两个限制条件**  
    * **分析**：最终团队数是两个限制值的较小值。例如，当c=4、m=4、x=1时，`min(c,m)=4`，但总人数9/3=3，因此取3。  
    * 💡 **学习笔记**：答案是两个“上限”中的较小者，这是贪心策略的核心。

### ✨ 解题技巧总结
- **问题抽象**：将复杂问题转化为两个数学限制条件的比较，避免复杂分类讨论。  
- **边界处理**：当c或m为0时，直接返回0（无法满足“至少1名c和m”的条件）。  
- **公式简化**：通过观察样例和数学推导，直接得出`min(min(c,m), (c+m+x)/3)`的简洁公式，避免冗余计算。

**本题通用核心C++实现参考**
* **说明**：此代码综合了多个优质题解的思路，采用最简洁的公式直接计算答案，适合竞赛场景。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    #include <algorithm> // 用于min函数
    using namespace std;

    int main() {
        int q;
        cin >> q;
        while (q--) {
            int c, m, x;
            cin >> c >> m >> x;
            int limit1 = min(c, m);         // 限制1：至少1名c和m
            int limit2 = (c + m + x) / 3;   // 限制2：总人数足够3人/团队
            cout << min(limit1, limit2) << '\n';
        }
        return 0;
    }
    ```
* **代码解读概要**：  
  代码首先读取查询次数q，然后对每个查询读取c、m、x。计算两个限制值（limit1和limit2），最后输出它们的较小值。核心逻辑仅需3行，简洁高效。

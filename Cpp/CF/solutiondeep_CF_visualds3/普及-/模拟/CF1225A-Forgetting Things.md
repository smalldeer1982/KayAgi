# Forgetting Things

## 题目描述

### 题意简述

求不定方程

$$a+1=b$$

且 $a$ 的第一位为 $d_a$，$b$ 的第一位为 $d_b$ 的一组解。

## 样例 #1

### 输入

```
1 2
```

### 输出

```
199 200
```

## 样例 #2

### 输入

```
4 4
```

### 输出

```
412 413
```

## 样例 #3

### 输入

```
5 7
```

### 输出

```
-1
```

## 样例 #4

### 输入

```
6 2
```

### 输出

```
-1
```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：`编程技巧应用（分类讨论）`

🗣️ **初步分析**：
解决这道题的关键在于“分类讨论”——也就是把可能的输入情况分成几类，每类单独处理。就像整理书包时，把书本、文具、水杯分开装，不同类别用不同方法处理。本题中，我们需要根据da和db的关系，分情况构造符合条件的a和b。

- **题解思路**：大部分题解将情况分为三类：
  1. **da+1=db**（如样例1的1和2）：此时a是da后面跟多个9（如199），b是db后面跟多个0（如200）；
  2. **da=db**（如样例2的4和4）：此时a是da后面跟0（如412），b是db后面跟1（如413）；
  3. **da=9且db=1**（特殊进位）：此时a=9，b=10；
  其他情况无解，输出-1。
  
- **核心难点**：如何覆盖所有可能的情况，避免遗漏（比如da=9且db=1的特殊进位）；如何构造符合首位要求的a和b（比如确保a的首位确实是da，b的首位是db）。

- **可视化设计**：我们将设计一个“像素数字工厂”动画，用8位像素风格展示不同情况下a和b的构造过程。例如，当da=1、db=2时，动画会显示数字1“长出”多个9变成199，接着199+1变成200（db=2的0结尾），同时用闪烁高亮首位数字，配合“叮”的音效提示关键步骤。

## 3. 核心难点辨析与解题策略

<difficulty_intro>
在解决这个问题时，我们需要重点关注以下三个关键点，掌握它们能帮助我们更高效地构造解：
</difficulty_intro>

1.  **关键点1：如何覆盖所有可能的情况？**
    * **分析**：题目要求a+1=b，且a和b的首位分别为da、db。可能的情况只有三种：da+1=db（a末尾补9，b末尾补0）、da=db（a末尾补0，b末尾补1）、da=9且db=1（特殊进位，如9→10）。其他情况（如da=5、db=7）无法构造，直接输出-1。优质题解通过条件判断覆盖了这三种情况。
    * 💡 **学习笔记**：分类讨论的关键是穷举所有可能的输入组合，确保没有遗漏。

2.  **关键点2：如何构造符合首位要求的a和b？**
    * **分析**：构造时需保证首位不变。例如，当da=1、db=2时，a=199（首位是1），b=200（首位是2）；当da=4、db=4时，a=412（首位是4），b=413（首位是4）。构造方式可以是在da或db后补0、9等数字，只要首位不变即可。
    * 💡 **学习笔记**：构造解时，只需保证首位正确，末尾数字可灵活调整（如补一个或多个0/9）。

3.  **关键点3：如何处理特殊进位情况？**
    * **分析**：当da=9时，a+1会导致进位（如9+1=10），此时b的首位变为1（db=1）。这是唯一可能的da>db的情况，其他da>db的情况（如da=6、db=2）无解。
    * 💡 **学习笔记**：进位是唯一可能让db<da的情况，但仅当da=9时成立。

### ✨ 解题技巧总结
<summary_best_practices>
- **穷举所有可能**：先列出da和db的所有可能关系（da+1=db、da=db、da=9且db=1），再逐一验证。
- **构造灵活解**：构造a和b时，只需保证首位正确，末尾数字可随意补（如补一个9或0），无需复杂计算。
- **特殊情况优先**：将特殊情况（如da=9且db=1）放在条件判断的最前面，避免被其他条件覆盖。
</summary_best_practices>

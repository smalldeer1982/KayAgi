# 题目信息

# Monitor

## 题目描述

Reca company makes monitors, the most popular of their models is AB999 with the screen size $ a×b $ centimeters. Because of some production peculiarities a screen parameters are integer numbers. Recently the screen sides ratio $ x:y $ became popular with users. That's why the company wants to reduce monitor AB999 size so that its screen sides ratio becomes $ x:y $ , at the same time they want its total area to be maximal of all possible variants. Your task is to find the screen parameters of the reduced size model, or find out that such a reduction can't be performed.

## 样例 #1

### 输入

```
800 600 4 3
```

### 输出

```
800 600
```

## 样例 #2

### 输入

```
1920 1200 16 9
```

### 输出

```
1920 1080
```

## 样例 #3

### 输入

```
1 1 1 2
```

### 输出

```
0 0
```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：数学（比例化简、最大公约数应用）

🗣️ **初步分析**：
解决“Monitor”问题的关键在于运用数学中的比例化简和最大公约数（GCD）计算。简单来说，比例化简就像把分数约分成最简形式——比如4:6可以约成2:3。在本题中，我们需要先将给定的比例x:y约简为最简整数比，再计算原屏幕尺寸能容纳多少个这样的“最简单元”，从而得到最大可能的新尺寸。

- **题解思路**：大部分题解的核心思路是：  
  1️⃣ 用GCD将x和y约简为最简比（如x=4,y=3约简后还是4:3；x=8,y=6约简后是4:3）；  
  2️⃣ 计算原尺寸a和b分别能容纳多少个“x单元”和“y单元”（即a/x和b/y）；  
  3️⃣ 取这两个值的最小值作为倍数，乘以x和y得到新尺寸；  
  4️⃣ 若原尺寸无法容纳至少一个“x单元”或“y单元”（即a<x或b<y），则输出0 0。  
  不同题解的差异主要在代码实现细节（如输入输出方式、变量命名），核心逻辑一致。  

- **核心算法流程**：关键步骤是比例化简和倍数计算。例如，样例2中输入1920 1200 16 9，约简后16:9（已是最简），计算1920/16=120，1200/9≈133.33，取最小值120，新尺寸为16×120=1920，9×120=1080，与样例输出一致。  

- **可视化设计思路**：计划采用8位像素风格动画，用“像素方块”表示“x单元”和“y单元”，动态展示原尺寸a和b如何“切割”成这些单元，高亮显示取最小值的过程，并通过音效（如“叮”声）提示关键操作。

## 3. 核心难点辨析与解题策略

<difficulty_intro>
在解决这个问题时，我们可能会遇到以下关键点或难点。结合优质题解的共性，我为大家提炼了思考方向和策略：
</difficulty_intro>

1.  **关键点1：如何正确约简比例x:y？**  
    * **分析**：若x和y有公约数（如x=8,y=6的公约数是2），直接使用原比例会导致计算错误（例如误将8:6当作最大单元）。优质题解通过GCD计算x和y的最大公约数，将其约简为最简比（如8/2=4，6/2=3，得到4:3），确保后续计算的“单元”是最小可能的。  
    * 💡 **学习笔记**：比例约简是本题的第一步，也是关键——只有最简比才能正确计算“单元”数量。

2.  **关键点2：为什么取min(a/x, b/y)作为倍数？**  
    * **分析**：假设x=4,y=3，原尺寸a=800,b=600。a/x=200表示a能容纳200个x单元，b/y=200表示b能容纳200个y单元。若取较大值（如200），则x×200=800≤a，y×200=600≤b，刚好填满。若原尺寸a=1920,b=1200，x=16,y=9，则a/x=120，b/y≈133.33，取较小的120，才能保证y×120=1080≤b=1200。因此，取最小值是为了确保新尺寸的长和宽都不超过原尺寸。  
    * 💡 **学习笔记**：倍数取min(a/x, b/y)是为了同时满足长和宽的约束。

3.  **关键点3：如何处理“无法调整”的情况？**  
    * **分析**：若约简后的x>a或y>b（如样例3中x=1,y=2，原尺寸a=1,b=1，y=2>b=1），则无法找到任何非负的c和d使得新尺寸满足比例，此时应输出0 0。优质题解通过特判a<x或b<y来处理这种情况。  
    * 💡 **学习笔记**：特判边界条件（如a<x或b<y）是避免错误的关键。

### ✨ 解题技巧总结
- **比例约简优先**：处理比例问题时，先约简为最简比，避免后续计算错误。  
- **边界条件特判**：在代码中优先检查“无法调整”的情况（如a<x或b<y），提前返回结果。  
- **变量命名清晰**：使用如base（倍数）、gcd（最大公约数）等直观的变量名，提高代码可读性。

**本题通用核心C++实现参考**  
* **说明**：此代码综合了RagnaLP等优质题解的思路，逻辑清晰且高效，适合直接学习和使用。  
* **完整核心代码**：
    ```cpp
    #include <iostream>
    #include <algorithm> // 用于std::min和__gcd（部分编译器可能需要替换为自定义gcd）
    using namespace std;

    int main() {
        int a, b, x, y;
        cin >> a >> b >> x >> y;

        // 约简x和y为最简比
        int g = __gcd(x, y);
        x /= g;
        y /= g;

        // 特判：原尺寸无法容纳至少一个单元
        if (a < x || b < y) {
            cout << "0 0" << endl;
            return 0;
        }

        // 计算最大倍数（取a/x和b/y的最小值）
        int base = min(a / x, b / y);
        int new_a = x * base;
        int new_b = y * base;

        cout << new_a << " " << new_b << endl;
        return 0;
    }
    ```
* **代码解读概要**：  
  代码首先读取输入，然后通过`__gcd`函数约简x和y为最简比。接着检查原尺寸是否能容纳至少一个“x单元”或“y单元”（特判），若不能则输出0 0。若能，则计算最大倍数`base`（取a/x和b/y的最小值），最后输出新尺寸。

* **核心代码片段**：
    ```cpp
    int gcd(int n, int m) {
        if (n < m) swap(n, m);
        return n == 0 ? n : gcd(m, n % m);
    }

    int main() {
        cin >> a >> b >> x >> y;
        int k = gcd(x, y);
        x /= k, y /= k;
        if (a < x || b < y) {
            printf("0 0");
            return 0;
        }
        int base = min(a / x, b / y);
        printf("%d %d", x * base, y * base);
    }
    ```
* **代码解读**：  
  `gcd`函数通过辗转相除法计算最大公约数，用于约简x和y。主函数中，先约简比例，再检查原尺寸是否足够。若足够，计算`base`（a/x和b/y的最小值），最后输出新尺寸。这里的`base`是“能容纳的最大单元数”，确保新尺寸不超过原尺寸。  
* 💡 **学习笔记**：自定义gcd函数在没有内置函数时很实用，变量名`base`直观表示“倍数基数”。

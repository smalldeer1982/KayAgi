# Dasha and Stairs

## 题目描述

老虎Dasha在她前往编程学校的路上遇到了第一个挑战 — 一个巨大的楼梯！

这些台阶被从一到无穷编号。据我们所知，老虎喜欢所有有斑纹的东西，那可能就像是他们身上的颜色。所以在她的路上的一些时间她计算了两个值 — 台阶编号的奇偶数的数量。

你需要检查一下是否存在一段从第$l$ 个台阶到第$r$ 个台阶（$1\leq l\leq r$ ），Dasha认为其（奇偶）值是合适的。

## 说明/提示

在第一个样例中一段合适的区间为$1$ 到$5$ 。这段区间包含了两个偶数台阶 — $2$ 和$4$ ,和三个奇数：$1$ ，$3$ 和$5$ 。

（注：由于题面或我的问题，这样翻译其实题意并不明，附上题目大意）

#### 题目大意

给你两个数，分别为偶数的数量$a$ 和奇数的数量$b$ ，问是否存在一段连续区间$[l,r]$ 其奇偶数的数目为$a$ 和$b$ （就是所谓的合适的）。

### 输入

```
2 3
```

### 输出

```
YES
```

## 样例 #2

### 输入

```
3 1
```

### 输出

```
NO
```

- **题解思路**：所有优质题解都基于同一核心规律——连续区间内奇数和偶数的数量差绝对值不超过1，且不能同时为0（因为区间长度至少为1）。

## 3. 核心难点辨析与解题策略

<difficulty_intro>
解决这道题的过程中，我们需要突破以下几个关键点：
</difficulty_intro>

1.  **关键点1**：如何发现“奇偶数量差≤1”的规律？
    * **分析**：连续整数序列的奇偶是交替出现的（如1奇、2偶、3奇……）。假设区间长度为$k$，当$k$为偶数时，奇偶数量相等（各$k/2$个）；当$k$为奇数时，奇偶数量差1（多的那个是区间起点的奇偶性）。因此，任意连续区间的奇偶数量差只能是0或1。
    * 💡 **学习笔记**：观察小例子（如区间[1,5]有3奇2偶，[2,5]有2奇2偶）能快速发现规律。

2.  **关键点2**：为什么要特判“$a=0$且$b=0$”的情况？
    * **分析**：题目要求区间$[l, r]$满足$1 \leq l \leq r$，即区间至少包含1个台阶。若$a=0$且$b=0$，说明区间内没有台阶，与题意矛盾。
    * 💡 **学习笔记**：边界条件（如空输入、最小/最大值）是编程中容易出错的地方，需要特别注意。

3.  **关键点3**：如何验证规律的正确性？
    * **分析**：可以通过反证法。假设存在一个区间奇偶数量差≥2，例如$a=3$，$b=1$（差2）。但连续序列中，奇偶交替出现，最多差1，因此这样的区间不存在。
    * 💡 **学习笔记**：通过反例验证规律是数学推导的常用方法。

### ✨ 解题技巧总结
- **观察小例子**：通过列举短区间（如[1,2]、[1,3]）观察规律，快速找到核心条件。
- **边界条件检查**：题目中“区间至少有一个台阶”对应“$a$和$b$不能同时为0”，需单独处理。
- **简洁条件表达**：用`abs(a - b) <= 1`代替复杂的分支判断，提高代码可读性。

## 4. C++核心代码实现赏析

<code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：此代码综合了多个优质题解的思路，通过判断奇偶数量差和排除全零情况，完整解决问题。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    #include <cmath>
    using namespace std;

    int main() {
        int a, b;
        cin >> a >> b;
        // 条件1：奇偶数量差绝对值≤1；条件2：不同时为0
        if (abs(a - b) <= 1 && (a != 0 || b != 0)) {
            cout << "YES" << endl;
        } else {
            cout << "NO" << endl;
        }
        return 0;
    }
    ```
* **代码解读概要**：
    > 代码首先读取输入的$a$（偶数数量）和$b$（奇数数量）。通过`abs(a - b) <= 1`判断奇偶数量差是否符合要求，再通过`(a != 0 || b != 0)`排除全零情况。两个条件同时满足时输出“YES”，否则输出“NO”。

**题解一：作者 清清老大**

    ```cpp
    if(abs(a - b) <= 1 && !(a == 0 && b == 0)) {
        puts("YES");
    } else {

    }
    ```
* **代码解读**：
    > `abs(a - b) <= 1`确保奇偶数量差不超过1；`!(a == 0 && b == 0)`等价于“$a$和$b$不同时为0”。两个条件用`&&`连接，只有同时满足时才输出“YES”。这里用`puts`代替`cout`，输出更高效。
* 💡 **学习笔记**：逻辑非`!`可以简化复杂的条件判断，使代码更易读。

**题解二：作者 Eason_AC**
* **亮点**：用`(n || m)`简洁处理全零情况（当$n$或$m$至少一个非零时为真）。
* **核心代码片段**：
    ```cpp
    printf(abs(n - m) <= 1 && (n || m) ? "YES" : "NO");
    ```
* **代码解读**：
    > 这是一个三目运算符的经典应用。`abs(n - m) <= 1`判断数量差，`(n || m)`判断是否全零。两个条件都满足时输出“YES”，否则“NO”。代码仅用一行完成所有逻辑，简洁高效。
* 💡 **学习笔记**：三目运算符适合简单的条件输出，能让代码更紧凑。

**题解三：作者 x义x**
* **亮点**：先特判全零情况，再判断数量差，逻辑清晰。
* **核心代码片段**：
    ```cpp
    if(a == 0 && b == 0) { cout << "NO"; return 0; }
    if(abs(a - b) <= 1) cout << "YES"; else cout << "NO";
    ```
* **代码解读**：
    > 首先处理全零情况（直接输出“NO”并结束程序），再判断数量差是否≤1。这种“先处理特殊情况，再处理一般情况”的逻辑顺序，能避免条件嵌套过深，提高代码可读性。
* 💡 **学习笔记**：将特殊情况提前处理，是减少代码复杂度的常用技巧。

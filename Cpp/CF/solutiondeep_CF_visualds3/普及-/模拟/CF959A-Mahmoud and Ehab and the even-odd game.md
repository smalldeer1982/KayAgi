# 题目信息

# Mahmoud and Ehab and the even-odd game

## 题目描述

Mahmoud和Ehab正在玩一款叫做“奇偶”的游戏。Ehab选择了一个他最喜欢的整数n然后他们开始玩这个游戏（从Mahmoud开始）在每个人的回合中，他必须选择一个数a，然后从n中减去它。

条件：

     1<=a<=n.
     如果轮到Mahmoud，a必须是偶数。
     但如果轮到Ehab，a必须是奇数。
     
如果当前的玩家不能选择任何满足条件的数字，他就会输。请输出谁获胜了。

## 说明/提示

在样例1中，Mahmoud不能选择任何一个整数，因为没有一个正偶数小于（等于）1。所以Ehab获胜了。

在样例2中，Mahmoud应当选择a为2。从n中减去2。轮到Ehab的回合了，此时n=0。因为没有小于（等于）0的正奇数，所以Mahmoud获胜了。

## 样例 #1

### 输入

```
1
```

### 输出

```
Ehab```

## 样例 #2

### 输入

```
2
```

### 输出

```
Mahmoud```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：数学逻辑（奇偶性判断）

🗣️ **初步分析**：  
这道题就像一场“奇偶取数游戏”——Mahmoud（先手）只能取偶数，Ehab（后手）只能取奇数，谁没法取谁输。解决问题的关键其实藏在**奇偶性**里：  
- 如果n是**偶数**，Mahmoud可以直接取走整个n（因为偶数≤n），这样Ehab面对0，没有奇数可取，Mahmoud赢！  
- 如果n是**奇数**，Mahmoud只能取一个偶数（比如2、4…），剩下的n会变成奇数（奇数-偶数=奇数）。这时Ehab可以取走剩下的所有奇数，Mahmoud输！  

简单来说，**n是偶数→Mahmoud赢，n是奇数→Ehab赢**。是不是像“猜硬币”一样，看n的“奇偶脸”就能直接知道结果？  

**可视化设计思路**：  
我打算做一个**8位像素风的“取数对战”动画**——屏幕中间显示大数字n（比如“1”或“2”），左边是Mahmoud（蓝色像素人），右边是Ehab（红色像素人）。当n是偶数时，Mahmoud会跳起来“拿走”n（数字变成0），Ehab摊手表示无奈，伴随“叮~”的胜利音效；当n是奇数时，Mahmoud只能拿走一个小偶数（比如n=3时拿走2，剩下1），Ehab立刻拿走1，数字变成0，Ehab欢呼，播放“耶~”的音效。动画还会有“单步执行”“自动播放”按钮，让你慢慢看每一步的变化！

1. **关键点1：理解“无法取数”的条件**  
   * **分析**：Mahmoud输的情况是“没有偶数≤n”（比如n=1），Ehab输的情况是“没有奇数≤n”（比如n=0）。题解中都抓住了“n的奇偶性直接导致某一方无法操作”的核心。  
   * 💡 **学习笔记**：游戏的胜负条件往往藏在“无法行动”的边界情况里。


2. **关键点2：先手的“最优策略”**  
   * **分析**：Mahmoud作为先手，最优选择是“直接取完n（如果n是偶数）”，这样能立刻结束游戏。题解中都用到了这个策略，因为这是赢的最快方式。  
   * 💡 **学习笔记**：先手要赢，往往要找“一步致胜”的方法。


3. **关键点3：奇偶性的“传递性”**  
   * **分析**：奇数-偶数=奇数（比如3-2=1），偶数-奇数=奇数（比如4-1=3），但本题中Mahmoud只能取偶数，所以奇数的n会让Mahmoud无法一步取完，进而输掉。题解中都利用了这个数学性质。  
   * 💡 **学习笔记**：数学性质是解决游戏问题的“秘密武器”。

  #include <iostream>
  using namespace std;

  int main() {
      int n;

      }
      return 0;
  }
  ```  
* **代码解读概要**：  
  1. 读取输入的n；  
  2. 用`n%2==1`判断n是否为奇数；

**题解一：小闸蟹（亮点：用`puts`提高效率）**

* **代码解读**：

**题解二：lukelin（亮点：快速读取函数）**  
* **亮点**：`read`函数用于快速读取大整数，避免`cin`的慢速度（虽然本题用不上，但竞赛中常用）。  
* **核心代码片段**：  
  ```cpp
  inline ll read(){
      ll x = 0; int zf = 1; char ch = ' ';
      while (ch != '-' && (ch < '0' || ch > '9')) ch = getchar();
      if (ch == '-') zf = -1, ch = getchar();
      while (ch >= '0' && ch <= '9') x = x * 10 + ch - '0', ch = getchar(); return x * zf;
  }
  ```  
* **代码解读**：  
  这个函数通过`getchar`逐个读取字符，转换成整数，比`cin`快很多。如果以后遇到“输入很大”的题目，可以用这个函数。  
* 💡 **学习笔记**：快速读取是竞赛中的“必备技能”。


**题解三：applese（亮点：用`long long`防止溢出）**

* **核心代码片段**：  
  ```cpp

  cin >> a;
  ```  
* **代码解读**：  
  `int`只能存储到2e9左右，`long long`能存储到9e18，避免输入大数字时“溢出”错误。这是一个“防坑”的好习惯。  
* 💡 **学习笔记**：用`long long`存储大整数，避免溢出。

# 题目信息

# Table Decorations

## 题目描述

您正在为宴会装饰桌子，有 $r$ 个红色，$g$ 个绿色和 $b$ 蓝色的气球，一张桌子需要三个颜色不完全相同的气球来装饰。

您需要求出可以用所需方式进行装饰的桌子的最大数量 $t$。

## 样例 #1

### 输入

```
5 4 3
```

### 输出

```
4
```

## 样例 #2

### 输入

```
1 1 1
```

### 输出

```
1
```

## 样例 #3

### 输入

```
2 3 3
```

### 输出

```
2
```

### Kay的C++算法解析：Table Decorations 深入学习指南

#### 1. 题目解读与核心算法识别
✨ **本题主要考察**：`贪心算法`

🗣️ **初步分析**：
> 解决"Table Decorations"的关键在于理解**气球分配的约束条件**。想象你是一位宴会策划师，手上有红/绿/蓝三色气球包，每桌需要3个气球且不能全同色。贪心算法就像高效分配气球：优先用光数量少的气球，避免某种颜色过剩。

- **核心思路**：桌数受限于两个条件：  
  ① 气球总数限制：`(r+g+b)/3`  
  ② 任意两色组合限制：`min(r+g, r+b, g+b)`  
- **难点解析**：当某颜色极多时（如r=7,g=1,b=1），虽总数满足3桌，但受限于绿+蓝=2，只能布置2桌。
- **可视化设计**：采用**像素气球堆**动画，左侧显示三色气球山，右侧动态计算四个约束值（红绿堆/红蓝堆/绿蓝堆/总气球堆），分配时高亮当前最小约束值。

#### 3. 核心难点辨析与解题策略
1. **约束条件识别**  
   *分析*：需同时考虑总气球数和任意两色组合的可行性。如样例3（2,3,3）：总球数8→理论2.6桌，但红+绿=5、红+蓝=5、绿+蓝=6，取整后均支持2桌。  
   💡 **学习笔记**：桌数由最紧张的两色组合决定！

2. **贪心策略证明**  
   *分析*：当最大颜色气球过半时（如r>2*(g+b)），每桌最多消耗2个红色+1个其他色，此时桌数≤g+b。否则可混合分配（如1红1绿1蓝）。  
   💡 **学习笔记**：贪心的本质是避免单一颜色浪费。

3. **数据范围处理**  
   *分析*：气球数≤10¹⁸，需用`long long`。优质题解均用`read()`或`cin`读入`long long`。  
   💡 **学习笔记**：大数据必查整数溢出！

### ✨ 解题技巧总结
- **约束转化法**：将"颜色不完全相同"转化为数学约束（∑气球/3 和 min(两色和)）
- **极值验证法**：构造极端用例（如7:1:1）验证算法鲁棒性
- **防御性编程**：大数据类型统一用`long long`

#### 4. C++核心代码实现赏析
**通用核心实现**
```cpp
#include <iostream>
#include <algorithm>
using namespace std;
int main() {
    long long r, g, b;
    cin >> r >> g >> b;
    cout << min({(r+g+b)/3, r+g, r+b, g+b});
    return 0;
}
```
* **说明**：综合两题解精华，C++11初始化列表简化嵌套min
* **解读概要**：  
  `min({A,B,C,D})`同时计算四个约束值：  
  `A`=总气球限制 → `(r+g+b)/3`  
  `B,C,D`=两色组合限制 → `r+g`, `r+b`, `g+b`

**题解一片段赏析**
```cpp
min(min(min((r+g+b)/3,r+g),r+b),b+g)
```
* **亮点**：纯数学实现，无冗余操作
* **解读**：  
  > 嵌套`min`从内向外计算：  
  > ① 先比`(r+g+b)/3`和`r+g`  
  > ② 结果再比`r+b`  
  > ③ 最终比`g+b`  
  > 如同筛选最小孔径的漏斗
* 💡 **学习笔记**：嵌套min是约束判定的高效写法

**题解二片段赏析**
```cpp
cout<<min(min(min((r+g+b)/3,r+g),r+b),b+g);
```
* **亮点**：IO与算法分离，强调核心逻辑
* **解读**：  
  > `cin>>r>>g>>b`集中处理输入  
  > 核心算法仅1行 → 避免干扰  
  > 注意：整数除法自动向下取整
* 💡 **学习笔记**：代码模块化提升可读性

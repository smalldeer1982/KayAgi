# Hungry Sequence

## 题目描述

Iahub and Iahubina went to a date at a luxury restaurant. Everything went fine until paying for the food. Instead of money, the waiter wants Iahub to write a Hungry sequence consisting of $ n $ integers.

A sequence $ a_{1} $ , $ a_{2} $ , ..., $ a_{n} $ , consisting of $ n $ integers, is Hungry if and only if:

- Its elements are in increasing order. That is an inequality $ a_{i}&lt;a_{j} $ holds for any two indices $ i,j $ $ (i&lt;j) $ .
- For any two indices $ i $ and $ j $ $ (i&lt;j) $ , $ a_{j} $ must not be divisible by $ a_{i} $ .

Iahub is in trouble, so he asks you for help. Find a Hungry sequence with $ n $ elements.

## 样例 #1

### 输入

```
3
```

### 输出

```
2 9 15
```

## 样例 #2

### 输入

```
5
```

### 输出

```
11 14 20 27 31
```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：`构造法应用` (通过设计“天生符合条件”的序列直接解决问题)

🗣️ **初步分析**：  
解决“Hungry Sequence”的关键，是**找到一组“自带条件buff”的数字**——不需要逐个检查，它们天生满足“递增+互不整除”的要求。这就像搭积木：你不用每块都打磨，直接选一组刚好能拼成房子的积木就行！  

本题有两种经典构造思路：  
1. **连续整数法**：输出`n到2n-1`的连续数字（比如n=3时输出3、4、5）。核心逻辑是：两个数相差小于较小数时，大数肯定不能被小数整除（比如3和4差1<3，所以4不被3整除；3和5差2<3，所以5不被3整除）。  
2. **质数序列法**：输出前n个质数（比如n=3时输出2、3、5）。核心逻辑是：质数之间互质，自然不会有整除关系（除了1，但题目要求a_i≥2）。  

**可视化设计思路**：  
我们会用`8位像素风`演示连续整数法——想象一个复古游戏里的“数字传送带”：输入n后，传送带上依次滚出n、n+1…2n-1，每出现一个数字，旁边的像素屏幕会用“绿色对勾”显示“检查通过”，并播放“叮”的音效。通过“单步执行”，你能清楚看到每个数字如何满足条件！

## 3. 核心难点辨析与解题策略

<difficulty_intro>
构造题的核心不是“计算”，而是“观察”。以下三个关键点，帮你打通构造思路：
</difficulty_intro>

1.  **关键点1：怎么想到用“构造法”而不是“计算法”？**  
    * **分析**：很多问题需要“算答案”（比如求最大值），但构造题需要“找答案”——找到一组天生符合条件的结构。比如本题，“连续整数”或“质数”都是“自带条件buff”的结构，比“逐个检查每个数是否符合条件”高效得多。  
    * 💡 **学习笔记**：构造法的关键是“观察条件的结构特征”，而不是“暴力枚举”。

2.  **关键点2：连续整数法的正确性怎么证明？**  
    * **分析**：假设序列是`a₁=n, a₂=n+1,…,aₙ=2n-1`。对于任意i<j，`aⱼ - aᵢ = j-i < n ≤ aᵢ`（因为i≥1，aᵢ≥n）。如果aⱼ能被aᵢ整除，那么aⱼ=k*aᵢ（k≥2），所以aⱼ≥2*aᵢ≥2n，但aⱼ≤2n-1，矛盾！因此连续整数法一定正确。  
    * 💡 **学习笔记**：构造的序列要“证明正确”，反证法是常用工具。

3.  **关键点3：质数筛法的效率怎么保证？**  
    * **分析**：质数筛法有两种：埃氏筛（O(n log log n)）和欧拉筛（O(n)）。对于n≤1e5，埃氏筛已经足够快——比如找前1e5个质数，埃氏筛只需要遍历到约1e6（第1e5个质数是1299709，小于1e7）。  
    * 💡 **学习笔记**：选择筛法要权衡“时间”和“空间”——埃氏筛代码简单，欧拉筛更高效但代码稍复杂。


### ✨ 解题技巧总结
- **技巧1：找“结构特征”**：遇到“构造符合条件的序列”问题，先想“有没有天生符合条件的结构”（比如连续数、质数、等差数列）。  
- **技巧2：用旧知识迁移**：比如用“互质”“相差1的数不整除”等小学数学知识，简化问题。  
- **技巧3：证明构造的正确性**：构造出序列后，一定要用逻辑推导证明它符合条件，避免“想当然”。

### 本题通用核心C++实现参考

* **完整核心代码**：

```cpp
#include <iostream>
using namespace std;

int main() {
    int n;
    cin >> n;
    for (int i = n; i <= 2 * n - 1; ++i) {
        cout << i << " ";
    }
    return 0;
}
```
* **代码解读概要**：  
  1. 读入n；  
  2. 循环输出从n到2n-1的所有整数；  
  3. 直接构造符合条件的序列，没有多余步骤！

### 题解一：连续整数法（作者：yydfj）

* **核心代码片段**：

* **代码解读**：  
  这行循环是整个题解的核心——从n开始，输出到2n-1，刚好n个数。为什么选这个范围？因为前面证明过，这个范围内的数两两不整除！  
* 💡 **学习笔记**：构造法的代码往往很简洁，关键是“想对思路”。


### 题解二：质数筛法（作者：Rosmarinus）
* **亮点**：用埃氏筛找质数，思路直接，适合巩固质数知识！  
* **核心代码片段**：
```cpp
bool p[maxn]; // p[i]为true表示i是合数
void Work() {
    for (int i = 2; i < maxn; i++) {
        if (!p[i]) { // i是质数
            printf("%d ", i);
            n--;
            if (n <= 0) return;
            // 标记i的倍数为合数
            for (int j = i + i; j < maxn; j += i) {
                p[j] = true;
            }
        }
    }
}
```
* **代码解读**：  
  1. 数组`p`标记合数（初始都是false）；  
  2. 从2开始遍历：如果`p[i]`是false（i是质数），输出它，然后标记所有i的倍数为合数；  
  3. 直到输出n个质数，停止。  
  问：为什么要标记倍数？因为质数的倍数都是合数，这样下次遇到就不会再输出了！  
* 💡 **学习笔记**：埃氏筛的核心是“标记质数的倍数”，从而快速找到所有质数。

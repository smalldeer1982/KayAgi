## 题目描述

Misha and Vasya participated in a Codeforces contest. Unfortunately, each of them solved only one problem, though successfully submitted it at the first attempt. Misha solved the problem that costs $ a $ points and Vasya solved the problem that costs $ b $ points. Besides, Misha submitted the problem $ c $ minutes after the contest started and Vasya submitted the problem $ d $ minutes after the contest started. As you know, on Codeforces the cost of a problem reduces as a round continues. That is, if you submit a problem that costs $ p $ points $ t $ minutes after the contest started, you get ![](https://cdn.luogu.com.cn/upload/vjudge_pic/CF501A/0c1954e97a47d9a6cd1567e3d0ad4f4a76ada737.png) points.

Misha and Vasya are having an argument trying to find out who got more points. Help them to find out the truth.

## 样例 #1

### 输入

```
500 1000 20 30
```

### 输出

```
Vasya
```

## 样例 #2

### 输入

```
1000 1000 1 1
```

### 输出

```
Tie
```

## 样例 #3

### 输入

```
1500 1000 176 177
```

### 输出

```
Misha
```

## 3. 核心难点辨析与解题策略

<difficulty_intro>
在解决这道题的过程中，我们需要关注以下三个核心难点，并掌握对应的解决策略：
</difficulty_intro>

1.  **关键点1**：正确理解并计算得分公式  
    * **分析**：题目给出得分公式是`max(3p/10, p - p*t/250)`，需注意两个得分项的计算方式。例如，`3p/10`是直接取原分的30%，而`p - p*t/250`是原分减去随时间的扣分项（每过1分钟扣`p/250`分）。优质题解均严格按照此公式实现计算。
    * 💡 **学习笔记**：公式类问题需逐字核对题目描述，确保代码与公式完全一致。

2.  **关键点2**：选择合适的数据类型  
    * **分析**：若使用`int`计算（如`a*3/10`），可能因整数除法丢失精度（如`500*3/10=150`正确，但`501*3/10=150`会丢失0.3）。因此，更安全的方式是用`double`存储得分项，保留小数精度。
    * 💡 **学习笔记**：涉及分数或除法时，优先用`double`类型避免精度丢失。

3.  **关键点3**：浮点数比较的精度处理  
    * **分析**：直接比较两个`double`数（如`x==y`）可能因精度误差误判。例如，`x=100.0000001`和`y=100.0`实际应视为相等，但直接比较会认为不等。优质题解（如_Clown_的题解）通过引入`EPS`（如`1e-6`）判断是否接近，解决了这一问题。
    * 💡 **学习笔记**：浮点数比较时，用`abs(x - y) < EPS`代替直接比较。

### ✨ 解题技巧总结
<summary_best_practices>
- 公式类问题：先画“计算流程图”，明确每一步的输入和输出，再转化为代码。
- 数据类型选择：不确定是否会丢失精度时，优先用`double`。
- 浮点数比较：养成用`EPS`的习惯（如`1e-6`），避免因精度问题出错。

## 4. C++核心代码实现赏析

<code_intro_overall>
综合优质题解的思路，我们提炼一个兼顾清晰性和鲁棒性的通用核心实现，帮助大家快速掌握整体框架。
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：此代码综合了_Clown_和Provider的题解思路，使用`double`保留精度，并用`EPS`处理浮点数比较，适合竞赛场景。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    #include <cmath> // 用于abs函数计算精度差
    using namespace std;

    const double EPS = 1e-6; // 定义精度误差范围

    int main() {
        double a, b, c, d;
        cin >> a >> b >> c >> d;

        // 计算Misha的两个得分项并取最大值
        double misha_score1 = 0.3 * a;
        double misha_score2 = a - (a * c) / 250.0;
        double misha_total = max(misha_score1, misha_score2);

        // 计算Vasya的两个得分项并取最大值
        double vasya_score1 = 0.3 * b;
        double vasya_score2 = b - (b * d) / 250.0;
        double vasya_total = max(vasya_score1, vasya_score2);

        // 比较得分（考虑精度误差）
        if (misha_total - vasya_total > EPS) {
            cout << "Misha";
        } else if (vasya_total - misha_total > EPS) {
            cout << "Vasya";
        } else {
            cout << "Tie";
        }

        return 0;
    }
    ```
* **代码解读概要**：
    > 代码首先读取输入的四个参数（`a,b,c,d`），然后分别计算Misha和Vasya的两个得分项（`0.3*p`和`p - p*t/250`），用`max`函数取较大值作为最终得分。最后通过比较两个最终得分（考虑`EPS`精度误差），输出结果。

# 题目信息

# Login Verification

## 题目描述

登录名是由小写和大写拉丁字母、数字和下划线符号“_”组成的任意序列。
如果登录名与已经存在的登录名类似，则禁止注册。
如果我们可以通过以下类型的操作组成的序列将S转化成T，则认为S,T是类似的。
- 将小写字母转换为大写字母，将大写字母转换为小写字母
- 将字母“O”更改为数字“0”,将数字“0”更改为字母“O”
- "1""l""L"相互转化
例如，
登录名«codeforces»和«codefwrces»，
«oo0oo00oo0oo0oo0oo_lol»和«oo0oo0oo0oo0oo0oo0oo0oo0oo_1oo»是类似的，
而«codeforces»和«code_forces»则不是。
给你一个登录名的列表,其中没有两个类似的登录名。
检查新登录名是否和已有登录名类似。

✨ **本题主要考察**：`模拟与字符串处理` 

🗣️ **初步分析**：
解决“Login Verification”的关键在于将所有登录名转换为统一的“标准形式”，从而快速判断是否存在类似登录名。简单来说，“模拟”在这里指的是严格按照题目要求的三种转换规则（大小写互转、O/0互转、1/l/L互转），将每个字符串处理成唯一的标准形式。例如，我们可以把所有字符转小写，再将O→0，l→1，i→1，这样不同登录名的“相似性”就会被统一为相同的标准字符串。

- **题解思路对比**：多数题解采用“转换函数+判重”的思路。差异在于转换细节（如转大写/小写）和判重方式（vector+find、map、直接遍历）。例如，Miller2019的题解通过vector存储转换后的字符串，用find函数判重；hensier的题解则直接逐个比较转换后的字符串。
- **核心算法流程**：首先定义转换函数，将输入字符串按规则标准化；然后将已有登录名转换后存储；最后将新登录名转换后，检查是否存在于存储的列表中。

## 3. 核心难点辨析与解题策略

<difficulty_intro>
在解决本题时，以下三个关键点需要特别注意：
</difficulty_intro>

1.  **关键点1：正确实现字符转换规则**  
    * **分析**：题目要求处理三种转换规则，需确保每个字符都被正确映射。例如，O和0互转，l、L、1互转，大小写互转。若转换顺序错误（如先处理O→0再转小写，可能漏掉大写O的情况），会导致标准形式错误。优质题解（如Miller2019）通常先统一转小写/大写，再处理特殊字符，避免遗漏。
    * 💡 **学习笔记**：转换顺序很重要！先统一大小写，再处理特殊字符（如O、l、i），能简化逻辑。

2.  **关键点2：判重数据结构的选择**  
    * **分析**：判重可以用vector+find（简单直观）、map（查询快）或直接遍历（代码简洁）。vector+find的时间复杂度为O(n*m)，适合小规模数据；map的查询复杂度为O(log n)，适合大规模数据。选择时需根据题目数据范围决定。
    * 💡 **学习笔记**：数据规模较小时，vector足够；数据规模大时，map更高效。

3.  **关键点3：处理边界条件**  
    * **分析**：需注意字符串长度不同的情况（直接排除）、全小写/大写的情况（如样例5中“abc”与“aBc”转换后相同吗？实际样例5输出No，说明大小写转换后需统一为同一形式）。优质题解（如hensier）通过转换函数统一处理，避免了边界错误。
    * 💡 **学习笔记**：转换后的字符串长度必须相同，否则直接不相似。

### ✨ 解题技巧总结
- **统一转换规则**：将所有字符先转小写/大写，再处理特殊字符（O→0，l→1等），避免混合处理导致的错误。
- **输入输出优化**：使用`ios::sync_with_stdio(false)`关闭同步，提升cin/cout速度，尤其在处理大量数据时。
- **判重前先转换**：新登录名和已有登录名必须都经过转换，否则无法正确比较（如bigdoge的第一次提交因未转换原用户名导致错误）。

**本题通用核心C++实现参考**
* **说明**：此代码综合了Miller2019和hensier的思路，采用转换函数统一处理字符，并用vector存储转换后的字符串，适合初学者理解核心逻辑。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    #include <vector>
    #include <algorithm>
    #include <cctype>
    using namespace std;

    string standardize(string s) {
        for (char &c : s) {
            c = tolower(c); // 先统一转小写
            if (c == 'o') c = '0';
            else if (c == 'l' || c == 'i') c = '1';
        }
        return s;
    }

    int main() {
        ios::sync_with_stdio(false);
        cin.tie(nullptr);

        string reg;
        int n;
        cin >> reg >> n;

        string target = standardize(reg);
        vector<string> existing;

        for (int i = 0; i < n; ++i) {
            string s;
            cin >> s;
            existing.push_back(standardize(s));
        }

        if (find(existing.begin(), existing.end(), target) != existing.end()) {
            cout << "No\n";
        } else {
            cout << "Yes\n";
        }

        return 0;
    }
    ```
* **代码解读概要**：代码首先定义`standardize`函数，将字符串转小写并处理O、l、i为0、1；主函数读取新登录名和已有登录名，转换后存储到vector中；最后用find函数检查是否存在重复，输出结果。

    ```cpp
    string change(string str) {
        string ans=str;
        int len=str.size();
        for(register int t=0;t<len;t++) {
            ans[t]=tolower(ans[t]);
            switch(ans[t]) {
                case 'o': ans[t]='0'; break;
                case 'l': ans[t]='1'; break;
                case 'i': ans[t]='1'; break;
            }
        }
        return ans;
    }
    ```
* **代码解读**：`change`函数将字符转小写后，用switch处理O、l、i。`tolower`确保大小写统一，switch结构直观。例如，字符'O'转小写后是'o'，再被替换为'0'，避免了大写O漏处理的问题。
* 💡 **学习笔记**：switch语句适合处理多条件分支，代码更易读。

**题解二：作者hensier**
* **亮点**：转换函数简洁，主函数直接比较。
* **核心代码片段**：
    ```cpp
    string change(string s) {
        for(int i=0;s[i];i++) {
            s[i]=tolower(s[i]);
            if(s[i]=='o')s[i]='0';
            if(s[i]=='l'||s[i]=='i')s[i]='1';
        }
        return s;
    }
    ```
* **代码解读**：`change`函数通过遍历每个字符，先转小写，再处理O、l、i。循环条件`s[i]`利用了字符串以'\0'结尾的特性，简洁高效。例如，字符'I'转小写后是'i'，被替换为'1'，符合题目要求。
* 💡 **学习笔记**：字符串遍历可以用`for(int i=0; s[i]; i++)`简化，避免计算长度。

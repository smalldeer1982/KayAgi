# Vladik and flights

## 题目描述

Vladik is a competitive programmer. This year he is going to win the International Olympiad in Informatics. But it is not as easy as it sounds: the question Vladik face now is to find the cheapest way to get to the olympiad.

Vladik knows $ n $ airports. All the airports are located on a straight line. Each airport has unique id from $ 1 $ to $ n $ , Vladik's house is situated next to the airport with id $ a $ , and the place of the olympiad is situated next to the airport with id $ b $ . It is possible that Vladik's house and the place of the olympiad are located near the same airport.

To get to the olympiad, Vladik can fly between any pair of airports any number of times, but he has to start his route at the airport $ a $ and finish it at the airport $ b $ .

Each airport belongs to one of two companies. The cost of flight from the airport $ i $ to the airport $ j $ is zero if both airports belong to the same company, and $ |i-j| $ if they belong to different companies.

Print the minimum cost Vladik has to pay to get to the olympiad.

## 说明/提示

In the first example Vladik can fly to the airport $ 2 $ at first and pay $ |1-2|=1 $ (because the airports belong to different companies), and then fly from the airport $ 2 $ to the airport $ 4 $ for free (because the airports belong to the same company). So the cost of the whole flight is equal to $ 1 $ . It's impossible to get to the olympiad for free, so the answer is equal to $ 1 $ .

In the second example Vladik can fly directly from the airport $ 5 $ to the airport $ 2 $ , because they belong to the same company.

## 样例 #1

### 输入

```
4 1 4
1010
```

### 输出

```
1```

## 样例 #2

### 输入

```
5 5 2
10110
```

### 输出

```
0```

🗣️ **初步分析**：  
解决“Vladik and flights”的关键，在于**将复杂的路径规划问题转化为简单的字符比较问题**。想象一下，机场像一排彩色的积木，每个积木涂成红色（0公司）或蓝色（1公司）。Vladik要从积木`a`走到积木`b`，规则是：同色积木间移动免费，不同色则要付“步数差”的费用。但其实，**只要`a`和`b`同色，直接走过去就行（费用0）；如果不同色，总能找到一个中间积木，先走到同色的相邻积木（费用1），再免费走到`b`**。比如样例1中，`a=1`（红）和`b=4`（蓝）不同色，但`2`号积木是蓝，所以`1→2`（费用1），`2→4`（免费），总费用1。  

**核心算法流程**：  
1. 读取输入的机场数量`n`、起点`a`、终点`b`，以及每个机场的公司（字符串或数组）。  
2. 直接比较`a`和`b`位置的公司是否相同（注意题目中`a`和`b`是1-based索引，代码中需要调整为0-based）。  
3. 输出结果：相同则0，不同则1。

## 3. 核心难点辨析与解题策略

在解决本题时，新手常遇到以下3个关键点：  

### 1. **难点1：如何理解“最小费用”的逻辑？**  
* **分析**：很多同学会误以为需要计算路径的最短步数，但实际上，**不同公司的最小费用必然是1**。因为只有两个公司，`a`所在的公司的机场和`b`所在的公司的机场之间必然有相邻的（比如`a`旁边的机场如果是同公司，就继续找，直到找到不同公司的）。所以，只要`a`和`b`不同公司，费用就是1。  
* 💡 **学习笔记**：问题的本质往往隐藏在“规则”背后，学会用逻辑推理简化问题。  

### 2. **难点2：输入的索引处理（1-based vs 0-based）**  
* **分析**：题目中`a`和`b`是1-based（比如第一个机场是1），而C++中的字符串和数组是0-based（第一个元素是0）。所以需要将`a`和`b`减1，才能正确访问对应的字符。  
* 💡 **学习笔记**：输入的索引问题是常见的bug来源，一定要注意题目中的描述。  

### 3. **难点3：如何高效读取输入？**  
* **分析**：当`n`很大时（比如1e5），`cin`读取字符会比较慢，此时可以用`getchar()`或者快读函数。比如题解二中的`getchar()`读取字符，题解三中的跳过不需要的字符，都是高效的输入方式。  
* 💡 **学习笔记**：输入优化是竞赛中的重要技巧，能避免超时。  


### ✨ 解题技巧总结  
- **问题转化**：将复杂的路径问题转化为简单的字符比较，这是解决本题的关键。  
- **索引处理**：注意题目中的1-based索引，转换为0-based。  
- **输入优化**：用`getchar()`或快读函数处理大数据量的输入。  


## 4. C++核心代码实现赏析

### 本题通用核心C++实现参考  
* **说明**：本代码综合了题解一的简洁性和题解二的输入优化，适合大多数情况。  
* **完整核心代码**：  
  ```cpp
  #include <iostream>
  #include <string>
  using namespace std;

  int main() {
      int n, a, b;
      string s;
      cin >> n >> a >> b >> s; // 读取输入（s是0-based）
      if (s[a-1] == s[b-1]) { // 比较a和b位置的字符（转换为0-based）
          cout << 0 << endl;
      } else {
          cout << 1 << endl;
      }
      return 0;
  }
  ```  
* **代码解读概要**：  
  代码分为三步：① 读取输入；② 比较`a`和`b`位置的字符（注意减1）；③ 输出结果。逻辑清晰，适合新手入门。  


### 针对各优质题解的片段赏析  

**题解一：来源：piggymichael**  
* **亮点**：用`string`直接存储机场公司，代码简洁。  
* **核心代码片段**：  
  ```cpp
  string a;
  cin >> x >> y >> z >> a;
  if (a[y-1] == a[z-1]) cout << 0 << endl;
  else cout << 1 << endl;
  ```  
* **代码解读**：  
  `a`是存储机场公司的字符串（0-based）。`y-1`和`z-1`是转换后的索引，直接比较这两个位置的字符。这种方法最简洁，适合小数据量的情况。  
* 💡 **学习笔记**：`string`是处理字符序列的方便工具，适合需要直接访问字符的场景。  

**题解二：来源：wxwoo**  
* **亮点**：用`getchar()`读取字符，输入效率高。  
* **核心代码片段**：  
  ```cpp
  int f[100010];
  for (int i=1; i<=n; i++) {
      char c = getchar();
      f[i] = c - 48; // 将字符转换为整数（0或1）
  }
  ```  
* **代码解读**：`getchar()`逐个读取字符，`c-48`将字符`'0'`或`'1'`转换为整数0或1。这种方法适合大数据量的情况，因为`getchar()`比`cin`快。  
* 💡 **学习笔记**：`getchar()`是处理字符输入的高效方式，尤其适合竞赛中的大数据。  

**题解三：来源：dingcx**  
* **亮点**：不使用数组，节省内存。  
* **核心代码片段**：  
  ```cpp
  if (a > b) swap(a, b); // 保证a <= b
  for (int i=1; i<=a; i++) scanf("%c", &c1); // 读取a位置的字符
  for (int i=a+1; i<=b; i++) scanf("%c", &c2); // 读取b位置的字符
  printf("%d", !(c1 == c2)); // 相等则0，不等则1
  ```  
* **代码解读**：通过`for`循环跳过不需要的字符，只保留`a`和`b`对应的字符`c1`和`c2`。`!(c1==c2)`是一个技巧：如果`c1`等于`c2`，则`c1==c2`为`true`，`!`后为`false`（0）；否则为`true`（1）。  
* 💡 **学习笔记**：当不需要存储所有数据时，可以跳过无关数据，节省内存。

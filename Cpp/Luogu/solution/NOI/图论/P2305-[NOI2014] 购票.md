# [NOI2014] 购票

## 题目描述

今年夏天，NOI 在 SZ 市迎来了她三十周岁的生日。来自全国 $n$ 个城市的 OIer 们都会从各地出发，到 SZ 市参加这次盛会。

全国的城市构成了一棵以 SZ 市为根的有根树，每个城市与它的父亲用道路连接。为了方便起见，我们将全国的 $n$ 个城市用 $1\sim n$ 的整数编号。其中 SZ 市的编号为 $1$。对于除 SZ 市之外的任意一个城市 $v$，我们给出了它在这棵树上的父亲城市 $f_v$  以及到父亲城市道路的长度 $s_v$。

从城市 $v$ 前往 SZ 市的方法为：选择城市 $v$ 的一个祖先 $a$，支付购票的费用，乘坐交通工具到达 $a$。再选择城市 $a$ 的一个祖先 $b$，支付费用并到达 $b$。以此类推，直至到达 SZ 市。

对于任意一个城市 $v$，我们会给出一个交通工具的距离限制 $l_v$。对于城市 $v$ 的祖先 A，只有当它们之间所有道路的总长度不超过 $l_v$  时，从城市 $v$ 才可以通过一次购票到达城市 A，否则不能通过一次购票到达。  

对于每个城市 $v$，我们还会给出两个非负整数 $p_v,q_v$  作为票价参数。若城市 $v$ 到城市 A 所有道路的总长度为 $d$，那么从城市 $v$ 到城市 A 购买的票价为 $dp_v+q_v$。

每个城市的 OIer 都希望自己到达 SZ 市时，用于购票的总资金最少。你的任务就是，告诉每个城市的 OIer 他们所花的最少资金是多少。

## 说明/提示

从每个城市出发到达 SZ 的路线如下（其中箭头表示一次直达）：

城市 $2$：只能选择 $2 \rightarrow 1$，花费为 $2 \times 20 + 0 = 40$。

城市 $3$：只能选择 $3 \rightarrow 1$，花费为 $5 \times 10 + 100 = 150$。

城 市 $4$ ： 由于 $4 + 2 =6 \leq l_4 = 10$，故可以选择 $4\rightarrow1$。若选择 $4 \rightarrow 1$，花费为 $(4 +2) \times 10 + 10 = 70$ ； 若选择 $4 \rightarrow 2 \rightarrow 1$，则花费为 $(4\times 10 + 10) + (2 \times  20 + 0) =90$；因此选择 $4 \rightarrow 1$。

城市 $5$：只能选择 $5\rightarrow 2\rightarrow 1$，花费为 $(9 \times 1 +100) + (2 \times  20 + 0) = 149$；无法选择 $5 \rightarrow 1$，因为 $l_5 =10$，而城市 $5$ 到城市 $1$ 总路程为 $9 + 2 = 11 \gt 5$，城市 $5$ 不能直达城市 $1$。

城市 $6$：若选择 $6 \rightarrow 1$，花费为 $(5 + 5) \times 20 + 100 = 300$；若选择 $6 \rightarrow 3 \rightarrow 1$，花费为 $(5 \times 20 + 100) + (5 \times 10 + 100) = 350$；因此选择 $6 \rightarrow 1$。

城市 $7$：选择 $7 \rightarrow 4 \rightarrow 1$，花费为 $(4 \times 20 + 0) + ((4 + 2) \times 10 + 10) = 150$；

其他方案均比该方案差。

### 数据范围

对于所有数据，$n\leq 2 \times 10^5, 0 \leq p_v \leq 10^6,\ 0 \leq q_v \leq 10^{12},\ 1\leq f_v<v,\ 0<s_v\leq l_v \leq 2 \times 10^{11}$，且任意城市到 SZ 市的总路程长度不超过 $2 \times 10^{11}$。

- 当 $t=0$ 或 $2$ 时，对输入的所有城市 $v$，都有 $f_v=v-1$，即所有城市构成一个以 SZ 市为终点的链；

# 题解

## 作者：panyf (赞：89)

新做法：线段树套李超树+出栈序。

$O(n^2)$ 的 dp 方程：$f_i=\min(f_j+(d_i-d_j)p_i+q_i)$，其中 $j$ 为 $i$ 的祖先，且 $d_i-d_j\leq l_i$。初始值 $f_1=0$。

式子拆开：$f_i=d_ip_i+q_i+\min(-d_jp_i+f_j)$。

设 $y=f_i-d_ip_i-q_i$，$k=-d_j$，$x=p_i$，$b=f_j$，则有 $y=\min(kx+b)$。

数据类型 $t=0$ 时，显然是李超树模板题。

$t=1$ 时，边 dfs 边处理，当离开一个结点时需要撤销对应的直线。和维护凸壳相比，李超树的优势在于方便撤销（可持久化或者用栈记录修改操作）。时空复杂度均为 $O(n\log n)$。

$t=2$ 时，有了 $l$ 的限制，需要支持区间查询，考虑树套树，每次在 $\log n$ 棵李超树上修改和查询。时间复杂度 $O(n\log^2n)$，空间 $O(n\log n)$。

为什么空间复杂度不是 $O(n\log^2n)$？李超树与一般的线段树不同，信息可以在非叶子结点保存，因此动态开点的李超树空间复杂度可以做到 $O(n)$。具体方法是每次插入直线时遇到空结点就 return，这样每次只会新建一个结点。总的空间复杂度即为 $O(n\log n)$。

$t=3$ 时，直接结合 $t=1$ 和 $2$ 的做法，线段树套可撤销李超树，时空复杂度均为 $O(n\log^2n)$，卡空间后可以通过。

有没有不需要卡空间的做法？空间瓶颈是李超树的撤销。注意到要做的操作是树链查询最小值， 可以树链剖分套线段树套李超树，就不需要撤销了，空间复杂度 $O(\log n)$，时间复杂度 $O(n\log^3n)$，但常数很小，可以通过（尝试构造了卡树剖的数据效果并不理想，而且这个做法在 uoj 也可以通过）。

更好的做法是利用**出栈序**，也就是每个结点离开 dfs 的顺序。直接在点 $i$ 及其最远的满足 $d_i-d_j\leq l_i$ 的祖先 $j$ 的出栈序对应的区间上查询。容易发现区间内不在 $i$ 和 $j$ 的路径上的点都未被访问到，不会对答案产生贡献。这样就不需要树链剖分了。时间复杂度 $O(n\log^2n)$，空间复杂度 $O(n\log n)$。

upd：李超树的优势在于方便撤销~~以及码量小~~，而最终的做法并没有用到撤销。因此其他题解的树剖+线段树+凸壳上二分的做法也可以用类似的方式优化到 $O(n\log^2n)$。
```cpp
#include<bits/stdc++.h>
using namespace std;
#define ll long long
const int N=2e5+3,M=4e6+3;
int rt[N*3],n,he[N],ne[N],p[N],o[N],t[M],lc[M],rc[M],c[N],id,u,v,w,m;
ll s[N],q[N],l[N],d[N],f[N],a[N],b[N];
#define g(t,x) (a[t]*(x)+b[t])
void tupd(int&k,int l,int r){//李超树插入直线（动态开点）
	if(k){
		int m=l+r>>1;
		if(g(w,m)<g(t[k],m))swap(w,t[k]);
		if(l<r)a[w]<a[t[k]]?tupd(rc[k],m+1,r):tupd(lc[k],l,m);
	}else t[k=++id]=w;
}
ll tqry(int k,int l,int r){//李超树查询最小值
	if(!k)return 1e18;
	int m=l+r>>1;
	return min(g(t[k],w),w>m?tqry(rc[k],m+1,r):tqry(lc[k],l,m));
}
void upd(int k,int l,int r){//外层线段树单点修改
	w=v,tupd(rt[k],0,1e6);
	if(l==r)return;
	int m=l+r>>1;
	u>m?upd(k*2+1,m+1,r):upd(k*2,l,m);
}
ll qry(int k,int l,int r){//外层线段树区间查询
	if(u<=l&&r<=v)return tqry(rt[k],0,1e6);
	int m=l+r>>1;
	return u>m?qry(k*2+1,m+1,r):(m<v?min(qry(k*2,l,m),qry(k*2+1,m+1,r)):qry(k*2,l,m));
}
void pre(int x){//预处理出栈序
	for(int i=he[x];i;i=ne[i])pre(i);
	o[x]=++id;
}
void dfs(int x){
	a[x]=-d[m],b[x]=f[x],u=o[v=c[m]=x],upd(1,1,n);
	for(int i=he[x];i;i=ne[i]){
		++m,d[m]=d[m-1]+s[i],u=o[i],v=o[c[lower_bound(d,d+m,d[m]-l[i])-d]];
		w=p[i],f[i]=qry(1,1,n)+d[m]*w+q[i],dfs(i),--m;
	}
}
int main(){
	int i,j;
	scanf("%d%*d",&n);
	for(i=2;i<=n;++i)scanf("%d%lld%d%lld%lld",&j,s+i,p+i,q+i,l+i),ne[i]=he[j],he[j]=i;
	pre(1),id=0,dfs(1);
	for(i=2;i<=n;++i)printf("%lld\n",f[i]);
	return 0;
}
```

### 题意简述：

有一棵以 $1$ 号节点为根节点的带边权的树。

除了 $1$ 号节点的所有节点上都有人需要坐车到达 $1$ 号节点。

除了 $1$ 号节点，每个节点都有 $5$ 个参数 $f_u,s_u,p_u,q_u,l_u$。

$f_u$ 表示 $u$ 号点的父亲，$s_u$ 表示 $u$ 号点与父亲之间的边的权值，$p,q,l$ 为车票参数。

定义两个节点 $u$ 和 $v$ 之间的距离 $dis_{u,v}$ 为 $u$ 和 $v$ 在树上的简单路径上的边权之和。

从一个节点 $u$，可以坐车到达节点 $v$ 当且仅当 $dis_{u,v}\le l_u$，且需要花费 $dis_{u,v}\times p_u+q_u$ 的代价。

问除了 $1$ 号节点的每个节点坐车到 $1$ 号节点需要花费的最小总代价。

$1\le n\le 2\times 10^5$，保证输入的所有数都是非负整数，$l_u\ge s_u$，保证答案在 `long long` 范围内。

### 题解：

考虑树形 DP，令 $\mathrm{f}[u]$ 为 $u$ 到 $1$ 的最小总代价，$\mathrm{dis}[u]$ 为 $1$ 号节点到 $u$ 号节点的距离。

则有 $\mathrm{f}[u]=\min\limits_{v\in anc_u\:\&\:\mathrm{dis}[u]-\mathrm{dis}[v]\le l_u}(\mathrm{f}[v]+(\mathrm{dis}[u]-\mathrm{dis}[v])\times p_u+q_u)$。

令 $b_u=\mathrm{dis}[u]\times p_u+q_u$，则有：

$$\mathrm{f}[u]=b_u+\min\limits_{v\in anc_u\:\&\:\mathrm{dis}[u]-\mathrm{dis}[v]\le l_u}(\mathrm{f}[v]-\mathrm{dis}[v]\times p_u)$$

观察式子，发现 $v$ 必须是紧接着 $u$ 往上一段连续的祖先，而转移的方式是简单的加权值取 $\min$。

这提示我们考虑斜率优化，考虑两个合法转移点 $j$ 和 $k$，比较 $j$ 和 $k$ 转移的优劣：

$$\begin{aligned}\mathrm{f}[j]-\mathrm{dis}[j]\times p_u&\Leftrightarrow\mathrm{f}[k]-\mathrm{dis}[k]\times p_u\\\mathrm{f}[k]-\mathrm{f}[j]&\Leftrightarrow(\mathrm{dis}[k]-\mathrm{dis}[j])\times p_u\end{aligned}$$

发现已经有了斜率的表达式，$x$ 坐标是 $\mathrm{dis}$，$y$ 坐标是 $\mathrm{f}$。假设 $\mathrm{dis}[j]<\mathrm{dis}[k]$，则决策 $j$ 比 $k$ 优当且仅当：

$$\begin{aligned}\mathrm{f}[j]-\mathrm{dis}[j]\times p_u&<\mathrm{f}[k]-\mathrm{dis}[k]\times p_u\\\mathrm{f}[k]-\mathrm{f}[j]&>(\mathrm{dis}[k]-\mathrm{dis}[j])\times p_u\\\frac{\mathrm{f}[k]-\mathrm{f}[j]}{\mathrm{dis}[k]-\mathrm{dis}[j]}&>p_u\end{aligned}$$

即点 $(\mathrm{dis}[j],\mathrm{f}[j])$ 和点 $(\mathrm{dis}[k],\mathrm{f}[k])$ 之间的线段的斜率大于 $p_u$。

所以我们需要用单调栈维护一个合法点的下凸壳，这样可以根据不同的 $p_u$ 在凸壳上二分得到决策点。

但是这样有一个问题，我们知道合法的决策点是 $u$ 的近几层祖先，即 $u$ 到祖先的链上 $\mathrm{dis}$ 较大的节点，它们对应的点也是靠右的。

![](https://i.loli.net/2019/03/04/5c7cf62878f64.bmp)

考虑这张图，假设 A 点是非法转移点，而 B、C、D 都是合法转移点，此时维护的凸壳并不包含 B，然而 B 可能是最优决策点。

也就是说，我们需要考虑一段后缀中的最优转移点，然而我们又不能维护所有后缀的凸壳，应该怎么办呢？

有一个解决方法：维护一些区间中的凸壳，查询时选择一些区间的并恰好等于需要的后缀，在每个区间内分别查询最优决策并更新。

这提示我们使用树状数组维护后缀信息，也就是树状数组套单调栈。  
这样花费的空间是 $\Theta(n\log n)$ 的，修改时间 $\mathcal{O}(\log n)$，查询时间 $\mathcal{O}(\log^2 n)$。

还有最后一个问题，需要处理的不是序列上而是树上的转移，这并不难实现，我们维护可撤销单调栈，在回溯的时候撤销这次操作即可。

关于可撤销单调栈，考虑单调栈的每一次操作只会更改栈顶指针以及栈顶位置的值，我们维护历史每一次操作原先的栈顶指针以及栈顶位置原来的值即可实现撤销操作，这样插入和撤销均是 $\Theta(1)$ 的。

```cpp
#include <cstdio>
#include <vector>
#include <algorithm>

typedef long long LL;
const int MN = 200005;
const int MS = 2100005;
const LL Inf = 0x7fffffffffffffff;

int N, faz[MN];
std::vector<int> G[MN];
LL wgh[MN], dep[MN], dis[MN], p[MN], q[MN], b[MN], len[MN];
LL stds[MN], tds;
LL f[MN], *X = dis, *Y = f;
inline double Slope(int i, int j) {
	return X[i] == X[j] ? 1e60 : (double)(Y[j] - Y[i]) / (X[j] - X[i]);
}

int stkp[MS], valp[MS], tpp[MS];
int *istk[MN], *ival[MN], *itp[MN], it[MN];
inline void Push(int id, int u) {
	int t = it[id], tp = itp[id][t], *stk = istk[id];
	while (tp > 0 && Slope(stk[tp - 1], stk[tp]) >= Slope(stk[tp], u)) --tp;
	++t, ival[id][t] = stk[itp[id][t] = ++tp], stk[tp] = u, it[id] = t;
}
inline void Pop(int id) {
	istk[id][itp[id][it[id]]] = ival[id][it[id]];
	--it[id];
}
inline int chk(int id, LL slp) {
	int t = it[id], tp = itp[id][t], *stk = istk[id];
	if (!~tp) return -1;
	int lb = 0, rb = tp - 1, x = tp, mid;
	while (lb <= rb) {
		mid = (lb + rb) >> 1;
		if (Slope(stk[mid], stk[mid + 1]) <= slp) lb = mid + 1;
		else x = mid, rb = mid - 1;
	}
	return stk[x];
}
inline void PushAll(int i, int u) { for (; i <= N; i += i & -i) Push(i, u); }
inline void PopAll(int i) { for (; i <= N; i += i & -i) Pop(i); }
inline void GetDp(int u) {
	f[u] = Inf;
	int i = std::lower_bound(stds + 1, stds + tds + 1, dis[u] - len[u]) - stds;
	for (i = N - i + 1; i; i -= i & -i) {
		int v = chk(i, p[u]);
		if (~v) f[u] = std::min(f[u], f[v] - dis[v] * p[u] + b[u]);
	}
}

void DFS(int u) {
	dep[u] = dep[faz[u]] + 1;
	dis[u] = dis[faz[u]] + wgh[u];
	stds[++tds] = dis[u];
	b[u] = dis[u] * p[u] + q[u];
	GetDp(u);
	PushAll(N - dep[u], u);
	for (auto v : G[u]) DFS(v);
	PopAll(N - dep[u]);
	--tds;
}

int main() {
	scanf("%d%*d", &N);
	istk[1] = stkp, itp[1] = tpp, ival[1] = valp, itp[1][it[1] = 0] = -1;
	for (int i = 2; i <= N; ++i) {
		int lenl = ((i - 1) & (1 - i)) + 1;
		istk[i] = istk[i - 1] + lenl;
		itp[i] = itp[i - 1] + lenl;
		ival[i] = ival[i - 1] + lenl;
		itp[i][it[i] = 0] = -1;
	}
	for (int i = 2; i <= N; ++i) {
		scanf("%d%lld%lld%lld%lld", &faz[i], &wgh[i], &p[i], &q[i], &len[i]);
		G[faz[i]].push_back(i);
	}
	PushAll(N, 1), stds[tds = 1] = 0;
	for (auto u : G[1]) DFS(u);
	for (int i = 2; i <= N; ++i) printf("%lld\n", f[i]);
	return 0;
}
```

关于此题的 dp 部分，应该是非常简单的，并且已经被讲烂了，这里不再赘述。那么根据深度的单调性我们显然可以用 cdq 分治加单调栈来优化序列上的问题。

对于树上的情况，能够对一个点产生贡献的点是它的一个区间内的祖先。关于如何把序列的做法拓展到树上，我目前已知四种做法：
## 点分治
有根树点分是一个非常神奇的技巧。它的主要步骤大致为：

对于一个有根树首先找到重心，然后递归处理重心上方的那个子树，再计算重心到当前根这条路径对以重心为根的子树的贡献，再。这样就可以保证每个贡献都可以被计算到，并且计算一个点的贡献时它的 dp 值一定已经计算好。实现时需要注意一些边界问题。

一次分治的计算贡献部分是 $O(n\log n)$ 时间，根据主定理，总时间复杂度为 $O(n\log^2n)$。
## 树剖
点分，边分，链分三种分治没有本质区别，因此能够点分的题理论上必可以树剖。

我们很容易想到用树剖化树为链，然后直接套用序列的做法。但是序列的做法时间复杂度已经为 $O(n\log^2n)$，直接上树剖复杂度会达到 3log。具体来讲，我们需要做若干次链上的 cdq 分治，共计 $O(n)$ 个元素和 $O(n\log n)$ 次询问。虽然因为数据卡不满而可以通过，但我们不能满足于此。

实际上，我曾经发现过一个常用的树剖 trick，可以用于很多树剖题尤其是优化建图的砍 log，但至今只在一篇博客里看到过这个 trick。考虑一下我们用树剖对一个路径进行剖分的过程，我们会发现它实际上被剖分成了若干条重链的前缀和一条重链的一个可能不是前缀的区间。

那么，对于这些前缀，我们可以拎出来单独处理，只需对每条重链做一次扫描线，维护单调栈并二分即可。前缀的询问有 $O(n\log n)$ 个，每次需 $O(\log n)$ 时间。而 cdq 分治的区间询问降到了 $O(n)$ 个，总时间复杂度降到了 $O(n\log^2n)$。

实现方面，可以使用最直观的方式，先一次预处理把每个点挂到它的祖先上面去做扫描线和分治。当然也可以在处理到链时计算这条链对子树的贡献，这样就写成了为人熟知的静态链分即 dsu on tree 的形式。
## 树状数组
考虑使用树状数组维护单调栈。那么，我们每次修改的复杂度是均摊 $O(\log n)$，查询是 $O(\log^2n)$。但是这里是树上的问题，我们需要支持撤销操作。

每次对栈的修改，我们可以花费 $O(\log n)$ 时间二分出要弹到哪里，这样复杂度就不依赖于均摊了。然后我们考虑把单调栈可撤销化。每次修改只改变了栈顶位置和栈顶的一个元素，那么我们再开一个栈记录每次改变前的栈顶元素，撤销时直接改回去就行了。

如果使用线段树套单调栈，本质是相同的。

------------
以上三种解法，虽然看似不同，但核心思想是一致的：即是把树上的问题剖分到序列上，把树上的路径剖分成链。至于三种不同的做法，只是用不同的形式来保证复杂度而已。但树状数组套单调栈的解法与两种分治又有所不同，两种分治把问题化为了 $O(n)$ 条互不相交链上的问题，可以静态解决，但树状数组在处理不同点时剖分出的链是可能相交的，是一个动态的问题。

那么，除此之外，有没有从另一条路径解决问题的做法呢？
## 出栈序
直接给结论：进行一次 dfs，在每个点被访问完时记录时间戳。dp 时按这个顺序进行 dfs，如果当前点为 $u$，对它产生贡献的最浅的祖先为 $v$，那么直接在 $u$ 和 $v$ 的时间戳之间进行查询即可。可以发现这样不会涉及到没有被计算过的点。

那么我们还是需要将序列上的做法动态化，使用线段树套单调栈即可。

**upd**：插入的顺序不单调，所以不太能单调栈，但是可以李超树。

这个做法的思想就和上面不一样了。它不是将路径剖成若干链，而是直接找到一种方法，把树拍到序列上去而不影响结果。这种方法非常巧妙，在许多只涉及从上往下的路径的题可以用到。

DP方程显然：$f_i=min(f_j+(dis_i-dis_j)*p_i+q_i)$，dis是根到每个点的距离。

假设j的深度比k大且j这个决策比k更优，那么就有：

$$f_j+d_ip_i-d_jp_i+q_i<f_k+d_ip_i-d_kp_i+q_i$$

$$\frac{f_j-f_k}{d_j-d_k}<p_i$$

记这个斜率式为$slop(j,k)$，如果存在i的深度大于j，j的深度大于k且$slop(i,j)<slop(j,k)$，那么j一定不是最优决策。（因为如果$p_i>slop(j,k)$，则i比较优，否则k比较优），所以我们要维护一个上凸壳。

考虑用**点分治**来实现树上斜率优化，假设当前在处理一个以now为根的子树。

1.找到重心x。

2.递归处理包含now的那个连通块带上x

3.将子树x中，除x以外的点，按照其一趟车可以到达的最小深度，从大到小排序。

4.依次处理这些点，每次把新的可以一趟车到达的点加入凸壳中，单调栈维护一下。然后二分答案寻找当前这个点的最优决策。

5.递归处理x的子树。

复杂度是$O(nlog^2n)$的。

## 作者：Kinandra (赞：14)



标签: CDQ 分治, 点分治, 斜率优化.

#### Part 1

先考虑链上无距离限制的做法, 设 $f_u$ 表示 $u$ 点的答案, 容易得到转移式: $f_u=\min_v\{f_v+(dis_u-dis_v)\times p_u+q_u\}$, 其中 $dis_u$ 表示 $u$ 到根节点的距离.

对转移式稍加变形, 容易得到一个可以斜率优化的形式: $f_u=\min_v\{f_v-dis_v\times p_u+dis_u\times p_u+q_u\}$ , 单调栈维护, 从根向下求答案.

### Part 2

加上距离限制, 仍然考虑链的做法, 注意到可以转移到某个点的范围(即能到达的深度最低的祖先)并不具有单调性, 需要想办法来处理一下(显然这里直接在凸包上二分是错误的).

一个做法是使用 CDQ 分治, 先递归处理上半部分(深度浅的一半)的答案, 将下半部分(深度深的一半)按 **能到达的深度最低的祖先的深度** 从大到小排序(即范围由小到大)依次处理, 每次把可以查询到的位置的所有点加入单调栈, 在凸包上二分找最优位置就好了.

### Part 3

`出毒瘤题的最好方法就是把序列(链)上的问题放到树上, 把树上的问题放到仙人掌上` 

引用这句话并不是我想说这道题毒瘤, 而是想表达树上问题确实通常需要我们更深入的思考, 这道题可以很容易地得到一个十分自然的想法: 链上按区间分治, 树上就上点分治.

事实上点分治的想法是正确的, 处理一个以 $u$ 为根的子树时, 我们先找到子树的重心 $root$ , 处理以 $root$ 为根的子树以外的部分(可以认为是 $root$ 以上的部分), 然后处理 $root$ 所有父亲对 $root$ 子树内所有点的贡献就好了.

时间复杂度 $\mathcal O(n\log^2n)$, 其中点分治一个 $\log$, 在凸包上二分一个 $\log$ .

代码实现挺简单的.

```cpp

#include <bits/stdc++.h>
using namespace std;
long long read();

int n, num;
int fa[200005];
long long s[200005], x[200005], y[200005], l[200005];

vector<int> e[200005];
void add(int f, int t) { e[f].push_back(t); }

int root, tot;
int vis[200005], sz[200005], mx[200005];
void dfs1(int u) {
    sz[u] = 1, mx[u] = 0;
    for (int i = e[u].size() - 1, v; i >= 0; --i)
        if (!vis[v = e[u][i]])
            dfs1(v), sz[u] += sz[v], mx[u] = max(mx[u], sz[v]);
    if (root < 0 || max(mx[u], tot - sz[u]) < max(mx[root], tot - sz[root]))
        root = u;
    return;
}

struct P {
    long long x, y;
    P operator-(P a) { return (P){x - a.x, y - a.y}; }
} st1[200005], st2[200005];
int top1, top2;

long long d[200005];
void dfs2(int u) {
    if (l[u] - d[u] > 0) st1[++top1] = (P){l[u] - d[u], u};
    for (int i = e[u].size() - 1, v; i >= 0; --i)
        if (!vis[v = e[u][i]]) d[v] = d[u] + s[v], dfs2(v);
}

bool cmp(P a, P b) { return a.x < b.x; }

long long f[200005];

double slope(P a, P b) { return (double)(a.y - b.y) / (a.x - b.x); }

double sl[200005];
void ins(P a) {
    while (top2 > 1 && slope(a, st2[top2]) <= sl[top2]) --top2;
    st2[++top2] = a;
    sl[top2] = top2 > 1 ? slope(st2[top2], st2[top2 - 1]) : -1e18;
}

long long qry(long long k) {
    int l = 1, r = top2;
    long long rt = 0;
    while (l <= r) {
        int mid = l + r >> 1;
        if (sl[mid] <= k) {
            rt = st2[mid].y - st2[mid].x * k, l = mid + 1;
        } else
            r = mid - 1;
    }
    return rt;
}

void work(int u) {
    root = -1, dfs1(u), vis[root] = 1;
    int trt = root;
    if (trt != u) tot = tot - sz[trt], work(u);
    int v = trt;
    long long dis = 0;

    top1 = top2 = d[trt] = 0, dfs2(trt);
    sort(st1 + 1, st1 + 1 + top1, cmp);
    for (int i = 1; i <= top1; ++i) {
        long long lim = st1[i].x;
        int w = st1[i].y;
        while (v != fa[u] && dis + s[v] <= lim && fa[v])
            ins((P){(dis += s[v]), f[fa[v]]}), v = fa[v];
        if (top2) f[w] = min(f[w], qry(-x[w]) + d[w] * x[w] + y[w]);
    }

    for (int i = e[trt].size() - 1; i >= 0; --i)
        if (!vis[v = e[trt][i]]) tot = sz[v], work(v);
}

int main() {
    n = read(), num = read();
    for (int i = 2; i <= n; ++i) {
        fa[i] = read(), s[i] = read(), x[i] = read(), y[i] = read();
        l[i] = read(), add(fa[i], i), f[i] = 1000000000000000000ll;
    }

    tot = n, vis[0] = 1, work(1);

    for (int i = 2; i <= n; ++i) printf("%lld\n", f[i]);
    return 0;
}

long long read() {
    long long x = 0;
    char c = getchar();
    while (c < '0' || c > '9') c = getchar();
    while (c >= '0' && c <= '9') x = x * 10 + c - '0', c = getchar();
    return x;
}
```

```
	}

	if (maxn[x] < maxn[root]) root = x;
}

bool cmp(LL x, LL y) {
	return dis[x] - l[x] > dis[y] - l[y];
}

void dfs(LL x) {
	sta[++top] = x;
	for (LL i = head[x]; i; i = e[i].next) {
		LL y = e[i].v;
		if (!vis[y]) dfs(y);
	}
}

inline LL getAns(LL x, point poi) {
	return poi.y + (dis[x] - poi.x) * p[x] + q[x];
}

double getSlope(point poi) {
	return (double) poi.y / poi.x;
}

void solveQuery(LL x) {
	LL ans = INF;
	LL l = 1, r = cntq;
	while (l + 2 < r) {
		LL mid1 = l + (r - l) / 3;
		LL mid2 = mid1 + (r - mid1) / 2;
		LL ans1 = getAns(x, pnt[mid1]);
		LL ans2 = getAns(x, pnt[mid2]);
		if (ans1 < ans2) r = mid2;
		else l = mid1;
	}
	for (LL i = l; i <= r; ++i)
		ans = std::min(ans, getAns(x, pnt[i]));
	d[x] = std::min(d[x], ans);
}

void solve(LL rt) {
	root = 0; getRoot(rt);
	LL x = root; vis[x] = 1;
	if (x != rt) {
		sum = siz[rt] - siz[x];
		solve(rt);
		top = 0; dfs(x);
		std::sort(sta + 1, sta + top + 1, cmp);
		LL now = fa[x], j = 1;
		cntq = 0;
		pnt[++cntq] = (point) {dis[now], d[now]};
		while (j <= top && dis[sta[j]] - dis[now] > l[sta[j]]) ++j;
		for (LL i = j; i <= top; ++i) {
			while (now != rt && dis[sta[i]] - dis[fa[now]] <= l[sta[i]]) {
				now = fa[now];
				point tp = (point) {dis[now], d[now]};
				while (cntq > 1 && getSlope(pnt[cntq]-pnt[cntq - 1]) < getSlope(tp - pnt[cntq])) --cntq;
				pnt[++cntq] = tp;
			}
			solveQuery(sta[i]);
		}
 	}
 	else top = 0, dfs(x);
 	for (LL i = 1; i <= top; ++i)
 		if (dis[sta[i]] - dis[x] <= l[sta[i]]) 
 			d[sta[i]] = std::min(d[sta[i]], getAns(sta[i], (point) {dis[x], d[x]}));
 	for (LL i = head[x]; i; i = e[i].next) {
 		LL y = e[i].v;
 		if (!vis[y]) sum = siz[y], solve(y);
	 }	
}

int main() {
	scanf("%d %d", &n, &t);
	memset(d, 0x3f, sizeof d);
	d[1] = 0;
	LL w;
	for (LL i = 2; i <= n; ++i) {
		scanf("%d %lld %lld %lld %lld", &fa[i], &w, &p[i], &q[i], &l[i]);
		addEdge(fa[i], i, w);
	}
	getDistance(1);
	maxn[0] = 0x7fffffff, sum = n, root = 0;
	solve(1);
	for (LL i = 2; i <= n; ++i) printf("%lld\n", d[i]);
	return 0;
}
```

## 作者：Sol1 (赞：8)

新做法：线段树套单调栈 + DFS 序。

首先 DP 方程是显然的：设 $dp_i$ 为 $i\rightsquigarrow 1$ 的路径的最小花费，有方程：

$$dp_i=\min_{j\in\text{Anc}(i),d_i-d_j\leq l_i}dp_j+p_j(d_i-d_j)+q_j$$

其中 $\text{Anc}(i)$ 表示 $i$ 节点所有祖先构成的集合；$d_i$ 表示 $i$ 的深度。

直接 DP 可以获得 $31$ 分的高分。

发现转移复杂度较高，且式子中存在 $i$ 和 $j$ 的交叉项，故考虑斜率优化。

为了方便起见，我们先考虑链的情况。

对方程进行套路变形：

$$dp_u-p_ud_u-q_u+p_ud_v=dp_v$$

左侧要尽量小，即要求使用一条斜率为 $p_u$ 的直线得到最小斜率，故设 $X=d$，$Y=dp$，建立平面直角坐标系并维护右下凸包。由于 $p$ 无单调性，而 $d$ 有单调性，所以用单调栈 + 二分维护。

然后你愉快地认为你做完了结果发现还有一个 $l_i$ 的限制……

在没有 $l_i$ 限制的时候，我们使用了 $1$ 个单调栈。加上这个限制，相当于由全局询问变成了区间询问，所以在外面套一个线段树就 ok 了。

但是这个问题是树上的，那么怎么做呢？

直接可撤销？常数大还难写……

套个树链剖分？复杂度 3log……

考虑出栈序。具体来说，在一个节点的 DFS **返回** 的时候记录时间 $T_i$。然后重新 DFS，如果 $u$ 一步能到祖先 $v$，则此时直接查询 $[T_u,T_v]$ 就是对的。原因是区间中不在路径上的点一定仍然没有被 DP 到。

这部分建议自己画个树模拟一遍，就可以理解了。

那么在这个出栈序上面应用原来的做法，就得到了一个不需要撤销的 $O(n\log^2n)$ 做法。

那么这题就做完啦~

线段树维护可持久化单调队列维护凸包

设dp[i] 表示i号点到根节点的最少花费

dis[i] 表示 点i到根节点的距离

dp[i]= min { （dis[i]-dis[j]）* P[i] + Q[i] + dp[j] }   j是i的祖先且dis[i]-dis[j]<=L[i]

即 dp[i]+dis[j]*P[i]=dp[j]+dis[i]*P[i]+Q[i]

斜率优化，dp[i]=斜率为P[i] 的直线 截（dis[j],dp[j]）所得的截距最小值

如果不考虑L[i] 的限制：

那就可以用一个可持久化的单调队列维护 根节点到当前点的路径上所有的点构成的下凸壳


考虑如何去除兄弟节点的子树对单调队列的影响


即在一个节点退出dfs时，将单调队列恢复为这个节点开始dfs的情况

头指针：头指针的前移不会涉及单调队列元素的修改，因为从根往下走不能保证斜率单调，所以头指针不能出队，需要二分找到最小的 斜率>P[i] 的点来更新dp[i]

尾指针：尾指针的前移会涉及到单调队列元素的替换，但替换只会替换一个元素，所以记录下 替换的是谁，原来的尾指针是谁，退出的时候恢复即可

加上L[i]的限制：

用线段树维护，若线段树节点的区间为[l,r]，那这个节点维护的是当前链上  深度为[l,r]的点构成的凸包

即由原来的维护一个单调队列变成维护nlogn个单调队列

每个节点开一个vector？ 光开nlogn个vector就TLE了吧

开一个nlogn的数组v存储整条链上凸包内的点

在建树的时候，按节点的顺序给头指针分一个位置x，若节点的大小为y，

那么这个节点维护的单调队列 中的点会出现在v数组的[x,x+y-1]位置

这样线段树内只需要存储单调队列的头尾指针即可

## 作者：ssxvngn (赞：6)

一种比较新的优化方式，[FZzzz 的题解](https://www.luogu.com.cn/article/4s0jjuym)中稍微提到了这种做法，这里会说的更完整清晰一些。

令 $V$ 为 $p$ 的范围。

首先先对 $s_u$ 做前缀和 $dis_u = dis_{fa_u} + s_u$。  
那么条件就相当于变成了 $u\in \operatorname{subtree}(v), dis_u - dis_v \le l_u$。  
右边那个条件还可以转化为 $dis_v\ge dis_u - l_u$，更方便处理。

对于答案，考虑 DP 求解。  
设 $f_u$ 为 $u$ 点的答案，有转移式 $f_u = f_v + p_u(dis_u - dis_v) + q_u$，$v$ 的条件已在上文提到。  
拆一下贡献，就是 $(-p_u dis_v + f_v) + (p_u dis_u + q_u)$，左边是个直线的形式，而右边是个定值的形式。  
于是可以考虑用李超线段树维护。

考虑到 $dis_v\ge dis_u - l_u$ 这个限制不好处理。  
一种处理方式是考虑重剖，拆成 $\log$ 个 $\operatorname{dfn}$ 区间上的问题。

那么就变成了查询区间直线最小值了。  
这可以考虑线段树套李超线段树。  
具体来说考虑插入一条直线时在线段树经过的 $\log$ 个节点对应的李超线段树都插入这个直线。  
查询就可以分为线段树上 $\log$ 个区间，查询这些区间的李超线段树的 $\min$，最后取这些值中的 $\min$ 即可。  
那么插入是 $O(n\log n\log V)$ 的，查询因为分成了 $\log$ 个区间是 $O(n \log^2 n\log V)$ 的，空间复杂度因为李超线段树的空间实际上是点数级别的，是 $O(n\log n)$ 的。

考虑到这个查询实在太劣了，考虑优化。  
考虑重剖跳的重链的性质，能发现只有最后一条重链可能是不完整的，其余的重链都会跳到链顶。  
因为会有很多跳到链顶的重链，这启发可以对于每个点维护其到对应重链的链顶的这条链的李超线段树的信息。  
具体的，对于插入，考虑如果自己就是链顶，直接插入即可；否则可以类似主席树的想法，从父亲那里继承父亲到链顶的信息，再插入进去自己对应的直线的信息即可。  
插入的时间因为每个点只会插入一次复杂度是 $O(n\log V)$ 的，查询就可以只查询一个李超线段树，因为有 $\log$ 个区间复杂度是 $O(n\log n\log V)$ 的，空间同插入的复杂度是 $O(n\log V)$ 的。

那么此时就只剩下那条不完整的链了，那就可以直接用线段树套李超线段树来做，因为只会只会查询一次时间复杂度是 $O(n\log n\log V)$ 的。  

综上，时间复杂度 $O(n\log n\log V)$，空间复杂度 $O(n\log n)$。

代码比较长，其实挺好写的。

对于每一个点，在前面$\log$个组里分别二分，总复杂度$\log^2$。

首先考虑链上的$dp$，$dp[i]=\min\limits_{dis[i]-dis[j]\leq lim[i]}(P[i]\times (dis[i]-dis[j])+Q[i]+dp[j])$

典型的斜率优化，拿斜率为$P[i]$的直线去切$(dis[j],dp[j])$得到的最小截距。

如果不考虑$lim[i]$，可以直接维护一个下凸壳，三分凸壳找极值。

加上$lim[i]$的限制后，有一个暴力的做法，用线段树维护区间的下凸壳，时空复杂度就多了一个$log$。

然后扩展到树上的做法用树剖，然后复杂度再加一个$log$。

最后复杂度就是$O(nlog^3n)$，用了一片$STL$，这种复杂度还能跑$1800ms$。

这个题体现了OI题的很多题的出题思路，一个问题可以在序列轻松解决，就把它拿到树上来出题。

这题所有数据点都需要开long long，要不然读入就跪了。

算法一：对于1-3个点，我们可以直接暴力dp的去做，求每个点的dp值时可以暴力枚举他的祖先去更新，复杂度O($n ^ 2$)，期望得分30分，适合于考试的时候实在没有时间的时候去写，或者用作对拍的暴力。

算法二：不难发现，这是一个$1D/1D$的dp,观察式子发现可以斜率优化。

$t = 0 / t = 2$时，这是一个序列上的问题，而当$t = 0$时是一个斜率优化板子题。

$t =  2$时, 这个$l[i]$的限制，就感觉有点像单调队列优化和斜率优化的结合体，这个可以用离散化加李超线段树或者线段树维护凸包解决，复杂度O($n\log ^ 2n$)。

算法二结合算法一期望得分50分。

算法三：如果只想到了李超线段树，序列上问题虽然可以解决，但是拿到树上，可能就需要可持久化了。我个人并不知道如何可持久化的李超线段树，所以还是使用线段树维护凸包解决。

对于$t = 1$的点，每个点只会更新他子树内的点，对树进行树剖，处理完某个点后，对它子树内的点维护的凸包里插入这个点即可。

复杂度O($n\log n$)，结合上述算法期望得分80分。

算法四：我们之前的思路，大多数是用祖先去更新后代，不过因为有了$l[i]$这个点，每个点对之后的点的贡献区间就很难加以计算了。

这里有一个树上很常见的技巧，一个点向下走L深度的链有很多，向上走L深度的链只有一个。所以我们可以让一个点去找祖先要贡献代替祖先去更新后代。用树剖+线段树维护凸包即可实现。复杂度O($n\log ^3n$)，期望得分100分。

算法四这个方法在2014年估计是过不去的，因为那个时候的机器跑得很慢，不过在今天只要你常数正常，就能通过。

十分好写，没有任何细节，找祖先更新的时候需要倍增一下估计也不是什么难点。

考虑树形 dp，设 $dp_i$ 为 $i$ 节点到 SZ 市的最小费用，$dis_i$ 为 SZ 市到 $i$ 节点的距离。

显然初始化 $dp_1=0$，然后 $dis_i$ 可以提前预处理出来。

然后有 $dp_u=\min(dp_v+(dis_u-dis_v)\times p_u+q_u)$。（$v$ 为 $u$ 的祖先，$dis_u-dis_v\leq l_u$）。

拆开来：$dp_u=dis_u\times p_u+q_u+\min(-dis_v\times p_u+dp_v)$。

发现拆出来跟 $v$ 有关系的只有 $-dis_v\times p_u+dp_v$。

然后看到这个东西就联想到斜率优化。

那么思路就显而易见了：先对原树进行树剖，然后对于每个点 $i$ 维护一条直线 $f_i(x)=-dis_i\times x+dp_i$。

至于如何维护这些直线，我们可以在每个线段树节点开一个 ```vector```，那么可以证明线段树总的空间复杂度是 $O(n\log n)$，因为每一层只有 $n$ 条直线，一共有 $\log n$ 层。

然后要不断插入一些直线，再用类似半平面交的方法维护下凸壳。

至于 $l$ 的限制，我们可以先用倍增求出来每一个点最远能跳到哪个点。

顺便再推荐一道类似的好题：陶陶的难题2。

## 作者：ywy_c_asm (赞：2)

感谢$stO\space i207M\space Orz$神犇教我的带撤销二进制分组！这里具体讲一下如何实现。

就是这题一眼看上去是个dp对吧，我们把dp式子写出来：

$dp[i]=min\{dp[j]+q_i+p_i(deep_i-deep_j)\}$

然后这玩意是个典型的能斜率优化的式子对吧，就是这样：

$dp[j]=p_ideep_j+(dp[i]-q_i-p_ideep_i)$

于是每个决策点$j$都能表示为一个点$(dp[j],deep_j)$，我们每个点的决策区间是他爸爸到某个祖先的一条链，容易想到我们能在dfs的时候维护一个栈存他到祖先的一条链，然后我们可以把这个栈上的决策区间给他二分出来。然后我们在纸上画一下就会发现我们要维护的是一个下凸壳，然后要在下凸壳上二分。

然后我们要求dp，那就要把对应决策区间里的下凸壳求出来，也就是栈的这段区间。我们dfs的时候不是不断在栈上进行push、pop点吗？那么咱们就能把求答案的过程转化为这个形式：

1.在序列最后插入一个点。

2.删除序列结尾的点。

3.求序列的一段区间内点构成的的凸包并在这个凸包上二分斜率求答案，强制在线。

实际上对于那个询问3我们应该知道，既然是求最大值，那么这个求答案的过程可以看做是互相独立的，也就是说我们并不用直接搞出整个询问区间的凸包，而是可以拆成几段区间分别求凸包。

我们不妨考虑没有pop的时候咋做，咱们是不断的进行强制在线的push的，我们考虑对它进行**二进制分组**，就是我们一开始有一棵大小为$n$的空线段树，然后我们不断的往后面插入点，当我们发现一个节点已经插满的话就在这个上面进行类似线段树分治一样的暴力，把这段区间的凸包$O(n)$求出并用vector存在这个节点上，查询的时候就在线段树上查就行了。不难发现这个暴力总共只会做$O(nlogn)$次，空间复杂度也是$O(nlogn)$的。

那么有了pop呢？好像直接这么做的话复杂度就不对了，因为可能多次进行pop和push然后会被卡成$O(n^2)$的。我们可以把push操作改一下：我们并不把所有满的节点都暴力维护，而是对**每一层**我们不维护最后一个满了的节点。我们发现如果删除$2^k$次才会放弃掉一个大小为$2^k$的节点，这样复杂度就对了。注意特判叶子节点就行了。

这里有个值得注意的细节，二进制分组在理论上本来是**两块相同就合并**，但是实际上我们用初始为空的线段树来支持这个操作，但是一般的线段树的两个儿子的大小可能会相差1，这就导致我们使用分层维护的策略的时候可能会让一个叶子和非叶子处于同一层就会出错，所以我们应该像FFT那样开一个大小为2的整次幂的线段树这样就行了。

然后时间复杂度$O(nlog^2n)$，但是因为那个带删除的二进制分组的常数较大所以跑的甚至不如$O(nlog^3n)$的树剖快……

[P2305 [NOI2014]购票](https://www.luogu.com.cn/problem/P2305)  

可以简单的列出$dp$方程：  
设$f_u$为从$u$点到$1$号点的最小花费。 

$f_u=f_t+p_u*(dep[u]-dep[t])+q_u$。  

很明显是斜率优化，但是有限制$l_u$的限制。  

因为$l_u$不单调，肯定不能用单调队列。至于为什么不能直接在上面二分，其他的题解也讲得很清楚了。  

写这篇题解主要是为了说明这样一个地方：再用数据结构（线段树/树状数组）维护区间凸包的时候，不能直接暴力插入。因为每个点都可以把前面的所有点都删掉。每次插入的最高时间复杂度是$O(n)$的(可以用长链加菊花图卡掉)。    

但是这题并没有卡，所以暴力插入的也可以过，但是时间复杂度并不对。  

在代码上来说，就是不可以这样：  
下面的$t[x]$表示$x$维护区间的凸包（下标从$0$开始），$siz[x]$表示$x$凸包的实际点数。

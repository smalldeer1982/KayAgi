# [COCI 2008/2009 #4] ROT

## 题目描述

给定一个 $r\times c$ 的由字符组成的方桌，你需要输出这个桌子按顺时针方向旋转 $k$ 度之后的状态。

保证 $k$ 为 $45$ 的倍数。

## 说明/提示

#### 数据规模与约定

对于 $100\%$ 的数据，$1\le r,c\le 10$。

#### 说明

**题目译自 [COCI2008-2009](https://hsin.hr/coci/archive/2008_2009/) [CONTEST #4](https://hsin.hr/coci/archive/2008_2009/contest4_tasks.pdf) *T3 ROT***。

```

  r k r

    o```

可以通过样例来找规律。第一组数据为 $45$ 度，每一行前的空格数都是列数减去行数，然后每一次输出后面接一个空格就可以了。

## 代码

#include<iostream>
#include<cstdio>
#include<cmath>
using namespace std;
char fz[500][500];
int main(){

	for(int i=1;i<=r;i++){
		for(int j=1;j<=c;j++){
		    cin>>fz[i][j];
	    }
	}

	n%=360;
	if(n==0){//没转直接输出

			}
			cout<<endl;
		}
	}
	if(n==90){//90度的情况

			for(int j=r;j>=1;j--){
				cout<<fz[j][i];
			}

		}
	}
	if(n==135){//135度的情况
		for(int i=1;i<=r+c-1;i++){
			for(int j=1;j<=abs(c-i);j++)cout<<" ";
			for(int j=r;j>=1;j--){
				for(int g=1;g<=c;g++)

	if(x==315)//旋转315度

					if(j+m-k==i)

### 主要思路

这道题是个**大模拟**，主要还是要**仔细**。

难点： 45 度， 135 度 ， 225 度 ， 315 度。

因为是斜过来看，所以我们可以用 **90 度， 180 度， 270 度** 同样的办法来做这道题：**找规律**。

就比如样例，容易得出 45 度的规律：列数减去行数就是每一行前面要输出的空格数，然后在以**输出一个数，后面跟一个空格**的方式来输出。

用同样的方法，便可以得出 135 度， 225 度， 315 度的规律，输出即可。

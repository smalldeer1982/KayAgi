# 【MX-X18-T1】「FAOI-R6」520

## 题目描述

如图是一个小键盘：

![小键盘](https://cdn.luogu.com.cn/upload/image_hosting/qxqjjdip.png)

或者用表格的形式来看，如下：
|  |  $/$ | $*$  | $-$  |
|:-:|:-:|:-:|:-:|
| $7$  | $8$  | $9$  | $+$  |
| $4$  | $5$  | $6$  |  ^ |
| $1$  | $2$  | $3$  |   |
| $0$  | <  | $.$  |  ^ |

你可以敲击空白键外的其他键。

你可以在如下方式中选一种输入：
- 从左往右敲击同一横线上的连续三个键，如 `123`。
- 从右往左敲击同一横线上的连续三个键，如 `321`。
- 从上往下敲击同一竖线上的连续三个键，如 `963`。
- 从下往上敲击同一竖线上的连续三个键，如 `369`。

给你一个由三个字符组成的字符串，保证这些字符是 $0\sim 9$ 的数字或 `+-*/.`，你需要判断是否能输入这个串。

注意：你不能两次敲击同一个键，比如 `00.` 无法输入。

# 题解

## 作者：BZS_1902 (赞：6)

## 题意 ##
题目给了我们一个计算器的图片，并给出了我们一个字符串，代表依次按的键，让我们求出这几个按键是否是从右到左或从左到右或从上到下或从下到上这样连续按三次的，而且不能重复按一个键或按空格键。
## 思路 ##
本人使用了信息学中的杀神大法，打表，使用一个字符串数组，存储所有可能的排列方式，在数组里找有没有一样的即可。
打表数组代码如下：
```cpp
string ss[100] = {"789", "987", "89+", "+98", "456", "654", "56+", "+65", "123", "321", "741", "147", "410", "014", "852", "258", "520", "025", "963", "369", "63.", ".36", "/85", "58/", "69*", "*96", "/*-", "-*/"};
```
## 代码 ##
```cpp
#include <bits/stdc++.h>
using namespace std;
int main() {
	string s;
	cin >> s;
	string ss[100] = {"789", "987", "89+", "+98", "456", "654", "56+", "+65", "123", "321", "741", "147", "410", "014", "852", "258", "520", "025", "963", "369", "63.", ".36", "/85", "58/", "69*", "*96", "/*-", "-*/"};
	for (int i = 0; i < 28; i++) {
		if (s == ss[i]) {
			cout << "Yes";
			return 0;
		}
	}
	cout << "No";
	return 0;
}
```

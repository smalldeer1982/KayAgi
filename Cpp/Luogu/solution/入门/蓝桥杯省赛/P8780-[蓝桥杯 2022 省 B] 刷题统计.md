# [蓝桥杯 2022 省 B] 刷题统计

## 题目描述

小明决定从下周一开始努力刷题准备蓝桥杯竞赛。他计划周一至周五每天做 $a$ 道题目，周六和周日每天做 $b$ 道题目。请你帮小明计算，按照计划他将在第几天实现做题数大于等于 $n$ 题?

## 说明/提示

对于 $50 \%$ 的评测用例，$1 \leq a, b, n \leq 10^{6}$.

对于 $100 \%$ 的评测用例，$1 \leq a, b, n \leq 10^{18}$. 

蓝桥杯 2022 省赛 B 组 C 题。

10 20 99```

## 作者：2021sunzishan (赞：37)

一道比较水的题目。

## 题目大意：

一个人工作日做 $a$ 题，双休日做 $b$ 题。问：做题数量达到 $n$ 题需要多少天？

## 思路：

- 先求出这个人一周共做多少题。

- 再求出这个人做了几周，还剩几题。

- 如果剩的题数在工作日能做完，就加上需要的天数。否则，先加上工作日的 $5$ 天，再加上双休日需要的天数。

### 是不是非常的简单？非常的简洁明了？

如果还是不太懂，那就看看代码吧！

代码：
```cpp
#include <bits/stdc++.h>
using namespace std;
long long a,b,n; //千万不要忘记开long long！
int main(){
	cin>>a>>b>>n;
	long long s=5*a+b*2; //一周做的题数
	long long s1=n%s; //还剩几题
	long long s2=n/s; //做了几周
	if  (s1<=5*a){//如果工作日能完成
		cout<<s2*7+(s1+a-1)/a;  
  		//s2*7为整周的天数，(s1+a-1)/a为求工作日需要的天数
    }else{ 
		cout<<s2*7+5+(s1-5*a+b-1)/b;
   		//s2*7同理，5为工作日天数(s1-5*a+b-1)/b为双休日所需天数
   		//其中s1-5*a为除去工作日还剩的题数
    }
	return 0;
}

```

# [P8780 \[蓝桥杯 2022 省 B\] 刷题统计](https://www.luogu.com.cn/problem/P8780)

我们看了题以后可以知道，本题就是让你~~多刷题~~统计数量，主要运用**分类讨论**思想。这道题和[P5743 【深基7.习8】猴子吃桃](https://www.luogu.com.cn/problem/P5743)有异曲同工之妙。

## 补充样例说明

第 $1$ 到 $5$ 天，小计完成 $10 \times 5=50$，共计完成 $50$。  
第 $6$ 到 $7$ 天，小计完成 $20 \times 2=40$，共计完成 $90$。  
第 $8$ 天，小计完成 $10$，共计完成 $100$。  
故 $8$ 天可完成。

## 思路1 暴力

很容易先想到一个思路，最笨的办法进行模拟。当今天不是周末就 $n-a$，是周末就 $n-b$。

但是如何判断是不是周末呢？

我们可以用变量 $\texttt{today}$ 存储今天是星期 $\texttt{today}+1$。例如，  
当 $\texttt{today}=1$ 时，今天是星期二，不是周末，要 $n-a$。  
当 $\texttt{today}=6$ 时，今天是星期天，是周末，要 $n-b$。

为什么要用变量 $\texttt{today}$ 存储今天是星期 $\texttt{today}+1$ 而不是 $\texttt{today}$ 呢？

因为以这种方式话 $\texttt{today}$ 的取值范围是 $[0,6]$，由于不会出现星期八，就可以快捷的使用**取模**运算。  
每过一天就 `today%=7`，方便快捷的保证了 $\texttt{today}$ 不超限。

当然，如果您愿意使用 `if` 语句也无可厚非，使用三目运算符也不是说不行。

数据范围 $1 \leq a, b, n \leq 10^{18}$，十年 OI 一场空，不开 `long long` 见祖宗。

AC 代码如下：

```cpp
#include <bits/stdc++.h>
using namespace std;
long long a,b,n;
long long today,days;
int main()
{
    cin>>a>>b>>n;
    while(n>0){
        if(today<5){
            n-=a;
        }
        else{
            n-=b;
        }
        today++;
        today%=7;
        days++;
    }
    cout<<days<<endl;
    return 0;
}

```

## 思路2 简化后计算

因为每星期的刷题数量都是 $5a+2b$ 保持不变，所以可以以星期为周期来进行简化计算。

建立变量 $\texttt{weeks}$ 和 $\texttt{days}$，  
表示需要 $\texttt{weeks}$ 个星期又 $\texttt{days}$ 天才能完成刷题任务。

易得 $\texttt{weeks}=\lfloor \dfrac{n}{5a+2b} \rfloor$。

接下来就是模拟。

AC 代码如下：

```cpp
#include <bits/stdc++.h>
using namespace std;
long long a,b,n;
long long weeks,days;  //需要weeks个星期又days天
long long today;       //今天是星期today
int main()
{
    cin>>a>>b>>n;
    weeks=n/(a*5+b*2); //获取星期
    n-=weeks*(a*5+b*2);//剩下天数
    while(n>0){
        if(today<5){   //周一到周五
            n-=a;
        }
        else{          //周六到周日
            n-=b;
        }
        today++;
        today%=7;      //不会出现周八的情况
        days++;
    }
    cout<<weeks*7+days<<endl;
    return 0;
}

```

## A. 前言
本蒟蒻的第一篇题解。
## B. 题意分析
本题可采用~~暴力法~~**枚举法**。
 
 原题再现： 
小明计划周一至周五每天做 $a$ 道题目，周六和周日每天做 $b$ 道题目。请你帮小明计算，按照计划他将在第几天实现做题数大于等于 $n$ 题？

小明每天会刷多少题，我们已经知道了，那我们就可以**枚举**从“下个星期一”到他完成 flag 那天他刷的题数，将这个数量加入一个变量(代码中叫 $sum$)中，将其与 $n$ 比较。

## C. $\textcolor{green}{AC}$ $\textcolor{green}{Coding}$
读入数据的语句轻轻松松写出来——
	
```cpp
	long long a,b,n,sum = 0,cnt;
	ios::sync_with_stdio(false); //快读,用了以后就不能用scanf和printf了
	cin >> a >> b >> n;
```


请注意，对于 $100\%$ 的评测用例，$\textcolor{red}{1 \leq a, b, n \leq 10^{18}}$！



接下来就是**枚举**。

```cpp
	for(cnt = 1;sum <= n;cnt++) //枚举每一天
	{
		if(cnt % 7 <= 5 && cnt % 7 >= 1) //如果是周一到周五
			sum += a; //刷a道题
		else //周六周日
			sum += b; //刷b道题
	}
```
最后**输出**莫忘记！
```cpp
	cout << cnt;
```
可能有人会问，为什么要把计数变量的值输出前要 $-1$ 呢？
如果你仔细学了循环，你会发现《深基》里有一段话是这么说的：


------------
```cpp
for (循环变量初始值; 循环条件; 循环结束操作) 
{
	循环体执行语句;
}
```
如果符合循环条件，则进入循环。对于每次循环，运行循环体，
**然后进行每轮循环结束的操作**。


------------
意思是说：

（$1$）让循环变量获得一个初始值；

（$2$）判断是否满足循环条件；

（$3$）执行一次语句；

（$4$）进行循环结束操作；

（$5$）回到第二条。

所以，我们的 $cnt$ 变量最后要 $-1$。

## D. **完整程序**（我知道有些人只看这里）
```cpp
//P8780 [蓝桥杯 2022 省 B] 刷题统计 
#include<bits/stdc++.h>
using namespace std;

int main()
{
	long long a,b,n,sum = 0,cnt;
	ios::sync_with_stdio(false); //快读,用了以后就不能用scanf和printf了
	cin >> a >> b >> n;
	for(cnt = 1;sum <= n;cnt++)
	{
		if(cnt%7<=5&&cnt%7>=1)
			sum += a;
		else
			sum += b;
	}
	cnt--;
	cout << cnt;
	return 0;
}

```

## 作者：olegekei (赞：11)

因为数据范围很大，枚举天数肯定会超时，所以我们考虑以星期为单位进行更快的计算。

对于周一到周五，每天做 $a$ 道题目，周六周日每天做 $b$ 道题目，则一星期做题量为 $5a+2b$ 道题目，为了方便，我们记作 $c$。

枚举星期也会超时，这里我们用除法进行计算，每个星期都做 $c$ 道题，那么 $x$ 星期就能做 $x \cdot c$ 道题。

当做题数 $n$ 恰好可以在 $x$ 个星期完成时，用除法求出我们需要的 $x=\dfrac{n}{c}$，那么此时答案即为 $7x$（因为 $x$ 是星期数，转化成天数所以要乘 $7$）。

而当 $n$ 无法恰好在 $x$ 个星期完成时会剩余 $1 \sim 6$ 天，不足一星期，暴力枚举即可。
```cpp
#include<iostream>
using namespace std;
int main(){
long long a,b,n;
cin>>a>>b>>n;
long long c=5*a+2*b;
long long x=n/c;
long long ans=7*x;
n%=c;
for(int i=1;n>0;i++,ans++){
	if(i<=5){
		n-=a;
	}
	else n-=b;
}
cout<<ans;
return 0;
}
```

# P8780 刷题统计题解

我们首先考虑暴力，最差情况时间复杂度是 $\Theta(n)$，$n \le 10^{18}$，显然超时，我们考虑暴力的优化，发现每个星期的刷题数都相等，所以我们考虑把每个星期的刷题数存起来，记为 $week$，这样我们就可以求出需要几个整星期才能使剩余题目数小于 $week$，然后暴力解出剩余需要的天数。

Code：

```cpp
#include <bits/stdc++.h>
using namespace std;
int main() {
    long long a, b, n;
    cin >> a >> b >> n;
    long long week = 5 * a + 2 * b; // 一个星期可以做出的题目数
    long long ans = n / week * 7; // 至少需要几个整星期
    n %= week; // 剩余天数
    for (int i = 1; i <= 5; i++) {
        n -= a, ans++;
        if (n < 0) cout << ans, exit(0);
    }
    for (int i = 1; i <= 2; i++) {
        n -= b, ans++;
        if (n < 0) cout << ans, exit(0);
    }
    // 剩余天数保证不超过7天，所以只需要枚举七天
}
```

### 思考过程

我看到这题的第一反应是 $O(n)$ 模拟过掉这题，但是看到数据彻底歇菜（只能拿50分）。

所以，我们可以另辟蹊径。

小明不论做多少天的题，这些天都可以被分为若干个完整的周和几个零碎的天。这些完整的周刷的题数是一定的，所以可以先算一下这些题需要做多少周，再计算剩下的题目够做多少天。

周数可以用这个公式求出来：$n \div (5*a+2*b)$。因为 C++ 的`int`自动向下取整，所以不用管小数点。剩余天数直接用 $n$ 减去完整周中做的总题数即可求出。

最后算剩下的题够做几天，小蒟蒻也没有用特别复杂的方法，直接 `if` 往上堆了，各位大佬有更好的方法也可以替换掉这堆 `if`。

### 代码

```cpp
#include<bits/stdc++.h>
#define int long long
using namespace std;

int a,b,n;
int week;
int more;
int days;

signed main()
{
	std::ios::sync_with_stdio(false);
	cin>>a>>b>>n;
	week=5*a+2*b;
	days=n/week;
	if(n%week!=0)
	{
		more=n%week;
		     if(a>=more) cout<<days*7+1;
		else if(a*2>=more) cout<<days*7+2;
		else if(a*3>=more) cout<<days*7+3;
		else if(a*4>=more) cout<<days*7+4;
		else if(a*5>=more) cout<<days*7+5;
		else if(a*5+b>=more) cout<<days*7+6;
		else if(a*5+b*2>=more) cout<<days*7+7;
	}
	else cout<<week*7;
	return 0;
}
```

## 分析题意

### 数据规模

上来先看数据规模，这题的数据范围是 $1\leq a,b,n\leq10^{18}$，远超 `int` 的范围，因此我们需要使用 `long long`（或 `unsigned long long`）。

### 声明变量

本题的思路很简单，声明一个 `sum` 变量用于存储目前为止所做的题目之和，一个 `ans` 变量用于存储天数，一个 `day` 变量用于存储当前是星期几。题目中已经规定好从周一开始，因此我们可以直接使 `day = 1`。

### 处理数据

我们可以使用一个 `while` 循环，当 `sum >= n`时，代表做题的目标已经完成，就跳出循环。

题目中已经规定，工作日做 $a$ 道题目，周末做 $b$ 道题目，因此当 `day <= 5` 时，我们使 `sum += a`，`day >= 6` 时，我们使 `sum += b`。

我们还需要把天数 `ans` 和日期 `day` 加上 $1$，同时，为了保证 `day`$\in[1,7]$，我们需要在 `while` 循环的末尾使 `day = day % 7 + 1`。

## 代码实现

``` cpp
#include <iostream>
using namespace std;

// 别忘了用 long long
long long a, b, n, day = 1, ans, sum;

int main() {
    scanf("%lld%lld%lld", &a, &b, &n);
    while (sum < n) {
        if (day >= 6) sum += b; // 周末
        else sum += a; // 工作日
        day = day % 7 + 1;
        ++ans;
    } printf("%lld\n", ans);
    return 0;
}
```

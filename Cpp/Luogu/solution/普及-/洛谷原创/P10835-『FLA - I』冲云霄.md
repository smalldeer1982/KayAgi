**题目等价于询问是否存在正整数 $x$ 使得 $m$ 个 $x$ 的异或和为 $n$。**

#### 测试点 $1 \sim 2$

此时只有 $n=0$。

观察发现当 $m$ 为奇数时无法得到 $0$，$m$ 为偶数时任意 $m$ 个相同正整数异或得到的结果都是 $0$。

- 当 $m$ 为偶数时输出 `Yes`，否则输出 `No`。

#### 测试点 $3 \sim 4$

此时只有 $m=2$ 和 $m=3$。

考虑 $m=2$ 的情况，对于任意**正整数** $x$，由于 $x$ 和 $x$ 的每个二进制位都相同，所以 $x \oplus x=0$。因而能得到的整数只有 $0$。

考虑 $m=3$ 的情况，由于 $x \oplus x \oplus x=0 \oplus x=x$。故可以得到任意正整数，只有 $0$ 无法被得到。

- 当 $m=2$ 时，若 $n=0$ 输出 `Yes`，否则输出 `No`。
- 当 $m=3$ 时，若 $n \neq 0$ 输出 `Yes`，否则输出 `No`。

#### 测试点 $5$

因为 $x \oplus x=0$，$x \oplus 0=x$ 且 $m \geq 2$，所以当 $m > 3$ 时 $m$ 个相同正整数的异或等于 $m-2$ 个相同正整数的异或。

$$\underbrace{x \oplus x \oplus x \oplus \cdots \oplus x}_{m\ 个\ x} = 0 \oplus \underbrace{x \oplus \cdots \oplus x}_{m-2\ 个\ x} = \underbrace{x \oplus \cdots \oplus x}_{m-2\ 个\ x}$$

更进一步地，当 $m$ 为奇数时 $m$ 个 $x$ 的异或等于 $3$ 个 $x$ 的异或；当 $m$ 为偶数时 $m$ 个 $x$ 的异或等于 $2$ 个 $x$ 的异或，即 $0$。因此：

- 当 $m$ 为偶数且 $n=0$，输出 `Yes`。
- 当 $m$ 为奇数且 $n \neq 0$，输出 `Yes`。
- 其余情况输出 `No`。

单组数据时间复杂度 $O(1)$。

int main(){
    scanf("%d",&T);
    while(T--){
        scanf("%d%d",&n,&m);
        if(m%2==0&&n==0||m%2==1&&n!=0) puts("Yes");
        else puts("No");
    }
    return 0;
}

这道题考察的就是异或的运用。

### 做题过程

直接进入正题。

异或的定义决定了它的两条性质：

1. 两个相同的数异或结果为 $0$。
> 因为两个相同的数，他们的二进制位一定是相同的。而异或相同为 $0$，所以最终结果为 $0$。

2. 一个数与 $0$ 异或结果是它本身。

> 因为 $0$ 的二进制每位都是 $0$，而如果原数的二进制该二进制位是 $1$，即不同，则新数的该位也是 $1$；原数的该位如果是 $0$，同为假，则结果还是 $0$。

既然一个数与 $0$ 异或是本身，与本身异或又是 $0$，也就是说：偶数个相同的数异或为 $0$，奇数个相同的数异或为它们本身。

因为题目说了数列 $a$ 中所有元素的值都相同，所以如果 $n$ 为 $0$，那就要求数列 $a$ 有偶数项。

如果 $n$ 不为 $0$，则数列 $a$ 必须是奇数个。这时我们把数列 $a$ 的元素设为 $n$ 的值。

可以看出如果 $n$ 是 $0$ 并且 $m$ 为奇数，由于数列 $a$ 的元素不能是 $0$（正整数），所以结果肯定不能为 $0$；如果 $n$ 不是 $0$ 并且 $m$ 是偶数，那肯定不能达成 $n$。

### 编写程序

我们可判断 $m$ 的奇偶性时可以用与 $1$ 的方法（想必做这题的同学也知道位运算的性质了吧），这样只有最低位是 $1$ 才会返回 $1$，要不就返回 $0$。

```cpp
#include<bits/stdc++.h>
using namespace std;
int main(){
    int t;
    cin>>t;
    for(int i=0;i<t;i++){
        int n,m;
        cin>>n>>m;
        if((n==0&&(m&1)==0)||(n>0&&(m&1)==1))cout<<"Yes";
        else cout<<"No";
        cout<<endl;
    }
    return 0;
}
```

### 前置知识

#### 异或

`^` 是按位异或操作。异或也叫 $\text{xor}$，符号是 $\oplus$。

对于 $a$、$b$ 两个二进制值，如果 $a$、$b$ 不相同，则异或结果为 $1$；如果 $a$、$b$ 两个值相同，异或结果为 $0$。

对两个十进制数进行按位异或是先将它们转化为二进制数，再一位一位的进行异或，最终将结果转化为十进制。

异或的几个特性：

- 两个相同的数按位异或的结果为 $0$；

- 任何数与 $0$ 异或结果都为 $0$。

#### 与

`&` 是按位与操作：将二进制下两个运算量的每一位进行逻辑“与”操作。

与操作（同真则真）：当两个数都不为 $0$ 时，返回真。

在这道题里我只是想讲它的一个特性：

- 将一个数按位与 $1$，如果这个数是奇数，结果为 $1$；如果这个数是偶数，结果为 $0$。所以，按位与 $1$ 与和对 $2$ 取模的效果是一样的。

### 题目分析

题目中有一句非常关键的话：**数列 $a$ 中所有元素的值都相同**。所以结合上面异或的两条规律，我们就能推出：当 $m$ 为偶数时，$n$ 只会是 $0$；当 $m$ 为奇数时，$n$ 只会是那个共同的 $a_i$。

如果你还不太理解的话，请看接下来两段：

#### 如果 $m$ 是偶数

如果 $m$ 是偶数，因为数列 $a$ 中所有元素的值都相同，又因为相同的元素异或会两两抵消，所以异或后的 $\dfrac{m}{2}$ 个 $0$ 相异或得到的还是 $0$。

#### 如果 $m$ 是奇数

如果 $m$ 是奇数，那么 $m-1$ 个相同的数先异或得到 $0$，然后 $0$ 再与最后一个数列 $a$ 的剩余元素相异或，得到的原封不动的 $a_i$ 就是最终答案。

而我们可以控制 $a_i$ 的值（只要是正整数）。所以，只要 $n$ 不是 $0$，我们就能得到符合要求的结果。

### 代码编写

在判断奇偶性的这里，我推荐大家用按位与 $1$ 而不是对 $2$ 取模。因为在计算机中，位运算的速度是很快的，而四则运算和模运算（尤其是模运算）是非常慢的。所以在一些需要大量判断奇偶性而时间复杂度又很高的程序中，使用位运算可以节约时间。

接下来就分情况写出代码即可。总结下来，能满足的情况只有两种：$n$ 为 $0$ 且 $m$ 为偶数，和 $n$ 不为 $0$ 且 $m$ 为奇数。

对了，在判断中直接写一个值或变量的名字和判断它是否非 $0$ 是一样的。

#### 完整代码

	ios::sync_with_stdio(0),cin.tie(0),cout.tie(0);

## Step1. 读题&分析

对于这道题目，我们需要根据给定的整数 $ n $ 和 $ m $ 确定是否可以用相同的正整数 $ a $ 组成一个长度为 $ m $ 的序列，使得该序列所有元素的异或结果为 $ n $。

$$
a \oplus a \oplus \cdots \oplus a
$$

其中 $ a $ 出现了 $ m $ 次。由于异或的一个重要性质是 $ x \oplus x = 0 $，因此对于一个相同的数 $ a $ 异或 $ m $ 次的结果为：

- 如果 $ m $ 是偶数，结果是 $ 0 $。
- 如果 $ m $ 是奇数，结果是 $ a $。

1. 如果 $ m $ 是偶数，那么异或结果只能是 $ 0 $。所以，如果 $ n = 0 $ 时输出 `Yes`，否则输出 `No`。
2. 如果 $ m $ 是奇数，那么异或结果是 $ a $，因此只要 $ n $ 可以作为数列中的元素（即 $ n $ 是正整数），则输出 `Yes`，否则输出 `No`。

# [题目传送门](https://www.luogu.com.cn/problem/P10835)
## 题目大意

- 若 $x$ 和 $y$ 的第 $k$ 个二进制位相同，结果的第 $k$ 个二进制位为 $0$；
- 若 $x$ 和 $y$ 的第 $k$ 个二进制位不同，结果的第 $k$ 个二进制位为 $1$。
## 题目描述

给定**整数** $n$ 和 $m$，判断是否存在满足下列条件的数列 $a$。本题中数列元素的下标从 $1$ 开始。

- 数列 $a$ 的长度为 $m$，数列 $a$ 的每一项都为**正整数**。

- $a_1 \oplus a_2 \oplus \cdots \oplus a_m = n$，即数列 $a$ 的所有项异或得到的结果等于 $n$。

- 数列 $a$ 中所有元素的值都相同。

## 题目思路

对于第一组测试数据，数列 $a$ 可以是 $[3,3,3]$，此时 $a_1 \oplus a_2 \oplus a_3 = 3 \oplus 3 \oplus 3 = 3$。

对于第二组测试数据，数列 $a$ 可以是 $[2,2,2,2,2]$，此时 $a_1 \oplus a_2 \oplus a_3 \oplus a_4 \oplus a_5 = 2 \oplus 2 \oplus 2 \oplus 2 \oplus 2 = 2$。                

我们可以得出规律，当 $m$ 为奇数时，$a_1 \oplus a_2 \oplus \cdots \oplus a_m = n$ 这个数列是成立的。     
反之，当 $m$ 为偶数时，$a_1 \oplus a_2 \oplus \cdots \oplus a_m = n$ 则不成立。           
但是我们看第三个样例，发现当 $n=0$ 且 $m$ 为奇数的时候，这个数列是不成立的。如数列 $a$ 可以是 $[4,4,4]$，此时：$a_1 \oplus a_2 \oplus a_3 = 4 \oplus 4 \oplus 4  = 1$，$1 \neq 0$，因此不成立。   
反之，发现当 $n=0$ 且 $m$ 为奇数的时候，数列成立。如数列 $a$ 可以是 $[4,4,4,4]$，此时：$a_1 \oplus a_2 \oplus a_3 \oplus a_4 = 4 \oplus 4 \oplus 4 \oplus 4  = 0$，数列成立。

### 题意

给定整数 $n$ 和 $m$，问是否存在长度为 $m$、所有元素值相同且所有元素异或结果为 $n$ 的数列 $a$。

### 分析

不妨先来考虑 $n$ 的每一位如何由若干个相同元素异或得到。

我们知道对于两个二进制数 $x,y$，若两者某一位相同，则该位异或运算的结果为 $0$，反之为 $1$。所以易证任何个 $0$ 进行异或运算的结果仍为 $0$，而奇数个 $1$ 进行异或运算的结果为 $1$，偶数个 $1$ 进行异或运算的结果为 $0$。

有了上述结论，我们考虑如何通过元素值相同的数列异或得到 $n$。

对于二进制数 $n$（$n \in N^*$）的每一位 $n_i$，我们分 $0$ 和 $1$ 两种情况讨论。

- 如果 $n_i = 0$，由上述结论可知使数列 $a$ 的每一元素对应位数为 $0$ 即可。
- 如果 $n_i = 1$，由上述结论可知对于合法的数列 $a$，每一元素对应位数都应为 $1$。而由于奇数个 $1$ 进行异或运算的结果为 $1$，所以只当数列的长度 $m$ 为奇数时数列满足条件。

而我们知道对于任何正整数 $n$，其二进制表示中至少有一位为 $1$。

当 $n = 0$ 时，出于数列 $a$ 的每一项都为正整数，数列 $a$ 只能由 $m$ 个 $1$ 组成。而偶数个 $1$ 进行异或运算的结果为 $0$，所以只当数列的长度 $m$ 为偶数时数列满足条件。

通过上述推导，题意可简化为判断 $n$ 是否为 $0$ 和 $m$ 的奇偶性。

				cout << "Yes" << endl;

		} else {
			if (m % 2 == 0)
				cout << "No" << endl;
			else

## 思路
以下记按位异或运算的符号为 $\oplus$。

首先按位异或是满足结合律的，然后对于这道题，我们要用到按位异或的以下两个性质：
1. $b \oplus b = 0$；
2. $b \oplus 0 = b$。

有了这两个性质，我们就可以开始做题了。首先，数列 $a$ 所有元素的值都相同且都只能为正整数，现将这个值记为 $x$。根据上面两个性质，不难得出，**在数列长度为奇数时，这个序列的异或和是 $x$，长度为偶数时则是 $0$。**

由上述结论可知，满足以下两个条件中任意一个时：
1. $n = 0$ 且 $m$ 为偶数（此时我们再将 $x$ 设为任意正整数就是满足条件的数列 $a$）；
2. $n$ 为任意正整数且 $m$ 为奇数（此时我们再将 $x$ 设为 $n$ 就是满足条件的数列 $a$）。

就能保证数列 $a$ 一定存在。否则，数列 $a$ 就一定不存在。

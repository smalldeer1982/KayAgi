# 题目信息

# [NOIP 2007 普及组] 守望者的逃离

## 题目背景

NOIP2007 普及组 T3

## 题目描述

恶魔猎手尤迪安野心勃勃，他背叛了暗夜精灵，率领深藏在海底的娜迦族企图叛变。

守望者在与尤迪安的交锋中遭遇了围杀，被困在一个荒芜的大岛上。

为了杀死守望者，尤迪安开始对这个荒岛施咒，这座岛很快就会沉下去。到那时，岛上的所有人都会遇难。

守望者的跑步速度为 $17m/s$，以这样的速度是无法逃离荒岛的。庆幸的是守望者拥有闪烁法术，可在 $1s$ 内移动 $60m$，不过每次使用闪烁法术都会消耗魔法值 $10$ 点。守望者的魔法值恢复的速度为 $4$ 点每秒，只有处在原地休息状态时才能恢复。

现在已知守望者的魔法初值 $M$，他所在的初始位置与岛的出口之间的距离 $S$，岛沉没的时间 $T$。你的任务是写一个程序帮助守望者计算如何在最短的时间内逃离荒岛，若不能逃出，则输出守望者在剩下的时间内能走的最远距离。

注意：守望者跑步、闪烁或休息活动均以秒为单位，且每次活动的持续时间为整数秒。距离的单位为米。


## 说明/提示

对于 $30\%$ 的数据，$1  \le  T   \le  10$，$ 1   \le   S   \le  100$；

对于 $50\%$ 的数据，$1   \le  T   \le   10^3$，$ 1   \le  S  \le   10^4$；

对于 $100\%$ 的数据，$1   \le   T   \le   3\times  10^5$，$0   \le   M   \le   10^3$，$ 1   \le  S   \le   10^8$。

### 题解综合分析与结论
本题核心在于合理平衡魔法闪现与跑步的收益。最优策略为：优先使用闪现，并在每一时刻维护两种状态（闪现累积距离和跑步累积距离），通过贪心比较取最大值作为当前最优状态。该方法时间复杂度为 O(T)，满足题目约束。

---

### 精选题解及评分

#### 1. 作者：wzr1125（★★★★★）
**关键亮点**：
- 双变量贪心思路，分别维护闪现距离（`fla`）和跑步距离（`run`）
- 每次优先使用闪现，并在闪现距离超过跑步距离时更新跑步为闪现结果
- 时间复杂度 O(T)，代码简洁高效（仅需 20 行）

**核心代码**：
```cpp
int m,s,t,fla,run;
for(int i=1;i<=t;i++){ 
    if(m>=10) m-=10,fla+=60,run+=17;
    else m+=4, run +=17;
    if(fla>run) run=fla; // 关键：更新跑步为最优状态
    if(max(fla,run)>=s) return 0; // 提前终止判断
}
```

#### 2. 作者：a202401006（★★★★★）
**关键亮点**：
- 三目运算符简化魔法值处理
- 单循环内完成状态更新，逻辑紧凑
- 显式维护 `run = max(run+17, magic)` 体现贪心本质

**核心代码**：
```cpp
int run = 0, magic = 0;
for(int i=1; i<=t; i++){
    m >=10 ? (m-=10, magic+=60) : (m+=4);
    run = max(run+17, magic); // 核心：取闪现与跑步的叠加最优
    if(run >= s) { /* Yes 分支 */ }
}
```

#### 3. 作者：qhr2023（★★★★☆）
**关键亮点**：
- 动态规划与贪心的混合思路，用 `s1` 表示总最优距离
- 每轮优先处理闪现逻辑，再通过 `s1 = max(s1+17, s2)` 更新状态
- 代码极简（仅 15 行），空间复杂度 O(1)

**核心代码**：
```cpp
int s1=0, s2=0;
for(int i=1; i<=t; i++){
    m >=10 ? (m-=10, s2+=60) : (m+=4);
    s1 = max(s1+17, s2); // 核心：贪心叠加状态
    if(s1 >= s) { /* Yes 分支 */ }
}
```

### 最优关键思路
**双状态贪心更新**：维护闪现与跑步两种移动方式的累积距离，每一步优先使用闪现，并在闪现收益更高时将跑步状态同步至闪现结果。这一策略保证了每时刻的最优性，且无需复杂的分支判断。

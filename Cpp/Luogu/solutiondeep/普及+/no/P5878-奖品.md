# 题目信息

# 奖品

## 题目描述

学校刚开完运动会，准备为尽可能多的同学评奖，并为每个人颁发一份奖品。一份奖品包括 $N$ 个物品，如：$5$ 支铅笔、$10$ 本练习薄等。每份奖品完全一样。虽然学校的保管室里还有一些办去年运动会后剩余的物品。在商店里，每种物品都有很多，但是，只有两种包装：大盒或小盒，并且不拆开买。

现在的问题是，充分利用这 $M$ 元钱，最多可准备多少分这样的奖品？


## 说明/提示

对于全部的数据，满足：

$1 \le N \le 100$，$1 \le M \le 10^5$。

$10 \le x, pm \le 100$，$1 \le y, sm \le 100$，$sm < sv \le 100$，$pm<pv\le 100$。

## 样例 #1

### 输入

```
2 100
10 8 10 10 13 11
12 20 6 10 17 24
```

### 输出

```
5```

## 样例 #2

### 输入

```
3 65
10 5 7 10 13 14
10 5 8 11 14 15
10 5 9 12 15 16
```

### 输出

```
2```

# AI分析结果



### 算法分类：二分

### 题解分析与结论
该题核心思路是二分答案+验证，所有题解均基于二分法。验证函数的核心在于高效计算满足每个物品需求的最小花费。各题解的主要差异在于验证函数的实现方式：
1. **动态规划背包法**：对每个物品构建完全背包模型，计算花费j元能获得的最大数量，判断是否满足需求。
2. **数学枚举法**：直接枚举小包装购买数量，计算对应大包装需求，取最小总花费。

### 精选题解
#### 1. 风急风也清 (⭐⭐⭐⭐⭐)
- **亮点**：直接枚举小包装数量，避免动态规划的高时间复杂度，代码简洁高效。
- **核心代码**：
  ```cpp
  int money = 1000000, need = x[i] * mid - y[i];
  for (int j = 0; j <= (need + sm[i]-1)/sm[i]; j++) {
      int bn = (max(0, need - j * sm[i]) + sv[i]-1) / sv[i];
      money = min(money, j * pm[i] + bn * pv[i]);
  }
  ```
- **个人心得**：强调特判`bn`可能为负数的情况，避免错误。

### 最优思路总结
**二分答案 + 数学枚举优化**：
1. 二分奖品数量，检查是否可用M元满足。
2. 对每个物品，枚举小包装购买数量，计算对应大包装数量，取总花费最小。
3. 注意向上取整和边界处理（如负数需求直接跳过）。

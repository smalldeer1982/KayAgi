# 题目信息

# [NOIP 2006 提高组] 金明的预算方案

## 题目描述

金明今天很开心，家里购置的新房就要领钥匙了，新房里有一间金明自己专用的很宽敞的房间。更让他高兴的是，妈妈昨天对他说：“你的房间需要购买哪些物品，怎么布置，你说了算，只要不超过 $n$ 元钱就行”。今天一早，金明就开始做预算了，他把想买的物品分为两类：主件与附件，附件是从属于某个主件的，下表就是一些主件与附件的例子：

| 主件 | 附件 |
| :----------: | :----------: |
| 电脑 | 打印机，扫描仪 |
| 书柜 | 图书 |
| 书桌 | 台灯，文具 |
| 工作椅 | 无 |

如果要买归类为附件的物品，必须先买该附件所属的主件。每个主件可以有 $0$ 个、$1$ 个或 $2$ 个附件。每个附件对应一个主件，附件不再有从属于自己的附件。金明想买的东西很多，肯定会超过妈妈限定的 $n$ 元。于是，他把每件物品规定了一个重要度，分为 $5$ 等：用整数 $1 \sim 5$ 表示，第 $5$ 等最重要。他还从因特网上查到了每件物品的价格（都是 $10$ 元的整数倍）。他希望在不超过 $n$ 元的前提下，使每件物品的价格与重要度的乘积的总和最大。

设第 $j$ 件物品的价格为 $v_j$，重要度为 $w_j$，共选中了 $k$ 件物品，编号依次为 $j_1,j_2,\dots,j_k$，则所求的总和为：

$$v_{j_1} \times w_{j_1}+v_{j_2} \times w_{j_2}+ \dots +v_{j_k} \times w_{j_k}$$

请你帮助金明设计一个满足要求的购物单。


## 说明/提示

#### 数据规模与约定

对于全部的测试点，保证 $1 \leq n \leq 3.2 \times 10^4$，$1 \leq m \leq 60$，$0 \leq v_i \leq 10^4$，$1 \leq p_i \leq 5$，$0 \leq q_i \leq m$，答案不超过 $2 \times 10^5$。

### 综合分析与结论
各题解均围绕分组背包变形展开，核心思路是将主附件组合视为分组，枚举所有可能的购买组合。差异主要体现在数据结构设计、代码简化技巧及优化细节处理。Anguei 的预处理+lambda优化、ClV_Csy 的详细结构体设计、wangkaiwei 的边读入边处理附件均值得学习。

### 精选题解及亮点

#### 1. Anguei（⭐⭐⭐⭐⭐）
**核心亮点**：  
- 采用二维数组存储主附件信息，逻辑清晰  
- 使用 lambda 表达式简化状态转移方程中的重复计算  
- 完整处理所有五种组合情况，代码简洁高效  

**代码核心**：  
```cpp
auto cost2 = [v, p, i](int x, int y) { return v[i][x] + v[i][y]; };
auto rpp = [v, p, i](int x) { return v[i][x] * p[i][x]; };
// 枚举四种购买组合的转移条件
if (j >= cost2(0, 1)) 
    f[j] = max(f[j], f[j - cost2(0,1)] + rpp(0)+rpp(1));
```

#### 2. ClV_Csy（⭐⭐⭐⭐）
**核心亮点**：  
- 通过结构体明确区分主附件属性  
- 详尽的代码注释与 Q&A 分析，适合新手理解  
- 显式处理空附件情况，避免无效计算  

**代码核心**：  
```cpp
struct node { int v1=0, v2=0, v3=0, s1=0, s2=0, s3=0; };
// 动态规划时显式判断附件存在性
if (a[i].v1 + a[i].v2 <= j)
    dp[j] = max(dp[j], ... + a[i].s1 + a[i].s2);
```

#### 3. wangkaiwei（⭐⭐⭐⭐）
**个人心得**：  
> "边读入边处理附件，避免二次遍历。虽然变量命名略简（lc/rc），但通过结构体直接绑定主附件关系，提升处理效率。"

**代码核心**：  
```cpp
struct thing{int v,p,q,lc,rc,s;};
// 动态规划时直接访问结构体中的附件指针
if(a[i].lc!=0 && j >= a[i].v + a[a[i].lc].v)
    dp[j] = max(dp[j], ... + a[a[i].lc].s);
```

### 关键思路与技巧
1. **分组枚举法**：将主件及其附件组合视为一个分组，枚举所有可能的购买组合（0-4种附件），转化为标准分组背包问题。
2. **预处理优化**：通过二维数组或结构体存储主附件信息，避免动态规划时频繁计算组合价格。
3. **空间压缩技巧**：使用滚动数组优化空间至 O(N)，并通过逆序枚举金额保证无后效性。
4. **代码可读性**：通过 lambda/结构体/辅助函数封装重复逻辑，如 Anguei 的 `cost2` 函数。

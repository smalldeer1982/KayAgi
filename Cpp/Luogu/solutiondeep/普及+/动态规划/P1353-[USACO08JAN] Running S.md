# 题目信息

# [USACO08JAN] Running S

## 题目描述

奶牛们打算通过锻炼来培养自己的运动细胞，作为其中的一员，贝茜选择的运动方式是每天进行 $n$ 分钟的晨跑。在每分钟的开始，贝茜会选择下一分钟是用来跑步还是休息。


贝茜的体力限制了她跑步的距离。更具体地，如果贝茜选择在第 $i$ 分钟内跑步，她可以在这一分钟内跑 $d_i$ 米，并且她的疲劳度会增加 $1$。不过，无论何时贝茜的疲劳度都不能超过 $m$。  

如果贝茜选择休息，那么她的疲劳度就会每分钟减少 $1$，但她必须休息到疲劳度恢复到 $0$ 为止。在疲劳度为 $0$ 时休息的话，疲劳度不会再变动。晨跑开始时，贝茜的疲劳度为 $0$ 。

还有，在 $n$ 分钟的锻炼结束时，贝茜的疲劳度也必须恢复到 $0$，否则她将没有足够的精力来对付这一整天中剩下的事情。

请你计算一下，贝茜最多能跑多少米。


## 说明/提示

【数据范围】  
对于 $100\%$ 的数据，$1\le n \le 10^4$，$1\le d_i \le 1000$，$1\le m \le 500$。

【样例说明】

贝茜在第 $1$ 分钟内选择跑步（跑了 $5$ 米），在第 $2$ 分钟内休息，在第 $3$ 分钟内跑步（跑了 $4$ 米），剩余的时间都用来休息。  
因为在晨跑结束时贝茜的疲劳度必须为0，所以她不能在第 $5$ 分钟内选择跑步。  
最终跑的总距离为 $9$。



## 样例 #1

### 输入

```
5 2
5
3
4
2
10
```

### 输出

```
9
```

# AI分析结果



### 综合分析与结论
本题核心是动态规划，需处理疲劳度恢复的约束。各题解围绕状态定义（疲劳度与时间的二维状态）展开，主要差异在于状态转移方式和优化思路。刷表法通过预更新未来状态减少冗余计算，填表法则显式处理历史状态转换。一维优化和前缀和技巧能有效降低空间复杂度。

### 高星题解推荐

#### 1. 今宵満月（4.5星）
**关键亮点**：刷表法的独特应用，清晰的状态转移逻辑，代码简洁高效。  
**核心思路**：用`f[i][j]`表示第i分钟疲劳j时的最大距离。按顺序处理三种转移：  
- 保持休息（`f[i][0]`继承前一状态）  
- 开始休息j分钟（更新`f[i+j][0]`）  
- 继续跑步（更新`f[i+1][j+1]`）  
**实现技巧**：转移顺序确保正确性，避免状态覆盖问题。
```cpp
for(int i=1; i<=n; ++i) {
    for(int j=0; j<=min(i, m); ++j) {
        if(j==0) f[i][0] = max(f[i-1][0], f[i][0]);
        else f[i+j][0] = max(f[i+j][0], f[i][j]);
        f[i+1][j+1] = max(f[i+1][j+1], f[i][j]+d[i+1]);
    }
}
```

#### 2. Log_x（4星）
**关键亮点**：填表法标准实现，分情况讨论清晰，引入辅助函数提升可读性。  
**核心思路**：`f[i][j]`的三种转移来源：  
1. 休息j分钟后恢复（从`i-j`分钟转移）  
2. 继续休息（继承`f[i-1][0]`）  
3. 跑步（从`j-1`疲劳度累加距离）  
**优化点**：逆序更新j避免覆盖，使用`CkMax`优化比较。
```cpp
for(int i=1; i<=n; ++i) {
    for(int j=1; j<=Min(i, m); ++j)
        CkMax(f[i][0], f[i - j][j]);
    CkMax(f[i][0], f[i - 1][0]);
    for(int j=m; j>=1; --j)
        CkMax(f[i][j], f[i - 1][j - 1] + d[i]);
}
```

#### 3. D10s（4星）
**关键亮点**：一维空间优化，逆向思维简化状态转移。  
**核心思路**：`F[i]`表示从i开始且疲劳度为0的最大距离。枚举后续连续跑步j分钟，利用前缀和快速计算区间和。  
**优化技巧**：前缀和预处理跑步距离，逆向更新减少状态数。
```cpp
for(int i=n; i; i--) {
    for(int j=1; j<=m; j++) {
        if(i+2*j-1 > n) break;
        f[i] = max(f[i], f[i+2*j] + s[i+j-1] - s[i-1]);
    }
    f[i] = max(f[i], f[i+1]);
}
```

### 关键技巧总结
1. **刷表法优化**：按未来状态更新，避免重复计算。
2. **逆向思维**：一维优化中从后向前处理，简化状态依赖。
3. **前缀和加速**：预处理区间和，快速计算连续跑步距离。
4. **状态转移顺序**：处理休息状态的转移前先处理继承情况（如`f[i][0]`的初始化）。

# 题目信息

# [NOIP 2017 普及组] 棋盘

## 题目背景

NOIP2017 普及组 T3

## 题目描述

有一个 $m \times m$ 的棋盘，棋盘上每一个格子可能是红色、黄色或没有任何颜色的。你现在要从棋盘的最左上角走到棋盘的最右下角。

任何一个时刻，你所站在的位置必须是有颜色的（不能是无色的）， 你只能向上、下、左、右四个方向前进。当你从一个格子走向另一个格子时，如果两个格子的颜色相同，那你不需要花费金币；如果不同，则你需要花费 $1 $ 个金币。

另外， 你可以花费 $2$ 个金币施展魔法让下一个无色格子暂时变为你指定的颜色。但这个魔法不能连续使用， 而且这个魔法的持续时间很短，也就是说，如果你使用了这个魔法，走到了这个暂时有颜色的格子上，你就不能继续使用魔法； 只有当你离开这个位置，走到一个本来就有颜色的格子上的时候，你才能继续使用这个魔法，而当你离开了这个位置（施展魔法使得变为有颜色的格子）时，这个格子恢复为无色。

现在你要从棋盘的最左上角，走到棋盘的最右下角，求花费的最少金币是多少？

# AI分析结果



### 综合分析与结论
本题核心为处理带魔法限制的最短路问题，各题解主要采用两种思路：**魔法转换+优先队列BFS**和**分层图状态扩展**。前者通过将魔法转化为特殊移动方式简化问题，后者通过多维状态记录颜色和魔法使用情况。优先队列BFS（类似Dijkstra）保证时间复杂度，分层图确保状态完备性。

---

### 高星题解推荐
#### 1. ZigZagKmp（★★★★★）
**关键亮点**：
- 将魔法转化为12种移动方式（普通4向+魔法8向），直接构建边权。
- 优先队列BFS高效处理非均匀边权，避免DFS超时。
- 代码简洁清晰，预处理终点无色特判。

**核心代码**（优先队列BFS）：
```cpp
struct node{ int x,y,c,w; };
priority_queue<node> q;
int dx[]={0,1,0,-1,1,1,-1,-1,0,2,0,-2}; // 12方向
int dy[]={1,0,-1,0,1,-1,1,-1,2,0,-2,0};
int dw[]={0,0,0,0,2,2,2,2,2,2,2,2}; // 魔法额外花费

void bfs() {
    memset(dis,0x3f,sizeof(dis));
    dis[1][1]=0;
    q.push({1,1,a[1][1],0});
    while(!q.empty()) {
        node cur = q.top(); q.pop();
        for(int i=0; i<12; i++) { // 遍历所有移动方式
            node nxt = {cur.x+dx[i], cur.y+dy[i], a[nxt.x][nxt.y], cur.w + dw[i]};
            if(nxt.c) { // 目标格子有色
                if(cur.c != nxt.c) nxt.w++;
                if(dis[nxt.x][nxt.y] > nxt.w) {
                    dis[nxt.x][nxt.y] = nxt.w;
                    q.push(nxt);
                }
            }
        }
    }
}
```

#### 2. dxzier（★★★★☆）
**关键亮点**：
- 分层图思想，状态包括位置、颜色、是否使用魔法。
- 处理魔法连续使用限制严谨，状态转移清晰。

**核心思路**：
```python
状态 = (x, y, 当前颜色, 是否使用魔法)
转移规则：
1. 普通移动：计算颜色异同花费。
2. 魔法移动：仅当未使用魔法时，支付2金币并指定颜色。
```

---

### 关键思路总结
1. **魔法转换技巧**：将跳跃/斜向移动视为魔法，统一处理为固定边权。
2. **优先队列优化**：确保每次扩展当前最小路径，时间复杂度O(M logN)。
3. **状态分层设计**：多维状态记录额外条件，避免重复搜索。

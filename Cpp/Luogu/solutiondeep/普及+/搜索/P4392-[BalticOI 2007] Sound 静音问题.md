# 题目信息

# [BalticOI 2007] Sound 静音问题

## 题目描述

数字录音中，声音是用表示空气压力的数字序列描述的，序列中的每个值称为一个采样，每个采样之间间隔一定的时间。 

很多声音处理任务都需要将录到的声音分成由静音隔开的几段非静音段。为了避免分成过多或者过少的非静音段，静音通常是这样定义的：$m$ 个采样的序列，该序列中采样的最大值和最小值之差不超过一个特定的阈值 $c$。 

请你写一个程序，检测 $n$ 个采样中的静音。

## 样例 #1

### 输入

```
7 2 0
0 1 1 2 3 2 2```

### 输出

```
2
6```

# AI分析结果



### 综合分析与结论
该题核心在于高效查询固定长度区间的最大/最小值差。主流解法包括单调队列、ST表、线段树、树状数组等，其中**单调队列**和**ST表**在时间/空间效率上表现最优。

#### 1. Uni_Tune 的单调队列解法（5星）
**关键亮点**：
- 时间复杂度O(n)，空间O(n)，理论最优解

#### 2. Schwarzkopf_Henkal 的优化ST表解法（4星）
**关键亮点**：
- 预处理时仅计算log2(m)层，空间优化至O(n logm)
- 查询复杂度O(1)，整体效率O(n logm)
- 针对题目特点进行空间优化，避免MLE

**核心代码片段**：
```cpp

maxv = max(stg[i][k], stg[i+m-(1<<k)][k]);

```

#### 3. 寒鸽儿 的单调队列解法（4星）
**个人心得**：
- 强调队列中存储下标而非值，便于判断元素是否在窗口内
- 通过`i-m+1`计算窗口左边界，确保队列元素有效性

### 关键思路总结
1. **滑动窗口最值**：优先选择单调队列，时间复杂度严格线性
2. **静态区间查询**：ST表在空间优化后是最佳选择，线段树更通用但常数大
3. **空间优化技巧**：根据问题特点限制预处理范围（如ST表仅处理到logm层）

# 题目信息

# [NOIP 2011 提高组] 铺地毯

## 题目描述

为了准备一个独特的颁奖典礼，组织者在会场的一片矩形区域（可看做是平面直角坐标系的第一象限）铺上一些矩形地毯。一共有 $n$ 张地毯，编号从 $1$ 到 $n$。现在将这些地毯按照编号从小到大的顺序平行于坐标轴先后铺设，后铺的地毯覆盖在前面已经铺好的地毯之上。

地毯铺设完成后，组织者想知道覆盖地面某个点的最上面的那张地毯的编号。注意：在矩形地毯边界和四个顶点上的点也算被地毯覆盖。

【数据范围】

对于 $30\%$ 的数据，有 $n \le 2$。  
对于 $50\%$ 的数据，$0 \le a, b, g, k \le 100$。  
对于 $100\%$ 的数据，有 $0 \le n \le 10^4$, $0 \le a, b, g, k \le {10}^5$。   

noip2011 提高组 day1 第 $1$ 题。

# AI分析结果

### 综合分析与结论
本题主要考查对地毯铺设问题的模拟与求解，核心是判断给定点被哪张地毯覆盖。众多题解思路相近，均意识到开二维数组会超空间，转而采用直接判断点是否在各地毯内的方法。不同在于遍历地毯顺序及数据存储方式。

### 通用建议与扩展思路
1. **思路优化**：从后往前遍历地毯能更快找到最上面覆盖点的地毯，因为后铺地毯在上面，一旦找到即停止遍历，减少不必要计算。
2. **数据结构选择**：使用结构体数组存储地毯信息，代码结构更清晰，方便管理和操作。
3. **代码实现**：注意边界条件处理，如未找到覆盖地毯时输出 -1 。

同类型题或类似算法套路：此类题目属于模拟与枚举类型，常涉及在一定范围内按特定规则操作对象，通过枚举所有可能情况判断条件是否满足。解决这类问题关键是合理选择数据结构存储信息，优化枚举顺序减少计算量。

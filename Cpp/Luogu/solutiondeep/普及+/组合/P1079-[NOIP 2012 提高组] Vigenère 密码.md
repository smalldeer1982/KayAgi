# [NOIP 2012 提高组] Vigenère 密码

## 题目描述

16 世纪法国外交家 Blaise de Vigenère 设计了一种多表密码加密算法 Vigenère 密码。Vigenère 密码的加密解密算法简单易用，且破译难度比较高，曾在美国南北战争中为南军所广泛使用。

 
在密码学中，我们称需要加密的信息为明文，用 $M$ 表示；称加密后的信息为密文，用 $C$ 表示；而密钥是一种参数，是将明文转换为密文或将密文转换为明文的算法中输入的数据，记为 $k$。在 Vigenère 密码中，密钥 $k$ 是一个字母串，$k=k_1,k_2,…,k_n$。当明文 $M=m_1,m_2,…,m_n$ 时，得到的密文 $C=c_1,c_2,…,c_n$，其中 $c_i$=$m_i \operatorname{\circledR} k_i$，运算 $\circledR$ 的规则如下表所示：

Vigenère 加密在操作时需要注意：

1. $\circledR$ 运算忽略参与运算的字母的大小写，并保持字母在明文 $M$ 中的大小写形式；
2. 当明文 $M$ 的长度大于密钥 $k$ 的长度时，将密钥 $k$ 重复使用。

例如，明文 $M=\texttt{Helloworld}$，密钥 $k=\texttt{abc}$ 时，密文 $C=\texttt{Hfnlpyosnd}$。

\textsf{密文} & \tt H & \tt f & \tt n & \tt l & \tt p & \tt y & \tt o & \tt s & \tt n & \tt d \\  \hline
\end{array}
$$


## 说明/提示

# AI分析结果

### 综合分析与结论
这些题解的核心都是根据Vigenère密码的规则，通过密钥对密文进行解密得到明文。主要思路围绕对密码表规律的挖掘，进而得出解密公式。各题解的算法要点和难点解决方式如下：
 - **ingu题解**：利用ASCII码特性，通过按位与操作获取字母在字母表顺序，计算偏移量进行解密，简洁高效。
 - **huluobotou题解**：观察密码表斜行规律，将斜行加和存入数组辅助解密，思路独特但实现稍繁琐。
 - **LightCoder题解**：采用Queue打表模拟密码表生成，再根据密文和密钥的大小写查表解密，直观但代码量较大。
 - **Sinwind题解**：分析密码表得出解密公式，将字符统一大写处理，逻辑清晰。
 - **qhr2023题解**：发现密文、明文和密钥对应字母和对26取模的规律，通过取模运算解决密钥循环使用问题，代码简洁。
 - **ShirasuAzusaSaikou题解**：推导解密公式，通过判断密文大小写确定明文起始字母，利用取模运算处理密钥循环。
 - **Kawaii_qiuw题解**：观察密码表得出加密解密的取模关系，通过取模运算处理密文和密钥位数对应问题。

```cpp
#include <iostream>
#include <string>
#include <cctype>

using namespace std;

string key_text;	//密匙
string cipher_text;	//密文
int key_len;		//密匙长度
int cipher_len;     //密文长度

//c是密文，k是密匙，返回解密后的明文m
char decryption(char c, char k)
{
    char temp_c = toupper(c);
    char temp_k = toupper(k);
    //从A/a算起
    char m =(isupper(c))? 'A' : 'a';
    //A/a后第几个
    m += (temp_c >= temp_k)? (temp_c - temp_k) : (temp_c + 26 - temp_k);

    return m;
}

int main(void)
{
    cin >> key_text;
    cin >> cipher_text;

    key_len = key_text.length();
    cipher_len = cipher_text.length();

    for(int i = 0; i < cipher_len; i++)
    {
        cout << decryption(cipher_text[i], key_text[i % key_len]);
    }

    return 0;
}
```

- **qhr2023题解（4星）**

        return (c-k+26)%26+'a';
    else if (k>='a'&&c<='Z')

        return (c-k+26)%26+'A';

}

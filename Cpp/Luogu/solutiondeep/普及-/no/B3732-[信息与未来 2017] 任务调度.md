# 题目信息

# [信息与未来 2017] 任务调度

## 题目描述

乌龟因为动作太慢，有 $n$ 个任务已经超过截止日期
了。乌龟处理第 $i$ 个任务需要 $a_i$ 单位时间。从 $0$ 时刻开始，乌龟可以选择某项任务，完成它，然后再开始另一项任务，如此往复直到所有任务都被完成。

由于已经超过截止日期，乌龟会为此受到一定的惩罚，惩罚值等于所有任务完成时刻之和。例如，有 2 个
任务分别需要 $10$ 和 $20$ 单位时间完成。如果先完成任务 1，惩罚值为 $10+30=40$；如果先完成任务 2，惩罚值为 $20+30=50$。

乌龟希望你求出惩罚值最小的完成任务的顺序。

---

试题中使用的生成数列 $R$ 定义如下：整数 $0\leq R_1\lt 201701$ 在输入中给出。

对于 $i\gt 1,R_i=(R_{i−1}\times 6807+2831)\mod 201701$。

## 说明/提示

对于 $100\%$ 的数据，$1\leq n\leq10^3$。

>本题原始满分为 $15\text{pts}$。

### 综合分析与结论
本题的核心问题是如何安排任务的顺序，使得所有任务的完成时刻之和最小。这是一个典型的贪心算法问题，通过合理的排序策略可以找到最优解。具体来说，应该按照任务的处理时间从小到大进行排序，这样可以使得短任务先完成，减少长任务的等待时间，从而最小化总惩罚值。

### 关键代码实现
```cpp
#include <iostream>
#include <vector>
#include <algorithm>

using namespace std;

int main() {
    int n, R1;
    cin >> n >> R1;
    vector<int> a(n);
    a[0] = R1;
    for (int i = 1; i < n; ++i) {
        a[i] = (a[i-1] * 6807 + 2831) % 201701;
    }
    sort(a.begin(), a.end());
    long long total_time = 0, penalty = 0;
    for (int i = 0; i < n; ++i) {
        total_time += a[i];
        penalty += total_time;
    }
    cout << penalty << endl;
    return 0;
}
```
**核心实现思想**：
1. **生成任务时间**：根据给定的生成规则生成每个任务的处理时间。
2. **排序**：将任务按处理时间从小到大排序。
3. **计算惩罚值**：依次累加任务的处理时间，并计算总惩罚值。

通过这种贪心策略，可以有效地最小化总惩罚值。

# 题目信息

# 【深基4.习9】打分

## 题目描述

现在有 $n(n \le 1000)$ 位评委给选手打分，分值从 $0$ 到 $10$。需要去掉一个最高分，去掉一个最低分（如果有多个最高或者最低分，也只需要去掉一个），剩下的评分的平均数就是这位选手的得分。现在输入评委人数和他们的打分，请输出选手的最后得分，精确到 $2$ 位小数。 

## 说明/提示

数据保证，$3 \leq n \leq 1000$，每个评委打出的分值为为 $0$ 到 $10$（含 $0$ 与 $10$）之间的整数。

## 样例 #1

### 输入

```
5
9 5 6 8 9```

### 输出

```
7.67```

### 综合分析与结论
本题的核心逻辑是通过模拟评委打分的过程，去掉一个最高分和一个最低分，然后计算剩余分数的平均值。所有题解都采用了类似的方法，主要区别在于如何找到最大值和最小值以及如何计算平均值。以下是各题解的要点对比：

1. **Lithium_Chestnut** 的题解通过排序后直接跳过第一个和最后一个元素来找到最小值和最大值，然后计算剩余元素的和。这种方法简单直接，但排序的时间复杂度为O(n log n)，对于本题的数据范围来说是可以接受的。
2. **C171501** 和 **AveMarina** 的题解在输入时动态更新最大值和最小值，并累加所有分数的总和，最后减去最大值和最小值。这种方法的时间复杂度为O(n)，更为高效。
3. **Lylighte** 的题解提供了两种方法，一种是动态更新最大值和最小值，另一种是排序后跳过首尾元素。这两种方法都适用于本题，且代码清晰易读。
4. **FR_qwq** 的题解虽然使用了Pascal语言，但思路与其他题解一致，动态更新最大值和最小值，并计算剩余分数的平均值。

### 所选高星题解
1. **Lithium_Chestnut** (5星)
   - 关键亮点：通过排序直接找到最小值和最大值，代码简洁明了。
   - 核心代码：
     ```cpp
     sort(a+1,a+n+1);
     for(int i=2;i<=n-1;i++) ans+=a[i];
     printf("%.2lf",ans/(n-2));
     ```

2. **C171501** (5星)
   - 关键亮点：在输入时动态更新最大值和最小值，时间复杂度更低。
   - 核心代码：
     ```cpp
     maxn=max(maxn,m);
     minn=min(minn,m);
     sum+=m;
     sum-=maxn;
     sum-=minn;
     printf("%.2f",sum/(n-2));
     ```

3. **Lylighte** (5星)
   - 关键亮点：提供了两种方法，代码清晰且注释详细。
   - 核心代码：
     ```cpp
     tot = maxn = minn = score[1];
     for(int i=2; i<=n; i++){
         if(score[i]>maxn) maxn = score[i];
         if(score[i]<minn) minn = score[i];
         tot += score[i];
     }
     cout << fixed << setprecision(2) << double(tot-maxn-minn)/(n-2) << endl;
     ```

### 最优关键思路或技巧
- **动态更新最大值和最小值**：在输入时动态更新最大值和最小值，可以避免排序的开销，时间复杂度为O(n)。
- **排序后跳过首尾元素**：通过排序后直接跳过第一个和最后一个元素来找到最小值和最大值，代码简洁，适用于数据量较小的情况。

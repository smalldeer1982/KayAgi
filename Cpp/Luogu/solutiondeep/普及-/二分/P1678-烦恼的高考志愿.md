# 题目信息

# 烦恼的高考志愿

## 题目背景

计算机竞赛小组的神牛 V 神终于结束了高考，然而作为班长的他还不能闲下来，班主任老 t 给了他一个艰巨的任务：帮同学找出最合理的大学填报方案。可是 v 神太忙了，身后还有一群小姑娘等着和他约会，于是他想到了同为计算机竞赛小组的你，请你帮他完成这个艰巨的任务。


## 题目描述

现有 $m$ 所学校，每所学校预计分数线是 $a_i$。有 $n$ 位学生，估分分别为 $b_i$。

根据 $n$ 位学生的估分情况，分别给每位学生推荐一所学校，要求学校的预计分数线和学生的估分相差最小（可高可低，毕竟是估分嘛），这个最小值为不满意度。求所有学生不满意度和的最小值。


## 说明/提示

数据范围：

对于 $30\%$ 的数据，$1\leq n,m\leq1000$，估分和录取线 $\leq10000$；

对于 $100\%$ 的数据，$1\leq n,m\leq100000$，估分和录取线 $\leq 1000000$ 且均为非负整数。


## 样例 #1

### 输入

```
4 3
513 598 567 689
500 600 550
```

### 输出

```
32```

# AI分析结果



### 综合分析与结论
本问题核心是快速为每个学生找到最接近的学校分数线，典型解法为排序后二分查找。所有题解中，基于二分法的实现（包括手写二分与STL的lower_bound）是时间效率最高、实现最简洁的方案。其他方法如双指针贪心、混合排序等存在实现复杂度或正确性风险，而平衡树等数据结构解法在本题场景下显得冗余。

### 高评分题解推荐

#### 1. 作者：Viston（5星）
**关键亮点**：  
- 使用STL的lower_bound大幅简化二分查找实现  
- 通过三次边界判断（完全小于、完全大于、中间值）确保正确性  
- 代码简洁，可读性极佳  

**核心代码**：
```cpp
sort(c+1,c+a+1); // 学校分数排序
e = lower_bound(c+1,c+a+1,d) - c; // 查找第一个>=d的位置
ans += min(abs(c[e]-d), abs(d-c[e-1])); // 比较相邻两个元素
```

#### 2. 作者：06ray（4星）
**关键亮点**：  
- 手写二分模板清晰展示查找过程  
- 通过特判处理极端情况（学生分低于所有学校）  
- 变量命名规范，逻辑易于跟踪  

**核心实现**：
```cpp
sort(a+1,a+n+1); // 学校分数排序
while(l < r) { // 二分查找右边界
    mid = (l+r)/2;
    if(a[mid] <= b[i]) l = mid+1;
    else r = mid; 
}
ans += min(abs(a[l-1]-b[i]), abs(a[l]-b[i])); // 比较左右临近值
```

#### 3. 作者：AK_黄铮_IMO（4星）
**关键亮点**：  
- 使用双指针维护当前最优解  
- 预处理排序后线性扫描，理论时间复杂度更优（O(n+m)）  
- 通过绝对差值比较动态调整指针  

**代码要点**：
```cpp
sort(a, a+m); sort(b, b+n); // 双排序
int k = 0;
for(int i=0; i<n; i++) {
    while(k < m && abs(a[k]-b[i]) <= abs(a[k+1]-b[i])) 
        k++;
    sum += abs(a[k]-b[i]);
}
```

### 最优技巧总结
**核心思路**：  
1. **排序预处理**：将学校分数排序后，即可使用二分法快速定位  
2. **边界值比较**：对每个学生分数，只需比较二分点前后的两个学校分数  
3. **STL工具运用**：直接使用lower_bound可避免手写二分错误  

**拓展方向**：  
- 类似问题：寻找最近元素（如LeetCode 658. Find K Closest Elements）  
- 变形场景：带权值的最小差值（如考虑学校剩余名额权重）

# 题目信息

# [语言月赛 202307] 扶苏和串

## 题目背景

众所周知，每个月入门赛的字符串题都是扶苏来枚举 idea 出出来的。

## 题目描述

给定一个 01 字符串 $s$，你可以任选 $s$ 的一个非空子串，把这个子串在 $s$ 中**翻转**一次。

问你能得到字典序最小的字符串是什么？

形式化的，你可以选择一个区间 $[l, r]$ 满足 $1 \leq l \leq r \leq |s|$，构造一个串 $t$ 满足：

$$t_i = \begin{cases}s_i, &i < l \text{ 或 } i > r \\ s_{r - (i - l)}, & l \leq i \leq r\end{cases}$$

这里字符串的下标从 $1$ 开始。

最小化字符串 $t$ 的字典序。

## 说明/提示

### 样例 1 解释

$s = \texttt{\underline{10}1}$，翻转下划线标出的子串，得到 $t = \texttt{011}$

### 样例 2 解释

$s = \texttt{00\underline{10100}}$，翻转下划线标出的子串，得到 $\texttt{0000101}$。

### 数据规模与约定

下面用 $|s|$ 表示输入字符串的长度。

- 对 $20\%$ 的数据，$|s| \leq 2$。
- 对 $40\%$ 的数据，$|s| \leq 8$。
- 另有 $10\%$ 的数据，$s$ 只含字符 $\texttt 1$。
- 另有 $10\%$ 的数据，$s$ 只含字符 $\texttt 0$。
- 对 $100\%$ 的数据，$1 \leq |s| \leq 100$。$s$ 只含字符 $\texttt{0,1}$。

# AI分析结果

### 综合分析与结论

本题要求通过翻转一个子串，使得字符串的字典序最小。题解主要采用枚举子串的左右端点并翻转的方法，寻找最小字典序的字符串。虽然该方法在时间复杂度上较高（O(n^2)），但由于字符串长度限制在100以内，该方法在时间和空间上都是可行的。

### 所选高星题解

#### 题解1：一扶苏一（4星）
- **关键亮点**：思路清晰，代码简洁，直接枚举所有可能的子串翻转情况，并通过字符串拼接和比较找到最小字典序的字符串。
- **代码实现**：
  ```cpp
  for (int l = 0; l < s.length(); ++l) {
    for (int r = l + 1; r < s.length(); ++r) {
      string x = s.substr(0, l);
      string y = s.substr(l, r - l + 1);
      string z = s.substr(r + 1);
      reverse(y.begin(), y.end());
      string tmp = x + y + z;
      ans = min(tmp, ans);
    }
  }
  ```
- **个人心得**：作者提到使用 `s.substr` 和 `reverse` 函数简化了代码实现，使得逻辑更加清晰。

### 最优关键思路或技巧

1. **枚举所有可能的子串**：通过双重循环枚举所有可能的子串左右端点，确保不遗漏任何可能的翻转情况。
2. **字符串翻转与拼接**：使用 `reverse` 函数翻转子串，并通过字符串拼接生成新的字符串，便于比较字典序。
3. **最小字典序比较**：通过 `min` 函数直接比较字符串的字典序，简化了比较逻辑。

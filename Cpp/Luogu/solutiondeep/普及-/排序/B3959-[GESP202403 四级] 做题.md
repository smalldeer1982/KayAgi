# 题目信息

# [GESP202403 四级] 做题

## 题目描述

小杨同学为了提高自己的实力制定了做题计划，在第 $k$ 天时，他必须要完成 $k$ 道题，否则他就会偷懒。

小杨同学现在找到了一个题库，一共有 $n$ 套题单，每一套题单中有一定数量的题目。但是他十分挑剔，每套题单他只会使用一次，每一天也只能使用一套题单里的题目，之后那套题单就会弃之不用。对于每套题单，他不必完成题单内所有的题。

那么问题来了，小杨同学最多做题几天才偷懒呢？

## 说明/提示

### 数据规模与约定
对全部的测试数据，保证 $1 \leq n \leq 10^6$，$1 \leq a_i \leq 10^9$。

## 样例 #1

### 输入

```
4
3 1 4 1
```

### 输出

```
3```

# AI分析结果

### 综合分析与结论
这些题解均围绕贪心算法解决小杨做题天数的问题。核心思路是先对题单按题目数量从小到大排序，再依次检查题单，若题单题目数大于等于当前天数，则天数加一。部分题解使用双重循环或优先队列，但本质相同。

const int N=1e6+19;
int a[N];

}

#include<iostream>
#include<cstdio>
#include<algorithm>
using namespace std;
int a[1000010];
int main() {
    int n;
    scanf("%d",&n);
    for (int i = 0; i < n; ++i) {
        cin >> a[i];
    }
    sort(a, a + n);
    int days = 0;
    for (int i = 0; i < n; ++i) {

            ++days;
        }
    }
    printf("%d",days);

}
```

### 最优关键思路或技巧
- **贪心策略**：优先选择题目数量少的题单，使小杨能在较早时间完成，保证后续有更多题单可用。
- **排序预处理**：对题单按题目数量从小到大排序，方便后续查找满足条件的题单。

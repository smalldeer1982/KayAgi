## 题目描述

小 A “刷题”十分猖狂，明目张胆地“刷题”。他现在在小书童里发现了 $n$ 样他喜欢的“题目”，每“题”都有他的需要时间，而老师布置了 $m$ 项作业，每项作业都有它的需要时间及分值，老师规定 $k$ 分以上算及格。小 A 只剩 $r$ 个单位时间，他想在及格的基础上更多地“刷题”。

## 说明/提示

### 数据范围及约定


对于 $100\%$ 的数据，$n\le 10$，$m\le 10$，$k\le 50$，$r\le 150$。数据保证没有不能及格的情况。

# AI分析结果



### 综合分析与结论
各题解核心思路均为01背包处理作业（求及格所需最短时间）+ 贪心处理刷题（按时间升序刷题）。差异主要体现在背包维度设计、及格时间查找方式及代码可读性。最优解需满足两个条件：保证及格的最短作业时间 + 剩余时间的最优分配。

**个人心得**  
"从前往后扫一次，一旦及格就停止" 体现问题转化思维，将复杂约束简化为顺序查找  
**核心代码**  
```cpp
// 01背包处理作业
for(int j=r;j>=w[i];j--)
    f[j] = max(f[j],f[j-w[i]] + c[i]);

// 找最短及格时间
for(int i=1;i<=r;i++)
    if(f[i] >= k) { stt = r - i; break; }

// 贪心刷题
sort(a+1,a+n+1);
for(int i=1; i<=n; i++) {
    stt -= a[i];
    if(stt <= 0) break;
    stn++;
}
```

### 最优技巧总结
1. **维度转换思维**：将作业处理转化为01背包问题，根据需求选择背包维度（时间→分数或分数→时间）
2. **贪心策略前置**：对题目时间预先排序，确保后续线性扫描即为最优解  
3. **双阶段处理**：明确分离及格约束与刷题优化两个阶段，降低问题复杂度  
4. **剪枝优化**：在及格扫描时使用break提前终止，避免无效计算

# 题目信息

# zzc 种田

## 题目背景

可能以后 zzc 就去种田了。


## 题目描述

田地是一个巨大的矩形，然而 zzc 每次只能种一个正方形,而每种一个正方形时 zzc 所花的体力值是正方形的周长，种过的田不可以再种，zzc 很懒还要节约体力去泡妹子，想花最少的体力值去种完这块田地，问最小体力值。

## 说明/提示

$1\le x,y\le 10^{16}$。

## 样例 #1

### 输入

```
1 10```

### 输出

```
40```

## 样例 #2

### 输入

```
2 2 ```

### 输出

```
8```

# AI分析结果



### 综合分析与结论
本题核心是贪心策略结合数论优化，通过类似GCD的辗转相除法，每次切割最大可能正方形块。关键难点在于处理1e16量级数据时的时间复杂度控制，必须通过取模运算而非逐次相减。最优解法时间复杂度为O(log(min(x,y)))，与GCD算法同阶。

---

### 高分题解推荐
#### 1. 北海_Beihai（★★★★★）
**核心亮点**：  
- 巧妙借鉴GCD算法框架，通过swap与取模快速缩小问题规模  
- 代码极简（仅7行），时间复杂度最优  
- 数学建模能力强，将种田问题转化为数论问题  

**关键代码**：
```cpp
while(x&&y){
    swap(x,y);
    ans += 4*y*(x/y);
    x %= y;
}
```

#### 2. Zerosking（★★★★☆）
**核心亮点**：  
- 递归解法的优化范例，通过取模避免递归深度爆炸  
- 详细记录调试过程（MLE问题与优化思路）  
- 完整呈现从基础递归到优化的思考路径  

**调试经验**：  
> "当数据为1e10和1时，递归调用次数高达1e10次导致爆栈。通过取模运算一次性处理多个正方形块，将调用次数降为O(log n)级。"

**关键优化**：

#### 3. 人间凡人（★★★★☆）
**核心亮点**：

- 对比基础解法和优化解法的差异

**核心逻辑**：
```cpp
while(x!=0 && y!=0){
    z = min(x,y);
    ans += 4*z*(max(x,y)/z); // 一次处理多个正方形
    max(x,y) %= z;
}
```

---

### 最优解题框架
**关键思路**：  
1. **贪心选择**：每次以短边为正方形边长  
2. **数论优化**：用取模替代逐次相减，计算 `max(x,y)/min(x,y)` 个正方形  
3. **循环终止**：当任一维度归零时结束  

**通用模板**：
```cpp
long long x,y,ans=0;
while(x && y){
    ans += 4 * min(x,y) * (max(x,y)/min(x,y));
    max(x,y) %= min(x,y);
}
// 当x或y为0时，ans即为答案
```

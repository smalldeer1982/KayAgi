# 题目信息

# L 国的战斗之间谍

## 题目背景

L 国即将与 I 国发动战争！！


## 题目描述

俗话说的好：“知己知彼，百战不殆”。L 国的指挥官想派出间谍前往I国，于是，选人工作就落到了你身上。

你现在有 $N$ 个人选，每个人都有这样一些数据：$A$（能得到多少资料）、$B$（伪装能力有多差）、$C$（要多少工资）。已知敌人的探查间谍能力为 $M$（即去的所有人 $B$ 的和要小于等于 $M$）和手头有 $X$ 元钱，请问能拿到多少资料？


## 说明/提示

数据范围：$1\le n\le 100$，$1\le m\le 1000$，$1\leq x\leq 1000$。

## 样例 #1

### 输入

```
3 10 12
10 1 11
1 9 1
7 10 12
```

### 输出

```
11```

# AI分析结果

### 综合分析与结论
本题是典型的二维 0 - 1 背包问题，限制条件为伪装能力总和不超过 $M$ 且费用不超过 $X$，目标是获取最多资料。题解主要分为动态规划和深度优先搜索（DFS）两类。

动态规划类题解思路清晰，通过状态转移方程 `f[j][k]=max(f[j][k],f[j - b[i]][k - c[i]] + a[i])` 求解，部分题解还进行了空间压缩优化。深度优先搜索类题解通过递归枚举所有可能的选人组合，但在数据规模较大时可能超时，部分题解加入剪枝优化提高效率。

    - **核心代码**：
```cpp
#include<iostream>
#include<algorithm>
using namespace std;

int x,y,z;
int f[1005][1005];
int main(){
    int n,m,p;
    cin>>n>>m>>p;
    for(int i=1;i<=n;i++){
        cin>>x>>y>>z;
        for(int j=m;j>=y;j--)
            for(int k=p;k>=z;k--)
                f[j][k]=max(f[j][k],f[j-y][k-z]+x);
    }
    cout<<f[m][p];
    return 0;
} 
```
核心思想：使用二维数组 `f[j][k]` 表示伪装能力至多为 $j$ 且总费用为 $k$ 时的最大获取资料量，通过三重循环遍历更新状态。

#include<bits/stdc++.h>
using namespace std;
int f[1010][1010];
int main(){
    int n,m,x;
    cin>>n>>m>>x;
    for(int i=1;i<=n;i++){
        int a,b,c;
        cin>>a>>b>>c;
        for(int j=m;j>=b;j--)

}
```
核心思想：同样使用二维数组 `f[j][k]` 记录状态，通过三重循环更新，最后输出 `f[m][x]` 得到结果。

### 最优关键思路或技巧
- **动态规划**：使用二维数组记录状态，通过状态转移方程更新，避免重复计算。
- **空间压缩**：在动态规划中，若当前状态只依赖于上一层状态，可将多维数组压缩为低维数组，减少空间开销。
- **深度优先搜索剪枝**：在 DFS 中，当当前状态不满足条件（如伪装能力或费用超过限制）时，直接返回，避免无效搜索。

### 可拓展之处
同类型题如多维背包问题，有多个限制条件，可通过增加维度的方式扩展状态转移方程。类似算法套路还有分组背包、完全背包等，可根据物品选取规则的不同进行调整。

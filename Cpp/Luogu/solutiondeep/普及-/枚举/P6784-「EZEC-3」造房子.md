# 题目信息

# 「EZEC-3」造房子

## 题目背景

pigstd 要去造房子了。

## 题目描述

pigstd 有 $a$ 个 A 材料和 $b$ 个 B 材料，造**第** $i$ 层楼需要 $i$ 个 A 材料与 $i$ 个 B 材料。

但是 pigstd 觉得房子不够高，于是他拿出了 $c$ 块钱，每块钱都可以用来买 $1$ 个 A 材料或者 $1$ 个 B 材料。

现在 pigstd 想知道，他最多能建多少层楼的房子。

## 说明/提示

**【样例 1 说明】**

pigstd 买 $2$ 个 A 材料和 $1$ 个 B 材料后就有 $3$ 个 A 材料和 $3$ 个 B 材料，最多可以建 $2$ 层楼的房子。

（花费 $1+2$ 个 A 材料和 $1+2$ 个 B 材料）

**【样例 2 说明】**

pigstd 买 $3$ 个 A 材料后就有 $4$ 个 A 材料和 $5$ 个 B 材料，最多可以建 $2$ 层楼的房子。

（花费 $1+2$ 个 A 材料和 $1+2$ 个 B 材料）

**【数据规模与约定】**

对于 $100\%$ 的数据，$0 \le a,b,c \le 10^{12}$。

| 数据点编号 | 满足的条件 |
| :----------: | :----------: |
|$1-4,22$| $c = 0$ |
|$5-8,21$| $a,b,c \le 10^{3}$ |
|$9-12$| $a,b,c \le 10^{6}$ |
|$13-20$| $a,b,c \le 10^{12}$ | 

## 样例 #1

### 输入

```
1 2 3```

### 输出

```
2```

## 样例 #2

### 输入

```
1 5 3```

### 输出

```
2```

### 综合分析与结论
这些题解核心思路都是先合理分配 $c$ 使 $a$ 和 $b$ 尽可能接近，再据此计算能建的最大楼层数。不同题解在计算最大楼层数时采用了不同方法，复杂度有所差异。
- **思路对比**：多数题解先对 $a$、$b$、$c$ 进行分类讨论来分配 $c$，之后通过枚举、二分或数学公式计算最大楼层数。
- **算法要点**：
    - **枚举**：顺序枚举楼层数，直到不满足条件。
    - **二分**：利用楼层数与所需材料数的单调性进行二分查找。
    - **数学公式**：通过解一元二次不等式得到最大楼层数的计算公式。
- **解决难点**：
    - **材料分配**：合理分配 $c$ 使 $a$ 和 $b$ 接近，多数题解采用分类讨论的方法。
    - **楼层数计算**：不同复杂度的计算方法各有优劣，数学公式法复杂度最低。

#include<algorithm>

#define LL long long

LL a,b,c;
int main(void){
    scanf("%lld%lld%lld",&a,&b,&c);
    LL x=max(a,b),y=min(a,b);
    LL m=x-y;
    if(m>c){
        LL ans=(LL)(((LL)(floor(sqrt(8*(y+c)+1)))-1)/2);

}

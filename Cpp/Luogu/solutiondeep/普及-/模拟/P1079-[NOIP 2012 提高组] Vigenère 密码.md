# [NOIP 2012 提高组] Vigenère 密码

## 题目描述

16 世纪法国外交家 Blaise de Vigenère 设计了一种多表密码加密算法 Vigenère 密码。Vigenère 密码的加密解密算法简单易用，且破译难度比较高，曾在美国南北战争中为南军所广泛使用。

 
在密码学中，我们称需要加密的信息为明文，用 $M$ 表示；称加密后的信息为密文，用 $C$ 表示；而密钥是一种参数，是将明文转换为密文或将密文转换为明文的算法中输入的数据，记为 $k$。在 Vigenère 密码中，密钥 $k$ 是一个字母串，$k=k_1,k_2,…,k_n$。当明文 $M=m_1,m_2,…,m_n$ 时，得到的密文 $C=c_1,c_2,…,c_n$，其中 $c_i$=$m_i \operatorname{\circledR} k_i$，运算 $\circledR$ 的规则如下表所示：

Vigenère 加密在操作时需要注意：

1. $\circledR$ 运算忽略参与运算的字母的大小写，并保持字母在明文 $M$ 中的大小写形式；
2. 当明文 $M$ 的长度大于密钥 $k$ 的长度时，将密钥 $k$ 重复使用。

例如，明文 $M=\texttt{Helloworld}$，密钥 $k=\texttt{abc}$ 时，密文 $C=\texttt{Hfnlpyosnd}$。

### 综合分析与结论
这些题解主要围绕Vigenère密码的解密算法展开，思路核心均是依据给定的密文和密钥，通过逆向操作得到明文。
- **思路**：多数题解通过分析密码表规律，得出解密公式。如通过将字母映射为数字，发现 (明文数字 + 密钥数字) % 26 = 密文数字，从而逆推得到明文。部分题解利用ASCII码特性简化计算，也有通过打表、模拟队列操作来解析密码表。
- **算法要点**：处理密钥循环使用，通常采用取模运算；处理字母大小写，通过判断ASCII码值实现。
- **解决难点**：主要难点在于准确解析密码表规律并正确实现逆向解密，同时处理好密钥循环及字母大小写问题。

多数题解思路清晰，但部分代码可读性欠佳，存在变量命名不规范、代码冗余等问题。综合质量来看，以下题解相对较好：
1. **ingu**：利用ASCII码特性简化计算，代码简洁，思路清晰。
2. **Sinwind**：逻辑清晰，将解密过程封装为函数，提高代码可读性。
3. **3_14**：代码结构清晰，通过函数实现字符与数字转换及解密过程，易于理解。

// 将字符转换为0-25的数字
int charToNum(char c) {
    if (c >= 'A' && c <= 'Z') {
        return c - 'A';
    } else if (c >= 'a' && c <= 'z') {
        return c - 'a';
    }
    return 0;
}
// 将数字转换为字符
char numToChar(int num, bool isUpper) {
    if (isUpper) {
        return 'A' + num;
    } else {
        return 'a' + num;
    }
}
string vigenereDecrypt(const string& ciphertext, const string& key) {
    string plaintext;
    int keyLength = key.length();
    int ciphertextLength = ciphertext.length();
    for (int i = 0; i < ciphertextLength; ++i) {
        char c = ciphertext[i];
        bool isUpper = (c >= 'A' && c <= 'Z');
        int cNum = charToNum(c);
        int kNum = charToNum(key[i % keyLength]);
        int mNum = (cNum - kNum + 26) % 26;
        char m = numToChar(mNum, isUpper);

        plaintext.push_back(m);
    }
    return plaintext;
}

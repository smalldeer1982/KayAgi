# 题目信息

# [NOIP 2007 提高组] 统计数字

## 题目描述

某次科研调查时得到了 $n$ 个自然数，每个数均不超过 $1.5 \times 10^9$。已知不相同的数不超过 $10^4$ 个，现在需要统计这些自然数各自出现的次数，并按照自然数从小到大的顺序输出统计结果。


## 说明/提示

- $30\%$ 的数据满足：$1  \le  n  \le  1000$；
- $60\%$ 的数据满足：$1  \le  n  \le  50000$；
- $100\%$ 的数据满足：$1  \le  n  \le  200000$，每个数均不超过 $1.5 \times 10^9$。

NOIP 2007 提高第一题


## 样例 #1

### 输入

```
8
2
4
2
4
5
100
2
100```

### 输出

```
2 3
4 2
5 1
100 2
```

# AI分析结果

### 综合分析与结论
这些题解主要围绕如何统计给定自然数序列中各数出现次数并按序输出的问题，提出了多种思路。由于数据范围大，常规数组计数排序不可行，难点在于选择合适的数据结构和算法。主要分为两类思路：
1. **基于排序的方法**：先将输入的数排序，相同的数会相邻，遍历排序后的序列，统计相邻相同数的个数并输出。此方法时间复杂度主要在排序部分，为$O(nlogn)$。
2. **使用STL容器map和set**：map可作为“超级桶”记录各数出现次数，set用于去重并保持有序，遍历set输出对应map中的次数。map单次修改操作时间复杂度为$O(logn)$，整体时间复杂度也为$O(nlogn)$。

综合质量来看，多数题解思路清晰，但部分代码可读性欠佳，优化程度也有限，整体达到4星及以上的较少。

### 通用建议与扩展思路
对于此类统计问题，若数据范围大，优先考虑特殊数据结构如map、unordered_map等。若要求有序，map更合适；若对顺序无要求且追求高效，unordered_map是更好选择。在代码实现上，注意容器操作的细节，如迭代器使用、元素插入删除等。同时，可考虑优化输入输出，如使用快读快写。

同类型题可考察对不同数据结构的应用，如给定字符序列统计各字符出现次数并按特定顺序输出，或结合其他条件如区间统计等。类似算法套路是根据数据特点选择合适的数据结构，利用其特性解决问题。

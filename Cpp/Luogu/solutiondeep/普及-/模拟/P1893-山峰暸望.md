# 题目信息

# 山峰暸望

## 题目描述

一天，Bessie 在眺望美丽的威斯康星的群山的时候，她突然产生了疑问：哪座山是最宽的？

她决定在地平线上，利用她的新式大量程山峰高度测量仪依次做 $N$（$1 \le N \le 10,000$）次高度测量，每次的高度值为 $H_i$（$1 \le H_i \le 1,000,000,000$）。

一座山定义为一段连续的高度序列，序列中的高度一开始单调上升（或者不变），然后单调下降（或者不变）。举例来说，$2, 3, 3, 5, 4, 4, 1$ 这一段高度序列就是一座山。如果在她的视线范围内有一段单调上升或者单调下降的序列，也算是一座山。注意，两座山之间可能会有部分重叠。

山的宽度定义为在这个山上进行的测量的次数（也就是序列的长度）。例如序列 $2, 3, 3, 5, 4, 4, 1$ 的宽度为 $7$。

请帮 Bessie 找到最宽的山。

```plain
          **********                *****

  bbbbbbbbbbbbbbbbbbbbbbbbbbbb             ddddd          ffffffffff  
                                                             hhhhhhhhh  山的编号
```

山标记为 `a`、`b` 等等。显然，山 `b` 有着最大的宽度，宽度为 $28$。


## 说明/提示

**【样例说明】**

在最宽的山处测量到的高度为 $2, 3, 5, 4, 1$。其他的山包括 $3, 2$ 和 $1, 6$。

---

**【提示】**

如果你知道一座山的最高的部分（也就是山峰），你会发现，找到这座山的宽度是很容易的哦。

# AI分析结果

### 综合分析与结论
这些题解主要围绕寻找最宽山峰展开，思路可分为两类：
1. **枚举山峰法**：枚举每个高度作为山峰，向左右扩展找到山脚，确定山的宽度，如AFOier、Leap_Frog等题解。时间复杂度为O(n^2)，优点是思路和代码简单直接，但数据规模大时效率低。
2. **动态规划法**：通过状态转移方程记录以每个点结尾的单调递增长度和以每个点开始的单调递减长度，进而计算山的宽度，如万弘的题解。时间复杂度为O(n)，效率更高。

多数题解思路清晰，但部分代码可读性欠佳，部分题解虽用暴力法通过但未优化。整体无高质量题解。

### 通用建议与扩展思路
1. **优化算法**：对于类似寻找连续序列特征的问题，若暴力枚举超时，可考虑动态规划等优化方法，通过记录中间状态减少重复计算。
2. **代码实现**：注重代码的可读性和规范性，合理命名变量、添加注释，便于理解和维护。
3. **拓展思路**：此类题目可拓展到更复杂的地形分析，如寻找多个山峰的特定组合，或在不同约束条件下找最长序列。类似算法套路可应用于股票价格趋势分析、信号处理中寻找特定波形等场景。

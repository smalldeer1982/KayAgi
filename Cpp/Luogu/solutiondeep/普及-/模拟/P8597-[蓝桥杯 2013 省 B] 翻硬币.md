# 题目信息

# [蓝桥杯 2013 省 B] 翻硬币

## 题目背景

小明正在玩一个“翻硬币”的游戏。

## 题目描述

桌上放着排成一排的若干硬币。我们用 `*` 表示正面，用 `o` 表示反面（是小写字母，不是零），比如可能情形是 `**oo***oooo`，如果同时翻转左边的两个硬币，则变为 `oooo***oooo`。现在小明的问题是：如果已知了初始状态和要达到的目标状态，每次只能同时翻转相邻的两个硬币，那么对特定的局面，最少要翻动多少次呢？


## 说明/提示

source：蓝桥杯 2013 省 B 组 H 题

## 样例 #1

### 输入

```
**********
o****o****```

### 输出

```
5```

## 样例 #2

### 输入

```
*o**o***o***
*o***o**o***```

### 输出

```
1```

# AI分析结果

### 综合分析与结论
这三道题解都围绕翻硬币问题，采用贪心策略解题。
 - **思路方面**：“sw2022”和“Vct14”思路相近，从前往后比较两个字符串对应字符，不同就翻转相邻硬币，前者翻转相邻两个，后者只翻转后一个（因后续不再使用前一个）。“哈士奇憨憨”则是统计不同段长度总和，认为每个硬币最多翻转两次，不同段需从开始不同处翻转。
 - **算法要点**：均利用贪心思想，基于相邻硬币翻转特点，以局部最优决策达全局最优解。
 - **解决难点**：难点在于证明贪心策略正确性，“哈士奇憨憨”给出了简要证明，说明不同段需翻转及中间相同硬币需回翻的原因。同时要理解相邻硬币翻转两次等效于不翻，从而避免重复无效操作。

综合来看，“sw2022”和“Vct14”的思路更直接易懂，代码实现简洁清晰，“哈士奇憨憨”的思路虽独特但理解难度稍高。

```cpp
#include<bits/stdc++.h>
using namespace std;
string a,b;
int main()
{
    cin>>a>>b;
    int l=a.size(),s=0,i;
    for(i=0;i<l;i++)
        if(a[i]!=b[i])
        {
            a[i]=a[i]=='o'?'*':'o';
            a[i+1]=a[i+1]=='o'?'*':'o';
            s++;
        }
    printf("%d\n",s);
    return 0;
}
```
    - **核心实现思想**：遍历两个字符串，发现对应字符不同时，翻转当前字符及下一个字符，并记录翻转次数，最后输出总翻转次数。

 - **作者：Vct14（4星）**
    - **关键亮点**：在贪心策略基础上，优化了翻转操作，仅翻转下一枚硬币，减少操作次数，代码简洁明了。
    - **核心代码**：
```cpp
#include<bits/stdc++.h>
using namespace std;

char a[1005];
char b[1005];

int main(){
    cin>>a>>b;
    int len=strlen(a);
    int i=0,sum=0;
    while(i<len){
        if(a[i]!=b[i]){
            a[i+1]=(a[i+1]=='*'?'o':'*');
            sum++;
        }
        i++;
    }
    cout<<sum;
    return 0;
}
```
    - **核心实现思想**：遍历字符串，若当前字符与目标字符串对应字符不同，只翻转下一枚硬币并累加翻转次数，遍历结束输出总次数。

### 最优关键思路或技巧
 - **贪心策略**：利用相邻硬币翻转两次等效于不翻的特性，从前往后比较，发现不同就翻转相邻硬币，每次决策保证局部最优，最终达全局最优。

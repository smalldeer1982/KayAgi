# 题目信息

# 生日

## 题目描述

cjf 君想调查学校 OI 组每个同学的生日，并按照年龄从大到小的顺序排序。但 cjf 君最近作业很多，没有时间，所以请你帮她排序。


## 说明/提示

数据保证，$1<n<100$，$1\leq |s|<20$。保证年月日实际存在，且年份 $\in [1960,2020]$。

## 样例 #1

### 输入

```
3
Yangchu 1992 4 23
Qiujingya 1993 10 13
Luowen 1991 8 1
```

### 输出

```
Luowen
Yangchu
Qiujingya
```

# AI分析结果

### 综合分析与结论

本题的核心是对多个学生的生日进行排序，要求按照年龄从大到小输出。难点在于多关键字排序（年、月、日）以及处理生日相同时的输入顺序问题。大多数题解采用了结构体存储学生信息，并通过自定义比较函数进行排序。常见的排序方法包括`sort`、`stable_sort`以及手写的冒泡排序等。

题解的质量参差不齐，部分题解思路清晰、代码简洁，而有些则存在冗余或不够优化的地方。以下是对题解的综合分析：

1. **数据结构**：大多数题解使用了结构体来存储学生的姓名、生日等信息，这是处理多关键字排序的常见方式。
2. **排序算法**：大多数题解使用了`sort`函数，部分题解使用了`stable_sort`，少数题解使用了手写的冒泡排序。`sort`和`stable_sort`的时间复杂度为O(nlogn)，而冒泡排序为O(n^2)，在本题数据规模较小的情况下，冒泡排序也能通过，但效率较低。
3. **比较函数**：比较函数的设计是关键，大多数题解通过嵌套的`if-else`语句实现多关键字排序，部分题解使用了更简洁的逻辑表达式。
4. **输入顺序处理**：部分题解在处理生日相同时，通过记录输入顺序来确保后输入的学生先输出。

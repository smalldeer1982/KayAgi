# 题目信息

# 第 k 小整数

## 题目描述

现有 $n$ 个正整数，要求出这 $n$ 个正整数中的第 $k$ 个最小整数（相同大小的整数只计算一次）。


## 说明/提示

$n \leq 10000$，$k \leq 1000$，正整数均小于 $30000$。

## 样例 #1

### 输入

```
10 3
1 3 3 7 2 5 1 2 4 6
```

### 输出

```
3
```

### 综合分析与结论
题目要求从一组正整数中找出第k小的整数，且相同大小的整数只计算一次。核心思路是通过排序和去重来解决问题。大部分题解采用了排序（如快排、桶排）和去重（如使用`unique`函数或手动去重）的方法，部分题解还使用了更高级的数据结构（如平衡树、线段树）来优化查询效率。

   - **核心代码：**
     ```cpp
     int search_k(int l, int r) {
         if (l == r && l == k) return a[k];
         int i = l, j = r, p = a[l];
         while (i < j) {
             while (i < j && a[j] > p) j--;
             if (i < j) swap(a[i], a[j]);
             while (i < j && a[i] <= p) i++;
             if (i < j) swap(a[i], a[j]);
         }
         a[i] = p;
         if (i == k) return a[k];
         else if (i > k) return search_k(l, i-1);
         else return search_k(i+1, r);
     }
     ```

### 最优关键思路或技巧
1. **排序与去重结合**：通过排序和去重（如`unique`函数）可以快速找到第k小的数，适合数据量较大的情况。
2. **桶排序**：利用数组标记数字是否出现，适合数据范围较小的情况，时间复杂度为O(n)。
3. **快速选择算法**：借鉴快速排序的思想，通过递归选择区间来找到第k小的数，时间复杂度平均为O(n)。

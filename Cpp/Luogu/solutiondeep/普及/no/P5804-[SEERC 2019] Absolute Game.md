# 题目信息

# [SEERC 2019] Absolute Game

## 题目描述

Alice 和 Bob 在玩一个游戏。Alice 有一个包含 $n$ 个整数的数列 $a$，Bob 有一个包含 $n$ 个整数的数列 $b$。每一回合中，玩家需要从他的数列中删去一个数字。玩家轮流进行回合，Alice 先手。

当两个数列中都只剩下一个数字的时候，游戏结束。令 Alice 的数列剩下的数字为 $x$，Bob 的数列剩下的数字为 $y$。Alice 想要最大化 $x$ 与 $y$ 之差的绝对值，而 Bob 想最小化这个值。两个玩家都以最优策略游戏。

请算出游戏结束时的结果。

## 说明/提示

第一个样例中，$x=14, y=10$，因此两个数之差为 $4$。

第二个样例中，两个数列都只剩下一个数字了，因此 $x=14, y=42$。

## 样例 #1

### 输入

```
4
2 14 7 14
5 10 9 22```

### 输出

```
4```

## 样例 #2

### 输入

```
1
14
42```

### 输出

```
28```

### 题解分析与结论
题目本质是博弈论中的贪心策略问题。双方最优策略下，最终结果等价于：Alice 选择某个数 $a_i$，Bob 必须选择与 $a_i$ 最接近的 $b_j$（使得差最小）。最终答案为所有 $a_i$ 对应的最小差的**最大值**。正确性基于双方策略的对抗性：Alice 会优先保留能产生最大最小差的数，而 Bob 则总能保留对应最优的 $b_j$。

### 高分题解推荐

#### 题解作者：Miraik（5星）
**关键亮点**  
- 通过严谨的数学归纳证明策略正确性  
- 代码使用排序+二分优化至 $O(n \log n)$  
- 代码简洁且高效，处理边界清晰  

**核心代码**  
```cpp
sort(a+1,a+n+1); sort(b+1,b+n+1);
for(int i=1;i<=n;i++){
    int j=lower_bound(b+1,b+n+1,a[i])-b;
    int c=inf;
    if(j<=n) c=min(c,b[j]-a[i]);
    if(j>1) c=min(c,a[i]-b[j-1]);
    ans=max(ans,c);
}
```
**实现思想**  
1. 对两数组排序以便二分查找；  
2. 对每个 $a_i$，用 `lower_bound` 找到插入位置 $j$；  
3. 比较 $b[j]$ 和 $b[j-1]$ 计算最小差；  
4. 取所有最小差中的最大值。

### 最优关键思路
1. **贪心博弈**：双方策略对抗下，结果等价于 Alice 选择最优起点，Bob 被动响应。  
2. **最邻近匹配**：对每个 $a_i$，在有序的 $b$ 中二分查找最邻近元素，计算最小差。  
3. **极值选择**：取所有最小差中的最大值作为最终答案。

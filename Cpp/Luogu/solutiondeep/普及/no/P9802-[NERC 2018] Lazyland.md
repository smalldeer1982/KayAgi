# 题目信息

# [NERC 2018] Lazyland

## 题目背景

翻译自 [NERC 2018](https://neerc.ifmo.ru/archive/2018/neerc-2018-statement.pdf) L 题。

## 题目描述

一个城市里有 $n$ 个人，和 $k$ 种职业，其中，每个人都有一个现在正在从事的职业 $a_i$，但是很遗憾，由于某些职业的空缺，使得这个城市根本无法继续正常生存下去。

你作为一城之主，自然不希望看到自己的城市这样子下去，你决定说服其中的某些人，使得每种职业都有人在做，对于每个人 $i$，你需要耗费 $b_i$ 的时间去说服。

你需要求出你去说服的时间的最小值。

## 说明/提示

对于所有的数据，保证 $1 \leq k \leq n \leq 10^5$，$1 \leq a_i \leq k$，$1 \leq b_i \leq 10^9$。

对于样例一，分别令 $1$，$6$，$8$ 号市民去从事 $2$，$4$，$6$ 号职业。

## 样例 #1

### 输入

```
8 7
1 1 3 1 5 3 7 1
5 7 4 8 1 3 5 2```

### 输出

```
10```

## 样例 #2

### 输入

```
3 3
3 1 2
5 3 4```

### 输出

```
0```

# AI分析结果



**算法分类**: 贪心

**综合分析**  
本题的核心是通过贪心策略，在确保每个职业至少保留一人的前提下，选择说服时间最小的人填补空缺。所有题解均采用贪心思路，差异主要体现在数据结构的选择和实现细节上。

**精选题解**

1. **佬头 (★★★★★)**  
   - **关键亮点**: 使用优先队列动态维护候选池，避免显式排序，代码简洁高效。  
   - **核心思路**:
     - 统计每个职业的最大说服时间，其余加入优先队列。
     - 计算需填补的空缺数，从队列中取最小元素求和。
   ```cpp
   priority_queue<int, vector<int>, greater<int>> q;
   for(int i = 1; i <= n; ++i) {
       if (!maxx[a[i]]) maxx[a[i]] = b, k--;
       else if (maxx[a[i]] < b) q.push(maxx[a[i]]), maxx[a[i]] = b;
       else q.push(b);
   }
   while(k--) ans += q.top(), q.pop();
   ```

2. **mbzdf (★★★★)**  
   - **关键亮点**: 显式处理每个职业的最大值，通过排序候选池取最小值。  
   - **核心思路**:
     - 对每个职业保留最大值，其余加入候选数组。
     - 排序候选数组后取前 $k - A$ 个元素求和。
   ```cpp
   for(int i = 1; i <= k; ++i) {
       if (t[i]) cnt++;
       if (t[i] <= 1) continue;
       // 保留最大值，其余加入候选数组
   }
   sort(c + 1, c + 1 + op);
   for (int i = 1; i <= k - cnt; ++i) ans += c[i];
   ```

3. **UniGravity (★★★★)**  
   - **关键亮点**: 使用 `vector` 存储每个职业的候选，直接排序处理。  
   - **核心思路**:
     - 遍历每个职业，保留最大值，其余加入候选列表。
     - 排序候选列表后取前 $need$ 个元素。

   for (int i = 1; i <= k; i++) {
       if (a[i].size() > 1) {
           int maxa = -1;
           // 保留最大值，其余加入候选
       }
   }
   sort(can + 1, can + cnt + 1);
   ```

**最优关键思路**  
每个职业保留说服时间最大的人，剩余人加入候选池。排序候选池后取最小 $k - A$ 个元素求和（$A$ 是已有职业数）。

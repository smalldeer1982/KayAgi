# 题目信息

# 跑步

## 题目描述

路人甲准备跑 $n$ 圈来锻炼自己的身体，他准备分多次（$\gt1$）跑完，每次都跑正整数圈，然后休息下再继续跑。

为了有效地提高自己的体能，他决定每次跑的圈数都必须比上次跑的多。

可以假设他刚开始跑了 $0$ 圈，那么请问他可以有多少种跑完这 $n$ 圈的方案？

## 说明/提示

#### 数据规模与约定

对于 $100\%$ 的数据，保证 $5\le n\le 500$。

## 样例 #1

### 输入

```
212```

### 输出

```
995645335```

# AI分析结果

### 综合分析与结论

本题的核心是求解将 $n$ 圈分成多次跑步的方案数，且每次跑步的圈数必须严格递增。通过分析，可以将问题转化为**背包问题**或**动态规划问题**。大多数题解都采用了动态规划的思路，但实现方式有所不同，主要分为两类：

1. **二维DP**：使用 $dp[i][j]$ 表示跑 $i$ 圈且最后一次跑 $j$ 圈的方案数。这种方法的转移方程较为直观，但空间复杂度较高。
2. **一维DP**：使用 $dp[i]$ 表示跑 $i$ 圈的方案数，通过优化枚举顺序将空间复杂度降为线性。这种方法更高效，且代码简洁。

最优解通常采用**一维DP**，并结合**01背包**的思想，通过从后向前枚举来避免重复计算。此外，初始化 $dp[0] = 1$ 是关键，确保状态转移的正确性。

### 最优关键思路与技巧
1. **一维DP优化**：通过从后向前枚举，将空间复杂度降为线性，同时避免了重复计算。
2. **01背包思想**：将每次跑步的圈数视为物品，背包容量为总圈数，转化为01背包问题。
3. **初始化技巧**：初始化 $dp[0] = 1$，确保状态转移的正确性。

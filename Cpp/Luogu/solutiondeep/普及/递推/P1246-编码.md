# 编码

## 题目描述

编码工作常被运用于密文或压缩传输。这里我们用一种最简单的编码方式进行编码：把一些有规律的单词编成数字。

字母表中共有 $26$ 个字母 $\mathtt{a,b,c,\cdots,z}$，这些特殊的单词长度不超过 $6$ 且字母按升序排列。把所有这样的单词放在一起，按字典顺序排列，一个单词的编码就对应着它在字典中的位置。

例如：

- $\verb!a! \to 1$；
- $\verb!b! \to 2$；
- $\verb!z! \to 26$；
- $\verb!ab! \to 27$；
- $\verb!ac! \to 28$。

你的任务就是对于所给的单词，求出它的编码。


## 样例 #1

### 输入

```
ab

```

### 输出

```
27

```

### 综合分析与结论

本题的核心在于如何高效地计算给定升序单词的编码。题解中主要分为以下几种思路：

1. **组合数计算**：通过组合数计算比当前单词小的单词数量，逐步累加得到编码。这种方法思路清晰，但实现较为复杂，需要对组合数有较好的理解。
2. **递推表格**：通过构建递推表格，计算每个字母开头、每个长度的单词数量，然后累加得到编码。这种方法较为直观，但需要预处理表格，代码量较大。
3. **暴力枚举**：直接枚举所有可能的单词，逐个比对。这种方法实现简单，但效率较低，适合小规模数据。
4. **DFS生成**：通过DFS生成所有可能的单词，并记录编码。这种方法代码简洁，但时间复杂度较高。

综合来看，**组合数计算**和**递推表格**是较为高效的解法，适合大规模数据处理。而**暴力枚举**和**DFS生成**则适合小规模数据或作为辅助验证手段。

# [yLOI2019] 棠梨煎雪

## 题目背景

> 岁岁花藻檐下共将棠梨煎雪，  
> 自总角至你我某日辗转天边。  
> 天淡天青，宿雨沾襟，  
> 一年一会信笺却只见寥寥数言。

——银临《棠梨煎雪》

## 题目描述

扶苏正在听《棠梨煎雪》的时候，@[spitfirekindergarten](https://www.luogu.org/space/show?uid=61795) 发来一道 [IOI2018 集训队作业题](http://uoj.ac/problem/425)：我切了这集训队题，辣鸡扶苏快过来做题。扶苏定睛一看，这不 s\* 题嘛，写了一发交上去才发现自己看错题目了。但是写完的代码不能浪费，于是就有了这道题。

歌词中的主人公与她的朋友一年会有一次互相写信给对方，一共通信了 $m$ 年。为了简化问题，我们认为她们每封信的内容都是一条二进制码，并且所有二进制码的长度都是 $n$。即每封信的内容都是一个长度为 $n$ 的字符串，这个字符串只含字符 ``0`` 或 ``1``。

这天她拿出了朋友写给她的所有信件，其中第 $i$ 年的写的信件编号为 $i$。由于信件保存时间过久，上面有一些字符已经模糊不清，我们将这样的位置记为 ``?``，``?`` 字符可以被解释为 ``0`` 或 ``1``。由于她的朋友也是人，符合人类的本质，所以朋友在一段连续的时间中书写的内容可能是相同的。现在她想问问你，对于一段连续的年份区间 $[l,r]$ 中的所有信件，假如朋友在这段时间展示了人类的本质，所写的是同一句话，那么这一句话一共有多少种可能的组成。也即一共有多少字符串 $S$，满足在这个区间内的所有信件的内容都可能是 $S$。

一个长度为 $n$ 的只含 ``0,1,?`` 的字符串 $A$ 可能是一个字符串 $B$ 当且仅当 $B$ 满足如下条件：

- $B$ 的长度也是 $n$ 。
- $B$ 中只含字符 ``0,1``。
- $A$ 中所有为 ``0`` 的位置在 $B$ 中也是 ``0``。
- $A$ 中所有为 ``1`` 的位置在 $B$ 中也是 ``1``。
- $A$ 中为 ``?`` 的位置在 $B$ 中可以为 ``0`` 也可以是 ``1``。

同时她可能会突然发现看错了某年的信的内容，于是她可能会把某一年的信的内容修改为一个别的只含 ``0``,``1``,``?`` 的长度为 $n$ 的字符串。

## 说明/提示

### 样例 1 解释

- 对于第一次询问，只有串 ``010`` 符合要求。
- 对于第二次询问，由于第二个串的第一位为 ``0``，第三个串的第一位为 ``1``，故没有串符合要求。
- 修改后将第三个串修改为 ``0??``。
- 对于第四次询问，有两个串符合要求，分别为 ``000`` 和 ``010``。
- 对于第五次询问，只有 ``010`` 符合要求。

故答案为 $1,0,2,1$，他们的异或和再异或 $0$ 的值为 $2$。

## 综合分析与结论  

### 核心思路  
题目要求在动态修改字符串的情况下，快速查询区间内所有字符串的共同合法解数目。关键点在于：  
1. **冲突检测**：若某一位在区间内同时出现必须为0和必须为1的情况，则无解  
2. **自由位统计**：未被确定的位（所有字符串该位均为?）的数量决定解的数目（2^k）  

### 算法实现  
1. **状态压缩**：每个字符串用两个整数的二进制位表示：  
   - `Va`：必须为0的位（1表示必须为0）  
   - `Vb`：必须为1的位（1表示必须为1）  
2. **线段树维护**：每个区间节点存储 `Va |` 和 `Vb |` 的结果  
3. **冲突判断**：合并时检查 `Va & Vb` 是否非零  
4. **自由位计算**：`(~Va & ~Vb)` 的二进制中1的个数即为k  

### 可视化设计  
- **动画演示**：  
  1. 线段树节点显示二进制位的状态（红：冲突位，绿：自由位）  
  2. 合并区间时高亮左右子节点的 `Va/Vb`，动态计算合并结果  
  3. 冲突检测时闪烁红色警示，自由位统计时显示绿色高亮  
- **复古像素风格**：  
  - 用8位像素块表示二进制位（0: 蓝色块，1: 黄色块，?: 灰色块）  
  - 线段树结构以网格形式展开，合并过程伴随经典音效

## 关键代码实现  

### 线段树合并核心逻辑  
```cpp
struct Node {
    unsigned Va, Vb; // 必须为0和必须为1的位
    Node *ls, *rs;
};

void pushup(Node *x) {
    x->Va = x->ls->Va | x->rs->Va; // 合并必须为0的位
    x->Vb = x->ls->Vb | x->rs->Vb; // 合并必须为1的位
}

void query(Node *x, int l, int r) {
    if (覆盖区间) {
        FindA |= x->Va;  // 累计必须为0的位
        FindB |= x->Vb;  // 累计必须为1的位
        return;
    }
    // 递归查询左右子树
}
```

### 冲突检测与计算  
```cpp
unsigned conflict = FindA & FindB;
if (conflict) ans = 0;
else {
    unsigned free_bits = (~FindA & ~FindB) & ((1<<n)-1);
    ans = 1 << __builtin_popcount(free_bits); // 计算2^k
}
```

## 个人心得  
- **位运算优先级**：`&` 和 `^` 的优先级不同，需加括号明确  
- **状态压缩技巧**：用两个整数代替数组存储，大幅减少内存占用  
- **线段树优化**：避免递归过深，指针式实现提升效率  

通过将字符串状态压缩为位运算操作，巧妙地将 O(n) 判断转化为 O(1) 的位运算，是此类区间统计问题的经典优化思路。

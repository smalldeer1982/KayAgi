# 题目信息

# 「Wdsr-1」笨蛋结构

## 题目背景

众所周知，琪露诺是笨蛋。

## 题目描述

琪露诺希望维护一个长度为 $n$ 的整数序列 $a$，初始值都为 $0$。

现在琪露诺想要进行 $q$ 次操作，每次选择序列中的一段区间 $[s,s+l-1]$ 并给出两个数字 $w,k$，使对所有的 $i \in [1,l]$，$a_{s+i-1}$ 加上 $w\times i^k$ 。  

琪露诺不希望 $k$ 很大，因此她给出了一个整数 $m$，满足 $0\le k\le m$。

为了不让头脑简单的琪露诺感到困惑，你只需要输出 依次进行完所有操作后，序列中的每个数字对 $2^{64}$ 取模（即 $\text{unsigned long long}$ 自然溢出）后的结果即可。

为了帮助你更好的理解题意，这里给出一段伪代码：

$$\def\b#1{\textbf{ #1 }}\def\t#1{\text{ #1 }}\def\s{\quad}
\def\l{\underline{\kern{300pt}}\cr[-10pt]}
\def\r{\overline{\underline{\kern{300pt}}}}
\begin{aligned}
&\r\cr&\b{Algorithm:}\t{An easy structure}\cr[-13pt]&\l\cr
&\begin{aligned}
    \t{1.}&\b{input}n,m,q \cr
    \t{2.}&\b{for}i=1\b{to} q \b{do} \cr
    \t{3.}&\s\b{input} s,l,w,k \cr
    \t{4.}&\s\b{for} j=1 \b{to} l \b{do}\cr
    \t{5.}&\s\s a[s+j-1] \gets a[s+j-1]+w\times \t{pow}(j,k) \cr
    \t{6.}&\s\b{end}\cr
    \t{7.}&\b{end}\cr
    \t{8.}&\b{for} i=1 \b{to} n \b{do}\cr
    \t{9.}&\s\b{output} a[i]\cr
    \t{10.}&\b{end}\cr
    \end{aligned}\cr[-12pt]
&\r\end{aligned}
%Made by @离散小波变换° .
%You can find his contributions by searching "JoesSR".
$$
其中 $\rm pow(a,b)$ 的含义为 $a^b$。

        int _a=_get(1,_n),_b=_get(1,_n); if(_a>_b) swap(_a,_b);
        int _l=_b-_a+1,_k=_get(0,_m); u64 _w=_get();
        _S[i]=_a,_L[i]=_l,_W[i]=_w,_K[i]=_k;
    }

    u64 ret=n^_get(); for(int i=1;i<=n;i++) ret^=_get()+R[i];

## 综合分析与结论  
### 核心思路  
所有题解均采用「二项式展开 + 差分数组」的核心策略：  
1. **多项式分解**：将每次操作中的 `w*(i^k)` 展开为 `ΣC(k,j)*x^j*(1-s)^{k-j}`，其中 `x` 是位置坐标  
2. **差分维护系数**：对展开后的每个次项 `x^j` 单独维护差分数组，实现 O(1) 区间修改  
3. **前缀和合成结果**：通过前缀和恢复系数后，计算各次项的总和  

### 解决难点  
- **暴力复杂度**：直接模拟的复杂度为 O(ql)（q=1e6，l=5e5）无法通过  
- **多项式拆分**：通过二项式定理将位置相关的多项式转换为固定次项的线性组合  
- **差分优化**：为每个次项单独维护差分数组，将整体复杂度降至 O(qm + nm)（m≤9）  

### 可视化设计思路  
1. **动画分步演示**：  
   - **二项式展开**：以气泡动画展示 `(x-s+1)^k` 展开为多项式的过程  
   - **差分操作**：用高亮颜色标记当前操作的区间端点，显示差分数组的加减操作  
   - **前缀和计算**：以流动效果从左到右更新前缀和数组  
2. **复古像素风格**：  
   - 用 8-bit 像素块表示数组元素，不同次项使用不同颜色（如红色为 x^0，蓝色为 x^1）  
   - 每次差分操作时播放短促的“滴答”音效，前缀和计算时播放连贯的音阶音效  
3. **AI自动演示**：  
   - 自动遍历所有操作，展示分解、差分、合成的完整流程  
   - 提供速度调节（1x/2x/5x）和单步调试按钮

## 最优思路与代码实现  
### 核心代码（耶梦加得版）  
```cpp  
// 预处理组合数  
for(int i = 0; i <= m; ++i) {
    C[i][0] = 1ull;
    for(int j = 1; j <= i; ++j) 
        C[i][j] = C[i - 1][j - 1] + C[i - 1][j];
}

// 处理每个操作  
for(int i = 1; i <= q; ++i) {
    u64 a = w[i];
    for(int j = k[i]; j >= 0; --j) {
        c[s[i]][j] += a * C[k[i]][j];
        c[s[i] + l[i]][j] -= a * C[k[i]][j];
        a *= (1 - s[i]); // 计算 (1-s)^{k-j}
    }
}

// 计算最终结果  
for(int i = 1; i <= n; ++i) {
    u64 v = 1ull; // x^j 的当前值
    for(int j = 0; j <= m; ++j) {
        c[i][j] += c[i - 1][j]; // 前缀和
        r[i] += c[i][j] * v;
        v *= i; // 更新 x^j
    }
}
```

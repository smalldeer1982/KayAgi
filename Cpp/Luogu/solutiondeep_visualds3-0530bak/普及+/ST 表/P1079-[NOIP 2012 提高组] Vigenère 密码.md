# [NOIP 2012 提高组] Vigenère 密码

## 题目描述

16 世纪法国外交家 Blaise de Vigenère 设计了一种多表密码加密算法 Vigenère 密码。Vigenère 密码的加密解密算法简单易用，且破译难度比较高，曾在美国南北战争中为南军所广泛使用。

 
在密码学中，我们称需要加密的信息为明文，用 $M$ 表示；称加密后的信息为密文，用 $C$ 表示；而密钥是一种参数，是将明文转换为密文或将密文转换为明文的算法中输入的数据，记为 $k$。在 Vigenère 密码中，密钥 $k$ 是一个字母串，$k=k_1,k_2,…,k_n$。当明文 $M=m_1,m_2,…,m_n$ 时，得到的密文 $C=c_1,c_2,…,c_n$，其中 $c_i$=$m_i \operatorname{\circledR} k_i$，运算 $\circledR$ 的规则如下表所示：

Vigenère 加密在操作时需要注意：

1. $\circledR$ 运算忽略参与运算的字母的大小写，并保持字母在明文 $M$ 中的大小写形式；
2. 当明文 $M$ 的长度大于密钥 $k$ 的长度时，将密钥 $k$ 重复使用。

例如，明文 $M=\texttt{Helloworld}$，密钥 $k=\texttt{abc}$ 时，密文 $C=\texttt{Hfnlpyosnd}$。

## 题解思路、算法要点与解决难点

### 核心思路
1. **数学推导**：将密文字符的 ASCII 码减去密钥字符的偏移量，通过模 26 运算处理循环问题。  
2. **大小写统一**：利用 `&31` 或 `toupper/tolower` 统一字符处理，保留原始大小写格式。  
3. **密钥循环**：通过取模操作实现密钥的周期性复用。

### 解决难点
1. **偏移量计算**：正确处理 `密文 - 密钥` 可能产生的负数（需加 26 后再取模）。  
2. **大小写分离**：保持明文字符与密文字符的大小写一致。  
3. **循环复用密钥**：用 `i % key_length` 实现密钥无限循环。

## 最优思路与技巧提炼

### 关键技巧
1. **ASCII 位运算**：用 `c & 31` 获取字母序（A=1, B=2,..., 无视大小写）。  
2. **统一偏移计算**：`(c - k_offset + 26) % 26` 处理负数并循环。  
3. **密钥循环指针**：`i % key_len` 自动复用密钥。

### 代码模板
```cpp
string k, c;
for (int i=0; i<c.size(); i++) {
    int k_offset = (k[i % k.size()] & 31) - 1; // 密钥字母序
    char decrypted = (c[i] & 31) - k_offset;
    decrypted = decrypted > 0 ? decrypted : decrypted + 26;
    decrypted += c[i] & 32; // 恢复小写（若原字符为小写）
    cout << decrypted;
}
```

## 代码实现（核心逻辑）

```cpp
#include <iostream>
using namespace std;

int main() {
    string k, c;
    cin >> k >> c;
    for (int i = 0; i < c.size(); i++) {
        char key = k[i % k.size()];
        // 统一为小写计算偏移量
        int offset = (tolower(key) - 'a');
        // 解密计算
        char decrypted = tolower(c[i]) - offset;
        if (decrypted < 'a') decrypted += 26;
        // 恢复原始大小写
        if (isupper(c[i])) decrypted = toupper(decrypted);
        cout << decrypted;
    }
    return 0;
}
```

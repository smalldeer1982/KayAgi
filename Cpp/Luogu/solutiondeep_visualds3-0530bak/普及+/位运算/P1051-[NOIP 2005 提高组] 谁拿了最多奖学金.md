# 题目信息

# [NOIP 2005 提高组] 谁拿了最多奖学金

## 题目描述

某校的惯例是在每学期的期末考试之后发放奖学金。发放的奖学金共有五种，获取的条件各自不同：

1. 院士奖学金，每人 $8000$ 元，期末平均成绩高于 $80$ 分（$>80$），并且在本学期内发表$1$篇或$1$篇以上论文的学生均可获得；
2. 五四奖学金，每人 $4000$ 元，期末平均成绩高于 $85$ 分（$>85$），并且班级评议成绩高于 $80$ 分（$>80$）的学生均可获得；
3. 成绩优秀奖，每人 $2000$ 元，期末平均成绩高于 $90$ 分（$>90$）的学生均可获得；
4. 西部奖学金，每人 $1000$ 元，期末平均成绩高于 $85$ 分（$>85$）的西部省份学生均可获得；
5. 班级贡献奖，每人 $850$ 元，班级评议成绩高于 $80$ 分（$>80$）的学生干部均可获得；

只要符合条件就可以得奖，每项奖学金的获奖人数没有限制，每名学生也可以同时获得多项奖学金。例如姚林的期末平均成绩是 $87$ 分，班级评议成绩 $82$ 分，同时他还是一位学生干部，那么他可以同时获得五四奖学金和班级贡献奖，奖金总数是 $4850$ 元。

现在给出若干学生的相关数据，请计算哪些同学获得的奖金总数最高（假设总有同学能满足获得奖学金的条件）。


## 说明/提示

**【数据范围】**

对于 $100\%$ 的数据，满足 $1 \le  N \le  100$。

**【题目来源】**

## 综合分析与结论

### 核心算法流程
1. **输入处理**：逐行读取学生数据，包含姓名、成绩、身份标识等字段。
2. **条件判断**：通过 5 个独立条件判断奖学金叠加，使用多个 `if` 语句而非 `else if` 确保多条件叠加。
3. **打擂台法**：实时维护最高奖学金学生信息，避免全量排序。
4. **总和统计**：在计算奖学金时累加总和。

### 解决难点
- **多条件叠加处理**：每个奖学金条件相互独立，需用独立 `if` 分支处理。
- **并列处理逻辑**：当最高奖学金相同时，保留输入顺序中的第一个，通过遍历顺序天然保证。
- **输入格式解析**：正确处理字符型字段（如 Y/N 标识）与数值型字段的混合输入。

### 可视化设计思路
1. **动画方案**：
   - **学生卡片流**：以 8-bit 像素风格卡片展示每个学生信息，卡片背景色区分当前处理状态。
   - **条件高亮**：当判断某一奖学金条件时，对应字段（如期末成绩、论文数）闪烁黄色边框。
   - **奖金累加特效**：每次满足条件时，右侧弹出 +8000/+4000 等像素数字特效，总奖学金槽同步增长。
2. **复古交互**：
   - **音效设计**：条件满足时播放 8-bit 上升音效，最高记录刷新时播放经典 FC 过关音效。
   - **自动演示模式**：按学生输入顺序自动播放计算过程，支持暂停/继续/步进控制。
3. **数据可视化**：
   - **排行榜**：右侧显示实时最高奖学金学生像素头像与金额，背景为动态条形图对比。

## 最优思路提炼

### 关键技巧
- **逻辑表达式乘法**：将条件判断转化为 `(condition) * value`，避免分支语句。
- **输入即计算**：在读取每个学生数据时即时计算奖金，减少内存占用。
- **打擂台法优化**：无需存储全部结果，遍历时直接维护最高记录与总和。

### 代码实现片段
```cpp
// 核心计算逻辑（qhr2023 方案简化版）
int calculate(int a, int b, char c, char d, int e) {
    return (a>80 && e>=1)*8000 +
           (a>85 && b>80)*4000 +
           (a>90)*2000 +
           (a>85 && d=='Y')*1000 +
           (b>80 && c=='Y')*850;
}
```

# [NOIP 2012 提高组] Vigenère 密码

## 题目描述

16 世纪法国外交家 Blaise de Vigenère 设计了一种多表密码加密算法 Vigenère 密码。Vigenère 密码的加密解密算法简单易用，且破译难度比较高，曾在美国南北战争中为南军所广泛使用。

 
在密码学中，我们称需要加密的信息为明文，用 $M$ 表示；称加密后的信息为密文，用 $C$ 表示；而密钥是一种参数，是将明文转换为密文或将密文转换为明文的算法中输入的数据，记为 $k$。在 Vigenère 密码中，密钥 $k$ 是一个字母串，$k=k_1,k_2,…,k_n$。当明文 $M=m_1,m_2,…,m_n$ 时，得到的密文 $C=c_1,c_2,…,c_n$，其中 $c_i$=$m_i \operatorname{\circledR} k_i$，运算 $\circledR$ 的规则如下表所示：

Vigenère 加密在操作时需要注意：

1. $\circledR$ 运算忽略参与运算的字母的大小写，并保持字母在明文 $M$ 中的大小写形式；
2. 当明文 $M$ 的长度大于密钥 $k$ 的长度时，将密钥 $k$ 重复使用。

例如，明文 $M=\texttt{Helloworld}$，密钥 $k=\texttt{abc}$ 时，密文 $C=\texttt{Hfnlpyosnd}$。

## 2. 综合分析与结论  
### 核心思路
所有题解均围绕 Vigenère 密码的解密公式展开，核心公式为：  
`明文 = (密文字符 - 密钥字符 + 26) % 26`  
其中需注意：  
1. 忽略密钥和密文的大小写  
2. 密钥循环使用（取模运算）  
3. 保持明文字符的大小写与密文一致  

### 关键难点与解决方案
- **字符偏移计算**：通过 ASCII 码的位运算（如 `x & 31`）快速获取字母序号  
- **大小写处理**：统一转为大写或小写计算偏移，最终还原原始大小写  
- **循环密钥**：通过 `i % key_len` 实现密钥的循环使用  

### 可视化设计思路（基于字符转换过程）
1. **网格展示**：绘制 26x26 字母矩阵，高亮当前密文和密钥对应的行列  
2. **动态标记**：用不同颜色标记当前处理的密文字符、密钥字符、计算偏移量  
3. **步骤分解**：分步显示：
   - 密文字符转数字序号  
   - 密钥字符转数字序号  
   - 计算 `(密文序号 - 密钥序号 + 26) % 26`  
   - 转换回明文字符  
4. **复古像素风格**：用 8-bit 字体和颜色渲染网格，添加音效（如字符转换时的电子音）

## 4. 最优思路与代码实现
### 最优技巧
```cpp
// 核心解密逻辑（作者：ingu）
for (int i=0; i<c.length(); i++) {
    int t = (k[i%k.length()] & 31) - 1; // 位运算快速取字母序号
    c[i] = (c[i] & 31) - t > 0 ? c[i]-t : c[i]-t+26;
}
```
- `x & 31`：将字母（无论大小写）映射到 1-26  
- 三目运算符处理负数偏移（等效 `(x+26) % 26`）

### 完整代码
```cpp
#include <iostream>
using namespace std;
int main() {
    string k,c;
    cin>>k>>c;
    for(int i=0; i<c.length(); i++) {
        int t=(k[i%k.length()]&31)-1;
        c[i]=(c[i]&31)-t>0?c[i]-t:c[i]-t+26;
    }
    cout<<c;
    return 0;
}
```

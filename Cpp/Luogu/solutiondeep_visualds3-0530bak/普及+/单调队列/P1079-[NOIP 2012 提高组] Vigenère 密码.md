# [NOIP 2012 提高组] Vigenère 密码

## 题目描述

16 世纪法国外交家 Blaise de Vigenère 设计了一种多表密码加密算法 Vigenère 密码。Vigenère 密码的加密解密算法简单易用，且破译难度比较高，曾在美国南北战争中为南军所广泛使用。

 
在密码学中，我们称需要加密的信息为明文，用 $M$ 表示；称加密后的信息为密文，用 $C$ 表示；而密钥是一种参数，是将明文转换为密文或将密文转换为明文的算法中输入的数据，记为 $k$。在 Vigenère 密码中，密钥 $k$ 是一个字母串，$k=k_1,k_2,…,k_n$。当明文 $M=m_1,m_2,…,m_n$ 时，得到的密文 $C=c_1,c_2,…,c_n$，其中 $c_i$=$m_i \operatorname{\circledR} k_i$，运算 $\circledR$ 的规则如下表所示：

Vigenère 加密在操作时需要注意：

1. $\circledR$ 运算忽略参与运算的字母的大小写，并保持字母在明文 $M$ 中的大小写形式；
2. 当明文 $M$ 的长度大于密钥 $k$ 的长度时，将密钥 $k$ 重复使用。

例如，明文 $M=\texttt{Helloworld}$，密钥 $k=\texttt{abc}$ 时，密文 $C=\texttt{Hfnlpyosnd}$。

## 综合分析与结论
### 核心思路与难点
**核心思路**：Vigenère 密码解密的核心是通过反向位移操作恢复明文。每个密文字符的位移量由密钥对应位置的字符决定，需要考虑大小写统一处理和密钥循环使用。

**核心难点**：
1. **位移计算**：需处理负数位移（`(密文字符 - 密钥字符 + 26) % 26`）
2. **密钥循环**：密钥长度短于密文时需循环使用（`i % key_length`）
3. **大小写处理**：保持密文字符的大小写形式

### 算法流程与可视化设计
1. **可视化流程**：
   - **高亮当前密文字符**（红色）和对应的密钥字符（蓝色）
   - 显示当前位移计算过程（如 `H - a → H - 0 → H`）
   - 使用动画展示字符回退过程（如 `Y → W`）
2. **复古像素风格**：
   - **颜色方案**：8-bit 风格调色板（红/蓝高亮，黑底白字）
   - **Canvas 绘制**：网格化展示字符变化，每个字符显示为 16x16 像素块
   - **音效设计**：
     - `beep` 音效：每次字符解密完成
     - `success` 音效：最终输出明文时播放

## 最优思路与代码实现
### 关键技巧
1. **ASCII 位运算**：`char & 31` 快速获取字母序数（A=1, B=2,...）
2. **统一位移公式**：`(c - k + 26) % 26` 处理所有正负情况
3. **循环密钥索引**：`i % key_len` 自动处理密钥重复

### 完整代码
```cpp
#include <iostream>
using namespace std;

int main() {
    string k, c;
    cin >> k >> c;
    for (int i = 0; i < c.size(); ++i) {
        char key = tolower(k[i % k.size()]);
        int shift = key - 'a';
        // 解密公式
        c[i] = (c[i] - (islower(c[i]) ? 'a' : 'A') - shift + 26) % 26 
              + (islower(c[i]) ? 'a' : 'A');
    }
    cout << c;
    return 0;
}
```

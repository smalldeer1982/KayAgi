# 如何得到 npy

## 题目背景

作为年级第一大风流人物，Steve 总会给自己找很多东西，包括但不限于 npy。Steve 看上了 Ada，并试图接近她，然而 Ada 并不是那么乐意。

## 题目描述

**提示：你可以阅读题目描述末尾的形式化题面。**

Steve 所在的校园有 $n$ 间教室，编号为 $1$ 到 $n$，有 $n-1$ 条走廊将其连通。也就是说，教室和走廊构成了一棵树。每条走廊都有一定的长度 $w_i$，经过这条走廊的时间等于其长度的数值。

Steve 喜欢在校园里游荡。当然，他希望最后能走到自己的教室 $s$ 或 Ada 的教室 $t$。但由于学校过于错综复杂，且 Steve 不想走回头路，所以他想到了如下方案：

对于每个教室（Steve 和 Ada 的教室除外，这两个教室周围不应该有任何标牌），在一条与其相连的边立上标牌。每次走到这个教室，就从立了标牌的边出。

Steve 可能会在学校的任何一个教室出现，所以一方面，Steve 需要让他从每个教室都能跟着标牌回到他的或 Ada 的教室。另一方面，他希望从学校所有教室走到目的地的**时间总和**尽可能小。

由于 Steve 又要去找 Ada 了，所以请你帮他完成这个任务。

#### 形式化题意

给定一棵 $n$ 个节点的无根树和两个关键点 $s,t$，要求对所有边定向，满足：

- 每条边要么是有向边，要么被删除；
- 每个点（除 $s,t$）出度恰好为 $1$，$s,t$ 出度为 $0$；
- 每个点都可以顺着有向边到达 $s$ 或 $t$。

求每个点到 $s$ 或 $t$ 的距离总和最小值。

## 综合分析与结论

### 核心算法流程
题目要求将树划分为两个子图，每个点只能选择更接近的终点（s或t），并构造边方向使得总距离最小。关键在于：
1. **计算每个点到s/t的最短距离**：两次DFS或BFS计算每个点到s和t的距离。
2. **确定分割边**：在s到t的路径上，找到一条边使得断开后的总距离最小。这条边是树的分界，将树分为s的子树和t的子树。
3. **构造边的方向**：除分割边外，每个子树内的边按指向父节点方向标记。

### 解决难点
- **分割边的选择**：需遍历s到t的路径，动态计算断开每条边后的总距离，选择最优解。
- **正确构造方向**：确保每个非s/t节点出度为1，且所有路径最终指向s或t。

### 可视化设计思路
- **动画方案**：以树状结构展示，用不同颜色标记s和t的子树，高亮当前分割边，动态更新总距离。
- **交互控制**：允许用户手动选择分割边，观察子树划分和总距离变化。
- **像素风格**：使用8-bit风格绘制树节点，分割边断裂时播放音效，用不同颜色箭头表示边方向。

## 最优思路提炼

1. **距离计算**：两次DFS分别计算每个点到s和t的最短距离。
2. **动态分割边**：在s-t路径上枚举每条边作为分割边，计算断开后的总距离，选择最小值。
3. **贪心构造方向**：每个子树的边方向指向父节点，确保出度约束。

## 核心代码实现（分割边选择）

```cpp
void dfs(int id, int fa) {
    for (auto p : edges[id]) {
        if (p.to != fa) {
            // 计算子树大小和总距离
            dfs(p.to, id);
            siz[id] += siz[p.to];
            sm[id] += sm[p.to];
        }
    }
}

// 遍历s-t路径找最优分割边
int find_split_edge() {
    ll max_offset = 0;
    for (int x = t; x != s; x = parent[x]) {
        ll current = t_sum[parent[x]] + s_sum[x];
        if (current > max_offset) {
            max_offset = current;
            split_edge = x;
        }
    }
    return split_edge;
}
```

# 题目信息

# [CSP-J 2022] 逻辑表达式

## 题目描述

逻辑表达式是计算机科学中的重要概念和工具，包含逻辑值、逻辑运算、逻辑运算优先级等内容。

在一个逻辑表达式中，元素的值只有两种可能：$0$（表示假）和 $1$（表示真）。元素之间有多种可能的逻辑运算，本题中只需考虑如下两种：“与”（符号为 `&`）和“或”（符号为 `|`）。其运算规则如下：

$0 \mathbin{\&} 0 = 0 \mathbin{\&} 1 = 1 \mathbin{\&} 0 = 0$，$1 \mathbin{\&} 1 = 1$；  
$0 \mathbin{|} 0 = 0$，$0 \mathbin{|} 1 = 1 \mathbin{|} 0 = 1 \mathbin{|} 1 = 1$。

在一个逻辑表达式中还可能有括号。规定在运算时，括号内的部分先运算；两种运算并列时，`&` 运算优先于 `|` 运算；同种运算并列时，从左向右运算。

比如，表达式 `0|1&0` 的运算顺序等同于 `0|(1&0)`；表达式 `0&1&0|1` 的运算顺序等同于 `((0&1)&0)|1`。

此外，在 C++ 等语言的有些编译器中，对逻辑表达式的计算会采用一种“短路”的策略：在形如 `a&b` 的逻辑表达式中，会先计算 `a` 部分的值，如果 $a = 0$，那么整个逻辑表达式的值就一定为 $0$，故无需再计算 `b` 部分的值；同理，在形如 `a|b` 的逻辑表达式中，会先计算 `a` 部分的值，如果 $a = 1$，那么整个逻辑表达式的值就一定为 $1$，无需再计算 `b` 部分的值。

现在给你一个逻辑表达式，你需要计算出它的值，并且统计出在计算过程中，两种类型的“短路”各出现了多少次。需要注意的是，如果某处“短路”包含在更外层被“短路”的部分内则不被统计，如表达式 `1|(0&1)` 中，尽管 `0&1` 是一处“短路”，但由于外层的 `1|(0&1)` 本身就是一处“短路”，无需再计算 `0&1` 部分的值，因此不应当把这里的 `0&1` 计入一处“短路”。

### 综合分析与结论

**题目核心**：处理带短路逻辑的中缀表达式求值，需统计运算符优先级、括号及短路次数。

**难点对比**：
1. **运算符优先级处理**：分治方法通过递归层级处理，线性扫描利用短路特性直接跳过计算。
2. **短路统计**：需避免嵌套短路被重复计数，分治法通过递归返回值判断，线性法通过指针跳跃实现。
3. **括号处理**：分治法递归处理括号内表达式，线性法直接跳过括号内部计算。

**最优思路**：
1. **分治递归法**（SkyWave）：预处理每个位置的括号层级和最后出现的运算符，递归时直接定位当前层级的最高优先级运算符。时间复杂度O(n)。
2. **线性扫描法**（Fishing_Boat）：利用短路特性直接跳过无效计算，维护`off`标记控制跳跃逻辑。时间复杂度O(n)。

### 核心代码实现

**分治递归法（SkyWave）**：
```cpp
int dfs(int l, int r) {
    if (c1[r] >= l) { // 处理最后出现的|
        int ans = dfs(l, c1[r]-1);
        if (ans == 1) {++cnt1; return 1;}
        return ans | dfs(c1[r]+1, r);
    }
    if (c2[r] >= l) { // 处理最后出现的&
        int ans = dfs(l, c2[r]-1);
        if (ans == 0) {++cnt2; return 0;}
        return ans & dfs(c2[r]+1, r);
    }
    // 处理括号或直接返回值
}
```

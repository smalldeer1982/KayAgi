# 题目信息

# 谁能赢呢？

## 题目描述

小明和小红经常玩一个博弈游戏。给定一个 $n\times n$ 的棋盘，一个石头被放在棋盘的左上角。他们轮流移动石头。每一回合，选手只能把石头向上，下，左，右四个方向移动一格，并且要求移动到的格子之前不能被访问过。谁不能移动石头了就算输。

假如小明先移动石头，而且两个选手都以最优策略走步，问最后谁能赢？


## 说明/提示

对于 $20\%$ 的数据，保证 $1\le n\le 10$；

对于 $40\%$ 的数据，保证 $1\le n\le 1000$；

对于 $100\%$ 数据，保证 $1\le n\le 10000$。

## 样例 #1

### 输入

```
2
0```

### 输出

```
Alice```

## 综合分析与结论  
### 核心思路与算法流程  
1. **棋盘建模与二分图性质**  
   - 将棋盘黑白染色，左上角为黑色，形成二分图（相邻格子颜色不同）。  
   - 游戏规则等价于在二分图上交替选择未访问节点，无法选择者输。  

2. **胜负判定与奇偶性**  
   - **偶数棋盘**：存在包含起点的完美匹配。先手每一步按匹配策略走，最终后手无路可走。  
   - **奇数棋盘**：最大匹配不包含起点。后手通过完美匹配策略使先手无法继续移动。  

3. **代码实现**  
   直接判断 `n` 的奇偶性：  
   ```cpp  
   if (n % 2 == 0) 先手赢 (Alice)；否则后手赢 (Bob)
   ```

### 可视化设计思路  
- **动画方案**：以 8 位像素风格渲染棋盘，动态展示黑白染色与匹配路径。  
  - **颜色标记**：起点用红色高亮，匹配边用绿色线条，当前移动路径用闪烁方块。  
  - **步进控制**：允许单步执行，观察每一步的匹配策略和对手应对。  
  - **音效提示**：移动时播放“点击”音效，胜负时播放胜利/失败音效。  
- **复古游戏化**：模拟经典红白机界面，棋盘以网格像素呈现，背景音乐循环播放 8 位风格旋律。

## 最优思路提炼  
1. **二分图博弈模型**：将棋盘转化为二分图，分析最大匹配是否包含起点。  
2. **奇偶性结论**：偶数棋盘存在完美匹配且包含起点，奇数棋盘反之。  
3. **极简代码实现**：仅需判断 `n` 的奇偶性，时间复杂度 **O(1)**。

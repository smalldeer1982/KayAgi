# 题目信息

# 硬币问题

## 题目描述

今有面值为 1、5、11 元的硬币各无限枚。

想要凑出 $n$ 元，问需要的最少硬币数量。

## 说明/提示

#### 样例解释

对于样例数据 1，最佳方案是 $15=5+5+5$，使用到 3 枚硬币。

对于样例数据 2，最佳方案是 $12=11 + 1$，使用到 2 枚硬币。

#### 数据规模与约定

对于 $100\%$ 的数据，保证 $n\leq 10^6$。

## 样例 #1

### 输入

```
15```

### 输出

```
3```

## 样例 #2

### 输入

```
12```

### 输出

```
2```

### 题解思路与核心难点分析

#### 核心思路
1. **状态定义**：设 `f[i]` 表示凑出 `i` 元所需的最少硬币数。
2. **状态转移**：对于每个 `i`，考虑最后一步使用的硬币面值（1、5、11），从 `i-1`、`i-5`、`i-11` 的状态转移而来，取三者最小值后加 1：
   $$ f[i] = \min(f[i-1], f[i-5], f[i-11]) + 1 $$
3. **边界条件**：`f[0] = 0`（凑 0 元不需要硬币），其他初始化为极大值（如 `INF`）。

#### 解决难点
- **贪心陷阱**：直接贪心（优先用大面值）会陷入局部最优，需通过动态规划全局比较。
- **越界处理**：当 `i < 5` 或 `i < 11` 时，需跳过对应面值的转移，避免数组越界。
- **时间复杂度**：$O(n)$，满足 $n \leq 10^6$ 的数据规模。

#### 可视化设计思路
- **动态表格**：以网格展示 `f[i]` 的更新过程，每个单元格显示当前状态值。
- **高亮与对比**：
  - 当前操作元素 `i` 标记为黄色。
  - 转移来源 `i-1`、`i-5`、`i-11` 高亮为绿色（若存在）。
  - 最小值来源用红色边框标记。
- **动画效果**：步进更新，每次循环播放硬币音效，成功更新时播放上扬音效。

### 最优思路提炼
1. **状态转移设计**：将大问题分解为三个子问题（`i-1`、`i-5`、`i-11`）的最小值。
2. **避免贪心错误**：通过动态规划强制全局比较，而非局部最优。
3. **空间优化**：仅需一维数组，滚动更新。

---

### 类似算法套路
- **完全背包问题**：若硬币面值不固定，需遍历所有面值，时间复杂度为 $O(n \cdot k)$（$k$ 为硬币种数）。
- **最短路径问题**：将硬币面值视为边权，求到达 `n` 的最短路径。

### 代码实现
```cpp
#include <iostream>
#include <vector>
#include <climits>
using namespace std;

int main() {
    int n;
    cin >> n;
    vector<int> f(n + 1, INT_MAX);
    f[0] = 0; // 初始条件
    
    for (int i = 1; i <= n; ++i) {
        if (i >= 1) f[i] = min(f[i], f[i-1] + 1);
        if (i >= 5) f[i] = min(f[i], f[i-5] + 1);
        if (i >= 11) f[i] = min(f[i], f[i-11] + 1);
    }
    
    cout << f[n];
    return 0;
}
```

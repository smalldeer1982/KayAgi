## 题目描述

每周都有若干名题解审核志愿者处理题解审核工作。每周末轮换时，本周的志愿者中会有若干人退出工作，其余人继续下周工作；此外，还有若干新人参与下周工作。

每周末，扶苏会给小粉兔提供如下三组信息：

- 本周的题解审核志愿者名单。
- 本周有哪些题解审核志愿者退出工作。
- 下周有哪些新人参与工作。

因为小粉兔忙着玩原神，所以小粉兔求着扶苏来处理轮换。但是扶苏忙着给洛谷入门赛造题，所以请你来给出下周的题解审核志愿者名单。扶苏承诺，只要你正确地给出名单，她将把小粉兔在头图中给出的辈分分一半给你。

## 综合分析与结论  

### 核心算法流程  
1. **输入处理**  
   - 本周名单存入集合（自动去重且有序）  
   - 删除退出者（集合的O(1)查找+删除）  
   - 插入新增者（集合的O(1)去重插入）  
2. **输出处理**  
   - 遍历集合直接输出（自动字典序）  

### 解决难点对比  
| 方法 | 时间复杂度 | 空间复杂度 | 排序实现 |  
|------|------------|------------|----------|  
| 数组标记法 | O(n*m + (n+l)log(n+l)) | O(n+l) | 手动排序 |  
| STL set法 | O(n logn + m logn + l logn) | O(n) | 自动维护 |  

**核心优化点**：  
- 利用红黑树特性自动维护有序性  
- 无需手动处理空字符串标记  
- 避免多次全量排序

## 最优思路与技巧提炼  

### 关键数据结构  
**红黑树集合（set）**：  
- 自动维护元素唯一性  
- 插入/删除/查找的O(logn)复杂度  
- 遍历即有序输出  

### 思维突破点  
- **去重与排序统一处理**：利用集合特性将三个操作（过滤退出者、合并新增者、排序）合并为一个数据结构的自然行为  
- **输入顺序无关性**：无论初始输入顺序如何，最终集合自动维护正确顺序

## 核心代码实现（C++）  
```cpp  
#include <iostream>
#include <set>
using namespace std;

int main() {
    ios::sync_with_stdio(false); // 加速IO
    int n, m, l;
    cin >> n >> m >> l;
    
    set<string> volunteers;
    
    // 读取初始名单
    for (int i = 0; i < n; ++i) {
        string name;
        cin >> name;
        volunteers.insert(name);
    }
    
    // 删除退出者
    for (int i = 0; i < m; ++i) {
        string name;
        cin >> name;
        volunteers.erase(name);
    }
    
    // 加入新增者
    for (int i = 0; i < l; ++i) {
        string name;
        cin >> name;
        volunteers.insert(name);
    }
    
    // 输出结果
    for (const auto& name : volunteers) {
        cout << name << '\n';
    }
    
    return 0;
}
```

## 个人调试心得  
- **输入加速**：`ios::sync_with_stdio(false)` 对1e5级数据量至关重要  
- **容器选择**：尝试过unordered_set但需要额外排序，最终回归set  
- **边界测试**：当m=0且l=0时验证原样输出，确保无多余操作

# [NOIP 2012 提高组] Vigenère 密码

## 题目描述

16 世纪法国外交家 Blaise de Vigenère 设计了一种多表密码加密算法 Vigenère 密码。Vigenère 密码的加密解密算法简单易用，且破译难度比较高，曾在美国南北战争中为南军所广泛使用。

 
在密码学中，我们称需要加密的信息为明文，用 $M$ 表示；称加密后的信息为密文，用 $C$ 表示；而密钥是一种参数，是将明文转换为密文或将密文转换为明文的算法中输入的数据，记为 $k$。在 Vigenère 密码中，密钥 $k$ 是一个字母串，$k=k_1,k_2,…,k_n$。当明文 $M=m_1,m_2,…,m_n$ 时，得到的密文 $C=c_1,c_2,…,c_n$，其中 $c_i$=$m_i \operatorname{\circledR} k_i$，运算 $\circledR$ 的规则如下表所示：

Vigenère 加密在操作时需要注意：

1. $\circledR$ 运算忽略参与运算的字母的大小写，并保持字母在明文 $M$ 中的大小写形式；
2. 当明文 $M$ 的长度大于密钥 $k$ 的长度时，将密钥 $k$ 重复使用。

例如，明文 $M=\texttt{Helloworld}$，密钥 $k=\texttt{abc}$ 时，密文 $C=\texttt{Hfnlpyosnd}$。

## 综合分析与结论

### 核心思路与难点
1. **核心算法**：  
   - 每个明文字符 $m_i = (c_i - k_i + 26) \bmod 26$，其中 $c_i$ 为密文字符，$k_i$ 为密钥字符（忽略大小写后对应的字母偏移量）。  
   - 密钥循环使用：通过 $i \bmod \text{len}(k)$ 实现密钥的周期性重复。  
   - 大小写处理：保持密文字符的大小写形式，仅根据其本身决定明文字符的大小写。

2. **解决难点**：  
   - **模运算边界处理**：减法可能产生负数，需通过 $+26$ 确保结果非负后再取模。  
   - **密钥对齐**：循环使用密钥时需正确处理索引的周期性变化。  
   - **大小写统一**：将密钥和密文字符统一为小写或大写后再计算偏移量，但保留密文字符的原始大小写。

3. **可视化设计思路**：  
   - **动画演示**：展示密钥循环对齐过程，高亮当前处理的密文字符和对应的密钥字符，以颜色区分大小写。  
   - **步进控制**：单步执行时显示 $(c_i - k_i + 26) \bmod 26$ 的计算步骤，并动态更新明文结果。  
   - **复古像素风格**：用 8-bit 风格网格展示密钥循环和字符变换，音效在每次解密成功时触发短促音效。

## 最优思路与技巧提炼

### 核心代码片段（ingu 的解法）
```cpp
for (int i=0; i<c.length(); i++) {
    int t = (k[i%k.length()] & 31) - 1;  // 统一为小写并计算偏移量
    c[i] = (c[i] & 31) - t > 0 ? c[i] - t : c[i] - t + 26;
}
```

### 关键技巧
1. **位运算统一大小写**：`k[i] & 31` 将字符转换为字母表序号（A=1, B=2, ... Z=26）。  
2. **单行边界处理**：通过三目运算符直接判断是否需要补 26。  
3. **循环索引简化**：`i % k.length()` 实现密钥的自动重复。

function decrypt(c, k) {
    const offset = (c.toLowerCase().charCodeAt(0) - 'a'.charCodeAt(0)) 
                 - (k.toLowerCase().charCodeAt(0) - 'a'.charCodeAt(0));
    return String.fromCharCode(
        (offset + 26) % 26 + (c === c.toUpperCase() ? 'A' : 'a').charCodeAt(0)
    );
}

## 题目描述

下面程序想要实现的功能为：

> 给出两个正整数 $x, y$，保证它们的最小公倍数（$\mathrm{lcm}$）不大于 $10^9$。求它们的最小公倍数。

```cpp
#include <iostream>
#include <algorithm>

int main() {
  int x, y;
  std::cin >> x >> y;
  int ans = x * y / std::__gcd(x, y);
  std::cout << ans << std::endl;
}
```

但是这个程序存在一些问题，请你提交一个程序，你提交的程序**应当输出一组这道题的输入数据，使得上面的程序输出错误的结果**。

提示：
1. `__gcd(x, y)` 可以求出 $x$ 和 $y$ 的最大公因数。
2. $\mathrm{lcm}(x,y)
\times\mathrm{gcd}(x,y)=x
\times y$

## 综合分析与结论

原程序的问题在于 `x * y` 可能导致 int 溢出。正确解法是构造两个大数，使其乘积超出 int 范围但 LCM 仍符合题目条件。例如输入 `1000000000 1000000000`，此时乘积溢出，但实际 LCM 为 1e9，符合题目要求。原程序因溢出导致错误，而正确计算应优先除法避免溢出（如 `x / gcd * y`）。

## 最优思路或技巧提炼

**关键技巧**：利用数值溢出构造 Hack 数据。选择两个大数使其乘积超过 int 上限，但通过高 GCD 值保证 LCM 仍在合法范围内。例如，两个相同的超大数（如 1e9），此时 LCM 为自身，但乘积溢出。

## 同类型题或类似算法套路

类似问题常出现在需要先乘后除的场景。通用解法是调整运算顺序，先用除法避免中间结果溢出。例如计算组合数时，优先约分再乘。

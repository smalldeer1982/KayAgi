# 题目信息

# [CSP-X2020 山东] 侠盗阿飞

## 题目描述

侠盗阿飞获得了一笔意外之财 $w$ 元钱，他想用这笔钱去帮助需要帮助的人。现在知道有 $n$ 个需要帮助的人以及他们每个人需要的钱数 $x_i$ 元（$i=0,1,2,3,\dots,n-1$），阿飞应该如何支配这笔钱使得能得到帮助的人数最多？

## 说明/提示

对于 $30\%$ 的数据，$x_i$ 为升序序列（$x_0\lt x_1\lt x_2\lt x_3\lt \dots$）。

对于 $100\%$ 的数据，$0\leq n\leq 500$，$0 \lt x_i\leq 5\times 10^4$，$0\leq w\leq 2\times 10^9$。


## 样例 #1

### 输入

```
10 5
1 2 3 4 5```

### 输出

```
4```

## 样例 #2

### 输入

```
1000 10
20 20 150 110 180 50 200 140 120 200```

### 输出

```
9```

# AI分析结果



**唯一算法分类**：贪心算法

---

**综合分析与结论**  
本题的目标是在给定预算下帮助最多的人，属于典型的贪心算法应用。核心思路是将所需金额从小到大排序，依次累加直到总和超过预算，此时的人数即为最大值。  
- **算法流程**：  
  1. 对所需金额数组进行升序排序。  
  2. 遍历排序后的数组，累加金额，若总和不超过预算则计数加一。  
  3. 当累加金额超过预算时停止遍历，返回当前计数。  

**关键难点**在于理解贪心策略的最优性，即选择金额最小的人群可以最大化帮助人数。此外，需处理边界条件如输入为空或所有金额总和未超预算的情况。

---

1. **贪心累加法 (5星)**  
   - **思路**：直接排序后逐个累加，实时判断总和是否超限。  
   - **亮点**：代码简洁，时间复杂度为 $O(n \log n)$，高效且易实现。  
   - **代码可读性**：使用显式循环和条件判断，逻辑清晰。  

2. **前缀和二分法 (4星)**

   - **优化点**：牺牲部分可读性换取理论上的效率提升，但实际差异不大。

**最优思路或技巧提炼**  
- **贪心策略**：排序后优先选择金额小的个体，确保每一步选择局部最优，最终全局最优。  
- **前缀和与二分结合**：通过预处理前缀和，快速定位最大合法人数，适用于需要多次查询的场景。

**代码实现**  
```cpp
#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

int main() {
    int n, w;
    cin >> n >> w;
    vector<int> x(n);
    for (int i = 0; i < n; ++i) cin >> x[i];
    sort(x.begin(), x.end());
    int sum = 0, count = 0;
    for (int num : x) {
        if (sum + num > w) break;
        sum += num;
        count++;
    }
    cout << count << endl;
    return 0;
}
```

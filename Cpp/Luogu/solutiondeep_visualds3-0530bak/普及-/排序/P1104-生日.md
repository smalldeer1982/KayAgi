# 生日

## 题目描述

cjf 君想调查学校 OI 组每个同学的生日，并按照年龄从大到小的顺序排序。但 cjf 君最近作业很多，没有时间，所以请你帮她排序。


## 说明/提示

数据保证，$1<n<100$，$1\leq |s|<20$。保证年月日实际存在，且年份 $\in [1960,2020]$。

```
Qiujingya 1993 10 13

```

### 输出

## 综合分析与结论  
### 核心思路  
所有题解的核心都是对年月日进行多级排序，主要分为两类实现方式：  
1. **结构体 + 多级比较**（主流方案）  
   - 按年、月、日逐级比较  
   - 日期相同时按输入顺序逆序排列  
   - 使用 `sort` 或 `stable_sort` 实现  
2. **日期转数值 + 单关键字排序**（需补零处理）  
   - 将年月日合并为整数（如 `19920423`）  
   - 需确保月份/日期补零（如 `8月1日 → 0801`）  

### 解决难点  
- **多级排序逻辑**：需处理年→月→日→输入顺序的四层优先级  
- **稳定排序要求**：日期相同时要求保留输入顺序，通过比较输入编号逆序实现  
- **数值转换陷阱**：直接计算天数（如 `年*365+月*30+日`）会因月份天数差异导致错误  

### 可视化设计  
- **高亮当前比较字段**：用不同颜色标记正在比较的年/月/日  
- **结构体字段展开**：将每个学生的年月日分开展示，比较时动态连线  
- **数值转换演示**：展示 `1992 → 04 → 23` 合并为 `19920423` 的过程  
- **复古像素风格**：  
  - 用 8-bit 风格显示排序过程，配电子音效（交换时触发 "哔" 声）  
  - Canvas 绘制网格，每个学生占一行，年月日分列三列

## 最优思路与技巧  
### 关键技巧  
1. **结构体多级比较**：逐层判断年→月→日，最后处理输入顺序  
2. **输入编号逆序**：日期相同时，`return a.num > b.num` 保证后输入的优先  
3. **稳定排序优化**：使用 `stable_sort` 可省略输入顺序判断（但主流题解未采用）  

### 同类型题套路  
- **多关键字排序**：学生成绩（总分→语文→数学）、日程安排（年→月→日）  
- **稳定排序需求**：需要保留原始顺序时（如投票系统按得分→提交时间排序）

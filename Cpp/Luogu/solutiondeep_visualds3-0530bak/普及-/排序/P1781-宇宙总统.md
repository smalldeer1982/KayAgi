# 题目信息

# 宇宙总统

## 题目描述

地球历公元 6036 年，全宇宙准备竞选一个最贤能的人当总统，共有 $n$ 个非凡拔尖的人竞选总统，现在票数已经统计完毕，请你算出谁能够当上总统。


## 说明/提示

票数可能会很大，可能会到 $100$ 位数字。

$1 \leq n \leq 20$。

## 样例 #1

### 输入

```
5
98765
12365
87954
1022356
985678
```

### 输出

```
4
1022356```

# AI分析结果



**唯一算法分类**: 字符串处理与比较

---

### **题解思路与核心难点**

1. **核心逻辑**  
   票数为超长字符串，比较需分两步：  
   - **先比长度**：位数越多数值越大  
   - **长度相同再比字典序**（等同于高精度数比较）

2. **解决难点**  
   - **避免高精度运算**：直接利用字符串特性比较，复杂度O(L)（L为字符串长度）  
   - **实时更新最大值**：无需存储全部数据，空间复杂度优化至O(1)

### **最优思路提炼**

1. **字符串特性利用**  
   - 直接比较长度与字典序，无需高精度计算  
   - **关键操作**：`str.size()`获取长度，`operator>`比较字典序

2. **实时更新代替全排序**  
   - 遍历时仅维护当前最大值，空间复杂度O(1)  
   - **适用场景**：数据流处理或n较大时更优

### **代码实现（核心逻辑）**

```cpp
#include <iostream>
#include <string>
using namespace std;

int main() {
    int n, id = 1;
    string max_val = "";
    cin >> n;
    for (int i = 1; i <= n; ++i) {
        string s;
        cin >> s;
        if (s.size() > max_val.size() || (s.size() == max_val.size() && s > max_val)) {
            max_val = s;
            id = i;
        }
    }
    cout << id << endl << max_val;
    return 0;
}
```

---

### **总结**

- **思维陷阱**：试图用数值类型（如`long long`）直接存储会溢出  
- **调试经验**：比较逻辑必须**先长度后字典序**，否则如`"99"`与`"100"`会误判  
- **扩展性**：相同思路可应用于大数排序、数据流中实时求Top-K

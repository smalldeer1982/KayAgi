## 题目描述

然而，XHY同学并不会做PPT，于是她找到了CYD大神求助。她想把N张4\*3的照片不重叠，无缝隙地拼成一个矩形作为背景图片。为了美观，她希望背景图片的长与宽尽可能接近，并且在长宽均相等的情况下，使得旋转的照片张数尽可能少。（4\*3的不算旋转，3\*4的算作旋转，且照片只能有这两种尺寸）。

请你帮CYD大神写一个程序，帮XHY同学设计一种满足她要求的照片排列方式，使得旋转的照片张数最少。

### 题解思路、算法要点与解决难点
**核心问题**：在面积固定的矩形中，找到长宽最接近且旋转照片最少的排列方式。  
**关键思路**：  
1. **面积约束**：总面积为12N，枚举可能的边长组合(a,b)。  
2. **长宽接近优先**：从sqrt(12N)向下枚举a，第一个合法的(a,b)即为长宽最接近的。  
3. **旋转次数计算**：将边分解为4x+3y形式，计算旋转次数的最小值。  

**解决难点**：  
- **分解验证**：如何快速验证边能否分解为4x+3y（枚举x，检查剩余部分是否被3整除）。  
- **旋转次数推导**：当b=4x+3y时，旋转次数为min(a*x/3, N - a*x/3)。  

**算法流程**：  
1. 枚举a从sqrt(12N)到3，检查是否整除。  
2. 确定a和b的倍数关系（3或4）。  
3. 分解b为4x+3y，计算旋转次数并取最小。

### 核心代码片段
```cpp
int main() {
    int n, ans = INT_MAX;
    cin >> n;
    int S = 12 * n;
    // 从sqrt(S)开始枚举a
    for (int a = sqrt(S); a >= 3; a--) {
        if (S % a != 0) continue;
        int b = S / a;
        // 处理a为3的倍数的情况
        if (a % 3 != 0) swap(a, b);
        // 分解b为4x+3y
        for (int x = 0; 4 * x <= b; x++) {
            int rem = b - 4 * x;
            if (rem % 3 == 0) {
                int rotate = x * a / 3;
                ans = min(ans, min(rotate, n - rotate));
            }
        }
        break; // 第一个合法的a即为最优
    }
    cout << ans;
    return 0;
}
```

通过枚举长宽并分解边，结合数学推导与暴力验证，该算法高效解决了问题，适用于N≤900的数据范围。

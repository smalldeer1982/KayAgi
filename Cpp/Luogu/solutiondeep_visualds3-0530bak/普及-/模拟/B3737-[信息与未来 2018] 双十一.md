# [信息与未来 2018] 双十一

## 题目描述

每年 $11$ 月 $11$ 日，各大网上商店都会有促销活动，因此大家都希望 $11$ 月 $11$ 日在周末，就可以更愉快地购物啦。请你写一个程序计算一段时间中，$11$ 月 $11$ 日是周末（周六或周日）的数量。
以下关于日期的定义和事实能帮到你：
- $1900$ 年 $1$ 月 $1$ 日是星期一。
- 每年的 $1, 3, 5, 7, 8, 10, 12$ 月有 $31$ 天；$4, 6, 9, 11$ 月有 $30$ 天；闰年的 $2$ 月有 $29$ 天，非闰年的 $2$ 月有 $28$ 天。
- 闰年的计算方法：不能被 $100$ 整除的年份称为普通年。普通年能被 $4$ 整除的为闰年，因此$2004$ 年是闰年，$1999$ 年不是闰年；能被 $100$ 整除的年份称为世纪年。世纪年能被 $400$ 整除的是闰年，因此 $2000$ 年是闰年，$1900$ 年不是闰年。

## 说明/提示

### 样例解释
#### 样例 $1$
$2018$ 年 $11$ 月 $11$ 日是星期日。
#### 样例 $2$
$2018$ 年到 $2100$ 年之间共有 $23$ 个 $11$ 月 $11$ 日是周末。
### 数据规模

### 综合分析与结论
本题要求计算指定时间段内每年11月11日是周末的数量。核心在于确定每个年份的11月11日的星期几。通过模拟日期计算，结合闰年判断，累加天数并取模得到星期数。

**核心难点**：
1. **闰年判断**：需准确区分普通年与世纪年的闰年规则。
2. **天数累加**：从基准年（1900年）到目标年的总天数计算，避免重复计算。

**解决方案**：
- **预处理前缀和**：预计算1900年到各年份的总天数，避免每次重复遍历。
- **直接计算月份天数**：通过公式简化每月天数累加，提升效率。

### 代码粘贴与核心实现思想
```python
def is_leap(year):
    if year % 100 == 0:
        return year % 400 == 0
    else:
        return year % 4 == 0

max_year = 3000
sum_days = [0] * (max_year + 2)  # sum_days[y]表示1900到y-1年的总天数
sum_days[1900] = 0
for y in range(1901, max_year + 1):
    prev_year = y - 1
    if is_leap(prev_year):
        sum_days[y] = sum_days[y - 1] + 366
    else:
        sum_days[y] = sum_days[y - 1] + 365

x, y = map(int, input().split())
count = 0
for year in range(x, y + 1):
    total = sum_days[year]  # 1900到year-1年的总天数
    feb = 29 if is_leap(year) else 28
    sum2 = 276 + feb  # 1-10月总天数
    total += sum2 + 11  # 加上11月11日
    weekday = total % 7
    if weekday in {5, 6}:
        count += 1
print(count)
```

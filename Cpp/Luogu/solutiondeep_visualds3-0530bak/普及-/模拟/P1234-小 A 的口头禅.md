# 题目信息

# 小 A 的口头禅

## 题目描述

小 A 最近有了一个口头禅 “呵呵”，于是他给出了一个矩形，让你求出里面有几个 “$\verb!hehe!$”（方向可以是向上、向下、向左、向右的直线）。


## 说明/提示

$1 \leq n,m \leq1000$。



## 样例 #1

### 输入

```
5 5
heheh
heheh
heheh
heheh
heheh
```

### 输出

```
10
```

## 综合分析与结论

### 核心思路
题目要求在二维矩阵中统计所有方向（上下左右）的连续 "hehe" 或 "eheh" 序列数量。核心解法为：
1. **线性扫描矩阵**：遍历每个可能的起点 (i,j)
2. **四方向检查**：对每个起点，检查上下左右四个方向是否能组成目标序列
3. **边界处理**：确保检查时不越界

### 解决难点
- **方向覆盖**：需正确处理四个方向的连续字符检查
- **反向序列**：需同时统计 "hehe" 和其反向形式 "eheh"
- **边界判断**：检查 i±3、j±3 是否超出矩阵范围

### 可视化设计思路
1. **网格动画**：用 Canvas 绘制矩阵，高亮当前检查的起点 (i,j)
2. **路径追踪**：匹配成功时，用不同颜色标记匹配路径（如绿色箭头）
3. **实时计数**：右侧面板显示已找到的序列数量
4. **像素风格**：采用 8-bit 色块表示字符，音效提示匹配成功

## 最优思路提炼
1. **双模式检查**：同时处理 "hehe" 和 "eheh" 两种模式
2. **方向分离**：独立处理水平（横向）和垂直（纵向）检查
3. **预判边界**：在循环条件中提前限制 j <= m-3 / i <= n-3

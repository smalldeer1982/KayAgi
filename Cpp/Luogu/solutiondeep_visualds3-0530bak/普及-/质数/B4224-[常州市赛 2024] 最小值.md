# [常州市赛 2024] 最小值

## 题目背景

搬运自 <http://czoj.com.cn/p/950>。数据为民间数据。

## 题目描述

素数是指一个大于 $1$ 的自然数，除了 $1$ 和它本身外，不能被其他自然数整除，换句话说就是该数除了 $1$ 和它本身以外不再有其他的因数。比如 $2,3,5,7,11,13\cdots$ 都是素数，而 $4,6,8,9,10\cdots$ 都是合数（有 $3$ 个以上因数）。特别地，$0$ 和 $1$ 既不是素数，也不是合数。

小 Y 想知道最小的素数，那当然是 $2$。不过他加上了一些限制，就是只通过某些数字构成的素数最小是多少？比如只通过 $3$ 构成的最小的素数是 $3$，只通过 $1$ 构成的最小的素数是 $11$，只通过 $8$ 和 $9$ 构成的最小的素数是 $89$ 等等。另外需要特别指出的是：给出的数字无须全部用到，比如只通过 $3$ 和 $5$ 构成的最小的素数是 $3$，而不是 $53$。

## 说明/提示

对于所有数据，$1\le n\le10$，保证可以使用的数字互不相同。

|测试点编号|$n$|
|:-:|:-:|

### 题解思路与难点分析
**核心思路**：  
通过贪心策略按数值顺序构造可能的最小素数。优先检查单数字素数（2、3、5、7），再依次检查两位数、三位数等特定组合（如11、89、409、449）。

**解决难点**：  
1. **贪心条件顺序**：需严格按数值从小到大判断，避免漏解（如先检查2再3，确保最小）。
2. **组合覆盖**：需覆盖所有可能的合法组合（如89需8和9同时存在，409需0、4、9）。
3. **效率优化**：贪心策略时间复杂度为 O(1)，远优于暴力枚举的 O(N)。

**关键对比**：  
- **贪心法**：通过逻辑推导直接构造目标素数，效率极高，但需严格验证所有可能的最小组合。
- **暴力法**：逐个生成数字并验证，正确性有保障，但数据范围较大时效率低。

### 最优思路提炼
**贪心构造法**：  
1. **单数字优先**：直接判断2、3、5、7是否存在。
2. **两位数组合**：若存在1则11，存在8和9则89。
3. **三位数组合**：如409（需0、4、9）和449（仅需4、9）。
4. **严格顺序**：按数值从小到大判断，确保首次找到的即最小解。

```cpp
#include<bits/stdc++.h>
using namespace std;
int mp[15], n;
int main() {
    cin >> n;
    for (int i = 1; i <= n; i++) {
        int x; cin >> x;
        mp[x] = 1;
    }
    // 按数值顺序检查所有可能
    if (mp[2]) cout << 2;
    else if (mp[3]) cout << 3;
    else if (mp[5]) cout << 5;
    else if (mp[7]) cout << 7;
    else if (mp[1]) cout << 11;
    else if (mp[8] && mp[9]) cout << 89;
    else if (mp[4] && mp[0] && mp[9]) cout << 409;
    else if (mp[4] && mp[9]) cout << 449;
    else cout << -1;
    return 0;
}
```

# 题目信息

# 骑士的工作

## 题目背景

你作为一个村的村长，保卫村庄是理所当然的了。今天，村庄里来了一只恶龙，他有 $n$ 个头，恶龙到处杀人放火。你着急了。不过天无绝人之路，现在来了一个骑士团。里面有 $m$ 位成员（往下看）。


## 题目描述

每个人都可以砍掉**至多**一个大小不超过 $z_i$ 的头，需要 $z_i$ 个金币，求最小花费。


## 说明/提示

对于所有数据，$1 \le n,m \le 2 \times 10^4$，$1 \leq z_i \leq 10^5$。

## 样例 #1

### 输入

```
2 3
5 
4
7 
8
4```

### 输出

```
11```

# AI分析结果



### 综合分析与结论

**题目核心**：在满足每个骑士最多砍一个头的约束下，寻找最小总花费的贪心策略。  
**难点**：如何快速匹配龙头与骑士，确保花费最小且满足约束。  
**关键贪心策略**：双排序后双指针匹配，每次选择当前最小可行骑士（局部最优推导全局最优）。

### 最优思路与技巧提炼

**核心贪心逻辑**：  
1. **双排序**：将龙头和骑士能力分别升序排序。  
2. **双指针贪心匹配**：  
   - 指针i遍历龙头，指针j遍历骑士。  
   - 若当前骑士能砍当前龙头，累加花费并双指针后移。  
   - 否则仅移动骑士指针，尝试更大能力的骑士。  

**数学验证**：  
- 排序后，若存在更优解，则必然存在交换后总花费更小，与贪心选择矛盾。  
- 因此每一步选择当前最小可行骑士必为全局最优。

### 核心代码实现

**双指针贪心关键代码**：  
```cpp
sort(a, a+n); // 龙头排序
sort(b, b+m); // 骑士排序
int i=0, j=0, cost=0;
while (i < n && j < m) {
    if (a[i] <= b[j]) { // 可匹配
        cost += b[j];
        i++; j++;
    } else j++; // 换更大骑士
}
if (i == n) cout << cost;
else cout << "you died!";
```

# 题目信息

# [NOIP 2010 普及组] 三国游戏

## 题目描述

小涵很喜欢电脑游戏，这些天他正在玩一个叫做《三国》的游戏。

在游戏中，小涵和计算机各执一方，组建各自的军队进行对战。游戏中共有 $N$ 位武将（$N$ 为偶数且不小于 $4$），任意两个武将之间有一个“默契值”，表示若此两位武将作为一对组合作战时，该组合的威力有多大。游戏开始前，所有武将都是自由的（称为自由武将，一旦某个自由武将被选中作为某方军队的一员，那么他就不再是自由武将了），换句话说，所谓的自由武将不属于任何一方。

游戏开始，小涵和计算机要从自由武将中挑选武将组成自己的军队，规则如下：小涵先从自由武将中选出一个加入自己的军队，然后计算机也从自由武将中选出一个加入计算机方的军队。接下来一直按照“小涵 $\to$ 计算机 $\to$ 小涵 $\to\dots$ ”的顺序选择武将，直到所有的武将被双方均分完。然后，程序自动从双方军队中各挑出一对默契值最高的武将组合代表自己的军队进行二对二比武，拥有更高默契值的一对武将组合获胜，表示两军交战，拥有获胜武将组合的一方获胜。

已知计算机一方选择武将的原则是尽量破坏对手下一步将形成的最强组合，它采取的具体策略如下：任何时刻，轮到计算机挑选时，它会尝试将对手军队中的每个武将与当前每个自由武将进行一一配对，找出所有配对中默契值最高的那对武将组合，并将该组合中的自由武将选入自己的军队。 下面举例说明计算机的选将策略，例如，游戏中一共有$6$个武将，他们相互之间的默契值如下表所示：

| 武将编号 |  1   |  2   |  3   |  4   |  5   |  6   |
| :------: | :--: | :--: | :--: | :--: | :--: | :--: |
|  **1**   |      | $5$  | $28$ | $16$ | $29$ | $27$ |
|  **2**   | $5$  |      | $23$ | $3$  | $20$ | $1$  |
|  **3**   | $28$ | $23$ |      | $8$  | $32$ | $26$ |
|  **4**   | $16$ | $3$  | $8$  |      | $33$ | $11$ |
|  **5**   | $29$ | $20$ | $32$ | $33$ |      | $12$ |
|  **6**   | $27$ | $1$  | $26$ | $11$ | $12$ |      |

双方选将过程如下所示：

|        | 小涵 | 轮到计算机时可选的自由武将 |  计算机    |  计算机选将说明    |
| :----: | :--: | :----------: | :--: | :--: |
| 第一轮 | $5$ | $1,2,3,4,6$ | $\color{magenta}4$ | 小涵手中的 $5$ 号武将与 $4$ 号的默契值最高，所以计算机选择 $4$ 号。 |
| 第二轮 | $5,3$ | $1,2,6$ | $4,\color{magenta}1$ |  小涵手中的 $5$ 号和 $3$ 号武将与自由武将中配对可产生的最大默契值为 $29$，是由 $5$ 号与 $1$ 号配对产生的，所以计算机选择 $1$ 号。  |
| 第三轮 | $5,3,6$ | $2$ | $4,1,\color{magenta}2$ |      |

小涵想知道，如果计算机在一局游戏中始终坚持上面这个策略，那么自己有没有可能必胜？如果有，在所有可能的胜利结局中，自己那对用于比武的武将组合的默契值最大是多少？  

假设整个游戏过程中，对战双方任何时候均能看到自由武将队中的武将和对方军队的武将。为了简化问题，保证对于不同的武将组合，其默契值均不相同。

## 题解思路与核心难点

**核心思路**  
本题本质是博弈论中的必胜策略推导问题。通过分析计算机的破坏策略，发现人类玩家无法获取任何武将组合的最高默契值。核心解法是寻找每个武将的次高默契值中的最大值，该值即为人类玩家必胜的保证。

**解决难点**  
1. **破坏策略推导**：计算机总会破坏当前人类武将的最强组合，导致最高默契值无法被人类获取  
2. **次优解证明**：需严格证明次大默契值中的最大值即为人类可能获得的最优解  
3. **数据结构优化**：对每个武将的n个默契值进行快速排序（O(n log n)），总时间复杂度为O(n² log n)  

**算法流程**  
1. 构建对称的默契值矩阵  
2. 对每个武将的默契值数组进行排序  
3. 取每个武将次大默契值（排序后倒数第二元素）  
4. 所有次大值中取最大值即为答案

## 最优思路提炼

**关键策略**  
1. **次大值定理**：每个武将的第二高默契值构成人类必胜的可行解集合  
2. **默契矩阵对称性**：利用对称性快速构建完整默契值矩阵  
3. **破坏策略反制**：通过优先选择次大值最大组合，确保计算机无法获得更高默契值

}

## 核心代码实现

**完整代码**  
```cpp
#include <cstdio>
#include <algorithm>
using namespace std;

int a[510][510], n, ans;

int main() {
    scanf("%d", &n);
    for(int i=1; i<n; ++i)
        for(int j=i+1; j<=n; ++j) {
            scanf("%d", &a[i][j]);
            a[j][i] = a[i][j];
        }
    
    for(int i=1; i<=n; ++i) {
        sort(a[i]+1, a[i]+n+1);
        ans = max(ans, a[i][n-1]);
    }
    
    printf("1\n%d", ans);
    return 0;
}
```

# [NOIP 2012 提高组] Vigenère 密码

## 题目描述

16 世纪法国外交家 Blaise de Vigenère 设计了一种多表密码加密算法 Vigenère 密码。Vigenère 密码的加密解密算法简单易用，且破译难度比较高，曾在美国南北战争中为南军所广泛使用。

 
在密码学中，我们称需要加密的信息为明文，用 $M$ 表示；称加密后的信息为密文，用 $C$ 表示；而密钥是一种参数，是将明文转换为密文或将密文转换为明文的算法中输入的数据，记为 $k$。在 Vigenère 密码中，密钥 $k$ 是一个字母串，$k=k_1,k_2,…,k_n$。当明文 $M=m_1,m_2,…,m_n$ 时，得到的密文 $C=c_1,c_2,…,c_n$，其中 $c_i$=$m_i \operatorname{\circledR} k_i$，运算 $\circledR$ 的规则如下表所示：

Vigenère 加密在操作时需要注意：

1. $\circledR$ 运算忽略参与运算的字母的大小写，并保持字母在明文 $M$ 中的大小写形式；
2. 当明文 $M$ 的长度大于密钥 $k$ 的长度时，将密钥 $k$ 重复使用。

例如，明文 $M=\texttt{Helloworld}$，密钥 $k=\texttt{abc}$ 时，密文 $C=\texttt{Hfnlpyosnd}$。

## **题解思路与算法要点**  
### **核心思路**  
Vigenère 密码的解密核心在于逆向运算：已知密文 $C$ 和密钥 $k$，通过计算 $m_i = (C_i - k_i + 26) \bmod 26$ 得到明文。关键点包括：  
1. **字符数值转换**：将字母统一映射到 0-25 的数值（如 `A=0, B=1`）。  
2. **密钥循环使用**：通过取模运算 `i % key_length` 实现密钥的重复扩展。  
3. **大小写保留**：根据密文字符的大小写决定明文字符的大小写。  

### **解决难点**  
- **负数处理**：当 $C_i - k_i$ 为负数时，需通过 `+26` 后再取模。  
- **密钥长度不足**：通过循环取模动态扩展密钥。  
- **大小写混合处理**：统一忽略密钥和密文的大小写计算偏移，但保留密文的大小写形式。  

---

## **题解评分（≥4星）**  
1. **ingu 的题解（5星）**  
   - **亮点**：利用位运算 `x & 31` 快速获取字母序号，代码极简（仅10行）。  
   - **代码可读性**：高效但需理解位运算技巧。  
   - **优化点**：直接操作字符，无需显式类型转换。  

2. **Sinwind 的题解（4星）**

## **代码实现（核心逻辑）**  
```cpp
#include <iostream>
using namespace std;

int main() {
    string k, c;
    cin >> k >> c;
    for (int i = 0; i < c.length(); i++) {
        // 统一转换为小写计算偏移
        char key = tolower(k[i % k.length()]);
        char cipher = tolower(c[i]);
        // 计算明文偏移
        int offset = (cipher - 'a' - (key - 'a') + 26) % 26;
        // 保留原大小写
        c[i] = isupper(c[i]) ? 'A' + offset : 'a' + offset;
    }
    cout << c << endl;
    return 0;
}
```

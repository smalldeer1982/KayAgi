# 题目信息

# [NOI2016] 网格

## 题目描述

跳蚤国王和蛐蛐国王在玩一个游戏。

他们在一个 $n$ 行 $m$ 列的网格上排兵布阵。其中的 $c$ 个格子中 $(0 \leq c \leq n\cdot m)$，每个格子有一只蛐蛐，其余的格子中，每个格子有一只跳蚤。

我们称占据的格子有公共边的两只跳蚤是相邻的。

我们称两只跳蚤是连通的，当且仅当这两只跳蚤相邻，或存在另一只跳蚤与这两只跳蚤都连通。

现在，蛐蛐国王希望，将某些（零个，一个或多个）跳蚤替换成蛐蛐，使得在此之后存在至少两只跳蚤不连通。

例如：图 $1$ 描述了一个 $n=4$，$m=4$，$c=2$ 的情况。

![](https://cdn.luogu.com.cn/upload/image_hosting/96tey4uv.png)

这种情况下蛐蛐国王可以通过将第二行第二列，和第三行第三列的两只跳蚤替换为蛐蛐，从而达成他的希望，如右图所示。并且，不存在更优的方案，但是可能存在其他替换两只跳蚤的方案。

你需要首先判断蛐蛐国王的希望能否被达成。如果能够达成，你还需要最小化被替换的跳蚤的个数。

### 综合分析与结论
本题核心在于拓扑缩点与离散化处理，通过提取关键点构建等效图进行连通性分析。各题解的共同思路是：
1. **离散化处理**：仅保留蛐蛐周围 5×5 区域的跳蚤（部分题解扩展到 3×3 或更广范围），将 1e9 量级的网格缩小到 O(c) 级别
2. **割点判断**：在缩点后的等效图上运行 Tarjan 算法判断割点
3. **特殊边界处理**：对单行单列的特殊网格进行特判

核心难点：
1. **伪割点陷阱**：直接对 3×3 区域缩点可能产生假的割点，需扩展至 5×5 区域确保正确性
2. **连通性验证**：需同时考虑四邻域跳蚤连通性 + 八邻域蛐蛐块对跳蚤的分割作用
3. **超大网格处理**：通过哈希表或坐标映射实现离散化存储

### 最优思路提炼
1. **关键点提取**：对每个蛐蛐周围的 5×5 区域跳蚤进行离散化存储，确保不遗漏潜在割点
2. **复合连通判断**：
   - 对跳蚤四邻域 BFS 染色
   - 对蛐蛐八邻域 BFS 检查跨区域连通性
3. **割点有效性验证**：仅当割点位于蛐蛐的 3×3 邻域时才视为有效候选
4. **单行特判**：当 n=1 或 m=1 时答案恒为 1（只需切割任意中间点）

### 核心代码片段（Thinking 题解）

```cpp
// 离散化建图核心逻辑
for(int i=0;i<c;++i)
    for(int x=Max(1,xi[i]-2);x<=xi[i]+2;x++)
        for(int y=Max(1,yi[i]-2);y<=yi[i]+2;y++)
            if(!exist(x,y)) add_to_graph(x,y);

// Tarjan 割点判定
void tarjan(int u,int fa){
    dfn[u]=low[u]=++cnt;
    int ch=0;
    for(int i=head[u];i;i=e[i].next){
        int v=e[i].to;
        if(!dfn[v]){
            tarjan(v,u);
            low[u]=min(low[u],low[v]);
            if(low[v]>=dfn[u] && check_near(u)) // 检查是否在3×3区域
                iscut[u]=1;
        } else if(v!=fa) 
            low[u]=min(low[u],dfn[v]);
    }
    if(fa==0 && ch==1) iscut[u]=0;
}
```

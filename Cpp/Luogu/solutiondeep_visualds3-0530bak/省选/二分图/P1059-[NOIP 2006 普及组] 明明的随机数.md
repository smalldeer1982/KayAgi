# 题目信息

# [NOIP 2006 普及组] 明明的随机数

## 题目描述

明明想在学校中请一些同学一起做一项问卷调查，为了实验的客观性，他先用计算机生成了 $N$ 个 $1$ 到 $1000$ 之间的随机整数 $(N\leq100)$，对于其中重复的数字，只保留一个，把其余相同的数去掉，不同的数对应着不同的学生的学号。然后再把这些数从小到大排序，按照排好的顺序去找同学做调查。请你协助明明完成“去重”与“排序”的工作。


## 说明/提示

NOIP 2006 普及组 第一题


## 样例 #1

### 输入

```
10
20 40 32 67 40 20 89 300 400 15
```

### 输出

```
8
15 20 32 40 67 89 300 400
```

### **综合分析与结论**
本题核心为去重与排序，各题解主要采用三种思路：**桶排序标记法、排序后遍历去重、STL容器自动处理**。  
**核心难点**在于高效实现去重与有序输出。桶排序以O(n)时间标记存在性，直接遍历值域输出，无需显式排序，最优于小范围数值。而排序+遍历或STL方法虽通用，但时间略高。  

**可视化设计**：  
- **桶数组标记**：用像素块表示1~1000的桶，输入时对应块亮起（颜色标记），重复输入无变化。  
- **遍历输出阶段**：指针从左至右扫描，亮起块触发音效并加入结果列表，展示“去重即保留首次标记”逻辑。  
- **复古风格**：16色像素调色板，扫描指针为8位箭头，音效采用芯片声模拟标记与完成音。

### **最优思路提炼**
1. **桶标记法**：  
   - 数组下标直接映射数值，标记存在性，遍历值域输出非空项，天然有序。  
   - **适用场景**：数值范围有限（如本题1~1000）。  

2. **STL容器法**：  
   - `set`或`unordered_set`（后需排序）自动去重，适合数值范围大但数据量小的场景。

#include<bits/stdc++.h>

int main() {
    bool book[1001] = {0};
    int n, x, cnt = 0;

        cin >> x;
        if (!book[x]) cnt++;
        book[x] = 1;
    }

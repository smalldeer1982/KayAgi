# 题目信息

# [NOIP 2006 普及组] 明明的随机数

## 题目描述

明明想在学校中请一些同学一起做一项问卷调查，为了实验的客观性，他先用计算机生成了 $N$ 个 $1$ 到 $1000$ 之间的随机整数 $(N\leq100)$，对于其中重复的数字，只保留一个，把其余相同的数去掉，不同的数对应着不同的学生的学号。然后再把这些数从小到大排序，按照排好的顺序去找同学做调查。请你协助明明完成“去重”与“排序”的工作。


## 说明/提示

NOIP 2006 普及组 第一题


## 样例 #1

### 输入

```
10
20 40 32 67 40 20 89 300 400 15
```

### 输出

```
8
15 20 32 40 67 89 300 400
```

# AI分析结果



## 唯一算法分类
**桶排序/哈希标记法**

## 综合分析与结论
### 核心算法流程
题目要求同时完成去重与排序，核心逻辑可分解为：
1. **标记存在性**：通过哈希表/布尔数组记录每个数值是否出现过（桶排序思想）
2. **顺序遍历输出**：由于数值范围限定在1-1000，遍历该范围即可得到有序结果

### 可视化设计要点
采用 **8位像素风格** 展示算法过程：
- **桶数组可视化**：用1000个像素方块表示桶，初始为灰色（未激活）
- **输入阶段**：每个输入数值触发对应像素块闪烁（黄色光效），并永久变为绿色（已标记）
- **统计阶段**：从左到右扫描像素块，绿色方块触发蓝色高亮并计入结果列表
- **音效设计**：标记时播放短促「滴」声，统计时播放上升音阶，结果输出后播放胜利音乐

```cpp
bool book[1001] = {0}; // 标记数组
int final = 0;
for(int i=0; i<n; i++) {
    if(!book[a[i]]) final++; // 首次出现则计数
    book[a[i]] = 1;          // 永久标记
}
// 输出时直接遍历1-1000
```

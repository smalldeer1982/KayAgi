# 题目信息

# [WC2006] 水管局长

## 题目背景

SC 省 MY 市有着庞大的地下水管网络，嘟嘟是 MY 市的水管局长（就是管水管的啦）。

## 题目描述

每天供水公司可能要将一定量的水从 $u$ 处送往 $v$ 处，嘟嘟需要为供水公司找到一条从 $u$ 至 $v$ 的水管的路径，接着通过信息化的控制中心通知路径上的水管进入准备送水状态，等到路径上每一条水管都准备好了，供水公司就可以开始送水了。嘟嘟一次只能处理一项送水任务，等到当前的送水任务完成了，才能处理下一项。

在处理每项送水任务之前，路径上的水管都要进行一系列的准备操作，如清洗、消毒等等。嘟嘟在控制中心一声令下，这些水管的准备操作同时开始，但由于各条管道的长度、内径不同，进行准备操作需要的时间可能不同。

供水公司总是希望嘟嘟能找到这样一条送水路径，路径上的所有管道全都准备就绪所需要的时间尽量短。嘟嘟希望你能帮助他完成这样的一个选择路径的系统，以满足供水公司的要求。另外，由于 MY 市的水管年代久远，一些水管会不时出现故障导致不能使用，你的程序必须考虑到这一点。

不妨将 MY 市的水管网络看作一幅简单无向图（即没有自环或重边）：水管是图中的边，水管的连接处为图中的结点。整张图共有 $n$ 个节点和 $m$ 条边，节点从 $1$ 至 $n$ 编号。


## 说明/提示

#### 数据规模与约定

对于全部的测试点，保证：

- $1 \leq n \leq 10^3$，$1 \leq m, q \leq 10^5$。
- $1 \leq k \leq 2$，$1 \leq u, v \leq n$，$1 \leq t \leq 10^9$。
- 给出的图无重边无自环，保证在宣布一条水管报废之前，该水管一定存在于图上且没有报废，
- 宣布报废的水管不超过 $5 \times 10^3$ 条，且在任意时刻，图一定是联通的。

## 题解思路与解决难点

### 核心思路
1. **离线倒序处理**：将删边操作转化为加边，避免动态删边的复杂性
2. **LCT维护最小生成树**：使用LCT维护动态MST，支持路径最大值查询与边权替换
3. **边转点技巧**：将边视为带权节点（边权存储为点权），通过`link(u, edge_node)`和`link(edge_node, v)`建立连接

### 关键步骤
```cpp
// 加边操作伪代码
void add_edge(u, v, w) {
    if u和v不连通:
        link(u, edge_node)
        link(v, edge_node)
    else:
        find_max_edge = query_max(u, v)
        if w < find_max_edge.w:
            cut_max_edge()
            link(u, new_edge_node)
            link(v, new_edge_node)
}
```

### 难点突破
- **环处理**：加边后形成的环中，通过LCT快速找到最大权边并替换
- **边权维护**：将边转化为带权节点进行动态管理
- **离线映射**：使用map存储边编号，快速定位操作对应的边

## 最优技巧提炼

### 关键算法
1. **动态MST维护**：通过LCT维护最小生成树性质，保证每次操作后仍是MST
2. **离线时间逆转**：将破坏性操作转化为构造性操作，简化处理逻辑

### 实现技巧
- **边转点存储**：用`n+i`节点表示第i条边，解决LCT无法直接存边权的问题
- **双关键字排序**：先按边权排序建MST，再按字典序映射边编号

## 核心代码实现
```cpp
// LCT维护边权最大值关键代码
struct LCT {
    int ch[MAXN][2], fa[MAXN], rev[MAXN];
    int val[MAXN], mx[MAXN]; // mx存储当前子树最大节点编号

    void pushup(int x) {
        mx[x] = x;
        if(val[mx[ch[x][0]]] > val[mx[x]]) mx[x] = mx[ch[x][0]];
        if(val[mx[ch[x][1]]] > val[mx[x]]) mx[x] = mx[ch[x][1]];
    }
    
    void cut(int x, int y) {
        makeroot(x), access(y), splay(y);
        fa[x] = ch[y][0] = 0;
        pushup(y);
    }
    
    int query(int x, int y) {
        makeroot(x), access(y), splay(y);
        return val[mx[y]];
    }
};
```

## 调试心得
1. **边排序陷阱**：初始按(u,v)字典序排序导致二分查找错误，改为优先按边权排序
2. **LCT节点初始化**：忘记设置`mx[i] = i`导致查询错误
3. **map去重处理**：对(u,v)和(v,u)进行统一存储，避免重复键问题

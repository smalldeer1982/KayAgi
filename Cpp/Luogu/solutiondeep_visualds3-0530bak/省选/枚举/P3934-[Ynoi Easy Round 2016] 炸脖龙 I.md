## 题目描述

您正在打 galgame，然后您觉得这个 gal 不知所云，于是您弃坑了，开始写数据结构题：

给一个长为 $n$ 的序列，$m$ 次操作，每次操作：

1. 区间 $[l,r]$ 加 $x$；
2. 对于区间 $[l,r]$，查询：

$$a[l]^{a[l+1]^{a[l+2]^{\dots ^{a[r]}}}} \mod p$$

## 说明/提示

Idea：nzhtl1477，Solution：nzhtl1477，Code：nzhtl1477，Data：nzhtl1477

对于100%的数据，$n , m \le 500000$ , 序列中每个数在$[1,2\cdot 10^9]$内，$p \le 2 \cdot 10^7 $, 每次加上的数在$[0,2\cdot 10^9]$内

共10组数据

## 综合分析与结论  

### 核心算法流程  
1. **扩展欧拉定理递归**  
   利用公式 $a^b \equiv a^{b\ \text{mod}\ \varphi(p) + \varphi(p)} \ (\text{mod}\ p)$ 递归处理指数塔，最多递归 $\log p$ 层  
2. **暴力计算前5层**  
   当指数塔长度超过5层时，直接视为超过 $\varphi(p)$，避免复杂判断  
3. **树状数组维护区间加**  
   通过差分树状数组实现 $O(\log n)$ 的区间加和单点查询  

### 解决难点对比  
| 题解方案             | 欧拉定理应用                      | 指数判断方法                  | 代码实现特点                |
|----------------------|-----------------------------------|-----------------------------|---------------------------|
| Leap_Frog（奈芙莲树） | 预处理前5层暴力判断               | 直接计算前5层数值            | 树状数组+递归，代码最简洁  |
| NaCly_Fish           | 结构体存储是否超过模数            | 快速幂内部标记是否取模       | 结构体返回双值，逻辑清晰    |
| zcysky               | 预处理phi数组后线性处理           | 判断递归层数是否超过log p    | 树状数组+循环，避免递归爆栈 |

### 1. Leap_Frog（★★★★☆）
**亮点**：  
- 首创「奈芙莲树」命名，代码仅70行  
- 预处理前5层暴力判断，避免复杂逻辑  
```cpp
int ren(int l,int r,int P) {
    if(a[l]%P==0) return 0;
    int ls = min(l+5,r); // 关键！暴力前5层
    for(int i=l+1; i<=ls; i++) 
        if(a[i]==1) { ls=i; break; }
    //...后续递归
}
```

### 2. NaCly_Fish（★★★★★）  
**亮点**：  
- 使用 `node` 结构体同时返回值和是否超过模数  
- 详细推导扩展欧拉定理应用场景  
```cpp
struct node { int val; bool flag; };
node power(int a, int t, int p) {
    node res = node(1,0);
    if(a >= p) res.flag = 1, a %= p;
    //...快速幂过程
}
```

### 3. zcysky（★★★★☆）  
**亮点**：  
- 线性处理phi数组避免递归爆栈  
- 完整的时间复杂度证明  
```cpp
int query(int l, int r, int p) {
    int now = l;
    while(p > 1 && now < r) { 
        modn[++now] = phi[p];
        p = phi[p];
    } // 线性预处理phi链
}
```

   ```cpp
       phi[1] = 1;
       for(int i=2; i<=2e7; i++) {
           if(!vis[i]) prime[++cnt]=i, phi[i]=i-1;
           for(int j=1; j<=cnt && i*prime[j]<=2e7; j++) {
               //...筛法计算phi
           }
       }
   }
   ```

2. **递归终止条件优化**  
   - 当 $p=1$ 时直接返回0  
   - 遇到 $a_i=1$ 时提前终止指数计算  

3. **树状数组差分技巧**  
   维护差分数组实现区间加：
   ```cpp
   void add(int l, int r, int v) {
       update(l, v);

   }
   ```

## 个人心得摘录
> **Leap_Frog调试经历**：  
> “快速幂的底数必须一进入就取模，为此调了整整3小时...原来以为 $a^b \text{ mod } p$ 直接计算即可，结果当 $a$ 本身超过 $p$ 时，未取模直接计算会导致后续判断错误”

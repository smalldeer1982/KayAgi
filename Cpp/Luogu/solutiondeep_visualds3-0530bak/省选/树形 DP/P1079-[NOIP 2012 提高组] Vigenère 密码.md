# [NOIP 2012 提高组] Vigenère 密码

## 题目描述

16 世纪法国外交家 Blaise de Vigenère 设计了一种多表密码加密算法 Vigenère 密码。Vigenère 密码的加密解密算法简单易用，且破译难度比较高，曾在美国南北战争中为南军所广泛使用。

 
在密码学中，我们称需要加密的信息为明文，用 $M$ 表示；称加密后的信息为密文，用 $C$ 表示；而密钥是一种参数，是将明文转换为密文或将密文转换为明文的算法中输入的数据，记为 $k$。在 Vigenère 密码中，密钥 $k$ 是一个字母串，$k=k_1,k_2,…,k_n$。当明文 $M=m_1,m_2,…,m_n$ 时，得到的密文 $C=c_1,c_2,…,c_n$，其中 $c_i$=$m_i \operatorname{\circledR} k_i$，运算 $\circledR$ 的规则如下表所示：

Vigenère 加密在操作时需要注意：

1. $\circledR$ 运算忽略参与运算的字母的大小写，并保持字母在明文 $M$ 中的大小写形式；
2. 当明文 $M$ 的长度大于密钥 $k$ 的长度时，将密钥 $k$ 重复使用。

例如，明文 $M=\texttt{Helloworld}$，密钥 $k=\texttt{abc}$ 时，密文 $C=\texttt{Hfnlpyosnd}$。

$$

\textsf{密文} & \tt H & \tt f & \tt n & \tt l & \tt p & \tt y & \tt o & \tt s & \tt n & \tt d \\  \hline

$$

## 综合分析与结论

### 核心算法流程
1. **字符数值转换**：将字母统一映射到 0-25 的数值（A/a→0，B/b→1，...，Z/z→25）
2. **偏移量计算**：`明文 = (密文字符数值 - 密钥字符数值 + 26) % 26`
3. **循环密钥**：通过取模运算实现密钥的循环使用 `key[i % key_len]`
4. **大小写保留**：根据密文字符的大小写决定明文字符的大小写形式

### 可视化设计思路
- **动态字符高亮**：用不同颜色区分当前处理的密文字符、对应的密钥字符、生成的明文字符
- **数值转换动画**：在字符旁显示数值转换过程（如 'Y'→24，'C'→2）
- **模运算演示**：展示 `(24-2+26)%26=22→'W'` 的计算过程
- **密钥循环提示**：用进度条形式展示密钥重复使用过程

## 题解清单（≥4星）

1. **ingu（⭐⭐⭐⭐⭐）**
   - 亮点：位运算取字符顺序 `x&31` 处理大小写，三目运算符优化负数修正
   - 代码行数最少（仅9行核心代码）

2. **qhr2023（⭐⭐⭐⭐）**
   - 亮点：统一公式 `(c-k+26)%26` 处理所有情况，函数式封装解密逻辑
   - 支持任意大小写组合的场景

## 核心代码实现

### ingu 题解关键片段
```cpp
for (int i=0;i<c.length();i++) {
    int t=(k[i%k.length()]&31)-1; // 取密钥字符顺序值
    c[i]=(c[i]&31)-t>0 ? c[i]-t : c[i]-t+26;
}
```

### 通用解密函数
```cpp
char decrypt(char c, char k) {
    bool is_upper = isupper(c);
    int c_num = tolower(c) - 'a';
    int k_num = tolower(k) - 'a';
    int m_num = (c_num - k_num + 26) % 26;
    return is_upper ? 'A'+m_num : 'a'+m_num;
}
```

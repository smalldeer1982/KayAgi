# [NOIP 2012 提高组] Vigenère 密码

## 题目描述

16 世纪法国外交家 Blaise de Vigenère 设计了一种多表密码加密算法 Vigenère 密码。Vigenère 密码的加密解密算法简单易用，且破译难度比较高，曾在美国南北战争中为南军所广泛使用。

 
在密码学中，我们称需要加密的信息为明文，用 $M$ 表示；称加密后的信息为密文，用 $C$ 表示；而密钥是一种参数，是将明文转换为密文或将密文转换为明文的算法中输入的数据，记为 $k$。在 Vigenère 密码中，密钥 $k$ 是一个字母串，$k=k_1,k_2,…,k_n$。当明文 $M=m_1,m_2,…,m_n$ 时，得到的密文 $C=c_1,c_2,…,c_n$，其中 $c_i$=$m_i \operatorname{\circledR} k_i$，运算 $\circledR$ 的规则如下表所示：

Vigenère 加密在操作时需要注意：

1. $\circledR$ 运算忽略参与运算的字母的大小写，并保持字母在明文 $M$ 中的大小写形式；
2. 当明文 $M$ 的长度大于密钥 $k$ 的长度时，将密钥 $k$ 重复使用。

例如，明文 $M=\texttt{Helloworld}$，密钥 $k=\texttt{abc}$ 时，密文 $C=\texttt{Hfnlpyosnd}$。

$$

\textsf{密钥} & \tt a & \tt b & \tt c & \tt a & \tt b & \tt c & \tt a & \tt b & \tt c & \tt a \\  \hline
\textsf{密文} & \tt H & \tt f & \tt n & \tt l & \tt p & \tt y & \tt o & \tt s & \tt n & \tt d \\  \hline
\end{array}
$$

### 题解思路与解决难点
#### 核心思路对比
1. **ASCII 偏移法**（ingu、ShirasuAzusaSaikou 等）  
   - 核心公式：`明文 = (密文字符 - 密钥字符偏移 + 26) % 26`  
   - 关键点：  
     - 通过 `k[i%k_len]` 实现密钥循环  
     - 利用 ASCII 码后 5 位特性简化计算（`x & 31` 得到字母序号）  
     - 大小写统一处理：保持原字符大小写，仅计算偏移量  

2. **斜行加和法**（huluobotou）  
   - 预计算斜行加和数组 `x[]`，利用 `x[密文字符序号] - 密钥字符序号` 得到明文  
   - 缺点：预处理复杂且代码冗余，实际效率低于直接偏移法  

3. **查表法**（LightCoder）  
   - 构造真·密码表矩阵，直接查表解密  
   - 实现繁琐，需处理队列操作和大小写判断  

#### 解决难点
- **密钥循环**：所有题解均通过 `i % k_len` 实现  
- **大小写处理**：  
  - 统一转小写计算偏移，最终根据密文字符大小写还原（如 `(c[i] >= 'a' ? 'a' : 'A')`）  
  - 或通过 `isupper()` 函数动态判断  
- **负数修正**：`+26` 后取模保证结果非负

### 最优思路提炼
1. **核心公式**：  
   `明文 = (密文字符 - 密钥字符偏移 + 26) % 26`  
   - 密钥偏移：`密钥字符转小写后 - 'a'`  
   - 负数修正：`+26` 保证结果在 `[0,25]`  

2. **代码优化技巧**：  
   - 使用 `k[i % k_len]` 自动循环密钥  
   - 统一转小写计算偏移，保留原字符大小写

### 完整代码示例（ingu 风格）
```cpp
#include <iostream>
using namespace std;
int main() {
    string k, c;
    cin >> k >> c;
    int k_len = k.length();
    for (int i = 0; i < c.length(); i++) {
        int offset = (k[i % k_len] & 31) - 1; // 统一转大写计算偏移
        char decrypted = c[i] - offset;
        if ((c[i] & 31) <= offset) decrypted += 26; // 处理负数
        cout << decrypted;
    }
    return 0;
}
```

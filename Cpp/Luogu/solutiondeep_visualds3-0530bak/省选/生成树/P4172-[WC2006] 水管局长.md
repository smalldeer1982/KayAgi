# 题目信息

# [WC2006] 水管局长

## 题目背景

SC 省 MY 市有着庞大的地下水管网络，嘟嘟是 MY 市的水管局长（就是管水管的啦）。

## 题目描述

每天供水公司可能要将一定量的水从 $u$ 处送往 $v$ 处，嘟嘟需要为供水公司找到一条从 $u$ 至 $v$ 的水管的路径，接着通过信息化的控制中心通知路径上的水管进入准备送水状态，等到路径上每一条水管都准备好了，供水公司就可以开始送水了。嘟嘟一次只能处理一项送水任务，等到当前的送水任务完成了，才能处理下一项。

在处理每项送水任务之前，路径上的水管都要进行一系列的准备操作，如清洗、消毒等等。嘟嘟在控制中心一声令下，这些水管的准备操作同时开始，但由于各条管道的长度、内径不同，进行准备操作需要的时间可能不同。

供水公司总是希望嘟嘟能找到这样一条送水路径，路径上的所有管道全都准备就绪所需要的时间尽量短。嘟嘟希望你能帮助他完成这样的一个选择路径的系统，以满足供水公司的要求。另外，由于 MY 市的水管年代久远，一些水管会不时出现故障导致不能使用，你的程序必须考虑到这一点。

不妨将 MY 市的水管网络看作一幅简单无向图（即没有自环或重边）：水管是图中的边，水管的连接处为图中的结点。整张图共有 $n$ 个节点和 $m$ 条边，节点从 $1$ 至 $n$ 编号。


## 说明/提示

#### 数据规模与约定

对于全部的测试点，保证：

- $1 \leq n \leq 10^3$，$1 \leq m, q \leq 10^5$。
- $1 \leq k \leq 2$，$1 \leq u, v \leq n$，$1 \leq t \leq 10^9$。
- 给出的图无重边无自环，保证在宣布一条水管报废之前，该水管一定存在于图上且没有报废，
- 宣布报废的水管不超过 $5 \times 10^3$ 条，且在任意时刻，图一定是联通的。

## 【唯一算法分类】  
**LCT (Link-Cut Tree)**  

---

## 【综合分析与结论】  
**核心思路**：  
1. **逆序处理**：将删边操作转化为加边，避免动态删边的复杂性。  
2. **边转点**：将边视为带权节点，通过`link(u, edge_id)`和`link(edge_id, v)`维护边权。  
3. **动态MST**：每次加边时检查是否形成环，若形成环则删除环中最大边（保证MST性质）。  
4. **LCT维护路径最大值**：利用LCT的`split`操作快速查询路径最大边权。  

**核心难点**：  
- **边权处理**：LCT默认维护点权，需将边转化为带权节点。  
- **动态维护MST**：需在加边时高效找到环并替换最大边。  
- **离线处理与映射**：逆序操作需用`map`记录边编号，避免重复查找。  

**可视化设计思路**：  
1. **动画流程**：  
   - **初始化**：展示逆序后的最终MST结构（像素网格表示节点与边）。  
   - **加边操作**：高亮新边（红色），分裂路径（蓝色路径），替换最大边（黄色闪烁）。  
   - **查询操作**：显示路径遍历过程，当前最大边高亮（绿色）。  
2. **复古风格**：  
   - **颜色方案**：节点用8位色块（红/蓝/绿），边权用灰阶表示。  
   - **音效**：加边成功（短促“滴”声），替换边（“咔嚓”声），查询完成（上升音调）。  
3. **AI自动演示**：按逆序步骤自动执行加边/查询，用户可暂停/单步观察LCT结构变化。

## 【最优思路与技巧提炼】  
### 关键思路  
1. **逆序离线处理**：将动态删边转化为静态加边，简化问题。  
2. **边权转点权**：通过中间节点存储边权，实现LCT对边权的维护。  
3. **环检测与替换**：加边时若形成环，删除最大边以保持MST性质。  

### 代码实现片段  
```cpp  
// 边转点核心操作（fy0123题解）  
void link(int x, int y, int id) {  
    makeroot(x);  
    if (findroot(y) != x) {  
        link(x, id + n);  
        link(id + n, y);  
    } else {  
        int mx_id = query_max_edge(x, y);  
        if (e[id].w < e[mx_id - n].w) {  
            cut(e[mx_id - n].u, mx_id);  
            cut(mx_id, e[mx_id - n].v);  
            link(x, id + n);  
            link(id + n, y);  
        }  
    }  
}  
```

## 【个人心得摘录】  
- **逆序处理是关键**：将不可逆操作（删边）转为可逆操作（加边），极大简化逻辑。  
- **边权转点权陷阱**：需注意节点编号分配，避免与原始节点冲突（如`id + n`）。  
- **LCT调试技巧**：通过`splay`和`access`逐步验证路径分裂的正确性。

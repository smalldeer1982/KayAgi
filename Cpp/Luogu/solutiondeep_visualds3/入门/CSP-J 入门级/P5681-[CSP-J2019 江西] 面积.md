# 题目信息

# [CSP-J2019 江西] 面积

## 题目描述

Alice 有一个边长为 $a$ 的正方形，Bob 有一个长宽分别为 $b,c$ 的矩形，请你告诉他们俩谁的图形面积更大。

## 说明/提示

【数据范围】   
对于 $30\%$ 的数据，$1 \le a,b,c \le 100$，$b=c$；  
对于 $80\%$ 的数据，$1\le a,b,c \le 10^4$；  
对于 $100\%$ 的数据，$1\le a,b,c \le 10^9$。

【样例 $1$ 解释】  
正方形面积为  $25$，矩形面积为 $24$。  

【样例 $2$ 解释】  
正方形面积为  $49$，矩形面积为 $50$。  

## 样例 #1

### 输入

```
5 4 6```

### 输出

```
Alice```

## 样例 #2

### 输入

```
7 5 10```

### 输出

```
Bob```

### 3. 核心难点辨析与解题策略

1. **整数溢出陷阱**  
   * **分析**：当 \(a, b, c \le 10^9\) 时，\(a^2\) 或 \(b \times c\) 可能高达 \(10^{18}\)。`int` 仅支持到约 \(2^{31}-1\)（\(2 \times 10^9\)），必须用 `long long` 存储变量和结果。  
   * 💡 **学习笔记**：十年OI一场空，不开 `long long` 见祖宗！

2. **面积比较的边界条件**  
   * **分析**：题目明确要求正方形面积严格大于矩形时才输出 "Alice"，相等或小于均输出 "Bob"。部分题解通过特判提前处理（如ZolaWatle），但直接比较更简洁。  
   * 💡 **学习笔记**：严格遵循题目输出描述，避免想当然。

3. **类型转换技巧**  
   * **分析**：混合使用 `int` 和 `long long` 时，乘前需显式转换（如 `1ll * a * a`）。否则 `int * int` 仍按 `int` 计算，导致溢出后赋给 `long long`。  
   * 💡 **学习笔记**：在运算前升级数据类型，而非依赖赋值隐式转换。

#### ✨ 解题技巧总结
- **技巧1（预见性数据类型选择）**：面对大数运算，优先评估结果范围，主动选用 `long long` 或 `unsigned long long`。  
- **技巧2（边界条件优先处理）**：先厘清比较逻辑中的等值情况，避免后续调试耗时。  
- **技巧3（代码极简主义）**：在保证可读性前提下，善用三目运算符等语法减少冗余代码。

### 4. C++核心代码实现赏析

**本题通用核心C++实现参考**  
* **说明**：综合优质题解思路，突出数据类型安全与边界处理。  
* **完整核心代码**：  
  ```cpp
  #include <iostream>
  using namespace std;
  int main() {
      long long a, b, c;
      cin >> a >> b >> c;
      long long square = a * a;
      long long rectangle = b * c;
      if (square > rectangle) cout << "Alice";
      else cout << "Bob";
      return 0;
  }
  ```
* **代码解读概要**：  
  > 输入边长 \(a\) 和矩形尺寸 \(b, c\) → 显式用 `long long` 存储面积结果 → 直接比较后输出。变量名 `square`/`rectangle` 增强可读性。

**题解一（ctq1999）片段赏析**  
* **亮点**：`1ll` 显式类型转换避免中间溢出  
* **核心代码片段**：  
  ```cpp

  S2 = 1ll * b * c;

* 💡 **学习笔记**：乘前类型转换是处理混合运算溢出的黄金法则。

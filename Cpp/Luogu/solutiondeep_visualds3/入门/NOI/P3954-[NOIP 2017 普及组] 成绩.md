# 题目信息

# [NOIP 2017 普及组] 成绩

## 题目背景

NOIP2017 普及组 T1

## 题目描述

牛牛最近学习了 C++ 入门课程，这门课程的总成绩计算方法是：

$$
\text{总成绩}=\text{作业成绩}\times 20\%+\text{小测成绩}×30\%+\text{期末考试成绩} \times 50\%
$$

牛牛想知道，这门课程自己最终能得到多少分。


## 说明/提示

**样例 1 说明**

牛牛的作业成绩是 $100$ 分，小测成绩是 $100$ 分，期末考试成绩是 $80$ 分，总成绩是 $100 \times 20\%+100 \times 30\%+80 \times 50\%=20+30+40=90$。

**样例 2 说明**

牛牛的作业成绩是 $60$ 分，小测成绩是 $90$ 分，期末考试成绩是 $80$ 分，总成绩是 $60 \times 20\%+90 \times 30\%+80 \times 50\%=12+27+40=79$。

**数据说明**

对于 $30\%$ 的数据，$A=B=0$。

对于另外 $30\%$ 的数据，$A=B=100$。

对于 $100\%$ 的数据，$0≤A,B,C≤100$ 且 $A,B,C$ 都是 $10$ 的整数倍。

## 样例 #1

### 输入

```
100 100 80 ```

### 输出

```
90```

## 样例 #2

### 输入

```
60 90 80 ```

### 输出

```
79```

#### 3. 核心难点辨析与解题策略
1. **浮点精度陷阱**  
   *分析*：浮点运算如`0.2×60`可能存储为11.999999。优质解均用整数运算规避，通过预乘系数再除10保证精确整除  
   💡 **学习笔记**：当输入有倍数特性时，整数运算永保精度安全

2. **运算顺序优化**  
   *分析*：`a/5`与`a/10×2`数学等价，但后者避免大数溢出风险（虽本题数据范围小）。StudyingFather的乘加顺序最易理解  
   💡 **学习笔记**：乘除混合时，先乘后除可减少中间值误差

3. **输出格式控制**  
   *分析*：浮点解需显式取整。石破天惊的`printf("%.0f")`会四舍五入，而`(int)`强制截断更符合题意  
   💡 **学习笔记**：输出整数时，明确指定取整方式比依赖默认转换更可靠

✨ **解题技巧总结**  
- **数据特性转化**：利用"10的倍数"特性将浮点问题转化为整数问题  
- **表达式分解**：复杂权重可拆解为基本运算组合（如30%=3/10）  
- **防御性输出**：浮点结果必须显式取整，并用边界值（0/100）验证

#### 4. C++核心代码实现赏析
**通用核心实现**  
```cpp
#include <cstdio>
int main() {
    int a, b, c;
    scanf("%d%d%d", &a, &b, &c);
    printf("%d", (a*2 + b*3 + c*5)/10); // 整数运算典范
    return 0;
}
```
* **说明**：综合自StudyingFather解法，规避浮点误差的最优方案  
* **解读概要**：读取三成绩→按权重系数(2,3,5)线性组合→除以10输出整数。全程整型运算，时间复杂度O(1)

**题解一片段（StudyingFather）**  
```cpp
score = (a * 2 + b * 3 + c * 5) / 10;
```
> **解读**：为什么权重用整数？因为20%=2/10，30%=3/10，50%=5/10。分子相加后除10，恰似把百分比换算统一为分母10的分数相加  
> 💡 **学习笔记**：系数选择本质是通分过程，使分母统一为10

**题解二片段（Reywmp）**  
```cpp
int ans = a/5 + b*3/10 + c/2;
```
> **解读**：将系数分解为基本运算：20%=1/5，50%=1/2。注意`b*3/10`必须写为乘法优先，若写`b/10*3`会因整除损失精度  
> 💡 **学习笔记**：整数除法顺序影响结果，乘除混合时乘法在前

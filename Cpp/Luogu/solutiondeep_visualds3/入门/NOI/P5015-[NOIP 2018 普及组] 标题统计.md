# [NOIP 2018 普及组] 标题统计

## 题目背景

NOIP2018 普及组 T1

## 题目描述

凯凯刚写了一篇美妙的作文，请问这篇作文的标题中有多少个字符？ 注意：标题中可能包含大、小写英文字母、数字字符、空格和换行符。统计标题字符数时，空格和换行符不计算在内。 

## 说明/提示

**样例 1 说明**   

标题中共有 3 个字符，这 3 个字符都是数字字符。   

**样例 2 说明**

标题中共有 $ 5$ 个字符，包括 $1$ 个大写英文字母， $1$ 个小写英文字母和 $2$ 个数字字符， 还有 $1$ 个空格。由于空格不计入结果中，故标题的有效字符数为 $4$ 个。 

**数据规模与约定**

规定 $|s|$ 表示字符串 $s$ 的长度（即字符串中的字符和空格数）。    
对于 $40\%$ 的数据，$1 ≤ |s| ≤ 5$，保证输入为数字字符及行末换行符。   
对于 $80\%$ 的数据，$1 ≤ |s| ≤ 5$，输入只可能包含大、小写英文字母、数字字符及行末换行符。   
对于 $100\%$ 的数据，$1 ≤ |s| ≤ 5$，输入可能包含大、小写英文字母、数字字符、空格和行末换行符。 

## 样例 #1

### 输入

```
234 ```

### 输出

```
3```

## 样例 #2

### 输入

```
Ca 45 ```

### 输出

```
4```

## 3. 核心难点辨析与解题策略

1.  **难点：正确处理含空格的输入**
    * **分析**：普通cin在空格处停止读取，就像水遇到油墨会分流。优质解法采用两种破解之道：getline读取整行（如吸水海绵），或利用cin自动跳过空格的特性（如磁铁只吸金属）
    * 💡 **学习笔记**：输入含空格时，getline和cin特性是两把不同的钥匙

2.  **难点：准确排除无效字符**
    * **分析**：题目要求忽略空格和换行符，就像数苹果时要剔除烂果。getline方案中需主动过滤空格；cin方案因自动跳过而天然过滤。关键在理解getline已去除换行符，只需额外处理空格
    * 💡 **学习笔记**：getline去换行，cin跳空格——根据需求选工具

3.  **难点：小规模数据的特殊处理**
    * **分析**：当数据规模很小（|s|≤5）时，可以像题解1那样用分支代替循环，如同用手工雕刻代替机器量产。这种优化建立在对题目约束的敏锐观察上
    * 💡 **学习笔记**：数据规模是隐藏的提示卡

### ✨ 解题技巧总结
-   **技巧A：输入特性利用** - 深刻理解cin/getline的行为差异，根据场景选择最优工具
-   **技巧B：逆向思维应用** - 与其思考"保留什么"，不如思考"排除什么"（如空格）
-   **技巧C：规模导向优化** - 小规模数据可用分支替代循环，大幅简化代码逻辑

## 4. C++核心代码实现赏析

**本题通用核心C++实现参考**
* **说明**：综合优质题解精髓，采用最简洁高效的cin特性解法
* **完整核心代码**：
    ```cpp
    #include <iostream>
    using namespace std;
    
    int main() {
        char c;
        int ans = 0;
        while (cin >> c) { // 魔法发生地：cin自动跳过空格/换行
            ans++;        // 每次读取的都是有效字符
        }
        cout << ans;
        return 0;
    }
    ```
* **代码解读概要**：
    > 此代码像精密的齿轮组：1) `cin>>c`在循环条件中完成字符读取与空白跳过 2) 每读入一个有效字符，`ans`计数器自增 3) 循环结束时输出总数。全程无需显式处理空格，完美利用语言特性

---
**题解一 (Lhc_fl)**
* **亮点**：数据规模洞察力，化循环为分支
* **核心代码片段**：
    ```cpp
    int ans=0;
    char c;
    if(cin>>c)ans++; 
    if(cin>>c)ans++;  // 五个if如同五道安检门
    if(cin>>c)ans++;  // 只允许有效字符通过
    if(cin>>c)ans++;
    if(cin>>c)ans++;
    ```
* **代码解读**：
    > 每个`if`都是一次独立尝试：`cin>>c`尝试读取非空白字符，成功则条件为真执行`ans++`。为什么能这样？因为题目保证最多5个字符！如同知道迷宫只有5个房间，我们直接检查每个房间而不用地图
* 💡 **学习笔记**：有限数据规模是简化代码的黄金机会

**题解二 (Malkin_Moonlight)**
* **亮点**：多解法教学，输入处理大师课
* **核心代码片段**：
    ```cpp
    string s;
    getline(cin, s);  // 吸管吸取整行内容
    int ans = 0;
    for(int i=0; i<s.size(); i++) {
        if (s[i] != ' ') ans++;  // 手动过滤空格
    }
    ```
* **代码解读**：
    > `getline`像吸管一样吸取整行输入存入`s`，包括空格但不含换行符。遍历字符串时，`s[i]!=' '`如同筛子滤掉空格颗粒，只保留有效字符计入`ans`。注意：为什么不用处理换行符？因为getline已帮我们去除！
* 💡 **学习笔记**：getline+过滤是处理含空格输入的经典组合拳

**题解三 (cff_0102)**
* **亮点**：极致简洁，语言特性妙用
* **核心代码片段**：
    ```cpp
    int s=0;
    char c;
    while(cin>>c) s++;  // 循环读取直到尽头
    ```
* **代码解读**：
    > `while(cin>>c)`构成简洁有力的循环：每次读取跳过空白字符，将有效字符存入`c`，同时循环持续。`s++`如同机械计数器般精准累加。当无更多字符时，cin返回false结束循环
* 💡 **学习笔记**：cin的特性让计数器无需关心空白符

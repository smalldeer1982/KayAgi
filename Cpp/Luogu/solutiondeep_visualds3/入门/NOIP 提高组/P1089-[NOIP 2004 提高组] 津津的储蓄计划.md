# [NOIP 2004 提高组] 津津的储蓄计划

## 题目描述

津津的零花钱一直都是自己管理。每个月的月初妈妈给津津 $300$ 元钱，津津会预算这个月的花销，并且总能做到实际花销和预算的相同。

为了让津津学习如何储蓄，妈妈提出，津津可以随时把整百的钱存在她那里，到了年末她会加上 $20\%$ 还给津津。因此津津制定了一个储蓄计划：每个月的月初，在得到妈妈给的零花钱后，如果她预计到这个月的月末手中还会有多于 $100$ 元或恰好 $100$ 元，她就会把整百的钱存在妈妈那里，剩余的钱留在自己手中。


例如 $11$月初津津手中还有 $83$ 元，妈妈给了津津 $300$ 元。津津预计$11$月的花销是 $180$ 元，那么她就会在妈妈那里存 $200$ 元，自己留下 $183$ 元。到了 $11$ 月月末，津津手中会剩下 $3$ 元钱。


津津发现这个储蓄计划的主要风险是，存在妈妈那里的钱在年末之前不能取出。有可能在某个月的月初，津津手中的钱加上这个月妈妈给的钱，不够这个月的原定预算。如果出现这种情况，津津将不得不在这个月省吃俭用，压缩预算。


现在请你根据 $2004$ 年 $1$ 月到 $12$ 月每个月津津的预算，判断会不会出现这种情况。如果不会，计算到 $2004$ 年年末，妈妈将津津平常存的钱加上 $20\%$ 还给津津之后，津津手中会有多少钱。

## 3. 核心难点辨析与解题策略

### 1. 关键点1：流程顺序不能错！  
**问题**：如果先减花销再加300，会误判“钱不够”（比如某月光花销290元，先减的话`money`会变-290，但加300后实际是够的）。  
**解决**：严格按题目顺序写代码——`money +=300 → money -= cost`。所有优质题解都遵守这个顺序！  
💡 学习笔记：模拟题的“顺序”比“聪明”更重要，顺序错了全错！

### 2. 关键点2：如何计算“整百的钱”？  
**问题**：比如203元要存200元，不是100元，怎么算？  
**解决**：用整数除法和取余——`deposit = (money / 100) * 100`（整百部分），`money -= deposit`（留下零头）。比如`203/100=2`，`2*100=200`，刚好是要存的钱。  
💡 学习笔记：整百计算的公式是“`(x//100)*100`存起来，`x%100`留着”。

### 3. 关键点3：年末利息避免浮点数误差！  
**问题**：`save*1.2`可能出现误差（比如100*1.2=120，但浮点数可能算成119.999...）。  
**解决**：用整数运算——因为存的钱是100的倍数，所以`save*120/100`（比如100*120=12000，÷100=120），完全没有误差！  
💡 学习笔记：涉及金钱的计算，优先用整数运算！

### ✨ 解题技巧总结  
1. **先写流程大纲**：把每个月的步骤列出来（加钱→花钱→存钱），再转成代码，避免混乱。  
2. **变量名“见名知意”**：用`money`代替`n`，用`save`代替`s`，调试时不容易错。  
3. **用标记处理异常**：比如`flag`标记是否超支，把“异常情况”和“正常情况”分开处理。


## 4. C++核心代码实现赏析

### 本题通用核心C++实现参考  
* **说明**：综合优质题解的优点，变量名直观，流程严格，避免浮点数误差。  
* **完整核心代码**：  
```cpp
#include <iostream>
using namespace std;

int main() {
    int money = 0;  // 津津手上的钱（元）
    int save = 0;   // 存在妈妈那里的钱（元）
    for (int month = 1; month <= 12; month++) {
        int cost;
        cin >> cost;
        money += 300;  // 月初得到300元
        money -= cost; // 减去本月花销
        if (money < 0) { // 钱不够，输出月份并结束
            cout << "-" << month << endl;
            return 0;
        }
        // 存整百的钱到妈妈那里
        int deposit = (money / 100) * 100;
        save += deposit;
        money -= deposit;
    }
    // 年末计算总金额：存的钱×1.2 + 手上的钱（整数运算）
    int total = (save * 120) / 100 + money;
    cout << total << endl;
    return 0;
}
```  
* **代码解读概要**：  
  循环12个月，每步先加300元，再减花销；不够就输出月份；够的话计算整百存款（`deposit`），存到`save`里，`money`留零头。最后用`save*120/100`算利息，加上`money`输出总数。


### 题解片段赏析——wurang的“while循环存钱”  
* **亮点**：用`while`循环直观展示“一次次存100元”的过程。  
* **核心代码片段**：  
```cpp
while(n>=100) {
    n-=100;
    s+=100;
}
```  
* **代码解读**：  
  只要`n`（手上的钱）≥100，就每次减100（存给妈妈），`s`（存的钱）加100。比如`n=203`，会循环2次：第一次`n`变103，`s`变100；第二次`n`变3，`s`变200。像“一次次递百元大钞”，新手一看就懂！  
* 💡 学习笔记：`while`循环适合“重复做某件事直到条件不满足”。


### 题解片段赏析——Graph_Theory的“问号表达式”  
* **亮点**：用一句话代替`if-else`，简洁高效。  
* **核心代码片段**：  
```cpp
cout << (safe ? cash + save * 120 : -i);
```  
* **代码解读**：  
  问号表达式的格式是“`条件? 真结果 : 假结果`”。如果`safe`为真（没超支），输出`cash+save*120`；否则输出`-i`（当前月份的负数）。一句话解决两种情况，代码更短！  
* 💡 学习笔记：简单的`if-else`可以用问号表达式简化。

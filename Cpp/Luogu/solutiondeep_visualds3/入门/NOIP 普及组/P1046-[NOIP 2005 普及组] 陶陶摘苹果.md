# [NOIP 2005 普及组] 陶陶摘苹果

## 题目描述

陶陶家的院子里有一棵苹果树，每到秋天树上就会结出 $10$ 个苹果。苹果成熟的时候，陶陶就会跑去摘苹果。陶陶有个 $30$ 厘米高的板凳，当她不能直接用手摘到苹果的时候，就会踩到板凳上再试试。


现在已知 $10$ 个苹果到地面的高度，以及陶陶把手伸直的时候能够达到的最大高度，请帮陶陶算一下她能够摘到的苹果的数目。假设她碰到苹果，苹果就会掉下来。


## 说明/提示

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：模拟  

🗣️ **初步分析**：
解决“陶陶摘苹果”的关键是**模拟**——就像你玩“过家家”时模仿妈妈做饭一样，我们要按照题目规则“模仿”陶陶摘苹果的过程：先算陶陶站在板凳上能碰到的最大高度（身高+30厘米），再逐个检查10个苹果，统计高度≤这个值的数量。  

模拟的核心是“**按规则一步步执行**”，本题的规则很明确：  
1. 陶陶的最大触及高度 = 身高 + 30（板凳）；  
2. 苹果高度≤这个值 → 能摘到，计数器加1。  

**核心算法流程**：输入苹果高度→计算最大高度→遍历统计→输出结果。  
**可视化设计思路**：我们会用8位像素风模拟这个过程——红色像素块代表苹果，黄色小人是陶陶，棕色方块是板凳。每检查一个苹果，会用闪烁高亮当前苹果；能摘到的苹果会“掉下来”（向下移动动画），同时计数器数字跳一下。还会加“叮”的音效强化记忆哦！

## 3. 核心难点辨析与解题策略

<difficulty_intro>
这道题看似简单，但新手常踩3个“小坑”。结合优质题解，我帮你提炼了应对策略：
</difficulty_intro>

1.  **关键点1：题意理解——板凳高度怎么用？**  
    * **分析**：很多同学一开始会分开判断“不用板凳能不能摘到”“用了能不能摘到”，但其实**板凳是“叠加”在身高上的**——陶陶站在板凳上的最大高度是“身高+30”，不管原来能不能摘到，直接用这个值比较就行。比如题解里的`H += 30`或`y = x + 30`，都是先算好最大高度再比较。  
    * 💡 **学习笔记**：模拟题的第一步是“把题目规则转化为数学表达式”！

2.  **关键点2：统计数量——怎样写得更简洁？**  
    * **分析**：常规写法是`if (apple[i] <= h) cnt++`，但RedContritio用` s += !(H < height[i]) `把判断和累加合并了——`H < height[i]`是“不能摘到”，`!`反转后就是“能摘到”，直接加1。这种技巧能减少代码行数，还能锻炼“逻辑转数值”的思维。  
    * 💡 **学习笔记**：善用C++的“逻辑值等于1/0”的特性，可以简化统计代码！

3.  **关键点3：边界条件——刚好等于的情况要算吗？**  
    * **分析**：题目说“碰到苹果就会掉下来”，所以苹果高度**等于**陶陶最大高度时，是能摘到的。所有优质题解都用了`<=`而不是`<`，这就是边界条件的处理——千万不要漏了“等于”的情况！  
    * 💡 **学习笔记**：模拟题要仔细看题目中的“等于”“大于”等关键词，边界条件往往是得分点！

### ✨ 解题技巧总结
- **技巧1：提前计算不变量**：比如先算`H += 30`，避免循环里重复计算；  
- **技巧2：用逻辑值简化统计**：`s += 条件`比`if(条件) s++`更简洁；  
- **技巧3：边界条件要盯死**：遇到“≤”“≥”一定要想清楚是否包含等于的情况。

## 4. C++核心代码实现赏析

<code_intro_overall>
先看一份**通用核心实现**——它综合了所有优质题解的优点，逻辑清晰，适合入门：
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：本代码是模拟题的“标准模板”，直接对应题目规则，适合刚学C++的同学模仿。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    using namespace std;
    int main() {
        int apple[10], h, cnt = 0;  // apple存苹果高度，h是陶陶身高，cnt是计数器
        for (int i = 0; i < 10; i++) cin >> apple[i];  // 读10个苹果
        cin >> h;  // 读陶陶身高
        h += 30;  // 加上板凳高度，得到最大触及高度
        for (int i = 0; i < 10; i++) {
            if (apple[i] <= h) cnt++;  // 统计能摘到的苹果
        }
        cout << cnt;  // 输出结果
        return 0;
    }
    ```
* **代码解读概要**：代码分四步——读苹果→读身高→算最大高度→统计数量。每一步都对应题目规则，没有多余的逻辑，是模拟题的“最直接写法”。

接下来看3份优质题解的**核心片段**，学习它们的“精妙技巧”：

**题解一：RedContritio（赞327）**  
* **亮点**：用逻辑值累加代替`if`判断，代码更简洁。
* **核心代码片段**：
    ```cpp
    H += 30;
    for(int i=0;i<10;i++) s += !(H < height[i]);
    ```
* **代码解读**：  
  ① `H += 30`：先算最大高度，避免重复计算；  
  ② `!(H < height[i])`：如果`H < height[i]`（不能摘到），结果是`false`（0）；否则是`true`（1）；  
  ③ `s += ...`：直接把“能摘到的数量”加给`s`，不用写`if`！  
* 💡 **学习笔记**：逻辑值和数值的转换，是C++里超实用的小技巧！

**题解二：installb（赞28）**  
* **亮点**：用“前缀和”递推，统计更高效（适合多次查询）。
* **核心代码片段**：
    ```cpp
    int a[2001] = {0};  // a[i]表示≤i高度的苹果数
    for(i=0;i<10;i++) { cin>>b; a[b]++; }  // 统计每个高度的苹果数
    for(i=1;i<=251;i++) a[i] += a[i-1];  // 递推前缀和
    printf("%d\n", a[c]);  // c是陶陶最大高度
    ```
* **代码解读**：  
  ① `a[b]++`：比如高度150的苹果，`a[150]`加1；  
  ② `a[i] += a[i-1]`：`a[i]`等于“≤i-1的苹果数”加“i高度的苹果数”，也就是“≤i的苹果总数”；  
  ③ `a[c]`：直接取≤c的苹果数，不用遍历！  
* 💡 **学习笔记**：前缀和能把“多次查询”的时间从O(n)降到O(1)，超适合数据量大的情况！

**题解三：mairuisheng（赞15）**  
* **亮点**：代码极简，逻辑直白，入门友好。
* **核心代码片段**：
    ```cpp
    ha += 30;  // 站在凳子上的高度
    for(i=1;i<=10;i++) if(a[i]<=ha) ans++;  // 统计答案
    ```
* **代码解读**：  
  这就是“模拟题的本质”——直接按照题目说的做：先算最大高度，再逐个比较，统计数量。代码没有任何技巧，但胜在“一看就懂”，是新手的“入门模板”。  
* 💡 **学习笔记**：模拟题的第一要务是“正确”，再追求“简洁”！

<insights_intro>

> **参考经验 (来自 RedContritio)**：“每次调用寄存器计算H+30是浪费，所以提前算好H += 30。”  
> **点评**：这个细节太重要了！循环里重复计算同一个值，会让程序变慢（虽然这题数据小，但好习惯要从小养成）。以后写循环，先把“不变的量”算好，再用在循环里！

> **参考经验 (来自 installb)**：“用前缀和统计数量，比遍历更快。”  
> **点评**：前缀和是“空间换时间”的经典技巧——虽然这题用不到，但如果题目要多次查询“≤X高度的苹果数”，前缀和能把每次查询的时间从O(10)降到O(1)，超高效！

> **参考经验 (来自 niuniudundun)**：“一开始分两次判断‘不用板凳’和‘用板凳’，后来发现可以合并成一次判断。”  
> **点评**：这是新手常犯的“冗余逻辑”错误——陶陶站在板凳上的最大高度已经包含了“不用板凳”的情况（比如苹果高度≤身高，肯定也≤身高+30），所以不用分开判断！

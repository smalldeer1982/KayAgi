# 题目信息

# [NOIP 2017 普及组] 成绩

## 题目背景

NOIP2017 普及组 T1

## 题目描述

牛牛最近学习了 C++ 入门课程，这门课程的总成绩计算方法是：

$$
\text{总成绩}=\text{作业成绩}\times 20\%+\text{小测成绩}×30\%+\text{期末考试成绩} \times 50\%
$$

牛牛想知道，这门课程自己最终能得到多少分。


## 说明/提示

**样例 1 说明**

牛牛的作业成绩是 $100$ 分，小测成绩是 $100$ 分，期末考试成绩是 $80$ 分，总成绩是 $100 \times 20\%+100 \times 30\%+80 \times 50\%=20+30+40=90$。

**样例 2 说明**

牛牛的作业成绩是 $60$ 分，小测成绩是 $90$ 分，期末考试成绩是 $80$ 分，总成绩是 $60 \times 20\%+90 \times 30\%+80 \times 50\%=12+27+40=79$。

**数据说明**

对于 $30\%$ 的数据，$A=B=0$。

对于另外 $30\%$ 的数据，$A=B=100$。

对于 $100\%$ 的数据，$0≤A,B,C≤100$ 且 $A,B,C$ 都是 $10$ 的整数倍。

## 样例 #1

### 输入

```
100 100 80 ```

### 输出

```
90```

## 样例 #2

### 输入

```
60 90 80 ```

### 输出

```
79```

> 解决这道题的核心是**计算加权平均分**——就像你期末成绩由作业、小测、考试按不同比例算总分一样。但直接用浮点数（比如`0.2`）计算可能会出问题！比如样例2中，用`60*0.2+90*0.3+80*0.5`在某些电脑上会算出`78.9999999`，转成整数就变成`78`，和正确结果`79`不符。

## 3. 核心难点辨析与解题策略

<difficulty_intro>
这道题的“难点”不是公式本身，而是**如何用严谨的方式实现公式**。结合优质题解，我总结了3个关键思考点：
</difficulty_intro>

1.  **关键点1：正确理解加权公式**  
    * **分析**：题目中的公式是`作业×20% + 小测×30% + 考试×50%`，千万不要把比例搞反！比如考试占50%，是权重最大的部分，计算时要乘0.5（或5/10）。  
    * 💡 **学习笔记**：先把题目中的比例转换成数学表达式，再开始写代码！

2.  **关键点2：避开浮点运算的误差**  
    * **分析**：浮点数（比如`double`）在计算机中是近似存储的，比如`0.1`无法精确表示。当我们计算`60×0.2`时，可能得到`11.9999999`而不是`12`，转成整数就会出错。解决方法是**用整数运算代替浮点**——把比例转换成分子（比如20%→2），总和乘以分子后再除以分母（100→10，因为所有比例的分母都是10的倍数）。  
    * 💡 **学习笔记**：如果结果需要是整数，优先用整数运算！

3.  **关键点3：利用题目给出的数据特性**  
    * **分析**：题目明确说`A、B、C`都是10的倍数，这是“突破口”！比如`A=60`，`A÷10=6`，`6×2=12`（刚好是`60×20%`的结果）。如果没有这个条件，我们可能需要用`round()`函数四舍五入，但有了这个条件，直接整数运算就够了。  
    * 💡 **学习笔记**：读题时要注意“数据说明”部分，里面往往藏着优化的小技巧！

### ✨ 解题技巧总结
<summary_best_practices>
通过这道题，我们可以学到3个通用技巧：
</summary_best_practices>
-   **技巧1：先想数学，再写代码**：把问题转换成数学公式，再考虑如何用代码实现（比如把浮点转整数）。
-   **技巧2：避开浮点陷阱**：如果结果是整数，尽量用整数运算（乘法优先于除法，避免小数）。
-   **技巧3：关注数据范围**：题目中的“数据说明”能帮我们简化问题（比如本题的“10的倍数”）。

**本题通用核心C++实现参考**
* **说明**：本代码综合了题解一的思路，用全整数运算计算总分，是竞赛中最推荐的写法。
* **完整核心代码**：
    ```cpp
    #include <stdio.h>

        return 0;
    }
    ```
* **代码解读概要**：
    > 代码分三步：1. 用`scanf`输入三个成绩；2. 计算`(a×2 + b×3 + c×5)`（对应20%、30%、50%的比例），再除以10得到总分；3. 用`printf`输出结果。所有运算都是整数，完全没有浮点误差！

    ```
* **代码解读**：
    > 作者把比例转换成了最简分数：`20%=1/5`（所以`a÷5`）、`30%=3/10`（所以`b×3÷10`）、`50%=1/2`（所以`c÷2`）。因为`a`是10的倍数，`a÷5`肯定是整数（比如`60÷5=12`）；`b×3`是30的倍数（比如`90×3=270`），`270÷10=27`；`c÷2`也是整数（比如`80÷2=40`）。这样加起来结果正确！

* **亮点**：用“拆分10倍”的方式计算。
* **核心代码片段**：

    printf("%d", a/10*2 + b/10*3 + c/10*5);
    ```
* **代码解读**：
    > 因为`a`是10的倍数，`a÷10`就是“去掉末尾的0”（比如`60÷10=6`），然后乘以2就是`6×2=12`（对应`60×20%`）。同理，`b÷10×3`是`90÷10×3=27`，`c÷10×5`是`80÷10×5=40`，加起来就是79。这种写法更直观，容易理解！
* 💡 **学习笔记**：拆分数据的“倍数特征”能让代码更易懂！

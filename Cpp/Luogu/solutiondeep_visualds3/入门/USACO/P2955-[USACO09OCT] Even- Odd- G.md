# [USACO09OCT] Even? Odd? G

## 题目描述

Bessie那惨无人道的二年级老师搞了一个有 $N$ 个正整数 $I$ 的表叫Bessie去判断“奇偶性”（这个词语意思向二年级的学生解释，就是“这个数是单数，还是双数啊？”）。Bessie被那个表的长度深深地震惊到了，竟然跟栋栋的泛做表格一样多道题！！！毕竟她才刚刚学会数数啊。

写一个程序读入 $N$ 个整数，如果是双数，那么在单立的一行内输出 `even`，如果是单数则类似地输出 `odd`。

## 说明/提示

**【样例解释】**

两个整数：$1024\text{ 和 }5931$

$1024$ 可被 $2$ 整除，$5931$ 不能

**【数据范围】**

$1\le N \le 100，1\le l_j \le 10^{60}$

## 样例 #1

### 输入

```
2 
1024 
5931 
```

### 输出

```
even 
odd 
```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：编程技巧应用（字符串/字符处理）

🗣️ **初步分析**：
解决这道题的关键，其实藏在**小学算术**里——一个数是奇数还是偶数，**只看它的最后一位数字**！比如1234是偶数（最后一位4），5679是奇数（最后一位9）。就像判断苹果是不是红苹果，不用咬开看果肉，只看果皮颜色就行～

但题目里的数大到什么程度呢？最大有**10^60**——比地球的年龄（约45亿年，即4.5×10^9）还大得多！普通的`int`或`long long`根本装不下（`long long`最多只能存到9×10^18）。这时候，我们需要用**字符串**或者**逐个字符读取**的技巧，把大数当成“字符序列”来处理，只取最后一位判断奇偶。

### 核心算法流程
1. **输入处理**：把每个大数读成字符串（或逐个字符读，只保留最后一位）；
2. **取最后一位**：字符串的最后一个字符就是大数的最后一位数字；
3. **判断奇偶**：把最后一位字符转换成数字（比如`'5'`→5），看它能不能被2整除；
4. **输出结果**：能整除输出`even`，否则输出`odd`。

## 3. 核心难点辨析与解题策略

### 关键点1：大数存不下，怎么办？
- **问题**：普通数据类型（`int`/`long long`）装不下10^60的数，会“溢出”（比如`long long`存10^19就会变成负数）。
- **解决**：用字符串或字符逐个读取——把大数当成“字符组成的句子”，不计算它的值，只看最后一个“字”（最后一位）。
- 💡 **学习笔记**：遇到“超大数”问题，先想“能不能用字符串处理”！

### 关键点2：怎么取最后一位？
- **问题**：字符串的下标从0开始，比如“1024”的长度是4，最后一位是`c[3]`（不是`c[4]`！）。
- **解决**：用`strlen(c)-1`得到最后一位的下标（`strlen`返回字符串的实际长度）。如果是逐个读字符，就一直更新“最后一位”变量（比如`s = x`，直到读完）。
- 💡 **学习笔记**：字符串下标要“减1”，别忘啦！

### 关键点3：字符怎么转数字？
- **问题**：字符串里的`'5'`是字符，不是数字5，直接用`'5'%2`会算错吗？其实**字符的ASCII码奇偶性和数字本身一致**（比如`'0'`→48→偶，`'1'`→49→奇），但最好还是用`c[len-1]-'0'`转成数字，更直观！
- **解决**：用字符减`'0'`（比如`'5'-'0'=5`），或者减48（因为`'0'`的ASCII是48）。
- 💡 **学习笔记**：字符转数字，记住`- '0'`！

### ✨ 解题技巧总结
1. **看数据范围**：遇到“10^60”这种大数，直接放弃普通类型，选字符串；
2. **抓核心规律**：奇偶性只看最后一位，不用处理整个数；
3. **边读边处理**：不用存所有数，读一个处理一个，省空间；
4. **细节要注意**：吞掉输入的回车（比如`getchar()`），处理EOF（防止死循环）。


## 4. C++核心代码实现赏析

### 本题通用核心C++实现参考
* **说明**：综合了woshishei和学而思李老师的思路，是最直观的字符串解法。
* **完整核心代码**：
```cpp
#include <iostream>
#include <cstring> // strlen函数需要的头文件
using namespace std;

int main() {
    int n;
    cin >> n; // 输入要判断的数的个数
    char num[65]; // 每个数最多60位，开65够了

    for (int i = 0; i < n; ++i) {
        cin >> num; // 读入一个大数（字符串）
        int len = strlen(num); // 得到字符串长度
        int last_digit = num[len - 1] - '0'; // 最后一位转数字

        if (last_digit % 2 == 0) {
            cout << "even" << endl;
        } else {
            cout << "odd" << endl;
        }
    }

    return 0;
}
```
* **代码解读概要**：
  1. 读入`n`（要判断的数的个数）；
  2. 循环`n`次，每次读入一个字符串`num`；
  3. 用`strlen(num)`得到字符串长度`len`，取最后一位`num[len-1]`；
  4. 转成数字`last_digit`，判断奇偶并输出。

### 各优质题解的片段赏析

#### 题解一：woshishei的核心片段
* **亮点**：边输入边输出，不存所有数，空间最优。
* **核心代码片段**：
```cpp
cin >> n;
for(i=1;i<=n;i++) {
    cin>>c;
    len=strlen(c);
    if((c[len-1]-'0')%2==0)
        cout<<"even\n";
    else
        cout<<"odd\n";
}
```
* **代码解读**：
  - 循环里每次读一个字符串`c`，直接处理，不用数组存所有数。`c[len-1]-'0'`把最后一位转成数字，判断奇偶。
* 💡 **学习笔记**：边读边处理，是节省空间的好方法！

#### 题解二：HFColin的核心片段
* **亮点**：用`getchar()`逐个读，只存最后一位，空间极致优化。
* **核心代码片段**：
```cpp
while (n--) {
    while (((x=getchar())!=EOF)&&x!='\n') s=x;
    s-='0';
    if (s%2) printf("odd\n");
    else printf("even\n");
}
```
* **代码解读**：
  - `getchar()`逐个读字符，直到遇到换行或EOF，`s`一直更新为最后一个字符。比如读“5931”时，`s`最后是`'1'`。转成数字后判断奇偶。
* 💡 **学习笔记**：遇到超长大数，用`getchar()`逐个读，只存关键位！

#### 题解四：JustinRochester的核心片段
* **亮点**：读入优化+位运算，效率超高。
* **核心代码片段**：
```cpp
int read(){
    int abs=0;char c=getchar();
    while(!isdigit(c)) c=getchar();
    while(isdigit(c)) abs=c-'0',c=getchar();
    return abs;
}

int main(){
    int n;scanf("%d",&n);
    while(n--){
        int x=read();
        printf((x&1)?"odd\n":"even\n");
    }
}
```
* **代码解读**：
  - `read()`函数跳过非数字字符，只保留最后一位数字（`abs=c-'0'`）。`x&1`判断奇偶：奇数的二进制最后一位是1，结果为真；偶数为假。
* 💡 **学习笔记**：位运算`&1`比`%2`更快，竞赛里常用！

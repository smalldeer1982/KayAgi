# 题目信息

# 配对碱基链

## 题目描述

脱氧核糖核酸（DNA）由两条互补的碱基链以双螺旋的方式结合而成。

而构成 DNA 的碱基共有 $4$ 种：

- 分别为腺嘌呤（$A$）

- 鸟嘌呤（$G$）

- 胸腺嘧啶（$T$）

- 胞嘧啶（$C$）

我们知道，在两条互补碱基链的对应位置上，腺嘌呤总是和胸腺嘧啶配对，鸟嘌呤总是和胞嘧啶配对。

你的任务就是根据一条单链上的碱基序列，给出对应的互补链上的碱基序列。

## 样例 #1

### 输入

```
ATATGGATGGTGTTTGGCTCTG
```

### 输出

```
TATACCTACCACAAACCGAGAC
```

## 3. 核心难点辨析与解题策略

在解决这个问题时，新手常遇到以下3个关键点：

1. **关键点1：正确建立碱基对应关系**  
   * **分析**：配对规则是`A↔T`、`C↔G`，别记反！比如有的同学会把`A`换成`C`，这就错了。优质题解的做法是：**把规则写死在代码里**（用map、if-else或switch），避免手动计算错误。  
   * 💡 **学习笔记**：固定规则直接“硬编码”，比复杂的计算更可靠！

2. **关键点2：字符串的遍历与长度**  
   * **分析**：C++中字符数组用`strlen()`算长度，`string`用`size()`。遍历的时候下标要从0开始（比如数组长度是5，下标是0~4）。如果从1开始，会漏掉第一个字符，或者越界访问！  
   * 💡 **学习笔记**：字符串的下标从0开始，记牢！

3. **关键点3：字符类型的正确比较**  
   * **分析**：字符变量（比如`s[i]`）要和**字符常量**（比如`'A'`，单引号）比较，不能用双引号（`"A"`是字符串，类型不对）。比如`if(s[i] == "A")`会报错！  
   * 💡 **学习笔记**：字符用单引号，字符串用双引号，别搞混！

### ✨ 解题技巧总结
- **技巧A：用映射表简化代码**：如果有固定的“键值对应”关系（比如本题的碱基配对），用`map`或数组存规则，比多个`if-else`更简洁。  
- **技巧B：优先用string处理字符串**：`string`的`size()`、`[]`操作更方便，不用考虑`\0`结尾的问题，适合新手。  
- **技巧C：注意细节**：比如字符的单引号、数组的大小、下标的起始位置，这些小细节往往是bug的根源！


## 4. C++核心代码实现赏析

在深入分析具体题解之前，先看一个**通用核心实现**——结合了`string`的方便和`map`的简洁：

### 本题通用核心C++实现参考
* **说明**：本代码用`string`存储输入，`map`建立碱基对应关系，逻辑清晰，适合新手学习。
* **完整核心代码**：
  ```cpp
  #include <iostream>
  #include <string>
  #include <map>
  using namespace std;

  int main() {
      string s;
      cin >> s;
      // 建立碱基配对规则
      map<char, char> pair = {{'A', 'T'}, {'T', 'A'}, {'C', 'G'}, {'G', 'C'}};
      // 遍历输出每个字符的互补碱基
      for (int i = 0; i < s.size(); ++i) {
          cout << pair[s[i]];
      }
      return 0;
  }
  ```
* **代码解读概要**：  
  1. 用`string s`读入输入的碱基链；  
  2. 用`map<char, char> pair`存配对规则（比如`pair['A']`就是`'T'`）；  
  3. 遍历`s`的每个字符，输出对应的互补碱基。


### 各优质题解的片段赏析

#### 题解一：墨忧（map的用法）
* **亮点**：用map把配对规则“可视化”，代码简洁。
* **核心代码片段**：
  ```cpp
  map <char,char> ch;
  ch['A']='T';
  ch['T']='A';
  ch['C']='G';
  ch['G']='C';
  for(int i=0;i<lena;i++) cout<<ch[a[i]];
  ```
* **代码解读**：  
  `map<char, char>`是一个“键值对”容器——`ch['A'] = 'T'`表示“键`A`对应的值是`T`”。遍历的时候，直接用`ch[a[i]]`就能拿到`a[i]`的互补碱基，不用写多个`if`！  
* 💡 **学习笔记**：map是处理“一一对应”问题的好工具！

#### 题解二：午尘（if-else的直观写法）
* **亮点**：逻辑直白，适合新手理解配对规则。
* **核心代码片段**：
  ```cpp
  for (int i=0;i<strlen(s);i++)
      if (s[i]=='A') cout<<"T";
      else if (s[i]=='G') cout<<"C";
      else if (s[i]=='T') cout<<"A";
      else if (s[i]=='C') cout<<"G";
  ```
* **代码解读**：  
  逐个判断`s[i]`是什么字符，然后输出对应的结果。比如`s[i]`是`A`，就输出`T`；是`G`就输出`C`。这种写法虽然长，但逻辑一看就懂，适合刚开始学条件判断的同学。  
* 💡 **学习笔记**：复杂的规则可以拆成多个简单的`if-else`！

#### 题解四：rui_er（switch的优化写法）
* **亮点**：用switch代替if-else，效率更高，代码更工整。
* **核心代码片段**：
  ```cpp
  rep(i, 0, n-1) {
      switch(s[i]) {
          case 'A': putchar('T'); break;
          case 'C': putchar('G'); break;
          case 'G': putchar('C'); break;
          case 'T': putchar('A'); break;
      }
  }
  ```
* **代码解读**：  
  `switch`是多分支判断的另一种方式——根据`s[i]`的值，跳转到对应的`case`执行。比如`s[i]`是`A`，就执行`putchar('T')`（输出`T`）。`break`用来跳出switch，避免执行后面的case。  
* 💡 **学习笔记**：当判断条件是“等于某个值”时，switch比if-else更高效！

## 7. 学习心得与经验分享 (若有)

本次分析的题解中，有几个非常实用的经验：

> **参考经验（来自 rui_er）**：“C风格字符数组中，字符串后还会有一位`\0`表示结尾，因此数组需要开大一位。”  
> **点评**：这个细节很容易被忽略！比如输入字符串长度是10，数组要开11位（比如`char s[11]`），否则`strlen()`会读到`\0`后面的乱码，导致长度计算错误。下次用字符数组时，记得多开一位！

> **参考经验（来自 午尘）**：“字符比较要用单引号，比如`s[i]=='A'`，不能用双引号。”  
> **点评**：这是新手常犯的错误！双引号是字符串常量（类型是`const char*`），单引号是字符常量（类型是`char`），两者不能直接比较。记住：字符用单引号，字符串用双引号！

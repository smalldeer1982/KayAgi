# [语言月赛 202405] 交题解

## 题目描述

小 S 想要在 Luogu 上提交一篇非常美妙的题解，但是小 Q 不希望看到小 S 的题解比自己的点赞更多，于是她故意在小 S 的题解中加入了一些奇怪的字符。

具体来说，原本的题解可以看作一个只包含**小写或大写字母**的字符串 $S$，而小 Q 在其中若干位置插入了一些随机字符，这些字符可能是数字或标点符号，但一定**不是字母**，且一定在**键盘**上出现。我们称被添加字符后的字符串为 $S'$。

你的任务是，将 $S'$ 中**非大小写字母**的字符全部去除，然后将得到的字符串输出。

## 说明/提示

### 样例 1 解释

在字符串 `I*Love*Luogu**333#$$` 中，字符 `*`，`#`，`$` 属于标点符号，字符 `3` 属于数字，这些都是后添加的字符。删除后，字符串变为 `ILoveLuogu`。

### 样例 2 解释

在字符串 `L2u4o6gu##$%Lov()es^me` 中，字符 `#`，`$`，`%`，`(`，`)`，`^`，\` 属于标点符号，字符 `2`，`4`，`6` 属于数字，这些都是后添加的字符。删除后，字符串变为 `LuoguLovesme`。

### 数据范围

对于前 $30 \%$ 的数据，保证 $S'$ 中只包含拉丁字母。  
对于另 $30 \%$ 的数据，保证 $S'$ 中只包含拉丁字母和下划线 `_`。  
对于 $100 \%$ 的数据，保证 $S'$ 的长度在 $1$ 到 $10^6$ 之间, $S'$ 中只出现**键盘上主键盘区中的字母、标点符号、数字**，不含空格、换行符等结束符号。

## 样例 #1

### 输入

```
I*Love*Luogu**333#$$
```

### 输出

```
ILoveLuogu
```

## 样例 #2

### 输入

```
L2u4o6gu##$%Lov()es^me
```

### 输出

```
LuoguLovesme
```

## 样例 #3

### 输入

```
[]{}->%_;'pxprpx
```

### 输出

```
pxprpx
```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：`模拟` (字符串的线性遍历与字符筛选)

🗣️ **初步分析**：  
解决「交题解」的关键，就像**在水果摊分拣苹果**——把筐里的水果一个个拿起来看，只有苹果（字母）才放进袋子，其他水果（非字母）直接丢掉。这里的“模拟”，就是**严格按照题目规则，逐个处理字符串中的每一个字符**。  

在本题中，“规则”很明确：只保留大小写字母，其他字符（数字、标点）全部删除。核心难点是**如何准确判断一个字符是不是字母**？题解给出的方案是用**ASCII码范围**——大写字母的ASCII值在`'A'~'Z'`（65~90）之间，小写字母在`'a'~'z'`（97~122）之间，只要满足其中一个条件就是字母。

## 3. 核心难点辨析与解题策略

### 核心难点与解决方法
1. **关键点1：如何准确判断“字母”？**  
   * **分析**：很多同学会误以为“字母就是a-z和A-Z”，但编程中要“用代码说话”——ASCII码是字符的“身份证号”，大写字母和小写字母的身份证号范围是连续的，所以直接用范围判断最可靠。  
   * 💡 **学习笔记**：字符判断别瞎猜，ASCII范围是“硬标准”！

2. **关键点2：如何处理1e6长度的大字符串？**  
   * **分析**：线性遍历（一个一个字符看）是最优解，因为每个字符只需要判断一次，时间复杂度O(n)。如果用其他方法（比如正则表达式），反而会变慢。  
   * 💡 **学习笔记**：大字符串处理，“线性遍历”永远是首选！

3. **关键点3：输入输出会不会超时？**  
   * **分析**：对于1e6长度的字符串，`cin`和`cout`可能会有点慢（因为默认同步C语言的输入输出）。可以加一句`ios::sync_with_stdio(false); cin.tie(0);`来加速，题解虽然没写，但这是竞赛中的“隐藏技巧”。  
   * 💡 **学习笔记**：处理大数据，输入输出优化不能忘！

### ✨ 解题技巧总结
- **字符筛选技巧**：用ASCII范围判断字符类型（字母、数字、标点）；  
- **字符串遍历技巧**：用`for`循环遍历`string`的每个字符（`s.size()`获取长度）；  
- **效率优化技巧**：处理大输入时，关闭`cin`与C的同步（`ios::sync_with_stdio(false);`）。


## 4. C++核心代码实现赏析

### 本题通用核心C++实现参考
* **说明**：本代码综合了题解的核心思路，并补充了输入输出优化，适用于所有测试用例。  
* **完整核心代码**：
  ```cpp
  #include <iostream>
  #include <string>
  using namespace std;

  int main() {
      ios::sync_with_stdio(false); // 关闭同步，加速cin
      cin.tie(0); // 解除cin与cout的绑定，进一步加速

      string s;
      cin >> s;

      for (int i = 0; i < s.size(); ++i) {
          if ((s[i] >= 'a' && s[i] <= 'z') || (s[i] >= 'A' && s[i] <= 'Z')) {
              cout << s[i];
          }
      }

      return 0;
  }
  ```
* **代码解读概要**：  
  1. 第一部分：导入必要的头文件（`iostream`处理输入输出，`string`处理字符串）；  
  2. 第二部分：输入优化（让`cin`更快）；  
  3. 第三部分：读入字符串`s`；  
  4. 第四部分：遍历`s`的每个字符，判断是否是字母，是的话输出；  
  5. 最后返回0，结束程序。

---

### 题解一：核心代码片段赏析
* **亮点**：用最简洁的条件判断覆盖所有字母情况，没有冗余代码。  
* **核心代码片段**：
  ```cpp
  for (int i = 0; i < s.size(); ++i) {
      if ((s[i] >= 'a' && s[i] <= 'z') || (s[i] >= 'A' && s[i] <= 'Z')) {
          cout << s[i];
      }
  }
  ```
* **代码解读**：  
  - 循环变量`i`从0到`s.size()-1`，逐个访问字符串的每个字符；  
  - 条件判断用`||`（逻辑或）连接两个条件：**要么是小写字母，要么是大写字母**；  
  - 只要满足条件，就用`cout`输出这个字符——是不是像“挑苹果”一样直接？  
* 💡 **学习笔记**：简单的条件判断，往往是解决问题的“最强武器”！

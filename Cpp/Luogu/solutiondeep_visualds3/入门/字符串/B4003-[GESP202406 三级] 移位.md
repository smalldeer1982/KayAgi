# 题目信息

# [GESP202406 三级] 移位

## 题目描述

小杨学习了加密技术移位，所有大写字母都向后按照⼀个固定数目进行偏移。偏移过程会将字母表视作首尾相接的环，例如，当偏移量是 $3$ 的时候，大写字母 A 会替换成 D，大写字母 Z 会替换成 C，总体来看，大写字母表 ABCDEFGHIJKLMNOPQRSTUVWXYZ 会被替换成 DEFGHIJKLMNOPQRSTUVWXYZABC。

注：当偏移量是 $26$ 的倍数时，每个大写字母经过偏移后会恰好回到原来的位置，即大写字母表 ABCDEFGHIJKLMNOPQRSTUVWXYZ 经过偏移后会保持不变。

## 说明/提示

**【样例解释】**

当偏移量是 $3$ 的时候，大写字母 A 会替换成 D，大写字母 Z 会替换成 C，总体来看，大写字母表 ABCDEFGHIJKLMNOPQRSTUVWXYZ 会被替换成 DEFGHIJKLMNOPQRSTUVWXYZABC。

**【数据范围】**

对于全部数据，保证有 $1\leq n\leq 100$。

## 样例 #1

### 输入

```
3```

### 输出

```
DEFGHIJKLMNOPQRSTUVWXYZABC```

## 3. 核心难点辨析与解题策略

<difficulty_intro>
这道题的「坑」都藏在「细节」里。结合题解，我帮你提炼了3个核心难点和解决方法：
</difficulty_intro>

1.  **难点1：如何让字母「绕圈」（Z后接A）？**  
    * **分析**：字母表是「环形」的，总共有26个字母——所以任何偏移量超过26的部分都是「无效的」（比如偏移27等于偏移1）。用「模26」运算能把任何大数字压缩到0~25之间，正好对应A~Z的位置（`'A' + 0`是A，`'A' +25`是Z）。  
    * 💡 **学习笔记**：遇到「循环/环形」问题，先想「模周期数」（本题周期是26）！

2.  **难点2：如何计算每个字母的「最终位置」？**  
    * **分析**：第`i`个原始字母（比如i=1是A），偏移n位后，总偏移量是「原始位置 + 偏移量」——但原始位置是从0开始算的（A是0，Z是25），所以要把`i`（从1开始）转换成`i-1`。最终公式就是`(n + i -1) %26`。  
    * 💡 **学习笔记**：字符的「位置」要和「数值」对应，别搞混「从0开始」还是「从1开始」！

3.  **难点3：如何把「数值」转回「字符」？**  
    * **分析**：大写字母的ASCII码是连续的——`'A'`是65，`'B'`是66，…，`'Z'`是90。所以只要把「偏移后的数值」加到`'A'`上，再强制转换成`char`类型，就能得到对应的字母。  
    * 💡 **学习笔记**：字符本质是「数值」，可以直接做算术运算！

### ✨ 解题技巧总结
- **技巧1：用模运算简化循环逻辑**：遇到环形问题（比如星期、月份、字母表），模「周期数」能快速解决绕圈问题；  
- **技巧2：字符与数值互转**：大写字母的ASCII码是连续的，`'A' + k`（k∈0~25）直接对应第k+1个字母；  
- **技巧3：简洁循环处理重复操作**：26个字母的处理逻辑完全一样，用`for`循环遍历1~26即可，不用写26次重复代码！

**本题通用核心C++实现参考**
* **说明**：此代码来自chen_zhe的题解，是「移位问题」的最简洁实现，覆盖所有边界情况。
* **完整核心代码**：
  ```cpp
  #include <iostream>
  using namespace std;

  int main() {
      int n;
      cin >> n;
      for (int i = 1; i <= 26; ++i) {
          cout << (char)('A' + (n + i - 1) % 26);
      }
      return 0;
  }
  ```
* **代码解读概要**：
  > 1. 输入偏移量`n`；  
  > 2. 用`for`循环遍历1~26（对应A~Z）；  
  > 3. 计算每个字母的「最终偏移」：`(n + i -1) %26`（把i从1转成0开始，加偏移量后模26）；  
  > 4. 把「数值」转成「字符」：`(char)('A' + 偏移量)`，然后输出。

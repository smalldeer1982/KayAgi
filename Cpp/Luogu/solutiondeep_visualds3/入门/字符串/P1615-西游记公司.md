# 西游记公司

## 题目背景

一道极其无厘头的题目。

## 题目描述

事情是这样的：  

西游记中的孙沙猪三徒弟在西天取经之后开始进入厦门大学经贸系学习经济，在 $1$ 个小时的学习后，他们用特殊手段毕业了。

然后，他们创办了三个公司：“花果山生态旅游山庄集团公司” “高老庄猪肉美食城有限公司” 和 “流沙河轮渡有限公司”。虽然这三家公司出自 “经卷出版社” 社长唐三藏的徒弟们，但是每家公司都仍然保持着 $0$ 元以下的财政收入。  

于是，他们想到了一个无聊的方法：向别人偷！其中，孙悟空偷猪八戒的平底锅之前，猪八戒可以一直抢沙和尚的笔记本。

现在，作为猪八戒手下首席智士的你，要帮助既聪明又愚蠢的猪八戒用那种不正当的手段挽救这个公司！  

你可以这样做：你已经知道孙悟空什么时候要来偷猪八戒的东西，而猪八戒又要在什么时间去抢沙和尚的东西，又知道猪八戒每秒钟可以偷沙和尚 $n$ 台笔记本，帮猪八戒算一算，自己在有限的时间内能偷到沙和尚多少台笔记本，使得他有足够的时间来防御大师兄。

## 说明/提示

对于 $100\%$ 的数据，保证两个时间合法，$n<32768$。

## 样例 #1

### 输入

```
00:0:00
0:00:10
10```

### 输出

```
100```

✨ **本题主要考察**：编程技巧应用（输入处理 + 时间单位转换）

🗣️ **初步分析**：
解决这道题的关键，其实是做两件“接地气”的事：  
1. **把时间“拆成秒”**：就像把“1小时2分3秒”换成“1×3600 + 2×60 + 3 = 3723秒”——毕竟“每秒偷n个”，只有秒数才能直接相乘。  
2. **正确读入带冒号的时间**：比如输入“00:0:00”，我们得把“00”（时）、“0”（分）、“00”（秒）准确读出来，不能被冒号卡住。  

这道题的核心逻辑就像“攒硬币”：  
- 1小时 = 3600个“1秒硬币”，1分钟 = 60个“1秒硬币”，秒就是1个硬币。  
- 两个时间的“硬币总数差”× 每秒偷的数量 = 最终答案！  

**核心难点**：  
- 如何让程序“忽略冒号”读入时间？

## 3. 核心难点辨析与解题策略

<difficulty_intro>
这道题的“坑”不多，但新手容易在三个地方“栽跟头”。结合优质题解的经验，我帮你总结了**解决策略**：
</difficulty_intro>

### 1. 难点1：如何读入带冒号的时间？
* **问题**：输入是“00:0:00”这种带冒号的格式，直接用`cin`或`scanf`会“卡”在冒号上。  
* **解决策略**：  
  - 用**字符变量吸收冒号**：比如`cin>>h1>>ch>>m1>>ch>>s1`——`ch`会自动读入冒号，不影响数字的读取。  
  - 用**`scanf`的格式化输入**：比如`scanf("%lld:%lld:%lld", &h1, &m1, &s1)`——`:`会告诉`scanf`“这里有个冒号，跳过它读数字”。  
* 💡 **学习笔记**：处理带分隔符的输入，要么“用字符吃分隔符”，要么“用格式化输入指定分隔符”！

### 2. 难点2：如何把时间转换成秒？
* **问题**：时、分、秒是60进制，不能直接相减（比如1小时2分≠12分）。  
* **解决策略**：把所有时间都转换成“从0点开始的总秒数”——  
  总秒数 = 小时×3600 + 分钟×60 + 秒  
  比如“0:00:10”就是`0×3600 + 0×60 +10 =10秒`，“00:0:00”是0秒，差值是10秒，×10就是100（对应样例输出）。  
* 💡 **学习笔记**：处理不同进制的单位，一定要“统一成最小单位”（比如秒）再计算！

### 3. 难点3：如何避免数字溢出？
* **问题**：如果时间差是10小时（36000秒），每秒偷32767个，结果是36000×32767=1,179,612,000，超过了`int`的最大值（约20亿？不，`int`通常是4字节，最大值是2,147,483,647，这里刚好没超，但更大的数就会溢出）。  
* **解决策略**：用**`long long`类型**存储时间和结果——`long long`是8字节，可以存到9e18，绝对够！  
* 💡 **学习笔记**：只要涉及“大数相乘/相加”，先想“要不要用`long long`”！

### ✨ 解题技巧总结
- **技巧1**：处理带分隔符的输入，用字符变量或格式化输入跳过分隔符。  
- **技巧2**：单位转换要统一成最小单位（比如秒），避免进制错误。  
- **技巧3**：数据可能溢出时，用`long long`代替`int`。

### 本题通用核心C++实现参考
* **说明**：本代码综合了Right的“极简输入”和chenyuxuan02的“清晰转换”，用`cin`读入，逻辑直白。
* **完整核心代码**：
  ```cpp
  #include <iostream>
  using namespace std;

  int main() {
      long long h1, m1, s1, h2, m2, s2, t;
      char ch; // 用来吸收冒号
      cin >> h1 >> ch >> m1 >> ch >> s1; // 读入开始时间：时:分:秒
      cin >> h2 >> ch >> m2 >> ch >> s2; // 读入结束时间：时:分:秒
      cin >> t; // 读入每秒偷的数量

      long long time1 = h1 * 3600 + m1 * 60 + s1; // 开始时间转秒
      long long time2 = h2 * 3600 + m2 * 60 + s2; // 结束时间转秒
      cout << (time2 - time1) * t << endl; // 差值×速率=答案

      return 0;
  }
  ```
* **代码解读概要**：  
  1. 用`char ch`吸收冒号，读入两个时间的时、分、秒；  
  2. 把两个时间都转换成“总秒数”（`time1`和`time2`）；  
  3. 计算秒数差，乘以每秒偷的数量，输出结果。

1. **场景初始化**：

3. **计算差值与结果**：  
   - 两个总秒数（0和10）会“碰撞”，弹出“10-0=10”的红色文字，伴随“嗒”的音效；

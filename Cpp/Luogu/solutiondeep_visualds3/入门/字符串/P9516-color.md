# 题目信息

# color

## 题目描述

洛谷的名字颜色和咕值直接挂钩，具体细则见 [洛谷咕值系统介绍](https://www.luogu.com.cn/discuss/83952)。

给出某洛谷用户的五项咕值指标的得分，输出该用户的名字颜色。保证该用户不是棕名和紫名。

## 说明/提示

**【样例说明】**

三个样例的咕值分别为 $263,140,0$，名字颜色分别为红、绿、灰。

**【数据范围】**

对于 $20\%$ 的数据，满足五项指标分数均 $\le 10$；

对于另外 $20\%$ 的数据，满足五项指标分数均 $\ge 90$；

对于 $100\%$ 的数据，保证五项指标分数均 $\in [0, 100]$。

## 样例 #1

### 输入

```
100 14 9 55 85```

### 输出

```
Red```

## 样例 #2

### 输入

```
100 10 10 10 10```

### 输出

```
Green```

## 样例 #3

### 输入

```
0 0 0 0 0```

### 输出

```
Gray```

## 3. 核心难点辨析与解题策略

<difficulty_intro>
这道题看似简单，但新手容易在**规则记忆**和**条件逻辑**上栽跟头。结合优质题解，我总结了3个核心关键点：
</difficulty_intro>

1. **关键点1：正确计算咕值总和**  
   * **分析**：题目明确说“五项咕值指标的得分总和”，所以必须把5个数加起来——所有优质题解都用了`a+b+c+d+e`，没有例外。新手要注意：不要漏加任何一个数，也不要算错顺序（加法交换律，顺序不影响结果）。  
   * 💡 **学习笔记**：求和是模拟题的基础操作，先确认“要算什么”再写代码。

2. **关键点2：准确记忆区间边界**  
   * **分析**：颜色对应的区间是：  
     - 灰：0~99 → `sum < 100`；  
     - 蓝：100~119 → `sum >=100 且 sum <120`；  
     - 绿：120~169 → `sum >=120 且 sum <170`；  
     - 橙：170~229 → `sum >=170 且 sum <230`；  
     - 红：230~500 → `sum >=230`。  
     所有优质题解都严格遵循这个边界，比如题解一用`sum <=99`（等价于`sum<100`），题解三用`sum>=230`，都是正确的。  
   * 💡 **学习笔记**：边界是模拟题的“生命线”，一定要对照题目说明反复确认！

3. **关键点3：条件判断的逻辑正确性**  
   * **分析**：条件判断要“不重叠、不遗漏”。比如题解一从低到高判断，每个`else if`会自动排除前面的情况（比如`sum<=119`时，已经排除了`sum<=99`）；题解三从高到低判断，`sum>=230`不满足时，才会判断`sum>=170`，同样不会重叠。  
   * 💡 **学习笔记**：条件判断的顺序不影响结果，但要保证“每个情况只被判断一次”。


### ✨ 解题技巧总结
- **技巧1：变量名要“见名知意”**：用`sum`代替`qwq`，用`total`代替`ans`，让代码自己“说话”。  
- **技巧2：边界用“原题数值”**：比如直接写`sum < 100`，而不是`sum <=99`（虽然等价，但前者更贴近题目描述）。  
- **技巧3：测试样例验证**：写完代码后，用样例输入（比如样例1的总和是100+14+9+55+85=263）测试，确认输出是否正确。

**本题通用核心C++实现参考**
* **说明**：本代码基于题解一的思路，用最简结构实现“输入→求和→判断”，变量名直观，边界判断严格。
* **完整核心代码**：
  ```cpp
  #include <iostream>
  using namespace std;

  int main() {
      int a, b, c, d, e;
      cin >> a >> b >> c >> d >> e;
      int sum = a + b + c + d + e;
      if (sum < 100) {
          cout << "Gray";
      } else if (sum < 120) {
          cout << "Blue";
      } else if (sum < 170) {
          cout << "Green";
      } else if (sum < 230) {
          cout << "Orange";
      } else {
          cout << "Red";
      }
      return 0;
  }
  ```
* **代码解读概要**：  
  1. 用`cin`读取5个整数（咕值指标）；  
  2. 计算总和`sum`；  
  3. 用`if-else if`从低到高判断`sum`的区间，输出对应颜色。

接下来看三个优质题解的**核心片段**，体会不同的写法风格：

**题解一：作者Yamchip**  
* **亮点**：条件判断完全贴合题目区间描述，变量名`sum`直观。
* **核心代码片段**：
  ```cpp
  sum = a + b + c + d + e;
  if(sum <= 99) cout << "Gray";
  else if(sum <= 119) cout << "Blue";

* **代码解读**：

* 💡 **学习笔记**：整数的边界判断，`<=`和`<`可以灵活转换，只要逻辑正确。

  int qwq=a+b+c+d+e;//计算所有咕值指标的综合

  else if(qwq<120)cout<<"Blue";
  else if(qwq<170)cout<<"Green";
  else if(qwq<230)cout<<"Orange";
  else cout<<"Red";

* **代码解读**：  
  变量名`qwq`虽然可爱，但注释`//计算所有咕值指标的综合`弥补了这一点——新手能立刻明白这个变量是总和。条件判断用`qwq<100`，更贴近题目中的“0~99”区间描述。  
* 💡 **学习笔记**：注释是代码的“说明书”，关键变量一定要加注释！

**题解三：作者fkcufk**  
* **亮点**：从高到低判断，符合“先看最高等级”的思维习惯。
* **核心代码片段**：
  ```cpp

  if(sum>=230)cout<<"Red";
  else if(sum>=170)cout<<"Orange";
  else if(sum>=120)cout<<"Green";
  else if(sum>=100)cout<<"Blue";
  else cout<<"Gray";
  ```
* **代码解读**：  
  先判断`sum>=230`（红名），如果不满足，再判断`sum>=170`（橙名）……这种写法的好处是“先处理最高级”，逻辑同样正确。比如`sum=263`（样例1），直接进入第一个条件，输出红。  
* 💡 **学习笔记**：条件判断的顺序可以灵活调整，只要覆盖所有情况。


## 5. 算法可视化：像素动画演示 (核心部分)

<visualization_intro>
为了让大家“看”到“求和→判断颜色”的过程，我设计了一个**8位像素风动画**——像玩FC游戏一样，直观感受算法的每一步！
</visualization_intro>

### 🎮 动画演示主题：咕值小方块的“颜色变身”
我们用**5个小像素块**代表5个咕值指标，**1个大像素块**代表总和，通过“合并小方块→大方块变色”模拟算法流程。

### 🎨 设计思路
- **复古风格**：用FC红白机的8位色彩（比如灰色#C0C0C0、蓝色#0000FF、绿色#00FF00），营造怀旧感；  
- **交互性**：支持“单步执行”（点击下一步看每个操作）、“自动播放”（每秒走一步）；  
- **音效强化**：输入每个小方块时播放“滴”声，合并成大方块时播放“叮”声，变色时播放“嗡”声，输出颜色时播放“胜利音效”（比如FC游戏的“通关音”）。

### 🕹️ 动画帧步骤
1. **初始化界面**：  
   - 屏幕左侧显示5个空的小像素块（20x20像素，灰色边框），右侧显示一个大像素块（50x50像素，初始灰色）；  
   - 底部控制面板有“开始”“单步”“重置”按钮，以及速度滑块（1x~5x）。

2. **输入咕值**：  
   - 点击“开始”后，每个小方块依次亮起，显示输入的数值（比如样例1的100、14、9、55、85），每亮一个播放“滴”声；  
   - 小方块的颜色随数值变化（比如100是红色，14是浅蓝），帮助区分大小。

3. **计算总和**：  
   - 5个小方块向中间的大方块移动，合并成一个大数值（比如263），播放“叮”声；  
   - 大方块上显示总和，背景变成浅黄色（代表“计算完成”）。

4. **判断颜色**：  
   - 大方块根据总和的区间变色：比如263≥230，变成红色，播放“嗡”声；  
   - 屏幕下方弹出文字气泡：“总和263，对应颜色Red！”，同时播放胜利音效。

5. **重置与重播**：  
   - 点击“重置”，所有方块回到初始状态，可重新输入数值；  
   - 点击“单步”，可以一步步看“输入→求和→变色”的过程。

### 📝 设计理由
- 用“小方块合并”模拟求和，让抽象的“加法”变成直观的“物理合并”；  
- 用“颜色变化”代表区间判断，让“条件逻辑”变成可见的“视觉反馈”；  
- 音效强化了“关键操作”的记忆（比如“叮”声=求和完成，“嗡”声=颜色确定）。

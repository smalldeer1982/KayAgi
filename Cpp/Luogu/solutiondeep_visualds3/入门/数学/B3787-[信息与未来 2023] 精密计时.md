# 题目信息

# [信息与未来 2023] 精密计时

## 题目描述

小小有一个非常精密的计时器，每秒可以计数 100 次 (两个连续的时钟计数之间恰好间隔百分之一秒)。例如，以下是从 $13:01:02.37$ 到 $13:01:03.01$ 的计时过程：
```plain
13:01:02.37 (13 时 1 分 2 秒.37)
13:01:02.38
13:01:02.39
...
13:01:02.98
13:01:02.99 (13 时 1 分 2 秒.99)
13:01:03.00 (13 时 1 分 3 秒.00)
13:01:03.01
```
小小记录了计时器上的两个时刻，你能帮助他计算这两个时刻之间经过了多少个 “百分之一秒”吗？

## 说明/提示

对于 $100\%$ 的数据，两个时刻都来自 $24$ 小时制的同一天 （$00:00:00.00$ 到 $23:59:59.99$），且保证后一个时刻晚于前一个时刻。

>本题原始满分为 $15\text{pts}$。

## 样例 #1

### 输入

```
01:02:34.56 01:03:00.10```

### 输出

```
2554```

## 样例 #2

### 输入

```
00:00:00.00 23:59:59.99```

### 输出

```
8639999```

## 3. 核心难点辨析与解题策略

<difficulty_intro>
解决这道题时，大家常犯的错误集中在「单位记混」「输入读错」「数据溢出」这三个点。结合优质题解，我帮你总结了应对策略：
</difficulty_intro>

1.  **关键点1：记准单位转换系数**  
    * **难点**：比如把「1小时」算成3600百分秒（漏乘100），结果就会差100倍！  
    * **解决策略**：**逐层推导**——从大单位到小单位，每一步都乘进制：1小时=60分钟→60×60=3600秒→3600×100=360000百分秒。记不住系数时，就一步步推！  
    * 💡 **学习笔记**：单位转换的核心是「找对每个单位的「最小单位倍数」」。

2.  **关键点2：正确读取时间格式**  
    * **难点**：输入是「HH:MM:SS.ms」，有冒号和小数点，直接读字符串容易出错。  
    * **解决策略**：**利用cin的自动分隔功能**——cin读整数时会自动跳过非数字字符（比如冒号、小数点），所以用`cin >> h1 >> c >> m1 >> c >> s1 >> c >> ms1`就能轻松读入，不用逐字符处理！  
    * 💡 **学习笔记**：能让编译器帮你做的事，就别自己动手！

3.  **关键点3：避免数据溢出**  
    * **难点**：一天的总百分秒是8640000，虽然int（约2e9）能装下，但如果题目改成跨天（比如两天），int就不够了。  
    * **解决策略**：**用long long类型**——long long能装下更大的数（约9e18），彻底避免溢出问题。  
    * 💡 **学习笔记**：计算大数时，优先用long long，没错的！

### ✨ 解题技巧总结
<summary_best_practices>
通过这道题，我总结了3个通用技巧，帮你解决类似问题：
</summary_best_practices>
- **技巧1：统一单位**：遇到不同单位的计算（比如时间、货币、文件大小），先转换成最小的公共单位，简化计算。  
- **技巧2：巧用cin分隔**：读取带分隔符的数值（比如「a:b:c」），用char变量接收分隔符，比字符串处理更简单。  
- **技巧3：选对数据类型**：根据计算结果的最大值选类型（比如long long处理大数），避免溢出。

## 4. C++核心代码实现赏析

<code_intro_overall>
先看一份**通用核心代码**——它综合了优质题解的思路，简洁到「一眼就能看懂」！
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：本代码来自hahaloo的题解，是「直接转换总时间」的典型实现，逻辑清晰、代码简洁。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    using namespace std;

    int main() {
        int h1, m1, s1, ms1, h2, m2, s2, ms2;
        char c; // 接收冒号和小数点
        cin >> h1 >> c >> m1 >> c >> s1 >> c >> ms1;
        cin >> h2 >> c >> m2 >> c >> s2 >> c >> ms2;

        long long t1 = h1 * 360000LL + m1 * 6000LL + s1 * 100LL + ms1;
        long long t2 = h2 * 360000LL + m2 * 6000LL + s2 * 100LL + ms2;

        cout << t2 - t1 << endl;
        return 0;
    }
    ```
* **代码解读概要**：  
  1. 用`cin`读入两个时间的「时、分、秒、百分秒」，`char c`自动接收冒号和小数点；  
  2. 把每个时间转换成总百分秒（比如`h1*360000LL`中的`LL`是为了确保计算时用long long类型）；  
  3. 输出两个总时间的差值。


<code_intro_selected>
接下来，我们剖析两个题解的「核心片段」，看看它们的亮点！
</code_intro_selected>

**题解一：作者hahaloo**
* **亮点**：直接转换总时间，代码极简，适合竞赛！
* **核心代码片段**：
    ```cpp
    long long t1 = h1 * 360000LL + m1 * 6000LL + s1 * 100LL + ms1;
    long long t2 = h2 * 360000LL + m2 * 6000LL + s2 * 100LL + ms2;
    ```
* **代码解读**：  
  这两行是「灵魂代码」！比如样例1中的起始时间`01:02:34.56`：  
  - `h1=1` → 1×360000=360000（1小时的百分秒数）；  
  - `m1=2` → 2×6000=12000（2分钟的百分秒数）；  
  - `s1=34` →34×100=3400（34秒的百分秒数）；  
  - `ms1=56` →直接加56；  
  总和是360000+12000+3400+56=375456，正好是起始时间的总百分秒！  
  结束时间同理，最后相减就是答案。
* 💡 **学习笔记**：统一单位是解决这类问题的「万能钥匙」！

**题解二：作者Lydia1010__**
* **亮点**：分步计算差值，帮你理解单位进位！
* **核心代码片段**：
    ```cpp
    for(int i=1;i<=11;i++){
        char e; cin>>e;
        if(i==1 || i==2) x=x*10+e-'0'; // 小时
        if(i==4 || i==5) y=y*10+e-'0'; // 分钟（跳过第3位的冒号）
        if(i==7 || i==8) z=z*10+e-'0'; // 秒（跳过第6位的冒号）
        if(i==10 || i==11) o=o*10+e-'0'; // 百分秒（跳过第9位的小数点）
    }
    ```
* **代码解读**：  
  这个循环读11个字符（比如「01:02:34.56」共11个字符），根据位置判断是小时（第1、2位）、分钟（第4、5位）、秒（第7、8位）、百分秒（第10、11位）。比如`i=1`读入'0'，`x`变成0；`i=2`读入'1'，`x`变成1（小时是01）。这种方法需要准确记住分隔符的位置，但能帮你更细致理解字符串的拆分过程。
* 💡 **学习笔记**：如果觉得cin的方法抽象，可以用这种「笨办法」先练手，再过渡到更简洁的写法！

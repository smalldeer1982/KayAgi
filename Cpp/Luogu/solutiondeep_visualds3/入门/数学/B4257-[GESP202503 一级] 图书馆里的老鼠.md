# 题目信息

# [GESP202503 一级] 图书馆里的老鼠

## 题目描述

图书馆里有 $n$ 本书，不幸的是，还混入了一只老鼠，老鼠每 $x$ 小时能啃光一本书，假设老鼠在啃光一本书之前，不会啃另一本。请问 $y$ 小时后图书馆里还剩下多少本完整的书。

## 说明/提示

对于所有测试点，保证 $1\leq n,x,y\leq 1000$，保证 $y$ 小时后至少会剩下⼀本完整的书。

## 样例 #1

### 输入

```
10
2
3```

### 输出

```
8```

## 样例 #2

### 输入

```
5
2
4```

### 输出

```
3```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：编程技巧应用（分支结构 + 数学计算）

🗣️ **初步分析**：
> 解决这道题的关键，是**把“剩下多少本完整的书”转化为“老鼠啃了/开始啃了多少本书”**——因为完整的书 = 总书数 - 老鼠碰过的书（包括正在啃的）。  
> 打个比方：如果老鼠每2小时啃完一本书，3小时后它会啃完1本，再啃第2本的一半——这时候第2本已经不完整了，所以老鼠总共“碰过”2本书。总共有10本的话，剩下的就是10-2=8本（对应样例1）。  
> 核心算法流程很简单：  
> 1. 计算老鼠“碰过”的书的数量`d`；  
> 2. 用总书数`n`减去`d`，就是答案。  
> 其中最关键的是`d`的计算——要么用**分支结构**判断“时间是否刚好啃完整数本”，要么用**数学公式**直接计算（比如`(y+x-1)/x`）。  
> 可视化设计上，我打算做一个**8位像素风的“老鼠啃书模拟器”**：屏幕上有一排像素书（比如黄色方块），老鼠（棕色像素点）每过`x`帧啃完一本（书变成灰色），`y`帧后统计灰色书的数量，剩下的黄色书就是答案。动画里会用**闪烁**标记当前啃的书，用“咔吱”音效模拟啃书声，让你直观看到`d`的变化！

## 3. 核心难点辨析与解题策略

<difficulty_intro>
这道题看似简单，但第一次做时容易“卡”在三个关键点上。结合题解的思路，Kay帮你把“难点”变成“得分点”！
</difficulty_intro>

1.  **关键点1**：理解“老鼠碰过的书”=“完整啃完的书 + 正在啃的那本”
    * **分析**：比如y=3小时、x=2小时，老鼠啃完1本（2小时），剩下1小时在啃第2本——所以“碰过”2本，不是1本！很多人会漏掉“正在啃的那本”，导致答案错。题解里用“d是啃过或开始啃的书”直接点破了这点。
    * 💡 **学习笔记**：问题转化是解题的第一步——把“剩下多少”变成“少了多少”，思路瞬间开阔！

2.  **关键点2**：如何用分支判断“时间是否刚好啃完整数本”
    * **分析**：当`y % x == 0`（比如y=4，x=2），说明刚好啃完`y/x`本（2本）；如果有余数（比如y=3，x=2），说明还在啃下一本，要加1（`3/2+1=2`）。题解的分支代码把这个逻辑写得明明白白，连小白都能看懂。
    * 💡 **学习笔记**：分支结构是“处理不同情况”的神器——遇到“如果...否则...”的问题，先想分支！

3.  **关键点3**：不用分支，用数学公式直接算`d`
    * **分析**：`(y + x - 1) / x`其实是“向上取整”的技巧——比如`(3+2-1)/2=4/2=2`（刚好是我们要的d），`(4+2-1)/2=5/2=2`（整数除法自动向下取整，但加x-1后就变成向上取整了）。这个公式能把两行分支代码变成一行，超简洁！
    * 💡 **学习笔记**：数学公式能简化代码——多积累“向上取整”“向下取整”的技巧，编程会更高效！

### ✨ 解题技巧总结

## 4. C++核心代码实现赏析

<code_intro_overall>
先看一份“能直接运行”的通用代码——综合了题解的思路，把输入、计算、输出全写齐了！
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：本代码来自题解的核心逻辑，补充了输入输出部分，是能直接AC的完整代码！
* **完整核心代码**：
    ```cpp
    #include <iostream>
    using namespace std;

    int main() {
        int n, x, y;
        cin >> n >> x >> y;  // 输入总书数、每本书的时间、总时间
        int d;
        if (y % x == 0) {
            d = y / x;  // 刚好啃完d本
        } else {
            d = y / x + 1;  // 啃完d本，还在啃下一本
        }
        cout << n - d << endl;  // 剩下的完整书数
        return 0;
    }
    ```
* **代码解读概要**：代码分三步：1. 读入三个变量`n`（总书数）、`x`（每本书时间）、`y`（总时间）；2. 用分支计算`d`（老鼠碰过的书）；3. 输出`n-d`（剩下的完整书）。逻辑像“流水账”一样顺，完全不用绕弯！

接下来看题解里的“精华片段”——分支计算`d`和拓展的数学公式，这两个片段能帮你掌握“核心中的核心”！

**题解一：(来源：洛谷作者chen_zhe)**
* **亮点**：用最直白的分支逻辑计算`d`，还拓展了“不用分支的数学技巧”，兼顾新手和进阶学习者！
* **核心代码片段（分支版）**：
    ```cpp
    if (y % x == 0) {
        d = y / x;
    } else {
        d = y / x + 1;
    }
    cout << n - d << endl;
    ```
* **代码解读**：
    > 这段代码是题解的“灵魂”！`y % x == 0`判断“总时间是否刚好是x的整数倍”——如果是，说明老鼠刚好啃完`y/x`本（比如y=4，x=2，就是2本）；如果不是，说明老鼠还在啃下一本，要加1（比如y=3，x=2，就是1+1=2本）。最后用`n-d`算出剩下的书，完美对应题目要求！
* 💡 **学习笔记**：分支结构是“处理多情况”的基础，一定要练熟！

* **核心代码片段（数学优化版）**：
    ```cpp
    int d = (y + x - 1) / x;
    cout << n - d << endl;
    ```
* **代码解读**：
    > 这个片段超厉害！`(y + x - 1) / x`其实是“向上取整”的公式——比如y=3，x=2，`3+2-1=4`，`4/2=2`（刚好是我们要的d）；y=4，x=2，`4+2-1=5`，`5/2=2`（整数除法自动向下取整，但结果刚好对）。这个公式把两行分支变成一行，代码更简洁！
* 💡 **学习笔记**：多积累数学小技巧，代码会越来越“优雅”！

## 题目描述

cjf 君想调查学校 OI 组每个同学的生日，并按照年龄从大到小的顺序排序。但 cjf 君最近作业很多，没有时间，所以请你帮她排序。


## 说明/提示

数据保证，$1<n<100$，$1\leq |s|<20$。保证年月日实际存在，且年份 $\in [1960,2020]$。

```
3
Yangchu 1992 4 23
Qiujingya 1993 10 13
Luowen 1991 8 1
```

### 输出

```
Luowen
Yangchu
Qiujingya
```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：`多关键字排序 + 结构体应用`

🗣️ **初步分析**：  
解决“生日排序”的关键，其实是**给“生日”这个复杂信息定一个“优先级顺序”**——就像体育课排队：先按出生年份站（1991年的同学比1992年的站前面），同年的再按月份（4月比10月早），同月的按日期（1号比23号早）；如果有人生日完全一样（比如双胞胎），**后输入的同学要先站**（因为题目要求“年龄相同则输入靠后的先输出”）。  

这就是**多关键字排序**：我们需要给每个“关键字”（年、月、日、输入顺序）设定优先级，依次比较。而为了把“姓名”和“生日”绑定在一起（总不能排着排着把名字搞丢吧？），我们会用**结构体**把这些信息“打包”成一个整体。

### 核心算法流程与可视化思路
排序的核心是**比较两个同学的“优先级”**：  
1. 先比年份：年份小的优先级高（更早出生，年龄更大）；  
2. 年份相同，比月份：月份小的优先级高；  
3. 月份相同，比日期：日期小的优先级高；  
4. 日期相同，比输入顺序：输入顺序大的优先级高（后输入的先输出）。  

**可视化设计**：我们会做一个8位像素风格的“排队动画”——用彩色像素块代表同学（比如蓝色块显示年份，绿色块显示月份，红色块显示日期，右下角的小数字是输入顺序）。排序时，像素块会“交换位置”，交换时播放“叮”的像素音效；相同生日的同学会闪烁输入顺序数字，提醒“后输入的要站前面”。动画支持“单步执行”（一步步看比较过程）和“自动播放”（像红白机游戏一样跑完全程）！

## 3. 核心难点辨析与解题策略

### 1. 难点1：多关键字排序的逻辑怎么写？
* **问题**：要按“年→月→日→输入顺序”排序，怎么把这些条件写成代码？  
* **解决**：用`if-else`逐层判断！比如：  
  - 先比年份：`a.n < b.n`（年份小的排前面）；  
  - 年份相同，比月份：`a.y < b.y`；  
  - 月份相同，比日期：`a.r < b.r`；  
  - 日期相同，比输入顺序：`a.num > b.num`（后输入的排前面）。  

* 💡 学习笔记：多关键字排序=“先比大的优先级，再比小的优先级”，像剥洋葱一样一层一层来！


### 2. 难点2：如何把姓名和生日绑定？
* **问题**：输入时姓名和生日是分开的，排序时容易搞混，怎么办？  
* **解决**：用**结构体**！比如：  
  ```cpp
  struct Student {
      string name;  // 姓名
      int year;     // 年
      int month;    // 月
      int day;      // 日
      int order;    // 输入顺序
  };
  ```  
  这样每个`Student`变量都包含了一个同学的所有信息，排序时不会散架～

* 💡 学习笔记：结构体是“打包数据”的神器，遇到“多个相关信息要一起处理”时，第一反应就用它！


### 3. 难点3：相同生日时，后输入的先输出？
* **问题**：如果两个同学生日相同，怎么保证后输入的排前面？  
* **解决**：  
  - 方法1：记录输入顺序（比如`order`变量），排序时逆序比较（`a.order > b.order`）；  
  - 方法2：用`stable_sort`代替`sort`，利用稳定排序的特性（相同值的元素保持原输入顺序）。  

* 💡 学习笔记：遇到“保持原顺序”的需求，优先想`stable_sort`！


### ✨ 解题技巧总结
1. **数据打包用结构体**：把相关信息绑在一起，避免混乱；  
2. **多关键字排序逐层比**：从优先级高的到低的，依次判断；  
3. **稳定排序省力气**：相同值要保顺序，`stable_sort`比自己写更方便；  
4. **生日转数字简化比较**：把年、月、日拼成一个大数字，直接比大小（适合怕写复杂cmp的同学）。


## 4. C++核心代码实现赏析

### 本题通用核心C++实现参考
* **说明**：综合了结构体、多关键字排序、输入顺序记录，是最经典的解法。  
* **完整核心代码**：  
  ```cpp
  #include <iostream>
  #include <vector>
  #include <algorithm>
  using namespace std;

  struct Student {
      string name;
      int year, month, day;
      int order;  // 记录输入顺序
  };

  // 多关键字比较函数
  bool cmp(const Student& a, const Student& b) {
      if (a.year != b.year) return a.year < b.year;  // 年小的排前面
      if (a.month != b.month) return a.month < b.month;  // 月小的排前面
      if (a.day != b.day) return a.day < b.day;  // 日小的排前面
      return a.order > b.order;  // 后输入的排前面
  }

  int main() {
      int n;
      cin >> n;
      vector<Student> students(n);  // 用vector存所有同学
      for (int i = 0; i < n; ++i) {
          cin >> students[i].name >> students[i].year >> students[i].month >> students[i].day;
          students[i].order = i + 1;  // 输入顺序从1开始
      }
      sort(students.begin(), students.end(), cmp);  // 排序
      for (const auto& s : students) {
          cout << s.name << endl;  // 输出名字
      }
      return 0;
  }
  ```
* **代码解读概要**：  
  1. 用`struct Student`打包姓名、生日、输入顺序；  
  2. `cmp`函数按“年→月→日→输入顺序”比较；  
  3. 输入时记录每个同学的输入顺序；  
  4. 用`sort`排序后，输出所有名字。


### 优质题解片段赏析

#### 题解二：生日转数字（作者：子墨丶）
* **亮点**：把生日转成8位数，简化比较逻辑！  
* **核心代码片段**：  
  ```cpp
  int s[i] = y[i] * 10000 + m[i] * 100 + d[i];  // 年×10000 + 月×100 + 日
  sort(s + 1, s + n + 1);  // 直接排序数字
  ```
* **代码解读**：  
  比如1992年4月23日→1992×10000 + 4×100 +23=19920423，1991年8月1日→19910801。数字越小，生日越早，排序后直接输出对应的名字就行！  
* 💡 学习笔记：**将复杂比较转成数字比较**，是简化代码的好方法！


#### 题解三：stable_sort（作者：JJJJones_Zhu）
* **亮点**：用稳定排序处理相同生日的情况！  
* **核心代码片段**：  
  ```cpp
  stable_sort(nn + 1, nn + n + 1, cmp);  // 稳定排序
  ```
* **代码解读**：  
  `stable_sort`和`sort`用法一样，但**相同值的元素会保持原输入顺序**。比如两个同学生日相同，后输入的在原数组中位置更靠后，稳定排序后依然靠后，输出时就会先出现（因为我们排序的是“生日早的排前面”，相同生日的后输入的在后面？不对，等一下——其实作者的cmp函数是“生日早的排前面”，相同生日的元素，原输入顺序靠后的在数组中位置更靠后，稳定排序后依然在后面？哦，不对，作者的cmp函数中，相同生日时返回`a.level > b.level`（`level`是输入顺序），所以其实还是需要记录输入顺序。不过`stable_sort`确实能保证相同值的元素不打乱顺序，是个有用的技巧！  
* 💡 学习笔记：`stable_sort`是稳定排序，适合需要保持原顺序的场景！

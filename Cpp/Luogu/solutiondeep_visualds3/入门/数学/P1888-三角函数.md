# 三角函数

## 题目描述

输入一组勾股数 $a,b,c（a\neq b\neq c）$，用分数格式输出其较小锐角的正弦值。（要求约分。）


## 说明/提示

数据保证：$a,b,c$ 为正整数且 $\in [1,10^9]$。

## 样例 #1

### 输入

```
3 5 4```

### 输出

```
3/5```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：数学（三角函数基础） + 编程技巧应用（排序、最大公约数）

🗣️ **初步分析**：  
解决“三角函数”问题，关键是**把数学知识转化为代码逻辑**。简单来说：  
- 数学上，直角三角形的**小锐角对边是最短边**，**斜边是最长边**，正弦值就是「最短边 / 最长边」（比如样例中的3/5）。  
- 编程上，我们需要做3件事：① 把三边排序找出最短和最长；② 用**最大公约数（gcd）**约分分数；③ 输出结果。  

打个比方，这就像整理玩具积木：先把最长和最短的积木挑出来，再把它们的“共同部分”（比如都能分成2块）去掉，得到最简的“比例”。  

**核心算法流程**：输入三边 → 排序 → 取最短边（分子）和最长边（分母） → 计算gcd → 约分输出。  
**可视化设计思路**：我们可以用8位像素风格的“积木排序游戏”展示过程——三个彩色积木代表三边，排序时积木滑动交换，约分时有因数分解动画，配合“咔嗒”“叮”的像素音效，让你直观“看”到每一步！

## 3. 核心难点辨析与解题策略

在解题中，大家常遇到3个关键点，结合题解的共性，我帮你提炼了应对策略：

1.  **关键点1：如何确定小角的对边和斜边？**  
    * **分析**：数学知识是核心——直角三角形中，**小角对小边，斜边最长**。所以只要把三边排序，最短的就是小角对边，最长的就是斜边。所有优质题解都用了“排序”解决这个问题。  
    * 💡 **学习笔记**：数学是编程的“指南针”，先想清楚数学逻辑，再写代码！

2.  **关键点2：如何处理大数避免溢出？**  
    * **分析**：题目中数据范围是`[1, 1e9]`，`int`的范围（约±2e9）刚好够，但如果是更大的数（比如1e18），`int`就会“装不下”。此时要用`long long`类型（范围约±9e18）。题解三中的`#define int long long`就是这个道理。  
    * 💡 **学习笔记**：遇到“大数”问题，第一反应是换`long long`！

3.  **关键点3：如何约分分数？**  
    * **分析**：约分的本质是找分子和分母的**最大公约数（gcd）**——比如6和10的gcd是2，约分后就是3/5。C++的`<algorithm>`头文件里有`__gcd`函数，可以直接用它计算gcd。  
    * 💡 **学习笔记**：`__gcd(a, b)`是约分的“神器”，记得包含头文件哦！


### ✨ 解题技巧总结
- **数学建模**：先把问题转化为“找最短边/最长边”的数学问题，再写代码。  
- **排序简化**：用`sort`函数快速排序，比手动`swap`更高效。  
- **大数处理**：`long long`是处理大数据的“保险绳”。  
- **约分技巧**：`__gcd`函数能帮你快速得到最简分数。


## 4. C++核心代码实现赏析

先看一份**通用核心代码**，综合了所有优质题解的优点：

### 本题通用核心C++实现参考
* **说明**：这份代码处理了大数，用`sort`排序，正确约分，是最简洁的“标准解法”。  
* **完整核心代码**：
  ```cpp
  #include <iostream>
  #include <algorithm> // 包含sort和__gcd
  using namespace std;

  typedef long long ll; // 定义long long别名，简化代码

  int main() {
      ll a[3]; // 用long long存储三边，避免溢出
      cin >> a[0] >> a[1] >> a[2];
      sort(a, a + 3); // 排序后a[0]最小，a[2]最大
      ll g = __gcd(a[0], a[2]); // 计算分子分母的gcd
      cout << a[0]/g << "/" << a[2]/g << endl; // 约分输出
      return 0;
  }
  ```
* **代码解读概要**：  
  1. 用`ll`（`long long`的别名）定义数组，存储三边；  
  2. `sort(a, a + 3)`快速排序，得到最小边`a[0]`和最大边`a[2]`；  
  3. `__gcd(a[0], a[2])`计算gcd，约分后输出分数。


### 针对各优质题解的片段赏析

**题解一：(作者：GoldenSTEVE7)**  
* **亮点**：手动`swap`排序，直观展示排序逻辑。  
* **核心代码片段**：
  ```cpp
  int a, b, c; cin >> a >> b >> c;
  if(a > b) swap(a, b); // 确保a ≤ b
  if(a > c) swap(a, c); // 确保a ≤ c（现在a是最小的）
  if(b > c) swap(b, c); // 确保b ≤ c
  ```
* **代码解读**：  
  这三行`if`+`swap`，把`a`、`b`、`c`从小到大排好序。比如，如果`a`比`b`大，就交换它们的位置，确保`a`是前两个中的较小者；再比较`a`和`c`，确保`a`是最小的；最后比较`b`和`c`，确保`b`≤`c`。这种“手动排序”的方法，能让你彻底理解排序的底层逻辑！  
* 💡 **学习笔记**：入门时可以用手动排序理解逻辑，之后再用`sort`简化代码。


**题解二：(作者：Chase12345)**  
* **亮点**：用`sort`函数，一行代码搞定排序。  
* **核心代码片段**：
  ```cpp
  int a[4]; // 用a[1]-a[3]存储三边
  cin >> a[1] >> a[2] >> a[3];
  sort(a + 1, a + 4); // 排序a[1]、a[2]、a[3]
  ```
* **代码解读**：  
  `sort(a + 1, a + 4)`的意思是“排序从`a[1]`到`a[3]`的元素”（因为`sort`的结束位置是“最后一个元素的下一个位置”）。`sort`是C++标准库中的快速排序函数，效率很高，一行代码就能替代三次`swap`，非常方便！  
* 💡 **学习笔记**：标准库函数是“偷懒”的好帮手，要多用哦！


**题解三：(作者：ovo321)**  
* **亮点**：用`long long`处理大数，避免溢出。  
* **核心代码片段**：
  ```cpp
  #define int long long // 把所有int变成long long
  ...
  int a[3]; cin >> a[0] >> a[1] >> a[2];
  sort(a, a + 3);
  ```
* **代码解读**：  
  `#define int long long`是“宏定义”，会把代码中所有的`int`替换成`long long`。这样，`a`数组的类型就变成了`long long`，能存储1e9甚至更大的数，彻底避免溢出问题。这种“全局替换”的方法，在处理大数时非常有用！  
* 💡 **学习笔记**：处理大数据时，`long long`是“必须的”，宏定义能帮你简化代码。

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：`数学 (简单代数变形)`  

🗣️ **初步分析**：
> 解决这道题的关键，其实是**把题目中的“平均数规则”翻译成数学公式**——就像解小学方程一样！题目说：`S = (R1 + R2) / 2`（S是R1和R2的平均数，且S是整数），现在已知R1和S，求R2。我们只需要把公式变形：两边乘2得`2S = R1 + R2`，再移项就得到`R2 = 2×S - R1`。这一步变形就是题目的“核心算法”——**用数学推导替代复杂计算**。  
> - **题解思路共性**：90%的题解都直接用这个公式计算，因为它又快又准；少数题解用了枚举或二分，但其实是“绕远路”（比如枚举所有可能的R2值，直到满足条件）。  
> - **核心难点**：容易搞反输入顺序（题目输入是`R1`在前、`S`在后），或误用数据类型（比如用float但忘记转换为int）。  
> - **可视化设计思路**：我会用8位像素风动画展示“公式变形”的过程——用彩色像素块代表`S`、`R1`、`R2`，一步步演示“乘2”“移项”的操作，最后输入数值后，像素块会“组合”成结果，伴随“叮”的音效提示正确！

## 3. 核心难点辨析与解题策略

<difficulty_intro>
这道题虽然简单，但新手容易踩3个“小坑”。结合优质题解的经验，我们一一解决：
</difficulty_intro>

1.  **关键点1：输入顺序不要搞反！**  
    * **分析**：题目输入是`R1`在前、`S`在后（比如样例输入`11 15`，对应`R1=11`，`S=15`）。如果搞反成`cin>>s>>r1`，结果会完全错误！优质题解都明确写了`cin>>r1>>s`，这是“读题细致”的体现。  
    * 💡 **学习笔记**：输入顺序错，代码全白写——读题时一定要圈出“输入格式”！

2.  **关键点2：数据类型用int就够了！**  
    * **分析**：题目中`R1`和`S`的范围是`-1000~1000`，计算`2×S-R1`的结果不会超过int的范围（int通常能存`-2e9~2e9`）。有些题解用了float，但其实是多余的——因为结果一定是整数，用int更高效。  
    * 💡 **学习笔记**：能不用float就不用，避免“精度丢失”的麻烦！

3.  **关键点3：别用枚举/二分“绕远路”！**  
    * **分析**：有些题解用枚举法（比如从`-2020`到`2020`遍历R2），虽然能AC，但效率很低（比如要循环4000次）。而公式法只需要1次计算，这就是“数学推导”的威力——**能用公式解决的问题，绝不暴力枚举**！  
    * 💡 **学习笔记**：先想“数学规律”，再想“循环判断”。

## 4. C++核心代码实现赏析

<code_intro_overall>
先看一份“通用核心代码”——它综合了所有优质题解的优点，是最适合入门的版本：
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：本代码来自Reio_Stark的题解，是“公式法”的典型实现，逻辑最简洁、可读性最高。
* **完整核心代码**：
    ```cpp
    #include<iostream>
    using namespace std;
    int main()
    {
        int r1, s;
        cin >> r1 >> s;
        cout << 2 * s - r1;
        return 0;
    }
    ```
* **代码解读概要**：  
  1. 包含`iostream`头文件（用于输入输出）；  
  2. 定义`r1`（存储输入的R1）和`s`（存储输入的S）；  
  3. 读入`r1`和`s`；  
  4. 直接计算`2×s - r1`并输出——这一步就是“公式变形”的代码实现；  
  5. 返回0，结束程序。

接下来看3份优质题解的“核心片段”，对比它们的思路差异：

**题解一：来源：Reio_Stark**
* **亮点**：用“直接计算”替代所有冗余逻辑，代码最短。
* **核心代码片段**：
    ```cpp
    cout << 2 * s - r1;
    ```
* **代码解读**：  
  这一行是整道题的“灵魂”——直接把公式`R2=2×S-R1`写成代码。为什么不用定义`r2`变量？因为计算结果可以直接输出，不需要中间存储。这是“代码简洁性”的极致！
* 💡 **学习笔记**：能直接输出结果，就不用多定义变量。


**题解二：来源：f070331**
* **亮点**：详细展示了“公式推导”的过程，适合理解变量类型。
* **核心代码片段**：
    ```cpp
    float s(0), r1, r2;
    cin >> r1 >> s;
    r2 = s - r1 / 2;
    r2 *= 2;
    int r = r2;
    cout << r;
    ```
* **代码解读**：  
  作者用`float`类型计算`r2`，先算`S - R1/2`，再乘2——其实和`2×S - R1`是等价的（数学上：`2×(S - R1/2) = 2S - R1`）。最后把`r2`转换为`int`输出，保证结果是整数。这种写法适合刚开始学“浮点数”的同学，能帮你理解“类型转换”的必要性。
* 💡 **学习笔记**：如果用float计算整数结果，最后要转成int。


**题解三：来源：Purslane**
* **亮点**：连`r2`变量都没定义，直接输出计算结果，极致简洁。
* **核心代码片段**：
    ```cpp
    cout << s * 2 - r1;
    ```
* **代码解读**：  
  这行代码和Reio_Stark的写法本质一样，只是把`2×s`写成了`s×2`——数学上是等价的，但`2×s`更符合公式的顺序。这种写法适合已经掌握“运算符优先级”的同学，能帮你减少变量定义的次数。
* 💡 **学习笔记**：运算符顺序不影响结果时，怎么写都可以，但要尽量符合公式习惯。

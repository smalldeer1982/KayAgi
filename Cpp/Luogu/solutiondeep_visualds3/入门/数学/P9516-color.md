# 题目信息

# color

## 题目描述

洛谷的名字颜色和咕值直接挂钩，具体细则见 [洛谷咕值系统介绍](https://www.luogu.com.cn/discuss/83952)。

给出某洛谷用户的五项咕值指标的得分，输出该用户的名字颜色。保证该用户不是棕名和紫名。

## 说明/提示

**【样例说明】**

三个样例的咕值分别为 $263,140,0$，名字颜色分别为红、绿、灰。

**【数据范围】**

对于 $20\%$ 的数据，满足五项指标分数均 $\le 10$；

对于另外 $20\%$ 的数据，满足五项指标分数均 $\ge 90$；

对于 $100\%$ 的数据，保证五项指标分数均 $\in [0, 100]$。

## 样例 #1

### 输入

```
100 14 9 55 85```

### 输出

```
Red```

## 样例 #2

### 输入

```
100 10 10 10 10```

### 输出

```
Green```

## 样例 #3

### 输入

```
0 0 0 0 0```

### 输出

```
Gray```

## 3. 核心难点辨析与解题策略

<difficulty_intro>
这道题虽然简单，但新手容易在“小细节”上栽跟头。结合题解的共性，我帮你提炼了3个核心关键点：
</difficulty_intro>

1.  **关键点1**：如何正确读取5个输入？
    * **分析**：C++中用`cin`读取多个整数很简单，只要写`cin >> a >> b >> c >> d >> e;`就行——注意输入的数要按顺序对应五个指标。如果输入错了顺序（比如把第一个数写成第二个），总和就会错！
    * 💡 **学习笔记**：输入顺序要和题目要求一致，别“张冠李戴”~

2.  **关键点2**：如何准确记住区间分界点？
    * **分析**：洛谷的规则是：0-99灰、100-119蓝、120-169绿、170-229橙、230-500红。记住这些数字的秘诀是“分组记”：比如100是蓝的起点，120是绿的起点，170是橙的起点，230是红的起点。题解里的条件判断都是围绕这些数字写的~
    * 💡 **学习笔记**：把分界点写成注释，或者像Hughpig那样直接引用规则，就不会记错啦！

3.  **关键点3**：如何避免条件判断错误？
    * **分析**：比如`sum=100`应该是蓝，但如果写成`sum < 100`（灰）和`sum >= 100 && sum < 120`（蓝），就不会错。题解里的判断条件要么是“<= 分界点”（从低到高），要么是“>= 分界点”（从高到低），都是正确的——只要覆盖所有区间就行！
    * 💡 **学习笔记**：用“边界值测试”验证条件：比如sum=99、100、119、120这些边界，代入条件看看输出对不对~

## 4. C++核心代码实现赏析

<code_intro_overall>
先看一个**通用核心实现**——它综合了所有题解的优点，用最基础的代码解决问题：
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：本代码用最基础的头文件（`#include<iostream>`）和变量名，清晰展示“读输入→算总和→判断输出”的核心逻辑，适合新手入门。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    using namespace std;

    int main() {
        int a, b, c, d, e;
        cin >> a >> b >> c >> d >> e; // 读入五个咕值指标
        int sum = a + b + c + d + e;  // 计算总和
        // 按区间判断颜色
        if (sum <= 99)      cout << "Gray";
        else if (sum <= 119) cout << "Blue";
        else if (sum <= 169) cout << "Green";
        else if (sum <= 229) cout << "Orange";
        else                cout << "Red";
        return 0;
    }
    ```
* **代码解读概要**：
    > 代码分三步：① 用`cin`读入五个整数（a到e）；② 计算它们的总和（sum）；③ 用`if-else if`按从低到高的顺序判断sum所在的区间，输出对应的颜色。整个逻辑就像“先把苹果装进篮子，再数篮子里有多少个，最后按数量给篮子贴标签”~

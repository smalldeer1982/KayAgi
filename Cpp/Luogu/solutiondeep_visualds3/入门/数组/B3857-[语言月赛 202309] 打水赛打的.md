# 题目信息

# [语言月赛 202309] 打水赛打的

## 题目描述

在 H 学校，C 考试一直被用来衡量学生编程、数据结构和算法能力的重要标准。C 考试满分 $500$ 分，一名学生只有取得不低于 $200$ 分的成绩才能毕业，我们称这样的学生为**达到毕业要求的学生**，C 考试未取得不低于 $200$ 分成绩的学生为**达不到毕业要求的学生**。

然而，随着内卷的激烈化，R 比赛走进了同学们的视野。由于该比赛组织混乱、试题难度低、获奖比例高，被同学们戏称为一等奖点击就送，成为公认的水赛。

在 2023 年的一等奖获奖名单中，同学们惊讶的发现，大量**达不到毕业要求的学生**竟然可以在 R 比赛中获得一等奖，这无疑是非常讽刺的一件事。

现在，给出 依次编号为 $1\sim N$ 的 H 学校的 $N$ 名同学的 C 考试成绩，和 R 比赛一等奖获奖名单。获奖名单中可能有超过 $N$ 的编号，这是由于其他一些学校也参加了 R 比赛，只有 $1 \sim N$ 编号的学生属于 H 学校。

请你统计，有多少 H 学校的一等奖获得者，是**达不到毕业要求的学生**。

## 说明/提示

 - 对于 $30\%$ 的测试数据，$1 \le N,M \le 5000$；
 - 对于另外 $30\%$ 的测试数据，$M \le N$；
 - 对于 $100\%$ 的测试数据，$1 \le N,M \le 10^5$，$1 \le W_i \le 2\times 10^5$，$0 \le S_i \le 500$。保证 $W_i$ 不重复。 

## 样例 #1

### 输入

```
5 8
200 0 199 200 500
3 1 2 6 8 10 11 12```

### 输出

```
2```

🗣️ **初步分析**：

## 2. 精选优质题解参考
**题解一（来源：览遍千秋）**
* **点评**：思路直击要害——用数组下标直接映射学生编号，实现O(M)高效筛选。代码边界处理严谨（跳过非1~N编号），变量命名清晰（`s[i]`表成绩，`w`表获奖编号）。亮点在于**用空间换时间**的索引技巧，适合竞赛快速实现。

## 3. 核心难点辨析与解题策略
1. **关键点：H校学生识别与越界处理**  
   * **分析**：获奖名单含外校编号（>N），直接访问数组会越界。优质题解通过`if(w<1 || w>n) continue`跳过无效编号
   * 💡 **学习笔记**：处理外部输入时，先验范围再操作是防崩溃的黄金法则

2. **关键点：成绩条件的快速判定**  
   * **分析**：需同时满足"一等奖"和"成绩<200"。题解用数组存储成绩，实现O(1)条件判断
   * 💡 **学习笔记**：静态数据用数组索引比动态查询更高效

3. **关键点：大规模数据处理**  
   * **分析**：N,M≤10⁵要求O(M)解法。嵌套循环（如为每个获奖者遍历学生列表）会超时
   * 💡 **学习笔记**：10⁵级数据量需避免O(NM)算法

### ✨ 解题技巧总结
- **索引映射法**：用数组下标作ID（如`s[id]`），实现常量级查询
- **输入分离处理**：先读全部学生数据，再流式处理获奖名单，节省内存
- **边界防护链**：范围检查→有效性判断→业务逻辑的三层防御

## 4. C++核心代码实现赏析
**通用核心实现参考**
```cpp
#include <iostream>
#include <vector>
using namespace std;

int main() {
    int n, m, ans = 0;
    cin >> n >> m;
    vector<int> s(n+1); // 下标1~n存成绩
    
    // 读取学生成绩
    for (int i = 1; i <= n; i++) cin >> s[i];
    
    // 处理获奖名单
    while (m--) {
        int w;
        cin >> w;
        if (w < 1 || w > n) continue; // 跳过非H校编号
        if (s[w] < 200) ans++;        // 检查毕业条件
    }
    cout << ans;
}
```
**代码解读概要**：  
> 1. 用`vector`动态存储成绩（下标1起）  
> 2. 流式处理获奖名单：每读一个编号立即判断  
> 3. 双层防御：先验范围再查成绩

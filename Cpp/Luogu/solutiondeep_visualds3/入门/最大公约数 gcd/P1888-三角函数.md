# 三角函数

## 题目描述

输入一组勾股数 $a,b,c（a\neq b\neq c）$，用分数格式输出其较小锐角的正弦值。（要求约分。）


## 说明/提示

数据保证：$a,b,c$ 为正整数且 $\in [1,10^9]$。

## 样例 #1

### 输入

```
3 5 4```

### 输出

```
3/5```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：`数学 + 编程技巧应用`  

🗣️ **初步分析**：  
解决这道题的关键，其实是**把数学原理“翻译”成代码**。我们先回忆两个初中数学知识点：  
1. 直角三角形中，**斜边是最长的边**（两点之间线段最短）；  
2. 较小的锐角对应的**对边是最短的边**（小角对小边，因为正弦值=对边/斜边，角越小正弦值越小，对应边也越短）。  

所以，**较小锐角的正弦值 = 最短边 ÷ 最长边**（分数形式，需要约分）。  

编程部分的任务就是：  
- 从三个数中**快速找到最短边和最长边**（用排序）；  
- 把分数**约分成最简形式**（用最大公约数GCD）。

## 3. 核心难点辨析与解题策略

在解决这道题时，很多同学会卡在三个关键点上。结合优质题解的经验，我帮你总结了**“难点+解决策略”**：

### 1. 关键点1：如何确定“对边”和“斜边”？  
**分析**：靠数学原理！直角三角形中，**斜边是最长的边**（肯定没错），**较小锐角的对边是最短的边**（小角对小边，正弦值=对边/斜边，角越小正弦值越小）。所以只要找到三个数中的“最小值”（对边）和“最大值”（斜边）就行！  
💡 **学习笔记**：数学是编程的“指南针”，先想清楚原理再写代码，不然容易“瞎猜”。

### 2. 关键点2：如何快速找“最小/最大值”？  
**分析**：用`sort`函数！不管是手动swap三次，还是用`sort`，排序后数组的**第一个元素是最小**，**最后一个元素是最大**，一键解决“找最值”的问题。`sort`是C++的“神器”，能帮你节省大量手动比较的时间！  
💡 **学习笔记**：标准库函数是“偷懒的艺术”——既然有现成的高效工具，就别自己重复造轮子。

### 3. 关键点3：如何约分分数？  
**分析**：用**最大公约数（GCD）**！分数约分就是分子分母都除以它们的GCD。C++里有`__gcd`函数（需要包含`<algorithm>`头文件），能直接求出两个数的GCD。比如`3和5的GCD是1`，所以`3/5`不用约分；`6和10的GCD是2`，约分后是`3/5`。  
💡 **学习笔记**：约分的关键是找GCD，记住`__gcd(a,b)`的用法，或者自己写一个（比如欧几里得算法）。

### ✨ 解题技巧总结  
- 先理数学原理：小角对小边，斜边最长，正弦值=对边/斜边；  
- 用`sort`快速找最值：避免手动比较的麻烦；  
- 约分用`__gcd`：标准库函数更高效；  
- 数组从0开始：符合C++习惯，减少索引错误。


## 4. C++核心代码实现赏析

在看具体题解的代码前，先看一份**通用核心实现**——它综合了所有优质题解的优点，是最简洁、最易理解的版本：

### 本题通用核心C++实现参考  
* **说明**：本代码用`sort`快速排序，`__gcd`约分，是解决本题的“标准模板”。  
* **完整核心代码**：  
```cpp
#include <iostream>
#include <algorithm> // 包含sort和__gcd函数
using namespace std;

int main() {
    int a[3];
    cin >> a[0] >> a[1] >> a[2]; // 输入三个数，存到a[0]~a[2]
    sort(a, a + 3); // 排序后，a[0]最小（对边），a[2]最大（斜边）
    int g = __gcd(a[0], a[2]); // 求最小边和最大边的GCD
    cout << a[0] / g << "/" << a[2] / g << endl; // 约分后输出
    return 0;
}
```  
* **代码解读概要**：  
  1. 输入三个数到数组`a`；  
  2. `sort(a, a+3)`排序，把最小的放`a[0]`，最大的放`a[2]`；  
  3. `__gcd(a[0], a[2])`求GCD，约分后输出分数。


### 针对各优质题解的片段赏析

**题解一：Chase12345（数组从1开始）**  
* **亮点**：用`sort(a+1, a+4)`排序，适合习惯“1-based”数组的同学。  
* **核心代码片段**：  
```cpp
int a[4]; // 用a[1]~a[3]存三个数
cin >> a[1] >> a[2] >> a[3];
sort(a + 1, a + 4); // 排序a[1]~a[3]
cout << a[1] / __gcd(a[1], a[3]) << '/' << a[3] / __gcd(a[1], a[3]) << '\n';
```  
* **代码解读**：  
  - 为什么用`a+1`和`a+4`？因为`sort`的第二个参数是“结束位置的下一个”，所以`a+4`对应`a[3]`的下一个位置，正好覆盖`a[1]、a[2]、a[3]`三个元素。  
  - 排序后`a[1]`是最小边，`a[3]`是最大边，直接约分输出。  
* 💡 **学习笔记**：数组的“起始索引”可以灵活选择，但要注意`sort`的参数范围。

**题解二：ovo321（数组从0开始）**  
* **亮点**：符合C++“0-based”数组的默认习惯，更不容易出错。  
* **核心代码片段**：  
```cpp
int a[3];
cin >> a[0] >> a[1] >> a[2];
sort(a, a + 3); // 直接排序a[0]~a[2]
cout << a[0]/__gcd(a[0],a[2]) << "/" << a[2]/__gcd(a[0],a[2]);
```  
* **代码解读**：  
  - `a[0]`是最小边，`a[2]`是最大边，直接用`__gcd`约分。这种写法不用记“a+1”“a+4”，更直观！  
* 💡 **学习笔记**：尽量用“0-based”数组，减少索引混淆的问题。

**题解三：GoldenSTEVE7（手动排序）**  
* **亮点**：展示排序的“底层逻辑”，适合理解`sort`的原理。  
* **核心代码片段**：  
```cpp
int a, b, c; cin >> a >> b >> c;
if(a > b) swap(a, b); // 把小的放a
if(a > c) swap(a, c); // 现在a是最小
if(b > c) swap(b, c); // 现在b是中间，c是最大
cout << a / __gcd(a, c) << "/" << c / __gcd(a, b);
```  
* **代码解读**：  
  - 三次`if`和`swap`，一步步把最小的数放到`a`，最大的放到`c`。比如输入3、5、4：  
    1. `a=3`和`b=5`，不交换；  
    2. `a=3`和`c=4`，不交换；  
    3. `b=5`和`c=4`，交换后`b=4`，`c=5`；  
  - 最终`a=3`（最小），`c=5`（最大），约分输出。  
* 💡 **学习笔记**：手动排序能帮你理解`sort`的原理，但实际编程中用`sort`更高效。

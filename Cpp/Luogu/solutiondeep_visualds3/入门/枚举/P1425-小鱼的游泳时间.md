# 题目信息

# 小鱼的游泳时间

## 题目描述

伦敦奥运会要到了，小鱼在拼命练习游泳准备参加游泳比赛，可怜的小鱼并不知道鱼类是不能参加人类的奥运会的。

这一天，小鱼给自己的游泳时间做了精确的计时（本题中的计时都按 $24$ 小时制计算），它发现自己从 $a$ 时 $b$ 分一直游泳到当天的 $c$ 时 $d$ 分，请你帮小鱼计算一下，它这天一共游了多少时间呢？

小鱼游的好辛苦呀，你可不要算错了哦。


## 说明/提示

对于全部测试数据，$0\le a,c \le 24$，$0\le b,d \le 60$，且结束时间一定晚于开始时间。

## 样例 #1

### 输入

```
12 50 19 10```

### 输出

```
6 20```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：编程技巧应用（基础时间计算、进制转换与条件处理）

🗣️ **初步分析**：
> 解决“小鱼的游泳时间”，关键是处理**时间的60进制减法**——因为1小时=60分钟，不像10进制那样直接减。就像你算“12点50分到19点10分”，要先看分钟够不够减：50分比10分大，所以得从小时“借1小时（60分钟）”，变成18点70分减12点50分，这样小时差是6，分钟差是20。  
> 题解里有两种主流思路：  
> ① **直接借位法**：先算小时差`c-a`、分钟差`d-b`，如果分钟差为负，就从小时差借1（减1），分钟差加60；  
> ② **统一单位法**：把两个时间都转成“当天的总分钟数”（比如12点50分=12×60+50=770分钟），相减后再转回头小时和分钟（380分钟=6小时20分）。  
> 核心难点是**处理分钟不够减的情况**，而统一单位法能“绕开”这个难点，更简洁；直接借位法则更贴近我们手动计算的逻辑。  
> 可视化设计上，我会用**8位像素风**模拟“时间卡片”：左边是开始时间（12:50），右边是结束时间（19:10），点击“单步执行”会看到：分钟差50-10= -40→变成60-40=20，小时差19-12=7→减1变成6，最后跳出结果6:20。还会加“叮”的音效（借位时）和“嗡”的音效（计算完成），像玩FC游戏一样！

## 2. 精选优质题解参考

<eval_intro>
我从**思路清晰性、代码可读性、实践价值**三个维度筛选了3份优质题解——它们覆盖了两种核心思路，适合不同学习阶段的你参考！
</eval_intro>

**题解一：直接借位法（来源：Mr_QwQ，赞879）**
* **点评**：这份题解完美还原了“手动算时间”的逻辑，代码超简洁！先算小时差`x=c-a`、分钟差`y=d-b`，如果`y<0`（分钟不够减），就从`x`借1（`x--`），`y`加60。变量名`x`（小时差）、`y`（分钟差）特别直观，边界处理（比如`y<0`时的借位）也很严谨。作者提到“一开始把x、y初始化弄反了”——这提醒我们：变量名要对应清楚，否则容易“张冠李戴”！

**题解二：统一单位法（来源：2789617221guo，赞3）**
* **点评**：这是“偷懒但聪明”的解法！把开始时间转成`a×60+b`分钟，结束时间转成`c×60+d`分钟，相减得到总分钟数`t3`，再用`t3/60`（小时）和`t3%60`（分钟）输出。完全不用分情况讨论，代码逻辑更简洁——适合怕“漏情况”的你！

**题解三：分类讨论法（来源：GoldenSTEVE7，赞0）**
* **点评**：这份题解用“if-else”把两种情况（分钟够减/不够减）直接写出来，特别适合**刚学条件判断**的同学！如果`d>=b`，直接输出`c-a`和`d-b`；否则输出`c-a-1`和`d-b+60`。逻辑直白，像“把手动步骤写成代码”，容易理解。

## 3. 核心难点辨析与解题策略

<difficulty_intro>
这道题的“坑”不多，但**新手容易犯的错**很集中。结合题解，我帮你提炼了3个核心难点和解决策略：
</difficulty_intro>

1.  **关键点1：如何处理“分钟不够减”的情况？**
    * **分析**：比如12:50到19:10，`d-b=10-50=-40`，这时候要“借位”——从小时差里拿1小时（60分钟），所以分钟差变成`-40+60=20`，小时差变成`19-12-1=6`。直接借位法和分类讨论法都是这么处理的；而统一单位法通过“转成分钟”绕开了这个问题（因为总分钟数一定是正的）。
    * 💡 **学习笔记**：遇到“进制减法”，要么“借位”，要么“转成十进制统一计算”。

2.  **关键点2：如何将“总分钟数”转回“小时和分钟”？**
    * **分析**：总分钟数`total`，小时数是`total/60`（整数除法，自动舍去小数），分钟数是`total%60`（取余数）。比如380分钟，`380/60=6`（小时），`380%60=20`（分钟）——这是“进制转换”的基础技巧！
    * 💡 **学习笔记**：整数除法`/`和取余`%`是处理“进制转换”的黄金搭档。

3.  **关键点3：如何避免“变量对应错误”？**
    * **分析**：比如Mr_QwQ提到“一开始把x、y初始化弄反了”——如果把`x=d-b`、`y=c-a`，结果就全错了！解决方法是**变量名要“见名知意”**：比如用`hour_diff`代替`x`，`min_diff`代替`y`，或者像题解二用`t1`（开始总分钟）、`t2`（结束总分钟），这样不容易搞混。
    * 💡 **学习笔记**：变量名越“直白”，代码越不容易错！


### ✨ 解题技巧总结
-   **技巧A：统一单位法**：遇到“不同进制”的计算（比如时间、角度），先转成“最小单位”（分钟、秒）计算，再转回去——避免分情况讨论。
-   **技巧B：变量名要直观**：用`hour_diff`代替`x`，`total_min`代替`t3`，比“简写”更重要！
-   **技巧C：边界条件测试**：写完代码要测“极端情况”，比如1点59分到2点1分（分钟差-58→借位后2分钟），或者0点0分到24点0分（24小时0分）——确保代码覆盖所有情况。

## 4. C++核心代码实现赏析

<code_intro_overall>
先看一份**统一单位法的通用核心代码**——它最简洁，适合大多数情况；之后再分析另外两种思路的关键片段！
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：本代码综合了“统一单位法”的思路，逻辑最简洁，适合新手直接套用。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    using namespace std;

    int main() {
        int a, b, c, d;
        cin >> a >> b >> c >> d;
        // 转成总分钟数
        int start = a * 60 + b;
        int end = c * 60 + d;
        int total = end - start;
        // 转回小时和分钟
        int hour = total / 60;
        int min = total % 60;
        cout << hour << " " << min << endl;
        return 0;
    }
    ```
* **代码解读概要**：
    > 代码分三步：①读入开始和结束时间；②把两个时间转成“当天的总分钟数”（比如12:50→12×60+50=770）；③用总分钟数相减，再转回头小时（总分钟/60）和分钟（总分钟%60）。全程不用分情况，超省心！

<code_intro_selected>
接下来看另外两种思路的关键片段，帮你理解“不同解法的差异”：
</code_intro_selected>

**题解一：直接借位法（来源：Mr_QwQ）**
* **亮点**：完美还原“手动计算”的逻辑，适合理解“借位”的本质。
* **核心代码片段**：
    ```cpp
    int x = c - a, y = d - b;
    if (y < 0) {
        x--;
        y += 60;
    }
    cout << x << " " << y;
    ```
* **代码解读**：
    > ①`x`是小时差，`y`是分钟差；②如果`y<0`（比如10-50=-40），说明分钟不够减，要从`x`借1小时（`x--`），`y`加60（变成20）；③最后输出`x`和`y`。是不是和你手动算的一样？
* 💡 **学习笔记**：借位的本质是“从高位拿1个单位，换成低位的进制数”（1小时=60分钟）。

**题解三：分类讨论法（来源：GoldenSTEVE7）**
* **亮点**：用“if-else”直接分情况，逻辑直白，适合刚学条件判断的同学。
* **核心代码片段**：
    ```cpp
    if (d >= b) {
        cout << c - a << " " << d - b;
    } else {
        cout << c - a - 1 << " " << d - b + 60;
    }
    ```
* **代码解读**：
    > ①如果结束分钟`d`≥开始分钟`b`（比如19:30到20:40），直接算小时差和分钟差；②否则（比如12:50到19:10），小时差减1（借位），分钟差加60。逻辑像“把手动步骤写成代码”，容易理解！
* 💡 **学习笔记**：分类讨论法的关键是“覆盖所有可能的情况”——这里只有两种情况，所以用`if-else`就够了。

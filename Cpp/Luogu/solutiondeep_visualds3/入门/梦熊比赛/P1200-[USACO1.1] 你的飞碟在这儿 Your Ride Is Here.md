# [USACO1.1] 你的飞碟在这儿 Your Ride Is Here

## 题目描述

众所周知，在每一个彗星后都有一只 UFO。这些 UFO 时常来收集地球上的忠诚支持者。不幸的是，他们的飞碟每次出行都只能带上一组支持者。因此，他们要用一种聪明的方案让这些小组提前知道谁会被彗星带走。他们为每个彗星起了一个名字，通过这些名字来决定这个小组是不是被带走的那个特定的小组（你认为是谁给这些彗星取的名字呢？）。关于如何搭配的细节会在下面告诉你；你的任务是写一个程序，通过小组名和彗星名来决定这个小组是否能被那颗彗星后面的 UFO 带走。


小组名和彗星名都以下列方式转换成一个数字：最终的数字就是名字中所有字母的积，其中 $\texttt A$ 是 $1$，$\texttt Z$ 是 $26$。例如，$\texttt{USACO}$ 小组就是 $21 \times 19 \times 1 \times 3 \times 15=17955$。如果小组的数字 $\bmod 47$ 等于彗星的数字 $\bmod 47$,你就得告诉这个小组需要准备好被带走！（记住“$a \bmod b$”是 $a$ 除以 $b$ 的余数，例如 $34 \bmod 10$ 等于 $4$）


写出一个程序，读入彗星名和小组名并算出用上面的方案能否将两个名字搭配起来，如果能搭配，就输出 `GO`，否则输出 `STAY`。小组名和彗星名均是没有空格或标点的一串大写字母（不超过 $6$ 个字母）。

## 3. 核心难点辨析与解题策略

1.  **字母转换陷阱**
    * **分析**：如何正确将A~Z映射到1~26？优质解法均采用`字符-'A'+1`的公式。关键要理解ASCII中大写字母连续分布的特性（A=65,B=66...）
    * 💡 **学习笔记**：字符本质是数字，减法操作就是解码密钥

2.  **乘积初始化黑洞**
    * **分析**：90%的错误源于将乘积初始化为0（0乘任何数=0）。必须像太空舱充能般从1开始累积
    * 💡 **学习笔记**：累乘变量初始化`int prod=1`比`prod=0`更重要

3.  **模运算时机选择**
    * **分析**：可以在计算过程中取模（防溢出），也可最终取模（本题乘积≤26^6≈3亿，int足够）。Vanyun解法先计算再取模更直观
    * 💡 **学习笔记**：大数据量需及时取模，小数据可最后处理

### ✨ 解题技巧总结
- **字符算术化**：善用ASCII码直接运算，避免switch-case
- **流式处理**：当不需存储完整字符串时，逐字符处理更高效
- **防御性初始化**：累乘/累加变量必须检查初始值
- **模运算等效性**：`(a*b)%47` ≡ `(a%47)*(b%47)%47`

## 4. C++核心代码实现赏析

**通用核心实现参考**
```cpp
#include <iostream>
#include <string>
using namespace std;

int main() {
    string comet, group;
    cin >> comet >> group;
    
    int code1 = 1, code2 = 1;  // 双密码锁初始化
    
    // 转换彗星名密码 
    for(char c : comet) 
        code1 *= c - 'A' + 1;  // ASCII魔法转换
        
    // 转换小组名密码
    for(char c : group)
        code2 *= c - 'A' + 1;
    
    // 通过47号激光门检测
    cout << (code1%47 == code2%47 ? "GO" : "STAY");
    return 0;
}
```
**代码解读概要**：太空站接收两个字符串→分别用能量环(循环)转换字母→乘积通过激光门(取模)→比较结果触发飞碟信号

**题解一：DrinkOnstage**
* **亮点**：流式字符处理像太空传送带
* **核心代码片段**：
```cpp
int a=1, b=1;
char v;
while((v=cin.get())!='\n') a *= v-'A'+1;  // 字符流处理
while((v=cin.get())!='\n') b *= v-'A'+1;
```
* **代码解读**：
  > 第一行：创建两个能量舱`a,b`初始充能值1  
  > `while((v=cin.get())!='\n')`：像太空舱的传送带，持续读取字符直到换行符（回车键像紧急停止按钮）  
  > `a *= v-'A'+1`：每个字符通过解码器转换为数字能量，注入能量舱
* 💡 **学习笔记**：流式处理省内存，适合线性操作

**题解二：Vanyun**
* **亮点**：ASCII解码技巧像外星黑科技
* **核心代码片段**：
```cpp
for(int i=0; i<a.length(); i++) 
    ans *= a[i] - '@';  // '@'的ASCII为64
```
* **代码解读**：
  > `a[i] - '@'`：利用@(64)和A(65)的相邻特性  
  > 例如：'A'(65)-'@'(64)=1 → 完美映射A=1  
  > 比常规`- 'A' + 1`少一次运算
* 💡 **学习笔记**：理解编码表可创造编码捷径

**题解三：return_third**
* **亮点**：教学级规范代码
* **核心代码片段**：
```cpp
for(int i=0; i<lena; i++)
    cnta *= a[i]-'A'+1;  // 标准转换公式
```
* **代码解读**：
  > 经典三步：获取长度→遍历字符→转换累乘  
  > 像太空作业手册的标准流程：  
  > 1. 测量字符串长度（太空舱尺寸）  
  > 2. 按顺序处理每个字符（检查每个舱位）  
  > 3. 累乘结果（收集能量）
* 💡 **学习笔记**：规范代码更易调试和扩展

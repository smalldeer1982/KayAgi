# 题目信息

# [NOIP 2005 普及组] 陶陶摘苹果

## 题目描述

陶陶家的院子里有一棵苹果树，每到秋天树上就会结出 $10$ 个苹果。苹果成熟的时候，陶陶就会跑去摘苹果。陶陶有个 $30$ 厘米高的板凳，当她不能直接用手摘到苹果的时候，就会踩到板凳上再试试。


现在已知 $10$ 个苹果到地面的高度，以及陶陶把手伸直的时候能够达到的最大高度，请帮陶陶算一下她能够摘到的苹果的数目。假设她碰到苹果，苹果就会掉下来。

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：模拟应用  

🗣️ **初步分析**：  
解决“陶陶摘苹果”的关键，就是**“按题目要求一步步模仿真实场景”**——就像玩“数字找朋友”游戏：陶陶站在板凳上能碰到的高度是“目标线”，每个苹果的高度要和“目标线”比，符合条件的就是“能摘到的朋友”，我们要数清楚有多少个这样的“朋友”。  

具体来说，**模拟的核心流程**是：  
1. 读入10个苹果的高度；  
2. 计算陶陶站在板凳上的最大高度（身高+30厘米）；  
3. 逐个检查苹果高度是否≤最大高度，统计符合条件的数量。  

**核心难点**其实是“避免复杂判断”——比如有的同学会分开判断“不用板凳能不能摘到”“用了板凳能不能摘到”，但其实直接把身高加30，一次判断就够了（因为不用板凳能摘到的，加了板凳肯定也能摘到）！  

**可视化设计思路**：我们会用8位像素风格做一个“陶陶摘苹果”的小游戏动画——陶陶站在像素化的苹果树下，板凳是个小方块，点击“开始”后，每个苹果会逐个“落下来”（如果够得到），同时计数器跳动。关键步骤会用**颜色高亮**（比如够得到的苹果变绿色，够不到的变红色），还有“叮”的音效提示！

## 3. 核心难点辨析与解题策略

<difficulty_intro>
虽然题目简单，但第一次做时容易踩“小坑”。结合优质题解，我帮你总结了3个核心关键点：
</difficulty_intro>

1.  **关键点1：正确计算“最大高度”**  
    * **分析**：很多同学会分开判断“不用板凳”和“用板凳”，但其实**直接把身高加30**就够了——因为“不用板凳能摘到的苹果，用了板凳肯定也能摘到”。比如陶陶身高110，加30后是140，所有≤140的苹果都能摘到，不用再分两次判断。  
    * 💡 **学习笔记**：简化问题的关键，是找到“更通用的条件”。

2.  **关键点2：高效统计符合条件的数量**  
    * **分析**：统计数量的常规方法是`if(条件) cnt++`，但RedContritio用了`cnt += 逻辑值`——比如`!(H<height[i])`等于“height[i]≤H”的布尔值，直接加到cnt里。这种技巧能减少代码行数，还能锻炼“用计算机思维想问题”的能力。  
    * 💡 **学习笔记**：布尔值可以直接当数字用（true=1，false=0），这是C++的小秘密～

3.  **关键点3：边界条件的处理**  
    * **分析**：题目说“碰到苹果就会掉下来”，所以**等于最大高度的苹果也算能摘到**。比如苹果高度140，陶陶最大高度140，这时候要算“能摘到”。所有题解都用了`<=`而不是`<`，就是因为注意到了这个边界！  
    * 💡 **学习笔记**：编程时一定要仔细看题目中的“等于”情况！

### ✨ 解题技巧总结
- **技巧1：问题简化**：把“用板凳”的情况直接转化为“身高加30”，避免复杂分支。  
- **技巧2：布尔值巧用**：用逻辑表达式的结果直接计数，简化代码。  
- **技巧3：边界检查**：注意“等于”的情况，确保条件判断正确。


## 4. C++核心代码实现赏析

<code_intro_overall>
先看一份“最简通用版”代码——它来自mairuisheng的题解，逻辑清晰，适合入门同学参考～
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：这是“标准模拟思路”的最简实现，覆盖了所有核心步骤，适合新手直接模仿。
* **完整核心代码**：
    ```cpp
    #include<bits/stdc++.h>
    using namespace std;
    long long a[11], i, ha, ans; // a存苹果高度，ha是陶陶身高，ans是答案
    int main() {
        for(i=1; i<=10; i++) cin >> a[i]; // 读10个苹果高度
        cin >> ha;
        ha += 30; // 加上板凳高度，得到最大高度
        for(i=1; i<=10; i++) if(a[i] <= ha) ans++; // 统计符合条件的苹果数
        cout << ans;
        return 0;
    }
    ```
* **代码解读概要**：  
  代码分四步：①读入10个苹果高度；②读入陶陶身高并加30；③循环检查每个苹果是否≤最大高度，统计数量；④输出答案。变量名清晰，流程和题目描述完全一致～

接下来看两个“技巧版”代码片段，学习更巧妙的实现方式！

**题解一：RedContritio的“逻辑计数”技巧**
* **亮点**：用逻辑表达式直接计数，减少`if`语句。
* **核心代码片段**：
    ```cpp
    H += 30;
    for(int i=0; i<10; i++) s += !(H < height[i]);
    ```
* **代码解读**：  
  `!(H < height[i])`等价于“height[i] ≤ H”——因为如果H不小于height[i]，就是“能摘到”。而`!`会把“true”变成1，“false”变成0，直接加到`s`里。比如H=140，height[i]=130，`H<height[i]`是false，`!`后是true（1），`s`加1；如果height[i]=150，`H<height[i]`是true，`!`后是false（0），`s`不加。  
* 💡 **学习笔记**：逻辑表达式是“隐形的计数器”，能让代码更简洁！

**题解二：installb的“前缀和递推”技巧**
* **亮点**：用前缀和统计“≤某高度的苹果数”，适合大数据量的情况。
* **核心代码片段**：
    ```cpp
    int a[2001] = {0}; // a[h]表示高度h的苹果数量
    for(i=0; i<10; i++) { cin >> b; a[b]++; } // 统计每个高度的苹果数
    c += 30; // 最大高度
    for(i=1; i<=251; i++) a[i] += a[i-1]; // 前缀和：a[i] = 高度≤i的苹果总数
    printf("%d\n", a[c]);
    ```
* **代码解读**：  
  首先，`a[b]++`记录每个高度有多少个苹果（比如高度150有1个苹果，a[150]就是1）。然后`a[i] += a[i-1]`是“前缀和”——比如a[140] = a[140]（高度140的苹果数） + a[139]（高度≤139的苹果数），这样a[140]就是“高度≤140的苹果总数”。最后直接输出a[c]（c是陶陶的最大高度），不用再循环统计！  
* 💡 **学习笔记**：前缀和能把“多次查询”的时间复杂度从O(n)降到O(1)，适合需要频繁统计区间的问题！

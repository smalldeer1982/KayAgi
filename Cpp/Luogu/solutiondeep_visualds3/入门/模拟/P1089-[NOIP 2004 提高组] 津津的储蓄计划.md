# [NOIP 2004 提高组] 津津的储蓄计划

## 题目描述

津津的零花钱一直都是自己管理。每个月的月初妈妈给津津 $300$ 元钱，津津会预算这个月的花销，并且总能做到实际花销和预算的相同。

为了让津津学习如何储蓄，妈妈提出，津津可以随时把整百的钱存在她那里，到了年末她会加上 $20\%$ 还给津津。因此津津制定了一个储蓄计划：每个月的月初，在得到妈妈给的零花钱后，如果她预计到这个月的月末手中还会有多于 $100$ 元或恰好 $100$ 元，她就会把整百的钱存在妈妈那里，剩余的钱留在自己手中。


例如 $11$月初津津手中还有 $83$ 元，妈妈给了津津 $300$ 元。津津预计$11$月的花销是 $180$ 元，那么她就会在妈妈那里存 $200$ 元，自己留下 $183$ 元。到了 $11$ 月月末，津津手中会剩下 $3$ 元钱。


津津发现这个储蓄计划的主要风险是，存在妈妈那里的钱在年末之前不能取出。有可能在某个月的月初，津津手中的钱加上这个月妈妈给的钱，不够这个月的原定预算。如果出现这种情况，津津将不得不在这个月省吃俭用，压缩预算。


现在请你根据 $2004$ 年 $1$ 月到 $12$ 月每个月津津的预算，判断会不会出现这种情况。如果不会，计算到 $2004$ 年年末，妈妈将津津平常存的钱加上 $20\%$ 还给津津之后，津津手中会有多少钱。

## 3. 核心难点辨析与解题策略

<difficulty_intro>
模拟题的“坑”往往藏在“操作顺序”和“细节处理”里。结合优质题解，我帮你提炼了3个核心难点和解决技巧：
</difficulty_intro>

1.  **关键点1：必须严格遵守“每月操作顺序”**  
    * **分析**：题目要求的顺序是“月初加300 → 减预算 → 存钱”。如果颠倒顺序（比如先存钱再加300），结果肯定错！比如样例1中11月，加300后是383，减180得203，再存200——如果先存钱再加300，就会变成先存83里的0，再加300得383，减180得203，结果一样？不对，比如某个月手上有50元，加300得350，减200得150，存100留50；如果先存钱（50不够存），加300得350，减200得150，存100——结果一样？哦，那为什么顺序重要？因为**“减预算后才知道有没有钱存”**！比如如果先存钱再加300，那“存的是上个月剩的钱”，而题目要求“本月加300、减预算后再存”——顺序错了，逻辑就错了！  
    * 💡 **学习笔记**：模拟题的核心是“还原题意顺序”，千万不要“自作主张”调整步骤！

2.  **关键点2：如何判断“钱不够”？**  
    * **分析**：钱不够的情况是“加300后减去预算，结果<0”。比如某个月预算是400，加300后是300，减400得-100——这时候要立刻输出“-当前月份”并终止程序。优质题解都用了“if(钱<0)”的判断，并且在判断后直接`return 0`或`break`，避免继续计算。  
    * 💡 **学习笔记**：模拟题中“终止条件”要早判断，避免不必要的计算！

3.  **关键点3：如何正确计算“整百的存款”？**  
    * **分析**：题目要求“把整百的钱存起来”——比如203元要存200，183元存100，50元不存。计算方法有两种：① 循环减100（像题解一）；② 用整数除法（`money/100*100`，像题解二、三）。两种方法都对，但整数除法更高效（比如203/100=2，乘100得200）。  
    * 💡 **学习笔记**：整数除法是模拟题中“取整”的神器，要学会用！

### ✨ 解题技巧总结
-   **技巧A：变量名要“见名知意”**：比如用`hold`表示手上的钱，`save`表示存的钱，比`a`、`b`好懂100倍！
-   **技巧B：早判断终止条件**：一旦出现钱不够，立刻输出结果并退出，避免后面的错误计算。
-   **技巧C：用整数运算代替循环**：比如`money/100*100`比`while(money>=100)`更高效，尤其当钱很多时！


## 4. C++核心代码实现赏析

<code_intro_overall>
先看一份**综合优质题解的通用核心代码**，帮你建立整体框架：
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：本代码综合了题解一、二、三的思路，用最清晰的变量名和步骤实现，适合新手入门。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    using namespace std;

    int main() {
        int hold = 0;    // 津津手上的钱
        int save = 0;    // 存给妈妈的钱
        bool ok = true;  // 是否没出现钱不够的情况

        for (int month = 1; month <= 12; month++) {
            int cost;
            cin >> cost;
            hold += 300;          // 月初加300元
            hold -= cost;         // 减去本月预算
            if (hold < 0) {       // 钱不够用
                cout << "-" << month << endl;
                ok = false;
                break;
            }
            // 存整百的钱
            int deposit = (hold / 100) * 100;
            save += deposit;
            hold -= deposit;
        }

        if (ok) {
            cout << hold + save * 1.2 << endl;
        }
        return 0;
    }
    ```
* **代码解读概要**：
    > 代码用`hold`和`save`分别记录手上和存的钱，用`ok`标记是否顺利到年底。循环12次处理每个月：① 读入本月预算；② 加300元；③ 减预算；④ 判断钱够不够；⑤ 存整百的钱。最后如果`ok`，计算“手上的钱 + 存的钱×1.2”输出。


<code_intro_selected>
接下来看优质题解的**核心片段**，学习它们的亮点：
</code_intro_selected>

**题解一：(来源：wurang)**
* **亮点**：用`while`循环直观模拟“存整百”的过程，像“一张一张给妈妈100元”，新手秒懂！
* **核心代码片段**：
    ```cpp
    while(n>=100) {
        n-=100;
        s+=100;
    }
    ```
* **代码解读**：
    > 当`n`（手上的钱）≥100时，每次减100给`n`（自己留的钱），加100给`s`（存的钱）。比如`n=203`，第一次循环后`n=103`、`s=100`；第二次循环后`n=3`、`s=200`——刚好存200元，和题目例子一致！
* 💡 **学习笔记**：循环是最直观的“模拟动作”方式，适合理解不深时用！

**题解二：(来源：sea_bird)**
* **亮点**：用`money/100>0`判断是否能存钱，再用`(money/100)*100`计算存的金额，高效又简洁！
* **核心代码片段**：
    ```cpp
    if(money/100>0) {
        mom+=(money/100)*100;
        money-=(money/100)*100;
    }
    ```
* **代码解读**：
    > `money/100`是整数除法，比如`money=183`时，`183/100=1`，所以存`1×100=100`元；`money=203`时，`203/100=2`，存200元。这种方法比循环少写很多代码，而且运行更快！
* 💡 **学习笔记**：整数除法是模拟题的“偷懒神器”，要学会用！

**题解三：(来源：yedalong)**
* **亮点**：用一行代码合并“加300、减预算、存整百”的计算，体现对逻辑的深刻理解！
* **核心代码片段**：
    ```cpp
    sum+=(300-s+ans)/100*100;
    ans=(300-s+ans)%100;
    ```
* **代码解读**：
    > `300-s+ans`是“本月加300、减预算后的总钱”（`s`是本月预算，`ans`是上月剩的钱）。`(...)`/100*100是存的整百金额，加到`sum`（存的钱）；`(...)%100`是剩下的钱，存在`ans`里。这行代码把多个步骤合并，超紧凑！
* 💡 **学习笔记**：当你完全理解逻辑后，可以尝试合并步骤，让代码更简洁！

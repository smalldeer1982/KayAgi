# 题目信息

# HXY玩卡片

## 题目描述

HXY 得到了一些卡片，这些卡片上标有数字 $0$ 或 $5$。

现在她可以选择其中一些卡片排成一列，使得排出的一列数字组成的数最大，且满足被 $90$ 整除这个条件。

同时，这个数必须为合法的某个非负整数，即不能含有前导 $0$，即 $0$ 不能作为这串数的首位。

但是特殊地，整数 $0$ 满足条件。

如果不能排出这样的数，输出 $-1$。

## 说明/提示

数据范围：

对于 $30\%$ 的数据，$n\le 10$。

对于 $20\%$ 的数据，给定的数字均为 $5$。

对于 $100\%$ 的数据，$n\le 1000$。

## 样例 #1

### 输入

```
4
5 0 5 0```

### 输出

```
0
```

## 样例 #2

### 输入

```
11
5 5 5 5 5 5 5 5 0 5 5```

### 输出

```
5555555550```

## 样例 #3

### 输入

```
2
5 5```

### 输出

```
-1
```

🗣️ **初步分析**：
> 解决这道题的关键，是先听懂「被90整除」的“数学暗号”——90=9×10，所以组成的数必须同时满足两个条件：  
> 1. **被10整除**：末位是0（所以必须有0卡片）；  
> 2. **被9整除**：所有数字之和是9的倍数（因为卡片只有0和5，0不影响和，所以5的个数必须是9的倍数，比如9个、18个…）。  
> 就像“做蛋糕”：0是“底座”（必须有），5是“奶油”（要凑够9的倍数才好吃），而且要把奶油堆在底座上面（5在前、0在后）才能最大！  

   - **题解思路共性**：所有优质题解都围绕“统计5和0的个数→判断条件→输出最大数”展开，核心是**用数学规律简化问题**（不用真的排列所有可能）。  
   - **核心难点**：① 想通“被90整除”的两个条件；② 处理特殊情况（比如只有0、5不够9个）；③ 确保数最大（5尽可能多且在前）。

## 3. 核心难点辨析与解题策略

<difficulty_intro>
这道题的“拦路虎”其实是**数学规律的转化**和**特殊情况的处理**，我们一个个拆解决！
</difficulty_intro>

1.  **关键点1：如何理解“被90整除”？**
    * **分析**：90=9×10，所以要同时满足“被10整除”（有0）和“被9整除”（5的个数是9的倍数）。就像“要进游乐场，必须有门票（0）+够身高（5的个数≥9）”，缺一个都不行～  
    * 💡 **学习笔记**：遇到“被XX整除”的问题，先拆成质因数相乘（比如90=2×3²×5），再对应数字的性质！

2.  **关键点2：特殊情况怎么处理？**
    * **分析**：① 没有0→直接输出-1（没有门票进不去）；② 有0但5<9→只能输出0（身高不够，只能玩“迷你项目”）；③ 有0且5≥9→输出最多的9倍数个5+所有0（身高够了，尽情玩！）。  
    * 💡 **学习笔记**：特殊情况要“优先判断”，比如先看有没有0，再看5的个数，避免逻辑混乱～

3.  **关键点3：如何让数最大？**
    * **分析**：要最大的数，就得让大的数字在前面（5比0大），所以5要尽可能多且排在前面，0全部跟在后面。比如18个5+3个0，就是“555555555555555555000”，比任何其他排列都大！  
    * 💡 **学习笔记**：“贪心原则”——要最大数，大数字放前面，多的数字放前面～

- **技巧B：优先处理特殊情况**：先判断“没有0”“5不够9个”这些简单情况，再处理一般情况，代码更清晰～

## 4. C++核心代码实现赏析

<code_intro_overall>
先看一份综合了所有优质题解思路的**通用核心代码**，帮你快速把握整体框架～
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：本代码综合了“绝顶我为峰”“Dilute”“wpy233”的思路，逻辑简洁、边界处理严谨，是最典型的实现方式～
* **完整核心代码**：
    ```cpp
    #include <iostream>
    using namespace std;

    int main() {
        int n, t5 = 0, t0 = 0;
        cin >> n;
        for (int i = 1; i <= n; ++i) {
            int a;
            cin >> a;
            if (a == 5) t5++;
            else t0++;
        }
        if (t0 == 0) { // 没有0，无法被10整除
            cout << -1;
            return 0;
        }
        if (t5 < 9) { // 5不够9个，只能输出0
            cout << 0;
            return 0;
        }
        // 输出最多的9倍数个5
        int max_five = t5 / 9 * 9;
        for (int i = 1; i <= max_five; ++i) cout << 5;
        // 输出所有0
        for (int i = 1; i <= t0; ++i) cout << 0;
        return 0;
    }
    ```
* **代码解读概要**：  
  1. 读入卡片数量`n`，统计5的个数`t5`和0的个数`t0`；  
  2. 特殊情况判断：没有0→输出-1，5不够9个→输出0；  
  3. 计算最多能输出的5的数量（`t5 / 9 * 9`，比如11个5→9个）；  
  4. 先输出所有5，再输出所有0，得到最大数～

<code_intro_selected>
接下来剖析3份优质题解的核心片段，看看它们的“小妙处”～
</code_intro_selected>

**题解一：(来源：绝顶我为峰)**
* **亮点**：用`t5/9*9`直接计算最多的5的数量，逻辑直白！
* **核心代码片段**：
    ```cpp
    if(t5>=9)// 有0且5≥9
    {
        for(int i=1;i<=t5/9*9;i++) cout<<5;
        for(int i=1;i<=t0;i++) cout<<0;
    }
    ```
* **代码解读**：  
  为什么用`t5/9*9`？比如t5=11，11/9=1（整数除法向下取整），1×9=9，刚好是不超过11的最大9的倍数！这行代码把“找最大9倍数”的逻辑浓缩成一句话，超高效～  
* 💡 **学习笔记**：整数除法的“向下取整”是处理“最大不超过X的Y倍数”的神器！

**题解二：(来源：Dilute)**
* **亮点**：专门处理“一个5都不用输出”的情况，边界更严谨！
* **核心代码片段**：
    ```cpp
    int m = five / 9 * 9;
    for(int i=1; i<=m; i++) printf("5");
    if(m == 0){ // 没有5可以输出，只输出0
        printf("0");
        return 0;
    }
    ```
* **代码解读**：  
  为什么要判断`m==0`？比如t5=5（不够9个），但前面已经判断过t0≠0，所以直接输出0！这行代码补上了“5不够9个”的情况，让逻辑更完整～  
* 💡 **学习笔记**：边界条件要“想全”，比如“没有5但有0”的情况！

**题解三：(来源：wpy233)**
* **亮点**：用`for(int i=1;i<=s-s%9;i++)`计算最多的5的数量，另一种思路！
* **核心代码片段**：
    ```cpp
    for(int i=1;i<=s-s%9;i++)// 输出5
        cout<<5;
    ```
* **代码解读**：  
  `s%9`是5的个数除以9的余数（比如s=11，余数是2），`s-s%9`就是减去余数得到最大的9倍数（11-2=9）！这和`t5/9*9`是一样的效果，只是换了种写法～  
* 💡 **学习笔记**：解决问题的方法可以有多种，选自己最容易理解的就行！

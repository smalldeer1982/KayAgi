## 题目描述

小明决定从下周一开始努力刷题准备蓝桥杯竞赛。他计划周一至周五每天做 $a$ 道题目，周六和周日每天做 $b$ 道题目。请你帮小明计算，按照计划他将在第几天实现做题数大于等于 $n$ 题?

## 说明/提示

对于 $50 \%$ 的评测用例，$1 \leq a, b, n \leq 10^{6}$.

对于 $100 \%$ 的评测用例，$1 \leq a, b, n \leq 10^{18}$. 

蓝桥杯 2022 省赛 B 组 C 题。

## 3. 核心难点辨析与解题策略

<difficulty_intro>
这道题的“坑”都藏在“大数”和“细节”里，我帮你提炼了3个核心难点和解决技巧：
</difficulty_intro>

1. **难点1：如何处理1e18的大数不超时？**
    * **分析**：暴力法（每天减题数）的时间复杂度是O(n)，1e18次操作会让程序“卡到天荒地老”。解决方案是**周期优化**——每周做的题数固定（5a+2b），用除法算整周数（`n/(5a+2b)`），把1e18次操作变成“最多7次剩余天数模拟”！
    * 💡 **学习笔记**：遇到“重复周期”的问题，先算整周期，再处理剩余，是解决大数问题的“万能钥匙”！

2. **难点2：如何计算剩余题数需要的天数？**
    * **分析**：剩余题数可能在“工作日”或“周末”完成。比如剩余99题，a=10，b=20——整周做50+40=90题，剩余9题，工作日1天就能完成。解决方案是**分类讨论**：如果剩余≤5a，用向上取整算工作日天数；否则，先加5天工作日，再算周末需要的天数。
    * 💡 **学习笔记**：分类讨论时，用“数学公式”代替“循环”，能让代码更高效！

3. **难点3：如何避免整数溢出？**
    * **分析**：a、b、n都是1e18，5a会变成5e18，超过int的范围（int最多约2e9）。解决方案是**用long long类型**（能存到9e18），所有变量都要声明为long long！
    * 💡 **学习笔记**：看到“1e18”这种大数，第一反应就是“开long long”，否则会“爆掉”！

## 4. C++核心代码实现赏析

<code_intro_overall>
先看一个**通用核心实现**——综合了“周期计算+数学公式处理剩余”，是这道题的“最优解”！
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：本代码来自题解一的优化版，用数学公式代替循环，效率最高！
* **完整核心代码**：
    ```cpp
    #include <iostream>
    using namespace std;

    int main() {
        long long a, b, n;
        cin >> a >> b >> n;
        long long week = 5 * a + 2 * b; // 每周做的题数
        long long full_weeks = n / week; // 完整的周数
        long long remaining = n % week;  // 剩余题数
        long long days = full_weeks * 7; // 整周的天数

        if (remaining == 0) {
            cout << days << endl;
            return 0;
        }
        // 处理剩余题数
        if (remaining <= 5 * a) {
            days += (remaining + a - 1) / a; // 向上取整算工作日天数
        } else {
            days += 5; // 先加5个工作日
            remaining -= 5 * a;
            days += (remaining + b - 1) / b; // 算周末天数
        }
        cout << days << endl;
        return 0;
    }
    ```
* **代码解读概要**：
    > 代码分3步：① 算每周做的题数`week`；② 算整周数`full_weeks`和剩余题数`remaining`；③ 处理剩余题数——如果剩余≤5a（工作日能做完），用向上取整算天数；否则，先加5天工作日，再算周末需要的天数。全程没有循环，效率极高！

* **代码解读**：
    > 这段代码的“灵魂”是`(s1 + a - 1)/a`——比如s1=9，a=10，(9+10-1)/10=18/10=1（整数除法向下取整），刚好是需要的天数！这种“向上取整”的公式，比循环判断快得多～
* 💡 **学习笔记**：向上取整的公式是`(x + y - 1) / y`，记住它，很多题都能用！

**题解二：作者_Cppsteve_（优化版）**
* **亮点**：先算整周，再模拟剩余天数，适合新手理解！
* **核心代码片段**：
    ```cpp
    long long weeks = n / (a*5 + b*2);
    n -= weeks * (a*5 + b*2); // 剩余题数
    while (n > 0) {
        if (today < 5) n -= a;
        else n -= b;
        today++;
        today %= 7;
        days++;
    }
    cout << weeks*7 + days << endl;
    ```
* **代码解读**：
    > 先算整周数`weeks`，再用`while`循环模拟剩余天数——`today`记录当前是星期几（0-6），`today<5`是工作日，否则是周末。循环最多跑7次，完全不超时～
* 💡 **学习笔记**：模拟剩余天数时，用`today%7`保持星期数在0-6，避免“星期八”的bug！

**题解三：作者olegekei**
* **亮点**：剩余天数用for循环模拟，逻辑超直观！
* **核心代码片段**：
    ```cpp
    long long c = 5*a + 2*b;
    long long x = n / c;
    long long ans = 7*x;
    n %= c;
    for (int i=1; n>0; i++, ans++) {
        if (i <=5) n -=a;
        else n -=b;
    }
    ```
* **代码解读**：
    > 用`for`循环模拟“周一到周日”——i从1到7，i≤5是工作日，否则是周末。每次循环`ans`加1（天数增加），直到剩余题数≤0。循环最多跑7次，效率很高～
* 💡 **学习笔记**：模拟剩余天数时，用`for`循环比`while`更直观，适合新手！

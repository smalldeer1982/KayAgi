## 题目描述

dXqwq 定义一个字符串四元组 $(a,b,c,d)$ 是“浴眼盯真”的，当且仅当：

- $a,b$ 的首字母均为 $\texttt{y}$。
- $c$ 恰好等于 $\texttt{ding}$。
- $d$ 恰好等于 $\texttt{zhen}$。

给定四个用空格分隔的字符串 $a,b,c,d$，保证其只包含小写英文字母，你需要判断 $(a,b,c,d)$ 是否是“浴眼盯真”的。

- Subtask 2（50 pts）：无特殊限制。 

对于 $100\%$ 的数据，$1\leq T\leq 100$，每个字符串长度不超过 $10$。

## 3. 核心难点辨析与解题策略

在解决这道题时，新手常踩的“坑”主要有3个：

### 1. 字符串首字符的访问
**难点**：容易误以为字符串的首字符是`str[1]`（比如`a[1]`），但C++的字符串是**零起始索引**（和数组一样），首字符是`str[0]`。  
**解决**：记住“字符串的第一个字符是`[0]`，第二个是`[1]`……”，比如`a`的首字符是`a[0]`。  
💡 **学习笔记**：字符串下标从0开始！

### 2. 字符串的完全比较
**难点**：容易用`c[0] == 'd'`代替`c == "ding"`（只检查第一个字符），但题目要求`c`**完全等于**`"ding"`。  
**解决**：直接用`==`运算符——C++的`string`类重载了`==`，会自动比较两个字符串的所有字符，完全一致才返回`true`。  
💡 **学习笔记**：`string`的`==`是“全字符比较”！

### 3. 多组数据的循环处理
**难点**：容易忘记循环T次（比如读了T但没写循环），或者循环条件写错（比如`for(int i=1; i<=T; i++)`写成`i<T`）。  
**解决**：用`while(T--)`或者`for(int i=0; i<T; i++)`——`while(T--)`更简洁（T从输入值开始，每次减1，直到0）。  
💡 **学习笔记**：多组数据用循环，循环次数等于T！

### ✨ 解题技巧总结
- **按题目顺序写条件**：逻辑更清晰，不容易漏条件；  
- **用`string`的内置操作**：`[ ]`访问字符、`==`比较字符串，比手动处理字符数组更方便；  
- **简化循环**：`while(T--)`比`for`循环更简洁，适合多组数据。


## 4. C++核心代码实现赏析

### 本题通用核心C++实现参考
* **说明**：综合了多个优质题解的简洁版本，保留核心逻辑，去掉冗余代码。  
* **完整核心代码**：
  ```cpp
  #include <iostream>
  #include <string>
  using namespace std;

  int main() {
      int T;
      cin >> T;
      while (T--) {
          string a, b, c, d;
          cin >> a >> b >> c >> d;
          if (a[0] == 'y' && b[0] == 'y' && c == "ding" && d == "zhen")
              cout << "Yes\n";
          else
              cout << "No\n";
      }
      return 0;
  }
  ```
* **代码解读概要**：  
  1. 读入数据组数`T`；  
  2. 循环`T`次，每次读入四个字符串`a,b,c,d`；  
  3. 逐一检查题目条件，输出结果。  
  核心是**条件判断语句**，直接对应题目要求，没有多余的代码。

### 题解一（作者：wmrqwq）核心片段赏析
* **亮点**：条件判断的简化技巧。  
* **核心代码片段**：
  ```cpp
  if(a[0]==b[0] && a[0]=='y' && c=="ding" && d=="zhen")
      cout<<"Yes"<<endl;
  else
      cout<<"No"<<endl;
  ```
* **代码解读**：  
  用`a[0]==b[0] && a[0]=='y'`代替`a[0]=='y' && b[0]=='y'`——如果`a`的首字符是`'y'`，且`b`的首字符等于`a`的，那`b`的首字符也一定是`'y'`。这种简化不影响逻辑，但让代码更短。  
* 💡 **学习笔记**：可以用等价条件简化代码，但要确保逻辑正确！

### 题解二（作者：dead_X）核心片段赏析
* **亮点**：快速读入函数。  
* **核心代码片段**：
  ```cpp
  inline int read(){
      int s=0,w=1;
      char ch=getchar();
      while(ch<'0'||ch>'9'){if(ch=='-')w=-1;ch=getchar();}
      while(ch>='0'&&ch<='9') s=s*10+ch-'0',ch=getchar();
      return s*w;
  }
  ```
* **代码解读**：  
  这个函数用`getchar()`逐字符读入数字，比`cin`快很多（因为`cin`默认有缓冲）。比如读`T`的时候，用`read()`代替`cin>>T`，能加快输入速度——在大数据量的题目中，这可能直接决定程序是否超时。  
* 💡 **学习笔记**：快速读入是竞赛必备技巧！

### 题解三（作者：shiyihang）核心片段赏析
* **亮点**：正则表达式的应用。  
* **核心代码片段**：
  ```cpp
  regex rab("^y.*");   // 匹配以y开头的字符串
  regex rc("^ding$");  // 匹配完全等于ding的字符串
  regex rd("^zhen$");  // 匹配完全等于zhen的字符串
  if (regex_match(a, rab) && regex_match(b, rab) && regex_match(c, rc) && regex_match(d, rd))
      cout << "Yes\n";
  else
      cout << "No\n";
  ```
* **代码解读**：  
  `regex_match`函数判断字符串是否完全匹配正则表达式：  
  - `^y.*`：`^`表示行首（字符串开头），`y`是首字符，`.*`表示后面可以跟任意字符；  
  - `^ding$`：`^`是开头，`$`是结尾，所以整个字符串必须是`ding`。  
  虽然本题用正则表达式有点“小题大做”，但它适合处理更复杂的字符串匹配（比如验证邮箱格式、电话号码）。  
* 💡 **学习笔记**：正则表达式是字符串处理的“瑞士军刀”！

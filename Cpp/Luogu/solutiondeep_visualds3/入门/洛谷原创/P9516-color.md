# 题目信息

# color

## 题目描述

洛谷的名字颜色和咕值直接挂钩，具体细则见 [洛谷咕值系统介绍](https://www.luogu.com.cn/discuss/83952)。

给出某洛谷用户的五项咕值指标的得分，输出该用户的名字颜色。保证该用户不是棕名和紫名。

## 说明/提示

**【样例说明】**

三个样例的咕值分别为 $263,140,0$，名字颜色分别为红、绿、灰。

**【数据范围】**

对于 $20\%$ 的数据，满足五项指标分数均 $\le 10$；

对于另外 $20\%$ 的数据，满足五项指标分数均 $\ge 90$；

对于 $100\%$ 的数据，保证五项指标分数均 $\in [0, 100]$。

## 样例 #1

### 输入

```
100 14 9 55 85```

### 输出

```
Red```

## 样例 #2

### 输入

```
100 10 10 10 10```

### 输出

```
Green```

## 样例 #3

### 输入

```
0 0 0 0 0```

### 输出

```
Gray```

## 3. 核心难点辨析与解题策略

### 核心难点1：记准区间的“边界线”  
**问题**：题目里的颜色区间有5个（比如Gray是0~99，Blue是100~119），很多同学会记混“100属于蓝名还是灰名”“230属于橙名还是红名”。  
**解决策略**：把规则写成“表格清单”，贴在电脑旁边：  
| 颜色   | 区间范围       |
|--------|----------------|
| Gray   | 0 ≤ sum ≤ 99   |
| Blue   | 100 ≤ sum ≤ 119|
| Green  | 120 ≤ sum ≤ 169|
| Orange | 170 ≤ sum ≤ 229|
| Red    | 230 ≤ sum ≤ 500|  
**学习笔记**：边界值是“条件判断的开关”，记错了就会“满盘皆输”！

### 核心难点2：条件判断的“顺序陷阱”  
**问题**：如果条件判断顺序错了（比如先判Red再判Gray），会导致结果错误吗？比如sum=50，先写`if(sum>=230)`会跳过，但`else if(sum<=99)`还是会正确判断——但从高到低或从低到高都可以，**关键是要覆盖所有区间**。  
**解决策略**：选一种“逻辑连贯”的顺序，比如从低到高（像Yamchip的题解）：先判最小的区间（Gray），再判更大的（Blue→Green→Orange→Red），确保每个sum只属于一个区间。  
**学习笔记**：条件判断的顺序要“顺藤摸瓜”，不能跳着来！

### 核心难点3：变量名的“可读性”  
**问题**：如果变量名叫`a1`“b2”，别人看代码会一头雾水——比如Hughpig用`qwq`表示总和，虽然可爱，但不如`sum`直观。  
**解决策略**：变量名要“见名知意”——用`sum`表示总和，`score1`-`score5`表示五个咕值（本题用a、b、c、d、e也可以，因为题目明确是“五项”）。  
**学习笔记**：好的变量名是“代码的说明书”，不用注释也能懂！

### ✨ 解题技巧总结  
- **技巧A**：把规则写成表格，避免记混边界；  
- **技巧B**：条件判断选“从低到高”或“从高到低”，确保覆盖所有情况；  
- **技巧C**：变量名要“直观”，比如`sum`比`qwq`好懂100倍！


## 4. C++核心代码实现赏析

### 本题通用核心C++实现参考  
* **说明**：本代码来自Yamchip的题解，是本题的“标准解法”——逻辑简洁、覆盖所有情况，新手直接抄都不会错！  
* **完整核心代码**：  
```cpp
#include <iostream>
using namespace std;

int main() {
    int a, b, c, d, e, sum;
    cin >> a >> b >> c >> d >> e;
    sum = a + b + c + d + e;
    if (sum <= 99)      cout << "Gray";
    else if (sum <= 119) cout << "Blue";
    else if (sum <= 169) cout << "Green";
    else if (sum <= 229) cout << "Orange";
    else                cout << "Red";
    return 0;
}
```
* **代码解读概要**：  
代码分三步：①用`cin`输入五个咕值；②计算总和`sum`；③用`if-else if`按区间判断`sum`，输出对应的颜色。整个流程像“流水线”一样顺畅，没有多余的步骤。


### 题解一：来源：Yamchip（核心片段赏析）  
* **亮点**：条件判断“逻辑连贯”，从低到高覆盖所有区间。  
* **核心代码片段**：  
```cpp
sum = a + b + c + d + e;
if (sum <= 99)      cout << "Gray";
else if (sum <= 119) cout << "Blue";
else if (sum <= 169) cout << "Green";
else if (sum <= 229) cout << "Orange";
else                cout << "Red";
```
* **代码解读**：  
这段代码是“灵魂”！比如sum=263（样例1）：  
- 先判断`sum<=99`？否；  
- 再判断`sum<=119`？否；  
- 继续`sum<=169`？否；  
- 再`sum<=229`？否；  
- 最后进入`else`，输出Red——完全符合样例结果！  
* 💡 **学习笔记**：从低到高的条件判断，是处理区间问题的“黄金法则”！


### 题解二：来源：Forever_LYD（核心片段赏析）  
* **亮点**：边界条件“写得明明白白”，适合新手理解。  
* **核心代码片段**：  
```cpp
if(ans<100)cout<<"Gray";
else if(ans>=100&&ans<120)cout<<"Blue";
else if(ans>=120&&ans<170)cout<<"Green";
else if(ans>=170&&ans<=230)cout<<"Orange";
else cout<<"Red";
```
* **代码解读**：  
比如`ans>=100&&ans<120`——直接告诉你“蓝名的范围是100到119”，像给新手“画了条线”。这种写法虽然长一点，但胜在“直观”，能避免边界错误（比如不会把119归到Green）。  
* 💡 **学习笔记**：明确写出边界条件，是新手“避坑”的好方法！

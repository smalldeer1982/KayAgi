# Judg.

## 题目描述

小 M 正在兴致勃勃地敲打着代码，他突然发现提交代码的评测机无响应了。

你查看了后台数据，请你根据这些已有数据判断他没有通过的测试点有哪些。

已知评测结果与测试点是否通过有如下关系：

|评测结果|测试点是否通过|
| :-----------: | :-----------: |
| $\texttt{AC}$ | 是 |
| $\texttt{WA}$ | 否 |
| $\texttt{TLE}$ | 否 |
| $\texttt{MLE}$ | 否 |


## 说明/提示

#### 【数据范围】

对于所有测试数据，满足 $1 \le n \le 10^5$，输入的字符串一定在 $\{\texttt{AC},\texttt{WA},\texttt{TLE},\texttt{MLE}\}$ 之中。

**本题开启捆绑测试，所有数据范围均相同的测试点捆绑为一个 $\text{Subtask}$。**

各测试点的附加限制如下表所示。

|测试点 |$n \le$  |
| :-----------: | :-----------: |
| $1 \sim 5$ | $10$ |
| $6 \sim 12$ | $2 \times 10^3$ |
| $13 \sim 20$ | $10^5$ |

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：编程技巧应用（模拟）

🗣️ **初步分析**：
解决这道题的关键是**模拟**——就像玩“按规则分类”的游戏：给你一堆卡片（评测结果），你要按照规则（不是AC的卡片要挑出来）把对应的编号记下来。  
简单来说，模拟的核心就是“**照题目说的做**”：读入n个评测结果，逐个判断是不是“AC”；如果不是，就把当前测试点的编号（从1开始）记下来，最后输出所有记下来的编号。  

**核心流程**：  
1. 读入测试点总数n；  
2. 循环n次：读入当前评测结果→判断是否为“AC”→不是的话记录编号；  
3. 输出所有记录的编号。  

**可视化设计思路**：  
我打算用**8位像素风**模拟一个“迷你评测机”：  
- 屏幕左侧是“测试点列表”（每个测试点是16x16的像素块，AC是绿色，WA/TLE/MLE是红色）；  
- 中间是“判断流程”（一个像素化的“指针”逐个指向测试点，判断时闪烁并伴随“叮”的音效）；  
- 右侧是“结果输出区”（每找到一个非AC结果，就弹出对应的编号像素块，伴随“啪”的音效）。  
控制面板会有“开始/单步/重置”按钮、速度滑块，还有8位风格的轻快背景音乐~

## 3. 核心难点辨析与解题策略

<difficulty_intro>
这道题看似简单，但新手容易踩3个“小坑”。让我们一起避开它们！
</difficulty_intro>

1.  **难点1：测试点编号从1开始**  
    * **分析**：题目中测试点编号是“第1个、第2个……第n个”，所以循环变量要从1开始（`i=1`），而不是0！如果写成`i=0`，输出的编号会比实际小1。  
    * **解决**：循环条件明确写成`for(int i=1; i<=n; ++i)`。

2.  **难点2：大规模数据的输入效率**  
    * **分析**：当n=1e5时，用普通的cin会很慢（因为cin默认和stdio同步），可能导致超时。  
    * **解决**：加入输入加速代码：`ios::sync_with_stdio(false); cin.tie(nullptr);`（关闭同步+解绑cin和cout），或者直接用scanf/printf。

3.  **难点3：输出格式的空格问题**  
    * **分析**：题目要求输出的编号之间用空格分隔，比如样例1输出“2 4 5”。如果用“边读边输出”，最后会多一个空格——但题目不扣分！（评测系统会忽略末尾的空格）  
    * **解决**：不用特意处理末尾空格，直接输出即可~


### ✨ 解题技巧总结
- **边读边处理**：能不用数组就不用，减少内存占用，提高效率；  
- **输入加速**：大规模数据必加`ios::sync_with_stdio(false); cin.tie(nullptr);`；  
- **小技巧谨慎用**：比如判断首字母，要确保题目中没有例外情况（本题中安全，但其他题可能有坑）；  
- **变量名要明确**：比如`cnt`（计数）、`r`（结果）比`a`、`b`更容易理解~


## 4. C++核心代码实现赏析

<code_intro_overall>
先看一个**通用核心实现**，它结合了“边读边输出”和“输入加速”，是本题的最优写法之一~
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：本代码综合了“边读边输出”的高效性和“输入加速”的优化，适合所有数据范围。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    #include <string>
    using namespace std;

    int main() {
        ios::sync_with_stdio(false);
        cin.tie(nullptr);
        int n;
        cin >> n;
        for (int i = 1; i <= n; ++i) {
            string s;
            cin >> s;
            if (s != "AC") {
                cout << i << " ";
            }
        }
        return 0;
    }
    ```
* **代码解读概要**：
    > 1. 先关闭cin和stdio的同步（`ios::sync_with_stdio(false)`），解绑cin和cout（`cin.tie(nullptr)`），加快输入速度；  
    > 2. 读入测试点总数n；  
    > 3. 循环n次：读入当前评测结果s→判断s是不是“AC”→不是的话输出i（测试点编号）和空格；  
    > 4. 结束程序。


<code_intro_selected>
接下来，我们剖析几个优质题解的“核心片段”，看看它们的亮点~
</code_intro_selected>

**题解一：(来源：nightwatch.ryan)**
* **亮点**：用数组存结果，适合需要“先收集再处理”的场景（比如本题如果要求按逆序输出，这个方法就很方便）。
* **核心代码片段**：
    ```cpp
    int n, r[200000];
    string s;
    int cnt;
    int main() {
        cin >> n;
        for (int i = 1; i <= n; i++) {
            cin >> s;
            if (s != "AC") {
                cnt++;
                r[cnt] = i;
            }
        }
        for (int i = 1; i <= cnt; i++) {
            cout << r[i] << " ";
        }
    }
    ```
* **代码解读**：
    > - `r[200000]`是存不通过编号的数组（大小设为2e5，足够装下所有情况）；  
    > - `cnt`是计数器，记录有多少个不通过的测试点；  
    > - 第一个循环：读入s→判断不是AC→cnt加1→把i存到r[cnt]；  
    > - 第二个循环：输出所有存好的编号。  
* 💡 **学习笔记**：数组适合“需要保存中间结果”的场景，但本题不需要，所以边读边输出更高效。


**题解二：(来源：ccxswl)**
* **亮点**：用“首字母判断”减少字符串比较的次数（只比较第一个字符，不用比较整个字符串）。
* **核心代码片段**：
    ```cpp
    for (int i = 1; i <= n; i++) {
        cin >> s;
        if (s[0] != 'A')
            cout << i << ' ';
    }
    ```
* **代码解读**：
    > - `s[0]`是字符串s的第一个字符（C++中字符串的索引从0开始）；  
    > - 因为WA/TLE/MLE的首字母都不是'A'，所以`s[0] != 'A'`等价于“不是AC”；  
    > - 这种方法比`s != "AC"`快一丢丢（少比较一个字符）。  
* 💡 **学习笔记**：小技巧要基于题目规则——如果有其他首字母为'A'的结果（比如“AE”），这个方法就错了！


**题解三：(来源：WBH_xh_WSD)**
* **亮点**：输入加速代码让大规模数据的读取速度飞起来！
* **核心代码片段**：
    ```cpp
    ios::sync_with_stdio(0);
    cin.tie(0);
    cout.tie(0);
    ```
* **代码解读**：
    > - `ios::sync_with_stdio(0)`：关闭cin和C语言stdio的同步（默认是同步的，所以cin很慢）；  
    > - `cin.tie(0)`：解绑cin和cout（默认cin会等待cout输出完再读入，解绑后能加快速度）；  
    > - `cout.tie(0)`：解绑cout和其他输出流（进一步加速）。  
* 💡 **学习笔记**：这三行代码是竞赛中的“必备加速神器”，遇到大n（比如1e5）一定要加！

<insights_intro>

## 3. 核心难点辨析与解题策略  
1. **难点：双条件排序逻辑**  
   * **分析**：需理解条件优先级——成绩为主排序键，名字为次排序键。当主键相等时才比较次键  
   * 💡 **学习笔记**：多条件排序像奥运颁奖：先按金牌数（主条件），金牌相同按银牌数（次条件）  

2. **难点：结构体数据组织**  
   * **分析**：用结构体将关联数据（姓名+成绩）绑定，避免维护多个独立数组  
   * 💡 **学习笔记**：结构体像"数据书包"，把属于同一实体的信息打包  

3. **难点：排序稳定性理解**  
   * **分析**：冒泡排序是稳定排序，保证相同分数学生原始顺序不被打乱  
   * 💡 **学习笔记**：稳定排序=整理扑克牌时，同花色的牌保持原有顺序  

**✨ 解题技巧总结**  
- **巧用逻辑运算符**：用`||`和`&&`优雅实现多级排序条件  
- **结构体封装**：相关数据打包处理，提升代码可读性  
- **测试边界值**：需测试同名同分、单元素输入等特殊情况

## 4. C++核心代码实现赏析  
**本题通用核心实现**  
* **说明**：基于题解思路优化，使用更高效的`sort`函数  
* **完整代码**：  
  ```cpp
  #include <iostream>
  #include <algorithm>
  using namespace std;

  struct Stu {
      string nm;
      int sc;
  };

  bool cmp(Stu a, Stu b) {
      if(a.sc != b.sc) return a.sc > b.sc;  // 分数降序
      return a.nm < b.nm;                   // 名字升序
  }

  int main() {
      Stu a[105];
      int n; cin >> n;
      for(int i=0; i<n; i++) 
          cin >> a[i].nm >> a[i].sc;
      
      sort(a, a+n, cmp);  // 调用排序函数
      
      for(int i=0; i<n; i++) 
          cout << a[i].nm << " " << a[i].sc << endl;
      return 0;
  }
  ```  
* **代码解读概要**：  
  > 定义`Stu`结构体→读取数据→用自定义比较函数`cmp`排序→输出结果。`cmp`函数实现核心排序逻辑：优先比较分数，分数相同再比较名字字典序。  

**题解片段赏析（chen_zhe）**  
* **亮点**：手写冒泡排序直观展示比较过程  
* **核心片段**：  
  ```cpp
  if (a[j].sc < a[j+1].sc || 
      (a[j].sc == a[j+1].sc && a[j].nm > a[j+1].nm)) 
  {
      swap(a[j], a[j+1]); // 交换操作
  }
  ```  
* **代码解读**：  
  > `a[j].sc < a[j+1].sc`检测是否需要按分数交换（后>前）。当分数相等时，`a[j].nm > a[j+1].nm`检测名字字典序是否需要交换（后<前）。双条件通过`||`连接形成完整规则。  
* 💡 **学习笔记**：比较函数返回`true`时表示两个元素需要交换位置

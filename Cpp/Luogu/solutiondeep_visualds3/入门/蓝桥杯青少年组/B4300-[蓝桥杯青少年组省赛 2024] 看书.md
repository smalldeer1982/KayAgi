# 题目信息

# [蓝桥杯青少年组省赛 2024] 看书

## 题目描述

一本书共 $n$ 页，小明计划第一天看 $x$ 页，此后每一天都要比前一天多看 $y$ 页。请问小明几天可以看完这本书？


## 样例 #1

### 输入

```
100 10 5```

### 输出

```
5```

#### 1. 题目解读与核心算法识别
✨ **本题主要考察**：`数学应用` (等差数列求和)  
🗣️ **初步分析**：
> 解决"看书"这道题，本质是等差数列求和问题。就像搭积木时逐层增加块数，小明每天阅读量构成等差数列：首项x，公差y。我们需要找到最小天数k，使累计阅读量≥总页数n。

- **核心算法流程**：
  1. 用等差数列求和公式：$S_k = kx + \frac{k(k-1)}{2}y$
  2. 解不等式 $S_k \geq n$ 的最小整数解k

#### 3. 核心难点辨析与解题策略
1. **难点1：数学建模转换**
   - **分析**：将生活场景转化为等差数列模型需抽象思维。优质题解通过标注"首项=首日阅读量，公差=每日增量"建立关联。

2. **难点2：解不等式取整**
   - **分析**：求根公式 $k=\lceil\frac{-x+\sqrt{x^2+2ny}}{y}\rceil$ 需处理非整数解。题解通过`ceil()`函数确保天数取整。
   - 💡 **学习笔记**：向上取整是处理"最后一天未读满"的标准方法。

3. **难点3：溢出风险**
   - **分析**：当y很小时，$k^2$可能超出int范围。题解使用`long long`避免溢出。
   - 💡 **学习笔记**：预估数据范围是竞赛编程的基本功。

#### 4. C++核心代码实现赏析
**通用核心实现（数学公式法）**
```cpp
#include <iostream>
#include <cmath>
using namespace std;

int main() {
    long long n, x, y;
    cin >> n >> x >> y;
    
    // 解二次方程 kx + k(k-1)y/2 >= n
    double k = (sqrt(x*x + 2*y*n) - x) / y;
    cout << (long long)ceil(k);
    return 0;
}
```
**代码解读概要**：通过求根公式直接计算最小整数天数，避免了循环开销。`sqrt()`用于解二次方程，`ceil()`确保天数取整。

**题解二核心片段（模拟法）**
```cpp
long long day = 0, pages = 0;
while (pages < n) {
    day++;
    pages += x + (day-1)*y; // 当日阅读量 = 首日 + (天数-1)*增量
}
cout << day;
```
**代码解读**：  
> 每日阅读量呈线性增长：第1天=x，第2天=x+y，第3天=x+2y...  
> `pages`累计总阅读量，当超过n时跳出循环  
> 💡 **学习笔记**：模拟法虽效率较低，但直观展示等差数列增长过程

# [语言月赛202212]  洛谷三角洲

## 题目描述

在平行的洛谷世界，有 A，B，C 三座城市，它们构成了洛谷三角洲。

两两城市之间各有一条双向道路。也就是说，一共有三条道路，A 市与 B 市之间有一条道路，A 市与 C 市之间有一条道路，B 市与 C 市之间有一条道路。

由于每条道路的拥堵程度不同，因此通过一条路需要的时间也不同。对于每一条路，双向的拥挤程度相同（即经过 A 市与 B 市之间的一条道路，由 A 市到 B 市和由 B 市到 A 市所需的时间相同）。

通过 A 市与 B 市之间的道路需要 $x$ 分钟，通过 B 市与 C 市之间的道路需要 $y$ 分钟，通过 C 市与 A 市之间的道路需要 $z$ 分钟。

![](https://cdn.luogu.com.cn/upload/image_hosting/g6n7j828.png)

由于业务需要，洛谷站长需要在三个市之间频繁往返。如果能够知道任意两个市之间通行的最短时间，那么他能够极大地提高工作效率。

所以他现在想知道：从 A 市到 B 市，从 B 市到 C 市，从 A 市到 C 市分别**至少**需要多少时间（单位：分钟）。但是他认为这个问题太难了，所以他把这个问题交给了聪明的你。


## 说明/提示

### 样例 1 解释

从 A 市到 B 市需要 $1$ 分钟，从 B 市到 C 市需要 $2$ 分钟。  
从 A 市到 C 市可以先从 A 市到 B 市，再从 B 市到 C 市，这样只需要花费 $1+2=3$ 分钟。  

### 数据规模与约定

- 对于 $20\%$ 的数据，满足 $x=y=z$；
- 对于另外 $20\%$ 的数据，满足 $x+y\ge z$，$x+z\ge y$，$y+z\ge x$；
- 对于 $100\%$ 的数据，满足 $1 \leq x,y,z\le 10^9$。

1 2 5```

### 输出

```
1
2
3```

### 输入

3 3 3```

### 输出

# 💡 Kay的C++算法解析：洛谷三角洲 深入学习指南 💡

今天我们来一起分析“洛谷三角洲”这道C++编程题。本指南将帮助大家梳理题目思路，理解核心算法，并掌握解题技巧。


## 1. 题目解读与核心算法识别

✨ **本题主要考察**：数学（最小值计算）

🗣️ **初步分析**：  
解决“洛谷三角洲”的关键，是用**数学中的“最小值比较”思路**——就像你从家到学校，要么直接走小路（快但绕吗？不，这里小路是直达），要么绕个便利店再去学校，选耗时更少的那条路就行！  

具体来说，三个城市A、B、C两两相连，每对城市间的最短时间只有两种可能：**直接走两市间的道路**，或者**绕第三个城市走两条路**。比如A到B，要么直接走x分钟的路，要么绕C走“A→C→B”（z+y分钟），取两者的最小值就是最短时间。其他城市对（B→C、A→C）同理。  

**核心算法流程**：对每对城市，计算两种路径的时间，取最小。比如：  
- A→B：min(直接走x，绕C的y+z)  
- B→C：min(直接走y，绕A的x+z)  
- A→C：min(直接走z，绕B的x+y)  

**可视化设计思路**：我们会用8位像素风展示三个城市（红A、蓝B、绿C）和三条道路（黄线标时间）。每一步会**高亮当前计算的城市对**（比如A和B闪烁），然后用动画演示“直接走”和“绕路”的路径（比如直接走是A→B的黄线变亮，绕路是A→C→B的黄线依次变亮），最后用**颜色标记更短的路径**（比如最短路径变绿），并伴随“叮”的音效强化记忆。

## 3. 核心难点辨析与解题策略

在解决这个问题时，大家容易遇到三个“小障碍”，我们一一拆解：

1. **难点1：想全所有可能的路径**  
   每对城市只有两种路径！比如A到B，不是“直接走”就是“绕C”，没有第三种可能（因为只有三个城市）。  
   * **解决策略**：画个三角形图，把每个城市对的两条路径写下来（比如A-B：x vs y+z），避免遗漏。  
   💡 **学习笔记**：复杂问题先画“关系图”，能快速理清思路！

2. **难点2：对应路径与时间的关系**  
   容易搞混“绕路”的时间组合——比如A到C的绕路是“A→B→C”，对应时间是x+y，而不是y+z！  
   * **解决策略**：用“起点→中间点→终点”的顺序记：A到C绕B，就是A→B（x）加B→C（y），所以是x+y。  
   💡 **学习笔记**：绕路的时间是“起点到中间点”加“中间点到终点”的时间！

3. **难点3：数据类型溢出**  
   x、y、z可以到1e9，相加会到2e9，超过`int`的范围（`int`最大约2e9？不对，`int`最大是2147483647，所以2e9会溢出！）。  
   * **解决策略**：用`long long`类型存储x、y、z，避免溢出。  
   💡 **学习笔记**：大数相加/相乘时，先想“会不会超过int范围”，用`long long`更安全！


### ✨ 解题技巧总结
- **技巧1：问题抽象**：把“找最短路径”抽象成“比较两种选择的最小值”，简化问题。  
- **技巧2：数据类型检查**：遇到大数（≥1e9），优先用`long long`。  
- **技巧3：画图辅助**：复杂关系用图表示，比文字更直观。


## 4. C++核心代码实现赏析

在深入分析具体题解的精妙片段之前，让我们先来看一个能够完整解决本题的通用核心C++实现参考。


### 本题通用核心C++实现参考
* **说明**：本代码综合了优质题解的思路，是最简洁高效的实现。  
* **完整核心代码**：
  ```cpp
  #include <iostream>
  #include <algorithm> // 用于min函数（注意：min对long long的支持需要C++11及以上）
  using namespace std;

  int main() {
      long long x, y, z; // 用long long存大数，避免溢出
      cin >> x >> y >> z;
      // 计算每对城市的最短时间
      long long ab = min(x, y + z);   // A→B的最短时间
      long long bc = min(y, x + z);   // B→C的最短时间
      long long ac = min(z, x + y);   // A→C的最短时间
      // 输出结果
      cout << ab << endl;
      cout << bc << endl;
      cout << ac << endl;
      return 0;
  }
  ```
* **代码解读概要**：  
  代码分三步：① 读取输入的三个道路时间（x、y、z）；② 用`min`函数计算每对城市的最短时间；③ 输出结果。关键是用`long long`避免溢出，用`min`直接比较两种路径的时间。


### 题解一：来源：「QQ红包」
* **亮点**：用最直白的“两种路径比较”思路，代码几乎没有冗余，适合入门者理解。  
* **核心代码片段**：
  ```cpp
  long long ab = min(x, y + z);
  long long bc = min(y, x + z);
  long long ac = min(z, x + y);
  ```
* **代码解读**：  
  这三行是题解的“灵魂”！比如`ab = min(x, y + z)`：`x`是A→B的直接时间，`y+z`是A→C→B的绕路时间，`min`函数会自动选更小的那个。这三行代码把“找最短路径”的逻辑直接转化为C++代码，非常巧妙！  
* 💡 **学习笔记**：把问题逻辑直接转化为代码，是编程的“终极目标”！

# 题目信息

# [语言月赛202303] Out for Dinner B

## 题目描述

Bessie 一家今晚要外出就餐。饭馆中只有三种菜，这三种菜的名称分别为 `A`、`B`、`C`。

饭店在进行折扣活动。如果我们称消费打 $Y$ 折，则意味着**支付金额**是**菜品金额**的 $\dfrac{Y}{10}$ 倍。具体的，如果购买了 `B` 菜且未购买 `C` 菜，则消费打**八折**；如果购买了 `C` 菜且未购买 `B` 菜，则消费打**七折**；如果 `B`、`C` 两道菜都购买了，则消费打**六折**。

这一次 Bessie 一家人只点了两道菜。这两道菜**可能是相同的**。最终结账时，**菜品金额**是 $x$ 元。给定两道菜的名字和 $x$，请你求出 Bessie 的**支付金额**。

## 说明/提示

### 样例 1 解释

Bessie 一家如果购买了 `B` 菜且未购买 `C` 菜，整次消费会打八折。他们本次未打折前需要支付 $100$ 元，因此打折后需要支付 $100 \times 80\% = 80$ 元。

### 数据规模与约定

对于 $100\%$ 的数据，保证 $10 \leq x \leq 10 ^ {18}$ 且 $x$ 是 $10$ 的倍数，$a, b$ 分别是大写英文字母 `A`、`B`、`C` 中的一个。

请注意，我们不保证 $a, b$ 一定不相同，这意味着可能出现 $a, b$ 相同的情况。


## 样例 #1

### 输入

```
100
AB```

### 输出

```
80```

## 样例 #2

### 输入

```
100
CA```

### 输出

```
70```

🗣️ **初步分析**：

题解的核心思路很明确：**先判断点的两道菜中有没有B和C，再对应折扣**。核心难点是“如何覆盖所有B、C的组合情况”（比如BC、CB、BB、CC、AC等），题解用了个巧办法——交换a和b让a≤b，这样只要判断a和b的顺序组合，就能减少一半的讨论量（比如BC和CB会变成同样的a=B、b=C）。

## 3. 核心难点辨析与解题策略

<difficulty_intro>
这道题的“坑”主要在“正确判断B和C的组合”和“处理大数”，我们一一拆解：
</difficulty_intro>

1.  **关键点1：如何覆盖所有B、C的组合？**
    * **分析**：两道菜的组合有8种（比如AB、BA、BC、CB等），直接判断每种情况会写很多if-else。题解的“交换排序”技巧太妙了——把a和b换成“小的在前”（比如BA变成AB，CB变成BC），这样只要判断a和b的顺序组合，比如a=B时b只能是B或C，a不是B时b只能是C或其他，瞬间减少一半工作量！
    * 💡 **学习笔记**：遇到对称的条件判断，试试“排序”变量，减少重复思考！

2.  **关键点2：如何处理“重复菜品”的情况？**
    * **分析**：比如点了两道B（BB），这时候“有B没C”，该打八折；点了两道C（CC），“有C没B”，打七折。题解的逻辑完全覆盖了这些情况——比如BB交换后还是BB，进入a=B、b=B的分支，y=8；CC交换后是CC，进入b=C的分支，y=7。
    * 💡 **学习笔记**：不要忽略“重复输入”的情况，分支逻辑要包含“两个相同菜”的判断！

3.  **关键点3：如何处理1e18的大数？**
    * **分析**：题目里x可以到1e18，用int会溢出（int最大约2e9），所以必须用long long（能存到9e18）。题解里用long long存x，计算时用x/10*y（先除10再乘y，避免中间溢出），比如x=1e18时，1e18/10=1e17，再乘6就是6e17，完全在long long范围内！
    * 💡 **学习笔记**：遇到大数先想“数据类型”，long long是处理大数的基础！

## 4. C++核心代码实现赏析

<code_intro_overall>
先看一个完整的核心实现，帮你建立整体框架！
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：本代码综合了题解的“交换排序”和“分支判断”思路，是最简洁的实现！
* **完整核心代码**：
    ```cpp
    #include <iostream>
    #include <algorithm> // 用于swap函数
    using namespace std;

    int main() {
        long long x;
        char a, b;
        cin >> x >> a >> b;

        // 交换a和b，让a ≤ b，减少分支
        if (a > b) swap(a, b);

        int y = 10; // 默认不打折（但题目中只有B/C影响折扣，所以实际不会用到10）
        if (a == 'B') {
            if (b == 'C') y = 6; // BC组合：六折
            else y = 8; // BB组合：八折
        } else if (b == 'C') {
            y = 7; // AC/CC组合：七折
        }

        cout << x / 10 * y << endl;
        return 0;
    }
    ```
* **代码解读概要**：
    > 代码像“流水线”：先读入x（金额）和a、b（两道菜）→ 交换a、b让顺序整齐→ 判断a和b的组合选折扣y→ 计算支付金额（x/10*y，因为折扣是y/10，比如8折就是x*(8/10)=x/10*8）→ 输出结果。核心是“交换排序+分支判断”，把复杂问题拆成简单步骤！

**题解一：来源：一扶苏一**

    ```cpp

* **代码解读**：
    > 1. **交换a和b**：比如输入是“CB”（a=C，b=B），a>b（C的ASCII码比B大），所以swap后a=B，b=C。这样不管输入是BC还是CB，都变成a=B、b=C，只需要判断一次！
    > 2. **分支判断**：如果a是B（比如BB、BC），那b如果是C就是六折（BC），否则是八折（BB）；如果b是C但a不是B（比如AC、CC），就是七折（因为有C没B）；如果a和b都不是B/C（比如AA），y还是10，但题目中只有B/C影响折扣，所以这种情况不影响结果。
* 💡 **学习笔记**：交换变量是“简化对称问题”的神器，比如处理两个变量的顺序无关问题时，先排序再判断！

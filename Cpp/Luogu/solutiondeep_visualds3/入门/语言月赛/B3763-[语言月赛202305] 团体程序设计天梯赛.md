# [语言月赛202305] 团体程序设计天梯赛

## 题目描述

七海参加了天梯赛。天梯赛的赛制较为独特，以下是天梯赛规则的节选。

>竞赛题目分 $3$ 个梯级：
>
> - 基础级设 $8$ 道题，其中 $5$ 分、$10$ 分、$15$ 分、$20$ 分的题各 $2$ 道，满分为 $100$ 分；
> - 进阶级设 $4$ 道题，每道题 $25$ 分，满分为 $100$ 分；
> - 登顶级设 $3$ 道题，每道题 $30$ 分，满分为 $90$ 分。

> 参赛队员的个人总分由其每题得分构成。但只有当该队员的基础题总分超过 $80$ 分时，其进阶部分的题目分数才被判为对其个人有效；其进阶题总分超过 $40$ 分时，其登顶部分的题目分数才被判为对其个人有效。

**请注意，这里的超过，指大于等于。**

现在，依照基础级-进阶级-登顶级的顺序给出某选手共 $15$ 题的分数，请你计算该名选手的有效得分。

## 说明/提示

对于 $100\%$ 的测试数据，输入的数据均为整数且符合题目实际情景。

## 样例 #1

### 输入

```
5 5 10 10 15 15 20 20
25 25 16 25
0 1 0```

### 输出

```
192
```

## 样例 #2

### 输入

```
0 0 0 0 0 0 1 1
25 25 25 25
30 30 30```

### 输出

```
2```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：`模拟` (算法分类标签)

🗣️ **初步分析**：  
模拟算法的核心，就像你跟着妈妈的食谱做番茄炒蛋：先切番茄、打鸡蛋，再下锅翻炒，每一步都要按规则来，不能乱顺序。本题的“食谱”就是天梯赛的得分规则——我们需要**模拟这个规则**，一步步算出选手的有效得分：

## 3. 核心难点辨析与解题策略

在解决模拟题时，我们常遇到以下3个关键点，结合题解的做法，我帮大家总结了应对策略：

### 关键点1：正确拆分三个阶段的题目数量  
**难点**：题目给了15题，要分成“基础8题、进阶4题、登顶3题”，数错数量就会全错！  
**策略**：用**循环次数控制输入**——基础用`for (int i=0; i<8; ++i)`，进阶`i<4`，登顶`i<3`，确保每阶段的题目数量正确。  
💡 学习笔记：模拟题的第一步，是“精准拆分规则中的步骤”！

### 关键点2：理解条件的顺序依赖  
**难点**：不是“基础≥80且进阶≥40”才能加登顶分，而是**必须先满足基础≥80，才能考虑进阶；满足进阶≥40，才能考虑登顶**（比如基础79，进阶100也不能加）。  
**策略**：用**嵌套if**实现顺序判断——先判断基础分，再判断进阶分，就像“闯关”一样，一关不过，后面的关都进不去。  
💡 学习笔记：条件有顺序时，要“按顺序判断”，不能颠倒！

### 关键点3：正确累加符合条件的分数  
**难点**：不是把所有分都加起来，而是“满足条件才加”（比如基础不够，进阶分再高也没用）。  
**策略**：用**逐步累加**的方式——先加基础分（total=L1），满足基础≥80再加进阶分（total+=L2），满足进阶≥40再加登顶分（total+=L3）。  
💡 学习笔记：累加分数要“看条件办事”，不能“一股脑全加”！


## 4. C++核心代码实现赏析

在看具体代码前，先看一个**通用核心实现**，帮你把握整体框架：

### 本题通用核心C++实现参考  
* **说明**：综合优质题解的思路，实现一个“直来直去”的模拟流程。  
* **完整核心代码**：  
```cpp
#include <iostream>
using namespace std;

int main() {
    int L1 = 0, L2 = 0, L3 = 0;
    // 读基础级8题的分数，求和L1
    for (int i = 0; i < 8; ++i) {
        int x;
        cin >> x;
        L1 += x;
    }
    // 读进阶级4题的分数，求和L2
    for (int i = 0; i < 4; ++i) {
        int x;
        cin >> x;
        L2 += x;
    }
    // 读登顶级3题的分数，求和L3
    for (int i = 0; i < 3; ++i) {
        int x;
        cin >> x;
        L3 += x;
    }
    // 按规则计算有效得分
    int total = L1;
    if (L1 >= 80) {
        total += L2;
        if (L2 >= 40) {
            total += L3;
        }
    }
    cout << total << endl;
    return 0;
}
```  
* **代码解读概要**：  
  1. 用三个变量`L1/L2/L3`分别存三个阶段的总分；  
  2. 用三个循环读入每个阶段的题目分数，求和；  
  3. 用嵌套if判断：先加基础分，满足条件再加进阶分，再满足条件加登顶分；  
  4. 最后输出有效得分。


### 题解一（览遍千秋）的片段赏析  
* **亮点**：用最简洁的代码，精准模拟规则。  
* **核心代码片段**：  
```cpp
// 计算有效得分的核心逻辑
int total = L1;
if (L1 >= 80) {
    total += L2;
    if (L2 >= 40) {
        total += L3;
    }
}
```  
* **代码解读**：  
  这段代码是整个题的“灵魂”！`total`一开始等于基础分L1——如果L1≥80（基础关过了），就加进阶分L2；如果L2≥40（进阶关过了），再加登顶分L3。就像“爬天梯”，每过一关，就往上走一步！  
* 💡 **学习笔记**：模拟题的代码要“简单直接”，跟着规则写，不要加多余的逻辑！

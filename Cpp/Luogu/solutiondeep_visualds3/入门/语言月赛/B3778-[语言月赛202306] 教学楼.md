# [语言月赛202306] 教学楼

## 题目描述

zyl 来到新的学校。学校很大，他想找到自己的教室在哪栋楼。

zyl 的班主任告诉了他教室在几楼，但学校有三座教学楼：图书馆，综合楼，艺术楼。这三栋教学楼的最高楼层数分别为 $3, 5, 9$。

zyl 得知，他想要找的教室在这座三栋教学楼中的一栋中。同时，zyl 掌握了如下信息：  

- 教室所在的楼层数 $n$。
- 三栋楼的**人流量** $a, b, c$（保证 $a, b, c$ 两两不同）。

显然，教室所在的楼层数一定**不大于**其所在的教学楼的最高楼层数。在满足该条件的基础上，他又得知，教室处在所有可能的教学楼中**人流量最小**的一栋。

现在，请你按照上述条件，帮助他找出他的教室处在哪栋教学楼。

## 说明/提示

### 样例 1 解释

教室在 $4$ 楼，图书馆的最高楼层为 $3$，因此教室不可能在图书馆。而剩下两栋楼中，综合楼的人流量较小，所以教室在综合楼。

### 数据规模与约定

- 对于 $40\%$ 的数据，$n\le3$。  
- 对于 $100\%$ 的数据，$1\leq n \leq 9$，$0\leq a,b,c\leq 20000$，且 $a,b,c$ 互不相同。

## 样例 #1

### 输入

```
4
20 50 100```

### 输出

```
comprehensive```

## 样例 #2

### 输入

```
2 
30 80 20```

### 输出

```
art```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：`模拟` (按照题目规则逐步执行)

🗣️ **初步分析**：  
模拟就像跟着食谱做蛋糕——题目给了明确的“步骤说明书”，我们要**一步一步照做**。本题的“说明书”是：先看教室楼层`n`属于哪个范围，再在符合条件的教学楼里找“人流量最小”的那栋。  

具体来说，三栋楼的“最高楼层限制”是：图书馆（≤3层）、综合楼（≤5层）、艺术楼（≤9层）。我们需要先把`n`放进对应的“楼层区间”，再在该区间的候选楼里选流量最小的。比如`n=4`时，只有综合楼（5层≥4）和艺术楼（9层≥4）符合条件，这时候比一比它们的流量，选小的那个。  

**核心算法流程**：  
1. 输入楼层`n`和三栋楼的人流量`a`（图书馆）、`b`（综合楼）、`c`（艺术楼）；  
2. 判断`n`的范围：  
   - `n≤3`：候选楼是全部三栋，找`a、b、c`中的最小值；  
   - `4≤n≤5`：候选楼是综合楼和艺术楼，找`b、c`中的最小值；  
   - `n≥6`：只有艺术楼符合条件，直接选它；  
3. 输出对应楼的名字。  

**可视化设计思路**：  
我们会用**8位像素风**还原这个过程——比如用蓝色像素块代表图书馆（3层）、绿色代表综合楼（5层）、红色代表艺术楼（9层），`n`用黄色像素数字显示在上方。当`n`确定后，**符合条件的楼会“亮起”**（比如闪烁），接着**流量最小的楼会“跳一下”**（像素动画），同时伴随“叮”的音效，帮你直观看到“谁是答案”。

## 3. 核心难点辨析与解题策略

在做这道题时，大家常犯的三个错误，我帮你总结好了对策：

### 关键点1：正确划分`n`的区间并对应候选楼  
**难点**：容易搞混“哪些楼符合条件”（比如`n=3`时，图书馆是允许的，但`n=4`时图书馆就不行了）。  
**对策**：画个“区间-楼”对照表：  
| `n`的范围 | 候选楼               |
|------------|----------------------|
| `1-3`      | 图书馆、综合楼、艺术楼 |
| `4-5`      | 综合楼、艺术楼         |
| `6-9`      | 艺术楼               |  
写代码前先把这个表贴在旁边，避免漏看或错看。

### 关键点2：在候选楼里找“流量最小”的楼  
**难点**：比如`n≤3`时，要比较三个数的最小值，容易漏掉情况（比如`a`不是最小，但`b`比`c`小）。  
**对策**：用“排除法”比较：  
- 先看`a`是不是比`b`和`c`都小？如果是，选图书馆；  
- 如果不是，再看`b`是不是比`c`小？如果是，选综合楼；  
- 剩下的就是艺术楼。  
这样一步步来，不会遗漏任何情况。

### 关键点3：准确输出楼名  
**难点**：楼名太长（比如`"comprehensive"`），容易拼错字母。  
**对策**：直接复制题目中的楼名字符串！比如题目里写的是`"comprehensive"`，你就直接把这个字符串粘到代码里，不要自己手写。

### ✨ 解题技巧总结  
- **画流程图**：把题目规则变成流程图，理清“先做什么，再做什么”；  
- **变量对应**：用`a、b、c`直接对应三栋楼的流量，避免混淆；  
- **复制字符串**：输出楼名时，直接复制题目中的内容，杜绝拼写错误。


## 4. C++核心代码实现赏析

### 本题通用核心C++实现参考  
* **说明**：本代码综合了题解的核心逻辑，是“模拟题目规则”的典型实现。  
* **完整核心代码**：  
```cpp
#include <iostream>
using namespace std;

int main() {
    int n, a, b, c;
    cin >> n >> a >> b >> c; // 输入楼层n和三栋楼的人流量
    
    if (n <= 3) {
        // 候选楼：图书馆(a)、综合楼(b)、艺术楼(c)
        if (a < b && a < c) {
            cout << "library";
        } else if (b < c) {
            cout << "comprehensive";
        } else {
            cout << "art";
        }
    } else if (n <= 5) {
        // 候选楼：综合楼(b)、艺术楼(c)
        if (b < c) {
            cout << "comprehensive";
        } else {
            cout << "art";
        }
    } else {
        // 只有艺术楼符合条件
        cout << "art";
    }
    
    return 0;
}
```
* **代码解读概要**：  
  代码分三步：  
  1. **输入**：读取`n`（楼层）和`a、b、c`（三栋楼的流量）；  
  2. **分支判断**：根据`n`的范围，进入对应的条件块；  
  3. **输出**：在每个条件块里比较流量，输出对应楼的名字。  
  核心逻辑都在`if-else`分支里，完全贴合题目规则。


### 题解一：(来源：未来姚班zyl)  
* **亮点**：用最直接的分支判断还原题目规则，代码几乎“零冗余”。

* **代码解读**：

  - 第二块`n<=5`：只需要比综合楼`b`和艺术楼`c`的流量，小的那个就是答案。

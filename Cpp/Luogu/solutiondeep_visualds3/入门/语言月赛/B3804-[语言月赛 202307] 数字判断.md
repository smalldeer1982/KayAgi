# [语言月赛 202307] 数字判断

## 题目背景

请注意，本题只有通过全部测试点才能获得分数。

## 题目描述

给定三个整数 $a, b, c$，请你判断它们是否满足如下**所有**条件：

1. $a + b + c \leq 100$。
2. $b$ 是 $5$ 的倍数。
3. $c$ 是 $7$ 的倍数。
3. $a - b > b - c$。

如果上述条件均满足，请输出 $\texttt{Yes}$，否则输出 $\texttt{No}$。

## 说明/提示

### 数据规模与约定

**本题采用捆绑测试，只有通过全部测试点才能获得分数**。

对全部的测试点，保证 $0 \leq a, b, c < 2^{31}$。

## 样例 #1

### 输入

```
50 10 7```

### 输出

```
Yes```

## 样例 #2

### 输入

```
100 100 100```

### 输出

```
No```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：模拟  

🗣️ **初步分析**：  
解决“数字判断”这道题，关键在于运用**模拟**的思路——就像老师检查作业时逐一核对“是否写名字、是否做完所有题、是否写对答案”一样，我们需要按照题目给出的4个条件，一个一个“核对”给定的三个数字`a`、`b`、`c`是否满足。  
在本题中，模拟的核心就是“按条件顺序检查”：先算三个数的和是否≤100，再看`b`是不是5的倍数，`c`是不是7的倍数，最后验证`a-b`是否大于`b-c`。只有**所有条件都满足**，才能输出`Yes`，否则输出`No`。  

核心难点有两个：① 三个数的和可能超过`int`的范围（比如`a`、`b`、`c`都是接近2³¹的数，和会达到3×2³¹，远超`int`的最大值2¹⁷左右），所以要用`long long`类型存储；② 确保所有条件都被正确翻译为代码（比如条件4是“`a-b > b-c`”，不能写成“`a-b ≥ b-c`”）。  

可视化设计上，我们会把四个条件做成“像素化的检查关卡”：每个条件对应一个16×16的像素格子，检查通过就变绿并播放“叮”的音效，不通过就变红并播放“咔”的音效。比如检查条件1时，三个数字的像素块会“飞”到条件1的格子里叠加，超过100的话块会“溢出”，伴随错误音效，直观展示“和超过100”的情况。

## 3. 核心难点辨析与解题策略

在解决这个问题的过程中，我们通常会遇到以下几个核心难点。结合优质题解的经验，我为大家提炼了对应的解决策略：

### 1. 关键点1：数据范围的“向前看”——避免求和溢出  
* **分析**：题目中说`a`、`b`、`c < 2³¹`，但三个数的和可能达到`3×2³¹`，远超`int`的最大值（约2¹⁷）。如果用`int`存储，求和时会发生“溢出”（结果变成负数或乱码），导致条件1判断错误。优质题解用`long long`定义`a`、`b`、`c`，完美解决了这个问题。  
* 💡 **学习笔记**：定义变量时，不仅要考虑变量本身的范围，还要考虑变量参与计算后的结果范围——比如求和、乘法等操作可能让结果变大，这时要用更大的类型（如`long long`）。

### 2. 关键点2：条件的“逻辑与”——所有条件都要满足  
* **分析**：题目要求“满足所有条件”，即四个条件必须同时为真。优质题解用`ans`变量逐步判断：初始`ans = true`，只要有一个条件不满足就设为`false`。这种写法比用多个`&&`连接条件更直观，也更容易调试（比如可以在每个条件判断后加`cout`，看是哪个条件导致`ans`变`false`）。  
* 💡 **学习笔记**：当需要满足多个条件时，用一个`bool`变量逐步“排除”不满足的情况，逻辑更清晰。

### 3. 关键点3：条件的“精准翻译”——避免运算符错误  
* **分析**：题目中的条件4是“`a - b > b - c`”，如果写成“`a - b ≥ b - c`”就会错误。优质题解直接按照题目写“`if (a - b <= b - c) ans = false`”，即“如果不满足条件4，就设`ans`为`false`”，完全符合题目要求。  
* 💡 **学习笔记**：翻译题目中的数学表达式时，一定要严格按照题目写，不要自己修改运算符（比如把`>`改成`≥`）。

### ✨ 解题技巧总结  
通过本题分析，我总结了以下通用技巧：  
- **变量类型要“超前”**：考虑变量参与计算后的结果范围，避免溢出。  
- **条件判断要“逐一”**：用`bool`变量逐步检查多个条件，逻辑更清晰。  
- **表达式翻译要“精准”**：严格按照题目写数学表达式，避免运算符错误。  


## 4. C++核心代码实现赏析

在深入分析具体题解的精妙片段之前，让我们先来看一个完整的核心C++实现参考，帮助大家把握整体框架。

### 本题通用核心C++实现参考  
* **说明**：此代码来自题解“一扶苏一”，是本题的典型实现，逻辑清晰且处理了数据范围问题。  
* **完整核心代码**：  
```cpp
#include <iostream>
using namespace std;

int main() {
    long long a, b, c;
    cin >> a >> b >> c;
    bool ans = true;
    // 条件1：a + b + c <= 100
    if (a + b + c > 100) ans = false;
    // 条件2：b是5的倍数
    if (b % 5 != 0) ans = false;
    // 条件3：c是7的倍数
    if (c % 7 != 0) ans = false;
    // 条件4：a - b > b - c
    if (a - b <= b - c) ans = false;
    // 输出结果
    if (ans) cout << "Yes" << endl;
    else cout << "No" << endl;
    return 0;
}
```  
* **代码解读概要**：  
该代码的核心逻辑是“读入数据→逐一检查条件→输出结果”。首先用`long long`定义`a`、`b`、`c`，避免求和溢出；然后用`ans`变量记录是否满足所有条件；接着依次检查四个条件，只要有一个不满足就设`ans`为`false`；最后根据`ans`的值输出`Yes`或`No`。


### 题解一：(来源：一扶苏一)  
* **亮点**：正确处理数据范围，条件判断清晰，代码规范。  
* **核心代码片段**：  
```cpp
long long a, b, c;
cin >> a >> b >> c;
bool ans = true;
if (a + b + c > 100) ans = false;
if (b % 5 != 0) ans = false;
if (c % 7 != 0) ans = false;
if (a - b <= b - c) ans = false;
```  
* **代码解读**：  
这段代码是题解的核心。首先用`long long`定义`a`、`b`、`c`——因为三个数的和可能超过`int`的范围。然后`ans`初始化为`true`（假设所有条件都满足）。接下来的四个`if`语句分别检查四个条件：  
- 如果条件1不满足（和>100），`ans`变`false`；  
- 如果条件2不满足（`b`不是5的倍数），`ans`变`false`；  
- 依此类推。  
最后，`ans`的值就是是否满足所有条件的结果。  
* 💡 **学习笔记**：处理可能溢出的数值时，要用`long long`；逐一检查条件的写法更直观，容易调试。

# 题目信息

# [语言月赛 202308] 小粉兔还会忘记考试吗

## 题目描述

小粉兔在本学期共修读了 $N$ 门课程，第 $i$ 门课程的课程代码为 $E_i$，每门课程都需要在结课后进行课程考试。

教务开始处理粉兔的成绩。教务共录入了 $M$ 条成绩信息，一条成绩信息由课程代码和成绩组成。由于成绩可能存在调整，一门课程的成绩可能被重复录入，后录入的成绩将覆盖先录入的成绩。课程成绩在 $60$ 分及以上，该门课程为及格；否则，为不及格。

由于粉兔记性很差，很可能忘记参加某些课程的课程考试。如果教务没有录入粉兔某门课程的成绩，则代表粉兔旷考了该门课程。旷考的课程亦为不及格。

小 F 就是这名教务，小 F 对粉兔旷考和不及格的门数非常感兴趣，请你帮助她统计。

## 说明/提示

对于 $100\%$ 的测试数据，$1 \le N,M \le 5 \times 10^3$，$1 \le E_i,P_i \le 10^9$，$E_i$ 互不相同，$0 \le S_i \le 100$。 保证存在一个整数 $j$，使得 $P_i=E_j$。

5 6

10 59```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：模拟  

🗣️ **初步分析**：  
模拟就像“玩过家家”——我们要照着题目描述的**真实流程**，一步步还原出来。比如这道题里，教务录入成绩的过程（后录的覆盖前录的）、区分“旷考”和“考0分”的需求，都需要我们用代码“模拟”出来。  

具体来说，题解的核心思路是：  
1. 用数组`E`存所有课程的代码（相当于“课程名单”）；  
2. 用数组`a`存每门课的**最新成绩**，初始值设为`-1`（重点！这样能区分“没录入成绩”和“考了0分”）；  
3. 处理每一条成绩时，找到对应的课程位置，更新`a`数组（覆盖旧成绩）；  
4. 最后统计：`a`中值为`-1`的是旷考，值非`-1`但小于60的是不及格。  

**核心难点**：如何区分“没录入成绩”和“考了0分”？  
**解决方案**：把`a`数组初始化为`-1`——没录入的课程保持`-1`（旷考），录入0分的课程会被设为`0`（真的不及格）。  

**可视化设计思路**：我们可以做一个“像素课程表”动画——每个课程是一个8×8的像素块，初始是灰色（代表`-1`，旷考）。录入成绩时，找到对应的像素块，先闪烁一下（表示“正在更新”），再变成对应颜色：绿色（≥60分）、红色（<60分）。统计时，红色块和灰色块会分别高亮，旁边显示“旷考数”和“不及格数”。动画还会加“叮”的音效（录入成功）和“滴”的音效（统计完成），像玩复古小游戏一样！

## 3. 核心难点辨析与解题策略

解决这道题的关键，在于搞定3个“小问号”：

### 1. 怎么区分“没录入成绩”和“考了0分”？  
**分析**：如果`a`数组初始为`0`，那么“没录入”和“考0分”的结果都是`0`，根本分不出来！  
**解决方案**：把`a`数组初始化为`-1`（一个不可能的成绩值）。这样：  
- 没录入成绩 → `a[i] = -1`（旷考）；  
- 考0分 → `a[i] = 0`（真的不及格）。  

💡 **学习笔记**：用“特殊值”标记特殊状态，是模拟题的常用技巧！


### 2. 怎么找到成绩对应的课程位置？  
**分析**：每条成绩给的是“课程代码”，我们需要知道它对应`E`数组里的第几个位置（比如课程代码`10`是`E[0]`还是`E[1]`）。  
**解决方案**：用循环遍历`E`数组，找到等于当前课程代码的下标`k`，然后更新`a[k]`。  

💡 **学习笔记**：遍历是“找对应关系”的基础方法，虽然慢但简单直接！


### 3. 怎么处理“重复录入覆盖旧成绩”？  
**分析**：题目说“后录入的覆盖先录入的”，所以每次找到位置后，直接把新成绩赋给`a[k]`就行——旧值会被自动覆盖。  
**解决方案**：不需要额外判断，直接赋值！比如第一次录入`10`课的成绩是`20`，`a[k]`变成`20`；第二次录入`10`课的成绩是`59`，`a[k]`直接变成`59`，旧值就没了。  

💡 **学习笔记**：模拟题要“懒一点”——题目说什么，我们就做什么，不用多猜！


### ✨ 解题技巧总结  
- **技巧A：用特殊值标记特殊状态**：比如用`-1`标记“未录入”，避免混淆；  
- **技巧B：模拟流程要“直译”题目**：题目说“后录覆盖先录”，我们就直接赋值覆盖；  
- **技巧C：统计时分开算**：先算旷考（`a[i]==-1`），再算不及格（`a[i]!=-1 && a[i]<60`），不要混在一起！


## 4. C++核心代码实现赏析

### 本题通用核心C++实现参考  
* **说明**：本代码综合了题解的核心思路，是一个完整的、易读的模拟实现。  

```cpp
#include <iostream>
#include <vector>
using namespace std;

int main() {
    int N, M;
    cin >> N >> M;
    vector<int> E(N);  // 存课程代码
    vector<int> a(N, -1);  // 存最新成绩，初始-1（旷考）
    
    // 输入课程代码
    for (int i = 0; i < N; ++i) {
        cin >> E[i];
    }
    
    // 处理M条成绩
    for (int i = 0; i < M; ++i) {
        int code, score;
        cin >> code >> score;
        // 找到对应课程的位置k
        for (int k = 0; k < N; ++k) {
            if (E[k] == code) {
                a[k] = score;  // 覆盖旧成绩
                break;  // 找到就退出循环，节省时间
            }
        }
    }
    
    // 统计旷考和不及格
    int absent = 0, fail = 0;
    for (int i = 0; i < N; ++i) {
        if (a[i] == -1) {
            absent++;
        } else if (a[i] < 60) {
            fail++;
        }
    }
    
    cout << absent << endl << fail << endl;
    return 0;
}
```

* **代码解读概要**：  
代码分四步：1. 读入课程数量`N`和成绩数量`M`；2. 读入所有课程代码存在`E`里；3. 处理每条成绩，找到对应课程位置更新`a`数组；4. 统计旷考（`a[i]==-1`）和不及格（`a[i]<60`）的数量，输出结果。核心是`a`数组的初始化和成绩更新的循环！


### 题解一核心代码片段赏析  
**题解一：来源：览遍千秋**  
* **亮点**：用最直接的循环查找对应课程，代码简洁到“一看就懂”！  
* **核心代码片段**：  
```cpp
// 处理M条成绩
for (int i = 0; i < M; ++i) {
    int code, score;
    cin >> code >> score;
    // 找到对应课程的位置k
    for (int k = 0; k < N; ++k) {
        if (E[k] == code) {
            a[k] = score;  // 覆盖旧成绩
            break;
        }
    }
}
```

* **代码解读**：  
这段代码是“成绩录入”的核心！每次读入课程代码`code`和成绩`score`后，用`k`循环遍历`E`数组——找到`E[k] == code`的位置，就把`a[k]`设为`score`（覆盖旧成绩），然后`break`退出循环（不用再找了）。是不是像“按名字找座位”？找到对应的座位，就把新成绩放上去～  

💡 **学习笔记**：循环里的`break`很重要！找到对应位置就停，能节省时间哦～

用8位像素风是因为它“复古又亲切”，像小时候玩的FC游戏；加入音效是为了“强化记忆”——比如录入成绩时“叮”一声，统计完成时“滴”一声，让你对关键操作印象更深；每完成一条成绩录入，就像“过了一小关”，增加成就感！


### 动画帧步骤与交互关键点  
1. **场景初始化（8位像素风）**：  
   - 屏幕左边是“课程列表”：N个8×8的像素块，初始是灰色（代表`-1`，旷考），每个块下面标着课程代码（比如`10`、`20`）；  
   - 右边是“控制面板”：有【开始】【单步】【重置】按钮，一个速度滑块（调节动画快慢），还有“当前成绩”显示区；  
   - 背景播放8位风格的轻松BGM（比如《超级马里奥》的轻快版）。  

2. **成绩录入演示**：  
   - 每处理一条成绩，对应的课程像素块会**闪烁3次**（表示“正在更新”），然后变成对应颜色：绿色（≥60分）、红色（<60分）；

   - 比如录入`10 20`时，`10`号课程的灰色块闪烁，然后变成红色（20<60），旁边显示“当前成绩：20”。  

3. **统计过程演示**：

4. **交互设计**：  
   - 【单步】：点击一次，执行一条成绩录入；  
   - 【自动播放】：按速度滑块的速度，自动完成所有录入；  
   - 【重置】：恢复初始状态，重新开始。

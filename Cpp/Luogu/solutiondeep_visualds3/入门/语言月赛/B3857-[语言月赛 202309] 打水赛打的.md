# [语言月赛 202309] 打水赛打的

## 题目描述

在 H 学校，C 考试一直被用来衡量学生编程、数据结构和算法能力的重要标准。C 考试满分 $500$ 分，一名学生只有取得不低于 $200$ 分的成绩才能毕业，我们称这样的学生为**达到毕业要求的学生**，C 考试未取得不低于 $200$ 分成绩的学生为**达不到毕业要求的学生**。

然而，随着内卷的激烈化，R 比赛走进了同学们的视野。由于该比赛组织混乱、试题难度低、获奖比例高，被同学们戏称为一等奖点击就送，成为公认的水赛。

在 2023 年的一等奖获奖名单中，同学们惊讶的发现，大量**达不到毕业要求的学生**竟然可以在 R 比赛中获得一等奖，这无疑是非常讽刺的一件事。

现在，给出 依次编号为 $1\sim N$ 的 H 学校的 $N$ 名同学的 C 考试成绩，和 R 比赛一等奖获奖名单。获奖名单中可能有超过 $N$ 的编号，这是由于其他一些学校也参加了 R 比赛，只有 $1 \sim N$ 编号的学生属于 H 学校。

请你统计，有多少 H 学校的一等奖获得者，是**达不到毕业要求的学生**。

## 说明/提示

 - 对于 $30\%$ 的测试数据，$1 \le N,M \le 5000$；
 - 对于另外 $30\%$ 的测试数据，$M \le N$；
 - 对于 $100\%$ 的测试数据，$1 \le N,M \le 10^5$，$1 \le W_i \le 2\times 10^5$，$0 \le S_i \le 500$。保证 $W_i$ 不重复。 

## 样例 #1

### 输入

```
5 8
200 0 199 200 500
3 1 2 6 8 10 11 12```

### 输出

```
2```

## 3. 核心难点辨析与解题策略

在解决这个问题的过程中，我们通常会遇到以下关键点：

1. **关键点1：如何高效存储并访问学生成绩？**  
   * **分析**：题目中N可以达到1e5，需要一个能快速访问的数据结构。数组是最佳选择——因为学生编号是1~N的连续整数，用数组`S[i]`存储编号i的成绩，访问时间是O(1)，完全满足大数据量的需求。题解中正是用数组`S`实现了这一点。  
   * 💡 **学习笔记**：连续编号的信息，优先用数组存储！

2. **关键点2：如何正确筛选本校学生？**  
   * **分析**：获奖编号W_i可能很大（比如样例中的6、8等），必须判断是否满足1≤W_i≤N。如果直接访问`S[W_i]`而不判断，会导致数组越界（比如W_i=6，N=5时，`S[6]`是不存在的）。题解中用“`if (W_i <1 || W_i >N) 跳过`”完美解决了这个问题。  
   * 💡 **学习笔记**：访问数组前，一定要检查索引是否合法！

3. **关键点3：如何避免重复计数？**  
   * **分析**：题目中明确说“W_i不重复”，所以每个获奖编号只会出现一次，不需要额外去重。这是题目给的“福利”，我们要善于利用题目中的条件简化代码。  
   * 💡 **学习笔记**：读题时要注意“保证条件”，比如“W_i不重复”可以省下去重的步骤！

## 4. C++核心代码实现赏析

在深入分析具体题解的片段前，先看一个完整的核心实现，帮大家把握整体框架：

### 本题通用核心C++实现参考
* **说明**：本代码综合了题解的思路，提供一个清晰且完整的核心实现，完全符合题目要求。
* **完整核心代码**：
```cpp
#include <iostream>
#include <vector>
using namespace std;

int main() {
    ios::sync_with_stdio(false); // 加速输入输出（应对1e5数据）
    cin.tie(nullptr);

    int N, M;
    cin >> N >> M;

    vector<int> S(N + 1); // S[1]~S[N]对应编号1~N的学生成绩
    for (int i = 1; i <= N; ++i) {
        cin >> S[i];
    }

    int ans = 0;
    for (int i = 0; i < M; ++i) {
        int W;
        cin >> W;
        if (W < 1 || W > N) { // 非本校学生，跳过
            continue;
        }
        if (S[W] < 200) { // 成绩不达标
            ++ans;
        }
    }

    cout << ans << endl;
    return 0;
}
```
* **代码解读概要**：  
  代码首先用`ios::sync_with_stdio(false)`和`cin.tie(nullptr)`加速输入输出（应对1e5级别的数据）；然后读取N和M，用`vector<int> S(N+1)`存储1~N号学生的C成绩；接着遍历每个获奖编号W，判断是否是本校学生（1≤W≤N），若是则检查成绩是否<200，满足则计数加1；最后输出计数结果。整个过程就是“读取数据→按规则判断→统计结果”，完全模拟题目要求。


### 题解核心片段赏析
**题解一：(来源：览遍千秋)**  
* **亮点**：用最简洁的代码实现题目要求，充分利用数组的快速访问特性，逻辑无冗余。
* **核心代码片段**：
```cpp
// 存储H校学生成绩
vector<int> S(N + 1);
for (int i = 1; i <= N; ++i) {
    cin >> S[i];
}

// 统计符合条件的人数
int ans = 0;
for (int i = 0; i < M; ++i) {
    int W;
    cin >> W;
    if (W >= 1 && W <= N && S[W] < 200) {
        ans++;
    }
}
```
* **代码解读**：  
  这段代码是题解的核心逻辑。首先，`vector<int> S(N+1)`创建了一个大小为N+1的数组——是不是很聪明？这样`S[W]`就能直接对应编号W的学生成绩，不用再做“`S[W-1]`”这样的转换。然后，遍历每个获奖编号W，用`W >=1 && W <=N`判断是否是本校学生，再用`S[W] <200`判断成绩是否达标，两个条件都满足就计数加1。整个过程没有多余的步骤，完全“按题目要求办事”。  
* 💡 **学习笔记**：数组的索引设计要贴合题目中的编号，能减少很多麻烦！

# 题目信息

# [语言月赛 202311] 食堂

## 题目描述

为了给师生提供良好的用餐体验，洛谷小学的食堂坚持现炒、现做每一道菜肴。

洛谷小学一共有 $a$ 名老师和 $b$ 名学生。食堂的营养师为每位师生的用餐进行配额：

- 一名学生，一次用餐需要 $R$ 克米饭，$V$ 克蔬菜，$M$ 克肉。
- 一名老师，一次用餐需要 $2R$ 克米饭，$3V$ 克蔬菜，$3M$ 克肉。

洛谷小学的食堂一天需要烹制两餐，分别为中午一餐、晚上一餐。其中，中午的一餐，学生和老师都需要，而晚上的一餐仅有老师需要。

现在请问，洛谷小学的食堂，一天要准备多少克米饭，多少克蔬菜，多少克肉呢？

## 说明/提示

**【样例解释】**

对于样例 $1$，洛谷小学有 $5$ 个老师和 $10$ 个学生。每天每个学生吃 $1$ 餐，每个老师吃 $2$ 餐，因此：

- 一个学生一餐吃 $200$ 克米饭。因此需要准备：$1\times 10\times 200+2\times 5\times 400=6000$ 克米饭。
- 一个学生一餐吃 $100$ 克蔬菜。因此需要准备：$1\times 10\times 100+2\times 5\times 300=4000$ 克蔬菜。
- 一个学生一餐吃 $150$ 克肉，因此需要准备：$1\times 10\times 150+2\times 5\times 450=6000$ 克肉。

因此，输出 $6000,4000,6000$。

**【数据范围】**

对于所有数据，保证：$1\leq a,b,R,V,M \leq 10000$。

## 样例 #1

### 输入

```
5 10 200 100 150```

### 输出

```
6000 4000 6000```

## 样例 #2

### 输入

```
15 120 150 200 180```

### 输出

```
27000 42000 37800```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：`数学（简单公式推导）` + `编程技巧应用（输入输出基础）`

🗣️ **初步分析**：
解决这道题的关键，就像帮食堂阿姨“算总账”——我们需要把题目里的“用餐规则”转化成**数学公式**，再用代码计算结果。简单来说，数学在这里的作用是“翻译”：把“1个学生吃1餐要R克米饭”翻译成`学生的米饭总量 = 学生人数 × 1餐量`，把“1个老师吃2餐，每餐要2R克米饭”翻译成`老师的米饭总量 = 老师人数 × 2餐 × 2R`。

题解的核心思路很直接：**拆解规则→推导公式→代码计算**。核心难点是**理清“谁吃几餐”和“每餐吃多少”的乘法关系**（比如老师的米饭系数是`2餐×2R=4R`，蔬菜是`2餐×3V=6V`），容易把系数搞混。解决方法是“一步一步拆”：先算1个人的总用量，再乘人数。

可视化设计上，我们会用**像素风格的食堂场景**展示计算过程：比如用像素小人代表老师/学生，用彩色方块代表食材（米饭黄、蔬菜绿、肉红），每一步计算时（比如算学生的米饭量），对应的食材方块会“增加”相应数量，同时用数字冒泡提示“这一步在算什么”（比如“学生的米饭：10×200=2000”）。动画还会加“叮”的像素音效，强化每一步的计算记忆～

## 3. 核心难点辨析与解题策略

<difficulty_intro>
这道题的“坑”都藏在“规则细节”里，我们逐一破解～
</difficulty_intro>

1.  **难点1：理清“用餐次数”和“每餐用量”的关系**  
    * **分析**：学生只吃1餐，老师要吃2餐；而老师每餐的用量是学生的倍数（比如米饭是2倍，蔬菜是3倍）。容易错把“老师的总用量”算成`a×2R`（忘记乘2餐），或者`a×3V`（忘记乘2餐）。  
    * **解决策略**：用“**个人总用量=每餐用量×餐数**”的公式拆解：比如老师的米饭个人总用量是`2R（每餐）×2（餐数）=4R`，蔬菜是`3V×2=6V`，肉是`3M×2=6M`。  
    * 💡 **学习笔记**：复杂的乘法关系，拆成“小步骤”算就不会错！

2.  **难点2：区分“变量顺序”和“输出顺序”**  
    * **分析**：题目输入是`a（老师）、b（学生）、R、V、M`，但计算时要注意“学生在前，老师在后”（比如米饭总量=学生的b×R + 老师的a×4R），输出顺序是“米饭→蔬菜→肉”，容易搞反。  
    * **解决策略**：写代码前先列好“输入变量表”和“输出公式表”，比如：  
      - 输入：a（老师）、b（学生）、R（学生米饭）、V（学生蔬菜）、M（学生肉）  
      - 输出：米饭= b*R + a*4*R；蔬菜= b*V + a*6*V；肉= b*M + a*6*M  
    * 💡 **学习笔记**：“先写公式再写代码”是避免顺序错误的秘诀！

3.  **难点3：C++中的“乘法表达”**  
    * **分析**：题目中的`bR`在C++里必须写成`b*R`（乘号不能省），否则会报错。  
    * **解决策略**：记住“所有运算都要写运算符”，比如`b*R`、`a*4*R`，不要偷懒！  
    * 💡 **学习笔记**：代码是“严格的语言”，每一步都要符合语法规则～

### ✨ 解题技巧总结
- **技巧1：规则拆解**：把题目中的“大规则”拆成“小公式”，比如“老师的总用量=每餐用量×餐数×人数”。  
- **技巧2：先算后写**：写代码前先在纸上算一遍样例（比如样例1中，米饭=10×200 +5×4×200=2000+4000=6000），确认公式正确。  
- **技巧3：输出检查**：输出时用空格分隔三个结果，顺序不要乱（米饭→蔬菜→肉）。


## 4. C++核心代码实现赏析

<code_intro_overall>
这道题的核心代码超简单！我们先看一个“通用模板”，帮你快速掌握整体框架～
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：本代码综合了题解的核心思路，是“最简洁的入门实现”，适合新手直接复用。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    using namespace std;

    int main() {
        int a, b, R, V, M;
        cin >> a >> b >> R >> V >> M;
        long long rice = 1LL * b * R + 1LL * a * 4 * R;
        long long vegetable = 1LL * b * V + 1LL * a * 6 * V;
        long long meat = 1LL * b * M + 1LL * a * 6 * M;
        cout << rice << " " << vegetable << " " << meat << endl;
        return 0;
    }
    ```
* **代码解读概要**：
    > 代码分三步：①用`cin`读取输入的5个变量（老师数a、学生数b、学生的三餐用量R/V/M）；②用公式计算三个食材的总量（注意用`long long`防止数值溢出，比如10000×10000×6=6e8，超过`int`的范围）；③用`cout`输出结果，用空格分隔。

**题解一：来源：chen_zhe**
* **亮点**：用“直接公式”代替复杂逻辑，代码可读性拉满！
* **核心代码片段**：
    ```cpp
    // 计算米饭总量：学生的b*R + 老师的a*4*R
    long long rice = b * R + a * 4 * R;
    // 计算蔬菜总量：学生的b*V + 老师的a*6*V
    long long vegetable = b * V + a * 6 * V;
    // 计算肉总量：学生的b*M + 老师的a*6*M
    long long meat = b * M + a * 6 * M;
    ```
* **代码解读**：
    > 这段代码的“聪明”在于**直接对应题目规则**：比如`a*4*R`就是“老师人数×每人大米饭总量（4R）”，`b*R`是“学生人数×每人大米饭总量（R）”。没有多余的逻辑，新手一眼就能看懂！为什么用`long long`？因为当a、b、R都是10000时，`a*4*R=10000×4×10000=4e8`，超过了`int`的最大值（约2e9？不对，`int`通常是4字节，最大值是2147483647，4e8是400000000，没超过？但为了保险，比如当a=10000，b=10000，R=10000时，`b*R=1e8`，`a*4*R=4e8`，总和是5e8，还是没超过`int`？但有些编译器的`int`是2字节，所以用`long long`更安全～  
    > 提示：计算时如果担心溢出，可以把其中一个数转成`long long`，比如`1LL * b * R`，这样整个表达式会自动提升为`long long`类型。
* 💡 **学习笔记**：“代码=题目规则的直译”是最棒的编程风格！

# 题目信息

# [语言月赛 202311] 数学选择题

## 题目描述

洛谷小学的数学考试由若干道选择题组成。选择题分为三档：简单题、中等题和困难题。每类题目都有评判标准：

- 简单题：回答正确得 $5$ 分，回答错误则不得分；
- 困难题：回答正确得 $20$ 分，回答错误倒扣 $20$ 分；

洛谷小学不鼓励学生为了做困难题而忽视基础，因此，如果简单题得到的总分，**不超过**给定的分值限制 $M$ 时，做困难题回答正确不得分，而回答错误一样倒扣 $20$ 分。同时，为了照顾学生情绪，如果学生所有题累计的得分小于 $0$，则按照 $0$ 分计算。

现在给定试卷中简单题的题数 $a$，困难题的题数 $b$，以及小 A 做对的简单题的题数 $c$，困难题的题数 $d$，以及给定的分值限制 $M$。问，小 A 最终能获得几分。

## 说明/提示

**【样例解释】**

- 对于样例 $1$，小 A 做对了 $7$ 道简单题获得了 $35$ 分，超过了给定的 $30$ 分限制，因此会正常计算困难题的得分。小 A 做对了 $4$ 个困难题获得 $80$ 分，但是错了一道要倒扣 $20$ 分，因此合计为 $35+80-20=95$ 分。
- 对于样例 $2$，小 A 做对了 $7$ 道简单题获得了 $35$ 分，并未超过给定的阈值 $40$，因此只计算做错困难题的倒扣分，合计为 $35-20=15$ 分。
- 对于样例 $3$，与样例 $2$ 一样，只计算做错困难题的倒扣分。小 A 只做对了 $1$ 个困难题，要倒扣 $80$ 分。此时小 A 累计的得分小于 $0$，则按照 $0$ 分作为最后的分值。

**【数据范围】**

对于所有数据，保证：$1 \leq a,b,c,d \leq 100$，$1 \leq c \leq a$，$1 \leq d \leq b$，$1 \leq M \leq 500$。

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：分支结构（if语句）

🗣️ **初步分析**：
分支结构就像生活中的“选择开关”——比如妈妈问你“作业写完没？”，写完就能看动画片，没写完就得继续写。在这道题里，“开关”是**简单题的总分是否超过M**：
- 如果超过（开关“开”）：困难题答对得20分、答错扣20分；
- 如果没超过（开关“关”）：困难题答对不得分，但答错还是扣20分。

**核心解题流程**：  
1. 计算简单题得分（5×做对的题数c）→ 记为s；  
2. 用s和M比大小，选择困难题的计算方式；  
3. 计算总得分，再把总分和0比（不能为负）。  

**核心难点**：  
- 别把“简单题做对的题数c”看成“总题数a”（很多人第一次会错！）；  
- 准确判断“困难题得分是否有效”（只有s>M时，答对才给分）；  
- 最后记得“总分不能低于0”。  

**可视化设计思路**：  
我们会做一个**8位像素风的“计分板小游戏”**：  
- 屏幕左边是“简单题计分器”：数字从0开始，每跳一下加5（对应做对一题），直到跳到5×c，伴随“滴”的像素音效；  
- 中间是“阈值指示灯”：如果计分器数字超过M，指示灯会从“红”变“绿”，触发右边“困难题计算区”的动画；  
- 困难题区：如果指示灯绿了，会弹出“+20×d”的黄色数字（答对得分）和“-20×(b-d)”的红色数字（答错扣分）；如果红了，只弹出红色数字；  
- 最后“总分框”：如果数字变成负数，会“啪”地弹回0，伴随“嗡”的提示音；正数则保持不变，播放“叮”的胜利音效。

## 3. 核心难点辨析与解题策略

<difficulty_intro>
解决这道题的“拦路虎”其实就3个——但只要把它们“拆穿”，问题就会变得简单！
</difficulty_intro>

1.  **难点1：简单题得分算错（把c看成a）**  
    * **分析**：题目说“做对的简单题数是c”，总题数a是“干扰项”！比如样例1中a=10，但c=7，所以简单题得分是5×7=35，不是5×10=50。  
    * 💡 **学习笔记**：读题时圈出“关键变量”——比如“做对的××题数”，别被“总题数”带偏！

2.  **难点2：困难题的条件判断（什么时候给答对分？）**  
    * **分析**：题目说“简单题总分≤M时，困难题答对不得分”——反过来，只有“简单题总分>M”时，答对才得20分。千万别搞反！  
    * 💡 **学习笔记**：条件判断要“反过来想”——题目说“不超过则不得分”，等于“超过才得分”。

3.  **难点3：忘记处理负分（总分小于0要置为0）**  
    * **分析**：比如样例3中，总分是35-80=-45，这时候要输出0，而不是-45。  
    * 💡 **学习笔记**：最后一步一定要“和0比大小”——用`max(总分, 0)`就能解决！

### ✨ 解题技巧总结
- **技巧1：圈画关键变量**：读题时用铅笔圈出“做对的题数”“阈值M”等核心数据，避免混淆；  
- **技巧2：写“伪代码”理清逻辑**：先在纸上写“如果s>M，那么hard=20d-20(b-d)；否则hard=-20(b-d)”，再翻译成C++；  
- **技巧3：最后一步检查负分**：养成“总分取max(0, 总分)”的习惯，避免漏掉题目要求。  


## 4. C++核心代码实现赏析

<code_intro_overall>
先看一份**通用核心代码**——它综合了题解的思路，把所有步骤“串”成了完整的程序，帮你建立整体框架！
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：本代码来自chen_zhe题解的核心逻辑，是“分支结构”的典型应用。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    #include <algorithm> // 用于std::max函数
    using namespace std;

    int main() {
        int a, b, c, d, M;
        cin >> a >> b >> c >> d >> M; // 输入总题数、做对题数、阈值
        
        int simple_score = 5 * c; // 简单题得分：5分/题
        int hard_score = 0;       // 困难题得分初始化为0
        
        // 分支判断：简单题得分是否超过M？
        if (simple_score > M) {
            // 超过：困难题答对得20分，答错扣20分
            hard_score = 20 * d - 20 * (b - d);
        } else {
            // 没超过：困难题答对不得分，答错扣20分
            hard_score = -20 * (b - d);
        }
        
        int total = simple_score + hard_score; // 计算总分
        cout << max(total, 0) << endl;         // 总分不能为负
        
        return 0;
    }
    ```
* **代码解读概要**：  
  1. **输入部分**：读取5个变量（简单题总数a、困难题总数b、简单题对题数c、困难题对题数d、阈值M）；  
  2. **简单题计算**：直接用“5×c”算出得分；  
  3. **分支判断**：用`if (simple_score > M)`决定困难题的计算方式；  
  4. **总分计算**：把简单题和困难题得分相加，再用`max(total, 0)`确保总分非负；  
  5. **输出结果**：打印最终得分。

接下来看题解中**最核心的“分支判断”片段**——这是整个程序的“大脑”！
</code_intro_selected>

**题解一：来源：chen_zhe**
* **亮点**：用最简洁的`if-else`覆盖了所有情况，没有多余的逻辑。
* **核心代码片段**：
    ```cpp
    int simple_score = 5 * c;
    int hard_score = 0;
    if (simple_score > M) {
        hard_score = 20 * d - 20 * (b - d);
    } else {
        hard_score = -20 * (b - d);
    }
    ```
* **代码解读**：  
  > 1. 先算`simple_score`（简单题得分）——这是“开关”的“触发条件”；  
  > 2. 如果`simple_score > M`（开关开）：困难题得分=答对的分（20×d）减去答错的分（20×(b-d)）；  
  > 3. 否则（开关关）：困难题得分=只扣答错的分（-20×(b-d)）——答对的分“被取消”了！  
  > 比如样例2中，`simple_score=35 ≤ M=40`，所以`hard_score=-20×(5-4)=-20`，总分35-20=15，正好是样例输出！
* 💡 **学习笔记**：分支结构的关键是“找对触发条件”——这里的条件是“simple_score > M”，而不是“<”或“≤”，千万别搞反！

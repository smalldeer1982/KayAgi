## 题目描述

香港天文台选定了 $8$ 个气象站。在台风影响期间，香港天文台会根据这些气象站的风速观测数据决定发布哪一个等级的预警。

- 因为已经受到台风影响，因此必然会发布 $1$ 号戒备信号；
- 若有 $4$ 个气象站的持续风力达到或超过 $41$ 千米/小时，则发布 $3$ 号强风信号；
- 若有 $4$ 个气象站的持续风力达到或超过 $63$ 千米/小时，则发布 $8$ 号烈风信号；
- 若有 $1$ 个气象站的持续风力达到或超过 $118$ 千米/小时，则发布 $10$ 号飓风信号；
- 多个条件同时满足，则发布其中最高等级的信号。

现在给定这些气象站的风力数据，请计算出香港天文台应当发布什么警告信号。

## 说明/提示

**【样例解释】**

第一组样例中，第一个气象观测站观测到了超过 $118$ 千米/小时的风速，因此应当发布 $10$ 号飓风信号。此为 2023 年超强台风苏拉影响香港的真实风速数据。

第二组样例中，只有 $3$ 个气象观测站的持续风力达到或超过 $41$ 千米/小时，因此只符合 $1$ 号戒备信号的标准。此为 2023 年强台风小犬影响香港的真实风速数据。

第三组样例中，恰好有 $4$ 个（第 $1,2,3,5$ 个）气象观测站的持续风力达到或超过 $63$ 千米/小时，因此符合 $8$ 号烈风信号的标准。此为 2008 年台风鹦鹉登陆香港的真实风速数据。

**【数据范围】**

对于所有风速观测数据 $x$，满足 $1 \leq x \leq 300$。

✨ **本题主要考察**：模拟应用（结合循环结构）

🗣️ **初步分析**：  
解决“风球”问题的核心是**模拟**——就像你帮老师统计班级里“数学考80分以上”“90分以上”的人数，再根据规则选出最高的奖励等级。模拟的核心思想是“按规则一步步还原现实逻辑”：我们需要先统计3个关键阈值（41、63、118千米/小时）的满足数量，再按“最高等级优先”的规则判断最终信号。  

在本题中，模拟的具体步骤是：  
1. 用循环读入8个风速数据，同步统计`cnt41`（≥41的数量）、`cnt63`（≥63的数量）、`cnt118`（≥118的数量）；  
2. 按**从高到低**的优先级判断：先看是否满足10号信号（`cnt118≥1`），再看8号（`cnt63≥4`），接着是3号（`cnt41≥4`），最后是1号。  

**核心难点**：优先级的处理——如果先判断低等级条件（比如先看3号），会导致高等级条件被覆盖（比如明明满足10号，却输出3号）。解决方案是**“从高到低”依次判断**，一旦满足某条件就直接输出，不再检查更低等级。

我们可以设计一个8位像素风的“气象站统计模拟器”：

- 判断阶段，屏幕中央会依次弹出“检查10号信号”“检查8号信号”的像素文字，满足条件时对应的信号等级（比如“10号”）会用大字体高亮，并播放“胜利”音效；

## 3. 核心难点辨析与解题策略

在解决这个问题的过程中，我们通常会遇到以下几个核心难点，结合优质题解的做法，我为大家总结了应对策略：

1. **难点1：正确统计多个阈值的满足数量**  
   * **分析**：需要同时统计三个不同阈值的数量，容易混淆每个计数器的条件（比如把`cnt63`的条件写成`x>63`而不是`x≥63`）。  
   * **策略**：明确每个计数器的定义（`cnt41`是≥41的数量，`cnt63`是≥63的数量，`cnt118`是≥118的数量），在循环中用三个`if`语句分别判断——注意是“分别”，而不是“else if”，因为一个风速可能同时满足多个条件（比如120同时满足≥41、≥63、≥118）。  
   * 💡 **学习笔记**：统计多个重叠条件的数量时，要用独立的`if`语句，不能用分支结构。

2. **难点2：处理多个条件同时满足的优先级**  
   * **分析**：题目要求“多个条件满足时输出最高等级”，如果判断顺序错误（比如先判断3号再判断10号），会得到错误结果。  
   * **策略**：按信号等级从高到低判断——先检查10号（最高），再检查8号，接着是3号，最后是1号。一旦满足某条件，立即输出结果并结束程序，避免后续低等级条件的干扰。  
   * 💡 **学习笔记**：优先级问题的解决关键是“从高到低”的判断顺序。

3. **难点3：循环结构的正确使用**  
   * **分析**：需要读入8个数据，容易写错循环的次数（比如写成`i<8`而不是`i<=8`）或者忘记初始化计数器。  
   * **策略**：循环变量从1到8（共8次），确保读入所有数据；计数器变量在循环前初始化为0（比如`int cnt41=0, cnt63=0, cnt118=0;`）。  
   * 💡 **学习笔记**：循环前初始化变量，循环次数要准确对应输入数量。


### ✨ 解题技巧总结
- **技巧A：变量命名要“见名知意”**：用`cnt41`而不是`a`，直接对应题目中的阈值，降低代码理解成本。  
- **技巧B：优先级判断用“从高到低”顺序**：避免逻辑错误的核心技巧。  
- **技巧C：循环处理重复输入**：遇到“读入n个数据”的问题，优先用循环简化代码。  


## 4. C++核心代码实现赏析

在深入分析具体题解的精妙片段之前，让我们先来看一个能够完整解决本题的通用核心C++实现参考。这有助于我们对整体解题框架有一个把握。

**本题通用核心C++实现参考**
* **说明**：本代码综合了优质题解的思路，提供一个清晰且完整的核心实现，覆盖输入、统计、判断、输出的全流程。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    using namespace std;

    int main() {
        int cnt41 = 0, cnt63 = 0, cnt118 = 0;
        // 读入8个风速数据并统计
        for (int i = 1; i <= 8; ++i) {
            int x;
            cin >> x;
            if (x >= 41) cnt41++;
            if (x >= 63) cnt63++;
            if (x >= 118) cnt118++;
        }
        // 按优先级判断输出
        if (cnt118 >= 1) {
            cout << 10 << endl;
        } else if (cnt63 >= 4) {
            cout << 8 << endl;
        } else if (cnt41 >= 4) {
            cout << 3 << endl;
        } else {
            cout << 1 << endl;
        }
        return 0;
    }
    ```
* **代码解读概要**：  
  代码分为两部分：第一部分用循环读入8个风速数据，同步统计三个计数器的数值；第二部分按“10号→8号→3号→1号”的优先级判断，输出最高等级的信号。关键在于循环中的三个`if`语句（统计）和`if-else if`结构（优先级判断）。

    ```cpp

    for (int i=1;i<=8;i++) {
        int x; cin >> x;
        if (x>=41) cnt41++;
        if (x>=63) cnt63++;

    }
    if (cnt118>=1) cout << 10;
    else if (cnt63>=4) cout << 8;
    else if (cnt41>=4) cout << 3;
    else cout << 1;
    ```
* **代码解读**：  
  这段代码是题解的核心。首先，三个计数器初始化为0，准备统计；然后用`for`循环读入8个数据，每个数据进来后，用三个`if`语句分别检查是否满足三个阈值，满足就增加对应的计数器——比如`x=126`会同时让`cnt41`、`cnt63`、`cnt118`各加1。接下来的`if-else if`结构按优先级判断：先看`cnt118`是否≥1（满足则输出10），否则看`cnt63`是否≥4（输出8），依此类推。  
  这里有个小细节：为什么用`if`而不是`else if`统计？因为一个风速可能同时满足多个条件（比如126≥41、≥63、≥118），需要同时统计三个计数器，而`else if`会导致只统计一个，这会出错！
* 💡 **学习笔记**：统计多个重叠条件时，用独立的`if`语句；判断优先级时，用`if-else if`结构。

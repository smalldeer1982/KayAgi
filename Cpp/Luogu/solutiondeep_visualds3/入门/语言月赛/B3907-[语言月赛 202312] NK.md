# 题目信息

# [语言月赛 202312] NK

## 题目描述

给定两个正整数 $N, K$，请你统计符合以下条件的正整数 $x$ 的数量：

- $1 \leq x \leq N ^ N$。
- $(x \bmod K)$ 是 $N$ 的倍数。
- $x$ 的个位是 $N$。

$x \bmod K$ 代表 $x$ 除以 $K$ 的余数，例如 $7 \bmod 3 = 1$。

## 说明/提示

### 数据规模与约定

- 对于 $30\%$ 的数据，$1 \leq N, K \leq 6$。
- 对于 $100\%$ 的数据，$1 \leq N \leq 9$，$1 \leq K \leq 10 ^ 6$。

## 样例 #1

### 输入

```
4 6```

### 输出

```
17```

## 样例 #2

### 输入

```
7 6```

### 输出

```
0```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：`枚举（优化后的高效枚举）`

🗣️ **初步分析**：  
解决这道题的核心思路是**“枚举”**——就像在一堆苹果里挑红苹果，我们需要逐一检查每个数是否符合条件。但直接“全部翻一遍”会很慢，所以要学会“挑着翻”：题目要求x的个位是N，而个位是N的数每10个才出现一次（比如N=4时，4、14、24……）。这就像找红苹果时，先把绿苹果直接跳过，只看红苹果，大大减少了工作量。  

题解的核心思路是：  
1. 先计算出上限`N^N`（比如N=4时是4^4=256）；  
2. 从x=N开始（第一个个位是N的数），每次加10枚举（跳过所有个位不是N的数）；  
3. 检查每个x是否满足“x mod K是N的倍数”（即`(x%K)%N == 0`）。  

**核心难点**：直接枚举所有数会超时（比如N=9时，`9^9≈3.8亿`次循环）。  
**解决方案**：利用“个位是N”的条件，将循环次数从`N^N`减少到`N^N / 10`（比如N=9时约3800万次，完全能通过）。  

**可视化设计思路**：

- 符合条件的数字会“闪烁金光”，并弹出“+1”的计数动画，伴随“叮”的提示音；

## 3. 核心难点辨析与解题策略

在解决这个问题的过程中，我们通常会遇到以下一些关键点或难点。结合优质题解的共性，我为大家提炼了几个核心的思考方向和策略：


### 1. 难点1：直接枚举的时间复杂度太高  
**问题**：当N=9时，`9^9≈3.8亿`次循环，电脑根本跑不完。  
**解决策略**：找“条件”缩小枚举范围——题目要求x的个位是N，而个位是N的数每10个出现一次。所以我们可以从x=N开始，每次加10，直接跳过所有个位不是N的数，循环次数瞬间减少到原来的1/10。  


### 2. 难点2：正确理解“x mod K是N的倍数”的条件  
**问题**：很多同学会误以为“x mod K等于N”，但题目是“x mod K是N的倍数”（比如N=4时，x mod K可以是0、4、8等）。  
**解决策略**：用数学表达式拆解条件——`(x % K) % N == 0`。比如x=14，K=6，N=4时：14%6=2，2%4=2≠0，不符合；x=24时，24%6=0，0%4=0，符合条件。  


### 3. 难点3：计算`N^N`时的溢出问题  
**问题**：当N=9时，`9^9=387420489`，用int类型（最大约21亿）没问题，但如果N更大（比如10），int就会溢出。不过本题N≤9，所以没问题。  
**解决策略**：对于更大的N，可以用`long long`类型存储`N^N`（比如`long long nn = 1;`）。  


### ✨ 解题技巧总结  
- **条件筛选法**：遇到枚举问题时，先找“必须满足的条件”（比如本题的“个位是N”），用条件缩小枚举范围，避免做无用功；  
- **数学表达式拆解**：把题目中的文字条件转化为代码能理解的数学式子（比如“x mod K是N的倍数”→`(x%K)%N==0`）；  
- **变量名规范**：用`nn`表示`N^N`，`cnt`表示计数，让代码一看就懂，减少调试时间。  


## 4. C++核心代码实现赏析

在深入分析具体题解的精妙片段之前，让我们先来看一个能够完整解决本题的通用核心C++实现参考。这有助于我们对整体解题框架有一个把握。


### 本题通用核心C++实现参考  
* **说明**：本代码来自Maxmilite的题解，是优化后的核心实现，逻辑清晰且高效。  
* **完整核心代码**：  
  ```cpp
  #include <iostream>
  using namespace std;

  int main() {
      int N, K;
      cin >> N >> K;

      // 计算N的N次方（nn = N^N）
      long long nn = 1; // 用long long避免溢出（N=9时nn=387420489，int足够，但用long long更安全）
      for (int i = 1; i <= N; ++i) {
          nn *= N;
      }

      int cnt = 0;
      // 从N开始，每次加10枚举（个位是N的数）
      for (long long x = N; x <= nn; x += 10) {
          if ((x % K) % N == 0) {
              cnt++;
          }
      }

      cout << cnt << endl;
      return 0;
  }
  ```
* **代码解读概要**：  
  1. **输入处理**：读取N和K；  
  2. **计算上限**：用循环算出`N^N`（注意用`long long`避免溢出）；  
  3. **优化枚举**：从N开始，每次加10，只检查个位是N的数；  
  4. **条件判断**：检查`(x%K)%N == 0`，符合条件就计数；  
  5. **输出结果**：打印符合条件的数的数量。  


### 题解一：(来源：Maxmilite)  
* **亮点**：用“步长10”的优化，把循环次数减少了90%，完美解决超时问题。  
* **核心代码片段**：  
  ```cpp
  // 从N开始，每次加10枚举
  for (long long x = N; x <= nn; x += 10) {
      if ((x % K) % N == 0) {
          cnt++;
      }
  }
  ```
* **代码解读**：  
  这段代码是整个题解的“灵魂”！为什么要从N开始？因为N是第一个个位是N的正整数（比如N=4时，第一个数是4）。为什么每次加10？因为个位每10个数循环一次（4→14→24→…），加10就能跳过所有个位不是N的数。条件`(x%K)%N == 0`则是判断“x mod K是N的倍数”——比如x=24，K=6，N=4时，24%6=0，0%4=0，符合条件，cnt加1。  
* 💡 **学习笔记**：枚举不是“蛮干”，而是“巧干”——用条件缩小范围，才能解决大数据的问题。

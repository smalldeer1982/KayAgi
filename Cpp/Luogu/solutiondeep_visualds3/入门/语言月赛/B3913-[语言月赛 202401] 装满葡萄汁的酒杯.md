# 题目信息

# [语言月赛 202401] 装满葡萄汁的酒杯

## 题目描述

小 H 有五个酒杯可供选择，它们的容量分别为 $100\text{mL}$、$150\text{mL}$、$300\text{mL}$、$400\text{mL}$ 和 $1000\text{mL}$。

现在，小 H 想要在这五个酒杯中选择一个来喝葡萄汁。小 H 打算喝 $a\text{ mL}(a \leq 1000)$ 的葡萄汁。请问，小 H 至少需要选择多大容量的酒杯，才能装下 $a\text{ mL}$ 的葡萄汁（葡萄汁不能溢出）。

## 说明/提示

### 样例解释 #1

如果要装 $50\text{ mL}$ 的葡萄汁，那么用 $100\text{ mL}$ 的酒杯就可以了。 

### 数据规模与约定

对于 $100\%$ 的数据，满足 $1 \leq a \leq 1000$。



## 样例 #1

### 输入

```
50```

### 输出

```
100```

## 样例 #2

### 输入

```
112```

### 输出

```
150```

## 样例 #3

### 输入

```
300```

### 输出

```
300```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：编程技巧应用（侧重基础分支结构的运用）

🗣️ **初步分析**：
解决这道题的核心，就像“给葡萄汁找合适的杯子”——我们有5个固定大小的杯子（100、150、300、400、1000mL），要选**最小的那个能装下a毫升葡萄汁的杯子**。这时候，**分支结构**（if-else语句）就是我们的“工具”：按杯子从小到大的顺序，依次判断“a是不是能装进这个杯子”，第一个能装下的，就是答案！

比如，a=50时，第一个杯子（100mL）就能装下，直接选它；a=112时，100装不下，但下一个150能装下，所以选150。这个过程就像“排队选杯子”，按顺序试，先到先得～

**核心算法流程**：读取a的值→按100→150→300→400→1000的顺序判断→输出第一个能装下a的容量。  
**可视化设计思路**：我们会用8位像素风模拟“选杯子”的过程——屏幕上显示5个像素化的酒杯（从小到大排列），a的值用闪烁的像素数字显示；每判断一个酒杯，对应的酒杯会“亮起来”，如果装不下就“变暗”，装下的话直接“闪烁+播放胜利音效”。这样你能直观看到“判断的顺序”和“答案的来源”～

## 3. 核心难点辨析与解题策略

<difficulty_intro>
这道题虽然简单，但新手容易在“细节”上栽跟头。我帮你提炼了3个核心难点，以及对应的解决方法～
</difficulty_intro>

1.  **难点1：为什么要按“从小到大”的顺序判断？**
    * **分析**：如果反过来按“从大到小”判断（比如先看1000，再看400），会选到**最大的能装下的杯子**，而题目要的是“最小的”。比如a=50，从大到小会选1000，但正确答案是100。所以顺序错了，结果就错！
    * 💡 **学习笔记**：解决“找最小满足条件的元素”问题，一定要按**从小到大**的顺序遍历。

2.  **难点2：边界条件怎么处理？**
    * **分析**：比如a=100，刚好等于第一个杯子的容量，这时候要选100；a=101，超过100，要选下一个150。这时候if条件里的“<=”就很关键——如果写成“<”，a=100就会错误地跳到150！
    * 💡 **学习笔记**：遇到“刚好等于”的情况，一定要用“<=”而不是“<”，否则会漏掉边界值。

3.  **难点3：如何理解“至少需要多大容量”？**
    * **分析**：“至少”意味着“刚好能装下，不能更小”。比如a=112，100装不下，150能装下，所以150是“最小的能装下的”。本质是找**第一个大于等于a的容量**。
    * 💡 **学习笔记**：“至少需要X”=“第一个≥a的X”，记住这个公式，很多类似问题都能解决！

### ✨ 解题技巧总结
- **技巧A：顺序遍历找答案**：找“最小满足条件的元素”，按从小到大顺序遍历，第一个满足的就是答案。
- **技巧B：边界条件用“<=”**：处理“刚好等于”的情况，避免遗漏。
- **技巧C：把问题翻译成“判断语句”**：比如“a≤100”对应“选100mL”，直接把题目要求写成代码逻辑，不用绕弯子。


## 4. C++核心代码实现赏析

<code_intro_overall>
先看一份通用的核心代码，帮你建立整体框架～

**本题通用核心C++实现参考**
* **说明**：本代码综合了题解的核心思路，用最简洁的分支结构解决问题，适合新手直接模仿。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    using namespace std;

    int main() {
        int a;
        cin >> a;  // 读取要喝的葡萄汁量
        if (a <= 100) {
            cout << 100 << endl;
        } else if (a <= 150) {
            cout << 150 << endl;
        } else if (a <= 300) {
            cout << 300 << endl;
        } else if (a <= 400) {
            cout << 400 << endl;
        } else {
            cout << 1000 << endl;
        }
        return 0;
    }
    ```
* **代码解读概要**：代码的逻辑和“选杯子”的过程完全一致——先读入a，然后按100→150→300→400→1000的顺序判断：如果a≤100，直接输出100；否则看a≤150吗？是的话输出150……直到最后，a超过400的话，直接输出1000。整个过程没有多余的步骤，每一行代码都对应一个“判断逻辑”。

**题解一：(来源：Maxmilite)**
* **亮点**：用“连续的else if”替代“嵌套if”，让代码更扁平、更好读！
* **核心代码片段**：
    ```cpp
    if (a <= 100) cout << 100 << endl;
    else if (a <= 150) cout << 150 << endl;
    else if (a <= 300) cout << 300 << endl;
    else if (a <= 400) cout << 400 << endl;
    else cout << 1000 << endl;
    ```
* **代码解读**：这段代码的妙处在于“else if的连续使用”——因为前面的条件已经排除了“a≤100”的情况，所以第二个条件“a≤150”其实等价于“100<a≤150”，不用再写多余的判断！比如，当a=112时，第一个if（a≤100）不成立，进入else if（a≤150），成立，输出150。这样写不仅省代码，还能让逻辑更清晰～
* 💡 **学习笔记**：连续的else if可以帮你“缩小判断范围”，避免重复写条件，是分支结构的“进阶技巧”！

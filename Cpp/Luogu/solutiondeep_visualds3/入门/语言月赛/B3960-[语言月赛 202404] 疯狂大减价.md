# 题目信息

# [语言月赛 202404] 疯狂大减价

## 题目背景

gza 去超市，超市大减价！

## 题目描述

超市提供了两张优惠券：

- 第一张：满 $100$ 元减 $20$ 元；
- 第二张：满 $200$ 元减 $50$ 元。

gza 可以以任意顺序使用优惠券。先使用的优惠券会对后使用的优惠券产生影响。例如，购买了 $210$ 元的商品，先使用第一张优惠券后，价格会变为 $190$ 元，此时，无法使用第二张优惠券。

给定初始需支付金额 $n$，求最终 gza 需要支付的金额的最小值。

## 说明/提示

### 数据规模与约定

对于 $100\%$ 的数据，$1 \leq n \leq 1000$。

## 样例 #1

### 输入

```
370
```

### 输出

```
300
```

## 样例 #2

### 输入

```
120```

### 输出

```
100```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：编程技巧应用（条件判断的逻辑设计）

🗣️ **初步分析**：
解决这道题的关键，就像我们去超市买东西时“看金额选优惠券”——不同的金额对应不同的优惠组合，我们需要用**条件判断**（if-else语句）把这些“规则”翻译成代码。简单来说，条件判断就是“如果满足A情况就做X，满足B情况就做Y”，像妈妈说“如果下雨就带伞，没下雨就戴帽子”一样直白。

在本题中，我们需要根据金额`n`的大小，选择最优的优惠券组合：
- 若`n`不足100元，没法用任何优惠券；
- 若`n`在100-199元之间，只能用“满100减20”的优惠券；
- 若`n`超过200元，两张优惠券都能用上（不管顺序如何，最优结果都是减70元）。

**核心算法流程与可视化设计**：
我们会把金额分成三个“区间”，每个区间对应固定的优惠。可视化时，我们会用**像素风格的超市场景**展示：金额数字用彩色像素块组成，当`n`进入某个区间时，对应的优惠券会闪烁（比如`n≥200`时，红、蓝两张优惠券轮流亮），然后金额数字“唰”地减少对应金额，伴随“叮”的像素音效——就像游戏里“捡金币”一样直观！

## 3. 核心难点辨析与解题策略

<difficulty_intro>
解决这道题时，大家容易卡壳的地方其实是“如何确定每个区间的最优优惠”和“条件判断的顺序”。我们结合题解的思路，拆解成3个关键点：
</difficulty_intro>

1. **关键点1：如何确定每个金额区间的最优优惠？**
   * **分析**：很多同学会想“优惠券的顺序会不会影响结果？”比如`n=210`，先用完100减20的券，剩下的190不够用200减50的券；但先用完200减50的券，剩下的160还能再用100减20的券——总优惠是70元！所以对于`n≥200`的情况，不管顺序如何，**最优结果都是减70元**。
   * 💡 **学习笔记**：遇到“多种组合”的问题，先枚举所有可能的情况，再找最小值！

2. **关键点2：条件判断的顺序为什么要“从小到大”？**
   * **分析**：如果先判断`n≥200`，再判断`n≤199`，会导致`n=150`的情况被错误归类到“减70”的分支吗？不会，但“从小到大”的顺序更符合我们的思维习惯，也更不容易出错——就像“先看有没有100元，再看有没有200元”。
   * 💡 **学习笔记**：条件判断的顺序要“从特殊到一般”或“从小到大”，避免逻辑漏洞！

3. **关键点3：如何验证结果是否正确？**
   * **分析**：题解里用了样例测试——输入370输出300，输入120输出100，这能快速验证代码的正确性。如果自己写代码，还可以测试边界情况（比如n=99、n=100、n=199、n=200）。
   * 💡 **学习笔记**：写代码前先想“边界情况”，写完后用样例和边界值测试！

### ✨ 解题技巧总结
- **拆解问题**：把复杂的“优惠券组合”拆成三个简单的金额区间，每个区间对应固定的优惠；
- **条件判断顺序**：从最小的区间到最大的区间，逻辑更清晰；
- **样例验证**：用题目给的样例和边界值测试代码，确保没错；
- **代码简洁**：能用基础语句解决的问题，不用复杂结构！


## 4. C++核心代码实现赏析

<code_intro_overall>
先看一个**通用核心实现**——它来自题解一，逻辑清晰、代码简洁，是解决本题的“标准模板”！
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：此代码来自Maxmilite的题解，是“条件判断解决优惠问题”的典型实现，逻辑直接、可读性极强。
* **完整核心代码**：
  ```cpp
  #include <iostream>
  using namespace std;

  int main() {
      int n;
      cin >> n;
      if (n <= 99) {
          cout << n << endl;
      } else if (n <= 199) {
          cout << n - 20 << endl;
      } else {
          cout << n - 70 << endl;
      }
      return 0;
  }
  ```
* **代码解读概要**：
  > 代码分三步：①读取输入的金额`n`；②用`if-else`判断`n`属于哪个区间；③输出对应的最小支付金额。比如输入370，进入`else`分支，输出`370-70=300`；输入120，进入第二个分支，输出`120-20=100`。

<code_intro_selected>
接下来，我们剖析题解一的核心代码——它的亮点就是“用最少的代码解决问题”！
</code_intro_selected>

**题解一：来源——Maxmilite**
* **亮点**：用极简的条件判断覆盖所有情况，没有冗余代码，变量名和逻辑都超容易理解。
* **核心代码片段**：
  ```cpp
  if (n <= 99) {
      cout << n << endl;
  } else if (n <= 199) {
      cout << n - 20 << endl;
  } else {
      cout << n - 70 << endl;
  }
  ```
* **代码解读**：
  > 这段代码是整个题解的核心！我们一步步看：
  > 1. 第一个`if`判断`n≤99`：如果金额不够100元，直接输出原价（没法用优惠券）；
  > 2. 第二个`else if`判断`n≤199`：如果金额在100-199之间，只能用“满100减20”的优惠券，输出`n-20`；
  > 3. 最后一个`else`：如果金额超过200元，两张优惠券都能用，总优惠70元，输出`n-70`。
  > 比如输入210，会进入`else`分支，输出`210-70=140`——这是最优结果！
* 💡 **学习笔记**：条件判断的“分支”要覆盖所有可能的情况，不要遗漏任何一种金额！

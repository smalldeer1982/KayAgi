## 题目描述

ZngivaeL 去中考。他有 4 门科目需要考试，每门都有一个评级（$\texttt{ABCD}$）。

Zngivael 有三种去向选择：重点高中、普通高中和职业高中。三所高中的录取难度由高到低依次为重点高中、普通高中、职业高中。

如果 Zngivael 的 4 门科目**全部**得到了 $\texttt{C}$ 及以上的评级（$\texttt{A, B, C}$），则可以选择前去**普通高中**，否则只能选择前去**职业高中**。

进一步地，进入**重点高中**需要 Zngivael 所有科目的评级都是 $\texttt{A}$ 或 $\texttt{B}$，以及**至少有一门科目**的评级是 $\texttt{A}$。

给定可爱 ZngivaeL 的每门课的评级，询问 ZngivaeL 能上的录取难度最高的高中。

## 说明/提示

### 样例解释

### 数据规模与约定

保证输入的四个字符均为大写字母 $\texttt{A, B, C, D}$ 其中之一。

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：模拟（按规则逐步验证条件）

🗣️ **初步分析**：
“模拟”是编程中最基础也最实用的技巧——**照着题目给的规则，一步一步执行**。比如妈妈让你“先写作业，再看电视”，你就按这个顺序做；这道题里，我们要“先检查有没有D，再检查有没有C，最后检查有没有A”，完全按照题目给的录取规则来。

### 核心逻辑梳理
我们的目标是找到**难度最高**的高中，所以判断顺序要“从低到高排除”：
1. **职业高中**：只要有一门是D→直接输出“Never give up.”；
2. **普通高中**：没有D，但有至少一门C→输出“This is ok.”；
3. **重点高中**：没有D也没有C（全A/B），且至少有一门A→输出“I'm so happy.”。

## 3. 核心难点辨析与解题策略

解决这类“条件判断”问题，3个最容易卡壳的点，我帮你总结了对策：

### 1. 难点1：条件的优先级——先检查哪个？
**问题**：为什么先查D，不是先查A？  
**对策**：录取规则是“难度越高，条件越严”。比如重点高中需要“全A/B+有A”，而职业高中只需要“有D”。所以要**从“最容易满足的低难度条件”开始**：先查D（职业），再查C（普通），最后查A（重点）。这样一旦满足低难度条件，直接输出结果，不用再查更高难度的。

### 2. 难点2：逻辑运算符——“或”和“与”要分清
**问题**：为什么查“有没有D”要用`x == 'D' || y == 'D'`？  
**对策**：`||`是“或”——只要有一个满足就为真（比如“有D”就是“只要一门是D”）；`&&`是“与”——所有都满足才为真（比如“全A/B”是“四门都不是C/D”）。题解里用`||`查“有没有某类评级”，用反向思维（没有D=全C及以上）简化了代码。

### 3. 难点3：边界条件——比如全B的情况
**问题**：输入“BBBB”为什么输出“This is ok.”？  
**对策**：重点高中需要“全A/B+至少一个A”，全B满足“全A/B”但不满足“有A”，所以只能上普通。解决这类问题的关键是**列全测试用例**：比如有D（ABCD）、有C（CCCA）、全A（AAAA）、全B（BBBB），确保代码覆盖所有情况。

### ✨ 解题技巧总结
- **按规则顺序判断**：从低难度到高难度排除，避免多余计算；
- **用逻辑运算符简化代码**：`||`查“有没有某类元素”，`&&`查“所有元素都满足”；
- **测试用例全覆盖**：写出所有可能的输入，验证代码是否正确。


## 4. C++核心代码实现赏析

### 本题通用核心C++实现参考
* **说明**：本代码综合了题解思路，用数组存储四门科目，更简洁高效，适合进阶学习。
* **完整核心代码**：
```cpp
#include <iostream>
using namespace std;

int main() {
    char score[4]; // 用数组存四门科目
    cin >> score[0] >> score[1] >> score[2] >> score[3];

    // 步骤1：检查有没有D（职业高中）
    bool hasD = false;
    for (int i = 0; i < 4; ++i) {
        if (score[i] == 'D') {
            hasD = true;
            break;
        }
    }
    if (hasD) {
        cout << "Never give up." << endl;
        return 0;
    }

    // 步骤2：检查有没有C（普通高中）
    bool hasC = false;
    for (int i = 0; i < 4; ++i) {
        if (score[i] == 'C') {
            hasC = true;
            break;
        }
    }
    if (hasC) {
        cout << "This is ok." << endl;
        return 0;
    }

    // 步骤3：检查有没有A（重点高中）
    bool hasA = false;
    for (int i = 0; i < 4; ++i) {
        if (score[i] == 'A') {
            hasA = true;
            break;
        }
    }
    cout << (hasA ? "I'm so happy." : "This is ok.") << endl;

    return 0;
}
```
* **代码解读概要**：
> 代码用数组`score`存储四门科目，通过三次遍历检查：
> 1. 第一次遍历查D→有则输出职业高中；
> 2. 第二次遍历查C→有则输出普通高中；
> 3. 第三次遍历查A→有则输出重点高中，否则输出普通高中。
> 数组的好处是**减少重复代码**（比如查D/C/A都用循环，不用写四次条件判断），适合处理多个同类数据。


### 题解一：来源：Maxmilite
* **亮点**：用单独变量存每门科目，条件判断直接明了，适合新手理解“每一步做什么”。
* **核心代码片段**：
```cpp
// 查有没有D
if (x == 'D' || y == 'D' || z == 'D' || w == 'D') {
    cout << "Never give up." << endl; 
    return 0;
}

// 查有没有C
if (x == 'C' || y == 'C' || z == 'C' || w == 'C') {
    cout << "This is ok." << endl; 
    return 0;
}

// 查有没有A
cout << (x == 'A' || y == 'A' || z == 'A' || w == 'A' ? "I'm so happy." : "This is ok.") << endl;
```
* **代码解读**：
> 这段代码是题解的“灵魂”——**每一行都对应一个录取条件**：
> 1. 第一部分：只要有一门是D，直接输出“Never give up.”（职业高中）；
> 2. 第二部分：如果有C，说明只能上普通，输出“This is ok.”；
> 3. 第三部分：用三元运算符（`? :`）简化判断——有A则重点，否则普通。
> 这里的关键是**条件顺序**：先查低难度的，一旦满足直接返回，避免多余计算。比如有D的话，不会再查C和A了。
* 💡 **学习笔记**：条件判断的顺序比逻辑更重要！要从“最容易满足的低难度条件”开始，这样代码更高效，也更符合人类逻辑。

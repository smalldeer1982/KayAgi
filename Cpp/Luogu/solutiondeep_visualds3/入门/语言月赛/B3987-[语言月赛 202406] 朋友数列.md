# 题目信息

# [语言月赛 202406] 朋友数列

## 题目描述

小 K 喜欢数列。设数列 $a,b$ 的长度均为 $n$，$a_i$ 表示数列 $a$ 中第 $i(1 \le i \le n)$ 个数字，$b_i$ 表示数列 $b$ 中第 $i(1 \le i\le n)$ 个数字。

如果对于所有的 $1 \le i \le n$，$a_i+b_i$ 的值均相同，并且满足 $-10^9 \le a_i,b_i \le 10^9$，则小 K 称它们是一对朋友数列。

现在，小 K 随手写下了 $n$ 和数列 $a$，小 K 希望你能帮助小 K 找到数列 $a$ 的一个朋友数列。

## 说明/提示

### 数据规模与约定

- 对于 $20\%$ 的测试数据，$n=1$；
- 对于 $100\%$ 的测试数据，$1 \le n \le 10^5$，$|a_i| \le 10^9$。

## 样例 #1

### 输入

```
5
1 2 3 4 5
```

### 输出

```
5 4 3 2 1
```

## 3. 核心难点辨析与解题策略

<difficulty_intro>
这道题看似简单，但刚接触时可能会卡在3个关键点上——我们一起拆解它们！
</difficulty_intro>

1.  **关键点1：理解“朋友数列”的条件**  
    * **难点**：容易误解成“所有`a_i + b_i`的和相等”（比如总和相同），但其实是**每个位置的和都相同**！  
    * **策略**：把条件写成数学式——`a₁+b₁ = a₂+b₂ = … = aₙ+bₙ = S`，这样立刻就能想到`b_i = S - a_i`。  
    * 💡 **学习笔记**：读题时把文字转成“数学公式”，能帮你快速找到解题方向！

2.  **关键点2：选一个“安全”的S**  
    * **难点**：担心选的`S`会让`b_i`超出范围（比如`|b_i| > 1e9`）。  
    * **策略**：选`S=0`！因为`|a_i| ≤ 1e9`，所以`b_i = -a_i`的绝对值也不会超过`1e9`，完美满足题目要求～  
    * 💡 **学习笔记**：选“0”“1”这种“中性数”，往往是构造题的“万能钥匙”！

3.  **关键点3：处理大规模数据（n≤1e5）**  
    * **难点**：如果用`cin`/`cout`不优化，可能会超时？  
    * **策略**：在C++里，可以加`ios::sync_with_stdio(false); cin.tie(0);`来加速输入输出（不过本题数据量不大，直接用也没问题）。  
    * 💡 **学习笔记**：循环处理1e5个数据时，要注意“输入输出效率”！

## 4. C++核心代码实现赏析

<code_intro_overall>
这道题的代码超简洁——我们先看一个**通用核心实现**，再拆解它的“巧妙之处”！
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：本代码综合了题解的核心思路，用最简洁的方式实现“朋友数列”的构造。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    using namespace std;

    int main() {
        ios::sync_with_stdio(false); // 加速输入输出（可选，处理大数据时推荐）
        cin.tie(0);
        
        int n;
        cin >> n;
        for (int i = 0; i < n; ++i) {
            long long a; // 用long long防止溢出（a_i可以到1e9，-a_i也不会超）
            cin >> a;
            cout << -a << " ";
        }
        return 0;
    }
    ```
* **代码解读概要**：
    > 代码分3步：① 加速输入输出（处理大数据时更快）；② 读入数列长度`n`；③ 循环`n`次——每次读一个`a_i`，输出它的相反数`-a_i`。整个过程像“流水线”一样：进一个数，出一个结果！

    ```cpp

    for (int i = 0; i < n; ++i) {
        long long x;
        cin >> x;
        cout << -x << " ";
    }
    ```
* **代码解读**：
    > 这段代码的“聪明”之处在于**不存数组**——直接把输入的`x`当成`a_i`，立刻计算`-x`并输出。比如输入`3`，输出`-3`；输入`5`，输出`-5`……完全符合`b_i = -a_i`的公式！  
    > 为什么用`long long`？因为`a_i`可以是`1e9`，`int`的范围是`-2e9`到`2e9`（刚好够），但用`long long`更保险——避免极端情况溢出～
* 💡 **学习笔记**：能“边读边处理”的问题，就不要额外存数组！

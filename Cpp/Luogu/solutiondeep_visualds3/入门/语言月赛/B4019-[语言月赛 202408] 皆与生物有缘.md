## 题目背景

> 他们都是生物课代表。

## 题目描述

某次试卷共有 $n$ 道题，一份试卷由两名老师评阅，最终的得分为两名老师评分总分的平均值，向上取整。

现在，给出每名老师为风试卷每一题打的分数，请问风生物最终的得分是多少？

## 说明/提示

### 样例解释

- 第一名老师给的总评分为 $20 + 20 + 19 + 20 + 20 = 99$ 分；
- 第二名老师给的总评分为 $20 + 19 + 20 + 20 + 20 = 99$ 分。

最终得分为 $99$ 分。

### 数据规模与约定

对于 $30\%$ 的测试数据，$n=1$；  
对于 $100\%$ 的测试数据，$1 \le n \le 2\times 10^5$，每题的得分不超过 $10^4$。

## 样例 #1

### 输入

```
5
20 20 19 20 20
20 19 20 20 20
```

### 输出

```
99
```

🗣️ **初步分析**：  
解决这道题的关键就像“按 recipe 做蛋糕”——题目已经把步骤写得明明白白，我们只要**模拟**这个流程就行！模拟的核心是“照规则办事”：先算两个老师的总分和，再求平均并向上取整。

## 3. 核心难点辨析与解题策略

<difficulty_intro>
模拟题的难点往往藏在“细节”里——比如数据范围、类型转换。结合题解，我帮你总结了3个核心问题及解决办法：
</difficulty_intro>

1.  **难点1：总和溢出怎么办？**  
    * **分析**：当n=2e5、每题10000分时，两个老师的总分和是`2e5×10000×2=4e9`，而`int`类型的最大值只有约2e9（`2^31-1`），直接用`int`会溢出（比如总和变成负数）。  
    * **解决**：用`long long`类型存储总和（`long long`能存到约9e18，完全够）！  
    * 💡 **学习笔记**：遇到“大数累加”时，先算最大可能值，再选合适的类型！

2.  **难点2：怎么正确向上取整？**  
    * **分析**：比如总和是99，除以2是49.5，向上取整是50；总和是100，除以2是50，向上取整还是50。直接用整数除法（`总和/2`）会把小数部分砍掉（比如99/2=49），不对！  
    * **解决**：①用`ceil`函数（需要把总和转成`double`，比如`ceil((double)总和/2)`）；②用整数运算技巧（`(总和+1)/2`，比如99+1=100→100/2=50，100+1=101→101/2=50.5→整数除法得50？不对，等一下，`(总和+1)/2`在整数除法中是对的吗？比如总和是m，向上取整m/2等于`(m+1)//2`（`//`是整数除法）：当m是偶数，`(m+1)//2 = m/2 + 0.5`→整数除法得m/2；当m是奇数，`(m+1)//2 = (m+1)/2`，比如m=99→(99+1)/2=50，正确！哦对，这是更高效的方法，不用浮点运算~  
    * 💡 **学习笔记**：向上取整整数除法的小技巧：`(a + b - 1) // b`（比如a/m的向上取整是`(a+m-1)//m`）！

3.  **难点3：输入时怎么更高效？**  
    * **分析**：如果用数组存每道题的分数（比如`a[1..n]`），会浪费内存（n=2e5时，数组占`2e5×4=8e5`字节，虽然不多，但没必要）。  
    * **解决**：直接读入分数并累加！比如`for(int i=1; i<=n; i++) { cin>>x; S1+=x; }`，省内存又快~  
    * 💡 **学习笔记**：不需要存储中间结果的话，就直接处理！


### ✨ 解题技巧总结
- **技巧1：预判数据范围**：遇到累加问题，先算最大可能值，选`long long`避免溢出。  
- **技巧2：用整数运算替代浮点**：向上取整可以用`(总和+1)/2`，避免`ceil`的精度问题。  
- **技巧3：边输入边处理**：不需要存每道题的分数，直接累加更高效。


## 4. C++核心代码实现赏析

<code_intro_overall>
先看一个**通用核心实现**——它综合了题解的优点，解决了溢出问题，代码简洁清晰~
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：本代码解决了溢出问题，用`long long`存总和，并用`ceil`函数处理向上取整，是最典型的模拟实现。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    #include <cmath>  // 包含ceil函数
    using namespace std;

    int main() {
        int n;
        long long total = 0;  // 用long long避免溢出
        cin >> n;

        // 读入第一个老师的分数并累加
        for (int i = 0; i < n; ++i) {
            int x;
            cin >> x;
            total += x;
        }

        // 读入第二个老师的分数并累加
        for (int i = 0; i < n; ++i) {
            int x;
            cin >> x;
            total += x;
        }

        // 计算向上取整的平均得分
        cout << (int)ceil(total / 2.0) << endl;
        return 0;
    }
    ```
* **代码解读概要**：  
  代码分三步：①读入n；②用两个循环分别累加两个老师的分数（`total`是总和）；③用`ceil`函数计算`total/2`的向上取整，转成`int`输出。关键是`long long`类型的使用——这能避免总和溢出！

* **亮点**：直接读入分数并累加，不用数组，代码更简洁！
* **核心代码片段**：
    ```cpp
    long long S1 = 0, S2 = 0;  // 分别存两个老师的总分
    for(int i = 1, x; i <= n; i++) {
        cin >> x; S1 += x;  // 读一个加一个
    }
    for(int i = 1, x; i <= n; i++) {
        cin >> x; S2 += x;
    }
    cout << (int)ceil((double)(S1 + S2) / 2);
    ```
* **代码解读**：  
  这段代码用`S1`和`S2`分别存两个老师的总分，读入`x`后直接加到对应的变量里——完全不用数组！比如第一个循环，每读一个`x`就加到`S1`，这样省了`n`个`int`的内存。最后`S1+S2`是总和，转成`double`后用`ceil`取整，再转成`int`输出。  
  问：为什么要用`double`？因为`ceil`函数需要`double`类型的参数，否则`(S1+S2)/2`是整数除法，会丢掉小数部分！
* 💡 **学习笔记**：不需要存中间结果时，边读边加是最优选择！

> **点评**：这是很多初学者的“踩坑点”——比如`ceil(99.0)`返回`99.0`，如果直接输出会显示`99`吗？不一定！有些编译器会输出`99.0000`，所以必须转成`int`。

# [语言月赛 202409] 灵感

## 题目描述

这天，迅风在欣赏某地的美景时，灵感大作，在上午及下午**分别**写下了两篇文章，而且迅风很喜欢数文章的字数。

具体地，如果迅风在下午写下的文章的字数之和**严格大于**他在上午写下的文章的字数之和，则认定他的灵感随着时间的推移越积越多。

现在给出迅风所写的四篇文章的字数，请你回答他的灵感是否随着时间的推移越积越多。

## 说明/提示

### 样例解释 1
第一个样例中，迅风在上午写下的文章的字数之和为 $3+2=5$，在下午写下的文章的字数之和为 $4+5=9$，由于 $9>5$，所以认定他的灵感随着时间的推移越积越多。
### 样例解释 2
第二个样例中，迅风在上午写下的文章的字数之和为 $3+3=6$，在下午写下的文章的字数之和为 $4+2=6$，由于 $6=6$，所以认定他的灵感不会随着时间的推移越积越多。

### 数据范围

对于 $100\%$ 的数据，$1\le a,b,c,d\le100$。

## 样例 #1

### 输入

```
3 2 4 5```

### 输出

```
Yes```

## 样例 #2

### 输入

```
3 3 4 2```

### 输出

```
No```

## 3. 核心难点辨析与解题策略

### 核心难点辨析与解题策略
新手容易栽在“小细节”上，我们一一拆解：

1. **难点1：输入顺序不能乱**  
   * 问题：若把a、c、b、d的顺序读错，总和就会错（比如上午总和变成a+c）。  
   * 解决：**记死顺序规则**——题目描述的顺序=输入顺序（上午1→上午2→下午1→下午2）。

2. **难点2：严格大于的条件要记牢**  
   * 问题：题目要求“下午严格大于上午”，等于的情况（比如6=6）要输出`No`。  
   * 解决：**条件写清楚**——用`>`而不是`≥`，可以用口诀记：“严格大于就是‘比你大一点’，等于不算！”

3. **难点3：总和计算别算错**  
   * 问题：比如把b+d当下午总和（应该是c+d）。  
   * 解决：**变量名要“见名知意”**——用`morning_sum`代替`x`，`afternoon_sum`代替`y`，一眼就知道变量对应什么。

### ✨ 解题技巧总结
- **技巧A**：变量名“见名知意”——复杂问题用长变量名（比如`total_morning`），简单问题可以简洁（比如`x`）；  
- **技巧B**：条件判断“抠字眼”——题目里的“严格大于”“至少”要对应正确的运算符（`>`、`≥`）；  
- **技巧C**：写完代码“模拟测试”——用样例1（3 2 4 5）和样例2（3 3 4 2）测试，快速发现错误。  


## 4. C++核心代码实现赏析

### 本题通用核心C++实现参考
* **说明**：本代码综合了题解思路，用“见名知意”的变量名，适合入门者理解。  
* **完整核心代码**：
```cpp
#include <iostream>
using namespace std;

int main() {
    int morning1, morning2, afternoon1, afternoon2;
    cin >> morning1 >> morning2 >> afternoon1 >> afternoon2;
    int total_morning = morning1 + morning2;
    int total_afternoon = afternoon1 + afternoon2;
    if (total_afternoon > total_morning) {
        cout << "Yes" << endl;
    } else {
        cout << "No" << endl;
    }
    return 0;
}
```
* **代码解读概要**：  
  1. 读入四个整数（上午两篇、下午两篇的字数）；  
  2. 计算上午总和`total_morning`和下午总和`total_afternoon`；  
  3. 判断“下午总和是否严格大于上午”，输出结果。  
  整个代码没有多余部分，完全围绕“解决问题”展开。

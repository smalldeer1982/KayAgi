# 题目信息

# [语言月赛 202411] Giants

## 题目描述

在《迪迪卫》中，一次魔法的**能量**和五个参数有关：能量类型 $type$；发起者的攻击力 $A$，魔法强度 $B$；承受者的物理抗性 $C$，魔法抗性 $D$。

**能量**的数值计算方法如下：

- 如果 $type=0$，则**能量**的数值为 $A+B$。
- 如果 $type=1$，则**能量**的数值为 $\max(A-C,0)+\max(B-D,0)$。

其中 $\max(x,y)$ 表示 $x,y$ 中的最大值。

现在给你这五个参数，请你计算本次魔法的**能量**的数值。

## 说明/提示

### 样例 #1 解释
能量类型为 $0$，数值为 $A+B=3+4=7$。

### 样例 #2 解释
能量类型为 $1$，数值为 $\max(A-C,0)+\max(B-D,0)=1+2=3$。

### 样例 #3 解释
能量类型为 $1$，数值为 $\max(A-C,0)+\max(B-D,0)=0+2=2$。

### 数据范围

对于 $100\%$ 的数据，$0\le A,B,C,D\le 1000,0\le type\le 1$。

对于 $40\%$ 的数据，$type=0$。

另有 $20\%$ 的数据，$B=D=0$。

## 样例 #1

### 输入

```
0 3 4 2 2
```

### 输出

```
7
```

## 样例 #2

### 输入

```
1 3 4 2 2
```

### 输出

```
3
```

## 样例 #3

### 输入

```
1 2 4 3 2
```

### 输出

```
2
```

## 3. 核心难点辨析与解题策略

<difficulty_intro>
这道题虽然简单，但新手容易在“细节”上栽跟头。结合题解的思路，我帮大家提炼了3个核心注意点和解决策略：
</difficulty_intro>

1.  **关键点1：准确理解公式逻辑**
    * **分析**：题目给了两种type的公式，很多同学会把type1的公式记混（比如漏加其中一个`max`，或者把A-C写成C-A）。解决办法很简单——**把公式“抄”到代码里**：type0就是`A+B`，type1就是`max(A-C,0) + max(B-D,0)`，完全不用自己“发明”逻辑。
    * 💡 **学习笔记**：基础题的核心是“照题办事”，不要想复杂！

2.  **关键点2：正确使用`std::max`函数**
    * **分析**：`max(x,y)`会返回x和y中的较大值——比如`max(A-C,0)`刚好解决“结果不能为负”的问题。如果手动写`if(A-C < 0) res = 0; else res = A-C;`，不仅代码变长，还容易写错条件。
    * 💡 **学习笔记**：C++标准库的函数是“工具”，能用就不要自己写！

3.  **关键点3：输入顺序不能错**
    * **分析**：题目中输入的顺序是`type A B C D`（第一个数是type，后面依次是A、B、C、D）。如果输入时写成`cin>>A>>B>>C>>D>>type`，整个计算就全错了！解决办法是**把输入顺序和题目描述“对齐”**——题目怎么写，代码就怎么读。
    * 💡 **学习笔记**：输入顺序是“隐形的规则”，一定要仔细看题！

### ✨ 解题技巧总结
-   **技巧A：“公式直译”法**：基础计算类题目，直接把题目中的公式翻译成代码，不要加额外逻辑；
-   **技巧B：善用标准库**：`std::max`「`std::min`」这类函数能帮你减少代码量，避免错误；
-   **技巧C：输入顺序对齐**：输入参数时，严格按照题目给出的顺序读入，不要调换位置。

# [语言月赛 202501] 数字谜

## 题目描述

给出一个两位数 $a$，问至少要把 $a$ 加上几（可以 $+0$），才能让 $a$ 的个位变成 $b$。

## 说明/提示

【样例 1 解释】

$47+2=49$，个位为 $9$。

【样例 2 解释】

$89+4=93$，个位为 $3$。

【样例 3 解释】

$a$ 的个位已经是 $0$，所以最少只要增加 $0$。

【数据范围】

$10\le a\le 90, 0\le b\le 9$。

## 样例 #1

### 输入

```
47 9
```

### 输出

```
2
```

## 样例 #2

### 输入

```
89 3
```

### 输出

```
4
```

## 样例 #3

### 输入

```
30 0
```

### 输出

```
0
```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：数学计算与编程技巧应用  

🗣️ **初步分析**：  
解决「数字谜」的关键，是让两位数`a`的**个位**快速变成目标数字`b`。我们可以把个位数字想象成「每10个数循环一次的小齿轮」——比如个位从0到9转一圈，再回到0。要让齿轮转到`b`，不需要一步步掰动齿轮（模拟加1），而是可以用**数学公式直接算出最少需要转多少格**。  

题解的核心思路是：**用`(90 + b - a) % 10`直接计算最小增量**。原理很简单：  
- 90是10的倍数（比如90=9×10），所以`90 + b - a`的结果，其个位数字等价于`b - a`的个位（因为90不影响个位）。  
- 取模10（`%10`）可以把结果限制在0~9之间，确保是「最小的非负增量」。  

比如样例1中，`a=47`、`b=9`：`90+9-47=52`，`52%10=2`——正好是需要加的数！  

**核心难点**：如何跳过「模拟加1」的繁琐步骤，直接用数学规律解决问题？  
**解决方案**：观察个位的周期性（每10个数重复），用模运算「跳过循环」，快速得到结果。  

**可视化设计思路**：  
我们会用「8位像素风」模拟这个「齿轮转动」的过程——  
- 屏幕左侧显示`a`的像素化数字（比如47用方块拼出），右侧显示目标`b`（9）；  
- 中间用动画展示公式计算：`90+9`的方块合并、`-47`的方块减少、`%10`的方块「截断」最后一位；  
- 每一步关键操作（比如加、减、取模）会伴随「叮」的像素音效，结果`2`会闪烁高亮，强化记忆。

## 3. 核心难点辨析与解题策略

### 核心难点与解决方法
1. **难点1：如何避免「一步步加1」的模拟？**  
   - 分析：模拟会让代码变长，而且当`a`很大时（比如90），需要加很多次才能得到结果。  
   - 解决：观察个位的周期性（每10个数重复），用「目标个位 - 当前个位 + 10」的非负余数（即`(b - a%10 +10)%10`），或题解中的`(90+b-a)%10`（两者等价）。  

2. **难点2：如何处理「b < 当前个位」的情况？**  
   - 分析：比如`a=89`（个位9）、`b=3`，直接减会得到负数（3-9=-6），但我们需要非负的增量。  
   - 解决：加10再取模（比如`(3-9+10)%10=4`），确保结果是「最小的正数」。  

3. **难点3：如何验证公式的正确性？**  
   - 分析：公式对不对，需要用样例测试。  
   - 解决：把样例代入公式——比如样例2的`89+3`，`90+3-89=4`，`4%10=4`，完全正确！  


### ✨ 解题技巧总结
- **用数学规律简化计算**：遇到周期性问题（比如个位、星期几），优先想「模运算」；  
- **多测试样例**：公式写好后，用样例数据验证，避免逻辑错误；  
- **代码极简原则**：能不用循环就不用，用数学公式替代重复操作。  


## 4. C++核心代码实现赏析

### 本题通用核心C++实现参考
* **说明**：本代码综合了题解的核心思路，是「数学公式法」的典型实现。  
* **完整核心代码**：
  ```cpp
  #include <iostream>
  using namespace std;

  int main() {
      int a, b;
      cin >> a >> b;
      int ans = (90 + b - a) % 10;
      cout << ans << endl;
      return 0;
  }
  ```
* **代码解读概要**：  
  1. 读取输入的两位数`a`和目标个位`b`；  
  2. 用公式`(90 + b - a) % 10`计算最小增量；  
  3. 输出结果。  
  整个代码只有5行核心逻辑，却完美解决了问题！


### 题解一（来自yummy）的核心片段赏析
* **亮点**：用一行公式解决问题，把「数学规律」直接转化为代码。  
* **核心代码片段**：
  ```cpp
  int ans = (90 + b - a) % 10;
  ```
* **代码解读**：  
  这行代码是题解的「灵魂」——  
  - `90 + b`：因为90是10的倍数，加上`b`后，个位就是`b`；  
  - `-a`：计算从`a`到「90+b」需要加多少；  
  - `%10`：取模10，得到「最小的非负增量」（因为超过10的部分会被「截断」，比如加12等价于加2）。  

  比如`a=89`、`b=3`：`90+3=93`，`93-89=4`，`4%10=4`——正好是需要加的数！  

* 💡 **学习笔记**：  
  代码的「简洁」往往来自「对问题本质的理解」——把数学规律写成代码，比写循环更高效！

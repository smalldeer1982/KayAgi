# 题目信息

# [语言月赛 202504] 洛谷月赛

## 题目背景

为了简化问题，本题中的资质要求和真实资质要求略有区别，请以本题为准。

**请入门选手注意，CCF 程序设计能力等级并不是 GESP 等级。**

## 题目描述

小 M 最近想出一场洛谷 div. 2 小月赛。他已经出好了一些题，并想要再找一个**有出题经验的**工作人员合作。（你可以认为整个团队只有他们两人。）

洛谷月赛对工作人员的 CCF 程序设计能力等级（简称“等级”）有所要求，具体如下：

- **至少一个工作人员**获得了 $\ge 8$ 级，或 $\ge 6$ 级且有出题经验，称为该比赛的**负责人**。
- **所有工作人员**至少需要 $3$ 级。

给出小 M 的等级、是否有出题经验，判断他找的合作者至少要获得几级，或报告两人无法满足资质要求。

## 说明/提示

【样例 1 解释】

小 M 自己是 $8$ 级，没有出题经验，但可以是负责人。他可以找另一个 $3$ 级，有出题经验的人合作。

但是如果合作者只有 $2$ 级，那么不满足洛谷的第二条要求。

【样例 2 解释】

小 M 自己是 $5$ 级，尽管有出题经验，但不可以是负责人，因此必须让对方负责比赛。由于合作者有出题经验，他只需要达到 $6$ 级即可。

【样例 3 解释】

由于小 M 自己只有 $2$ 级，因此无论对方有多强，都无法满足第二条要求。

【数据范围】

$0\le l\le 10$，$0\le e\le 1$。

## 3. 核心难点辨析与解题策略

<difficulty_intro>
这道题的“小弯”在于**把两个大条件拆成小M的具体情况**，我帮大家总结了3个核心难点和解决办法：
</difficulty_intro>

1.  **难点1：正确理解“负责人”的条件**  
    * 分析：题目说“负责人是≥8级，或≥6级且有出题经验”——要注意“且”和“或”的逻辑！比如小M等级7、经验1，虽然≥6但<8，但因为有经验，所以能当负责人；而等级6、经验0，就不能当。  
    * 💡 学习笔记：用“布尔表达式”写清楚条件，比如`(l >= 8) || (l >=6 && e ==1)`，比文字好记！

2.  **难点2：“所有工作人员≥3级”的隐藏关系**  
    * 分析：如果小M自己都<3级，不管合作者多厉害，都不满足第二个条件——比如小M等级2，直接Impossible！  
    * 💡 学习笔记：先判断“最严格的条件”（小M是否≥3），再处理其他问题，避免绕弯路。

3.  **难点3：合作者“有出题经验”的隐含条件**  
    * 分析：题目说要找“有出题经验的”合作者，所以合作者的经验是1——这意味着如果需要合作者当负责人，只要≥6级就行（因为满足“≥6且有经验”），不用到8级！  
    * 💡 学习笔记：读题要注意“隐藏信息”，比如“有出题经验的工作人员”不是装饰，是关键条件！

### ✨ 解题技巧总结
- **技巧A：条件拆解**：把大条件拆成小条件，比如先判断小M≥3，再判断能否当负责人，逐步缩小范围。  
- **技巧B：用变量简化逻辑**：比如用`can_leader`表示小M能否当负责人，代码里写`if(can_leader)`比重复写长条件更清晰。  
- **技巧C：边界测试**：写完代码要测边界情况，比如小M等级3、经验0（能否当负责人？不能，合作者要6级），小M等级6、经验1（能当负责人，合作者要3级）。

**本题通用核心C++实现参考**  
* **说明**：本代码综合了题解的思路，用最简洁的逻辑覆盖所有情况。  
* **完整核心代码**：
    ```cpp
    #include <iostream>
    using namespace std;

    int main() {
        int l, e;
        cin >> l >> e; // 读取小M的等级l和经验e（0/1）
        
        if (l < 3) {
            cout << "Impossible" << endl; // 小M<3，直接不可能
        } else {
            bool can_leader = (l >= 8) || (l >= 6 && e == 1); // 小M能否当负责人
            if (can_leader) {
                cout << 3 << endl; // 小M能当，合作者只要3级
            } else {
                cout << 6 << endl; // 小M不能当，合作者要当负责人（≥6级）
            }
        }
        return 0;
    }
    ```
* **代码解读概要**：  
  代码分三步：1. 读入小M的信息；2. 判断小M是否≥3级（否就输出Impossible）；3. 判断小M能否当负责人（是则合作者3级，否则6级）。用`bool can_leader`把“能否当负责人”这个核心问题变成了一个变量，代码特别好懂！

## 题目描述

小 A 正在撰写毕业论文。毕业论文需要满足字数要求、查重要求和 AIGC 检测要求。

假设小 A 的论文共有 $w$ 字，其中被认定为复制的字数为 $f$ 字，被认定为 AIGC 的字数为 $a$ 字。
下面是学院对以上三个要求的具体内容：

- **查重要求**。当字数不为 $0$ 时，重复率为 $\dfrac{f}{w}$，重复率不能超过 $20\%$。
- **AIGC 要求**。当字数不为 $0$ 时，AIGC 率为 $\dfrac{a}{w}$，AIGC 率不能超过 $30\%$。
- **字数要求**。字数不能少于 $r$ 字。

请判定小 A 的毕业论文是否满足要求。

## 说明/提示

**【样例 1 解释】**

重复率为 $25\%$，不符合要求。

**【样例 2 解释】**

重复率为 $20\%$，AIGC 率为 $25\%$，字数为 $20$ 字，均符合要求。

**【数据规模与约定】**

对于 $100\%$ 的测试数据：

- $0 \le w,f,a \le 10^6$
- $1 \le r \le 10^6$
- $w \ge f$，$w \ge a$

## 样例 #1

### 输入

```
20 5 5 10```

### 输出

```
Rejected```

## 样例 #2

### 输入

```
20 4 5 10```

### 输出

```
Accepted```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：模拟（规则判断的直接应用）

🗣️ **初步分析**：
解决这道题的核心是**“模拟”**——就像你帮老师检查同学的作业：先看字数够不够（>=r），再看重复率有没有超（≤20%），最后看AIGC率达不达标（≤30%），三个条件都满足才算“合格”。  

**模拟的核心思想**：按照题目给定的规则，一步步“复刻”判断过程。本题中，我们需要把“重复率≤20%”“AIGC率≤30%”这两个浮点数条件，转换成**整数乘法**（比如10×f ≤ 2×w）——这是避免浮点数误差的关键！比如样例1中，w=20，f=5，10×5=50，2×20=40，50>40，所以重复率超标，直接拒绝。

## 3. 核心难点辨析与解题策略

<difficulty_intro>
这道题的“坑”不多，但三个关键细节需要特别注意——一不小心就会错哦！
</difficulty_intro>

1.  **关键点1：条件的“与逻辑”**  
    * **分析**：三个要求必须**同时满足**（字数够+重复率低+AIGC率低），少一个都不行。比如样例1字数够（20≥10），但重复率超了，所以拒绝。解决方法：用`&&`（逻辑与）把三个条件连起来，只有全为真时才输出Accepted。  
    * 💡 **学习笔记**：多条件判断时，先明确“是要全满足还是满足一个”，逻辑运算符别用反！

2.  **关键点2：浮点数误差的规避**  
    * **分析**：如果直接算`f*1.0/w ≤0.2`，可能因为浮点数的精度问题出错（比如0.2在计算机里是近似值）。解决方法：把不等式两边**同乘一个整数**（比如10），转换成整数比较——10f ≤ 2w（因为0.2=2/10），10a ≤3w（0.3=3/10）。这样完全不用浮点数，绝对准确！  
    * 💡 **学习笔记**：竞赛中遇到百分比、比例问题，优先用整数乘法代替浮点数除法！

3.  **关键点3：w=0的边界处理**  
    * **分析**：题目中说“当字数不为0时”才计算重复率和AIGC率，但如果w=0，字数要求w≥r（r≥1）肯定不满足，所以直接拒绝。解决方法：不用单独处理w=0——因为三个条件里第一个就是w≥r，w=0时直接不满足，后面的条件不用判断（逻辑与的“短路效应”会自动跳过）。  
    * 💡 **学习笔记**：利用逻辑运算符的“短路”特性，可以简化边界条件的判断！

### ✨ 解题技巧总结
- **技巧A：规则拆解**：把复杂的问题拆成“多个小条件”，逐个检查（比如本题拆成“字数、查重、AIGC”三个小条件）；  
- **技巧B：整数替代浮点数**：遇到比例/百分比问题，用“交叉相乘”转换成整数运算，避免误差；  
- **技巧C：利用逻辑短路**：比如`条件1 && 条件2`，如果条件1不满足，条件2不会执行，节省时间还简化代码！


## 4. C++核心代码实现赏析

<code_intro_overall>
先看一个完整的核心实现——它把三个条件浓缩成一行判断，超简洁！
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：本代码综合了优质题解的思路，用最简洁的方式实现规则判断，避免了浮点数运算。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    using namespace std;

    int main() {
        long long w, f, a, r;  // 用long long避免溢出（比如w=1e6时，10*w=1e7，int也够，但long long更安全）
        cin >> w >> f >> a >> r;
        
        bool ok = true;
        if (w < r) ok = false;  // 字数不够
        if (10 * f > 2 * w) ok = false;  // 重复率超了
        if (10 * a > 3 * w) ok = false;  // AIGC率超了
        
        cout << (ok ? "Accepted" : "Rejected") << endl;
        return 0;
    }
    ```
* **代码解读概要**：  
  代码先读取四个输入（w：总字数，f：重复字数，a：AIGC字数，r：字数要求），然后用三个`if`依次检查三个条件——只要有一个不满足，`ok`就变成`false`。最后根据`ok`的值输出结果（三目运算符`? :`超好用！）。


<code_intro_selected>
接下来看题解中的“精华片段”——整数转换的关键代码！
</code_intro_selected>

**题解一：(来源：览遍千秋)**
* **亮点**：用“10×f ≤ 2×w”替代浮点数比较，彻底解决误差问题。
* **核心代码片段**：
    ```cpp
    if (w >= r && 10 * f <= 2 * w && 10 * a <= 3 * w) {
        cout << "Accepted" << endl;
    } else {
        cout << "Rejected" << endl;
    }
    ```
* **代码解读**：  
  这行代码把三个条件用`&&`连起来，**只有全满足才输出Accepted**。重点看`10*f <= 2*w`——原本是`f/w ≤0.2`，两边乘10后变成`10f ≤2w`（因为w是正数，不等号方向不变）。比如样例1中，10×5=50，2×20=40，50>40，所以条件不满足，直接进入else分支。  
  为什么用`10`？因为0.2和0.3的分母都是10（2/10、3/10），乘10刚好把小数变成整数——是不是超聪明？
* 💡 **学习笔记**：遇到`x/y ≤ k`（k是小数），把两边乘y的“最小公倍数分母”，转换成整数比较，永远是对的！

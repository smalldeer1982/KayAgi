# [语言月赛 202506] 票价计算

## 题目描述

某公司有三班同一时刻发车的通勤车，第 $i$ 班车的票价为 $a_i$ 元，运行时间为 $b_i$ 分钟。

小明的公司上班时间是 9:00，最慢的那班车恰好能在 9:00 到达公司。现在公司推出新政策：每在 9:00 前提前一分钟到达公司，可以获得 $c$ 元奖金。

请帮助小明选择一班车，使得实际花费（票价减去奖金）最少。

## 说明/提示

### 样例 1 解释

最慢的车是第 2 班车（运行时间 $6$ 分钟）：
- 选择第 1 班车：花费 $3 - (6 - 5) \times 1 = 2$ 元；
- 选择第 2 班车：花费 $10 - 0 = 10$ 元；
- 选择第 3 班车：花费 $7 - (6-2) \times 1 = 3$ 元。

最少花费为 $2$ 元（第 1 班车）。

### 样例 2 解释

最慢的车是第 1 班车（运行时间 $10$ 分钟）：
- 选择第 1 班车：花费 $5 - 0 = 5$ 元；
- 选择第 2 班车：花费 $8 - (10-8) \times 2 = 4$ 元；
- 选择第 3 班车：花费 $15 - (10-5) \times 2 = 5$ 元。

最少花费为 $4$ 元（第 2 班车）。

### 样例 3 解释

请留意，奖金完全有可能大于车票本身的价格，因此最终的输出中可能包含负数。

### 数据规模与约定

对于 $100\%$ 的数据，$1 \leq a_1, b_1, a_2, b_2, a_3, b_3, c \leq 100$。

## 样例 #1

### 输入

```
3 5 10 6 7 2 1
```

### 输出

```
1 2
```

## 样例 #2

### 输入

```
5 10 8 8 15 5 2
```

### 输出

2 4
```

## 样例 #3

### 输入

```
20 30 15 40 25 35 3
```

### 输出

1 -10
```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：数学基础应用  

🗣️ **初步分析**：  
解决这道题的关键就像“帮小明算一笔通勤账”——我们需要先找到“最慢的那班车”作为参考（就像大家都以9点到为基准，最慢车刚好卡点），再算每班车的实际花费（票价减去提前到的奖金），最后选最划算的那班。  

**数学基础应用**在这里的核心是“用算术运算解决实际问题”：就像你买东西时算“折扣后价格”，这里的“折扣”是提前到的奖金。题解的思路非常直接：  
1. 找运行时间的最大值T（最慢车的时间）；  
2. 用公式算每班车的实际花费 `cost_i = a_i - c*(T - b_i)`；  
3. 比较三个cost，选最小的（编号最小的优先）。  

**核心难点**是：① 正确理解T是“最长运行时间”；② 准确计算cost公式；③ 处理多个最小值的情况。解决方案就是“明确定义+公式推导+逐个比较”。  

可视化设计上，我们会用**像素风格**展示三个班车，高亮最大值的车作为参考，计算每个cost时闪烁对应车，最后选中的车放大并播放胜利音效——就像玩“选最优道具”的复古游戏，直观又有趣！

## 3. 核心难点辨析与解题策略

在解决这个问题时，我们通常会遇到以下3个关键点：

### 关键点1：理解T是“最慢车的运行时间”（即三个b的最大值）  
**分析**：题目说“最慢的车恰好9点到”，所以最慢车的运行时间是所有车中最长的（T = max(b1,b2,b3)）。如果把T错算成最小值，后面的奖金会变成负数（提前时间为负，相当于迟到），结果全错。  
**解决方案**：用`max`函数嵌套计算三个b的最大值，明确“最慢=运行时间最长”。  
💡 **学习笔记**：T是所有运行时间的最大值，这是计算奖金的核心参考点！

### 关键点2：正确计算实际花费 `cost_i = a_i - c*(T - b_i)`  
**分析**：实际花费是“票价减去奖金”，奖金是“提前分钟数×c”（提前分钟数= T - b_i，因为T是最慢时间，b_i越小提前越多）。容易出错的地方：① 把奖金算成“b_i×c”（没减T）；② 把公式写成“a_i + c*(T - b_i)”（把奖金算成花费）。  
**解决方案**：先推导公式，再用样例验证（比如样例1中，T=6，cost1=3-1*(6-5)=2，和样例一致）。  
💡 **学习笔记**：公式是“票价 - 奖金”，奖金是“提前时间×c”，提前时间= T - b_i！

### 关键点3：处理多个相同最小值时选编号最小的车  
**分析**：如果两班车cost相同（比如cost1=cost2=2），要选编号小的1号车。  
**解决方案**：按顺序比较——先看cost1是否最小，再看cost2，最后看cost3，确保编号小的优先。  
💡 **学习笔记**：相同结果选编号小的，这是题目要求的“优先规则”！


### ✨ 解题技巧总结  
- **问题拆解**：把“选最优车”拆成“算T→算cost→比大小”，复杂问题变简单；  
- **公式验证**：用样例验证公式是否正确，避免计算错误；  
- **顺序比较**：按编号顺序比较cost，确保相同结果选编号小的。


## 4. C++核心代码实现赏析

在深入分析具体片段前，先看一个**通用核心实现**，帮大家把握整体框架：


### 本题通用核心C++实现参考  
* **说明**：本代码综合了优质题解的思路，是“计算+比较”的典型实现。  
* **完整核心代码**：  
```cpp
#include <iostream>
#include <algorithm> // 用于max函数
using namespace std;

int main() {
    int a1, b1, a2, b2, a3, b3, c;
    cin >> a1 >> b1 >> a2 >> b2 >> a3 >> b3 >> c;
    
    // 计算最慢车的运行时间T（最大值）
    int T = max(max(b1, b2), b3);
    // 计算每班车的实际花费
    int cost1 = a1 - c * (T - b1);
    int cost2 = a2 - c * (T - b2);
    int cost3 = a3 - c * (T - b3);
    
    // 找最小的cost和对应的车号
    int min_cost = cost1;
    int car_num = 1;
    if (cost2 < min_cost) {
        min_cost = cost2;
        car_num = 2;
    }
    if (cost3 < min_cost) {
        min_cost = cost3;
        car_num = 3;
    }
    
    cout << car_num << " " << min_cost << endl;
    return 0;
}
```  
* **代码解读概要**：  
  1. 读取输入：三班车的a（票价）、b（运行时间）和c（奖金）；  
  2. 算T：用`max`函数找三个b的最大值；  
  3. 算cost：用公式计算每班车的实际花费；  
  4. 比大小：从cost1开始，依次比较cost2、cost3，更新最小cost和车号；  
  5. 输出结果：车号和最小cost。  


### 针对优质题解的片段赏析  

**题解一：(来源：Maxmilite)**  
* **亮点**：用最基础的代码实现核心逻辑，逻辑“直戳问题本质”。  
* **核心代码片段**：  
```cpp
int T = max(max(b1, b2), b3);
int cost1 = a1 - c * (T - b1);
int cost2 = a2 - c * (T - b2);
int cost3 = a3 - c * (T - b3);
```  
* **代码解读**：  
  第一行用`max`嵌套计算三个b的最大值T——这是所有计算的“参考点”（最慢车的时间）。接下来三行是**核心公式**：比如cost1是1号车的实际花费，等于票价a1减去奖金（c×提前时间T - b1）。这段代码没有多余的步骤，直接完成了“算参考点→算每个cost”的关键步骤，就像“用计算器算三个账”，非常好懂！

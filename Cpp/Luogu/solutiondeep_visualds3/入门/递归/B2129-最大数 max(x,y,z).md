# 最大数 max(x,y,z)

## 题目描述

已知：

$m=\dfrac{\max(a,b,c)}{\max(a+b,b,c) \times \max(a,b,b+c)}$

输入三个整数 $a$，$b$，$c$，求 $m$。

把求三个数的最大数 $\max(x,y,z)$ 分别定义成函数和过程来做。

## 说明/提示

### 数据规模与约定

对于全部的测试点，保证 $|a|,|b|,|c| \leq 50$。换句话说，$a,b,c$ 都在 $-50$ 到 $50$ 之间。

测试数据保证，分式的分母不为 $0$。

## 样例 #1

### 输入

```
1 2 3```

### 输出

```
0.200```

## 3. 核心难点辨析与解题策略

<difficulty_intro>
这道题的“坑”都藏在“细节”里。结合优质题解的经验，我帮你总结了三个最容易踩的坑和解决办法：
</difficulty_intro>

1.  **关键点1：如何正确求三个数的最大值？**
    * **问题**：C++自带的`max`函数只能比较两个数，直接写`max(a,b,c)`会报错！
    * **解决办法**：① 嵌套调用：`max(a, max(b,c))`（先比b和c，再和a比）；② C++11语法：`max({a,b,c})`（用大括号装多个参数，需要编译器支持）。
    * 💡 **学习笔记**：函数是“工具”，嵌套或用新特性都是“用工具的技巧”，核心是解决“多值比较”的问题！

2.  **关键点2：为什么结果总是整数？（避免整数除法）**
    * **问题**：如果`a,b,c`是整数，直接算`max(a,b,c)/(max(a+b,b,c)*max(...))`会触发“整数除法”（比如3/2=1，而不是1.5）。
    * **解决办法**：① 把变量定义为`double`类型（比如`double a,b,c`）；② 给整数乘`1.0`（比如`maxx(a,b,c)*1.0 / (...)`），强制转为浮点数。
    * 💡 **学习笔记**：除法中只要有一个操作数是浮点数，结果就是浮点数——这是C++的“隐形规则”，一定要记牢！

3.  **关键点3：如何输出保留三位小数？**
    * **问题**：直接输出`cout << m`会显示很多位小数，不符合题目要求。
    * **解决办法**：① C++风格：`cout << fixed << setprecision(3) << m`（需要`#include <iomanip>`）；② C语言风格：`printf("%.3lf", m)`（简单直接，新手推荐）。
    * 💡 **学习笔记**：输出格式是“面子工程”，但题目要求必须满足——两种方法选一个你记熟的就行！

### ✨ 解题技巧总结
<summary_best_practices>
通过这道题，我想给你分享三个“终身有用”的编程技巧：
</summary_best_practices>
-   **技巧1：重复逻辑要封装**：如果一段代码要写多次，就做成函数——比如本题的“三数求最大”，写一次函数，调用三次，比重复写三次嵌套`max`舒服10倍！
-   **技巧2：整数除法要小心**：只要涉及除法，先想“是不是要转浮点数”——用`double`类型或者乘`1.0`，永远没错！
-   **技巧3：输出格式早规划**：题目要求“保留三位小数”，写代码前就想好要用`printf`还是`cout`+`iomanip`，不要等到最后再改！

<code_intro_overall>
先看一个“入门友好版”的通用代码——它综合了newbie666和StayAlone的思路，用自定义函数，逻辑清晰，适合新手：

**本题通用核心C++实现参考**
* **说明**：本代码来自newbie666的题解，是“自定义函数+浮点数处理+printf输出”的标准模板，适合入门学习者。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    #include <algorithm> // 包含max函数
    using namespace std;

    // 自定义三数最大值函数：返回三个double中的最大值
    double max3(double a, double b, double c) {
        return max(a, max(b, c)); // 嵌套调用两数max
    }

    int main() {
        double a, b, c; // 用double避免整数除法
        cin >> a >> b >> c;
        // 计算m：分子是max(a,b,c)，分母是max(a+b,b,c)*max(a,b,b+c)
        double m = max3(a, b, c) / max3(a + b, b, c) / max3(a, b, b + c);
        printf("%.3lf", m); // 保留三位小数输出
        return 0;
    }
    ```
* **代码解读概要**：
    > 代码分三部分：① 自定义`max3`函数（封装三数求最大）；② 读取输入（`a,b,c`是double类型）；③ 计算`m`（用`max3`函数代替重复的嵌套`max`）；④ 输出`m`（`printf("%.3lf")`保留三位小数）。整个流程就像“按菜谱炒菜”：先准备工具（函数），再备菜（输入），再炒菜（计算），最后装盘（输出）！

**题解一：happybob（C++11极简写法）**
* **亮点**：用C++11的`max({a,b,c})`一键求多值最大，代码行数少到极致！
* **核心代码片段**：
    ```cpp
    #include <iostream>
    #include <algorithm>
    #include <iomanip> // 包含fixed和setprecision
    using namespace std;

        cin >> a >> b >> c;
        // 直接用max({...})求多值最大，cout控制输出格式

        return 0;
    }
    ```
* **代码解读**：
    > 这段代码的“精华”是`max({a,b,c})`——大括号里可以放任意多个同类型的数，`max`函数会直接返回最大值（需要C++11及以上版本）。输出部分用`cout << fixed << setprecision(3)`，意思是“固定小数点后三位”——比如结果是0.2，会显示0.200。
* 💡 **学习笔记**：C++的新特性能帮你“偷懒”，但要先确认编译器支持哦！

**题解二：StayAlone（对比臃肿与简洁写法）**
* **亮点**：用两个版本的代码对比，直观展示“函数封装”的好处！
* **核心代码片段（臃肿版）**：
    ```cpp
    // 直接嵌套max，代码很长
    printf("%.3lf", max(a, max(b, c)) / max(a + b, max(b, c)) / max(a, max(b, b + c)));
    ```
* **核心代码片段（简洁版）**：
    ```cpp
    // 自定义max1函数，代码瞬间变简洁
    double max1(double a, double b, double c) { return max(a, max(b, c)); }
    printf("%.3lf", max1(a, b, c) / max1(a + b, b, c) / max1(a, b, b + c));
    ```
* **代码解读**：
    > 臃肿版的代码里，每个`max`都要嵌套两次，看起来很“乱”；简洁版把嵌套逻辑封装成`max1`函数，调用时只要写`max1(a,b,c)`，代码可读性直接提升一个档次！
* 💡 **学习笔记**：函数封装不是“多此一举”，而是“让代码变好看的魔法”！

**题解三：__Cartesian__Tree__（整数转浮点数的小技巧）**
* **亮点**：用`*1.0`解决整数除法问题，适合变量是整数的情况！
* **核心代码片段**：
    ```cpp
    int a, b, c; // 变量是整数
    m = maxx(a, b, c) * 1.0 / (maxx(a + b, b, c) * maxx(a, b, b + c));
    ```
* **代码解读**：
    > 这里`maxx(a,b,c)`返回的是整数，但乘`1.0`后会变成浮点数（比如3*1.0=3.0），这样除法结果就不会是整数了。如果没有`*1.0`，比如3/(2*3)=3/6=0（整数除法），但乘了之后是3.0/(2*3)=0.5，结果正确！
* 💡 **学习笔记**：如果变量必须是整数，就用`*1.0`强制转浮点数——这是“急救小技巧”！

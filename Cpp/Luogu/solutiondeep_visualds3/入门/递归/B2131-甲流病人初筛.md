# 甲流病人初筛

## 题目描述

目前正是甲流盛行时期，为了更好地进行分流治疗，医院在挂号时要求对病人的体温和咳嗽情况进行检查，对于体温超过 $37.5$ 度（含等于 $37.5$ 度）并且咳嗽的病人初步判定为甲流病人（初筛）。现需要统计某天前来挂号就诊的病人中有多少人被初筛为甲流病人。

## 样例 #1

### 输入

```
5
Zhang 38.3 0
Li 37.5 1
Wang 37.1 1
Zhao 39.0 1
Liu 38.2 1```

### 输出

```
Li
Zhao
Liu
3```

## 3. 核心难点辨析与解题策略

<difficulty_intro>
模拟题看似简单，但新手容易在“细节”上栽跟头。结合题解的共性问题，我帮你提炼了3个核心难点和解决策略：
</difficulty_intro>

1.  **关键点1：数据类型别搞错！**  
    * **难点**：体温是`38.3`这样的小数，如果你用`int`（整数类型）存，会直接丢掉小数部分（比如`38.3`变成`38`），导致判断错误！  
    * **解决策略**：一定要用`double`（双精度浮点型）存体温！所有题解都用了`double`，这是模拟题的“基础常识”。  
    * 💡 **学习笔记**：小数用`double`/`float`，整数用`int`，字符串用`string`——变量类型要“对号入座”！

2.  **关键点2：条件判断要准确！**  
    * **难点**：题目说“体温≥37.5且咳嗽”，咳嗽的输入是`0`（不咳嗽）或`1`（咳嗽）。新手容易写成`cough==0`（反了），或者漏写`=`（比如`temp>37.5`，漏掉等于的情况）。  
    * **解决策略**：把条件拆成两部分：`temp >= 37.5`（体温够高）**且**`cough == 1`（咳嗽）。所有题解都用了`&&`（逻辑与）连接两个条件，这是最严谨的写法。  
    * 💡 **学习笔记**：条件判断前，先把题目要求“逐字翻译成代码”，再检查一遍！

3.  **关键点3：输入输出顺序要符合样例！**  
    * **难点**：样例要求“先输出符合条件的名字（每行一个），最后输出总数”。新手可能会把总数先输出，或者名字没换行，导致格式错误。  
    * **解决策略**：按照样例的顺序写代码：符合条件时立即`cout<<name<<endl;`（`endl`换行），最后再`cout<<ans<<endl;`。题解一和题解二都这么做，完全贴合样例格式。  
    * 💡 **学习笔记**：模拟题的输出格式“照葫芦画瓢”——样例怎么输出，你就怎么写！

## 4. C++核心代码实现赏析

<code_intro_overall>
先看一份**最简洁的通用实现**——来自题解一，它把模拟的核心流程浓缩到了10行左右，非常适合新手入门！
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：本代码来自“我和鱼过不去”的题解，是“在线处理”的典型写法，逻辑直白、代码简洁，覆盖了所有核心步骤。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    #include <string>
    using namespace std;

    int main() {
        int n, cough, ans = 0;
        string name;
        double temp;
        cin >> n;
        for (int i = 1; i <= n; ++i) {
            cin >> name >> temp >> cough;
            if (temp >= 37.5 && cough) {
                cout << name << endl;
                ans++;
            }
        }
        cout << ans << endl;
        return 0;
    }
    ```
* **代码解读概要**：
    > 代码分四步：1. 定义变量（`n`是病人数量，`ans`是符合条件的人数）；2. 读入`n`；3. 循环`n`次，每次读入一个病人的信息；4. 判断条件，符合就输出名字并计数；5. 最后输出总数。整个流程和“手动筛选”完全一样，没有多余的步骤！

<code_intro_selected>
接下来看三个题解的“亮点片段”，帮你理解不同的写法思路~
</code_intro_selected>

**题解一：(来源：我和鱼过不去)**
* **亮点**：用“重复变量”省内存，逻辑最直白。
* **核心代码片段**：
    ```cpp
    for (int i = 1; i <= n; ++i) {
        cin >> name >> temp >> cough;  // 每次循环重新读入name，不用存所有名字
        if (temp >= 37.5 && cough) {
            cout << name << endl;
            ans++;
        }
    }
    ```
* **代码解读**：
    > 这段循环是“在线处理”的核心——每次读一个病人的信息，直接判断，符合就输出。`name`变量在循环里“反复使用”，比如第一次读`Zhang`，第二次读`Li`时会覆盖掉之前的值，完全不用开数组存所有名字！这种写法的好处是**内存占用最少**，适合数据量不大的情况。
* 💡 **学习笔记**：模拟题优先“在线处理”，能省内存就省！

**题解二：(来源：墨忧)**
* **亮点**：用结构体“打包”数据，适合离线处理。
* **核心代码片段**：
    ```cpp
    struct person {
        string name;  // 姓名
        double tem;   // 体温
        int k;        // 是否咳嗽（1是，0否）
    } a[300];  // 最多存300个病人

    for (int i = 0; i < n; ++i) {
        cin >> a[i].name >> a[i].tem >> a[i].k;  // 把每个病人存到结构体数组里
    }
    ```
* **代码解读**：
    > 结构体`person`把“名字、体温、咳嗽”三个相关数据“打包”成一个“病人对象”，然后用数组`a`存所有病人。这种写法叫“离线处理”——先把所有数据存起来，再统一遍历判断。如果题目要求“第二次筛选”（比如再找“体温<37.5但咳嗽”的人），直接再遍历一遍数组就行，不用重新读入数据！
* 💡 **学习笔记**：需要多次处理数据时，用结构体存起来更方便！

**题解三：(来源：int127)**
* **亮点**：用数组缓存结果，适合需要排序或调整输出顺序的场景。
* **核心代码片段**：
    ```cpp
    string ill[210];  // 存符合条件的病人名字
    int cnt = 0;      // 符合条件的人数

    fr(i, 1, n, 1) {  // fr是for循环的宏定义，等价于for(int i=1;i<=n;i++)
        cin >> a[i].p >> a[i].t >> a[i].k;
        if (a[i].t >= 37.5 && a[i].k) {
            ill[++cnt] = a[i].p;  // 把符合条件的名字存到ill数组里
        }
    }

    fr(i, 1, cnt, 1) cout << ill[i] << '\n';  // 最后一起输出
    ```
* **代码解读**：
    > 这段代码把符合条件的名字存到`ill`数组里，最后再一起输出。如果题目要求“按名字字典序排序输出”，只需要在存完后加一句`sort(ill+1, ill+cnt+1);`就行！作者还提到可以用`vector<string> ill;`替代数组——`vector`不用预先定大小（比如`ill.push_back(a[i].p)`直接加元素），更灵活。
* 💡 **学习笔记**：需要调整输出顺序时，用数组或`vector`缓存结果！

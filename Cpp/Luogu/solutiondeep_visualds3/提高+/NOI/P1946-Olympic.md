# 题目信息

# Olympic

## 题目描述

奥运会正在如火如荼的进行着，金牌榜上也有许多队伍需要排名。你需要选择三个整数 $P_g,P_s$ 和 $P_b$，分别表示每获得一块金、银、铜牌所对应得分。并且满足 $1000 \ge P_g \ge P_s \ge P_b \ge 1$。队伍将依据他们获得的分数进行排序（高分在前）。现在，为了使你所在的队伍排名尽可能的靠前，由你来选择 $P_g,P_s,P_b$。

## 说明/提示

### 数据范围

- 对于 $10\%$ 的数据，保证最优解的 $P_g \le 10$；
- 对于 $30\%$ 的数据，保证最优解的 $P_g \le 100$。


## 样例 #1

### 输入

```
3

1 1 1

0 1 2

2 1 0```

### 输出

```
1 1 1```

#### 3. 核心难点辨析与解题策略
1. **难点：不等式组的分类处理**  
   * **分析**：当$cu_1=cu_i$需单独处理（直接比较总分），斜率正/负导致不等式方向相反。优质题解通过`if(k>0)`和`if(k<0)`分支解决。  
   * 💡 **学习笔记**：铜牌数相等时，胜负由金银牌总分决定！

2. **难点：临界点取值精度**  
   * **分析**：分数差除以铜牌数差需向上/向下取整（如`ceil(delta/k)`）。Seauy题解封装`CEIL/FLOR`函数避免重复逻辑。  
   * 💡 **学习笔记**：整数除法要特别注意余数处理！

3. **难点：枚举范围优化**  
   * **分析**：$P_b≤P_s≤P_g$的约束可提前剪枝。CodyTheWolf在`Solve(i,j)`中通过`temp[i].first<=y`跳过无效值。  
   * 💡 **学习笔记**：利用参数约束能显著减少计算量。

### ✨ 解题技巧总结
- **分层枚举法**：先固定高维参数（$P_g/P_s$），再优化低维（$P_b$）  
- **临界点离散化**：将连续不等式转化为离散事件点（Seauy的`discre[]`）  
- **数学归纳法**：把排名问题转化为"何时反超/被反超"的数学命题

**题解一核心片段**  
```cpp
if (k > 0) 
    temp[++tail] = make_pair(ceil(delta/k), true);
else 
    temp[++tail] = make_pair(floor(delta/k), false);
```
> **解读**：根据斜率正负决定临界点计算方式——斜率>0时需**向上取整**（`ceil`），确保$P_b$足够大使队伍反超；斜率<0时**向下取整**（`floor`），防止$P_b$过大导致被反超。  
> 💡 **学习笔记**：`make_pair`将临界点与方向绑定，便于后续统一排序处理。

**题解二核心片段**  
```cpp
if (B[1] > B[i]) 
    temp = CEIL(Base_diff, B[1]-B[i]);
```
> **解读**：`CEIL`自定义函数处理向上取整，避免浮点误差。`Base_diff`是金银牌总分差，当己方铜牌数多时，临界点计算决定何时实现反超。  
> 💡 **学习笔记**：封装数学运算提升代码可维护性。

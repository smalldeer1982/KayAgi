# [NOIP 2012 提高组] Vigenère 密码

## 题目描述

16 世纪法国外交家 Blaise de Vigenère 设计了一种多表密码加密算法 Vigenère 密码。Vigenère 密码的加密解密算法简单易用，且破译难度比较高，曾在美国南北战争中为南军所广泛使用。

 
在密码学中，我们称需要加密的信息为明文，用 $M$ 表示；称加密后的信息为密文，用 $C$ 表示；而密钥是一种参数，是将明文转换为密文或将密文转换为明文的算法中输入的数据，记为 $k$。在 Vigenère 密码中，密钥 $k$ 是一个字母串，$k=k_1,k_2,…,k_n$。当明文 $M=m_1,m_2,…,m_n$ 时，得到的密文 $C=c_1,c_2,…,c_n$，其中 $c_i$=$m_i \operatorname{\circledR} k_i$，运算 $\circledR$ 的规则如下表所示：

Vigenère 加密在操作时需要注意：

1. $\circledR$ 运算忽略参与运算的字母的大小写，并保持字母在明文 $M$ 中的大小写形式；
2. 当明文 $M$ 的长度大于密钥 $k$ 的长度时，将密钥 $k$ 重复使用。

例如，明文 $M=\texttt{Helloworld}$，密钥 $k=\texttt{abc}$ 时，密文 $C=\texttt{Hfnlpyosnd}$。

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：`字符串模拟` (编程技巧应用)

🗣️ **初步分析**：
> 解决Vigenère密码这道题，关键在于理解**位移密码**的数学本质。想象一个26齿的密码齿轮⚙️——密钥字符决定齿轮转动齿数，密文字母通过反向转动齿轮得到明文字母。本题中：
> - 核心公式：`m_i = (c_i - k_i + 26) % 26`（字母转为0-25数字计算）
> - 难点处理：保持密文大小写特征 + 密钥循环使用（取模实现）
> 
> 可视化设计思路：
> - 像素动画将展示26×26密码表，高亮当前密文/密钥字符的行列交点
> - 复古风格：FC红白机像素字体 + 齿轮转动音效
> - 交互设计：单步执行时显示当前字符计算过程，成功解密播放8-bit胜利音效

## 2. 精选优质题解参考

**题解一：(来源：ingu)**
* **点评**：此解法巧妙运用**位运算加速**（`k[i]&31`提取字母序号），将核心逻辑浓缩为3行代码。代码中`(c[i]&31)-t>0`的判断处理了字母表循环边界，体现了对ASCII编码特性的深刻理解。变量命名简洁（`k,c,t`）但含义清晰，边界处理严谨（`+26`防负数），竞赛实践价值极高。

## 3. 核心难点辨析与解题策略

1.  **难点：解密公式的边界处理**
    * **分析**：当`c_i < k_i`时直接相减会产生负数，优质题解均采用`(c_i - k_i + 26) % 26`策略。+26保证结果非负，%26实现字母表循环，如同密码齿轮旋转一圈复位。
    * 💡 **学习笔记**：模运算在循环数据结构中的核心作用

2.  **难点：大小写混合处理**
    * **分析**：密文大小写决定明文大小写，但密钥大小写不影响计算。Sinwind题解用`toupper`统一计算基准，再通过`isupper`还原大小写的方案最稳健。
    * 💡 **学习笔记**：字符处理应分离"计算用基准"和"最终表现形式"

3.  **难点：密钥循环机制**
    * **分析**：`i % key_length`实现密钥循环，如同磁带循环播放。所有优质题解都采用取模运算，但ingu用`i%k.length()`直接嵌入循环条件，代码最紧凑。
    * 💡 **学习笔记**：取模运算是循环访问容器的银弹

## 4. C++核心代码实现赏析

**本题通用核心C++实现参考**
* **说明**：综合优质题解思路，保留大小写处理最佳实践
```cpp
#include <iostream>
#include <cctype>
using namespace std;

int main() {
    string k, c;
    cin >> k >> c;
    for (int i = 0; i < c.size(); i++) {
        char base = isupper(c[i]) ? 'A' : 'a';     // 基准字母
        int offset = toupper(k[i % k.size()]) - 'A'; // 统一为数字偏移量
        c[i] = (c[i] - base - offset + 26) % 26 + base;
    }
    cout << c << endl;
    return 0;
}
```
* **代码解读概要**：
  > 1. 读取密钥`k`和密文`c`
  > 2. 遍历密文：用`isupper`确定基准字母（A或a）
  > 3. 计算偏移：`toupper`统一密钥字符为大写后转数字
  > 4. 解密公式：`(c_i - base - offset + 26) % 26 + base`
  > 5. 直接修改密文字符为明文输出

**题解一：(ingu)**
* **亮点**：位运算提取字母序号
* **核心代码片段**：
```cpp
int t=(k[i%k.length()]&31)-1;
c[i]=(c[i]&31)-t>0?c[i]-t:c[i]-t+26;
```
* **代码解读**：
  > - `k[i]&31`：通过位运算提取字母序号（A=1,...,Z=26）
  > - 三元运算符处理两种场景：直接减偏移量或+26循环
  > - 精妙之处：利用ASCII后5位相同特性省去大小写判断
* 💡 **学习笔记**：位运算可替代`toupper`转换

**题解二：(Sinwind)**
* **亮点**：函数封装与大小写分离
* **核心代码片段**：
```cpp
char decryption(char c, char k) {
    char temp_c = toupper(c);
    char temp_k = toupper(k);
    char base = isupper(c) ? 'A' : 'a';
    return (temp_c - temp_k + 26) % 26 + base;
}
```
* **代码解读**：
  > - 函数式设计：输入单个字符对，输出解密字符
  > - `toupper`统一计算基准，避免大小写干扰
  > - `isupper`记录原始大小写状态
  > - 公式计算后加回基准字母
* 💡 **学习笔记**：功能拆解提升可读性

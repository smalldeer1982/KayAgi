## 题目背景

与其编写苍白无力的背景，不如出更有质量的题。

## 题目描述

我们称[简单有向图](https://www.luogu.com.cn/paste/4oz6fep2) $G,H$ **本质相同**，当且仅当：

- 对于任意点对 $(u,v)$，若在图 $G$ 中从 $u$ 出发能走到 $v$，则在图 $H$ 中从 $u$ 出发能走到 $v$。反之若在图 $H$ 中从 $u$ 出发能走到 $v$，则在图 $G$ 中从 $u$ 出发也能走到 $v$。

若对于简单有向图 $G$，不存在其他简单有向图 $H$ 与其本质相同，则我们称图 $G$ 为 **单图**。

$T$ 次询问，每次询问给定一个正整数 $n$，请你回答 $n$ 个点的**有标号**单图数。

## 说明/提示

对于 $100\%$ 的数据满足：$1 \leq T,n \leq 1000$，$1\leq mod \leq 10^9$。

## 3. 核心难点辨析与解题策略

<difficulty_intro>
解决本题需突破三个核心难点：

1.  **关键点1：理解单图的结构约束**
    * **分析**：若存在三点链式结构（x→y→z），则x→z边的存在不影响传递闭包，违反单图定义。优质题解通过反例图示（如添加(5,2)边）直观说明该约束。
    * 💡 **学习笔记**：单图只能是孤立点、两点环或严格二分图（出点集→入点集）

2.  **关键点2：两点环的计数去重**
    * **分析**：将2k个点划分为k对时，若简单计算组合数会重复计数。(2k-1)!!的递推式（`f[i]=f[i-2]*(i-1)`）通过固定最小标号点选择配对点实现去重。
    * 💡 **学习笔记**：配对问题中"固定序法"可避免重复计数

3.  **关键点3：二分图结构的空边处理**
    * **分析**：每个出点必须至少连一条边（否则退化为入点），故方案为$(2^{m-i}-1)^i$而非$2^{i(m-i)}$。题解2通过"减1"操作排除全不连边的情况。
    * 💡 **学习笔记**："至少一条"约束常通过全集减补集处理

## 4. C++核心代码实现赏析

<code_intro_overall>
以下是综合优质题解提炼的通用实现，包含组合数预处理、两点环递推和二分图方案计算：
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：综合题解1的数学严谨性、题解2的代码可读性和题解3的变量命名规范
* **完整核心代码**：
```cpp
#include <bits/stdc++.h>
using namespace std;
typedef long long LL;
const int N = 1001;

int mod;
LL C[N][N], g[N], f[N]; // g:二分图方案, f:两点环方案

void init() {
    // 组合数预处理(杨辉三角)
    for (int i = 0; i < N; i++) {
        C[i][0] = 1;
        for (int j = 1; j <= i; j++)
            C[i][j] = (C[i-1][j] + C[i-1][j-1]) % mod;
    }
    
    // 两点环递推 f[2k] = (2k-1)!!
    f[0] = 1;
    for (int i = 2; i < N; i += 2) 
        f[i] = f[i-2] * (i-1) % mod;

    // 二分图方案 g[m]=ΣC(m,i)*(2^{m-i}-1)^i
    for (int m = 0; m < N; m++) {
        g[m] = 0;
        for (int i = 0; i <= m; i++) {
            LL ways = 1;
            if (m-i != 0) { // 计算(2^(m-i)-1)^i
                LL base = (1LL << (m-i)) % mod; // 2^(m-i)
                ways = (base - 1 + mod) % mod;   // 2^(m-i)-1
                LL tmp = 1;
                for (int j = 0; j < i; j++) 
                    tmp = tmp * ways % mod;
                ways = tmp;
            }
            g[m] = (g[m] + C[m][i] * ways) % mod;
        }
    }
}

int main() {
    int T, n;
    cin >> T >> mod;
    init();
    while (T--) {
        cin >> n;
        LL ans = 0;
        // 枚举两点环数量k
        for (int k = 0; 2*k <= n; k++) 
            ans = (ans + C[n][2*k] * f[2*k] % mod * g[n-2*k]) % mod;
        cout << ans << endl;
    }
    return 0;
}
```
* **代码解读概要**：
  1.  **预处理阶段**：`init()`函数预先计算组合数表（杨辉三角）、两点环方案（奇双阶乘）和二分图方案（含快速幂）
  2.  **主逻辑**：枚举两点环数量k，组合数选择2k个点，乘两点环方案和剩余点二分图方案
  3.  **边界处理**：`m-i=0`时特殊处理（`ways=1`），避免负指数幂

各优质题解的核心代码亮点分析：

**题解一：(Demeanor_Roy)**
* **亮点**：快速幂优化指数计算
* **核心代码片段**：
```cpp
inline int pwr(int x,int y) {
    int res=1;
    while(y) {
        if(y&1) res=(LL)res*x%mod;
        x=(LL)x*x%mod; y>>=1;
    }
    return res;
}
```
* **代码解读**：
  > 该快速幂函数通过二进制分解指数（`y>>=1`），将幂运算复杂度从O(n)降至O(logn)。`y&1`判断当前二进制位是否为1，是则累乘`res`。注意`(LL)`强制转换防止int溢出。
* 💡 **学习笔记**：模幂运算必用快速幂优化

**题解二：(0000pnc)**
* **亮点**：组合数学与代码的自然映射
* **核心代码片段**：
```cpp
for (int i=0; i<=n; i++) {
    ans += C[n][i] * pow(pow(2, n-i)-1, i);
}
```
* **代码解读**：
  > 直接对应数学公式$g(n)=\sum_{i=0}^n \binom{n}{i}(2^{n-i}-1)^i$的实现。`pow(2,n-i)`计算$2^{n-i}$，外层`pow`计算其i次方。虽然未用快速幂，但公式表达极其清晰。
* 💡 **学习笔记**：保持代码与数学公式的同构性提升可读性

**题解三：(听取MLE声一片)**
* **亮点**：孤立点的优雅处理
* **核心代码片段**：
```cpp
for(int j=0;j<=i;j++) {
    int x=j, y=i-j; // 出点x个, 入点y个
    f[i]=(f[i]+C[i][j]*ksm(ksm(2,y)-1,x))%mod;
}
```
* **代码解读**：
  > 将孤立点视为入点特例（y包含孤立点），通过`ksm(2,y)-1`统一处理"至少一条边"约束。变量命名`x/y`虽简短但配合注释清晰。

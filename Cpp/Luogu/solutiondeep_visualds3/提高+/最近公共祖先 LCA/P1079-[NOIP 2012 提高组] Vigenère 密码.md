# [NOIP 2012 提高组] Vigenère 密码

## 题目描述

16 世纪法国外交家 Blaise de Vigenère 设计了一种多表密码加密算法 Vigenère 密码。Vigenère 密码的加密解密算法简单易用，且破译难度比较高，曾在美国南北战争中为南军所广泛使用。

 
在密码学中，我们称需要加密的信息为明文，用 $M$ 表示；称加密后的信息为密文，用 $C$ 表示；而密钥是一种参数，是将明文转换为密文或将密文转换为明文的算法中输入的数据，记为 $k$。在 Vigenère 密码中，密钥 $k$ 是一个字母串，$k=k_1,k_2,…,k_n$。当明文 $M=m_1,m_2,…,m_n$ 时，得到的密文 $C=c_1,c_2,…,c_n$，其中 $c_i$=$m_i \operatorname{\circledR} k_i$，运算 $\circledR$ 的规则如下表所示：

Vigenère 加密在操作时需要注意：

1. $\circledR$ 运算忽略参与运算的字母的大小写，并保持字母在明文 $M$ 中的大小写形式；
2. 当明文 $M$ 的长度大于密钥 $k$ 的长度时，将密钥 $k$ 重复使用。

例如，明文 $M=\texttt{Helloworld}$，密钥 $k=\texttt{abc}$ 时，密文 $C=\texttt{Hfnlpyosnd}$。

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：`字符串处理与数学模拟`

🗣️ **初步分析**：
> 想象你是一个情报特工，手里有一份密文和密钥，需要还原原始信息。Vigenère密码就像个特殊的密码转盘——每个密文字母通过密钥字母"反向旋转"得到明文字母。核心在于发现加密表规律：**明文字母序号 = (密文字母序号 - 密钥字母序号 + 26) % 26**（序号从0开始计算）

- 题解呈现三种主要思路：①数学公式直接计算（qhr2023/Sinwind）②预先生成密码表再查表（LightCoder）③循环移位模拟（Jerry_zpl）。数学公式法最简洁高效，查表法最直观易懂。
- **可视化设计**：我们将创建像素风格的"密码破译机"动画。左侧输入带显示密文和循环密钥，中央转盘动态展示字母偏移过程（如密文'H'遇到密钥'a'时，转盘逆时针旋转0位→输出'H'），右侧输出明文字母。关键变量`当前字母序号`会以闪烁像素高亮，偏移计算过程显示为旋转动画。

## 3. 核心难点辨析与解题策略

### 难点一：理解密码表数学本质
**分析**：观察加密表发现，每个密钥字母对应一个凯撒密码偏移量。优质题解均将字母映射为0-25的数字（如A=0），通过`(密文数-密钥数+26)%26`解决循环偏移。  
💡 **学习笔记**：字母表本质是26进制的循环队列

### 难点二：处理大小写混合
**分析**：Sinwind题解给出黄金准则——**密文大小写决定明文大小写，密钥大小写不影响计算**。通过`isupper()`判断密文字母，计算时统一转大写，输出时还原大小写。  
💡 **学习笔记**：字符处理先标准化（转大写计算），后还原

### 难点三：密钥循环机制
**分析**：当密文长于密钥时，jerry_zpl题解展示高效解决方案：`key[i%key_len]`。类似音乐中的循环节奏，取模运算实现密钥无缝复用。  
💡 **学习笔记**：循环问题→取模是利器

### ✨ 解题技巧总结
- **数学建模法**：将字母操作转化为数字运算（A=0,B=1...），利用`(x+26)%26`处理循环
- **分治策略**：独立函数处理大小写转换（如Sinwind的`decryption`函数）
- **防御性编程**：输入长度>100时，`vector`比原生数组更安全（避免栈溢出）

## 4. C++核心代码实现赏析

### 通用核心实现
```cpp
#include <iostream>
#include <cctype>
using namespace std;

int main() {
    string key, cipher;
    cin >> key >> cipher;
    
    for(int i = 0; i < cipher.size(); i++) {
        char k = key[i % key.size()];
        // 统一转换为大写计算偏移
        int offset = (tolower(k) - 'a');
        // 保存原始大小写状态
        bool isUpper = isupper(cipher[i]);
        char base = isUpper ? 'A' : 'a';
        
        // 核心解密公式
        char plain = (cipher[i] - base - offset + 26) % 26 + base;
        cout << plain;
    }
    return 0;
}
```
**代码解读概要**：  
1. 密钥循环：`key[i % key.size()]`  
2. 大小写处理：`isupper()`判断+`base`变量  
3. 核心算法：`(cipher[i]-base-offset+26)%26+base`实现循环偏移

**题解二 (Sinwind)**  
```cpp
char decryption(char c, char k) {
    char temp_c = toupper(c);
    char temp_k = toupper(k);
    char base = isupper(c) ? 'A' : 'a';
    return base + (temp_c - temp_k + 26) % 26;
}
```
**亮点**：`toupper`标准化+`base`还原大小写  
**学习笔记**：标准化是处理字符混合的银弹

# [USACO1.1] 你的飞碟在这儿 Your Ride Is Here

## 题目描述

众所周知，在每一个彗星后都有一只 UFO。这些 UFO 时常来收集地球上的忠诚支持者。不幸的是，他们的飞碟每次出行都只能带上一组支持者。因此，他们要用一种聪明的方案让这些小组提前知道谁会被彗星带走。他们为每个彗星起了一个名字，通过这些名字来决定这个小组是不是被带走的那个特定的小组（你认为是谁给这些彗星取的名字呢？）。关于如何搭配的细节会在下面告诉你；你的任务是写一个程序，通过小组名和彗星名来决定这个小组是否能被那颗彗星后面的 UFO 带走。


小组名和彗星名都以下列方式转换成一个数字：最终的数字就是名字中所有字母的积，其中 $\texttt A$ 是 $1$，$\texttt Z$ 是 $26$。例如，$\texttt{USACO}$ 小组就是 $21 \times 19 \times 1 \times 3 \times 15=17955$。如果小组的数字 $\bmod 47$ 等于彗星的数字 $\bmod 47$,你就得告诉这个小组需要准备好被带走！（记住“$a \bmod b$”是 $a$ 除以 $b$ 的余数，例如 $34 \bmod 10$ 等于 $4$）


写出一个程序，读入彗星名和小组名并算出用上面的方案能否将两个名字搭配起来，如果能搭配，就输出 `GO`，否则输出 `STAY`。小组名和彗星名均是没有空格或标点的一串大写字母（不超过 $6$ 个字母）。

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：`模拟`（字符串处理与数值计算）

🗣️ **初步分析**：  
解决“你的飞碟在这儿”这道题，关键在于**模拟题目给定的计算流程**——就像按照菜谱一步步做菜一样，我们需要把字符串“转换成数字”、“计算乘积”、“取模比较”这三个步骤老老实实地实现出来。  
- **核心思路**：读取两个字符串（彗星名和小组名），将每个字符转换为对应的数字（A→1，B→2，…，Z→26），计算每个字符串的数字乘积，最后将两个乘积分别对47取模，若结果相等则输出`GO`，否则输出`STAY`。  
- **核心难点**：① 字符到数字的正确转换（比如A不能转换成0）；② 乘积的初始化（必须从1开始，否则所有结果都会是0）；③ 输入方式的选择（避免读取换行符等无关字符）。  
- **可视化设计思路**：我们可以用**8位像素风格**展示整个流程——比如左边是彗星名的“数字厨房”，右边是小组名的“数字厨房”，每个字符对应一个“食材块”（比如`C`对应`3`的像素块），“翻炒”（相乘）后得到“半成品”（乘积），再“过滤”（取模）得到“最终菜品”（模47的结果），最后比较两个“菜品”是否一样。  
- **游戏化元素**：加入“切菜音效”（字符转换）、“翻炒音效”（相乘）、“出锅音效”（取模），结果相同时播放“胜利音乐”（比如FC游戏的通关音效），否则播放“失败提示”（短促的蜂鸣）。

## 3. 核心难点辨析与解题策略

### 1. **关键点1：字符到数字的正确转换**  
* **分析**：题目要求A→1，B→2，…，Z→26。常见的转换方式有三种：① `ch - '@'`（@的ASCII是64，A是65，所以65-64=1）；② `ch - 'A' + 1`（A- A=0，加1得1）；③ `ch - 64`（64是@的ASCII，和第一种方式一样）。这三种方式都对，但要注意**不能漏掉“加1”**（比如`ch - 'A'`会得到0~25，不符合题目要求）。  
* 💡 **学习笔记**：字符转数字的关键是“找到基准值”（比如@或'A'），然后调整到题目要求的范围。

### 2. **关键点2：乘积的初始化**  
* **分析**：乘积是“连乘”，所以初始值必须是1（就像加法的初始值是0一样）。如果初始值是0，不管乘多少个数，结果都是0，这会导致所有测试用例都输出`GO`（因为0 mod 47等于0），肯定会错！  
* 💡 **学习笔记**：连乘的初始值是1，连加的初始值是0，这是编程的基本常识。

### 3. **关键点3：输入方式的选择**  
* **分析**：题目中的输入是两个字符串，中间用换行分隔。常见的输入方式有：① `cin >> string`（自动跳过空格和换行，适合本题）；② `gets`（读取整行，包括空格，但需要注意缓冲区问题）；③ `cin.get()`（逐个读取字符，适合处理特殊情况）。本题用`cin >> string`最方便，因为它会自动处理换行符。  
* 💡 **学习笔记**：选择输入方式时，要根据题目要求和数据格式来决定，优先选择简单、不易错的方式。


### ✨ 解题技巧总结  
- **技巧A：用`string`处理字符串**：`string`类有很多方便的函数（比如`length()`获取长度），比字符数组更易使用。  
- **技巧B：提前取模**：虽然本题数据很小，但乘积可能会很大（比如6个Z的乘积是26^6=308915776，超过int的范围吗？其实int通常是4字节，可以存到2^31-1=2147483647，所以没问题。但如果数据更大，提前取模（比如每乘一次就取模）可以避免溢出。  
- **技巧C：模块化代码**：把重复的功能写成函数（比如计算字符串的乘积），让主函数更简洁。


## 4. C++核心代码实现赏析

### 本题通用核心C++实现参考  
* **说明**：本代码综合了优质题解的思路，用`string`读取输入，`ch - 'A' + 1`转换字符，模块化函数计算乘积，逻辑清晰，适合初学者参考。  
* **完整核心代码**：  
  ```cpp
  #include <iostream>
  #include <string>
  using namespace std;

  int calculateProduct(string s) {
      int product = 1;
      for (char ch : s) { // 遍历字符串中的每个字符
          product *= (ch - 'A' + 1); // 转换为1~26的数字并相乘
      }
      return product % 47; // 取模47
  }

  int main() {
      string comet, team;
      cin >> comet >> team; // 读取两个字符串
      if (calculateProduct(comet) == calculateProduct(team)) {
          cout << "GO" << endl;
      } else {
          cout << "STAY" << endl;
      }
      return 0;
  }
  ```  
* **代码解读概要**：  
  1. `calculateProduct`函数：接收一个字符串，计算其字符乘积并取模47。  
  2. 主函数：读取彗星名和小组名，调用`calculateProduct`函数得到两个结果，比较后输出。


### 针对各优质题解的片段赏析

#### 题解一（来源：Vanyun）  
* **亮点**：用`@`的ASCII值转换字符，代码简洁。  
* **核心代码片段**：  
  ```cpp
  for (int i = 0; i < a.length(); i++) ans *= a[i] - '@'; 
  ```  
* **代码解读**：  
  `a[i]`是字符串中的第i个字符（比如`A`），`@`的ASCII是64，`A`的ASCII是65，所以`65-64=1`，刚好是`A`对应的数字。这个写法很巧妙，省去了“加1”的步骤。  
* 💡 **学习笔记**：可以用ASCII表中的“基准字符”来简化转换逻辑。

#### 题解二（来源：DrinkOnstage）  
* **亮点**：用`cin.get()`处理输入，灵活应对换行符。  
* **核心代码片段**：  
  ```cpp
  while ((v = cin.get()) != '\n') a *= v - 'A' + 1;
  ```  
* **代码解读**：`cin.get()`逐个读取字符，直到遇到换行符（`'\n'`）。这样可以处理任何长度的字符串，即使字符串中有空格（不过本题中没有空格）。  
* 💡 **学习笔记**：`cin.get()`适合处理需要逐字符读取的情况。

#### 题解三（来源：charliejiang_0w0_）  
* **亮点**：用函数封装转换逻辑，代码结构清晰。  
* **核心代码片段**：  
  ```cpp
  int convertToInt(string name) {
      int sum = 1;
      for (int i = 0; i < name.length(); i++) {
          sum *= charToInt1200(name[i]);
      }
      return sum % 47;
  }
  ```  
* **代码解读**：`convertToInt`函数负责将字符串转换为乘积并取模，主函数只需要调用这个函数，再比较结果即可。这种写法让代码更易读、易维护。  
* 💡 **学习笔记**：模块化代码是解决复杂问题的关键。

# [NOIP 2012 提高组] Vigenère 密码

## 题目描述

16 世纪法国外交家 Blaise de Vigenère 设计了一种多表密码加密算法 Vigenère 密码。Vigenère 密码的加密解密算法简单易用，且破译难度比较高，曾在美国南北战争中为南军所广泛使用。

 
在密码学中，我们称需要加密的信息为明文，用 $M$ 表示；称加密后的信息为密文，用 $C$ 表示；而密钥是一种参数，是将明文转换为密文或将密文转换为明文的算法中输入的数据，记为 $k$。在 Vigenère 密码中，密钥 $k$ 是一个字母串，$k=k_1,k_2,…,k_n$。当明文 $M=m_1,m_2,…,m_n$ 时，得到的密文 $C=c_1,c_2,…,c_n$，其中 $c_i$=$m_i \operatorname{\circledR} k_i$，运算 $\circledR$ 的规则如下表所示：

Vigenère 加密在操作时需要注意：

1. $\circledR$ 运算忽略参与运算的字母的大小写，并保持字母在明文 $M$ 中的大小写形式；
2. 当明文 $M$ 的长度大于密钥 $k$ 的长度时，将密钥 $k$ 重复使用。

例如，明文 $M=\texttt{Helloworld}$，密钥 $k=\texttt{abc}$ 时，密文 $C=\texttt{Hfnlpyosnd}$。

$$

### 💡 Kay的C++算法解析：Vigenère密码 深入学习指南 💡

#### 1. 题目解读与核心算法识别
✨ **本题主要考察**：`字符串处理与数学技巧应用`

🗣️ **初步分析**：
> Vigenère密码解密本质是模拟密码表的逆向操作，核心技巧是**数学运算与边界处理**。就像玩魔方时反向旋转还原一样，我们需要将密文字符按密钥字符"逆向旋转"对应位置。  
> - 题解普遍采用两种思路：**数学公式法**（字符转数字计算）和**查表法**（预建解密表）。数学法更简洁高效（O(n)时间复杂度），查表法更直观但代码量大  
> - 核心算法流程：  
>   ```plain
>   1. 循环处理每个密文字符c_i  
>   2. 取对应密钥字符k_j（j=i%len_k）  
>   3. 统一转小写计算：m_i = (c_i - k_j + 26) % 26  
>   4. 保留c_i的大小写特征输出  
>   ```  
> - 像素动画设计：采用**8位密码破译机**主题，左侧显示密文/密钥字符（像素方块），右侧显示ASCII数字计算过程。关键步骤高亮减法运算，负数时触发+26动画（红色闪烁），伴随"咔嗒"音效。成功解密时字符像素块翻转显示明文，并播放复古电子音效

#### 3. 核心难点辨析与解题策略
1. **难点：解密公式的符号处理**  
   * **分析**：当密文字符序号小于密钥字符时，直接相减会产生负数。优质题解均采用`(c-k+26)%26`策略，通过预加26确保结果非负  
   * 💡 **学习笔记**：模运算中的负数处理可类比钟表倒拨——倒拨3小时等于正拨21小时  

2. **难点：大小写一致性保持**  
   * **分析**：需保持密文原文大小写一致。高效做法如题解7：先用`tolower`计算，再根据原字符ASCII特征恢复大小写  
   * 💡 **学习笔记**：ASCII中大小写字母二进制后5位相同，可用`ch & 31`快速获取字母序  

3. **难点：密钥循环机制实现**  
   * **分析**：当密钥较短时需循环使用。所有优质题解均采用`i % key_len`实现循环索引，时间复杂度O(1)  
   * 💡 **学习笔记**：循环索引本质是数学上的周期性映射，类似昼夜更替  

**✨ 解题技巧总结**  
- **统一计算基准**：先转换为统一大小写再运算，避免条件嵌套  
- **模运算防负**：减法前预加模数（26）确保结果合法  
- **位运算加速**：利用`ch & 31`快速获取字母序（替代`-‘A’`）  
- **循环索引优化**：`%`运算实现密钥自动重复

#### 4. C++核心代码实现赏析
**本题通用核心C++实现参考**  
* **说明**：综合题解1/7优化，兼顾效率与可读性  
```cpp
#include <iostream>
#include <cctype>
using namespace std;

int main() {
    string k, c;
    cin >> k >> c;
    for (int i = 0; i < c.length(); i++) {
        char key = tolower(k[i % k.length()]);
        int offset = (key & 31) - 1; // 位运算取字母序
        char base = islower(c[i]) ? 'a' : 'A'; // 保留大小写
        char decrypted = (tolower(c[i]) - 'a' - offset + 26) % 26 + base;
        cout << decrypted;
    }
    return 0;
}
```
* **代码解读概要**：  
  1. 读入密钥`k`和密文`c`  
  2. 循环中：取密钥字符并统一小写，用位运算计算偏移量  
  3. 根据密文字符确定大小写基准  
  4. 核心计算：转小写→减偏移→加26防负→取模→转回原大小写  

**题解一核心片段**

```cpp
int t=(k[i%k.length()]&31)-1; // 位运算取密钥偏移
c[i]=(c[i]&31)-t>0?c[i]-t:c[i]-t+26; // 三目处理边界
```
* **亮点**：极简位运算与边界处理  
* **学习笔记**：`&31`等效`%32`但更快，适合字母序提取  

**题解二核心片段**  
```cpp
char decryption(char c, char k) {
    char temp_c = toupper(c); // 统一大写计算
    char temp_k = toupper(k);
    return (temp_c >= temp_k) ? 
           c - (temp_k - 'A') : // 直接减
           c - (temp_k - 'A') + 26; // 越界补偿
}
```
* **亮点**：函数封装提高可读性  
* **学习笔记**：函数化是复杂逻辑的"分治策略"

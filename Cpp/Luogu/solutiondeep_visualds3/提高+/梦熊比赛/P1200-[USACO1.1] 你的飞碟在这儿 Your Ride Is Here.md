# 题目信息

# [USACO1.1] 你的飞碟在这儿 Your Ride Is Here

## 题目描述

众所周知，在每一个彗星后都有一只 UFO。这些 UFO 时常来收集地球上的忠诚支持者。不幸的是，他们的飞碟每次出行都只能带上一组支持者。因此，他们要用一种聪明的方案让这些小组提前知道谁会被彗星带走。他们为每个彗星起了一个名字，通过这些名字来决定这个小组是不是被带走的那个特定的小组（你认为是谁给这些彗星取的名字呢？）。关于如何搭配的细节会在下面告诉你；你的任务是写一个程序，通过小组名和彗星名来决定这个小组是否能被那颗彗星后面的 UFO 带走。


小组名和彗星名都以下列方式转换成一个数字：最终的数字就是名字中所有字母的积，其中 $\texttt A$ 是 $1$，$\texttt Z$ 是 $26$。例如，$\texttt{USACO}$ 小组就是 $21 \times 19 \times 1 \times 3 \times 15=17955$。如果小组的数字 $\bmod 47$ 等于彗星的数字 $\bmod 47$,你就得告诉这个小组需要准备好被带走！（记住“$a \bmod b$”是 $a$ 除以 $b$ 的余数，例如 $34 \bmod 10$ 等于 $4$）


写出一个程序，读入彗星名和小组名并算出用上面的方案能否将两个名字搭配起来，如果能搭配，就输出 `GO`，否则输出 `STAY`。小组名和彗星名均是没有空格或标点的一串大写字母（不超过 $6$ 个字母）。

## 3. 核心难点辨析与解题策略

1.  **字母转换的准确性**
    * **分析**：必须确保A→1, B→2,..., Z→26的精确映射。优质解法均使用`c-'A'+1`或等效方法（如`c-'@'`），避免硬编码ASCII值
    * 💡 **学习笔记**：字符运算依赖ASCII码特性，理解'A'的ASCII值(65)是关键

2.  **乘积初始值陷阱**
    * **分析**：累乘变量必须初始化为1（非0），否则结果恒为0。Vanyun特别强调此易错点
    * 💡 **学习笔记**：乘法累加初始值=1，加法累加初始值=0

3.  **取模运算时机**
    * **分析**：比较前必须先对乘积取模（`product % 47`），而非比较乘积本身。题目明确要求"mod 47"
    * 💡 **学习笔记**：取模可防止整数溢出，但本题最大乘积26⁶≈3e8，int类型可容纳

### ✨ 解题技巧总结
- **字符运算技巧**：活用`c-'A'+1`实现字母序数转换
- **输入处理优化**：流式读取(cin.get())节省内存
- **模块化封装**：将重复操作(如字母转换)提取为函数
- **边界防御**：显式初始化变量，注释关键陷阱点

## 4. C++核心代码实现赏析

**本题通用核心C++实现参考**
* **说明**：综合优质题解思路，采用最清晰的实现方式
* **完整核心代码**：
```cpp
#include <iostream>
#include <string>
using namespace std;

int main() {
    string comet, group;
    cin >> comet >> group;
    
    int cometCode = 1, groupCode = 1;
    for (char c : comet) cometCode *= (c - 'A' + 1);
    for (char c : group) groupCode *= (c - 'A' + 1);
    
    if (cometCode % 47 == groupCode % 47) 
        cout << "GO" << endl;
    else 
        cout << "STAY" << endl;
    
    return 0;
}
```
* **代码解读概要**：
  1. 读取彗星名和小组名字符串
  2. 初始化乘积变量为1
  3. 遍历字符串：每个字母转换为1-26的数字并累乘
  4. 对乘积取模后比较结果
  5. 输出匹配结果

---

**题解一（Vanyun）**
* **亮点**：ASCII技巧转换（`- '@'`代替`- 'A' + 1`）
* **核心代码片段**：
```cpp
for(int i=0; i<a.length(); i++) 
    ans *= a[i] - '@';
```
* **代码解读**：
  > `'@'`的ASCII值为64，'A'为65，故`A-'@'=1`。这比显式`-64`或`-'A'+1`更直观且避免魔数
* 💡 **学习笔记**：利用字符常量简化计算，但需确保环境ASCII兼容

**题解二（DrinkOnstage）**
* **亮点**：流式读取省内存
* **核心代码片段**：
```cpp
while((v=cin.get())!='\n') 
    a *= v-'A'+1;
```
* **代码解读**：
  > `cin.get()`逐字符读取，遇换行符停止。避免存储整个字符串，对长输入更优（但本题≤6字母）
* 💡 **学习笔记**：输入结束符处理技巧，尤其适合行式输入

**题解三（Carl__2015）**
* **亮点**：函数封装转换逻辑
* **核心代码片段**：
```cpp
int turn(char c) {
```

* **代码解读**：
  > 将字母转换提取为`turn()`函数，隔离实现细节。主循环更简洁，且修改转换逻辑只需调整一处

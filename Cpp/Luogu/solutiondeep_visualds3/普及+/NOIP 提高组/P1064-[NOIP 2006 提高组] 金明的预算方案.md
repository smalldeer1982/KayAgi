# 题目信息

# [NOIP 2006 提高组] 金明的预算方案

## 题目描述

金明今天很开心，家里购置的新房就要领钥匙了，新房里有一间金明自己专用的很宽敞的房间。更让他高兴的是，妈妈昨天对他说：“你的房间需要购买哪些物品，怎么布置，你说了算，只要不超过 $n$ 元钱就行”。今天一早，金明就开始做预算了，他把想买的物品分为两类：主件与附件，附件是从属于某个主件的，下表就是一些主件与附件的例子：

| 主件 | 附件 |
| :----------: | :----------: |
| 电脑 | 打印机，扫描仪 |
| 书柜 | 图书 |
| 书桌 | 台灯，文具 |
| 工作椅 | 无 |

如果要买归类为附件的物品，必须先买该附件所属的主件。每个主件可以有 $0$ 个、$1$ 个或 $2$ 个附件。每个附件对应一个主件，附件不再有从属于自己的附件。金明想买的东西很多，肯定会超过妈妈限定的 $n$ 元。于是，他把每件物品规定了一个重要度，分为 $5$ 等：用整数 $1 \sim 5$ 表示，第 $5$ 等最重要。他还从因特网上查到了每件物品的价格（都是 $10$ 元的整数倍）。他希望在不超过 $n$ 元的前提下，使每件物品的价格与重要度的乘积的总和最大。

设第 $j$ 件物品的价格为 $v_j$，重要度为 $w_j$，共选中了 $k$ 件物品，编号依次为 $j_1,j_2,\dots,j_k$，则所求的总和为：

$$v_{j_1} \times w_{j_1}+v_{j_2} \times w_{j_2}+ \dots +v_{j_k} \times w_{j_k}$$

请你帮助金明设计一个满足要求的购物单。


## 说明/提示

对于全部的测试点，保证 $1 \leq n \leq 3.2 \times 10^4$，$1 \leq m \leq 60$，$0 \leq v_i \leq 10^4$，$1 \leq p_i \leq 5$，$0 \leq q_i \leq m$，答案不超过 $2 \times 10^5$。

#### 1. 题目解读与核心算法识别
✨ **本题主要考察**：`依赖背包问题`（动态规划变种）

🗣️ **初步分析**：
> 解决这道题的关键在于处理物品间的依赖关系（主件+附件），并将其转化为分组背包问题。想象你在整理行李箱：主件是行李箱本体，附件是各种收纳袋——必须先放行李箱才能放收纳袋。在本题中，我们为每个主件设计5种"打包方案"（不买/只主件/主+附1/主+附2/全买），然后用01背包思路选择最优组合。

- **核心流程**：① 预处理主附件关系 → ② 枚举每组打包方案 → ③ 背包DP求最优解  
- **可视化设计**：采用8位商店像素游戏风格，展示货架（主件+附件图标）和购物车（背包状态）。高亮当前处理的主件组，播放"叮"音效时显示价格/价值计算过程，用不同颜色区分五种打包方案的选择效果。

#### 2. 精选优质题解参考
**题解一（来源：Anguei）**  
* **点评**：此解法思路清晰直白，用二维数组预处理主附件关系堪称典范。亮点在于用lambda表达式`cost2()`/`cost3()`简化状态转移方程，避免冗长下标计算（如`v[i][0]+v[i][1]`）。代码规范：常量名前缀`kMaxN`明确作用域，逆序背包循环严格遵循01背包模板。实践价值极高——直接用于竞赛时只需注意附件可能晚于主件输入的特殊处理。

**题解二（来源：tuntunQwQ）**  
* **点评**：采用二维DP定义`f[i][j]`更符合问题直观，对初学者更友好。亮点是完整保留了物品索引维度，方便调试时追踪决策路径。虽然空间复杂度O(nm)略高于一维解法，但其清晰的五情况转移方程（直接并列if语句）对理解分组背包本质极具教学意义。

**题解三（来源：封禁用户）**  
* **点评**：结构体存储方案`node{ v1,v2,v3,s1,s2,s3 }`是亮点，将业务逻辑（主附件关系）与算法逻辑（DP转移）解耦。虽然二维DP稍显冗余，但"空附件默认为0"的严谨处理值得学习，特别适合处理边界数据（如单附件主件）。

#### 3. 核心难点辨析与解题策略
1. **依赖关系建模**  
   * **分析**：优质题解均用二维数组/结构体存储主附件，关键是通过`q_i`字段建立映射。注意附件输入顺序可能先于主件（如题解Anguei用`v[_q][1]`动态填充）
   * 💡 **学习笔记**：附件关系本质是树形结构（父节点=主件），但本题限制（最多2附件）允许扁平化存储

2. **分组方案枚举**  
   * **分析**：必须覆盖五种情况且避免组合冲突。如题解tuntunQwQ用独立if判断确保逻辑完备性，核心是计算组合价格`v_comb`和价值`w_comb`
   * 💡 **学习笔记**：组合数=2^(附件数)，本题最大4种有效组合（附件数≤2）

3. **背包状态转移**  
   * **分析**：一维DP需逆序循环防覆盖（Anguei），二维DP可正序但需继承前状态（tuntunQwQ）。关键推导：`dp[j] = max(..., dp[j - v_comb] + w_comb)`
   * 💡 **学习笔记**：当附件存在时，只更新主件索引对应的DP状态

✨ **解题技巧总结**  
- **技巧A（问题分解）**：将复杂依赖转化为"组内枚举+组间DP"两阶段  
- **技巧B（代码健壮性）**：预处理时初始化数组默认值（如`v[i][1]=0`）  
- **技巧C（空间优化）**：优先尝试一维DP（题解Anguei），N较大时节省80%内存

#### 4. C++核心代码实现赏析
**本题通用核心实现参考**  
* **说明**：综合优质题解优化，一维DP+lambda表达式平衡效率与可读性
* **完整核心代码**：
```cpp
#include <iostream>
#include <algorithm>
const int kMaxN = 32000, kMaxM = 60;
int v[kMaxM+5][3], p[kMaxM+5][3], f[kMaxN+5];

int main() {
    int n, m; std::cin >> n >> m;
    // 预处理主附件关系
    for(int i=1; i<=m; ++i) {
        int _v, _p, _q; std::cin >> _v >> _p >> _q;
        if(!_q) v[i][0]=_v, p[i][0]=_p;  // 主件
        else if(!v[_q][1]) v[_q][1]=_v, p[_q][1]=_p; // 第一个附件
        else v[_q][2]=_v, p[_q][2]=_p;   // 第二个附件
    }
    // 分组背包DP
    for(int i=1; i<=m; ++i) for(int j=n; j>=0; --j) {
        auto cost2 = [&](int x,int y){ return v[i][x]+v[i][y]; };
        auto val = [&](int x){ return v[i][x]*p[i][x]; };
        if(j >= v[i][0]) 
            f[j] = std::max(f[j], f[j-v[i][0]] + val(0));
        if(v[i][1] && j>=cost2(0,1)) 
            f[j] = std::max(f[j], f[j-cost2(0,1)] + val(0)+val(1));
        if(v[i][2] && j>=cost2(0,2)) 
            f[j] = std::max(f[j], f[j-cost2(0,2)] + val(0)+val(2));
        if(v[i][1]&&v[i][2] && j>=cost2(0,1)+v[i][2]) 
            f[j] = std::max(f[j], f[j-cost2(0,1)-v[i][2]] + val(0)+val(1)+val(2));
    }
    std::cout << f[n];
}
```
* **代码解读概要**：  
  1. **预处理阶段**：用`v[i][0..2]`存储主件及附件价格，`p[i][0..2]`存储重要度  
  2. **DP阶段**：外层遍历物品组，内层逆序遍历预算  
  3. **Lambda工具**：`cost2`计算组合价格，`val`计算价值贡献  
  4. **五情况转移**：按组合总价是否≤当前预算，更新最优解  

**题解一（Anguei）片段赏析**  
* **亮点**：Lambda表达式消除重复计算
* **核心代码片段**：
```cpp
auto cost2 = [v,p,i](int x,int y){ return v[i][x]+v[i][y]; };
auto rpp = [v,p,i](int x){ return v[i][x]*p[i][x]; };
if(j >= cost2(0,1)) 
    f[j]=max(f[j], f[j-cost2(0,1)] + rpp(0)+rpp(1));
```
* **代码解读**：  
  > 这里用Lambda将"组合价格计算"和"价值计算"抽象为工具函数，使转移方程`f[j]=max(...)`保持简洁。`cost2(0,1)`动态计算主件+附件1的总价，`rpp(0)+rpp(1)`计算联合价值，避免重复书写数组下标。  
* 💡 **学习笔记**：合理封装工具函数提升DP代码可维护性

**题解二（tuntunQwQ）片段赏析**  
* **亮点**：二维DP状态转移直观展现决策过程
* **核心代码片段**：
```cpp
for(int i=1;i<=m;i++) for(int j=0;j<=n;j++){
    f[i][j]=f[i-1][j];  // 继承之前状态
    if(v[i][0]<=j) 
        f[i][j]=max(f[i][j], f[i-1][j-v[i][0]]+v[i][0]*p[i][0]);
}
```
* **代码解读**：  
  > 二维DP中`f[i][j]`显式存储考虑前i组物品时的最优解。第一行`f[i][j]=f[i-1][j]`确保不选当前组时的结果继承，后续if语句在此基础上更新五种选择。虽然空间效率低，但调试时可查看整个DP表。  
* 💡 **学习笔记**：二维DP是理解状态转移的"教学用脚手架"

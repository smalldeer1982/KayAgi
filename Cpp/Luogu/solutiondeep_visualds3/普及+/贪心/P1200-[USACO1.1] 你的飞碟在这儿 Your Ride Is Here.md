# 题目信息

# [USACO1.1] 你的飞碟在这儿 Your Ride Is Here

## 题目描述

众所周知，在每一个彗星后都有一只 UFO。这些 UFO 时常来收集地球上的忠诚支持者。不幸的是，他们的飞碟每次出行都只能带上一组支持者。因此，他们要用一种聪明的方案让这些小组提前知道谁会被彗星带走。他们为每个彗星起了一个名字，通过这些名字来决定这个小组是不是被带走的那个特定的小组（你认为是谁给这些彗星取的名字呢？）。关于如何搭配的细节会在下面告诉你；你的任务是写一个程序，通过小组名和彗星名来决定这个小组是否能被那颗彗星后面的 UFO 带走。


小组名和彗星名都以下列方式转换成一个数字：最终的数字就是名字中所有字母的积，其中 $\texttt A$ 是 $1$，$\texttt Z$ 是 $26$。例如，$\texttt{USACO}$ 小组就是 $21 \times 19 \times 1 \times 3 \times 15=17955$。如果小组的数字 $\bmod 47$ 等于彗星的数字 $\bmod 47$,你就得告诉这个小组需要准备好被带走！（记住“$a \bmod b$”是 $a$ 除以 $b$ 的余数，例如 $34 \bmod 10$ 等于 $4$）


写出一个程序，读入彗星名和小组名并算出用上面的方案能否将两个名字搭配起来，如果能搭配，就输出 `GO`，否则输出 `STAY`。小组名和彗星名均是没有空格或标点的一串大写字母（不超过 $6$ 个字母）。

#### 3. 核心难点辨析与解题策略
1. **字符转换陷阱**  
   * **分析**：字母→数字需精确对应（A=1, B=2...）。易错点在于：  
     - 用`ch-'A'`漏加1 → 导致A=0  
     - 混淆ASCII值（误用`'a'`处理大写字母）  
   * 💡 **学习笔记**：`ch - 'A' + 1`是安全牌，`ch - '@'`是进阶技巧

2. **累乘初始化错误**  
   * **分析**：乘积变量初始值必须为1（非0）。因为`0*任何数=0`，会导致结果全0。优质题解都在变量声明时显式初始化`int ans=1`  
   * 💡 **学习笔记**：乘法的单位元是1，加法的单位元是0——这是数学基石！

3. **输入边界处理**  
   * **分析**：字符串长度≤6但需预留结束符`\0`。清蒸充电器题解发现：声明`char[6]`会越界，`char[7]`才安全。`getline`或`scanf`需明确输入截止条件  
   * 💡 **学习笔记**：C++中字符数组长度应≥内容长度+1

### ✨ 解题技巧总结
- **ASCII映射法**：字母运算本质是整数运算（`A=65, B=66...`）  
- **防御性初始化**：数值型变量声明时立即赋值  
- **流式处理**：数据无需全存储后再计算（参考DrinkOnstage解法）  
- **模块化验证**：复杂逻辑拆解为函数，逐单元测试

#### 4. C++核心代码实现赏析
**通用核心实现**  
```cpp
#include <iostream>
#include <string>
using namespace std;

int main() {
    string s1, s2;
    cin >> s1 >> s2;  // 输入彗星名和小组名
    
    long prod1 = 1, prod2 = 1;  // 初始化累乘器
    for (char c : s1) prod1 *= c - 'A' + 1;  // 字符转数字累乘
    for (char c : s2) prod2 *= c - 'A' + 1;
    
    cout << (prod1 % 47 == prod2 % 47 ? "GO" : "STAY");
    return 0;
}
```
**代码解读概要**：  
> 1. 读取两个字符串  
> 2. 通过范围循环遍历每个字符，`c-'A'+1`转为1-26的数字  
> 3. 累乘后对47取模比较  
> 4. 三目运算符直接输出结果  

**题解片段赏析**  
**题解一：Vanyun**  
```cpp
for(int i=0;i<a.length();i++) 
    ans *= a[i] - '@';  // '@'的ASCII=64
```
**解读**：  
> 魔法在于`@`位于`A`前一位（ASCII 64），减`@`等效于`ch-64`，使`A=1`。比主流的`ch-'A'+1`少一次运算，但需记住ASCII表  

**题解二：DrinkOnstage**  
```cpp
while((v=cin.get())!='\n') 
    a *= v - 'A' + 1;
```
**解读**：  
> `cin.get()`逐个读取字符直到回车。优势：  
> - 避免存储整个字符串  
> - 自动处理任意长度输入  
> 注意：输入结束需两次回车  

**题解三：muyang_233**  
```cpp
int ascii(char ch) {
    return ch - '@';  // 封装转换逻辑
}
void count() {
    for(int i=0;i<strlen(a);i++)
        ans1 *= ascii(a[i]);  // 调用转换函数
}
```
**解读**：  
> 将字符转换封装为独立函数，体现"单一职责原则"。主函数保持简洁，调试时可单独测试`ascii()`

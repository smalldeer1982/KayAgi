# [NOI Online #1 入门组] 文具订购

## 题目描述

小明的班上共有 $n$ 元班费，同学们准备使用班费集体购买 $3$ 种物品：

1. 圆规，每个 $7$ 元。
2. 笔，每支 $4$ 元。
3. 笔记本，每本 $3$ 元。

小明负责订购文具，设圆规，笔，笔记本的订购数量分别为 $a,b,c$，他订购的原则依次如下：

1. $n$ 元钱必须正好用光，即 $7a+4b+3c=n$。
2. 在满足以上条件情况下，成套的数量尽可能大，即 $a,b,c$ 中的最小值尽可能大。
3. 在满足以上条件情况下，物品的总数尽可能大，即 $a+b+c$ 尽可能大。

请你帮助小明求出满足条件的最优方案。可以证明若存在方案，则最优方案唯一。

 

## 说明/提示

#### 样例输入输出 3 解释

$a=2,b=4,c=1$ 也是满足条件 $1,2$ 的方案，但对于条件 $3$，该方案只买了 $7$ 个物品，不如 $a=1,b=2,c=6$ 的方案。 

#### 数据规模与约定

- 对于测试点 $1 \sim 6$，保证 $n \leq 14$。
- 对于测试点 $7 \sim 12$，保证 $n$ 是 $14$ 的倍数。
- 对于测试点 $13 \sim 18$，保证 $n \leq 100$。
- 对于全部的测试点，保证 $0 \leq n \leq 10^5$。

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：贪心策略 + 分类讨论  

🗣️ **初步分析**：  
解决“文具订购”问题的关键，是**按优先级满足三个条件**：  
1. **花光所有钱**：必须满足 \(7a + 4b + 3c = n\)；  
2. **成套数量最大**：\(a、b、c\) 的最小值要尽可能大（可以理解为“买尽可能多的‘一套’文具——1个圆规+1支笔+1本笔记本，共14元”）；  
3. **总数最大**：在满足前两个条件的基础上，尽量买便宜的文具（3元的笔记本），因为同样的钱能买更多件。  

### 核心算法逻辑
- **第一步**：计算最多能买多少套（\(max\_set = n // 14\)），此时 \(a = b = c = max\_set\)，剩余的钱为 \(remain = n \% 14\)；  
- **第二步**：处理余数 \(remain\)：如果余数是1、2、5（无法用3或4凑出），则减少1套（\(max\_set -= 1\)），余数变为 \(remain + 14\)；  
- **第三步**：用余数买尽可能多的笔记本（3元），剩下的钱买笔（4元），确保总数最大。  

### 可视化设计思路
我们会用**8位像素风格**模拟“文具店购物”：  
- 屏幕左侧显示当前剩余的钱（像素数字），右侧显示 \(a、b、c\) 的数量（像素方块，分别用蓝、红、绿表示）；  
- 成套购买时，钱减少14，三个颜色的方块各加1（伴随“叮~”的音效）；  
- 处理余数时，比如余数19，先买1支笔（钱减4，红方块加1），再买5本笔记本（钱减15，绿方块加5），每一步都用**闪烁**高亮当前操作的文具；

## 3. 核心难点辨析与解题策略

### 1. 难点1：理解“成套数量尽可能大”  
- **分析**：“成套”不是指“必须买整14元”，而是指a、b、c的最小值要大。比如n=33，买1套（a=1,b=1,c=1）后，余数19可以买1支笔+5本笔记本，此时a=1,b=2,c=6，最小值是1，比买0套（a=0,b=8,c=1，最小值0）更优。  
- 💡 **学习笔记**：优先最大化a、b、c的最小值，等价于优先买“一套”文具（14元）。


### 2. 难点2：处理无法凑出的余数（1、2、5）  
- **分析**：如果余数是1、2、5，无法用3或4凑出，此时需要**减少1套**（多出来14元），余数变为1+14=15、2+14=16、5+14=19，这些数都可以用3或4凑出。比如n=15，原套数1（余数1），减少1套后余数15，买5本笔记本（c=5），此时a=0,b=0,c=5，满足条件。  
- 💡 **学习笔记**：遇到无法凑出的余数，退一步（减少1套）就能解决。


### 3. 难点3：保证总数最大  
- **分析**：同样的钱，买便宜的文具（3元笔记本）能得到更多件。比如余数8，买2支笔（4×2=8，总数2）不如买1支笔+1本笔记本（4+3=7，不够）？不对，余数8应该买2支笔（总数2）吗？不，等一下——余数8的最优解是买2支笔吗？或者有没有更好的？比如余数8=4×2（总数2），或者3×1+4×1=7（不够），所以确实买2支笔。哦，对，**只有当余数能被3或4整除时，优先买3元的**。比如余数6=3×2（总数2），比4×1+3×0.5（不行）好。  
- 💡 **学习笔记**：总数最大=尽量买单价低的文具。


### ✨ 解题技巧总结
1. **优先级意识**：按题目条件的顺序解决问题（先花光钱，再套数，再总数）；  
2. **数学简化**：利用“3+4=7”的规律，把圆规换成笔+笔记本，简化问题；  
3. **分类讨论**：对余数逐一处理，把复杂问题拆成小问题。


## 4. C++核心代码实现赏析

### 本题通用核心C++实现参考（来自OMG_wc的题解，简洁高效）
* **说明**：该代码通过数学推导，直接枚举a的最大值，然后计算b和c，逻辑严谨且效率高。  
* **完整核心代码**：
```cpp
#include <iostream>
using namespace std;

int main() {
    int n;
    cin >> n;
    // 枚举a的最大可能值（从n//14往下）
    for (int a = n / 14; a >= 0; a--) {
        int t = n - a * 14; // 剩余的钱（需要用4b+3c凑）
        // 枚举b的可能值，计算c是否为整数
        for (int b = 0; 4 * b <= t; b++) {
            if ((t - 4 * b) % 3 == 0) {
                int c = (t - 4 * b) / 3;
                cout << a << " " << a + b << " " << a + c << endl;
                return 0;
            }
        }
    }
    cout << "-1" << endl; // 无解
    return 0;
}
```
* **代码解读概要**：  
  1. 先计算a的最大可能值（n//14）；  
  2. 计算剩余的钱t（需要用4b+3c凑）；  
  3. 枚举b的数量，检查t-4b是否能被3整除（即c是否为整数）；  
  4. 找到第一个符合条件的解，输出并结束程序（因为a是从大到小枚举的，第一个解就是最优解）。


### 题解一（lzqy_）核心代码片段赏析
* **亮点**：用分类讨论直接处理所有余数，思路直白。  
* **核心代码片段**：
```cpp
ans = n / 14;
n %= 14;
if (n < 6 && n != 4 && n != 3 && n != 0) {
    ans--;
    n += 14;
}
if (n == 3) cout << ans << " " << ans << " " << ans + 1 << endl;
if (n == 4) cout << ans << " " << ans + 1 << " " << ans << endl;

```
* **代码解读**：  
  - 先计算套数ans，余数n；  
  - 如果余数无法凑出（1、2、5），则ans减1，余数加14；  
  - 对每个余数，直接输出对应的a、b、c（比如n=3，买1本笔记本，c=ans+1）。  
* 💡 **学习笔记**：分类讨论是解决“多情况问题”的有效方法，即使代码长，逻辑也很清晰。


### 题解二（江户川·萝卜）核心代码片段赏析
* **亮点**：用打表法把余数对应的解存进数组，代码简洁。  
* **核心代码片段**：
```cpp
int m[14][3] = {
    {0,0,0}, {-1,-1,4}, {-1,0,3}, {0,0,1},
    {0,1,0}, {-1,0,4}, {0,0,2}, {0,1,1},
    {0,2,0}, {0,0,3}, {1,0,1}, {0,2,1},
    {0,0,4}, {0,1,3}
};
a = b = c = n / 14;
n %= 14;
if (n == 1 || n == 2 || n == 5) {
    a--; b--; c--;
}
a += m[n][0]; b += m[n][1]; c += m[n][2];
```
* **代码解读**：  
  - m数组存的是余数对应的解（比如m[3][2] = 1，表示余数3时c加1）；  
  - 处理无法凑出的余数（1、2、5），减少1套；  
  - 把余数对应的解加到a、b、c上。  
* 💡 **学习笔记**：打表法适合“情况有限”的问题，能快速得到答案。

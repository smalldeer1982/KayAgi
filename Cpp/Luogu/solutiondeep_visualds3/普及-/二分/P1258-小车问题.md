# 题目信息

# 小车问题

## 题目描述

甲、乙两人同时从 A 地出发要尽快同时赶到 B 地。出发时 A 地有一辆小车，可是这辆小车除了驾驶员外只能带一人。已知甲、乙两人的步行速度一样，且小于车的速度。问：怎样利用小车才能使两人尽快同时到达。

## 说明/提示

#### 数据规模与约定

对于 $100\%$ 的数据，保证 $0 \leq s, a, b \leq 10^9$。

## 样例 #1

### 输入

```
120 5 25
```

### 输出

```
9.600000
```

#### 3. 核心难点辨析与解题策略
1. **难点1：时间等式构建**  
   * **分析**：需明确小车返回接乙时，甲步行时间=小车往返时间+乙乘车时间。优质题解均通过`(s-x)/a = 2*(相遇时间) + (s-x)/b`建立等式。  
   * 💡 **学习笔记**：运动时间等式是优化问题的核心纽带。

2. **难点2：公式等价性验证**  
   * **分析**：表面不同的公式`x=(a+b)s/(3a+b)`与`x=2as/(3a+b)`实为代数等价变形（分子分母同乘(a+b)）。  
   * 💡 **学习笔记**：数学推导需检查多解本质一致性。

3. **难点3：方法选择依据**  
   * **分析**：数学推导法（O(1)）适合公式明确场景；二分法（O(log n)）适用未知模型或高维扩展。  
   * 💡 **学习笔记**：问题规模决定解法——小数据推公式，大数据用迭代。

✨ **解题技巧总结**  
- **技巧1：图形辅助建模**——画时间轴明确阶段分割点  
- **技巧2：边界鲁棒性**——验证`s=0`或`a=b`的退化情况

#### 4. C++核心代码实现赏析
**通用核心实现参考**  
```cpp
#include <iostream>
#include <iomanip>
using namespace std;
int main() {
    double s, a, b;
    cin >> s >> a >> b;
    double x = (a + b) * s / (3 * a + b); // 最优下车点
    double time = x / b + (s - x) / a;    // 总时间=乘车段+步行段
    cout << fixed << setprecision(6) << time;
    return 0;
}
```
**代码解读概要**：输入距离s、步行速a、车速b；计算关键下车点x；总时间为x段乘车时间与剩余步行时间之和。精度控制保证6位小数。

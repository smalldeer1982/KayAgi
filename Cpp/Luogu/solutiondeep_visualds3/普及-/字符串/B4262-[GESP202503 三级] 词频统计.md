# 题目信息

# [GESP202503 三级] 词频统计

## 题目描述

在文本处理中，统计单词出现的频率是一个常见的任务。现在，给定 $n$ 个单词，你需要找出其中出现次数最多的单词。在本题中，忽略单词中字母的大小写（即 `Apple`、`apple`、`APPLE`、`aPPle` 等均视为同一个单词）。

请你编写一个程序，输入 $n$ 个单词，输出其中出现次数最多的单词。

## 说明/提示

对于所有测试点，$1\leq n\leq 100$，每个单词的长度不超过 $30$，且仅由大小写字母组成。

## 样例 #1

### 输入

```
6
Apple
banana
apple
Orange
banana
apple```

### 输出

```
apple```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：`枚举与编程技巧应用` (涉及字符串处理、数据统计)

🗣️ **初步分析**：
解决“词频统计”的关键在于：先统一单词的大小写（比如把所有字母转成小写），再统计每个单词的出现次数，最后找出次数最多的那个。  
简单来说，这就像给每个“小客人”（单词）发一个“小写名字牌”，然后让它们去“计数器桶”里排队——每个桶对应一个唯一的小写单词，每来一个客人就往桶里加1。最后看看哪个桶里的数字最大，对应的就是出现次数最多的单词。  

题解中提到了两种思路：  
- **暴力枚举法**：逐个比较每个单词和其他所有单词，统计出现次数（时间复杂度O(n²)）。  
- **STL map法**：用`map<string, int>`自动统计（时间复杂度O(n log n)，更高效）。  

核心算法流程是：**小写转换→统计次数→找最大值**。可视化时，我们可以用像素动画展示“小写魔法”（字母逐个变灰小写）、“桶计数”（单词被丢进对应桶，数字跳动）、“找最大”（桶上数字高亮放大）等关键步骤。动画采用8位像素风，比如用FC游戏的“宝箱”代表map的桶，每个单词是像素小人，转换大小写时会有“闪光”效果，统计时“叮”的音效提示计数+1。

## 3. 核心难点辨析与解题策略

<difficulty_intro>
解决这道题时，我们需要重点关注以下几个关键点：
</difficulty_intro>

1.  **关键点1：如何统一单词的大小写？**  
    * **分析**：题目要求忽略大小写，所以必须将所有单词转为统一格式（如全小写）。可以用`tolower()`函数逐个字符转换，或手动计算（如大写字母`- 'A' + 'a'`）。例如，`Apple`转小写后是`apple`，这样才能和其他`apple`正确比较。  
    * 💡 **学习笔记**：处理字符串时，统一格式是避免“误判”的关键！

2.  **关键点2：选择高效的统计方式**  
    * **分析**：暴力法需要双重循环（O(n²)），适合小数据；`map`法自动统计（O(n log n)），更高效。如果n很大（比如1e5），必须用`map`或`unordered_map`。  
    * 💡 **学习笔记**：数据结构选得好，代码效率高！

3.  **关键点3：处理“并列最多”的情况**  
    * **分析**：题目可能隐含“唯一最多”，但实际中需考虑多个单词次数相同的情况。此时应选字典序最小的，或按题目要求处理。本题样例输出选`apple`（可能因先出现），需注意代码逻辑是否覆盖。  
    * 💡 **学习笔记**：边界条件要考虑，测试用例多验证！

### ✨ 解题技巧总结
- **统一格式优先**：处理字符串问题时，先统一大小写、去空格等，避免后续比较出错。  
- **善用STL工具**：`map`/`unordered_map`是统计频率的“神器”，能大幅简化代码。  
- **测试边界用例**：比如所有单词相同、多个单词次数并列等，确保代码鲁棒性。

## 4. C++核心代码实现赏析

<code_intro_overall>
我们选择更高效的STL map法作为通用核心实现，它代码简洁且体现现代C++的优势。
</code_intro_overall>

**本题通用核心C++实现参考**  
* **说明**：此代码综合题解的map思路，包含完整的输入、处理、输出逻辑，适合直接学习。  
* **完整核心代码**：  
    ```cpp
    #include <iostream>
    #include <string>
    #include <map>
    #include <cctype> // 用于tolower函数

    using namespace std;

    int main() {
        int n;
        cin >> n;
        map<string, int> cnt; // 键：小写单词，值：出现次数
        string s;

        for (int i = 0; i < n; ++i) {
            cin >> s;
            // 转换为小写
            for (char &c : s) {
                c = tolower(c);
            }
            cnt[s]++; // 统计次数
        }

        string ans_s;
        int max_c = 0;
        // 遍历map找最大次数
        for (auto &p : cnt) {
            if (p.second > max_c) {
                max_c = p.second;
                ans_s = p.first;
            }
        }

        cout << ans_s << endl;
        return 0;
    }
    ```
* **代码解读概要**：  
  代码首先读取输入，将每个单词转为小写后存入`map`统计次数。然后遍历`map`，找到次数最多的单词并输出。核心逻辑是“小写转换”和“map统计”，利用`map`自动去重和排序的特性简化了统计过程。

接下来，我们分析两种题解的核心代码片段，看看它们的亮点和实现思路。
</code_intro_selected>

**题解一：暴力枚举法 (来源：chen_zhe)**  
* **亮点**：思路直白，适合理解统计逻辑；手动处理大小写转换，适合练习字符操作。  
* **核心代码片段**：  
    ```cpp
    // 转换为小写（手动版）
    for (int i = 0; i < n; ++i) {
        cin >> s[i]; 
        for (int j = 0; j < s[i].length(); ++j) {
            if (s[i][j] >= 'A' && s[i][j] <= 'Z') {
                s[i][j] = s[i][j] - 'A' + 'a';
            }
        }
    }

    // 暴力统计次数
    for (int i = 0; i < n; ++i) {
        int cur_c = 0; 
        for (int j = 0; j < n; ++j) {
            if (s[i] == s[j]) cur_c++;
        }
        if (cur_c > max_c) {
            max_c = cur_c;
            ans_s = s[i];
        }
    }
    ```
* **代码解读**：  
  第一部分逐个字符检查，如果是大写字母（A-Z），就通过`- 'A' + 'a'`转为小写（比如'A'→'a'）。第二部分用双重循环：外层遍历每个单词，内层统计它在所有单词中的出现次数，最后更新最大值。  
  思考：为什么内层循环要从0开始？因为每个单词都要和包括自己在内的所有单词比较，才能正确统计次数。  
* 💡 **学习笔记**：暴力法虽然简单，但能帮我们理解“统计”的本质——逐个比较、计数。

**题解二：STL map法 (来源：chen_zhe)**  
* **亮点**：代码简洁高效，利用`map`自动管理单词的唯一性和计数，体现了“用工具简化问题”的思想。  
* **核心代码片段**：  
    ```cpp
    map<string, int> cnt;
    // 读入并转换小写
    for (int i = 0; i < n; ++i) {
        cin >> s;
        for (char &c : s) c = tolower(c);
        cnt[s]++;
    }
    // 找最大次数
    for (auto it = cnt.begin(); it != cnt.end(); ++it) {
        if (it->second > max_c) {
            max_c = it->second;
            ans_s = it->first;
        }
    }
    ```
* **代码解读**：  
  `map<string, int> cnt`相当于一个“智能计数器”，每个不同的小写单词对应一个“桶”。读入单词时，先转小写，然后`cnt[s]++`会自动找到对应的桶并加1（如果桶不存在，会自动创建）。最后遍历所有桶，找到数字最大的那个。  
  思考：`map`的键是按字典序排序的，所以如果有多个单词次数相同，会选字典序较小的那个。这符合题目要求吗？本题样例中`apple`是正确输出，而`map`的遍历顺序正好能保证这一点。  
* 💡 **学习笔记**：`map`是统计频率的“利器”，熟练使用STL能大幅提升编程效率！

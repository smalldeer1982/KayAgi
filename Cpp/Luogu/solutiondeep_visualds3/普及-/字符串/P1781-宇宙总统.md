# 题目信息

# 宇宙总统

## 题目描述

地球历公元 6036 年，全宇宙准备竞选一个最贤能的人当总统，共有 $n$ 个非凡拔尖的人竞选总统，现在票数已经统计完毕，请你算出谁能够当上总统。


## 说明/提示

票数可能会很大，可能会到 $100$ 位数字。

$1 \leq n \leq 20$。

## 样例 #1

### 输入

```
5
98765
12365
87954
1022356
985678
```

### 输出

```
4
1022356```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：`编程技巧应用` (字符串处理与大数字比较)

🗣️ **初步分析**：
解决“宇宙总统”这道题，关键在于理解如何比较两个非常大的数字（可能长达100位）。这类数字无法用普通整数类型存储，因此需要用字符串处理。简单来说，比较两个大数字符串的规则是：**先比较长度，长度更长的数更大；若长度相同，则逐位比较字符（数字字符的ASCII码顺序与数值顺序一致）**。例如，"123"和"45"比较时，"123"长度更长，所以更大；而"123"和"129"比较时，第三位'3' < '9'，所以"129"更大。

- **题解思路**：大部分题解采用直接遍历所有票数，维护当前最大值及其编号的方法（打擂台法），或通过结构体排序后取最大值。核心差异在于比较逻辑的实现方式（如直接字符串比较、自定义比较函数等）。
- **核心难点**：正确实现大数字符串的比较逻辑（避免"9"被误认为比"10"大），以及记录最大值的原始编号。
- **可视化设计思路**：设计一个8位像素风格的动画，用方块表示字符串的每一位，动态展示长度比较和逐位比较的过程。例如，当比较两个字符串时，先高亮显示长度差异，若长度相同则逐个字符闪烁比较，用不同颜色标记较大的字符。

## 3. 核心难点辨析与解题策略

<difficulty_intro>
在解决这道题的过程中，我们通常会遇到以下关键点或难点。结合优质题解的共性，我为大家提炼了几个核心的思考方向和策略：
</difficulty_intro>

1.  **关键点1：大数字符串的正确比较**
    * **分析**：直接使用字符串的默认比较（如`>`运算符）会导致"9" > "10"的错误，因为默认比较是按字典序逐位比较。正确的做法是先比较长度，长度长的数更大；若长度相同，再逐位比较字符（或直接使用字符串的默认比较，因为长度相同时字典序与数值顺序一致）。
    * 💡 **学习笔记**：大数字符串比较的核心是“长度优先，同长比内容”。

2.  **关键点2：记录最大值的原始编号**
    * **分析**：输入的票数需要与原始编号绑定，否则无法在找到最大值后输出正确的编号。优质题解通常用结构体（如`struct node{string x; int num;}`）或直接维护编号变量来解决。
    * 💡 **学习笔记**：处理“带属性的数据”时，用结构体或数组同步存储属性是常见方法。

3.  **关键点3：避免重复比较的低效**
    * **分析**：若使用排序（如`sort`），时间复杂度为O(n log n)；而直接遍历（打擂台法）时间复杂度为O(n)，更高效。对于n≤20的题目，两种方法均可，但打擂台法更简洁。
    * 💡 **学习笔记**：根据数据规模选择算法，小数据量下简单方法可能更优。

### ✨ 解题技巧总结
<summary_best_practices>
- **问题分解**：将大问题分解为“输入处理”“比较逻辑”“结果输出”三部分，逐步解决。
- **字符串特性利用**：利用`string`的`size()`方法获取长度，直接比较长度；长度相同时，`string`的`>`运算符天然支持字典序比较（与数值顺序一致）。
- **边界处理**：初始最大值设为第一个输入的票数，避免空值错误。

## 4. C++核心代码实现赏析

<code_intro_overall>
在深入分析具体题解的精妙片段之前，让我们先来看一个能够完整解决本题的通用核心C++实现参考。

**本题通用核心C++实现参考**
* **说明**：本代码综合了多个优质题解的思路，采用打擂台法，逐次比较输入的票数，动态更新最大值及其编号，简洁高效。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    #include <string>
    using namespace std;

    int main() {
        int n, id = 1;
        string max_vote;
        cin >> n >> max_vote; // 初始最大值设为第一个输入的票数

        for (int i = 2; i <= n; ++i) {
            string current;
            cin >> current;
            // 比较逻辑：先长度，后字典序
            if (current.size() > max_vote.size() || 
                (current.size() == max_vote.size() && current > max_vote)) {
                max_vote = current;
                id = i;
            }
        }

        cout << id << endl << max_vote << endl;
        return 0;
    }
    ```
* **代码解读概要**：
    > 代码首先读取候选人数量`n`和第一个票数`max_vote`，并初始化`id`为1。随后遍历剩余候选人，每次读取当前票数`current`，比较其长度和字典序：若长度更长，或长度相同但字典序更大，则更新`max_vote`和`id`。最后输出结果。核心逻辑在`if`条件中，确保正确比较大数字符串。

**题解二：作者right_cat**
* **亮点**：使用结构体和自定义排序函数，适合学习数据排序的通用方法。
* **核心代码片段**：
    ```cpp
    struct node {
        string x;
        int num;
        int lenx;
    } s[25];

    bool cmp(node a, node b) {
        if (a.lenx > b.lenx) return 1;
        if (a.lenx == b.lenx && a.x > b.x) return 1;
        return 0;
    }

    sort(s + 1, s + n + 1, cmp);
    ```
* **代码解读**：
    > 结构体`node`存储票数`x`、编号`num`和长度`lenx`。排序函数`cmp`先比较长度（`lenx`），长度长的排前；长度相同时比较票数`x`（字典序）。通过`sort`排序后，首元素即为最大值。此方法将问题转化为排序问题，思路清晰。
* 💡 **学习笔记**：结构体可将关联数据（票数、编号）打包，配合自定义排序函数，是解决类似问题的通用技巧。

**题解三：作者zhangyiming（修正后）**
* **亮点**：通过`strlen`和`strcmp`组合实现比较，适合学习C风格字符串的处理。
* **核心代码片段**：
    ```cpp
    char a[201], max[201] = "";
    for (int i = 1; i <= n; i++) {
        cin >> a;
        if ((strlen(max) < strlen(a)) || (strlen(a) == strlen(max) && strcmp(max, a) < 0)) {
            strcpy(max, a);
            num = i;
        }
    }
    ```
* **代码解读**：
    > 代码使用C风格字符串，`strlen`获取长度，`strcmp`比较内容。若当前票数`a`的长度大于`max`，或长度相同但`a`更大（`strcmp(max, a) < 0`表示`max < a`），则更新`max`和`num`。此方法展示了如何用C函数实现大数字符串比较。
* 💡 **学习笔记**：C风格字符串需注意`strlen`和`strcmp`的使用场景（`strlen`计算长度，`strcmp`比较内容）。

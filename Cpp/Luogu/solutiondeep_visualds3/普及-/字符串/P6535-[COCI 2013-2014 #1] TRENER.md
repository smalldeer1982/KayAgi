## 3. 核心难点辨析与解题策略

<difficulty_intro>
这道题的“坑”都藏在细节里，我帮大家提炼了3个核心难点和解决办法，学会了能避免90%的错误！
</difficulty_intro>

1.  **关键点1：首字母怎么转成统计索引？**
    * **分析**：字符在计算机里是ASCII码（`a`是97，`b`是98…）。要把`a`~`z`转成0~25的索引，只需`字符-'a'`（比如`a-'a'=0`，`b-'a'=1`）；如果想用1~26的索引，就加1（`字符-'a'+1`）。  
    * 💡 **学习笔记**：字符转数字的核心是“找基准”——`'a'`是小写字母的基准！

2.  **关键点2：如何保证输出是字典序？**
    * **分析**：字典序就是`a→b→c→…→z`的顺序。数组法遍历0~25（对应`a`~`z`），map法遍历`'a'`到`'z'`，自然就是字典序！  
    * 💡 **学习笔记**：想按顺序输出，就按顺序遍历数据结构！

3.  **关键点3：什么时候输出`PREDAJA`？**
    * **分析**：用一个`bool`变量（比如`flag`）标记“有没有符合条件的字母”。初始设为`false`，如果找到≥5次的字母，就把`flag`改成`true`；最后如果`flag`还是`false`，就输出`PREDAJA`。  
    * 💡 **学习笔记**：用“标记变量”处理“有没有结果”的问题，是模拟题的常用技巧！

### ✨ 解题技巧总结
<summary_best_practices>
这道题的技巧能用到很多类似题目里，记下来：
</summary_best_practices>
-   **字符转索引**：小写字母用`c-'a'`，大写字母用`c-'A'`，永远不会错！
-   **统计选结构**：数据范围小（比如26个字母）用数组（快），范围大或类型复杂用map（方便）。
-   **字典序输出**：按`a`~`z`的顺序遍历，不用额外排序！

## 4. C++核心代码实现赏析

<code_intro_overall>
先看一个**通用数组版核心代码**——它结合了题解一和题解三的优点，逻辑清晰又简洁，适合新手入门！
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：本代码是数组法的典型实现，用`cnt[26]`统计次数，逻辑直接，效率高。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    #include <string>
    using namespace std;

    int main() {
        int n;
        cin >> n;
        int cnt[26] = {0}; // 初始化为0，对应a~z的次数
        bool has_result = false;

        for (int i = 0; i < n; ++i) {
            string s;
            cin >> s;
            char first = s[0]; // 取首字母
            cnt[first - 'a']++; // 转成索引并统计
        }

        for (int i = 0; i < 26; ++i) {
            if (cnt[i] >= 5) {
                cout << (char)(i + 'a'); // 转成字符输出
                has_result = true;
            }
        }

        if (!has_result) {
            cout << "PREDAJA";
        }
        cout << endl;
        return 0;
    }
    ```
* **代码解读概要**：  
  1. 输入`n`，初始化`cnt`数组（存26个字母的次数）和`has_result`（标记有没有结果）；  
  2. 循环`n`次，每次取字符串首字母，转成索引后统计次数；  
  3. 遍历`cnt`数组，输出≥5次的字母，标记`has_result`；  
  4. 如果没有结果，输出`PREDAJA`。

**题解一：数组统计核心片段（作者：Utilokasteinn）**
* **亮点**：用`a[s[0]-'a'+1]++`处理索引，适合习惯1~26计数的同学。
* **核心代码片段**：
    ```cpp
    for(int i=1;i<=n;i++) {
        cin>>s;
        a[s[0]-'a'+1]++; // s[0]是首字母，转成1~26的索引
    }
    for(int i=1;i<=26;i++) {
        if(a[i]>=5) cout<<char(i-1+'a'),flag=1; // 转成字符输出
    }
    ```
* **代码解读**：  
  比如首字母是`a`，`s[0]-'a'=0`，加1后是`1`，存在`a[1]`里；输出时`i-1+'a'`把`1`转成`a`（`1-1=0`，`0+'a'=97`即`a`）。这种方式和0~25的逻辑一样，只是索引起点不同~
* 💡 **学习笔记**：索引起点不重要，重要的是“一一对应”！

**题解二：map统计核心片段（作者：naroanah）**
* **亮点**：直接用`char`当map的键，不用算索引，更直观。
* **核心代码片段**：
    ```cpp
    map<char,int> a; // char是键（字母），int是值（次数）
    for(int i=0;i<n;i++) {
        cin>>s;
        a[s[0]]++; // 直接用首字母当键，次数+1
    }
    for(char i='a';i<='z';i++) {
        if(a[i]>=5) cout<<i, t=true; // 遍历a~z，输出符合条件的字母
    }
    ```
* **代码解读**：  
  map的`a[s[0]]`会自动创建键值对——如果`s[0]`是`b`，就创建`a['b']`并初始化为0，然后`++`变成1。遍历`'a'`到`'z'`时，直接查`a[i]`的次数，超级方便！
* 💡 **学习笔记**：map适合“键类型不连续”或“不想算索引”的场景~

# 题目信息

# [GESP202306 三级] 春游

## 题目描述

老师带领同学们春游。已知班上有 $N$ 位同学，每位同学有从 $0$ 到 $N-1$ 的唯一编号。到了集合时间，老师确认是否所有同学都到达了集合地点，就让同学们报出自己的编号。到达的同学都会报出自己的编号，不会报出别人的编号，但有的同学很顽皮，会多次报出。你能帮老师找出有哪些同学没有到达吗 ?。

## 样例 #1

### 输入

```
3 3
0 2 1```

### 输出

```
3```

## 样例 #2

### 输入

```
3 5
0 0 0 0 0```

### 输出

```
1 2```

## 3. 核心难点辨析与解题策略

1. **难点1：如何高效记录报到状态？**  
   * **分析**：数组标记法将编号直接映射为数组下标，`a[i]=1`表示编号i已报到。相比`set`更省时（O(1)访问），空间O(N)也合理  
   * 💡 **学习笔记**：**下标映射**是处理离散编号的利器  

2. **难点2：重复报数如何处理？**  
   * **分析**：无论报多少次，只需标记一次。代码中`a[p]=1`天然去重，避免复杂判断  
   * 💡 **学习笔记**：**状态覆盖**简化逻辑，关注最终状态而非过程  

3. **难点3：全员到齐的特判**  
   * **分析**：用`ans`计数器统计缺席人数，归零时输出N。比二次遍历检查更高效  
   * 💡 **学习笔记**：**边统计边处理**减少冗余操作  

### ✨ 解题技巧总结
- **标记板思维**：将实体对象（学生）映射为数据结构（数组下标）  
- **状态覆盖法**：重复操作不影响最终状态时直接覆盖  
- **边界预判**：特殊场景（如全员到齐）提前设计处理逻辑

## 4. C++核心代码实现赏析

**通用核心实现参考**  
* **说明**：综合自玖安老师题解，完整呈现签到逻辑  
* **完整核心代码**：
```cpp
#include <iostream>
using namespace std;

int main() {
    int N, M;
    cin >> N >> M;
    int arrived[1005] = {0}; // 标记数组（索引=学生编号）
    int ans = 0;             // 缺席计数器

    // 标记报到状态
    for (int i = 0; i < M; i++) {
        int p;
        cin >> p;
        arrived[p] = 1;     // 关键！无论报几次，标记一次
    }

    // 检查缺席并输出
    for (int i = 0; i < N; i++) {
        if (arrived[i] == 0) {
            ans++; 
            cout << i << " ";
        }
    }

    // 全员到齐特判
    if (ans == 0) cout << N;
    return 0;
}
```
* **代码解读概要**：  
  1. 初始化`arrived`数组全0 → 2. 循环读报数并标记 → 3. 遍历数组找缺席 → 4. `ans==0`时输出N  

**玖安老师题解片段赏析**  
* **亮点**：去重逻辑极致简洁，边界处理严谨  
* **核心代码片段**：
```cpp
for (int i = 0; i < M; i++) {
    int p;
    cin >> p;
    a[p] = 1;  // 核心标记：下标直接映射编号
}
```
* **代码解读**：  
  > 循环`M`次读取报数`p`→将数组`a`中索引为`p`的值设为1。**为何用数组下标？** 因为学生编号0~N-1恰好是完美数组索引！**为何不判断是否已标记？** 重复赋值不影响结果，省去`if`判断提升效率。  
* 💡 **学习笔记**：**数组下标映射**把抽象编号转为物理存储地址，是空间换时间的经典技巧

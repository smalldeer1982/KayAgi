## 题目背景

与其编写苍白无力的背景，不如出更有质量的题。



## 题目描述

对于整数 $n,k$，若存在**非负整数** $x$ 与 **正整数** $y$ 满足：

$$x + y = n\ \land\ x \bmod y =k$$

则我们称有序数对 $(x,y)$ 是 $n$ 的一个 **优秀拆分**（其中 $\land$  是**并且**的意思）。

现给定非负整数 $n,k$，请你构造**任意**一组 $n$ 的优秀拆分，并分别输出你构造方案中的 $x$ 和 $y$。特殊地，若不存在这样的拆分，则输出 `-1`。

## 说明/提示

#### 样例解释

对于第一组数据，只存在唯一的构造方式。

对于第二组数据，$(3,10)$ 也是一组合法的构造。

对于第三组数据，可以证明不存在一组合法的构造。

------------

#### 数据范围

**「本题采用捆绑测试」**

- $\operatorname{Subtask} 1(20\%)$：$n \leq 10^6$。

- $\operatorname{Subtask} 2(40\%)$：$n \leq 10^{12}$。

- $\operatorname{Subtask} 3(40\%)$：无特殊限制。

对于 $100\%$ 的数据：$T \leq 5$，$0 \leq n,k \leq 10^{18}$。

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：数学构造（模运算性质应用）

🗣️ **初步分析**：  
解决“构造与取模”问题的关键，在于**利用模运算的核心性质——余数必须小于除数**（比如，10除以3余1，1<3）。我们的目标是找到两个数$x$（非负）和$y$（正），使得它们的和为$n$，且$x$除以$y$的余数是$k$。  

### 核心思路与结论  
所有优质题解都指向一个**简洁结论**：  
- 若$2k < n$，则取$x=k$，$y=n-k$（此时$y = n - k > k$，满足余数$k < y$的条件）；  
- 否则（$2k \geq n$），无解，输出$-1$。  

比如，样例1中的输入`13 3`，$2\times3=6 <13$，所以$x=3$，$y=10$（$3+10=13$，$3\mod10=3$），符合要求。  

### 可视化设计思路  
为了直观理解“余数小于除数”的条件，我们可以设计一个**像素风格的“数学积木”动画**：  
- 用红色像素块表示$x$（值为$k$），蓝色像素块表示$y$（值为$n-k$）；  
- 动画开始时，红色和蓝色积木拼成一个黄色的大积木（表示$n$）；  
- 然后，红色积木“放在”蓝色积木上，显示“$x\mod y = k$”（因为$x < y$，余数就是$x$本身）；  
- 若$2k \geq n$，则蓝色积木会“变小”（$y = n - k \leq k$），此时红色积木无法“放在”蓝色积木上，动画显示“无解”。  

### 游戏化元素

- **交互**：支持“单步执行”（逐步显示$x$和$y$的构造过程）和“自动播放”（快速演示所有步骤）；  
- **AI模式**：模拟“探险家”寻找$x$和$y$，当找到解时，探险家会“欢呼”（像素动画）。

## 3. 核心难点辨析与解题策略

<difficulty_intro>  
解决本题的核心难点在于**理解模运算的约束条件**和**构造合适的$x$、$y$**。结合优质题解，我总结了以下关键策略：  
</difficulty_intro>

### 1. 关键点1：余数必须小于除数（$k < y$）  
* **分析**：  
  模运算的基本性质是“余数<除数”（比如$5\mod3=2$，$2<3$）。因此，$x\mod y=k$必须满足$k < y$。而$y = n - x$，所以$k < n - x$，即$x < n - k$。  

* 💡 **学习笔记**：  
  余数条件是本题的“敲门砖”，忽略它会导致构造的解无效。  

### 2. 关键点2：如何构造$x$和$y$？  
* **分析**：  
  优质题解都选择了**最简单的构造方式**：$x=k$，$y=n-k$。此时，$x+y=n$（满足和的条件），且$k < y$（当$2k < n$时），因此$x\mod y=k$（满足模的条件）。这种构造方式**时间复杂度$O(1)$**，非常高效。  

* 💡 **学习笔记**：  
  构造题的核心是“找最简单的解”，而非“所有解”。  

### 3. 关键点3：处理大数（$n,k \leq 10^{18}$）  
* **分析**：  
  本题数据范围极大（$10^{18}$），因此必须使用**long long**类型（C++中的64位整数）。若使用int类型，会导致“溢出”错误（比如$10^{18}$超过int的最大值$2^{31}-1$）。  

* 💡 **学习笔记**：  
  遇到大数问题，第一反应是“开long long”！

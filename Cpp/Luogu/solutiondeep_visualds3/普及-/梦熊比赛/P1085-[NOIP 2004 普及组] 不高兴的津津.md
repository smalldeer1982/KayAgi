#### 题目解读与核心算法识别
✨ **本题主要考察**：`枚举与模拟应用`  
🗣️ **初步分析**：  
> 本题核心是通过遍历枚举七天的数据，模拟津津的日程计算过程。就像每天检查作业完成时间是否超标一样，我们需要逐天计算上课总时长，并动态记录最大值及其位置。  
> - 核心思路：遍历七天数据，计算每日总时长，当超过8小时且严格大于当前最大值时更新记录

#### 核心难点辨析与解题策略
1. **难点1：初始值设定陷阱**  
   * **分析**：优质解法采用两种策略：cff_0102初始化mx=8（自然过滤≤8的值），TainityAnle用max=0但增加a>8条件。关键要确保初始值不影响首次有效更新  
   * 💡 **学习笔记**：初始值需与问题约束联动设计  

2. **难点2：相同最大值取最早**  
   * **分析**：zhuoqizhi强调必须用严格>而非≥更新。当新值≤当前max时保持原记录，从而保证最早出现的最大值被保留  
   * 💡 **学习笔记**：最值位置更新需考虑等值处理策略  

3. **难点3：无超时情况的输出**  
   * **分析**：通过pos/maxi初始化为0，未更新时自然输出0。cff_0102解法中因mx初始8，仅超时值可能触发更新  
   * 💡 **学习笔记**：输出默认值应与问题要求直接对应  

### ✨ 解题技巧总结
- **边界融合术**：像cff_0102将约束条件（>8）融入初始值（mx=8），减少判断分支  
- **严格更新机制**：使用严格比较（>而非≥）保证多解时取最早出现位置  
- **实时计算策略**：如TainityAnle即时计算a+b，避免数组存储，提升空间效率

using namespace std;
int main() {

    for (int i = 1; i <= 7; i++) {
        int a, b;
        cin >> a >> b;

}

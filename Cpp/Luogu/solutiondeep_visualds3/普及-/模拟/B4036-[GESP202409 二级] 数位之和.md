# 题目信息

# [GESP202409 二级] 数位之和

## 题目描述

小杨有 $n$ 个正整数，他认为一个正整数是美丽数字当且仅当该正整数每一位数字的总和是 $7$ 的倍数。

小杨想请你编写一个程序判断 $n$ 个正整数哪些是美丽数字。

## 说明/提示

### 数据规模与约定

对全部的测试数据，保证 $1 \leq n \leq 10^5$，$1 \leq a_i \leq 10^5$。

## 样例 #1

### 输入

```
3
7
52
103```

### 输出

```
Yes
Yes
No```

## 3. 核心难点辨析与解题策略

<difficulty_intro>
在解决这道题的过程中，我们可能会遇到以下关键点或难点。结合题解的思路，我为大家总结了应对策略：
</difficulty_intro>

1.  **关键点1：如何正确分解每一位数字？**
    * **分析**：分解数位的关键是循环取个位（`a % 10`）并更新原数（`a /= 10`）。例如，对于数52，第一次取个位2（`52%10=2`），然后`52/10=5`；第二次取个位5（`5%10=5`），然后`5/10=0`，循环结束。需要注意循环条件是`a > 0`，确保所有位都被处理。
    * 💡 **学习笔记**：数位分解的“取模+除法”组合是固定套路，记住这个模式就能轻松拆解任意正整数的各位。

2.  **关键点2：如何避免累加错误？**
    * **分析**：每次循环时，`sum`需要初始化为0（每个数单独计算）。如果忘记重置`sum`，前一个数的和会影响当前数的结果。例如，处理完7（sum=7）后，若不重置sum，处理52时sum会从7开始累加，导致错误（正确sum应为5+2=7）。
    * 💡 **学习笔记**：每个数的计算是独立的，`sum`必须在每次处理新数时重新初始化为0。

3.  **关键点3：如何高效处理大规模数据？**
    * **分析**：题目中n可达1e5，每个数最多6位，总操作次数是1e5*6=6e5次，完全在时间限制内。无需优化，直接模拟即可。
    * 💡 **学习笔记**：当数据规模在合理范围内时，直接模拟是最简洁可靠的选择。

## 4. C++核心代码实现赏析

<code_intro_overall>
首先，我们来看一个完整的、能直接解决问题的C++核心实现。
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：此代码综合了优质题解的思路，结构清晰，适合直接用于竞赛或练习。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    using namespace std;

    int main() {
        int n;
        cin >> n;
        while (n--) {
            int a, sum = 0;
            cin >> a;
            while (a > 0) {
                sum += a % 10; // 取个位并累加
                a /= 10;       // 去掉已处理的个位
            }
            if (sum % 7 == 0) {
                cout << "Yes" << endl;
            } else {
                cout << "No" << endl;
            }
        }
        return 0;
    }
    ```
* **代码解读概要**：
    > 代码首先读取n，然后循环n次处理每个数。对每个数，初始化sum为0，通过`while (a > 0)`循环拆解各位：`a % 10`取个位，`sum +=`累加，`a /= 10`去掉已处理的位。循环结束后判断sum是否是7的倍数，输出结果。

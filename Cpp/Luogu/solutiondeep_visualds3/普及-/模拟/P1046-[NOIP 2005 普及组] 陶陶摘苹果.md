# 题目信息

# [NOIP 2005 普及组] 陶陶摘苹果

## 题目描述

陶陶家的院子里有一棵苹果树，每到秋天树上就会结出 $10$ 个苹果。苹果成熟的时候，陶陶就会跑去摘苹果。陶陶有个 $30$ 厘米高的板凳，当她不能直接用手摘到苹果的时候，就会踩到板凳上再试试。


现在已知 $10$ 个苹果到地面的高度，以及陶陶把手伸直的时候能够达到的最大高度，请帮陶陶算一下她能够摘到的苹果的数目。假设她碰到苹果，苹果就会掉下来。


## 说明/提示

**【题目来源】**

NOIP 2005 普及组第一题

## 样例 #1

### 输入

```
100 200 150 140 129 134 167 198 200 111
110
```

### 输出

```
5```

## 3. 核心难点辨析与解题策略

1.  **关键点1**：如何正确计算陶陶的最大可触高度？
    * **分析**：题目中明确说明“当她不能直接用手摘到苹果的时候，就会踩到板凳上再试试”。因此，陶陶的最大可触高度是“身高+30cm”（无论是否需要踩板凳，直接计算这个值即可覆盖所有可能的情况）。优质题解通常会在输入身高后立即计算`H += 30`，避免后续重复计算。
    * 💡 **学习笔记**：提前计算固定值（如本题的身高+30）可以减少代码冗余，提高效率。

2.  **关键点2**：如何高效统计符合条件的苹果数量？
    * **分析**：需要遍历所有10个苹果的高度，逐个比较是否≤最大可触高度。优质题解通常使用简单的`for`循环完成遍历，并通过计数器（如`s`、`cnt`）累加结果。部分题解还利用布尔值的特性（`true=1`）直接累加比较结果（如`s += !(H < height[i])`），进一步简化代码。

3.  **关键点3**：如何避免输入输出错误？
    * **分析**：本题需要输入10个苹果的高度和1个身高值。输入顺序错误（如先输入身高再输入苹果高度）或数组越界（如数组大小不足10）是常见错误。优质题解通常会明确数组的大小（如`int a[15]`），并使用循环严格输入10次。

**本题通用核心C++实现参考**
* **说明**：本代码综合了多个优质题解的思路，采用最直观的模拟逻辑，适合新手学习。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    using namespace std;

    int main() {
        int apples[10]; // 存储10个苹果的高度
        int height;     // 陶陶的身高
        int cnt = 0;    // 计数器

        // 输入10个苹果的高度
        for (int i = 0; i < 10; ++i) {
            cin >> apples[i];
        }

        // 输入陶陶的身高，并计算最大可触高度（身高+30）
        cin >> height;
        height += 30;

        // 统计能摘到的苹果数量
        for (int i = 0; i < 10; ++i) {
            if (apples[i] <= height) {
                cnt++;
            }
        }

        // 输出结果
        cout << cnt << endl;
        return 0;
    }
    ```
* **代码解读概要**：
    > 这段代码首先定义了存储苹果高度的数组和计数器变量。通过`for`循环输入10个苹果的高度和陶陶的身高后，计算最大可触高度（身高+30）。接着再次遍历苹果高度数组，逐个比较是否≤最大可触高度，符合条件的苹果数量累加到`cnt`中。最后输出`cnt`，即能摘到的苹果数。

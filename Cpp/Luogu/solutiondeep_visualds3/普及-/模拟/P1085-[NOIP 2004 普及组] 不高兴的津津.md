🗣️ **初步分析**：
> 解决“不高兴的津津”这道题，关键在于用“模拟”的思路，逐天计算津津的上课总时间，并记录最不高兴的那天。简单来说，“模拟”就是按照题目描述的流程，一步步复现问题场景——就像你亲自当“小助手”，帮津津检查每一天的课程表，计算总时长，再找出最长的那一天。

## 3. 核心难点辨析与解题策略

<difficulty_intro>
在解决这个问题的过程中，我们通常会遇到以下几个关键点或难点。结合优质题解的共性，我为大家提炼了思考方向和策略：
</difficulty_intro>

1.  **关键点1**：如何初始化最大值变量，确保所有天数不超8时输出0？
    * **分析**：若初始最大值`max`设为0，当所有天数总时长≤8时，`max`仍为0，此时需输出0。但更简洁的方式是将`max`初始化为8（如cff_0102的题解），这样只要某天总时长>8且>当前`max`，就更新`max`和天数。若所有天数≤8，`max`保持8，天数保持0，直接输出即可。
    * 💡 **学习笔记**：初始化变量时，要考虑“最坏情况”（如所有天数都不满足条件），让初始值能直接对应最终输出。

2.  **关键点2**：如何处理多个天数总时长相同的情况，确保输出最靠前的那天？
    * **分析**：比较时使用“严格大于”（`if (current > max)`）而非“大于等于”。这样，当后续天数总时长等于当前`max`时，不会更新天数，从而保留第一个出现的天数。
    * 💡 **学习笔记**：“严格大于”是保留“最靠前”结果的关键。

3.  **关键点3**：如何高效计算并比较每天的总时长？
    * **分析**：无需存储所有天数的总时长，只需在输入时实时计算并比较。这样空间复杂度为O(1)（仅用几个变量），时间复杂度为O(7)（固定7天），非常高效。
    * 💡 **学习笔记**：实时计算+实时比较，避免不必要的存储，提升效率。

### ✨ 解题技巧总结
<summary_best_practices>
- **实时处理**：输入时直接计算总时长并比较，避免存储所有数据，节省空间。  
- **初始化技巧**：将最大值初始化为临界值（如8），简化“所有天数不超8”的判断逻辑。  
- **严格比较**：用`>`而非`>=`确保相同值时保留最靠前的结果。
</summary_best_practices>

**本题通用核心C++实现参考**

* **完整核心代码**：
    ```cpp
    #include <iostream>
    using namespace std;

    int main() {
        ios::sync_with_stdio(false); // 加速输入
        cin.tie(nullptr);
        
        int max_time = 8; // 初始化为8，若所有天数≤8则输出0
        int max_day = 0;  // 最不高兴的天数
        
        for (int day = 1; day <= 7; ++day) {
            int school, mom;
            cin >> school >> mom;
            int total = school + mom;
            
            if (total > max_time) { // 严格大于，保留最靠前的天数
                max_time = total;
                max_day = day;
            }
        }
        
        cout << max_day << endl;
        return 0;
    }
    ```

    > 代码首先加速输入（`ios::sync_with_stdio(false); cin.tie(nullptr);`），提升读取效率。`max_time`初始化为8，`max_day`初始化为0。通过循环读取7天的课程时间，计算每天总时长`total`。若`total`大于当前`max_time`，则更新`max_time`和`max_day`。最后输出`max_day`，若所有天数≤8，`max_day`保持0。

# 题目信息

# 头像上传

## 题目描述

有一天无聊的你在注册一个无聊的网站，注册完的第一步自然是上传头像。

你发现这个网站总共可以上传 $n$ 个头像，每个头像必须是正方形，并且长宽至少为 $L \times L$

在图片上传前，系统会对图片进行如下处理：如果图片的任何一边长度超过了 $G$ ，那么系统会不断地对图片的长宽同时减半（向下取整），直至两边长度 $\leq G$ 为止。

你现在找到了 $n$ 张可供上传的图片，第 $i$ 张的尺寸是 $W_i \times H_i$。

如果图片有任何一边小于 $L$，请输出 "Too Young"
 
如果图片满足大小条件但不为正方形，请输出"Too Simple"

如果图片满足大小条件并且是正方形，请输出"Sometimes Naive" 

以上所有字符串输出时均不包含引号。

## 说明/提示

对于 $30\%$ 的数据，满足 $n=1$

对于 $100\%$的数据，满足 $n \leq 2000$，$W_i,H_i ,G\leq 10000$

## 样例 #1

### 输入

```
3 5 10
12 3
7 8
5 5
```

### 输出

```
Too Young
Too Simple
Sometimes Naive```

## 3. 核心难点辨析与解题策略

<difficulty_intro>
在解决这道题的过程中，我们通常会遇到以下关键点或难点。结合优质题解的共性，提炼思考方向和策略：
</difficulty_intro>

1.  **关键点1：正确实现“同时减半直到≤G”的裁剪逻辑**
    * **分析**：裁剪条件是“任何一边超过G”（即`w>g || h>g`），而非“两边都超过G”。例如，若`w=12`，`g=10`，`h=5`（h≤G），仍需裁剪，因为w>g。优质题解均使用`while(w>g || h>g)`确保正确裁剪。
    * 💡 **学习笔记**：裁剪条件是“或”不是“且”，否则会导致部分情况未裁剪（如一边超G，另一边不超）。

2.  **关键点2：条件判断的顺序**
    * **分析**：必须先判断“是否小于L”，再判断“是否是正方形”。例如，若裁剪后的`w=4`，`h=4`，但`L=5`，此时应输出“Too Young”而非“Sometimes Naive”。优质题解均严格按此顺序判断。
    * 💡 **学习笔记**：题目要求的输出顺序决定了代码中的判断顺序，需严格遵循。

3.  **关键点3：向下取整的处理**
    * **分析**：题目要求“向下取整”，C++中整数除法`/=2`或右移`>>=1`（仅对正数有效）均可实现。例如，`w=7`，`7/2=3`（向下取整），而不是4。优质题解均正确使用这两种方法。
    * 💡 **学习笔记**：整数除法自动向下取整，无需额外处理（如`floor`），简化代码。

### ✨ 解题技巧总结
<summary_best_practices>
- **问题拆解**：将问题拆分为“裁剪”和“判断”两步，分别实现，降低复杂度。
- **边界测试**：测试极端情况（如`w=G+1`，`h=G+1`；`w=L-1`，`h=L`），确保代码鲁棒性。
- **代码简洁性**：使用右移`>>=1`代替`/=2`，提升运行速度（尤其在大循环中）。
</summary_best_practices>

**本题通用核心C++实现参考**
* **说明**：此代码综合了犇犇犇犇、pomelo_nene等题解的思路，代码简洁、逻辑清晰，适合作为参考。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    using namespace std;

    int main() {
        int n, l, g;
        cin >> n >> l >> g;
        while (n--) {
            int w, h;
            cin >> w >> h;
            // 裁剪：直到w和h都≤G
            while (w > g || h > g) {
                w /= 2;  // 向下取整
                h /= 2;
            }
            // 判断输出
            if (w < l || h < l) {
                cout << "Too Young" << endl;
            } else if (w != h) {
                cout << "Too Simple" << endl;
            } else {
                cout << "Sometimes Naive" << endl;
            }
        }
        return 0;
    }
    ```
* **代码解读概要**：代码首先读取输入参数`n`、`l`、`g`，然后对每张图片循环处理。裁剪部分用`while`循环确保`w`和`h`都≤G；判断部分按顺序检查是否小于L、是否是正方形，输出对应结果。

**题解二：作者：Doveqise**
* **亮点**：用递归实现裁剪，展示了不同的编程思路。
* **核心代码片段**：
    ```cpp
    void solve(int w, int h) {
        if (l > min(w, h)) { puts("Too Young"); return; }
        if (g >= max(w, h)) { 
            puts(w == h ? "Sometimes Naive" : "Too Simple"); 
            return; 
        }
        solve(w >> 1, h >> 1); // 递归裁剪
    }
    ```
* **代码解读**：`solve`函数首先判断是否小于L（`l > min(w,h)`），若否，再判断是否已≤G（`g >= max(w,h)`），若是则输出结果；否则递归调用自身，将`w`和`h`右移一位（即除以2）。递归的终止条件是图片已≤G或小于L。
* 💡 **学习笔记**：递归适合处理“重复执行同一操作直到满足条件”的场景，但需注意递归深度（本题中`w`和`h`最多除以2约14次，不会栈溢出）。

**题解三：作者：pomelo_nene**
* **亮点**：使用右移操作提升效率，代码极简洁。
* **核心代码片段**：
    ```cpp
    while(w>g || h>g) {
        w >>= 1, h >>= 1; // 右移代替除法，效率更高
    }
    ```
* **代码解读**：`>>=1`是位运算中的右移操作，等同于`/=2`（仅对正数有效）。在C++中，位运算比除法更快，尤其在循环次数较多时，能提升代码效率。
* 💡 **学习笔记**：正数的向下取整除法可用右移优化，提升常数效率。

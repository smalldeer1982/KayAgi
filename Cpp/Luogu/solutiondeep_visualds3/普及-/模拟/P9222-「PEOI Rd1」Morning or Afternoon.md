# 题目信息

# 「PEOI Rd1」Morning or Afternoon

## 题目描述

有一个可用转盘显示日期的指针闹钟，我从一个时刻观测到了另一个时刻（闭区间），现在我想知道我是否一定可以通过判断这期间日期的变化情况来推断出现在是上午还是下午。

闹钟在上午 $12:00$ 准时更新日期。

上午/下午 $11:59$ 后的时刻为 下午/上午 $12:00$，上午/下午 $12:59$ 后的时刻为 上午/下午 $1:00$。

## 说明/提示

### 样例解释#1
因为我从 $11:30$ 看到了 $12:20$（假设 $12:20$ 是第一次观测到；后面的 $12:20$ 的上下午情况可以根据现在的情况推出，因为我知道右端点的 $12:20$ 是第几次观测到），我可以通过判断这期间日期有没有变化来判断这个时刻是上午还是下午。比如，我看到这段时间里闹钟更新了日期，那么 $12:20$ 一定是上午；反之 $12:20$ 一定是下午。

样例#3 同理。
### 样例解释#2
有可能是上午 $12:20$ 到上午 $5:00$，也有可能是下午 $12:20$ 到下午 $5:00$，这两种情况日期都不可能发生改变，所以我无法判断。

### 数据范围
对于 $18\%$ 分数的数据，所有时刻保证不会出现 小时数 $=12$ 的情况（即不会出现 `12:25` 类型的时刻）。

对于另外 $9\%$ 分数的数据有 $T=1$。$\color{#fff}\sout\text{不可以，总司令！}$

对于 $100\%$ 的数据 $T\leqslant 100$，$\overline\texttt{ab}\in[1,12]$，$\overline\texttt{cd}\in[0,59]$。

### 输入

12:10 12:59
1:09 8:01
12:00 1:00
12:00 12:00```

## 3. 核心难点辨析与解题策略

在解决这道题的过程中，我们可能会遇到以下核心难点。结合优质题解的思路，我们逐一分析：

1.  **关键点1**：如何正确理解“日期更新”与上下午判断的关系？
    * **分析**：题目中，日期仅在上午12:00更新。若两个时刻跨越了这个更新点（如从上午11:30到上午12:20），则日期会变化；若未跨越（如从上午12:20到上午5:00），日期不变。此时，通过日期是否变化即可判断上下午。
    * 💡 **学习笔记**：日期更新的本质是“时间从11:59跳到12:00”，这是判断的关键分界点。

2.  **关键点2**：如何将12小时制的时间转换为便于比较的格式？
    * **分析**：12小时制中，12:xx实际等同于0:xx（如上午12:20是上午的开始，下午12:20是下午的开始）。将12点转换为0点后，时间可统一用0-11小时表示，方便直接比较大小。
    * 💡 **学习笔记**：转换12点为0点是简化问题的核心技巧，类似“模12运算”的思想。

3.  **关键点3**：如何判断两个时间是否跨越了日期更新点？
    * **分析**：转换后，若开始时间（小时和分钟）严格大于结束时间，则说明两个时刻之间跨越了12:00（如转换后的11:30 > 0:20）；否则未跨越（如0:20 < 5:00）。
    * 💡 **学习笔记**：时间的大小关系直接反映了是否跨越更新点，比较逻辑是解题的临门一脚。

### ✨ 解题技巧总结
- **问题简化**：将复杂的上下午判断问题简化为时间大小比较问题，通过转换12点为0点降低复杂度。
- **边界处理**：特别注意12点的转换，避免因时间格式错误导致比较结果错误。

**题解一：来源：hycqwq**
* **亮点**：通过结构体重载比较运算符，将时间比较逻辑封装，代码模块化程度高。
* **核心代码片段**：
    ```cpp
    struct tim {
        int h, m;
        void read() {
            scanf("%d:%d", &h, &m);
            if (h == 12) h = 0; // 转换12点为0点
        }
    };
    bool operator<(tim x, tim y) { // 重载小于运算符
        if (x.h != y.h) return x.h < y.h;
        return x.m < y.m;
    }
    bool operator>(tim x, tim y) { return y < x; }
    ```
* **代码解读**：
    > `tim`结构体封装了时间的小时（h）和分钟（m），`read`方法负责读取时间并转换12点为0点。通过重载`<`和`>`运算符，直接比较两个时间的大小。例如，当调用`x > y`时，实际调用的是`y < x`，逻辑清晰。这种设计让主函数中的比较代码更简洁（`if (x > y)`），体现了面向对象的编程思想。
* 💡 **学习笔记**：数据结构的合理设计能简化核心逻辑，让代码更易维护和扩展。

**题解二：来源：what_can_I_do**
* **亮点**：将时间转换为分钟数，通过数值大小直接比较，避免了多条件判断。
* **核心代码片段**：
    ```cpp
    a = a * 60 + b; // 转换为分钟数
    c = c * 60 + d;
    if (a > c) puts("Yes");
    else puts("No");
    ```
* **代码解读**：
    > 每个时间点（如11:30）可以转换为从0点开始的分钟数（11*60+30=690）。将两个时间都转换为分钟数后，直接比较数值大小即可判断先后。这种方法将“小时和分钟的双重比较”简化为“单一数值比较”，逻辑更直观，减少了条件判断的复杂度。
* 💡 **学习笔记**：将复合数据转换为单一数值（如时间转分钟）是简化比较问题的常用技巧。

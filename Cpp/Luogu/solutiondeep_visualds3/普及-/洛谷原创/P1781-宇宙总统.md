# 宇宙总统

## 题目描述

地球历公元 6036 年，全宇宙准备竞选一个最贤能的人当总统，共有 $n$ 个非凡拔尖的人竞选总统，现在票数已经统计完毕，请你算出谁能够当上总统。


## 说明/提示

票数可能会很大，可能会到 $100$ 位数字。

$1 \leq n \leq 20$。

## 样例 #1

### 输入

```
5
98765
12365
87954
1022356
985678
```

### 输出

```
4
1022356```

## 1. 题目解读与核心算法识别

### ✨ 本题主要考察：字符串处理

🗣️ **初步分析**：  
解决“宇宙总统”的关键，在于**如何比较超级大的数字**——这些数字长到普通整数（比如`int`或`long long`）根本装不下！这时候，我们可以用**字符串**来“模拟”这些大数，就像用一张纸条写数字一样。  

字符串处理的核心思想在这里很简单：  
- 比较两个大数的大小，就像比较两张“数字纸条”：  
  1. 先看**纸条长度**（数字位数）：更长的纸条代表更大的数（比如“1022356”比“98765”长，所以更大）；  
  2. 如果长度一样，就**从左到右逐个看数字**（字符）：哪张纸条的某个位置数字更大，整张纸条就更大（比如“98765”和“87954”，第一个数字9>8，所以“98765”更大）。  

本题的题解思路大多围绕这个规则展开：要么用“打擂台”的方式逐个比较字符串（找最大值），要么用排序（把字符串按数字规则排好序再取最大）。核心难点是**正确实现大数的比较规则**，解决方案就是“先比长度，再比字典序”。  

**可视化设计思路**：  
我会用8位像素风做一个“数字擂台赛”动画——每个候选人的票数是一个像素化的“数字条”（比如“98765”是5个像素块，“1022356”是7个像素块）。  
- 长度比较时：更长的数字条会“发光”（高亮），伴随“叮”的音效；  
- 字符比较时：当前比较的数字块会“闪烁”，大的数字块变成红色，小的变成蓝色，伴随“嘀”的音效；  
- 找到最大值时：对应数字条会“跳起来”，播放胜利音效（比如FC游戏的“通关声”）。

## 3. 核心难点辨析与解题策略

### 🧩 核心难点1：超级大的数怎么存？  
**分析**：普通整数类型（比如`int`只能存10位，`long long`存18位）根本装不下100位的数。  
**解决方案**：用**字符串**（`string`）或**字符数组**（`char[]`）存储——字符串就像“数字的容器”，每个字符存一位数字。


### 🧩 核心难点2：怎么比较两个大数的大小？  
**分析**：字符串默认的“字典序”比较（比如“98765”>“1022356”）是错的，因为“9”比“1”大，但“1022356”的位数更多！  
**解决方案**：自定义比较规则：  
1. 先比**长度**（`s.size()`）：长度长的数更大；  
2. 长度相同，再比**字典序**（`s1>s2`）：字符串的字典序和数字的大小一致（比如“98765”>“87954”）。


### 🧩 核心难点3：怎么记住最大值的编号？  
**分析**：我们需要把“票数”和“候选人编号”绑定在一起，不能只存票数。  
**解决方案**：  
- 用**结构体**：把票数和编号装进一个结构体（比如`struct Node { string piao; int id; }`）；  
- 用**变量记录**：打擂台时，每次更新最大值的同时，更新编号变量（比如`id = i`）。


### ✨ 解题技巧总结  
- **技巧A**：用字符串模拟大数，避免溢出问题；  
- **技巧B**：比较大数时，先比长度再比字典序；  
- **技巧C**：用结构体或变量关联“数据”和“属性”（比如票数和编号）。


## 4. C++核心代码实现赏析

### 📌 本题通用核心C++实现参考  
* **说明**：综合了callG和zhoulan119的思路，是最简洁的“打擂台”实现。  
* **完整核心代码**：  
```cpp
#include <iostream>
#include <string>
using namespace std;

int main() {
    int n, id = 1;
    string max_piao;
    cin >> n >> max_piao; // 先读第一个候选人的票数

    for (int i = 2; i <= n; ++i) {
        string current;
        cin >> current;
        // 比较规则：先比长度，再比字典序
        if (current.size() > max_piao.size() || 
            (current.size() == max_piao.size() && current > max_piao)) {
            max_piao = current;
            id = i;
        }
    }

    cout << id << endl << max_piao << endl;
    return 0;
}
```  
* **代码解读概要**：  
  1. 先读入候选人数量`n`和第一个候选人的票数`max_piao`，编号`id`初始化为1；  
  2. 循环读入后面`n-1`个候选人的票数`current`；  
  3. 用“先比长度，再比字典序”的规则判断`current`是否更大：如果是，更新`max_piao`和`id`；  
  4. 最后输出最大票数的编号和票数。


### 📌 题解二（right_cat）核心代码赏析  
* **亮点**：用结构体+排序，清晰展示“如何组织数据”和“自定义排序规则”。  
* **核心代码片段**：  
```cpp
struct node {
    string x; // 票数
    int num;  // 编号
    int lenx; // 长度
};
bool cmp(node a, node b) {
    if (a.lenx > b.lenx) return true; // 长度长的排前面
    if (a.lenx == b.lenx && a.x > b.x) return true; // 长度相同，字典序大的排前面
    return false;
}

int main() {
    int n;
    cin >> n;
    node s[25];
    for (int i = 1; i <= n; ++i) {
        cin >> s[i].x;
        s[i].num = i;
        s[i].lenx = s[i].x.size();
    }
    sort(s + 1, s + n + 1, cmp); // 按自定义规则排序
    cout << s[1].num << endl << s[1].x << endl;
}
```  
* **代码解读**：  
  - `struct node`把票数、编号、长度绑在一起，方便处理；  
  - `cmp`函数是“排序的规则”：先比长度，再比字典序；  
  - `sort`函数会按照`cmp`的规则把结构体数组排好序，第一个元素就是最大的数。  
* 💡 **学习笔记**：结构体是组织“关联数据”的好工具，排序时可以用`cmp`函数定制规则。

# 题目信息

# 头像上传

## 题目描述

有一天无聊的你在注册一个无聊的网站，注册完的第一步自然是上传头像。

你发现这个网站总共可以上传 $n$ 个头像，每个头像必须是正方形，并且长宽至少为 $L \times L$

在图片上传前，系统会对图片进行如下处理：如果图片的任何一边长度超过了 $G$ ，那么系统会不断地对图片的长宽同时减半（向下取整），直至两边长度 $\leq G$ 为止。

你现在找到了 $n$ 张可供上传的图片，第 $i$ 张的尺寸是 $W_i \times H_i$。

如果图片有任何一边小于 $L$，请输出 "Too Young"
 
如果图片满足大小条件但不为正方形，请输出"Too Simple"

如果图片满足大小条件并且是正方形，请输出"Sometimes Naive" 

以上所有字符串输出时均不包含引号。

## 说明/提示

对于 $30\%$ 的数据，满足 $n=1$

对于 $100\%$的数据，满足 $n \leq 2000$，$W_i,H_i ,G\leq 10000$

## 样例 #1

### 输入

```
3 5 10
12 3
7 8
5 5
```

### 输出

```
Too Young
Too Simple
Sometimes Naive```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：`模拟`  

🗣️ **初步分析**：  
解决「头像上传」的关键，就是**严格按照题目描述的规则“模拟”每一步操作**。什么是“模拟”？比如妈妈让你把苹果洗干净、削皮、切成块——你只要按顺序做这三件事，就是“模拟”。在本题中，我们要对每个头像做三件事：  
1. **缩放**：如果头像的长或宽超过G，就同时把长和宽减半（向下取整），直到都不超过G；  
2. **判断大小**：如果缩放后的长或宽有一个小于L，输出“Too Young”；  
3. **判断形状**：如果大小符合要求但不是正方形，输出“Too Simple”；如果是正方形，输出“Sometimes Naive”。  

### 核心算法流程与可视化设计思路  
- **缩放过程**：用像素块表示头像的长和宽（比如长是12就用12个横向像素，宽是3就用3个纵向像素），每次缩放时，像素块**同时缩小一半**（比如12→6，3→1），直到都不超过G（比如G=10时，6和1都≤10，停止缩放）。  
- **判断过程**：用颜色标记结果——红色表示“Too Young”（比如宽1<L=5），黄色表示“Too Simple”（比如长7、宽8都≥5但不等），绿色表示“Sometimes Naive”（比如长5、宽5）。  
- **游戏化设计**：缩放时播放“叮”的像素音效，判断时播放“滴”的音效，输出绿色结果时播放“哒”的胜利音效；每处理一个头像得1分，全部处理完显示“通关”动画（比如像素星星闪烁）。

## 3. 核心难点辨析与解题策略

<difficulty_intro>
模拟题的“难点”往往不是算法，而是**容易漏掉题目中的细节**。结合优质题解的经验，我总结了3个最容易踩的坑和解决办法：
</difficulty_intro>

### 1. 缩放的条件：是“或”不是“且”  
* **坑点**：误以为“只有当长和宽都超过G时才缩放”（用`&&`），但题目说“任何一边超过G”（用`||`）。  
* **解决**：反复读题！题目中的“任何一边”就是“只要有一个满足”，对应C++的`||`运算符。  
* 💡 **学习笔记**：模拟题的条件要和题目**一字不差**！

### 2. 操作的顺序：先缩放，再判断L  
* **坑点**：先判断“长或宽< L”，再缩放——比如初始长12≥L=5，但缩放后长3<L，这时候应该输出“Too Young”，但先判断的话会漏掉这个情况。  
* **解决**：严格按照题目顺序：先缩放（处理超过G的情况），再判断大小（是否< L），最后判断形状。  
* 💡 **学习笔记**：模拟题的步骤要和题目**顺序一致**！

### 3. 向下取整的处理：C++整数除法自动完成  
* **坑点**：担心“减半后怎么向下取整”，比如12/2=6（正确），3/2=1（正确，因为C++整数除法会自动舍去小数部分）。  
* **解决**：直接用`w/=2`或`w>>1`（位运算），不用额外调用`floor`函数——C++会帮你搞定向下取整！  
* 💡 **学习笔记**：整数除法的“截断”特性，正好符合题目“向下取整”的要求！


## 4. C++核心代码实现赏析

<code_intro_overall>
先看一份**通用核心代码**——它综合了所有优质题解的优点，逻辑清晰，适合新手模仿！
</code_intro_overall>

### 本题通用核心C++实现参考  
* **说明**：这份代码来自题解一（犇犇犇犇），是模拟题的“标准写法”，覆盖了所有规则，且可读性极强。  
* **完整核心代码**：  
  ```cpp
  #include <iostream>
  using namespace std;

  int main() {
      int n, l, g; // n个头像，最小尺寸L，最大尺寸G
      cin >> n >> l >> g;
      for (int i = 1; i <= n; ++i) {
          int w, h; // 当前头像的长和宽
          cin >> w >> h;
          // 1. 缩放：长或宽超过G时，同时减半
          while (w > g || h > g) {
              w /= 2;
              h /= 2;
          }
          // 2. 判断大小：有一边<L → Too Young
          if (w < l || h < l) {
              cout << "Too Young" << endl;
          } 
          // 3. 判断形状：不是正方形 → Too Simple；是 → Sometimes Naive
          else if (w != h) {
              cout << "Too Simple" << endl;
          } else {
              cout << "Sometimes Naive" << endl;
          }
      }
      return 0;
  }
  ```
* **代码解读概要**：  
  1. 读入总头像数`n`、最小尺寸`l`、最大尺寸`g`；  
  2. 循环处理每个头像：读入长`w`和宽`h`；  
  3. 用`while`循环缩放，直到`w`和`h`都≤`g`；  
  4. 按顺序判断三个条件，输出对应结果。


<code_intro_selected>
接下来看几个**有亮点的代码片段**，学习不同的实现技巧！
</code_intro_selected>

### 片段一：递归实现缩放（来自Doveqise）  
* **亮点**：用递归代替`while`循环，代码更简洁。  
* **核心代码片段**：  
  ```cpp
  void solve(int w, int h) {
      if (l > min(w, h)) { puts("Too Young"); return; } // 大小不够
      if (g >= max(w, h)) { // 不需要缩放
          puts(w == h ? "Sometimes Naive" : "Too Simple");
          return;
      }
      solve(w >> 1, h >> 1); // 递归缩放（除以2）
  }
  ```
* **代码解读**：  
  - 函数`solve(w, h)`的作用是处理尺寸为`w×h`的头像；  
  - 首先判断“最小边< L”→输出“Too Young”；  
  - 然后判断“最大边≤G”→不需要缩放，直接判断形状；  
  - 否则递归调用`solve(w>>1, h>>1)`（`>>1`相当于除以2），继续缩放。  
* 💡 **学习笔记**：递归可以把“重复做一件事”的逻辑写得更简洁，但要注意终止条件！

### 片段二：位运算优化（来自YLWang）  
* **亮点**：用`>>1`代替`/2`，稍微提升运行速度。  
* **核心代码片段**：  
  ```cpp
  while (a > g || b > g) a >>= 1, b >>= 1;
  ```
* **代码解读**：  
  - `a >>= 1`等价于`a = a / 2`（向下取整）；  
  - 位运算比除法运算快一点，在竞赛中经常用来优化时间。  
* 💡 **学习笔记**：小技巧能让代码更高效，但不要为了优化而牺牲可读性！

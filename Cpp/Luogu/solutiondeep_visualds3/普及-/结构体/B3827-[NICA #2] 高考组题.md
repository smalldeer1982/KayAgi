# 题目信息

# [NICA #2] 高考组题

## 题目描述

高考是能决定每个考生命运的考试，因此作为组卷人的 Aya 将高考组卷视为重中之重。现在 Aya 有 $n$ 个备选的高考题。

对于每一道可能选入试卷的高考题，其有 $k$ 个指标来评判它各个方面的维度，例如难度、新颖度、与现实生活的结合性等。每个指标均为一个正整数。

Aya 认为，一个题被放入高考卷的必要性，是这些指标的平均值。他认为，必要性前二高的试题必须要被放进高考试卷。若有多个必要性前二大的试题则选择较早出现的那个。请告诉 Aya 哪个题目是必须要放进高考试卷的。

## 说明/提示

数据保证，$2 \leq n \leq 100$，$1 \leq k \leq 10$，$1 \leq$ 一道试题的各个指标 $\leq 100$。

## 样例 #1

### 输入

```
3 2
10 20
15 25
20 30```

### 输出

```
3
2```

## 样例 #2

### 输入

```
3 2
10 20
5 25
1 2```

### 输出

```
1
2```

#### 1. 题目解读与核心算法识别
✨ **本题主要考察**：`编程技巧应用`（打擂台求最值/排序应用）  
🗣️ **初步分析**：
> 本题核心是高效找出前两大值（必要性最高的两道题）。如同擂台比武：大当家（最大值）和二当家（次大值）守擂，新挑战者依次上台比拼。若挑战者强于大当家，则原大当家降为二当家；若只强于二当家，则直接取代二当家。  
- **核心难点**：需正确处理相同平均值情况（取输入顺序更早的题），并高效维护前两大值。
- **可视化设计**：像素动画将模拟"擂台比武"场景。每个题目显示为像素战士，头顶平均值数字。挑战时：① 与大当家数值碰撞比较（黄光特效）② 成功则原大当家退至二当家位（蓝光移动）③ 失败则与二当家碰撞（红光特效）。音效：升级时8-bit胜利音效，降级时低沉音效。
- **复古元素**：FC红白机风格界面，题目战士采用不同像素配色（红/蓝/绿），控制面板含"单步挑战"按钮和调速滑块。

#### 2. 精选优质题解参考
**题解一（loushujia - 17赞）**  
* **点评**：采用"打擂台"思路，仅用单次遍历（O(n)时间复杂度）即解决问题。代码中`mx1`/`mx2`变量如同大当家和二当家，`x1`/`x2`记录其题号。亮点在于发现初始版本未处理"大当家降级"逻辑后迅速修正（`mx2=mx1,x2=x1`），体现敏锐的问题定位能力。边界处理严谨（n≥2保证不会空输出），变量命名直观，是竞赛场景高效实现的典范。

**题解二（Peaceful_Evening - 8赞）**  
* **点评**：巧用`stable_sort`稳定排序，避免处理相同值顺序问题。将平均值比较转化为整数总和比较（`sum`代替`ave`），彻底规避浮点精度风险。结构体设计简洁（`id`+`jun`），代码可读性极强。虽时间复杂度O(nlogn)，但n≤100时完全可行，教学演示价值突出。

**题解三（lewis_lu_CH3COOH - 2赞）**  
* **点评**：结构体排序的经典实现。核心创新点在于用`num`（指标总和）替代平均值计算，避免浮点运算。自定义排序函数`cmp`先按总和降序，再按编号升序，严格满足题目要求。变量命名规范（`num`/`type`），适合初学者理解多条件排序思想。

#### 3. 核心难点辨析与解题策略
1. **难点一：高效维护动态前两名**  
   * **分析**：传统排序需O(nlogn)时间，而打擂台法仅需O(n)。关键在正确处理更新逻辑：当新值>最大值时，需将原最大值转移给次大值（见loushujia解法）。  
   * 💡 **学习笔记**：动态维护最值比全局排序更高效时，优先考虑打擂台。

2. **难点二：避免浮点精度陷阱**  
   * **分析**：5份题解中3份（Peaceful_Evening等）用整数总和替代平均值。因k相同且指标为整数，总和比较等价于平均值比较，且无精度损失。  
   * 💡 **学习笔记**：浮点数比较是常见错误源，能用整数运算时优先转换。

3. **难点三：处理相同值顺序**  
   * **分析**：题目要求相同值时取输入顺序早的（即编号小）。打擂台法中天然满足（先出现先记录）；排序法中需自定义比较规则（如`return a.id < b.id`）。  
   * 💡 **学习笔记**：多条件排序时，次要条件要显式声明。

### ✨ 解题技巧总结
- **技巧1：问题转化**：将平均值比较转化为总和比较，避免浮点陷阱（Peaceful_Evening）。  
- **技巧2：空间换效率**：打擂台法用额外变量存储状态，换时间效率（loushujia）。  
- **技巧3：稳定排序应用**：当需要保持原始顺序时，优先选`stable_sort`（Peaceful_Evening）。

#### 4. C++核心代码实现赏析
**通用核心实现（打擂台法）**  
```cpp
#include <cstdio>
int main() {
    int n, k, x1 = 0, x2 = 0, a;
    double mx1 = 0, mx2 = 0, p;
    scanf("%d%d", &n, &k);
    for (int i = 1; i <= n; i++) {
        p = 0;
        for (int j = 0; j < k; j++) {
            scanf("%d", &a);
            p += a;
        }
        p /= k; // 计算平均值
        if (p > mx1) {        // 挑战大当家成功
            mx2 = mx1; x2 = x1; // 大当家降级
            mx1 = p; x1 = i;    // 新大当家
        } else if (p > mx2) { // 只挑战二当家成功
            mx2 = p; x2 = i;
        }
    }
    printf("%d\n%d", x1, x2);
    return 0;
}
```
**代码解读概要**：  
> ① 外层循环遍历每道题 ② 内层循环累加指标值并求平均 ③ 核心擂台逻辑：先战大当家，成功则引发"降级链"反应；失败则战二当家。④ 最终输出守擂成功的两位题号。

---

**题解一（loushujia）核心片段**  
```cpp
if (mx1 < p) {          // 挑战大当家
    mx2 = mx1; x2 = x1; // 降级关键！
    mx1 = p; x1 = i; 
} else if (mx2 < p) {   // 挑战二当家
    mx2 = p; x2 = i;
}
```
**亮点**：降级逻辑一行代码实现状态转移  
**学习笔记**：状态更新要考虑"原状态的去向"，如同接力赛传递交接棒。

**题解二（Peaceful_Evening）核心片段**  
```cpp
stable_sort(s+1, s+n+1, [](p a, p b) {
    return a.jun > b.jun; // 仅需总和降序
});
```
**亮点**：利用稳定排序自动处理同值顺序  
**学习笔记**：当次要条件恰好是输入顺序时，`stable_sort`比自定义排序更简洁。

**题解三（lewis_lu_CH3COOH）核心片段**  
```cpp
bool cmp(node a, node b) {
    if (a.num != b.num) 
        return a.num > b.num; // 总和优先
    return a.type < b.type;   // 编号次要
}
```
**亮点**：显式声明双条件避免歧义  
**学习笔记**：多条件排序需明确主次关系，如同比赛先看成绩再看提交时间。

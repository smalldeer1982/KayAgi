## 题目描述

给定一个正整数，规定一次操作为选定 $l,r$，删去所有从后往前数第 $l\sim r$ 位的数字，并且将剩下的数字组成一个新的正整数。如 $123456$ 删去从后往前数的第 $2\sim 3$ 位就会变成 $1236$。

现在有 $T$ 组询问，每次询问给定一个正整数 $n$，你需要回答：对于这个正整数，能否通过**最多一次操作**（不操作也算）将其变为 $4$ 的倍数。

但是请注意，不能把所有的数位全都删完。

## 说明/提示

### 样例 1 解释

对第一组数据：删去从后往前数第 $2\sim 3$ 位，剩下的数是 $4$，是 $4$ 的倍数。

对第二组数据：可以证明没有任何一种方案能够达成目标。

对第三组数据：删去从后往前数第 $1$ 位，剩下的数是 $28$，是 $4$ 的倍数。

### 数据范围

对于前 $10\%$ 的数据，保证 $1\le n\le 9$。\
对于前 $30\%$ 的数据，保证 $1\le T\le 10,1\le n\le 100$。\
对于另外 $10\%$ 的数据，保证 $T=1$。\
对于前 $60\%$ 的数据，保证 $1\le T\le 10,1\le n\le 10^9$。\
对于 $100\%$ 的数据，$1\le T\le 10^2,1\le n\le 10^{18}$。

## 样例 #1

### 输入

```
3
234
1
286```

### 输出

```
Yes
No
Yes```

## 样例 #2

### 输入

```
1
2386```

### 输出

```
Yes```

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：数学（数论）+ 枚举

🗣️ **初步分析**：  
解决这道题的关键，是要用到一个**数论小技巧**——**判断一个数是不是4的倍数，只需要看它的最后两位**！就像判断一个年份是不是闰年，要先看后两位能不能被4整除一样。比如1234，最后两位是34，34÷4=8.5，所以1234不是4的倍数；而1236的最后两位是36，36÷4=9，所以1236是4的倍数。  

回到题目：我们可以**最多删一次数字**（也可以不删），要让剩下的数是4的倍数。那是不是要枚举所有可能的删法？不用！因为删完后，剩下的数如果是4的倍数，要么它是**1位**（本身得是4的倍数，比如4、8），要么它的**最后两位是4的倍数**（而这两位一定是原数里连续的两位）。所以我们只需要检查两种情况：  
1. 原数里有没有某一位是4的倍数（比如4、8）；  
2. 原数里有没有连续两位组成的数是4的倍数（比如12、28）。  

**可视化设计思路**：  
我们会用**8位像素风**展示数字串（比如“234”是三个8x8的像素块），用颜色高亮当前检查的位：  
- 检查单个数时，数字变**黄色**，播放“叮”的音效；如果满足条件，立刻变**绿色**，弹出“找到啦！”的提示；  
- 检查连续两位时，两个数字一起变**黄色**，播放“咚咚”的音效；满足条件则变**绿色**。  
动画还支持“单步执行”（一步步看检查过程）和“自动播放”（像贪吃蛇AI一样跑完全程），胜利时会播放复古“通关音效”（比如《超级马里奥》的“叮——”）。

## 3. 核心难点辨析与解题策略

### 核心难点1：理解“4的倍数只看最后两位”的数论特征  
**分析**：为什么不用看前面的位？比如1234，前面的“12”其实是12×100，而100是4的倍数（100÷4=25），所以12×100肯定是4的倍数。最后两位34才是“余数”——余数能被4整除，整个数就能被4整除！  

**策略**：记牢这个小技巧，以后遇到“判断4的倍数”的问题，直接看最后两位就行～

### 核心难点2：为什么只需要检查“1位或连续两位”？  
**分析**：不管怎么删数字，剩下的数要么是1位（得是4的倍数），要么长度≥2（最后两位得是4的倍数）。而剩下的数的最后两位，要么是原数里的连续两位（比如删中间的“3”，剩下“124”的最后两位是“24”，原数里就有“24”），要么剩下的数是1位（比如删成“4”）。  

**策略**：不用想复杂的删法，只盯着“1位”和“连续两位”就够了！

### 核心难点3：处理“超大数”的存储  
**分析**：n可以到10¹⁸，普通的`int`（最多存2×10⁹）根本装不下！这时候得用**字符串**存每一位数字——比如“123456789012345678”直接存在字符串里，想取哪一位就取哪一位。  

**策略**：遇到“大数”问题，第一反应就是用字符串！

### ✨ 解题技巧总结  
- **数论简化**：遇到倍数问题，先想有没有“只看末尾几位”的特征（比如2看最后1位，5看最后1位，8看最后3位）；  
- **字符串存大数**：超过`int`范围的数，用字符串存准没错；  
- **避免冗余枚举**：先想“有没有更聪明的办法”，别上来就暴力枚举所有可能。


## 4. C++核心代码实现赏析

### 本题通用核心C++实现参考  
* **说明**：综合洛谷网校题解的思路，实现了最简洁的核心逻辑——用字符串存数字，检查1位或连续两位。  
* **完整核心代码**：  
```cpp
#include <iostream>
#include <string>
using namespace std;

int main() {
    int T;
    cin >> T;
    while (T--) {
        string s;
        cin >> s;
        bool ok = false;
        
        // 检查情况1：有没有单个数是4的倍数（0、4、8）
        for (char c : s) {
            int digit = c - '0';
            if (digit % 4 == 0) {
                ok = true;
                break;
            }
        }
        
        // 情况1没找到，检查情况2：连续两位是4的倍数
        if (!ok) {
            for (int i = 0; i < (int)s.size() - 1; ++i) {
                int num = (s[i] - '0') * 10 + (s[i+1] - '0');
                if (num % 4 == 0) {
                    ok = true;
                    break;
                }
            }
        }
        
        cout << (ok ? "Yes" : "No") << endl;
    }
    return 0;
}
```
* **代码解读概要**：  
  1. 读入T组测试用例；  
  2. 把每个数存成字符串`s`（比如“234”）；  
  3. 先遍历`s`的每个字符，检查单个数是不是4的倍数；  
  4. 如果没找到，再遍历连续两位，计算它们的数值（比如“2”和“3”组成12），检查是不是4的倍数；  
  5. 最后输出“YES”或“NO”。


### 题解一：洛谷网校（核心片段赏析）  
* **亮点**：用字符串存大数，遍历检查两种情况，逻辑无冗余。  
* **核心代码片段**：  
```cpp
// 检查连续两位的关键代码
for (int i = 0; i < (int)s.size() - 1; ++i) {
    int num = (s[i] - '0') * 10 + (s[i+1] - '0');
    if (num % 4 == 0) {
        ok = true;
        break;
    }
}
```
* **代码解读**：  
  - `s[i] - '0'`：把字符（比如'2'）转成数字（2）；  
  - `(s[i]-'0')*10 + (s[i+1]-'0')`：把两个字符拼成两位数（比如'2'和'3'变成23）；  
  - 只要有一个两位数能被4整除，立刻标记`ok=true`，跳出循环——不用检查后面的了！  

* 💡 **学习笔记**：处理“字符转数字”时，记得用`c - '0'`（比如'0'的ASCII码是48，'1'是49，减完就是数字1）；遍历连续两位时，要注意`i`的范围是`0`到`size-2`（不然`i+1`会越界）！

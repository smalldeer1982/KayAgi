# 题目信息

# [NOIP 2005 普及组] 陶陶摘苹果

## 题目描述

陶陶家的院子里有一棵苹果树，每到秋天树上就会结出 $10$ 个苹果。苹果成熟的时候，陶陶就会跑去摘苹果。陶陶有个 $30$ 厘米高的板凳，当她不能直接用手摘到苹果的时候，就会踩到板凳上再试试。


现在已知 $10$ 个苹果到地面的高度，以及陶陶把手伸直的时候能够达到的最大高度，请帮陶陶算一下她能够摘到的苹果的数目。假设她碰到苹果，苹果就会掉下来。


## 说明/提示

**【题目来源】**

NOIP 2005 普及组第一题

100 200 150 140 129 134 167 198 200 111
110
```

5```

## 3. 核心难点辨析与解题策略

<difficulty_intro>
看似简单的题目中，也有需要注意的关键点。以下是解题时的核心难点及应对策略：

1.  **关键点1：正确理解“总高度”的计算**
    * **分析**：题目中“陶陶不能直接摘到就踩板凳”，但实际“身高+30”的总高度一定≥身高，因此所有能直接摘到的苹果（高度≤身高）必然也能被“总高度”覆盖。只需计算“身高+30”，无需单独比较“直接摘”和“踩板凳”两种情况。
    * 💡 **学习笔记**：抓住问题本质，避免冗余判断。

2.  **关键点2：高效遍历与统计**
    * **分析**：需要遍历10个苹果，逐个比较高度。优质题解通过提前计算总高度（如`H +=30`），避免每次循环重复计算，提升效率。
    * 💡 **学习笔记**：预处理关键值（如总高度），减少循环内计算量，是优化代码的常用技巧。

3.  **关键点3：代码的简洁性与可读性**
    * **分析**：变量命名（如`ans`、`cnt`）要直观，循环结构（如`for`）要清晰。例如，用`for(int i=0;i<10;i++)`明确遍历10次，避免越界。
    * 💡 **学习笔记**：好的代码不仅能运行，还能让人一目了然。

### ✨ 解题技巧总结
<summary_best_practices>

- **利用逻辑值特性**：`true`等价于1，`false`等价于0，可直接累加计数（如`s +=!(H<height[i])`）。

## 4. C++核心代码实现赏析

<code_intro_overall>
我们综合多个优质题解，提炼出一个简洁、高效的核心实现，帮助大家快速掌握解题框架。
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：此代码综合了RedContritio和mairuisheng的思路，兼顾简洁性与效率，适合作为模板学习。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    using namespace std;

    int main() {
        int apple[10], height, cnt = 0;
        // 输入10个苹果的高度
        for (int i = 0; i < 10; ++i) {
            cin >> apple[i];
        }
        // 输入陶陶身高并计算总高度
        cin >> height;
        height += 30;
        // 遍历统计可摘的苹果数
        for (int i = 0; i < 10; ++i) {
            if (apple[i] <= height) {
                cnt++;
            }
        }
        // 输出结果
        cout << cnt << endl;
        return 0;
    }
    ```
* **代码解读概要**：
    > 代码首先读取10个苹果的高度，然后读取陶陶的身高并加上30cm（板凳高度）得到总高度。接着遍历所有苹果，统计高度≤总高度的数量，最后输出结果。核心逻辑是“输入→计算总高度→遍历比较→统计”，结构清晰，适合初学者理解。

<code_intro_selected>
接下来，我们分析优质题解中的核心代码片段，学习其中的亮点和技巧。

**题解一：作者RedContritio**
* **亮点**：利用逻辑值直接累加，减少条件判断的冗余操作。
* **核心代码片段**：
    ```cpp
    for(int i=0;i<10;i++)s+=!(H<height[i]);
    ```
* **代码解读**：
    > 这段代码中，`H<height[i]`判断苹果是否高于总高度。若苹果≤总高度（即`H>=height[i]`），则`!(H<height[i])`为`true`（等价于1），累加到`s`中；否则为`false`（等价于0）。这样直接用逻辑值的结果累加，避免了`if`语句的条件判断，代码更简洁高效。
* 💡 **学习笔记**：逻辑值的布尔运算可以简化代码，尤其在统计符合条件的数量时非常实用。

**题解二：作者mairuisheng**
* **亮点**：代码简洁直白，变量命名直观，适合初学者理解。
* **核心代码片段**：
    ```cpp
    ha+=30;
    for(i=1;i<=10;i++)if(a[i]<=ha)ans++;
    ```
* **代码解读**：
    > 首先将身高`ha`加上30cm得到总高度，然后遍历每个苹果的高度`a[i]`，若苹果≤总高度则答案`ans`加1。变量`ha`（身高）、`ans`（答案）命名清晰，逻辑一目了然，是“模拟”思想的典型实现。
* 💡 **学习笔记**：简单问题用简单代码解决，清晰比“炫技”更重要。

**题解三：作者installb**
* **亮点**：用递推（前缀和）统计数量，展示了不同的解题思路。
* **核心代码片段**：
    ```cpp
    for(i=1;i<=251;i++)a[i]=a[i]+a[i-1];
    printf("%d\n",a[c]);
    ```
* **代码解读**：
    > 这段代码中，数组`a[i]`初始记录高度为`i`的苹果数量。通过递推`a[i] += a[i-1]`，`a[c]`最终表示“高度≤c”的苹果总数（即陶陶能摘到的数量）。这种方法将“遍历比较”转化为“前缀和累加”，虽然对本题来说稍显复杂，但在处理大量数据时（如苹果高度范围很大）可以提升效率。
* 💡 **学习笔记**：换一种思路（如前缀和）可能找到更高效的解法，多思考问题的不同模型。

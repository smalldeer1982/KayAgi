# 题目信息

# 01迷宫

## 题目描述

有一个仅由数字 $0$ 与 $1$ 组成的 $n \times n$ 格迷宫。若你位于一格 $0$ 上，那么你可以移动到相邻 $4$ 格中的某一格 $1$ 上，同样若你位于一格 $1$ 上，那么你可以移动到相邻 $4$ 格中的某一格 $0$ 上。

你的任务是：对于给定的迷宫，询问从某一格开始能移动到多少个格子（包含自身）。

## 说明/提示

对于样例，所有格子互相可达。

- 对于 $20\%$ 的数据，$n \leq 10$；
- 对于 $40\%$ 的数据，$n \leq 50$；
- 对于 $50\%$ 的数据，$m \leq 5$；
- 对于 $60\%$ 的数据，$n,m \leq 100$；
- 对于 $100\%$ 的数据，$1\le n \leq 1000$，$1\le m \leq 100000$。

## 样例 #1

### 输入

```
2 2
01
10
1 1
2 2
```

### 输出

```
4
4
```

# 「01迷宫」题解分析与核心思想提炼


## **一、题目核心要求**
给定一个由0和1组成的`n×n`迷宫，规则是：从0出发可走到相邻（上下左右）的1，从1出发可走到相邻的0。对于每个询问点`(x,y)`，求其能到达的格子数（包括自身）。


## **二、核心思想：连通块划分**
所有能互相到达的点构成一个**连通块**，块内所有点的答案（能到达的格子数）**完全相同**。因此，解决问题的关键是：
1. **预处理**：找出所有连通块，计算每个块的大小（即块内格子数）。
2. **查询**：对于每个询问点，直接返回其所属连通块的大小。


## **三、常见实现方法**
### 1. **广度优先搜索（BFS）**
- **思路**：从每个未访问的点出发，用BFS遍历其能到达的所有点（构成一个连通块），记录块的大小，并标记块内所有点的答案。
- **实现细节**：
  - 用队列存储待访问的点。
  - 用数组`vis`标记是否访问过（避免重复遍历）。
  - 用数组`ans`记录每个点的答案（即所属连通块的大小）。
- **示例**（来自「Nightfury12366」的题解）：
  ```cpp
  LL bfs(int x, int y) {
      LL sum = 0;
      queue<node> q;
      node s = {x, y, map[x][y]};
      sum++; num++;
      remenber[x][y] = num; // 标记所属连通块编号
      q.push(s);
      while (!q.empty()) {
          node u = q.front(); q.pop();
          for (int i = 0; i < 4; i++) {
              int nx = u.x + dx[i], ny = u.y + dy[i];
              if (nx < 1 || nx > n || ny < 1 || ny > n || remenber[nx][ny]) continue;
              if (map[u.x][u.y] != map[nx][ny]) {
                  remenber[nx][ny] = num;
                  sum++;
                  q.push({nx, ny, map[nx][ny]});
              }
          }
      }
      ans[num] = sum; // 记录连通块大小
      return sum;
  }
  ```


### 2. **深度优先搜索（DFS）**
- **思路**：与BFS类似，但用递归或栈遍历连通块。
- **注意**：DFS可能会因递归深度过大（如`n=1e3`）导致**栈溢出**，因此**不推荐**用于大规模数据。


### 3. **并查集（Disjoint Set Union, DSU）**
- **思路**：将每个格子视为一个节点，若两个格子连通（可互相到达），则合并它们的集合。记录每个集合的大小，查询时返回该节点所属集合的大小。
- **实现细节**：
  - 将二维坐标`(x,y)`哈希为一维（如`(x-1)*n + y`），方便并查集处理。
  - 用`fa`数组记录父节点，`size`数组记录集合大小。
- **示例**（来自「陈新月」的题解）：
  ```cpp
  int getf(int now) { // 路径压缩
      if (fa[now] == now) return now;
      else return fa[now] = getf(fa[now]);
  }
  void unite(int x1, int y1, int x2, int y2) { // 合并集合
      int u = cul_num(x1, y1), v = cul_num(x2, y2);
      int fu = getf(u), fv = getf(v);
      if (fu != fv) {
          fa[fv] = fu;
          size[fu] += size[fv];
      }
  }
  ```


## **四、难点与优化策略**
### 1. **避免重复计算**
- **问题**：若每次询问都重新搜索，时间复杂度为`O(m*n²)`（`m=1e5`，`n=1e3`），会超时。
- **解决**：**预处理所有连通块**，查询时`O(1)`返回。例如，用数组`ans`记录每个点的答案，或用并查集的`size`数组记录集合大小。


### 2. **标记连通块**
- **问题**：如何高效标记每个点所属的连通块？
- **解决**：
  - BFS/DFS时，用数组`remenber`标记每个点的连通块编号，用`ans`数组记录每个编号对应的大小（如「Nightfury12366」的题解）。
  - 直接将每个点的答案（连通块大小）存入数组`v`（如「Akashicw」的题解）。


### 3. **效率优化**
- **BFS优于DFS**：BFS用队列实现，不会爆栈，更适合大规模数据。
- **输入优化**：用`scanf`或`getchar`读取字符，避免`cin`的慢速度（如「陈新月」的`read`函数）。
- **空间优化**：用数组代替`queue`（如「Akashicw」的题解），减少容器的 overhead。


## **五、总结**
- **核心思想**：连通块划分，预处理每个块的大小，查询时直接返回。
- **最佳实现**：BFS预处理连通块，用数组标记每个点的答案，查询时`O(1)`返回。
- **关键优化**：预处理避免重复计算，用BFS确保效率，输入优化提升读取速度。


## **六、示例代码（BFS预处理）**
```cpp
#include <iostream>
#include <queue>
#include <cstring>
using namespace std;

const int N = 1001;
const int dx[] = {1, 0, -1, 0};
const int dy[] = {0, 1, 0, -1};

char map[N][N];
int ans[N][N]; // 记录每个点的答案（连通块大小）
bool vis[N][N]; // 标记是否访问过

int n, m;

struct Node {
    int x, y;
};

void bfs(int x, int y) {
    queue<Node> q;
    q.push({x, y});
    vis[x][y] = true;
    int cnt = 0;
    // 临时存储连通块内的点，以便后续赋值
    vector<Node> nodes;
    while (!q.empty()) {
        Node u = q.front();
        q.pop();
        nodes.push_back(u);
        cnt++;
        for (int i = 0; i < 4; i++) {
            int nx = u.x + dx[i];
            int ny = u.y + dy[i];
            if (nx < 1 || nx > n || ny < 1 || ny > n) continue;
            if (vis[nx][ny]) continue;
            if (map[u.x][u.y] != map[nx][ny]) {
                vis[nx][ny] = true;
                q.push({nx, ny});
            }
        }
    }
    // 将连通块内的所有点的答案设为cnt
    for (Node node : nodes) {
        ans[node.x][node.y] = cnt;
    }
}

int main() {
    cin >> n >> m;
    for (int i = 1; i <= n; i++) {
        cin >> (map[i] + 1); // 读取第i行的字符（从map[i][1]开始）
    }
    // 预处理所有连通块
    memset(vis, false, sizeof(vis));
    for (int i = 1; i <= n; i++) {
        for (int j = 1; j <= n; j++) {
            if (!vis[i][j]) {
                bfs(i, j);
            }
        }
    }
    // 处理查询
    while (m--) {
        int x, y;
        cin >> x >> y;
        cout << ans[x][y] << endl;
    }
    return 0;
}
```


## **七、拓展思考**
- **并查集的应用**：若迷宫是动态的（如不断添加或删除边），并查集的合并/查询操作更高效。
- **更大规模数据**：若`n=1e4`，`m=1e5`，需要更高效的输入方式（如`mmap`）和空间优化（如压缩存储）。
- **多源BFS**：若有多个起点，可同时将所有起点入队，一次BFS处理多个连通块。


**结论**：「01迷宫」的核心是连通块划分，预处理是关键，BFS是最适合的实现方式。通过预处理所有连通块，可将查询时间降为`O(1)`，满足大规模数据的要求。

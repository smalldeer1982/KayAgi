# [NOI2016] 网格

## 题目描述

跳蚤国王和蛐蛐国王在玩一个游戏。

他们在一个 $n$ 行 $m$ 列的网格上排兵布阵。其中的 $c$ 个格子中 $(0 \leq c \leq n\cdot m)$，每个格子有一只蛐蛐，其余的格子中，每个格子有一只跳蚤。

我们称占据的格子有公共边的两只跳蚤是相邻的。

我们称两只跳蚤是连通的，当且仅当这两只跳蚤相邻，或存在另一只跳蚤与这两只跳蚤都连通。

现在，蛐蛐国王希望，将某些（零个，一个或多个）跳蚤替换成蛐蛐，使得在此之后存在至少两只跳蚤不连通。

例如：图 $1$ 描述了一个 $n=4$，$m=4$，$c=2$ 的情况。

![](https://cdn.luogu.com.cn/upload/image_hosting/96tey4uv.png)

这种情况下蛐蛐国王可以通过将第二行第二列，和第三行第三列的两只跳蚤替换为蛐蛐，从而达成他的希望，如右图所示。并且，不存在更优的方案，但是可能存在其他替换两只跳蚤的方案。

你需要首先判断蛐蛐国王的希望能否被达成。如果能够达成，你还需要最小化被替换的跳蚤的个数。

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：`图论 - 割点与连通性`

🗣️ **初步分析**：
> 解决网格问题就像拆除城市的关键桥梁。核心思想是：**将巨大网格离散化，提取关键点（蛐蛐周围5x5区域）建图**。通过判断连通性和寻找割点，确定最少操作数。
> - **核心难点**：直接处理10^9网格不可行，需提取关键点；避免假割点需扩展至5x5区域；边界情况需特判
> - **算法流程**：
>   1. 特判-1（跳蚤<2或=2且相邻）
>   2. 提取关键点建图（蛐蛐周围5x5区域）
>   3. BFS/并查集判连通性（0）
>   4. Tarjan求割点（1）
>   5. 否则输出2
> - **可视化设计**：像素网格中，绿色跳蚤连通块随BFS蔓延；割点变红时播放"叮"声；关键步骤显示low/dfn值变化

## 3. 核心难点辨析与解题策略

1. **关键点提取范围**  
   *分析*：3x3区域可能导致假割点（如图1）。**解决方案**：提取蛐蛐周围5x5区域确保完备性  
   💡 学习笔记：空间换正确性，25倍区域覆盖所有可能割点  

   ![假割点示例](https://cdn.luogu.com.cn/upload/image_hosting/sshoskjo.png)  
   *图1：3x3区域中的假割点（黄圈）*

2. **连通性判断优化**  
   *分析*：直接BFS 10^9网格不可行。**解决方案**：对关键点建图后BFS，障碍块八连通处理  
   💡 学习笔记：并查集动态维护连通块，复杂度O(αc)

3. **边界情况处理**  
   *分析*：一行/列网格性质不同。**解决方案**：特判n=1/m=1，此时答案≤1  
   💡 学习笔记：边界退化时，割点定义变化需单独处理

### ✨ 解题技巧总结
- **离散化建图**：将海量网格抽象为O(c)关键点
- **分层判断**：严格遵循(-1→0→1→2)判断链
- **防御性编程**：哈希冲突处理（双重哈希）、数组越界检查

## 4. C++核心代码实现赏析

**本题通用核心C++实现参考**
```cpp
#include <cstdio>
#include <cstring>
#include <queue>
using namespace std;
const int dx[4]={0,1,0,-1}, dy[4]={1,0,-1,0};
const int MAXC=100000, HASHSIZE=1000117;

struct Hash { /* 哈希表实现省略 */ };
Hash pointMap; // 坐标到ID的映射

void buildGraph(int n, int m, const vector<PII>& crickets) {
    for(auto [x,y] : crickets) {
        for(int i=-2; i<=2; ++i) for(int j=-2; j<=2; ++j) {
            int nx=x+i, ny=y+j;
            if(nx<1||nx>n||ny<1||ny>m) continue;
            if(!pointMap.exist(nx,ny)) 
                pointMap.insert(nx,ny, ++idx);
            // 添加四连通边
            for(int d=0; d<4; ++d) {
                int tx=nx+dx[d], ty=ny+dy[d];
                if(pointMap.exist(tx,ty))
                    addEdge(pointMap.get(nx,ny), pointMap.get(tx,ty));
            }
        }
    }
}

int main() {
    int T; scanf("%d",&T);
    while(T--) {
        int n,m,c; scanf("%d%d%d",&n,&m,&c);
        vector<PII> crickets(c);
        for(int i=0;i<c;++i) scanf("%d%d",&crickets[i].x,&crickets[i].y);

        // 步骤1：特判-1
        if(1LL*n*m - c < 2) { puts("-1"); continue; }
        if(1LL*n*m - c == 2) { /* 检查是否相邻 */ }

        // 步骤2：建图
        int idx=0;
        buildGraph(n, m, crickets);

        // 步骤3：BFS判连通
        if(!isConnected()) { puts("0"); continue; }

        // 步骤4：Tarjan求割点
        for(int i=1;i<=idx;++i) if(!dfn[i]) tarjan(i,0);
        bool hasCut=false;
        for(int i=1;i<=idx;++i) 
            if(isCritical[i] && isCut[i]) hasCut=true;
        
        puts(hasCut ? "1" : "2");
    }
}
```
* **代码解读概要**：哈希优化坐标映射，5x5区域建图，四连通边处理。Tarjan标准实现，isCritical标记蛐蛐邻域点。

# [六省联考 2017] 寿司餐厅

## 题目描述

Kiana 最近喜欢到一家非常美味的寿司餐厅用餐。

每天晚上，这家餐厅都会按顺序提供 $n$ 种寿司，第 $i$ 种寿司有一个代号 $a_i$ 和美味度 $d_{i, i}$，不同种类的寿司有可能使用相同的代号。每种寿司的份数都是无限的，Kiana 也可以无限次取寿司来吃，但每种寿司每次只能取一份，且每次取走的寿司必须是按餐厅提供寿司的顺序连续的一段，即 Kiana 可以一次取走第 $1, 2$ 种寿司各一份，也可以一次取走第 $2, 3$ 种寿司各一份，但不可以一次取走第 $1, 3$ 种寿司。

由于餐厅提供的寿司种类繁多，而不同种类的寿司之间相互会有影响：三文鱼寿司和鱿鱼寿司一起吃或许会很棒，但和水果寿司一起吃就可能会肚子痛。因此，Kiana 定义了一个综合美味度 $d_{i, j} \ (i < j)$，表示在一次取的寿司中，如果包含了餐厅提供的从第 $i$ 份到第 $j$ 份的所有寿司，吃掉这次取的所有寿司后将获得的额外美味度。由于取寿司需要花费一些时间，所以我们认为分两次取来的寿司之间相互不会影响。注意在吃一次取的寿司时，不止一个综合美味度会被累加，比如若 Kiana 一次取走了第 $1, 2, 3$ 种寿司各一份，除了 $d_{1, 3}$ 以外，$d_{1, 2}, d_{2, 3}$ 也会被累加进总美味度中。

神奇的是，Kiana 的美食评判标准是有记忆性的，无论是单种寿司的美味度，还是多种寿司组合起来的综合美味度，在计入 Kiana 的总美味度时都只会被累加一次。比如，若 Kiana 某一次取走了第 $1, 2$ 种寿司各一份，另一次取走了第 $2, 3$ 种寿司各一份，那么这两次取寿司的总美味度为 $d_{1, 1} + d_{2, 2} + d_{3, 3} + d_{1, 2} + d_{2, 3}$，其中 $d_{2, 2}$ 只会计算一次。

奇怪的是，这家寿司餐厅的收费标准很不同寻常。具体来说，如果 Kiana 一共吃过了 $c \ (c > 0)$ **种**代号为 $x$ 的寿司，则她需要为这些寿司付出 $mx^2 + cx$ 元钱，其中 $m$ 是餐厅给出的一个常数。

现在 Kiana 想知道，在这家餐厅吃寿司，自己能获得的总美味度（包括所有吃掉的单种寿司的美味度和所有被累加的综合美味度）减去花费的总钱数的最大值是多少。由于她不会算，所以希望由你告诉她。

## 说明/提示

### 样例解释 1

在这组样例中，餐厅一共提供了 $3$ 份寿司，它们的代号依次为 $a_1 = 2, a_2 = 3, a_3 = 2$，计算价格时的常数 $m = 1$。

在保证每次取寿司都能获得新的美味度的前提下，Kiana 一共有 $14$ 种不同的吃寿司方案。以下列出其中几种：
1. Kiana 一个寿司也不吃，这样她获得的总美味度和花费的总钱数都是 $0$，两者相减也是 $0$；
2. Kiana 只取 $1$ 次寿司，且只取第 $1$ 个寿司，即她取寿司的情况为 $\{[1, 1]\}$，这样获得的总美味度为 $5$，花费的总钱数为 $1 \times 2^2 + 1 \times 2 = 6$，两者相减为 $-1$；
3. Kiana 取 $2$ 次寿司，第一次取第 $1, 2$ 个寿司，第二次取第 $2, 3$ 个寿司，即她取寿司的情况为 $\{[1, 2], [2, 3]\}$，这样获得的总美味度为 $5 + (-10) + 15 + (-10) + 15 = 15$，花费的总钱数为 $(1 \times 2^2 + 2 \times 2) + (1 \times 3^2 + 1 \times 3) = 20$，两者相减为 $-5$；
4. Kiana 取 $2$ 次寿司，第一次取第 $1$ 个寿司，第二次取第 $3$ 个寿司，即她取寿司的情况为 $\{[1, 1], [3, 3]\}$，这样获得的总美味度为 $5 + 15 = 20$，花费的总钱数为 $1 \times 2^2 + 2 \times 2 = 8$，两者相减为 $12$。

在 $14$ 种方案中，惟一的最优方案是列出的最后一种方案，这时她获得的总美味度减去花费的总钱数的值最大为 $12$。

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：网络流（最小割/最大权闭合子图）


🗣️ **初步分析**：  
解决“寿司餐厅”问题的关键，在于将“选区间得美味度”“付费用”的逻辑转化为**最大权闭合子图**模型。简单来说，**最大权闭合子图**就像“必须买零件才能装电脑”——选一个“电脑”节点（大区间），就必须选它的“CPU”“内存”等零件节点（小区间）；而“买零件的钱”（费用）则是负权值，需要从总收益中扣除。  

在本题中：  
- **区间美味度**：每个区间`[i,j]`对应一个节点，权值为`d[i,j]`（美味度）减去该区间包含的寿司的费用（比如`d[i,i]`要减去`a[i]`，因为吃第`i`个寿司要付`a[i]`元）。  
- **依赖关系**：选大区间`[i,j]`必须选小区间`[i+1,j]`和`[i,j-1]`（就像装电脑必须选零件），因此大区间节点向小区间节点连**无限大**容量的边（确保不会被割，即选大区间必选小区间）。  
- **费用处理**：代号`x`的固定费用`mx²`对应一个节点，权值为`-mx²`（负权，因为是支出）；每个寿司`i`（`d[i,i]`）必须连向其代号`a[i]`的节点（吃寿司就必须付固定费用）。  

**核心算法流程**：  
1. 建图：源点`S`连正权节点（美味度>0的区间），汇点`T`连负权节点（美味度<0的区间、代号固定费用）。  
2. 跑最小割（Dinic算法）：最小割对应“放弃的正收益+必须支付的负收益”，总正收益减去最小割就是最大净收益。  

**可视化设计思路**：  
用**8位像素风格**展示图结构：  
- 源点`S`（红色方块）、汇点`T`（蓝色方块）；  
- 区间节点（绿色方块，`[i,j]`越大，位置越靠右）；  
- 代号节点（黄色方块，在屏幕下方）；  
- 边用箭头表示（正权边为绿色，负权边为红色，依赖边为灰色）。  
动画会展示**BFS分层**（节点按层变色）、**DFS增广**（路径闪烁）、**最小割结果**（割掉的边变虚线），帮助直观理解“选哪些区间最划算”。

## 3. 核心难点辨析与解题策略

### 1. 如何将区间依赖关系转化为边？  
**难点**：选大区间`[i,j]`必须选小区间`[i+1,j]`和`[i,j-1]`，否则美味度会重复计算。  
**策略**：大区间节点向小区间节点连**无限大**容量的边（如`DinicFlow::insw(Id[i][j], Id[i+1][j], Inf)`）。这样，在最小割中，如果大区间节点被选（不割与`S`的边），小区间节点也必须被选（不割与`S`的边），否则会存在`S→大区间→小区间→T`的路径，不符合割的定义。


### 2. 如何处理费用中的`mx²`和`cx`？  
**难点**：`cx`是每个寿司的费用（选`d[i,i]`就要付`a[i]`），`mx²`是代号的固定费用（选任何该代号的寿司都要付`mx²`）。  
**策略**：  
- `cx`：将`d[i,i]`的权值减去`a[i]`（如`d[i][j] -= a[i]`），这样选`d[i,i]`的收益就自动扣除了`cx`的费用。  
- `mx²`：为每个代号`x`建一个节点，向汇点`T`连`mx²`容量的边（如`DinicFlow::insw(++cnt, 2, i*i)`）。同时，`d[i,i]`节点向代号`x`节点连无限大边（如`DinicFlow::insw(Id[i][j], cnt + A[i], Inf)`），确保选`d[i,i]`必选代号节点（即必须付`mx²`）。


### 3. 如何正确建立网络流图的源汇点？  
**难点**：正权节点（美味度>0）和负权节点（美味度<0、费用）的边要连对，否则最小割计算错误。  
**策略**：  
- 源点`S`连正权节点：边容量为节点权值（如`if (cost > 0) DinicFlow::insw(1, Id[i][j], cost), Ans += cost`），表示“放弃这个节点会损失`cost`的收益”。  
- 负权节点连汇点`T`：边容量为节点权值的绝对值（如`if (cost < 0) DinicFlow::insw(Id[i][j], 2, -cost)`），表示“选这个节点要支付`-cost`的费用”。


### ✨ 解题技巧总结  
- **模型转化**：遇到“选A必选B”的最大化问题，优先考虑**最大权闭合子图**模型。  
- **费用拆分**：将复杂费用（如`mx²+cx`）拆分成“固定费用”（代号节点）和“可变费用”（区间节点权值调整），简化建图。  
- **无限大边**：依赖关系用无限大边表示，确保不会被割，从而强制满足依赖条件。


## 4. C++核心代码实现赏析

### 本题通用核心C++实现参考  
**说明**：综合小粉兔、绚濑绘里、tzc_wk的题解，提炼出的标准实现，包含Dinic模板和建图逻辑。  
**完整核心代码**：  
```cpp
#include <cstdio>
#include <cstring>
#include <algorithm>
#include <queue>
using namespace std;

typedef long long LL;
const LL Inf = 1e18;

namespace Dinic {
    const int MN = 6060, MM = 16055;
    int N, S, T;
    int h[MN], iter[MN], nxt[MM*2], to[MM*2], tot = 1;
    LL w[MM*2];
    void ins(int u, int v, LL x) { nxt[++tot] = h[u], to[tot] = v, w[tot] = x, h[u] = tot; }
    void insw(int u, int v, LL x) { ins(u, v, x), ins(v, u, 0); }
    int lv[MN], que[MN], l, r;
    bool bfs() {
        memset(lv, 0, sizeof(lv));
        lv[S] = 1, que[l=r=1] = S;
        while (l <= r) {
            int u = que[l++];
            for (int i = h[u]; i; i = nxt[i])
                if (w[i] && !lv[to[i]]) lv[to[i]] = lv[u]+1, que[++r] = to[i];
        }
        return lv[T];
    }
    LL dfs(int u, LL f) {
        if (u == T) return f;
        LL d = 0;
        for (int &i = iter[u]; i; i = nxt[i]) {
            if (w[i] && lv[to[i]] == lv[u]+1) {
                LL t = dfs(to[i], min(f-d, w[i]));
                w[i] -= t, w[i^1] += t, d += t;
                if (d == f) break;
            }
        }
        return d;
    }
    LL solve() {
        LL ans = 0;
        while (bfs()) memset(iter, 0, sizeof(iter)), ans += dfs(S, Inf);
        return ans;
    }
}

const int MN = 105;
int N, M, A[MN], MxA;
int Id[MN][MN], cnt;
LL Ans = 0;

int main() {
    scanf("%d%d", &N, &M);
    for (int i = 1; i <= N; ++i) scanf("%d", &A[i]), MxA = max(MxA, A[i]);
    Dinic::S = 1, Dinic::T = 2, cnt = 2;
    for (int i = 1; i <= N; ++i)
        for (int j = i; j <= N; ++j) Id[i][j] = ++cnt;
    for (int i = 1; i <= N; ++i) {
        for (int j = i; j <= N; ++j) {
            int d; scanf("%d", &d);
            if (i == j) d -= A[i]; // 处理cx费用
            if (d > 0) Dinic::insw(Dinic::S, Id[i][j], d), Ans += d;
            else if (d < 0) Dinic::insw(Id[i][j], Dinic::T, -d);
            if (i != j) { // 依赖关系：大区间连小区间
                Dinic::insw(Id[i][j], Id[i+1][j], Inf);
                Dinic::insw(Id[i][j], Id[i][j-1], Inf);
            }
        }
    }
    if (M) { // 处理mx²费用
        for (int i = 1; i <= MxA; ++i) Dinic::insw(++cnt, Dinic::T, (LL)i*i);
        for (int i = 1; i <= N; ++i) Dinic::insw(Id[i][i], cnt - MxA + A[i], Inf);
    }
    Dinic::N = cnt;
    printf("%lld\n", Ans - Dinic::solve());
    return 0;
}
```  
**代码解读概要**：  
1. **Dinic模板**：实现了BFS分层和DFS增广，用于计算最小割。  
2. **节点编号**：`Id[i][j]`表示区间`[i,j]`的节点编号，代号节点编号从`cnt+1`开始。  
3. **建边逻辑**：正权区间连`S`，负权区间连`T`，大区间连小区间（依赖），代号节点连`T`（固定费用）。  
4. **结果计算**：总正收益减去最小割，得到最大净收益。


### 针对各优质题解的片段赏析

#### 题解一（小粉兔）：依赖关系建边  
**亮点**：用`Id[i][j]`统一管理区间节点编号，依赖边建得简洁。  
**核心代码片段**：

for (int i = 1; i <= N; ++i) for (int j = i; j <= N; ++j) {
    if (i == j) {
        if (M) DinicFlow::insw(Id[i][j], cnt + A[i], Inf); // 寿司连代号

#### 题解二（绚濑绘里）：节点编号管理  
**亮点**：用`ict`变量统一计数，避免节点编号重复。  
**核心代码片段**：

**代码解读**：  
- `ict`从0开始，每创建一个节点就加1，确保区间节点和代号节点编号不重复。  
- `id[i][j]`表示区间`[i,j]`，`idw[a[i]]`表示代号`a[i]`，清晰易懂。  
**学习笔记**：节点编号是网络流建图的基础，统一管理能避免错误。


#### 题解三（tzc_wk）：费用处理  
**亮点**：将`cx`费用融入区间权值，简化建图。  
**核心代码片段**：  
```cpp
for (int i = 1; i <= n; ++i)
    for (int j = i; j <= n; ++j) {
        if (i == j) d[i][j] -= a[i]; // 处理cx费用
        if (d[i][j] > 0) addedge(0, id[i][j], d[i][j]), sum += d[i][j];
        else addedge(id[i][j], cnt+1, -d[i][j]);
    }
```  
**代码解读**：  
- 当`i==j`（单个寿司），将`d[i][j]`减去`a[i]`，这样选`d[i][j]`的收益就自动扣除了`cx`的费用。  
- 正权区间连`S`（0号节点），负权区间连`T`（cnt+1号节点），符合最大权闭合子图的建图规则。  
**学习笔记**：费用处理可以通过调整节点权值来简化，避免额外建边。

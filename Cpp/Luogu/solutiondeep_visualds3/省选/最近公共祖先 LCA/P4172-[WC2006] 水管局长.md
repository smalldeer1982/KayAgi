# 题目信息

# [WC2006] 水管局长

## 题目背景

SC 省 MY 市有着庞大的地下水管网络，嘟嘟是 MY 市的水管局长（就是管水管的啦）。

## 题目描述

每天供水公司可能要将一定量的水从 $u$ 处送往 $v$ 处，嘟嘟需要为供水公司找到一条从 $u$ 至 $v$ 的水管的路径，接着通过信息化的控制中心通知路径上的水管进入准备送水状态，等到路径上每一条水管都准备好了，供水公司就可以开始送水了。嘟嘟一次只能处理一项送水任务，等到当前的送水任务完成了，才能处理下一项。

在处理每项送水任务之前，路径上的水管都要进行一系列的准备操作，如清洗、消毒等等。嘟嘟在控制中心一声令下，这些水管的准备操作同时开始，但由于各条管道的长度、内径不同，进行准备操作需要的时间可能不同。

供水公司总是希望嘟嘟能找到这样一条送水路径，路径上的所有管道全都准备就绪所需要的时间尽量短。嘟嘟希望你能帮助他完成这样的一个选择路径的系统，以满足供水公司的要求。另外，由于 MY 市的水管年代久远，一些水管会不时出现故障导致不能使用，你的程序必须考虑到这一点。

不妨将 MY 市的水管网络看作一幅简单无向图（即没有自环或重边）：水管是图中的边，水管的连接处为图中的结点。整张图共有 $n$ 个节点和 $m$ 条边，节点从 $1$ 至 $n$ 编号。


## 说明/提示

#### 数据规模与约定

对于全部的测试点，保证：

- $1 \leq n \leq 10^3$，$1 \leq m, q \leq 10^5$。
- $1 \leq k \leq 2$，$1 \leq u, v \leq n$，$1 \leq t \leq 10^9$。
- 给出的图无重边无自环，保证在宣布一条水管报废之前，该水管一定存在于图上且没有报废，
- 宣布报废的水管不超过 $5 \times 10^3$ 条，且在任意时刻，图一定是联通的。

#### 1. 题目解读与核心算法识别
✨ **本题主要考察**：最小生成树维护与LCT（Link-Cut Tree）动态处理  
🗣️ **初步分析**：
> 题目要求动态维护水管网络，支持删边操作和查询两点间路径的最大边权最小值。核心思路是**离线逆序处理**：将删边操作转化为加边操作，用LCT维护最小生成树（MST）。  
> - **算法核心**：最小生成树保证任意两点间路径的最大边权最小（Kruskal贪心思想）。离线后，先构建最终状态的MST，逆序处理时：
>   - **查询操作**：直接查询路径最大边权
>   - **加边操作**：若新边权小于环上最大边权，则替换（维护MST最优性）
> - **可视化设计**：
>   - **像素动画**：8位风格网格图，节点为像素方块，边为连线
>   - **关键步骤高亮**：加边时红色高亮新边，黄色标记环上最大边
>   - **交互控制**：步进按钮调节速度，"叮"声提示加边，"胜利"音效完成查询

#### 3. 核心难点辨析与解题策略
1. **难点：边权转化为点权**  
   - **分析**：LCT仅支持点权，需创建虚拟节点（编号`n+i`）代表边，权值为边权
   - **解决**：`link(u, n+i); link(v, n+i)` 将边节点接入树结构

2. **难点：动态维护MST最优性**  
   - **分析**：加边形成环时，需快速定位环上最大边
   - **解决**：LCT维护子树最大值节点，`split(u,v)`后直接查询

3. **难点：高效边查询**  
   - **分析**：需快速定位边在数据集中的位置
   - **解决**：预排序边并用`map<pair<u,v>, id>`建立映射（二分查找O(log m)）

💡 **学习笔记**：
- LCT维护动态MST的核心是**边替换策略**
- 逆序处理规避删边复杂性
- 边权转点权是通用技巧

✨ **解题技巧总结**：
- **离线逆序**：删边问题先读入所有操作再反向处理
- **虚点设计**：为每条边创建节点处理边权
- **环上剪枝**：仅当新边权<环最大边权时才替换
- **路径压缩**：LCT的`split(u,v)`高效提取路径信息

#### 4. C++核心代码实现赏析
**本题通用核心实现**（综合题解优化）：
```cpp
#include <map>
using namespace std;
const int N = 1e5 + 10;

struct LCT {
    int ch[N][2], fa[N], val[N], mx[N], rev[N];
    // 维护最大值节点
    void pushup(int x) { 
        mx[x] = x;
        if(ch[x][0] && val[mx[ch[x][0]]] > val[mx[x]]) mx[x] = mx[ch[x][0]];
        if(ch[x][1] && val[mx[ch[x][1]]] > val[mx[x]]) mx[x] = mx[ch[x][1]];
    }
    // LCT基础操作（access, splay, link, cut等）
    // ...
} T;

struct Edge { int u, v, w; } e[N];
map<pair<int, int>, int> edgeMap; // 边映射: (u,v) -> id

void addEdge(int id) {
    int u = e[id].u, v = e[id].v;
    if(T.findroot(u) != T.findroot(v)) {
        T.link(u, id + n); T.link(v, id + n);
    } else {
        T.split(u, v);
        int maxEdge = T.mx[v]; // 获取路径最大边节点
        if(e[id].w < T.val[maxEdge]) {
            T.cut(e[maxEdge - n].u, maxEdge);
            T.cut(e[maxEdge - n].v, maxEdge);
            T.link(u, id + n); T.link(v, id + n);
        }
    }
}
```

**关键代码解读**：
1. **边映射建立**：将无向边`(u,v)`映射到唯一ID，保证O(1)查询
2. **LCT维护**：`mx[x]`存储子树最大权值节点编号
3. **加边策略**：不连通直接连；连通则比较替换
4. **边权转化**：边`e[i]`对应LCT节点`n+i`，权值`val[n+i]=e[i].w`

# [NOIP 2012 提高组] Vigenère 密码

## 题目描述

16 世纪法国外交家 Blaise de Vigenère 设计了一种多表密码加密算法 Vigenère 密码。Vigenère 密码的加密解密算法简单易用，且破译难度比较高，曾在美国南北战争中为南军所广泛使用。

 
在密码学中，我们称需要加密的信息为明文，用 $M$ 表示；称加密后的信息为密文，用 $C$ 表示；而密钥是一种参数，是将明文转换为密文或将密文转换为明文的算法中输入的数据，记为 $k$。在 Vigenère 密码中，密钥 $k$ 是一个字母串，$k=k_1,k_2,…,k_n$。当明文 $M=m_1,m_2,…,m_n$ 时，得到的密文 $C=c_1,c_2,…,c_n$，其中 $c_i$=$m_i \operatorname{\circledR} k_i$，运算 $\circledR$ 的规则如下表所示：

Vigenère 加密在操作时需要注意：

1. $\circledR$ 运算忽略参与运算的字母的大小写，并保持字母在明文 $M$ 中的大小写形式；
2. 当明文 $M$ 的长度大于密钥 $k$ 的长度时，将密钥 $k$ 重复使用。

例如，明文 $M=\texttt{Helloworld}$，密钥 $k=\texttt{abc}$ 时，密文 $C=\texttt{Hfnlpyosnd}$。

## 1. 题目解读与核心算法识别

✨ **本题主要考察**：`编程技巧应用`

🗣️ **初步分析**：
> 解决 Vigenère 密码这道题，关键在于理解**字母的循环移位**操作。想象一个古老的密码转盘，外圈是密文字母，内圈是密钥字母，通过旋转内盘找到对应关系（类似旋转式密码锁）。在本题中，这种技巧主要用于实现加密规则的逆向运算。
   
- 核心思路是将字母映射为0-25的数字，通过模26运算实现循环移位。难点在于正确处理大小写和密钥循环使用。优质题解主要采用两种方案：① 利用ASCII位运算统一处理大小写（高效但抽象） ② 显式分类大小写（直观易理解）。
- 可视化设计重点：用像素转盘展示字母移位过程，高亮当前处理的密文/密钥字母，动态显示移位计算过程（如"A→-3→X"）。采用复古密码机风格，转动时有"咔嗒"音效，成功解密时播放8-bit胜利音效。

1.  **难点：循环移位边界处理**

## 4. C++核心代码实现赏析

**本题通用核心C++实现参考**
* **说明**：综合优质题解思路，平衡效率与可读性
* **完整核心代码**：
```cpp
#include <iostream>
#include <cctype>
using namespace std;

int main() {
    string key, cipher;
    cin >> key >> cipher;
    
    for (int i = 0; i < cipher.size(); i++) {
        char k = key[i % key.size()];
        char c = cipher[i];
        
        // 统一为小写计算偏移
        int offset = (tolower(k) - 'a');
        // 计算明文字母序号(0-25)
        int plain_num = (tolower(c) - 'a' - offset + 26) % 26;
        
        // 保持原始大小写
        if (isupper(c)) cout << char('A' + plain_num);
        else cout << char('a' + plain_num);
    }
    return 0;
}
```
* **代码解读概要**：
  1. 读取密钥key和密文cipher
  2. 遍历密文，用`i%key.size()`循环使用密钥
  3. `tolower()`统一字母大小写计算偏移量
  4. `(x+26)%26`确保序号非负
  5. 根据原密文大小写输出对应明文

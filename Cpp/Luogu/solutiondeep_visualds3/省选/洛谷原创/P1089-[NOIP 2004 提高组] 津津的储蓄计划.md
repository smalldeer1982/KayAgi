# [NOIP 2004 提高组] 津津的储蓄计划

## 题目描述

津津的零花钱一直都是自己管理。每个月的月初妈妈给津津 $300$ 元钱，津津会预算这个月的花销，并且总能做到实际花销和预算的相同。

为了让津津学习如何储蓄，妈妈提出，津津可以随时把整百的钱存在她那里，到了年末她会加上 $20\%$ 还给津津。因此津津制定了一个储蓄计划：每个月的月初，在得到妈妈给的零花钱后，如果她预计到这个月的月末手中还会有多于 $100$ 元或恰好 $100$ 元，她就会把整百的钱存在妈妈那里，剩余的钱留在自己手中。


例如 $11$月初津津手中还有 $83$ 元，妈妈给了津津 $300$ 元。津津预计$11$月的花销是 $180$ 元，那么她就会在妈妈那里存 $200$ 元，自己留下 $183$ 元。到了 $11$ 月月末，津津手中会剩下 $3$ 元钱。


津津发现这个储蓄计划的主要风险是，存在妈妈那里的钱在年末之前不能取出。有可能在某个月的月初，津津手中的钱加上这个月妈妈给的钱，不够这个月的原定预算。如果出现这种情况，津津将不得不在这个月省吃俭用，压缩预算。


现在请你根据 $2004$ 年 $1$ 月到 $12$ 月每个月津津的预算，判断会不会出现这种情况。如果不会，计算到 $2004$ 年年末，妈妈将津津平常存的钱加上 $20\%$ 还给津津之后，津津手中会有多少钱。

在本题中，模拟的流程是：  
1. 每月月初，妈妈给300元 → 2. 减去本月预算 → 3. 如果钱不够，立刻输出月份并停止 → 4. 如果够，把**整百的钱**存给妈妈（剩下的留着）。

## 3. 核心难点辨析与解题策略

在解决这个问题时，我们常遇到以下3个关键点：


### 1. 每月操作的顺序：先加钱还是先花钱？  
* **分析**：题目说“每个月的月初，在得到妈妈给的零花钱后”再处理——所以顺序是：**先加300元 → 再减本月预算 → 再存钱**。如果顺序搞反（比如先减预算再加钱），就会算错！  
* 💡 **学习笔记**：模拟题的核心是“按题目顺序执行”，一定要圈出题目中的“时间顺序词”（比如“月初”“之后”）。


### 2. 存钱的逻辑：如何计算“整百的部分”？  
* **分析**：题目要求“整百的钱”存起来，比如剩下203元→存200元，剩下99元→不存。计算方式有两种：  
  - 用除法取整：`(剩余钱)/100 * 100`（比如203/100=2→2×100=200）；  
  - 用循环减100：`while(剩余钱>=100) {剩余钱-=100; 存的钱+=100;}`。  
* 💡 **学习笔记**：整数除法和取模是模拟题的“神器”——`a/b`取整，`a%b`取余数，组合起来能处理很多“整份”问题。


### 3. 边界条件：刚好够花或刚好100元怎么办？  
* **分析**：题目说“多于或恰好100元”要存钱——比如剩下100元，要存100元，剩下0元。如果代码里写`if(剩余钱>100)`就错了，应该是`>=100`！  
* 💡 **学习笔记**：模拟题一定要注意“等于”的情况，比如题目中的“≥”“≤”不要漏看。


### ✨ 解题技巧总结  
- **技巧A：画流程图**：把题目中的步骤画成流程图（比如“月初→加300→减预算→够？→存整百→下一个月”），代码照着流程图写，不会乱。  
- **技巧B：用直观的变量名**：比如`hold_money`（手中的钱）、`save_money`（存的钱），比`n`、`s`更容易理解，也不容易写错。  
- **技巧C：测试边界案例**：比如输入“100”（每个月花100元），看年末能存多少；或者输入“301”（第一个月就不够），看是否输出“-1”。


## 4. C++核心代码实现赏析

在深入分析具体题解之前，先看一个**通用核心实现**——综合了多个题解的优点，逻辑清晰，适合入门：


### 本题通用核心C++实现参考  
* **说明**：本代码综合了优质题解的思路，用直观的变量名和明确的步骤，覆盖所有边界情况。  
* **完整核心代码**：  
```cpp
#include <iostream>
using namespace std;

int main() {
    int hold_money = 0;  // 津津手中的钱
    int save_money = 0;  // 存给妈妈的钱
    bool is_safe = true; // 是否有月份钱不够

    for (int month = 1; month <= 12; month++) {
        int cost;
        cin >> cost;
        hold_money += 300;       // 月初加300元
        hold_money -= cost;      // 减去本月预算

        if (hold_money < 0) {    // 钱不够，输出月份
            cout << "-" << month << endl;
            is_safe = false;
            break;
        }

        // 存整百的钱
        int save = (hold_money / 100) * 100;
        save_money += save;
        hold_money -= save;
    }

    if (is_safe) {  // 没有不够的情况，计算年末总钱数
        cout << hold_money + save_money * 1.2 << endl;
    }

    return 0;
}
```
* **代码解读概要**：  
代码分三部分：  
1. 初始化变量：`hold_money`（手中的钱）、`save_money`（存的钱）、`is_safe`（是否安全）；  
2. 循环12个月：每个月加300元→减预算→检查是否够→存整百；  
3. 年末计算：如果安全，`save_money*1.2`是妈妈给的利息，加上手中的钱输出。


### 针对各优质题解的片段赏析

#### 题解一：（来源：wurang）  
* **亮点**：用`while`循环存钱，逻辑最贴近题目描述（“每次存100元，直到剩下的钱不足100”）。  
* **核心代码片段**：  
```cpp
while(n>=100) {
    n-=100;
    s+=100;
}
```
* **代码解读**：  
这段代码是“存钱”的核心——只要手中的钱`n`≥100，就每次减100（自己留的钱减少），存的钱`s`加100。比如`n=203`，第一次循环后`n=103`，`s=100`；第二次循环后`n=3`，`s=200`——刚好存200元，和题目例子一致！  
* 💡 **学习笔记**：`while`循环适合“重复做某件事直到条件不满足”的场景，比如“存整百的钱”。


#### 题解二：（来源：sea_bird）  
* **亮点**：用`flag`标记是否安全，逻辑更清晰，即使循环中途退出，也能正确处理输出。  
* **核心代码片段**：  
```cpp
if(money<0) {
    cout<<"-"<<i;
    flag=false;
    break;
}
```
* **代码解读**：  
当`money`（手中的钱）小于0时，输出“-月份”，把`flag`设为`false`（表示不安全），然后`break`跳出循环。这样即使后面还有月份，也不会继续执行——完全符合题目“出现这种情况就停止”的要求！  
* 💡 **学习笔记**：`flag`变量是“状态标记器”，适合处理“是否发生某件事”的情况，比如“是否钱不够”“是否找到答案”。


#### 题解三：（来源：yedalong）  
* **亮点**：用一行代码算出要存的钱，简化重复逻辑。  
* **核心代码片段**：  
```cpp
sum+=(300-s+ans)/100*100;
ans=(300-s+ans)%100;
```
* **代码解读**：  
`300-s+ans`是“加300元减预算后剩下的钱”（`ans`是上月剩下的钱，`s`是本月预算）。`(...)//100*100`算出整百的部分（要存的钱），加到`sum`（存的钱）里；`ans`取余100，就是剩下的零钱。比如`ans=83`，`s=180`，则`300-180+83=203`，`203/100*100=200`（存200），`ans=203%100=3`——和题目例子完全一致！  
* 💡 **学习笔记**：合并重复计算能让代码更简洁——比如“加300减预算”只算一次，不用重复写。

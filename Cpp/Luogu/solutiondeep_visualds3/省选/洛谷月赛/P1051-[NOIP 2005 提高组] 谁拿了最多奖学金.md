# 题目信息

# [NOIP 2005 提高组] 谁拿了最多奖学金

## 题目描述

某校的惯例是在每学期的期末考试之后发放奖学金。发放的奖学金共有五种，获取的条件各自不同：

1. 院士奖学金，每人 $8000$ 元，期末平均成绩高于 $80$ 分（$>80$），并且在本学期内发表 $1$ 篇或 $1$ 篇以上论文的学生均可获得；
2. 五四奖学金，每人 $4000$ 元，期末平均成绩高于 $85$ 分（$>85$），并且班级评议成绩高于 $80$ 分（$>80$）的学生均可获得；
3. 成绩优秀奖，每人 $2000$ 元，期末平均成绩高于 $90$ 分（$>90$）的学生均可获得；
4. 西部奖学金，每人 $1000$ 元，期末平均成绩高于 $85$ 分（$>85$）的西部省份学生均可获得；
5. 班级贡献奖，每人 $850$ 元，班级评议成绩高于 $80$ 分（$>80$）的学生干部均可获得；

只要符合条件就可以得奖，每项奖学金的获奖人数没有限制，每名学生也可以同时获得多项奖学金。例如姚林的期末平均成绩是 $87$ 分，班级评议成绩 $82$ 分，同时他还是一位学生干部，那么他可以同时获得五四奖学金和班级贡献奖，奖金总数是 $4850$ 元。

现在给出若干学生的相关数据，请计算哪些同学获得的奖金总数最高（假设总有同学能满足获得奖学金的条件）。

## 说明/提示

**【数据范围】**

对于 $100\%$ 的数据，满足 $1 \le  N \le  100$。

## 3. 核心难点辨析与解题策略

<difficulty_intro>
模拟题的“坑”往往藏在细节里。结合优质题解的共性，我帮你提炼了3个核心难点和解决策略：
</difficulty_intro>

1.  **关键点1：准确理解奖学金条件**  
    * **分析**：题目里的条件都是“>”（比如期末成绩>80分），不是“≥”！比如如果学生期末成绩是80分，即使论文数≥1，也不能拿院士奖学金。解决方法是：**把题目中的条件一字不差地转化为代码**（比如`a>80`而不是`a>=80`）。  
    * 💡 **学习笔记**：模拟题的关键是“**严格按题目描述写代码**”，不要自己加“想当然”的条件。

2.  **关键点2：处理并列第一的情况**  
    * **分析**：题目要求“如果有多个学生奖金最多，输出输入最早的那个”。解决方法有两种：① 打擂台时，只有当当前学生的奖金**严格大于**最大值时才更新（比如`if(money>mx)`）；② 排序时，把“输入顺序”作为第二关键字（比如`if(x.money==y.money) return x.id<y.id`）。  
    * 💡 **学习笔记**：并列情况的处理，本质是“**确定优先级**”——奖金第一，输入顺序第二。

3.  **关键点3：正确累加总奖金和维护最大值**  
    * **分析**：总奖金需要累加所有学生的奖金，最大值需要实时更新。解决方法是：① 每处理一个学生，就把他的奖金加到`total`变量里；② 用`mx`变量存储当前最高奖金，`max_name`存储对应姓名，每处理一个学生就比较更新。  
    * 💡 **学习笔记**：维护“最大值”的常用方法是“**打擂台**”——遍历所有元素，不断更新最大值。


### ✨ 解题技巧总结
- **技巧A：用结构体存储复杂信息**：当需要处理多个关联的数据（比如姓名、成绩、是否干部）时，用`struct`把它们“打包”，代码更清晰。
- **技巧B：逻辑表达式简化代码**：用`(条件)*奖金`的方式，可以把多个`if`判断浓缩成一行，提高代码效率。
- **技巧C：打擂台找最大值**：不需要排序，遍历一次就能找到最大值，时间复杂度更低。

## 4. C++核心代码实现赏析

<code_intro_overall>
我们先来看一个**通用核心实现**——结合了结构体存储、打擂台找最大值、严格条件判断的优点，适合大多数情况：
</code_intro_overall>

**本题通用核心C++实现参考**
* **说明**：本代码综合了优质题解的思路，结构清晰，兼顾可读性和效率，是解决本题的“标准模板”。
* **完整核心代码**：
    ```cpp
    #include <iostream>
    #include <string>
    using namespace std;

    // 用结构体存储学生信息，把相关数据“打包”
    struct Student {
        string name;       // 姓名
        int score;         // 期末平均成绩
        int class_score;   // 班级评议成绩
        char is_leader;    // 是否学生干部（'Y'/'N'）
        char is_west;      // 是否西部学生（'Y'/'N'）
        int papers;        // 论文数量
        int money;         // 总奖金
    };

    int main() {
        int n;
        cin >> n;
        Student stu[105];  // 存储最多100个学生
        int max_money = -1; // 初始最大值设为-1（肯定会被更新）
        string max_name;    // 奖金最多的学生姓名
        int total = 0;      // 总奖学金

        for (int i = 0; i < n; ++i) {
            // 1. 读入学生信息
            cin >> stu[i].name >> stu[i].score >> stu[i].class_score 
                >> stu[i].is_leader >> stu[i].is_west >> stu[i].papers;
            
            // 2. 计算当前学生的奖金（严格按题目条件）
            stu[i].money = 0;
            if (stu[i].score > 80 && stu[i].papers >= 1)     stu[i].money += 8000; // 院士奖学金
            if (stu[i].score > 85 && stu[i].class_score > 80)stu[i].money += 4000; // 五四奖学金
            if (stu[i].score > 90)                           stu[i].money += 2000; // 成绩优秀奖
            if (stu[i].score > 85 && stu[i].is_west == 'Y')  stu[i].money += 1000; // 西部奖学金
            if (stu[i].class_score > 80 && stu[i].is_leader == 'Y') stu[i].money += 850; // 班级贡献奖
            
            // 3. 更新最大值（只有严格大于时才更新，保证输入早的优先）
            if (stu[i].money > max_money) {
                max_money = stu[i].money;
                max_name = stu[i].name;
            }
            
            // 4. 累加总奖金
            total += stu[i].money;
        }

        // 输出结果
        cout << max_name << endl;
        cout << max_money << endl;
        cout << total << endl;

        return 0;
    }
    ```
* **代码解读概要**：  
  代码分为4个核心步骤：① 读入学生信息→② 按规则计算奖金→③ 维护最大值→④ 累加总奖金。其中，`struct Student`把学生的所有信息“打包”，避免了零散的变量；`max_money`和`max_name`用打擂台的方式更新，确保找到奖金最多的学生；`total`变量累加所有学生的奖金，最后输出结果。

<insights_intro>
优质题解中的“经验分享”能帮你避开很多坑，比如：
</insights_intro>

> **参考经验 (来自 mcturtle)**：“奖金可以叠加，要用多个 `if`，不能用 `if else`！”  
> **点评**：这是模拟题的“经典坑”——如果用`if else`，满足一个条件后就不会检查后面的条件，而题目中奖金是可以叠加的（比如一个学生可以同时拿五四和班级贡献奖）。用多个`if`才能确保所有满足的条件都被检查到！

> **参考经验 (来自 Haohao123coding)**：“变量名不要定义得自己都看不懂，如 `a`、`b`、`c`。”  
> **点评**：好的变量名能让代码“自己说话”，比如用`class_score`代替`b`，别人一看就知道是班级评议成绩。这能帮你减少调试时间，也方便别人阅读你的代码。

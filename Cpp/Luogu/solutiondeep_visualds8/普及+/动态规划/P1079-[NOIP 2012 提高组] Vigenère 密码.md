# [NOIP 2012 提高组] Vigenère 密码

## 题目描述

16 世纪法国外交家 Blaise de Vigenère 设计了一种多表密码加密算法 Vigenère 密码。Vigenère 密码的加密解密算法简单易用，且破译难度比较高，曾在美国南北战争中为南军所广泛使用。

 
在密码学中，我们称需要加密的信息为明文，用 $M$ 表示；称加密后的信息为密文，用 $C$ 表示；而密钥是一种参数，是将明文转换为密文或将密文转换为明文的算法中输入的数据，记为 $k$。在 Vigenère 密码中，密钥 $k$ 是一个字母串，$k=k_1,k_2,…,k_n$。当明文 $M=m_1,m_2,…,m_n$ 时，得到的密文 $C=c_1,c_2,…,c_n$，其中 $c_i$=$m_i \operatorname{\circledR} k_i$，运算 $\circledR$ 的规则如下表所示：

Vigenère 加密在操作时需要注意：

1. $\circledR$ 运算忽略参与运算的字母的大小写，并保持字母在明文 $M$ 中的大小写形式；
2. 当明文 $M$ 的长度大于密钥 $k$ 的长度时，将密钥 $k$ 重复使用。

例如，明文 $M=\texttt{Helloworld}$，密钥 $k=\texttt{abc}$ 时，密文 $C=\texttt{Hfnlpyosnd}$。

$$

\textsf{密文} & \tt H & \tt f & \tt n & \tt l & \tt p & \tt y & \tt o & \tt s & \tt n & \tt d \\  \hline
\end{array}
$$

## 1. 题目解读与策略初探

✨ **本题核心挑战**：  
> 给定密文 `C` 与密钥 `k`，**在忽略大小写的前提下**，**逆向**推导出明文 `M`。关键在于：  
> 1. 理解“Vigenère 表”本质是 **字母循环移位**；  
> 2. 处理好 **大小写保持不变**；  
> 3. 密钥长度不足时要 **循环使用**。

✨ **核心算法标签**：模拟、循环移位、模运算

## 3. 解题策略深度剖析

### 🎯 核心难点与关键步骤（最优解法：公式法）

| 步骤 | 要点 | 代码实现提示 |
|---|---|---|
| **1. 统一大小写** | 把密钥字母全部转成小写或大写 | `k[i] = tolower(k[i]);` |
| **2. 计算偏移量** | 字母 → 0~25 的数字 | `offset = k[i % klen] - 'a';` |
| **3. 逆向移位** | 密文 - 偏移，负数时 +26 | `plain = (cipher - base - offset + 26) % 26 + base;` |
| **4. 保持大小写** | 原密文是大写则结果大写，小写则小写 | `base = isupper(c) ? 'A' : 'a';` |

💡 **学习笔记**：  
> - **模运算技巧**：`(a - b + mod) % mod` 是处理“循环减”万能模板。  
> - **大小写保持**：用 `isupper/ islower` 判断，再决定加 `'A'` 还是 `'a'`。

### ⚔️ 策略竞技场：不同解法对比

| 策略 | 核心思想 | 时间复杂度 | 代码长度 | 适用场景 |
|---|---|---|---|---|
| **公式法** | 直接套用 `(C - k + 26) % 26` | O(n) | 最短 | 考场首选 |
| **查表法** | 预生成 26×26 逆表，再查表 | O(n) | 中等 | 喜欢“打表”的同学 |
| **队列法** | queue 模拟字母循环，生成逆表 | O(26²+n) | 较长 | 训练 STL 与队列思维 |
| **暴力循环** | 对每个字符循环减 `offset` 次 | O(n·26) | 最长 | 不建议，但可拿部分分 |

## 4. C++核心代码实现赏析

### 本题通用核心C++实现参考

```cpp
#include <iostream>
#include <cctype>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    string k, c;
    cin >> k >> c;
    for (size_t i = 0; i < c.size(); ++i) {
        char ci = c[i];
        char ki = tolower(k[i % k.size()]);
        int offset = ki - 'a';
        int base = isupper(ci) ? 'A' : 'a';
        ci = (ci - base - offset + 26) % 26 + base;
        cout << ci;
    }
    return 0;
}
```

**代码解读概要**：  
> 1. 用 `tolower` 把密钥统一成小写，避免重复判断。  
> 2. `base` 保存原大小写基准。  
> 3. 一行公式完成“循环减”并恢复大小写，简洁高效。

### 各优质题解片段赏析

#### ① ingu 的位运算技巧
```cpp
int t = (k[i % k.length()] & 31) - 1;
c[i] = (c[i] & 31) - t > 0 ? c[i] - t : c[i] - t + 26;
```
> - `&31` 直接把 `'A'/'a'` 都映射到 1~26，省掉 `tolower` 分支。  
> - 一行完成解密，**极简主义**典范。

#### ② LightCoder 的队列打表
```cpp
for (int i = 1; i <= 26; ++i) q.push(i);
for (int i = 1; i <= 26; ++i) {
    Mid = q.front(); q.pop(); q.push(Mid);
    for (int j = 26; j; --j) Map[i][j] = q.front(), q.pop(), q.push(Map[i][j]);
}
```
> - 用队列模拟“循环右移”，生成逆向映射表 `Map[i][j]`。  
> - 查表 O(1)，**空间换时间**思想。

# 题目信息

# 足球

## 题目描述

我们当中有很多热爱中国足球的同学，我们都知道中超（中国足球超级联赛）的规则：

一场比赛中，若获胜（即你的得分严格大于对手得分）则获得 $3$ 的积分，若打平（即你的得分等于对手得分）则获得 $1$ 分，若失败（即你的得分严格小于对手得分）获得 $0$ 积分。

这个问题很简单，假设 $N$ 轮比赛中你一共攻入 $S$ 个球，丢掉 $T$ 个球，那么你可能获得的最大得分和最小得分是多少？

## 样例 #1

### 输入

```
1 1 1
1 1 2```

### 输出

```
1 1
3 2
```

## 1. 题目解读与策略初探

✨ **本题核心挑战**：  
> 在 N 轮比赛中，给定总进球 S、总丢球 T，如何“分配”这些进球/丢球到每一轮，才能  
> 1. 让积分尽可能高（贪心：尽量多赢）；  
> 2. 让积分尽可能低（贪心：尽量少赢/多输）。  
> 每轮比赛的积分规则：赢 3 分、平 1 分、输 0 分。

✨ **核心算法标签**：`贪心`、`分类讨论`、`模拟`

---

### 🔍 算法侦探：如何在题目中发现线索？

| 线索 | 发现过程 |
| --- | --- |
| **线索1：目标** | “最大得分”“最小得分” → 需要**极端化**地安排比赛结果。 |
| **线索2：约束** | 总进球 S、总丢球 T 是**全局资源**，必须**恰好分配**到 N 轮。 |
| **线索3：数据规模** | N ≤ 1e9，S,T ≤ 1e9，只能**O(1)** 数学推导，不能枚举每轮。 |

---

### 🧠 思维链构建：从线索到策略

1. 首先，**线索1**告诉我们需要求“最大值/最小值” → 自然想到**贪心**。
2. **线索2**要求我们把 S 个进球、T 个丢球“分配”到 N 场比赛 → 需要**分类讨论**如何分配。
3. **线索3**排除了暴力枚举 → 必须**数学推导**出每类情况下的积分公式。

## 3. 解题策略深度剖析

### 🎯 核心难点与关键步骤（贪心推导）

| 难点 | 分析 | 💡 学习笔记 |
| --- | --- | --- |
| **最大得分：如何“赢”得最多？** | 1. 若 `s < n`：最多赢 `s` 场（1:0），剩余 `n-s-1` 场平局（0:0），最后1场集中丢球（0:T）。<br>2. 若 `s ≥ n`：前 `n-1` 场1:0赢，最后1场用剩余进球 `s-(n-1)` 与丢球 T 比较：<br>- 若 `s-(n-1) > t` → 再赢1场 → 总分 `3n`。<br>- 若 `== t` → 平局 → 总分 `3(n-1)+1`。<br>- 若 `< t` → 输掉 → 总分 `3(n-1)`。 | 贪心原则：优先“1:0”赢，剩余丢球全部集中到1场输掉，避免平局浪费进球。 |
| **最小得分：如何“输”得最多？** | 1. 若 `s > t`：至少赢1场（s:0），剩余 `t` 场0:1输，剩下 `n-1-t` 场0:0平 → 总分 `3 + max(0, n-1-t)`。<br>2. 若 `s ≤ t`：可能全输，条件是 `t-s ≥ n-1` → 总分 `0`；否则 `n-(t-s)` 场平局 → 总分 `max(0, n-(t-s))`。<br>需与“赢1场+其余输”取 `min`：即 `min(3 + max(0, n-t-1), max(0, n-t+s))`。 | 贪心原则：优先“0:1”输，若无法全输，则用平局“填补”剩余场次。 |

---

### ⚔️ 策略竞技场：不同思路对比

| 策略 | 核心思想 | 优点 | 缺点 | 适用 |
| --- | --- | --- | --- | --- |
| **数学推导**（推荐） | 直接按 `s/t` 与 `n` 的关系分类讨论 | 简洁、O(1) | 需要严谨推导边界 | 100% |
| **暴力枚举** | 枚举每轮胜负平，检查是否满足总进球/丢球 | 思路直观 | O(3^N) 爆炸 | N≤10 对拍 |
| **贪心模拟** | 用“堆”模拟分配（如艮鳖肉思路） | 易于理解 | 本质仍是数学推导 | 100% |

---

### ✨ 优化之旅：从“能做”到“做好”

1. **起点**：暴力枚举每轮结果 → 指数级爆炸。
2. **瓶颈**：每轮独立决策导致状态爆炸。
3. **钥匙**：发现“进球/丢球”是**全局资源**，只需**统计**每类结果（赢/平/输）的**场次**。
4. **升华**：用 `s/t/n` 直接推导场次 → 数学公式一步到位。

## 4. C++核心代码实现赏析

### 本题通用核心实现（综合艮鳖肉 & quantum11）

```cpp
#include <cstdio>
using namespace std;
long long s, t, n;

int main() {
    while (scanf("%lld %lld %lld", &s, &t, &n) == 3) {
        long long mx = 0, mn = 0;

        // 最大得分
        if (s < n) {
            mx = 3 * s + (n - s - 1);
            if (t == 0) mx++; // 没有丢球，最后一场平局
        } else {
            mx = 3 * (n - 1);
            long long last = s - (n - 1);
            if (last > t) mx += 3;      // 全胜
            else if (last == t) mx += 1; // 最后一场平
            // 否则最后一场输，mx不变
        }

        // 最小得分
        if (s > t) {
            mn = 3 + (n - 1 - t > 0 ? n - 1 - t : 0);
        } else {
            long long a = 3 + (n - 1 - t > 0 ? n - 1 - t : 0); // 赢1场方案
            long long b = n - (t - s) > 0 ? n - (t - s) : 0;   // 不赢方案
            mn = a < b ? a : b;
        }

        printf("%lld %lld\n", mx, mn);
    }
    return 0;
}
```

---

### 代码解读概要

- **最大得分**：先保证“赢”的场次最多（1:0），剩余丢球集中到1场输掉（0:T），若无法输掉则平局。
- **最小得分**：先尝试“全输”或“尽量输”，若进球过多必须赢1场，则比较“赢1场+其余输”与“全平”的积分取小。

   - 赢球：胜利音效“↑↑”

# CNEASY - CN Tower (Easy)

## 题目描述

### 题面描述

Christy C. Coder 前往滑铁卢参加编程比赛。途中，她在多伦多停留观光。旅行的不幸之处在于，家乡的每个人都希望她把所有东西的照片都带回来。克里斯蒂讨厌拍照：这让她看起来像个游客！幸运的是，Christy 有一个让她拍照不再痛苦的计划。

CN 塔高 $553$ 米，是世界上最高的独立式建筑。塔上 $351$ 米处是 "360" 旋转餐厅，每 $72$ 分钟旋转 $360$ 度。在这里，克里斯蒂可以俯瞰整个城市，还可以用她新买的 $100$ 倍光学变焦相机拍摄所有地标性建筑的特写照片。由于餐厅本身也会旋转，她只需站在一个地方就能拍到各个方向的照片。

服务员坚持要她点菜，否则就走人，而克里斯蒂对菜单上的任何菜品都不感兴趣。因此，她必须在被赶走之前尽快行动。给定克里斯蒂想要拍照的地标性建筑的位置，你的任务是确定她必须在餐厅内停留的最短时间，以使餐厅旋转到足以让所有地标性建筑都出现在视野中。假设克里斯蒂始终将相机垂直对准窗户，以尽量减少玻璃造成的失真。请注意，多个地标可能占据相同的（角度）位置，而这些地标只需要拍摄一张照片就可以捕捉到。

由于只有当她开始拍照时，餐厅工作人员才会意识到她是一名游客，因此我们从她拍下第一张照片开始计算所需时间。因此，克里斯蒂可以不受餐厅工作人员的干扰，移动到餐厅的任何位置，然后开始拍照。

## 输入格式

第一行输入包括测试用例的数量。对于每个测试用例，第一行是一个整数 n（2n 行指定一个地标）。每个地标说明包括地标名称（长度不超过 40 个字符的大小写字母字符串）、空格和从 CN 塔到地标的罗盘角度 $d$（单位：度）（$0$ = 北，$90$ = 东，$180$ = 南，$270$ = 西）。请注意，$d$ 是一个实数，满足 $0 < d < 360$，$d$ 单位为百分之一度（即最多两位小数）。

## 输出格式

对于每个测试用例，输出一个整数，即 Christy 必须在餐厅中停留的最小秒数。如果时间不是整数秒，则将其四舍五入到最接近的秒（即，取数字的上限）。

## 样例 #1

### 输入

```
1
5
CasaLoma 231.0
OntarioParliament 123.0
SkyDome 75.0
RoyalYorkHotel 340.0
PearsonAirport 165.0
```

### 输出

```
3012
```

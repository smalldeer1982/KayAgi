# CONDUIT - I Conduit

## 题目描述

Irv Kenneth Diggit为一家公司工作，该公司挖掘战壕，但总是会损坏一些地下管道或电缆。
IRV的工作是确保计划开挖的地方下面没有地下管道或电缆。
他有几张不同的地图，每一家公用事业公司一张，显示它们的管道所在的位置，他需要绘制一张将它们结合在一起的大地图。
一种方法是简单地将每一张较小的地图一次一张地绘制到大地图上。
然而，这往往会浪费时间，更不用说办公室里钢笔绘图员的墨水了，因为在许多情况下，管道的部分相互重叠(尽管在地下的不同深度)。
IRV想要的是一种方法来确定来自不同地图的所有线段的情况下要绘制的最小线段数量。

## 输入格式

输入将由多个小组组成。
每组将从包含正整数n的单行开始，该正整数表示来自所有较小地图的线段的总数。
接下来的n行中的每一行都将包含格式中的一个线段的描述。
坐标值指定为最多两个小数位的且范围为0--1001的浮点值。
线段的最大数量为10000，并且所有线段的长度都不为零。
在最后一个小组之后，将有一行会包含0，表示输入结束；它不被计入处理。

## 输出格式

对于每个小组，在单行上输出需要在较大的合并地图上绘制的最小线段数量。

## 样例 #1

### 输入

```
3
1.0 10.0 3.0 14.0
0.0 0.0 20.0 20.0
10.0 28.0 2.0 12.0
2
0.0 0.0 1.0 1.0
1.0 1.0 2.15 2.15
2
0.0 0.0 1.0 1.0
1.0 1.0 2.15 2.16
0
```

### 输出

```
2
1
2
```

# MSE07E - Showstopper

## 题目描述

一个知名公司最近在其硬件视频处理解决方案中发现了一个小问题，现在他们正在开发一个软件补丁。为了发挥最佳性能，芯片通常成对使用，内存中的所有数据对象都应该被偶数次引用。但在某些情况下，这一规则被打破，导致恰好有一个数据对象被引用了奇数次。由于产品即将发布，这是他们需要解决的唯一问题。他们需要你的帮助，快速高效地解决这个关键问题。你能帮他们吗？

## 输入格式

输入文件由多个数据集组成，每个数据集之间由一个或多个空行分隔。每个数据集是一组以压缩格式存储的 32 位正整数（即引用）。

输入数据集中每一行包含三个用单个空格分隔的 32 位正整数 X、Y、Z，它们表示出以下的引用序列：X, X+Z, X+2\*Z, X+3\*Z, …, X+K\*Z, …（其中满足 (X+K\*Z) ≤ Y）。

## 输出格式

你的任务是分析输入数据，对于每个数据集，判断数据是否被破坏，进而找出哪个引用出现了奇数次，并统计该引用的次数。

对于每个输入数据集，你需要输出一行文本，内容是“no corruption”（表示数据没有被破坏）或者是两个用单个空格分开的整数（第一个是出现奇数次的引用，第二个是该引用的出现次数）。

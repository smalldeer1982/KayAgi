# FILRTEST - File Recover Testing

## 题目描述

在最近的编程比赛中，有一道名为“文件恢复”的题目。那道题目要求统计给定文本中重复出现的字符串数量。为了测试边缘情况，你准备了一些测试用例，需要生成一个有大量某个特定字符串重复出现的文本。由于测试用例的长度不能过长，你决定先确定一个总长度及一个字符串，然后在这个总长度内尽量多地放入该字符串的重复。如长度为14，字符串为“abcab”，则可以生成文本“abcabcabcabcab”，该文本长度为14，其中字符串“abcab”一共出现了4次（分别从位置1、4、7和10开始）。在动手实现之前，你想评估一下这个方法的效果。给定一个长度和一个字符串，你需要计算这个字符串最多能在该长度的文本中连续出现的次数。

## 输入格式

每个测试用例由一行构成。这一行包括一个整数 $K$（$1 \le K \le 10^9$）和一个由最多 $10^6$ 个小写字母组成的非空字符串 $S$。输入以一行“−1 *”结束。

## 输出格式

对于每个测试用例，输出一行，包含一个整数，表示字符串 $S$ 在长度为 $K$ 的文本中最多能连续出现的次数。

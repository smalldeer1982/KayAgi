# AHORCADO - B Beware, the end of the world

## 题目描述

世界末日在历史上一直是一个有争议的话题，例如999年和1999年。现在有人认为玛雅人预测了2012年的变化，特别是在2012年12月21日，还有人猜测EoW将伴随着某颗小行星撞击地球而来，或者行星排列与之有关。

在ACM（灾难缓解协会），他们100%确信僵尸启示录的结局可能会到来，因此他们正在准备僵尸探测机制，这是某种雷达。目前该机制正在测试中，因为在识别僵尸时出现了错误。有时它会给出假阳性指标。他们要求你，一个出色的程序员，制作一个程序，告诉他们演讲是否正确，基本上你会得到两个僵尸位置的二维快照，让我们称之为前一个和现在的快照，僵尸将用“*”表示，而空白将用“”表示，您出色的程序必须在前一个快照的基础上判断当前快照是否有效。如果出现以下情况，则当前快照视为有效：

1） 它的僵尸数量不少于前一张快照。

2） 上一个快照中的每个僵尸可能只在八个方向中的每一个方向上移动了一个正方形，或者它与以前在同一个正方形中。

## 输入格式

第一行将具有快照的尺寸，形式为R C。

2 R，C

然后是R行C字符，描述前一个快照，然后是另一组R行和C字符，每行都属于当前快照。这两个快照由一行字符“N”分隔。

最后一个测试用例后面跟着0 0。

在之前的快照中最多会有9个僵尸。

## 输出格式

对于每个测试用例，如果两个快照彼此对应，则打印VALID，否则打印NOT VALID。

## 样例 #1

### 输入

```
5 4\n....\n....\n.*..\n....\n....\nN\n....\n....\n..*.\n....\n....\n6 6\n......\n.*..*.\n......\n......\n......\n......\nN\n......\n....*.\n......\n....*.\n......\n..*...\n0 0\n\n
```

### 输出

```
\n\nVALID\nNOT VALID\n\n
```

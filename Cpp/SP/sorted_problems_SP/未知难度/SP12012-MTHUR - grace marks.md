# MTHUR - grace marks

## 题目描述

在 BITS，评分系统曾经是按绝对值计算，而不是现在常见的相对评分制。有一次，Mathur 教授组织了一场突击测验，结果许多学生措手不及，他们的成绩比往年差了很多。

Mathur 教授认为真正的能力应该在困难的条件下体现，因此他坚持进行这样严苛的测试。然而，最终由于学生们的就业情况不佳，教授意识到问题的严重性，决定给所有学生的成绩进行调整。

作为一名数学家，Mathur 教授提出了一种简单的加分策略。他为每位学生加上一个固定值来平衡差距。他选择那些能使调整后的成绩与去年的成绩绝对差之和最小的加分值。公式为：$\sum \left| a[i] - (b[i] + \text{grace}) \right|$，并且最小化该值。同时，Mathur 教授倾向于从他偏好的数字集中选择加分值。

最终，多亏了 Mathur 教授的这一策略，学生们顺利进入了“Microsoft”和“Adobe”等公司。现在，请你通过给定的输入数据，找出 Mathur 教授具体选择了哪个加分值。（如果有多个满足条件的加分值，请选择其中最小的一个）。

## 输入格式

第一行是一个整数 $t$，表示测试用例的数量。接下来的 $5 \times t$ 行中，每 5 行对应一个测试用例。

- 对于每个测试用例：
  - 第一行是一个整数 $n$，表示学生人数。
  - 第二行是 $n$ 个整数，表示每位学生去年的成绩。
  - 第三行是 $n$ 个整数，表示每位学生今年的成绩。
  - 第四行是一个整数 $m$，表示提供的加分值候选数目。
  - 第五行是 $m$ 个整数，表示可能的加分值选项。

## 输出格式

输出 Mathur 教授选择的加分值。

## 说明/提示

- 测试用例数量满足：$1 \leq t \leq 10$
- 学生人数满足：$1 \leq n \leq 1000$
- 成绩范围为：$0 \leq a[i], b[i] \leq 1000$
- 加分值数量满足：$1 \leq m \leq 1000$
- 加分值范围为：$0 \leq \text{grace} \leq 1000$

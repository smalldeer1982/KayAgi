# GLGRID - G-Line Grid

## 题目描述

21 世纪引入了多核处理器，随之而来的便是关于并行计算的广泛研究。随着时间的推移，处理器的数量将不断增加。目前，IIIT 的 Biloo 教授要求一名学生在多台 G-line 处理器上实现下面这段代码：

```
for(i=1; i<=x; i++){
    for(j=1; j<=x; j++){
        for(k=1; k<=a; k++){
            z = z % y;
        }
    }
    for(j=1; j<=b; j++){
        z = z / y;
    }
}
for(i=1; i<=c; i++){
    z = z % y;
}
```

经过实验，学生发现代码中唯一耗时显著的操作是取模（%）和除法（/），且这两种操作所需时间几乎相等。其他操作耗时可以忽略不计。他找到了一种算法，比如可以随机顺序执行这些操作。为测试该算法，他选用了 M 个处理器，并保证每个处理器恰好执行 M 次操作（% 或 /）。当处理器完全覆盖所有操作时，算法性能最佳。

学生感到困惑的是，只有在某些特定条件下，给定 a、b 和 c 的情况下，才能找到使该算法有效的 x 的值。他希望能蒙混过关来解出一些符合条件的 x，因此需要这些 x 尽可能小，以便让教授觉得他是通过手动计算的。

现在，给定参数 a、b、c 和 k，输出前 k 个能使算法有效的 x 值。

**注意：** x 的值必须大于或等于 0。

## 输入格式

第一行输入一个整数 t，表示测试用例的数量。接下来的每个测试用例由一行组成，包含用空格分隔的四个整数 a、b、c 和 k。

## 输出格式

对每个测试用例，输出能够使学生的算法有效的前 k 个 x 的值，每个值占一行。

## 说明/提示

- $1 \leq t \leq 10^5$
- $1 \leq a, b, c \leq 10^9$
- $1 \leq k \leq 100$

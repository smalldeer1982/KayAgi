# POLQUERY - Police Query

## 题目描述

为了更有效地抓捕逃犯，警方正在引入一套全新的计算机系统。该系统负责管理的区域内有 **N** 个城市和 **E** 条双向道路相连。城市的编号从 1 到 **N**。警方需要监控并阻止那些企图在城市间潜逃的罪犯。因此，他们希望通过查看地图来决定在哪里设置路障。

新系统需要回答以下两种类型的查询：

1. 给出两个城市 **A** 和 **B** 以及一条连接城市 **G $ _{1} $** 和 **G $ _{2} $** 的道路。如果封锁这条道路，罪犯能否从城市 **A** 到达城市 **B**？

2. 给出三个城市 **A**、**B** 和 **C**。如果切断整个城市 **C**，罪犯能否从城市 **A** 到达城市 **B**？

请为此编写程序来实现这些查询的功能。

## 输入格式

第一行输入两个整数 **N** 和 **E**，分别表示城市和道路的数量（2 ≤ **N** ≤ 300,000，1 ≤ **E** ≤ 300,000）。接下来 **E** 行中，每行包含两个不同的整数，这些整数的范围是 1 到 **N**，表示这条路连通的两个城市。任意一对城市之间最多有一条道路。

然后输入一个整数 **Q**（1 ≤ **Q** ≤ 300,000），表示有 **Q** 个查询。接下来的 **Q** 行中，每行有四或五个整数。第一个整数是查询类型，分为 **1** 或 **2**。

- 如果是类型 **1**，接下来的一行包括 4 个整数 **A**、**B**、**G $ _{1} $** 和 **G $ _{2} $**，分别代表要查询的城市和对应的道路。保证 **A** 和 **B** 是不同的城市，**G $ _{1} $** 和 **G $ _{2} $** 是一条存在的道路。

- 如果是类型 **2**，该行提供 3 个整数 **A**、**B** 和 **C**，它们会是不同的城市。

测试数据保证了最初可以从每个城市到达其他所有城市。

## 输出格式

对于每个查询，输出 "da"（是）或 "ne"（否），每个答案占一行。

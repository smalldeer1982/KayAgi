# EQ - Electronic queue

## 题目描述

火车站刚刚采用了一种新的电子排队系统。现在，乘客若想购买车票，必须先领取服务顺序号并等待叫号。

在这个车站，有 $N$ 个售票窗口；每个窗口一次只能服务一位乘客。当轮到你时，你会被指定到某个窗口购买和支付车票。如果你需要购买 $K$ 张票，你需要花费 5 分钟来选择列车、时间、座位等信息，随后再花 $K$ 分钟来打印票。如果有多个窗口可用，乘客会被分配到编号最小的窗口。

根据乘客到达车站的时间和他们想要购买的票数，计算所有乘客买票所需的总时间，包括等待和购买的时间。

## 输入格式

第一行是整数 $C$，表示测试用例的数量。

对于每个测试用例：
- 第一行是整数 $N$，表示售票窗口的数量。
- 第二行是整数 $P$，表示乘客的数量。
- 接下来的 $P$ 行中，每行有两个整数，分别表示每位乘客到达车站的时间和他们想购买的票数。乘客的到达时间各不相同，并按升序排列。

## 输出格式

对于每个测试用例，输出所有乘客购买车票所花费的总时间。

## 说明/提示

- $1 \le C \le 15$
- $1 \le N \le 50$
- $1 \le P \le 10000$
- 所有输入中的数字都是正数且小于 1000。

## 样例输入

```
1
2
3
1 1
2 10
3 2
```

## 样例输出

```
32
```
